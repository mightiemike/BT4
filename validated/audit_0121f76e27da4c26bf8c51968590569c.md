Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is `msg.sender` at the pool level — the direct caller of `pool.swap(...)`. When `MetricOmmSimpleRouter` is used, the pool's `msg.sender` is the router, not the end-user. If the pool admin allowlists the router to enable router-mediated swaps, every user on the internet can bypass the swap allowlist by routing through the router, completely nullifying the pool's access-control invariant.

## Finding Description

In `MetricOmmPool.swap`, the pool calls `_beforeSwap(msg.sender, ...)`:

```solidity
// metric-core/contracts/MetricOmmPool.sol line 230-240
_beforeSwap(
    msg.sender,   // ← this is whoever called pool.swap()
    recipient,
    ...
);
```

`_beforeSwap` in `ExtensionCalling.sol` (lines 149–177) forwards this `sender` argument verbatim to the extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the extension is called by the pool), and `sender` is the router's address when the user routes through `MetricOmmSimpleRouter`.

In `MetricOmmSimpleRouter.exactInputSingle` (lines 72–80) and `exactInput` (lines 104–112), the router calls `IMetricOmmPoolActions(pool).swap(...)` directly. At that point, `msg.sender` inside the pool is the **router's address**, so the extension receives `sender = router_address` and checks `allowedSwapper[pool][router_address]`.

For any router-mediated swap to work at all, the pool admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, `allowedSwapper[pool][router_address] == true` for every call through the router — regardless of who the actual end-user is. An address that is explicitly **not** on the allowlist can call `router.exactInputSingle(pool, ...)` and the extension passes because it sees the router, not the user.

The `DepositAllowlistExtension` does not share this flaw because it gates the explicit `owner` parameter (the position-owner address), not `sender`. The swap path has no equivalent explicit "real swapper" parameter.

## Impact Explanation

Direct policy bypass on curated pools. The swap allowlist exists to restrict which addresses may trade on a given pool (e.g., KYC-gated, institutional-only, or whitelist-only pools). Once the router is allowlisted — which is required for normal UX — any unprivileged user can trade on the pool by routing through `MetricOmmSimpleRouter`. The pool admin's curation is completely nullified. This can result in unauthorized users draining LP value through adversarial swaps on a pool designed for a closed set of counterparties, regulatory or compliance failure if the pool is gated for legal reasons, and loss of LP principal if the pool's pricing or liquidity assumptions depend on a controlled set of swappers. Severity: **High** — direct loss of access-control invariant with fund-impacting consequences for LPs on curated pools.

## Likelihood Explanation

High. The attack requires no special privilege. `MetricOmmSimpleRouter` is a standard public periphery contract. Any pool admin who wants users to be able to swap via the router must allowlist the router address — this is the expected operational pattern. Once the router is allowlisted, any user can call `router.exactInputSingle(...)` or `router.exactInput(...)` targeting the curated pool. No admin action, no special token, no flash loan needed. The only scenario where the bypass does not apply is if the pool admin intentionally blocks the router and forces all swappers to call the pool directly — but this breaks normal UX and is not the documented design intent.

## Recommendation

The `SwapAllowlistExtension` must gate the **end-user**, not the intermediary. Two sound approaches:

1. **Trusted forwarder pattern in `extensionData`:** Allowlist the router as a trusted forwarder. When `sender` is a known trusted forwarder, the extension reads the real user address from `extensionData` and checks that address instead. This requires the router to populate `extensionData` with the real `msg.sender` at the router level.

2. **Explicit `swapper` parameter at the pool level:** The pool's `swap` function could accept an explicit `swapper` parameter (separate from `msg.sender`) that the router populates with `msg.sender` at the router level. The extension then checks that explicit address. This requires a pool-level interface change.

3. **Short-term mitigation:** Do not allowlist the router; require direct pool calls for allowlisted pools. Document this constraint clearly. This is the safest short-term mitigation but breaks composability.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook.
  - Pool admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
  - Pool admin does NOT call setAllowedToSwap(pool, alice, true)  // alice is NOT allowlisted

Attack:
  - alice calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls IMetricOmmPoolActions(pool).swap(recipient=alice, ...)
  - Pool calls _beforeSwap(msg.sender=router_address, ...)
  - Extension checks: allowedSwapper[pool][router_address] == true  → PASSES
  - Alice's swap executes on the curated pool despite not being allowlisted

Expected:
  - Extension should check allowedSwapper[pool][alice] == false → REVERT NotAllowedToSwap

Contrast (direct call):
  - alice calls pool.swap(...) directly
  - Pool calls _beforeSwap(msg.sender=alice, ...)
  - Extension checks: allowedSwapper[pool][alice] == false → REVERTS correctly

Foundry test outline:
  1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  2. setAllowedToSwap(pool, address(router), true).
  3. Confirm alice (not allowlisted) calling pool.swap() directly reverts NotAllowedToSwap.
  4. Confirm alice calling router.exactInputSingle({pool: pool, ...}) succeeds — demonstrating bypass.
```
