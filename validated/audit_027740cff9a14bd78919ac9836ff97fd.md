### Title
Pending Builder Fees Not Settled Before `updateBuilder` Overwrites Owner, Redirecting Accrued Fees to New Owner - (File: `core/contracts/OffchainExchange.sol`)

---

### Summary

`updateBuilder` in `OffchainExchange.sol` overwrites the entire `Builder` struct — including the `owner` field — without first settling the `collectedBuilderFee` accumulated for that builder. Because `claimBuilderFee` gates payout on the **current** `builders[builderId].owner`, any fees earned by the old owner are silently redirected to the new owner after the update.

---

### Finding Description

Builder fees are accumulated per-builder in `collectedBuilderFee[quoteId][builderId]` during every matched order: [1](#0-0) 

When a builder's record is updated, `updateBuilder` unconditionally replaces the full `Builder` struct, including the `owner`: [2](#0-1) 

There is no call to `claimBuilderFee` or any other settlement step before this overwrite. When the new owner subsequently calls `claimBuilderFee`, the authorization check passes because it reads the **current** (already-updated) owner: [3](#0-2) 

The new owner then receives the full `collectedBuilderFee` balance, including all fees that accrued while the old owner held the position: [4](#0-3) 

The `collectedBuilderFee` mapping is keyed by `(quoteId, builderId)` — not by owner address — so there is no per-owner accounting. All historical accruals for a `builderId` are claimable by whoever is the owner at claim time. [5](#0-4) 

---

### Impact Explanation

The old builder owner permanently loses all fees that accrued before the ownership update. The new owner receives fees they did not earn. The asset delta is the full `collectedBuilderFee[quoteId][builderId]` balance at the time of the `updateBuilder` call — a direct, irreversible transfer of quote-token credit from the old owner's rightful claim to the new owner's subaccount.

**Impact: Medium** — direct loss of earned fees for the displaced builder owner; magnitude scales with trading volume and time between fee accrual and ownership change.

---

### Likelihood Explanation

Builder ownership updates are a normal protocol operation exposed via `ContractOwner.updateBuilders`: [6](#0-5) 

Any time a builder owner is rotated (key rotation, ownership transfer, builder reassignment) while `collectedBuilderFee` is non-zero, the misallocation occurs. Because builder fees accumulate continuously with every matched order and ownership changes are a routine admin operation, the conditions for this bug to manifest are regularly present.

**Likelihood: Medium**

---

### Recommendation

Before overwriting `builders[txn.builderId]`, settle all pending fees to the current owner. Concretely, inside `updateBuilder`, iterate over all spot product IDs and transfer any non-zero `collectedBuilderFee[productId][builderId]` to the current `builders[txn.builderId].owner` subaccount before applying the new `Builder` struct. This mirrors the pattern that `claimBuilderFee` already implements and ensures fees are always attributed to the owner who earned them.

---

### Proof of Concept

1. Builder #5 is registered with `owner = Alice`. Over time, matched orders accumulate `collectedBuilderFee[USDC_ID][5] = 1000e18`.
2. Admin calls `ContractOwner.updateBuilders([5], [Bob], ...)`, which enqueues and executes `updateBuilder`. `builders[5].owner` is now `Bob`. No fees are settled.
3. Bob calls `claimBuilderFee` (via the endpoint) with `sender = Bob's subaccount`, `builderId = 5`. The check at line 875 passes because `builders[5].owner == Bob`. Bob receives 1000e18 USDC.
4. Alice calls `claimBuilderFee` — the check at line 875 reverts because `builders[5].owner != Alice`. Alice receives nothing.

Alice's 1000e18 USDC in earned fees is permanently redirected to Bob. [2](#0-1) [7](#0-6)

    function claimBuilderFee(bytes32 sender, uint32 builderId)
        external
        onlyEndpoint
    {
        require(!RiskHelper.isIsolatedSubaccount(sender), ERR_UNAUTHORIZED);
        require(
            builders[builderId].owner == address(uint160(bytes20(sender))),
            ERR_UNAUTHORIZED
        );
        uint32[] memory productIds = spotEngine.getProductIds();
        for (uint32 i = 0; i < productIds.length; i++) {
            uint32 productId = productIds[i];
            int128 collectedFee = collectedBuilderFee[productId][builderId];
            if (collectedFee == 0) {
                continue;
            }
            emit ClaimBuilderFee(builderId, productId, sender, collectedFee);
            spotEngine.updateBalance(productId, sender, collectedFee);
            collectedBuilderFee[productId][builderId] = 0;
        }
    }
