### Title
Path-Dependent Transaction Hash via Lossy `ValidResourceBounds` Protobuf Round-Trip - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses `AllResources { l2_gas: 0, l1_data_gas: 0 }` into `L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of resource felts for each variant, the same on-chain transaction produces two distinct hash preimages depending on whether it is processed through the RPC/gateway path or the P2P block-sync path.

### Finding Description

**Serialization boundary — `ValidResourceBounds` protobuf round-trip**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes both `L1Gas(X)` and `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` to the identical wire bytes:

```
{ l1_gas: Some(X), l2_gas: Some(0), l1_data_gas: Some(0) }
``` [1](#0-0) 

The reverse converter then applies a heuristic to decide which Rust variant to reconstruct:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

This is a lossy round-trip: `AllResources { l2_gas: 0, l1_data_gas: 0 }` is permanently rewritten to `L1Gas` after one protobuf pass.

**Hash domain — `get_tip_resource_bounds_hash` is variant-sensitive**

The fee-fields hash that enters every V3 transaction hash includes a different number of resource felts depending on the variant:

```rust
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 felts
    ValidResourceBounds::AllResources(all) => {
        vec![get_concat_resource(&all.l1_data_gas, L1_DATA_GAS)?]    // 3 felts
    }
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
``` [3](#0-2) 

Concretely, for `tip = T`, `l1_gas = X`, `l2_gas = 0`, `l1_data_gas = 0`:

| Path | Variant | Hash preimage |
|---|---|---|
| RPC / gateway | `AllResources` | `poseidon(T, pack(L1_GAS,X), pack(L2_GAS,0), pack(L1_DATA_GAS,0))` |
| P2P block sync | `L1Gas` | `poseidon(T, pack(L1_GAS,X), pack(L2_GAS,0))` |

These are cryptographically distinct values.

**Attacker-reachable trigger**

The gateway stateless validator explicitly accepts transactions with `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }` as valid: [4](#0-3) 

Any unprivileged user can submit such a transaction. Once included in a block and propagated over P2P, every syncing node deserializes it with the wrong variant.

**End-to-end call path**

1. User submits `RpcInvokeTransactionV3 { resource_bounds: AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }, … }`.
2. Gateway calls `convert_rpc_tx_to_internal` → `tx_without_hash.calculate_transaction_hash` → `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash(AllResources, …)` → hash **H₁** (3-felt preimage). [5](#0-4) 
3. Transaction is included in a block. Block is propagated over P2P as `starknet_api::transaction::InvokeTransactionV3 { resource_bounds: ValidResourceBounds, … }`.
4. Receiving node deserializes via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` → `L1Gas(X)`.
5. Any subsequent call to `get_invoke_transaction_v3_hash` on the deserialized struct → `get_tip_resource_bounds_hash(L1Gas, …)` → hash **H₂** (2-felt preimage). **H₁ ≠ H₂**. [6](#0-5) 

### Impact Explanation

Any component on a syncing node that recomputes the transaction hash from stored fields — including `validate_transaction_hash`, the Starknet OS hint processor, or proof-verification logic — will derive **H₂** while the canonical chain records **H₁**. This causes:

- **Wrong hash bound to the executable payload**: the blockifier/OS receives a transaction whose recomputed hash does not match the signed hash, breaking signature and proof verification for that transaction.
- **Authoritative-looking wrong value from RPC**: `starknet_getTransactionByHash` on a syncing node may return a transaction whose fields, if re-hashed by a client, produce a different hash than the one in the receipt.

This matches the allowed impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"* (High).

### Likelihood Explanation

The trigger is a standard V3 invoke transaction with only `l1_gas` set (l2_gas and l1_data_gas both zero). This is a valid, common transaction shape explicitly tested in the gateway. No special privilege is required. The bug fires on every P2P block-sync of such a transaction.

### Recommendation

Preserve the `AllResources` vs `L1Gas` distinction across the protobuf boundary. Options:

1. **Add a discriminant field** to `protobuf::ResourceBounds` (e.g., `bool is_all_resources`) so the deserializer can reconstruct the correct variant without relying on zero-value heuristics.
2. **Always use `AllResources`** for V3 transactions in the P2P path and reject `L1Gas` for any transaction whose version is ≥ 3.
3. **Remove the heuristic collapse** in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` and instead return an error when `l1_data_gas` is absent (once the 0.13.2 compatibility window is closed, as the existing `TODO` already notes). [7](#0-6) 

### Proof of Concept

```
1. Craft: RpcInvokeTransactionV3 {
       resource_bounds: AllResourceBounds {
           l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
           l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
       },
       tip: 0, …
   }

2. Gateway accepts it (valid per stateless validator).
   Hash H₁ = poseidon(0,
                       pack(L1_GAS, 1000, 1),
                       pack(L2_GAS, 0, 0),
                       pack(L1_DATA_GAS, 0, 0))   ← 3 resource felts

3. Block is produced and synced over P2P.
   Protobuf wire: { l1_gas: (1000,1), l2_gas: (0,0), l1_data_gas: (0,0) }

4. Receiving node deserializes:
   l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → ValidResourceBounds::L1Gas({ max_amount: 1000, max_price_per_unit: 1 })

5. Recomputed hash H₂ = poseidon(0,
                                  pack(L1_GAS, 1000, 1),
                                  pack(L2_GAS, 0, 0))   ← only 2 resource felts

6. H₁ ≠ H₂  →  hash mismatch between originating sequencer and every syncing node.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
