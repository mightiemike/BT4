Audit Report

## Title
`uint32` overflow in `_afterTimelock` allows pool admin to bypass all parameter timelocks instantly — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`_afterTimelock` computes `uint32(block.timestamp + oracleStopLossConfig[pool_].timelock)`. When `timelock` is set to `type(uint32).max`, the addition in `uint256` space overflows the `uint32` cast and truncates to a value already in the past, causing `_requireElapsed` to pass immediately. No upper-bound validation exists on the timelock value. A pool admin can exploit this to bypass the timelock on every subsequent parameter change, defeating the LP-protection guarantee the extension is designed to provide.

## Finding Description

`_afterTimelock` at line 297–299 computes the execution deadline:

```solidity
function _afterTimelock(address pool_) private view returns (uint32) {
    return uint32(block.timestamp + oracleStopLossConfig[pool_].timelock);
}
```

`oracleStopLossConfig[pool_].timelock` is `uint32`. The addition is performed in `uint256` space (implicit promotion of `uint32` to match `block.timestamp`), then explicitly cast to `uint32`. In Solidity 0.8+, explicit casts truncate silently — they do not revert.

With `block.timestamp ≈ 1,753,000,000` and `timelock = type(uint32).max = 4,294,967,295`:

```
1,753,000,000 + 4,294,967,295 = 6,047,967,295
uint32(6,047,967,295) = 6,047,967,295 mod 4,294,967,296 ≈ 1,752,999,999
```

The result is approximately `block.timestamp − 1` — already in the past. `_requireElapsed` at line 301–303 then trivially passes:

```solidity
function _requireElapsed(uint32 executeAfter) private view {
    if (block.timestamp < executeAfter) revert OracleStopLossTimelockNotElapsed(...);
}
```

`proposeOracleStopLossTimelock` at line 78–84 has no validation on `newTimelock`:

```solidity
function proposeOracleStopLossTimelock(address pool_, uint32 newTimelock) external onlyPoolAdmin(pool_) {
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    uint32 executeAfter = _afterTimelock(pool_);
    sched.pendingTimelock = newTimelock;
    sched.pendingTimelockExecuteAfter = executeAfter;
    emit OracleStopLossTimelockProposed(pool_, newTimelock, executeAfter);
}
```

`initialize` at line 57–58 calls `_validateDrawdown` and `_validateDecay` but no analogous `_validateTimelock`. `_validateDrawdown` and `_validateDecay` at lines 305–311 enforce upper bounds, but no such function exists for the timelock.

**Attack path:**

1. Pool is initialized with `timelock = 0` (as in test setUp at line 49: `_initPool(address(mockPool), 0, 0, 0)`).
2. Admin calls `proposeOracleStopLossTimelock(pool, type(uint32).max)`. Since current `timelock = 0`, `executeAfter = uint32(block.timestamp)` — immediately executable (`block.timestamp >= block.timestamp`).
3. Admin calls `executeOracleStopLossTimelock(pool)`. `oracleStopLossConfig[pool].timelock` is now `type(uint32).max`.
4. Admin calls `proposeOracleStopLossDrawdown(pool, 0)`. `_afterTimelock` computes `uint32(block.timestamp + type(uint32).max)` → wraps to a past value.
5. Admin immediately calls `executeOracleStopLossDrawdown(pool)`. `_requireElapsed` passes. `drawdownE6 = 0`.
6. The stop-loss check at line 217 short-circuits (`if (drawdown == 0) return;`), permanently disabling LP value protection with no delay.

The same bypass applies to `proposeOracleStopLossDecay` and `proposeOracleStopLossHighWatermarks`, all of which call `_afterTimelock` internally.

## Impact Explanation

The NatDoc at lines 13–28 explicitly states: *"Drawdown and decay changes are timelocked so LPs can react."* The timelock is the sole mechanism protecting LPs from sudden stop-loss parameter changes by the pool admin. By bypassing it, the pool admin can immediately set `drawdownE6 = 0` (disabling stop-loss entirely), set watermarks to arbitrarily high values (bricking the pool via `OracleStopLossTriggered` on every swap), or set `decayPerSecondE8 = E8` (collapsing all watermarks to zero within one second). This constitutes an admin-boundary break (pool admin bypasses timelocks) and broken core pool functionality causing loss of funds (LP principal exposed to unbounded value loss).

## Likelihood Explanation

Requires the pool admin to act maliciously or be compromised. The pool admin is explicitly semi-trusted, constrained by the timelock as the enforced boundary. The initial `timelock` can be `0` (as shown in tests), making step 2 immediately executable with no waiting. The overflow arithmetic is deterministic and requires no special conditions beyond setting `timelock = type(uint32).max`. The bypass is repeatable on any pool where the admin can first set the timelock to `type(uint32).max`.

## Recommendation

Add a `_validateTimelock` function with a reasonable upper bound (e.g., 30 days) and call it in both `initialize` and `proposeOracleStopLossTimelock`:

```solidity
uint32 private constant MAX_TIMELOCK = 30 days;

function _validateTimelock(uint32 timelock) private pure {
    if (timelock > MAX_TIMELOCK) revert OracleStopLossTimelockTooLarge(timelock);
}
```

Call it in `initialize` alongside `_validateDrawdown`/`_validateDecay` (line 57–58), and in `proposeOracleStopLossTimelock` before storing the pending value (line 81). This eliminates the overflow path and caps the maximum delay to a value that cannot wrap a `uint32` cast for any realistic `block.timestamp`.

## Proof of Concept

```solidity
// Foundry test (extend OracleValueStopLossSubExtensionTest)
function test_timelockOverflowBypassesAllTimelocks() public {
    // setUp already called _initPool(address(mockPool), 0, 0, 0)

    vm.startPrank(admin);

    // Step 2-3: Set timelock to type(uint32).max — no validation, passes
    extension.proposeOracleStopLossTimelock(address(mockPool), type(uint32).max);
    extension.executeOracleStopLossTimelock(address(mockPool));

    // Step 4-5: Propose and immediately execute drawdown=0 — executeAfter wraps to past
    extension.proposeOracleStopLossDrawdown(address(mockPool), 0);
    extension.executeOracleStopLossDrawdown(address(mockPool)); // no vm.warp needed

    vm.stopPrank();

    // Step 6: Stop-loss is now permanently disabled
    (uint32 drawdown,,,) = extension.oracleStopLossConfig(address(mockPool));
    assertEq(drawdown, 0);
}
