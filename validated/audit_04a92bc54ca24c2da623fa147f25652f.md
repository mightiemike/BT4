### Title
Protobuf `ValidResourceBounds` Deserialization Silently Converts `AllResources` to `L1Gas`, Producing a Divergent Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf converter for `ValidResourceBounds` silently downgrades an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) to `L1Gas`. Because `get_tip_resource_bounds_hash` hashes a **different number of elements** for each variant, the transaction hash computed at gateway admission diverges from the hash recomputed by any node that receives the transaction over P2P. A valid, gateway-accepted transaction therefore carries a hash that cannot be reproduced from its own serialized bytes after a protobuf round-trip.

---

### Finding Description

**Step 1 – Hash domain split in `get_tip_resource_bounds_hash`**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` feeds a different number of field elements into the Poseidon absorb depending on the `ValidResourceBounds` variant:

```
L1Gas(X)          → poseidon(tip, L1_GAS_packed(X), L2_GAS_packed(0))          // 3 elements
AllResources{X,0,0} → poseidon(tip, L1_GAS_packed(X), L2_GAS_packed(0), L1_DATA_GAS_packed(0)) // 4 elements
```

Even when `l2_gas = 0` and `l1_data_gas = 0`, the two variants produce **different hashes** because the Poseidon state absorbs a different number of field elements. [1](#0-0) 

**Step 2 – Protobuf converter silently collapses the variant**

The protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter applies the following logic:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
} else {
    ValidResourceBounds::AllResources(...)
})
```

Any `AllResources` value whose `l2_gas` and `l1_data_gas` fields are zero is silently converted to `L1Gas`. The original variant is not preserved. [2](#0-1) 

**Step 3 – Gateway always produces `AllResources`**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (never `L1Gas`). The gateway converts it to `InternalRpcInvokeTransactionV3`, which also stores `AllResourceBounds`. The hash H₁ is computed with `ValidResourceBounds::AllResources` semantics (4-element Poseidon). [3](#0-2) 

**Step 4 – P2P receiver recomputes a different hash**

When the transaction is propagated to a peer via protobuf and the peer reconstructs the `InvokeTransactionV3` (or calls `calculate_transaction_hash` / `InvokeTransaction::create`), the deserialized `ValidResourceBounds` is now `L1Gas(X)`. The recomputed hash H₂ uses the 3-element Poseidon path. H₁ ≠ H₂. [4](#0-3) 

**Step 5 – Gateway validator does not block the trigger input**

The stateless validator only requires that at least one resource bound is non-zero. A transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` (X > 0) passes all gateway checks and is admitted with hash H₁. [5](#0-4) 

---

### Impact Explanation

Any node that receives the transaction over P2P and recomputes the hash (e.g., during executable-transaction construction, signature verification, or block-hash commitment) will derive H₂ ≠ H₁. This breaks the canonicalization invariant that a transaction's hash must be reproducible from its serialized bytes. Concretely:

- A peer node that calls `InvokeTransaction::create` or `calculate_transaction_hash` on the deserialized data will store H₂ in its mempool/state, diverging from the originating node's H₁.
- If the block is committed with H₁ but a verifying node recomputes H₂, the block's transaction list hash is wrong from that node's perspective, which can cause consensus disagreement or incorrect receipt/event attribution.
- The user's signature is over H₁; any component that re-derives the hash from the deserialized `L1Gas` form and checks it against the signature will reject a legitimately signed transaction.

This matches: **High – Transaction conversion or signature/hash logic binds the wrong hash or executable payload.**

---

### Likelihood Explanation

The trigger is an unprivileged user action: submit an `InvokeV3` transaction with `AllResourceBounds { l1_gas: X > 0, l2_gas: 0, l1_data_gas: 0 }`. This is a valid, gateway-accepted input. The protobuf round-trip happens automatically on every P2P propagation. No special privileges or malformed bytes are required.

---

### Recommendation

Preserve the `AllResources` variant unconditionally in the protobuf converter. Remove the downgrade heuristic:

```rust
// Before (broken):
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})

// After (canonical):
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

The `L1Gas` variant should only be produced when deserializing a pre-0.13.3 on-disk or wire format that was originally serialized as `L1Gas`. If backward compatibility with old protobuf peers is required, encode the variant explicitly in the protobuf schema (e.g., a `bounds_type` tag) rather than inferring it from zero values.

---

### Proof of Concept

```
1. Construct RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway computes H₁ via get_tip_resource_bounds_hash(AllResources{...}):
     poseidon(tip=0, L1_GAS_packed(1000,1), L2_GAS_packed(0,0), L1_DATA_GAS_packed(0,0))
     → 4-element absorb → H₁

3. User signs the transaction with H₁. Gateway accepts it.

4. Transaction is serialized to protobuf and sent to a peer node.

5. Peer deserializes:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas(ResourceBounds { max_amount: 1000, max_price_per_unit: 1 })

6. Peer computes H₂ via get_tip_resource_bounds_hash(L1Gas{...}):
     poseidon(tip=0, L1_GAS_packed(1000,1), L2_GAS_packed(0,0))
     → 3-element absorb → H₂

7. H₁ ≠ H₂.
   - Peer's executable transaction carries H₂; block commitment uses H₁.
   - Peer's signature check against H₂ fails for a legitimately signed transaction.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
