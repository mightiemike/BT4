### Title
`Batcher::new` Accepts `BatcherConfig` Without Enforcing the `input_stream_content_buffer_size >= n_concurrent_txs` Invariant That `update_dynamic_config` Enforces - (`File: crates/apollo_batcher/src/batcher.rs`)

---

### Summary

`Batcher::update_dynamic_config` validates the combined `BatcherConfig` before applying it, rejecting any config where `input_stream_content_buffer_size < n_concurrent_txs`. The constructor `Batcher::new` (and its production entry point `create_batcher`) stores the initial `BatcherConfig` directly without calling `config.validate()`, so the same invariant is never checked at startup. A node operator (or deployment config) that sets `n_concurrent_txs > input_stream_content_buffer_size` in the initial config file starts the batcher in a permanently invalid state that the dynamic-config update path would have rejected.

---

### Finding Description

`BatcherConfig` derives the `validator::Validate` trait and registers a cross-field schema validator:

```rust
#[derive(Clone, Debug, Default, Serialize, Deserialize, Validate, PartialEq)]
#[validate(schema(function = "validate_batcher_config"))]
pub struct BatcherConfig { ... }
```

The schema validator enforces:

```rust
fn validate_batcher_config(batcher_config: &BatcherConfig) -> Result<(), ValidationError> {
    if batcher_config.static_config.input_stream_content_buffer_size
        < batcher_config.dynamic_config.n_concurrent_txs
    {
        return Err(ValidationError::new(
            "input_stream_content_buffer_size must be at least n_concurrent_txs",
        ));
    }
    Ok(())
}
``` [1](#0-0) 

`update_dynamic_config` correctly calls `candidate.validate()` and rejects invalid configs:

```rust
pub(crate) fn update_dynamic_config(&mut self, dynamic_config: BatcherDynamicConfig) {
    let candidate =
        BatcherConfig { static_config: self.config.static_config.clone(), dynamic_config };
    if let Err(errors) = candidate.validate() {
        warn!("Dynamic config update rejected due to validation errors: ...");
        return;
    }
    self.config = candidate;
}
``` [2](#0-1) 

`Batcher::new`, however, stores the config directly with no call to `config.validate()`:

```rust
pub(crate) fn new(config: BatcherConfig, ...) -> Self {
    assert!(config.static_config.validation_only == mempool_client.is_none(), ...);
    Self { config, ... }
}
``` [3](#0-2) 

The production entry point `create_batcher` calls `Batcher::new` directly without validating the config first: [4](#0-3) 

The invalid config then propagates into `validate_block`, which creates the input transaction channel using the undersized `input_stream_content_buffer_size`:

```rust
let (input_tx_sender, input_tx_receiver) =
    tokio::sync::mpsc::channel(self.config.static_config.input_stream_content_buffer_size);
``` [5](#0-4) 

While `n_concurrent_txs` is used as the batch size for concurrent transaction execution: [6](#0-5) 

---

### Impact Explanation

When `input_stream_content_buffer_size < n_concurrent_txs`, the `validate_block` channel is created with a capacity smaller than the number of transactions the block builder requests per batch. The consensus layer sends transactions into this channel while the `ValidateTransactionProvider` drains them in batches of `n_concurrent_txs`. Because the channel buffer cannot hold a full batch, the sender blocks waiting for space while the receiver waits for a full batch — a deadlock. This causes block validation to stall and time out, meaning the batcher rejects valid proposed blocks, breaking consensus liveness and potentially causing the node to fall out of the validator set.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing**, and **Critical — Wrong state/receipt/revert result from blockifier/syscall/execution logic for accepted input**.

---

### Likelihood Explanation

The `BatcherConfig` is loaded from a JSON config file or CLI overrides at node startup. The default values (`input_stream_content_buffer_size = 400`, `n_concurrent_txs = 100`) satisfy the invariant, but deployment configs override both independently. The production deployment config sets `input_stream_content_buffer_size = 4000` and does not pin `n_concurrent_txs`, leaving the relationship entirely to operator discretion with no startup guard. Any operator who sets `n_concurrent_txs` above `input_stream_content_buffer_size` — a plausible tuning mistake — starts a silently broken batcher. [7](#0-6) 

---

### Recommendation

Add `config.validate()` (or an equivalent call) inside `Batcher::new` before storing the config, mirroring the guard already present in `update_dynamic_config`. Alternatively, call `config.validate()` inside `create_batcher` before passing the config to `Batcher::new`, and propagate the error to the caller so the node fails fast at startup rather than silently operating with an invalid configuration.

---

### Proof of Concept

The existing unit test `validate_batcher_config_failure` already demonstrates that `BatcherConfig::validate()` rejects the invalid combination:

```rust
#[test]
fn validate_batcher_config_failure() {
    let config = BatcherConfig {
        static_config: BatcherStaticConfig {
            input_stream_content_buffer_size: 99,
            ..Default::default()
        },
        dynamic_config: BatcherDynamicConfig { n_concurrent_txs: 100, ..Default::default() },
    };
    let error = config.validate().unwrap_err();
    assert!(error.to_string()
        .contains("input_stream_content_buffer_size must be at least n_concurrent_txs"));
}
``` [8](#0-7) 

The same config passed directly to `Batcher::new` (as done in `create_batcher`) is accepted without error, placing the batcher in the invalid state that `update_dynamic_config` would have rejected. A test reproducing the bug:

```rust
// This should panic/error but currently succeeds silently:
let config = BatcherConfig {
    static_config: BatcherStaticConfig {
        input_stream_content_buffer_size: 99,
        ..Default::default()
    },
    dynamic_config: BatcherDynamicConfig { n_concurrent_txs: 100, ..Default::default() },
};
// No validation called — batcher starts with invalid config
let batcher = create_batcher(config, ...).await;
// validate_block now creates a channel of size 99 while requesting batches of 100 — deadlock
```
