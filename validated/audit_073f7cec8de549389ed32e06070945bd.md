### Title
Deprecated `verify_transaction_inclusion` Accepts Internal Merkle Node as Valid Transaction — (`contract/src/lib.rs`, `merkle-tools/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` (v1) remains a publicly callable NEAR contract method with no access control. `compute_root_from_merkle_proof` applies `double_sha256` uniformly at every tree level without distinguishing leaf nodes from internal nodes. An attacker can supply an internal node hash as `tx_id` and a shorter proof path, causing the function to return `true` for a transaction that does not exist.

---

### Finding Description

`compute_root_from_merkle_proof` iterates over the proof elements and at each step computes:

```
current_hash = double_sha256(current_hash || proof_hash)   // or reversed
``` [1](#0-0) 

There is no domain separator, depth tag, or any other mechanism that distinguishes a leaf-level hash from an internal-node hash. `compute_hash` simply concatenates two 32-byte values and double-SHA256s them: [2](#0-1) 

`verify_transaction_inclusion` (v1) passes the caller-supplied `args.tx_id` directly into this function and compares the result to the stored `merkle_root`: [3](#0-2) 

The function is `pub`, carries only `#[pause]` (not `#[private]`), and has no role or trusted-relayer guard, so any external account can call it: [4](#0-3) 

The developers acknowledge the issue in the doc comment but rely on "a higher level of verification" that the contract itself does not enforce: [5](#0-4) 

---

### Impact Explanation

For a 4-transaction block with leaves `[tx0, tx1, tx2, tx3]`:

```
node01 = double_sha256(tx0 || tx1)
node23 = double_sha256(tx2 || tx3)
root   = double_sha256(node01 || node23)
```

An attacker calls `verify_transaction_inclusion` with:
- `tx_id    = node01`  (an internal node, not a real transaction)
- `tx_index = 0`
- `merkle_proof = [node23]`

`compute_root_from_merkle_proof(node01, 0, [node23])` computes `double_sha256(node01 || node23) = root`, which equals `header.block_header.merkle_root`, so the function returns `true`. The attacker has proven inclusion of a transaction that does not exist.

---

### Likelihood Explanation

- The function is publicly callable with no access control.
- The attacker only needs to know the internal node hash at any position in the tree, which is computable from any two known leaf transactions.
- The proof path is one element shorter than a real leaf proof, making it trivially constructable.
- No privileged role, leaked key, or social engineering is required.

---

### Recommendation

1. **Remove `verify_transaction_inclusion` (v1) entirely** or gate it with `#[private]` so it is no longer externally callable. The `#[deprecated]` Rust attribute is a compile-time hint only; it provides zero runtime protection.
2. Enforce the coinbase-proof mitigation at the contract level for all callers, not just those who opt into v2.
3. If backward compatibility must be preserved, add a depth/tree-height check: the length of `merkle_proof` must equal `ceil(log2(block_tx_count))`, and `tx_index` must be less than `block_tx_count`. Both values would need to be stored or supplied and validated.

---

### Proof of Concept

```rust
// In merkle-tools/src/lib.rs tests (unit-testable without NEAR runtime)
#[test]
fn test_internal_node_forgery() {
    // 4-leaf tree
    let tx0 = decode_hex("aaaa..."); // any 32-byte value
    let tx1 = decode_hex("bbbb...");
    let tx2 = decode_hex("cccc...");
    let tx3 = decode_hex("dddd...");

    // Compute internal nodes exactly as the contract does
    let node01 = compute_hash(&tx0, &tx1);   // internal node, NOT a real tx
    let node23 = compute_hash(&tx2, &tx3);
    let root   = compute_hash(&node01, &node23);

    // Attacker supplies node01 as tx_id with a 1-element proof
    let forged_root = compute_root_from_merkle_proof(node01.clone(), 0, &vec![node23]);

    // This assertion passes — the contract would return `true`
    assert_eq!(forged_root, root,
        "Internal node accepted as leaf: forgery succeeds");
}
``` [6](#0-5) 

The same call path is reachable on-chain via the public `verify_transaction_inclusion` method, which delegates directly to `compute_root_from_merkle_proof` with no additional guards. [7](#0-6)

#[must_use]
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
