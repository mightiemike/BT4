Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Gates the Router Address Instead of the End-User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of `MetricOmmPool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router's address, not the end-user's. A pool admin who allowlists the router to enable router-based swaps inadvertently grants swap access to every caller of the router, completely defeating the per-user allowlist.

## Finding Description
**Root cause in `MetricOmmPool.swap()`:** The pool passes `msg.sender` as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // sender = direct caller of pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this verbatim to every configured extension (lines 149–177 of `ExtensionCalling.sol`).

**Root cause in `SwapAllowlistExtension.beforeSwap`:**

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly (line 72–80 of `MetricOmmSimpleRouter.sol`), so `sender = router`. The check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**Bypass path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to specific addresses.
2. Admin calls `setAllowedToSwap(pool, router, true)` to enable router-based swaps for allowlisted users.
3. Any unprivileged user — not in the allowlist — calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
4. The router calls `pool.swap()`, so `sender = router` in `beforeSwap`.
5. `allowedSwapper[pool][router] == true` → check passes → swap executes for the unauthorized user.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` (lines 92–188 of `MetricOmmSimpleRouter.sol`). There is no field in the `beforeSwap` signature that carries the original end-user's address, and the router does not pass it.

## Impact Explanation
The swap allowlist is a core access-control mechanism. Once the router is allowlisted (a necessary step for any router-based swap flow), the allowlist is rendered completely ineffective: every public caller of `MetricOmmSimpleRouter` can swap against the pool regardless of whether the admin intended to permit them. This is a broken core pool functionality / admin-boundary break — the pool admin's access control is bypassed by an unprivileged path through a public periphery contract.

## Likelihood Explanation
The condition is easily met: any pool that uses `SwapAllowlistExtension` and also wants to support router-based swaps must allowlist the router. This is a standard operational step. Once done, any unprivileged user can exploit the bypass immediately and repeatably by calling any of the four `exact*` functions on `MetricOmmSimpleRouter`. No special privileges, flash loans, or oracle manipulation are required.

## Recommendation
Pass the original end-user's address through the call chain. One approach: add an `originator` field to the `beforeSwap` hook signature (or use `extensionData` to carry it), and have the router encode `msg.sender` into `extensionData` before calling `pool.swap()`. `SwapAllowlistExtension.beforeSwap` would then decode and gate on the originator rather than `sender`. Alternatively, check both `sender` (router) and the decoded originator, requiring the originator to be allowlisted.
