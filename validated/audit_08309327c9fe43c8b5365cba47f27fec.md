### Title
Hardcoded Cost of 1 in `op_substr` Ignores Output Size — (`File: src/more_ops.rs`)

### Summary
`op_substr` (opcode 12) charges a fixed cost of `1` regardless of the size of the substring it produces. Every other operator that produces a variable-length atom uses `malloc_cost()` to add `atom_len * MALLOC_COST_PER_BYTE` to the returned cost. `op_substr` skips this entirely, so an attacker can extract an arbitrarily large atom for a cost of 1, breaking the cost model that protects consensus nodes from being overloaded.

### Finding Description
In `src/more_ops.rs`, `op_substr` validates its arguments and then returns:

```rust
let r = a.new_substr(a0, start as u32, end as u32)?;
let cost: Cost = 1;
Ok(Reduction(cost, r))
``` [1](#0-0) 

The helper `malloc_cost` exists precisely to add a per-byte output charge:

```rust
fn malloc_cost(a: &Allocator, cost: Cost, ptr: NodePtr) -> Reduction {
    let c = a.atom_len(ptr) as Cost * MALLOC_COST_PER_BYTE;
    Reduction(cost + c, ptr)
}
``` [2](#0-1) 

Every comparable operator — `op_add`, `op_subtract`, `op_multiply`, `op_concat`, `op_strlen`, `op_gr`, `op_ash`, `op_lsh`, etc. — either calls `malloc_cost` or explicitly adds `atom_len * MALLOC_COST_PER_BYTE` to its returned cost. [3](#0-2) 

`op_concat`, the closest structural peer to `op_substr`, charges `CONCAT_COST_PER_BYTE + MALLOC_COST_PER_BYTE` per byte of output. [4](#0-3) 

`op_substr` charges nothing per byte of output. The hardcoded `1` is the direct analog of the hardcoded USDC peg: a constant is substituted for a value that must be computed dynamically from the actual data size.

The opcode is wired into `ChiaDialect::op()` at opcode 12 with no flag guard, so it is reachable in every execution mode (consensus, mempool, softfork guard). [5](#0-4) 

### Impact Explanation
An attacker submits a CLVM program that calls `(substr large_atom 0 N)` where `N` is close to the atom's full length. The returned `NodePtr` references an atom of up to `N` bytes, but the cost charged is `1`. That large atom can then be fed into downstream operators (`concat`, `sha256`, arithmetic) whose costs are proportional to their *input* sizes. The attacker therefore gets the downstream per-byte input costs "for free" on the substr side, allowing programs that consume far more node CPU/memory than the declared cost budget permits. In the worst case this enables a cost-limit bypass: a program that should exceed `max_cost` does not, because the substr step is essentially free. This is a consensus-critical cost model invariant: every byte produced must be charged for.

### Likelihood Explanation
The entry path is direct: any caller of `run_program` with attacker-controlled CLVM bytes can trigger opcode 12. No special flag is required. The opcode is enabled in all modes including `MEMPOOL_MODE`. [6](#0-5) 

### Recommendation
Replace the hardcoded `let cost: Cost = 1;` with a base cost constant (analogous to `STRLEN_BASE_COST`, etc.) and add `malloc_cost` on the result, mirroring every other variable-output operator:

```rust
let r = a.new_substr(a0, start as u32, end as u32)?;
let cost: Cost = SUBSTR_BASE_COST;
Ok(malloc_cost(a, cost, r))
```

where `SUBSTR_BASE_COST` is calibrated against actual node benchmarks.

### Proof of Concept
```
; CLVM program: extract 1 MB from a 1 MB atom for cost = 1
; (substr (q . <1MB_atom>) 0 1048576)
; Reported cost: 1
; Actual bytes produced: 1,048,576
; Those bytes can then be passed to sha256/concat at full per-byte cost,
; but the substr step paid nothing.
```

Concretely: construct a CLVM tree where `a0` is a 1 048 576-byte atom (within allocator limits), `start = 0`, `end = 1048576`. The call to `op_substr` returns `Reduction(1, r)` where `r` is a 1 MB atom. Feeding `r` into `op_sha256` costs `SHA256_BASE_COST + SHA256_COST_PER_BYTE * 1048576 ≈ 2 097 152` — but the substr step that produced the input paid only `1`. A program budget set to `2 097 153` would be accepted even though the true resource consumption is `2 097 153 + (1048576 * MALLOC_COST_PER_BYTE)` units. [7](#0-6)

pub fn op_substr(
    a: &mut Allocator,
    input: NodePtr,
    _max_cost: Cost,
    _flags: ClvmFlags,
) -> Response {
    let ([a0, start, end], argc) = get_varargs::<3>(a, input, "substr")?;
    if !(2..=3).contains(&argc) {
        Err(EvalErr::InvalidOpArg(
            input,
            format!("Substring takes exactly 2 or 3 arguments, got {argc}"),
        ))?;
    }
    let size = atom_len(a, a0, "substr")?;
    let start = i32_atom(a, start, "substr")?;

    let end = if argc == 3 {
        i32_atom(a, end, "substr")?
    } else {
        size as i32
    };
    if end < 0 || start < 0 || end as usize > size || end < start {
        Err(EvalErr::InvalidOpArg(
            input,
            "Invalid Indices for Substring".to_string(),
        ))?
    } else {
        let r = a.new_substr(a0, start as u32, end as u32)?;
        let cost: Cost = 1;
        Ok(Reduction(cost, r))
    }

    mut input: NodePtr,
    max_cost: Cost,
    _flags: ClvmFlags,

    let mut cost = CONCAT_BASE_COST;
    let mut total_size: usize = 0;
    let mut terms = Vec::<NodePtr>::new();
    while let Some((arg, rest)) = a.next(input) {
        input = rest;
        let len = match a.sexp(arg) {
            SExp::Pair(_, _) => {
                return Err(EvalErr::InvalidOpArg(arg, "concat on list".to_string()))?;
            }
            SExp::Atom => a.atom_len(arg),
        };
        cost += CONCAT_COST_PER_ARG;
        cost += len as Cost * (CONCAT_COST_PER_BYTE + MALLOC_COST_PER_BYTE);
        check_cost(cost, max_cost)?;
        if len > 0 {
            // skip NIL arguments, as an optimization
            total_size += len;
            terms.push(arg);
        }
