### Title
IBC NFT Refund Permanently Fails on Sink-Chain Path Due to Wrong Sender in `MintNFT` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`refundPacketToken` handles two cases when a packet times out or receives an error acknowledgement. For the source-chain path it correctly unescrows the NFT. For the sink-chain path it calls `MintNFT` with the **escrow address** as the `sender` argument. `MintNFT` enforces a denom-creator check against that `sender`. The escrow address is a deterministic hash of port+channel and is never the denom creator, so the creator check always fails. The NFT was already burned during the original send; the refund path is permanently broken, and the user's NFT is irrecoverably lost.
