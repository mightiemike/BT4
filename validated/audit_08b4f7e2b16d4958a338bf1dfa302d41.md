Audit Report

## Title
SwapAllowlistExtension checks the router address instead of the actual user, enabling complete allowlist bypass via MetricOmmSimpleRouter — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument against `allowedSwapper[pool][sender]`. Because `MetricOmmPool.swap()` passes its own `msg.sender` as `sender`, and `MetricOmmSimpleRouter` calls `pool.swap()` directly, the extension always evaluates the router's address — not the end user's. Any user who routes through the router inherits the router's allowlist status, rendering per-user access control completely inoperative.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // immediate caller of the pool
  recipient,
  ...
);
``` [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that value:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
``` [2](#0-1) 

Here `msg.sender` = pool, `sender` = immediate caller of the pool. When the call originates from `MetricOmmSimpleRouter.exactInputSingle`, the pool's `msg.sender` is the router:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
``` [3](#0-2) 

The same holds for `exactInput` (every hop), `exactOutputSingle`, `exactOutput`, and the recursive `_exactOutputIterateCallback` hops: [4](#0-3) [5](#0-4) 

In every case the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. Once the pool admin calls `setAllowedToSwap(pool, router, true)` — a routine step to enable the standard UI — the gate is open to every user on the network who calls any of the four router entry points. No existing guard in the extension or the pool checks the original `msg.sender` of the router call.

## Impact Explanation
A pool protected by `SwapAllowlistExtension` is a permissioned venue; only allowlisted counterparties should execute swaps. Once the router is allowlisted, any unprivileged user can drain liquidity from the pool at oracle-quoted prices. LPs deposited believing the pool was restricted; unauthorized traders can extract value at will. This constitutes broken core pool functionality with direct loss of LP assets, satisfying the contest's Critical/High impact threshold.

## Likelihood Explanation
The only precondition is that the pool admin allowlists the router — a natural, expected operational action for any pool that wants to support the standard periphery UI. No privileged attacker capability, no malicious token, no special timing is required. Any ordinary user who discovers the router is allowlisted can exploit it immediately and repeatedly. The misconfiguration is invisible: `isAllowedToSwap(pool, router)` returns `true` as intended, and there is no on-chain signal that this also opens the gate to all users.

## Recommendation
The extension must verify the original end user, not the immediate pool caller. Two concrete approaches:

1. **Preferred — router encodes real user in `extensionData`**: Have the router encode `msg.sender` into `extensionData` for every `pool.swap()` call. `SwapAllowlistExtension.beforeSwap` decodes and checks that value when `sender` is a recognized router address.

2. **Simplest — document and remove router from allowlist model**: Document that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)` and provide a separate router-aware extension for pools that need both router support and per-user gating.

## Proof of Concept
```
Setup
─────
1. Deploy pool with SwapAllowlistExtension on beforeSwap.
2. Pool admin: setAllowedToSwap(pool, alice, true)   // intended user
3. Pool admin: setAllowedToSwap(pool, router, true)  // enable router UI for alice

Attack
──────
4. Bob (never allowlisted) calls:
   MetricOmmSimpleRouter.exactInputSingle({
     pool:     <restricted pool>,
     tokenIn:  token0,
     amountIn: large,
     recipient: bob,
     ...
   })

5. Router calls pool.swap(bob, zeroForOne, ...)
   → pool's msg.sender = router

6. _beforeSwap(router, bob, ...)
   → extension checks allowedSwapper[pool][router] = true → passes

7. Swap executes; Bob receives token1 from the restricted pool.

Result: Bob, never allowlisted, successfully swaps against the pool.
        The per-user allowlist is completely bypassed.
```
