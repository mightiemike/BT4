### Title
Expired `delayed_declares` Bypass TTL Enforcement and Are Admitted into Active Pool - (File: `crates/apollo_mempool/src/mempool.rs`)

### Summary
`remove_expired_txs()` only scans `tx_pool` for TTL-expired transactions. Declare transactions held in the separate `delayed_declares` queue are never subject to TTL expiry. When `add_ready_declares()` eventually moves them into `tx_pool`, it performs no TTL check, so a Declare transaction that has already exceeded `transaction_ttl` is silently admitted into the active pool and can be returned by `get_txs()` for sequencing.

### Finding Description

The `Mempool` struct maintains two distinct storage areas for transactions:

- `tx_pool` — the main pool for all active transactions, subject to TTL enforcement.
- `delayed_declares` — a separate `AddTransactionQueue` holding Declare transactions waiting for `declare_delay` to elapse before being promoted to `tx_pool`. [1](#0-0) 

When `add_tx()` is called, it first invokes `remove_expired_txs()`, which exclusively operates on `self.tx_pool`: [2](#0-1) 

`remove_expired_txs()` calls `self.tx_pool.remove_txs_older_than(...)` — it never touches `self.delayed_declares`. Declare transactions sitting in `delayed_declares` are therefore invisible to TTL enforcement.

After `remove_expired_txs()`, `add_tx()` calls `add_ready_declares()`: [3](#0-2) 

`add_ready_declares()` checks only whether `declare_delay` has elapsed (`now - declare_delay < submission_time`). It contains **no TTL check**. Any declare whose `declare_delay` has elapsed is unconditionally moved into `tx_pool` via `add_tx_inner()`, regardless of whether it has also exceeded `transaction_ttl`.

The same ordering applies in `get_txs()`: `add_ready_declares()` runs first, promoting potentially-expired declares into `tx_pool`, before `prune_expired_nonqueued_txs()` can catch them on the pop path. [4](#0-3) 

The `AddTransactionQueue` type that backs `delayed_declares` has no TTL-aware removal method at all: [5](#0-4) 

The structural parallel to the Solidity bug is exact:
- `s_offersToUpkeep` ↔ `tx_pool` (the set checked for expiry)
- `delayed_declares` ↔ pending offers (never added to the expiry-checked set)
- `performUpkeep()` ↔ `remove_expired_txs()` (only iterates the main set)
- `add_ready_declares()` ↔ the acceptance path (promotes items without expiry check)

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

A Declare transaction that has exceeded `transaction_ttl` is promoted from `delayed_declares` into `tx_pool` as a live transaction. Once in `tx_pool`, it occupies capacity and — if its nonce matches the account nonce — is inserted into `tx_queue` and returned by `get_txs()` to the batcher for inclusion in a block. A transaction the protocol intended to discard is instead sequenced.

Additionally, `size_in_bytes()` sums both pools: [6](#0-5) 

Expired declares that linger in `delayed_declares` past their TTL continue to count against the mempool's `capacity_in_bytes` limit, causing valid transactions to be evicted.

### Likelihood Explanation

The default `declare_delay` is 1 second and `transaction_ttl` is 60 seconds: [7](#0-6) 

Under default configuration the window is narrow but non-zero: if the mempool is idle (no `add_tx` or `get_txs` calls) for `transaction_ttl` seconds after a declare is submitted, the declare will be promoted into `tx_pool` as an expired transaction on the next activity. When `declare_delay` is configured above `transaction_ttl` (a valid configuration with no schema-level guard), every declare submitted will be promoted into `tx_pool` already expired. Any user who can submit a Declare transaction can trigger this path without privilege.

### Recommendation

Add TTL enforcement inside `add_ready_declares()`: before calling `add_tx_inner(args)`, check whether `now - submission_time > transaction_ttl` and drop (and count as expired) any declare that has exceeded its TTL. Alternatively, extend `remove_expired_txs()` to also scan `delayed_declares` and remove entries whose `submission_time` is older than `transaction_ttl`. A config-level validation asserting `declare_delay < transaction_ttl` should also be added to `MempoolStaticConfig`.

### Proof of Concept

```
1. Configure mempool with declare_delay = 120s, transaction_ttl = 60s.
2. Submit a Declare transaction at T=0. It enters `delayed_declares`.
3. At T=60s, the TTL has elapsed. `remove_expired_txs()` is called on every
   subsequent `add_tx()` but only scans `tx_pool` — the declare is untouched.
4. At T=120s, `add_ready_declares()` fires (triggered by any `add_tx` or
   `get_txs` call). It checks only `now - declare_delay < submission_time`
   (120s - 120s = 0 < 0 is false), so the declare passes and is moved into
   `tx_pool` via `add_tx_inner()`.
5. The declare is now in `tx_pool` 60 seconds past its TTL. If its nonce
   matches the account nonce it is inserted into `tx_queue` and returned by
   the next `get_txs()` call for inclusion in a block.
6. `remove_expired_txs()` will only clean it up on the *following* `add_tx()`
   call, after it may already have been handed to the batcher.
```

// A queue to hold transactions that are waiting to be added to the tx pool.
struct AddTransactionQueue {
    elements: VecDeque<(DateTime, AddTransactionArgs)>,
    // Keeps track of the total size of the transactions in this queue.
    size_in_bytes: u64,
}

impl AddTransactionQueue {
    fn new() -> Self {
        AddTransactionQueue { elements: VecDeque::new(), size_in_bytes: 0 }
    }

    fn push_back(&mut self, submission_time: DateTime, args: AddTransactionArgs) {
        self.size_in_bytes = self
            .size_in_bytes
            .checked_add(args.tx.total_bytes())
            .expect("Overflow when adding a transaction to AddTransactionQueue.");
        self.elements.push_back((submission_time, args));
    }

    fn pop_front(&mut self) -> Option<(DateTime, AddTransactionArgs)> {
        let removed_element = self.elements.pop_front();
        if let Some((_, args)) = &removed_element {
            self.size_in_bytes = self
                .size_in_bytes
                .checked_sub(args.tx.total_bytes())
                .expect("Underflow when removing a transaction from AddTransactionQueue.");
        }
        removed_element
    }

    fn front(&self) -> Option<&(DateTime, AddTransactionArgs)> {
        self.elements.front()
    }

    fn contains(&self, contract_address: ContractAddress, nonce: Nonce) -> bool {
        self.elements.iter().any(|(_, tx_args)| {
            let tx = &tx_args.tx;
            tx.contract_address() == contract_address && tx.nonce() == nonce
        })
    }

    fn len(&self) -> usize {
        self.elements.len()
    }

    fn size_in_bytes(&self) -> u64 {
        self.size_in_bytes
    }
}

    pub fn get_txs(&mut self, n_txs: usize) -> MempoolResult<Vec<InternalRpcTransaction>> {
        // All transactions are enqueued in FIFO mode.
        if !self.is_fifo() {
            self.add_ready_declares();
        }
        let mut eligible_tx_references: Vec<TransactionReference> = Vec::with_capacity(n_txs);
        let mut n_remaining_txs = n_txs;

        let mut account_nonce_updates = AddressToNonce::new();
        while n_remaining_txs > 0 && self.tx_queue.has_ready_txs() {
            let chunk = self.tx_queue.pop_ready_chunk(n_remaining_txs);

            let (valid_txs, expired_txs_updates) = self.prune_expired_nonqueued_txs(chunk);
            account_nonce_updates.extend(expired_txs_updates);

            // In FIFO mode, all transactions are already enqueued. In fee-priority mode,
            // we need to enqueue the next eligible transaction for each address.
            if !self.is_fifo() {
                self.enqueue_next_eligible_txs(&valid_txs)?;
            }

            n_remaining_txs -= valid_txs.len();
            eligible_tx_references.extend(valid_txs);
        }

    fn add_ready_declares(&mut self) {
        let now = self.clock.now();
        while let Some((submission_time, _args)) = self.delayed_declares.front() {
            if now - self.config.static_config.declare_delay < *submission_time {
                break;
            }
            let (_submission_time, args) =
                self.delayed_declares.pop_front().expect("Delay declare should exist.");
            self.add_tx_inner(args);
        }
        self.update_state_metrics();

    fn remove_expired_txs(&mut self) -> AddressToNonce {
        let removed_txs = self
            .tx_pool
            .remove_txs_older_than(self.config.dynamic_config.transaction_ttl, &self.state.staged);

        for tx_ref in &removed_txs {
            self.decrement_stuck_txs_if_gap_account(tx_ref.address, 1);
        }

        let queued_txs = self.tx_queue.remove_txs(&removed_txs);

        self.log_and_count_expired_txs(&removed_txs);
        self.update_state_metrics();
        queued_txs
            .into_iter()
            .map(|tx| (tx.address, self.state.resolve_nonce(tx.address, tx.nonce)))
            .collect::<AddressToNonce>()
    }
