Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of user address, allowing any caller to bypass per-user swap allowlist via `MetricOmmSimpleRouter` — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap()`, so when `MetricOmmSimpleRouter` calls `pool.swap()`, the extension receives `sender = router`, not the originating user. `SwapAllowlistExtension.beforeSwap()` checks `allowedSwapper[pool][sender]` against this router address. If the pool admin allowlists the router (the only way to let any user route through it), every caller of the router — including explicitly excluded addresses — passes the allowlist check unconditionally.

## Finding Description
`MetricOmmPool.swap()` unconditionally forwards `msg.sender` as the `sender` argument: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap()` then gates the swap by checking that `sender` against the per-pool allowlist, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly with no encoding of the originating user into `extensionData`: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

**Two cases arise:**
- **Router not allowlisted:** All router-mediated swaps revert with `NotAllowedToSwap`, even for individually allowlisted users. The router is unusable with this pool.
- **Router allowlisted (required for any router-mediated swap to work):** `allowedSwapper[pool][router] == true` causes `beforeSwap` to pass for every caller of the router, including addresses explicitly excluded from the allowlist. The per-user gate is completely bypassed.

There is no existing guard that recovers the original user identity. The `extensionData` field passed by the router is `""` (empty) or user-supplied arbitrary bytes — neither encodes the actual caller in a verifiable way.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers) loses that restriction entirely for any user routing through `MetricOmmSimpleRouter`. A disallowed user executes swaps against the pool's liquidity at oracle-fair prices, extracting value the pool admin intended to reserve for allowlisted participants. This is a direct, fund-impacting breach of the pool's access-control invariant: unauthorized users trade against LP capital that was meant to be protected, constituting a medium-severity direct loss of the pool's access-control guarantee with fund-impacting consequences for LPs.

## Likelihood Explanation
Likelihood is medium. Both required conditions are the natural production configuration: (1) a pool deployed with `SwapAllowlistExtension` as a `beforeSwap` hook, and (2) the pool admin allowlisting the router — which is the only way to let any allowlisted user use the standard periphery router. `MetricOmmSimpleRouter` is the primary documented swap entrypoint, so the attack path is reachable by any user who reads the contract. No special privilege or front-running is required; a disallowed user simply calls `exactInputSingle` with the target pool.

## Recommendation
The `sender` forwarded to extensions must represent the economic actor, not the intermediary. The cleanest fix is for `MetricOmmSimpleRouter` to ABI-encode `msg.sender` (the actual user) into `extensionData` before calling `pool.swap()`, and for `SwapAllowlistExtension.beforeSwap()` to decode and check that address when present, falling back to `sender` for direct pool calls. Alternatively, the pool interface can be extended with an explicit `originator` field passed alongside `sender` through the entire hook call chain.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension as beforeSwap hook
  - Pool admin: setAllowedToSwap(pool, allowedUser, true)
  - Pool admin: setAllowedToSwap(pool, router, true)
    (required so allowedUser can use the router at all)

Attack:
  - disallowedUser calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(recipient, ...) [msg.sender = router]
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] → true
  - Swap executes for disallowedUser at oracle price
  - disallowedUser receives output tokens; allowlist policy is bypassed

Call path:
disallowedUser → MetricOmmSimpleRouter.exactInputSingle()
    → pool.swap(recipient, ...) [msg.sender = router]
        → _beforeSwap(sender=router, ...)
            → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                → allowedSwapper[pool][router] == true → PASSES
        → swap executes
```
