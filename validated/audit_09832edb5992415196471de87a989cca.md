### Title
NFT Permanently Locked via Transfer to Blocked Module Account — (`x/nft/keeper/keeper.go`)

### Summary

`Keeper.TransferOwner` performs no validation of the destination address against blocked module accounts. An NFT owner can transfer ownership to any valid bech32 address — including module accounts that have no signing key — permanently locking the NFT with no recovery path.

### Finding Description

The full call chain is:

`MsgTransferNFT` → `msgServer.TransferNFT` → `Keeper.TransferOwner` → `swapOwner` (`deleteOwner` / `setOwner`)

**Step 1 — `ValidateBasic` only checks bech32 validity:**

`MsgTransferNFT.ValidateBasic` accepts any valid bech32 address as recipient, including module account addresses. [1](#0-0) 

**Step 2 — `msgServer.TransferNFT` passes recipient directly to `TransferOwner` with no additional guard:** [2](#0-1) 

**Step 3 — `Keeper.TransferOwner` only checks denom existence and that `srcOwner` is the current owner. There is no check on `dstOwner`:** [3](#0-2) 

**Step 4 — `swapOwner` unconditionally writes the new owner into the KV store:** [4](#0-3) 

**Step 5 — The `Keeper` struct has no `AccountKeeper` or `BankKeeper` field at all:**

The interfaces defined in `expected_keepers.go` are never injected into the NFT `Keeper`, so no `BlockedAddr`-style check is even possible at the keeper level. [5](#0-4) [6](#0-5) 

**Step 6 — Recovery is impossible.** `BurnNFT` requires the caller to be both the current owner AND the denom creator simultaneously: [7](#0-6) 

Once a module account (e.g., `distribution`, `bonded_tokens_pool`) is the recorded owner, no real private key can sign a `MsgTransferNFT` or `MsgBurnNFT` on its behalf. The NFT is permanently locked.

### Impact Explanation

Permanent, irrecoverable loss of an economically valuable NFT. The NFT's ownership record points to an address from which no transaction can ever be signed. The NFT cannot be transferred, burned, or edited by any real key. This matches the Critical scope: permanent lock of an economically valuable NFT in a module account with no signing key.

### Likelihood Explanation

The attack requires only that the attacker own an NFT (trivially achievable by minting or receiving one) and know the bech32 address of any blocked module account (all are publicly known and derivable). No special privileges, governance, or operator access are needed. The transaction is a standard `MsgTransferNFT` signed by the NFT owner.

### Recommendation

Inject `AccountKeeper` (and optionally `BankKeeper`) into the NFT `Keeper` struct and add a blocked-address check in `TransferOwner` before writing the new owner:

```go
// In Keeper.TransferOwner, before setNFT/swapOwner:
if k.bankKeeper.BlockedAddr(dstOwner) {
    return sdkerrors.Wrapf(types.ErrUnauthorized, "recipient %s is a blocked module account", dstOwner)
}
```

Alternatively, add the check in `msgServer.TransferNFT` using `accountKeeper.GetAccount` and asserting the result is not a `ModuleAccountI`.

### Proof of Concept

```go
// keeper integration test (unmodified Go/Cosmos test setup)
func TestTransferToModuleAccountLocksNFT(t *testing.T) {
    ctx, keeper := setupKeeper(t)
    userA := sdk.AccAddress([]byte("userA"))
    moduleAccAddr := authtypes.NewEmptyModuleAccount("distribution").GetAddress()

    // Issue denom and mint NFT to userA
    _ = keeper.IssueDenom(ctx, "testdenom", "Test", "", "", userA)
    _ = keeper.MintNFT(ctx, "testdenom", "token1", "MyNFT", "", "", userA, userA)

    // Transfer to blocked module account — succeeds with no error
    err := keeper.TransferOwner(ctx, "testdenom", "token1", userA, moduleAccAddr)
    require.NoError(t, err)  // passes — no guard exists

    // NFT is now owned by module account
    nft, _ := keeper.GetNFT(ctx, "testdenom", "token1")
    require.Equal(t, moduleAccAddr.String(), nft.GetOwner().String())

    // No real key can sign as moduleAccAddr — NFT is permanently locked
    err = keeper.TransferOwner(ctx, "testdenom", "token1", moduleAccAddr, userA)
    require.Error(t, err) // would require signing as module account — impossible
}
```

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
```

func (k Keeper) swapOwner(ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress) {
	// delete old owner key
	k.deleteOwner(ctx, denomID, tokenID, srcOwner)

	// set new owner key
	k.setOwner(ctx, denomID, tokenID, dstOwner)
}
