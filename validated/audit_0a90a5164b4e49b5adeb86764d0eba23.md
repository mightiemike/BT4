### Title
Unvalidated `tx_id` Input in `verify_transaction_inclusion` Allows Merkle-Node Forgery as Proof of Transaction Inclusion - (File: `contract/src/lib.rs`)

### Summary
The deprecated but still-callable public function `verify_transaction_inclusion` accepts any 32-byte value as `tx_id` without checking whether it represents a real Bitcoin transaction hash or an internal Merkle tree node. An unprivileged NEAR caller can supply an internal node hash as `tx_id` together with a crafted `merkle_proof` that correctly reconstructs the block's `merkle_root`, causing the function to return `true` and falsely certify that a non-existent transaction was confirmed on Bitcoin.

### Finding Description
`verify_transaction_inclusion` in `contract/src/lib.rs` (lines 288–323) is a public, unguarded endpoint. Its only input checks are:

- `args.confirmations <= self.gc_threshold`
- `args.tx_block_blockhash` must be on the main chain
- `!args.merkle_proof.is_empty()`

After those guards it calls `merkle_tools::compute_root_from_merkle_proof(args.tx_id, ...)` and compares the result to `header.block_header.merkle_root`. There is no check that `args.tx_id` is a leaf-level transaction hash rather than an intermediate node hash. The contract's own documentation acknowledges this:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [1](#0-0) 

The `#[deprecated]` attribute is a Rust compiler hint only; it imposes no runtime restriction. The function remains fully callable by any NEAR account. The `#[pause]` macro can disable it, but only if a `PauseManager` has explicitly paused it — by default it is active. [2](#0-1) 

`compute_root_from_merkle_proof` in `merkle-tools/src/lib.rs` is a pure hash-chain computation; it applies no constraint on what the starting hash represents. [3](#0-2) 

`ProofArgs` places no restriction on `tx_id` either — it is a plain `H256`. [4](#0-3) 

### Impact Explanation
Any downstream NEAR contract that calls `verify_transaction_inclusion` and acts on a `true` result (e.g., releasing bridged funds, minting tokens, unlocking collateral) can be deceived into processing a Bitcoin transaction that never existed. The attacker does not need to mine a block or control any privileged role; they only need a valid main-chain block hash and knowledge of that block's Merkle tree structure (publicly available from any Bitcoin node or block explorer).

### Likelihood Explanation
The function is publicly reachable with no access control. Bitcoin block Merkle trees are fully public. Constructing a valid internal-node proof requires only arithmetic over known hashes. The deprecated label does not prevent calls. Any integrator that has not independently audited the deprecation notice and migrated to v2 is exposed.

### Recommendation
1. Remove or gate `verify_transaction_inclusion` behind an access-control role so it cannot be called by arbitrary accounts, or add a runtime `panic!` / `env::panic_str` to make it unconditionally revert.
2. Direct all callers to `verify_transaction_inclusion_v2`, which enforces a coinbase proof that anchors the leaf level and closes the internal-node substitution path.
3. Add an explicit check that `args.tx_index != 0` is consistent with the coinbase position, or validate minimum proof length relative to expected tree depth.

### Proof of Concept

**Setup:** A real Bitcoin block `B` is on the contract's main chain. Its Merkle tree has at least two transactions: `tx0` (coinbase) and `tx1`. The internal node `N = SHA256d(tx0 || tx1)` is known.

**Attack:**
1. Attacker calls `verify_transaction_inclusion` with:
   - `tx_id = N` (the internal node hash, not a real txid)
   - `tx_block_blockhash = B`
   - `tx_index = 0`
   - `merkle_proof = [<remaining siblings above N to reconstruct merkle_root>]`
   - `confirmations = 1`

2. `compute_root_from_merkle_proof(N, 0, proof)` correctly reconstructs `merkle_root` of block `B`.

3. The function returns `true`.

4. A downstream contract that trusts this result treats the fake `tx_id = N` as a confirmed Bitcoin transaction and executes its payout or unlock logic. [5](#0-4) [6](#0-5)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
