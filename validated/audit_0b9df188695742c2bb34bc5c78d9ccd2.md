Audit Report

## Title
`SwapAllowlistExtension` per-user allowlist is fully bypassed when swaps are routed through `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it unchanged to every configured extension. When `MetricOmmSimpleRouter` is the caller, `sender` is the router address, not the originating EOA. `SwapAllowlistExtension.beforeSwap()` checks `allowedSwapper[pool][sender]`, so it evaluates the router's allowlist status. Any pool that allowlists the router inadvertently grants swap access to every user of that router, completely defeating the per-user access control the extension is documented to provide.

## Finding Description
The full call chain is confirmed in production code:

1. `MetricOmmPool.swap()` captures `msg.sender` and passes it as `sender` to `_beforeSwap`: [1](#0-0) 

2. `ExtensionCalling._beforeSwap` encodes `sender` verbatim into the `abi.encodeCall` forwarded to each extension: [2](#0-1) 

3. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no mechanism to forward the originating EOA — the router is `msg.sender` from the pool's perspective: [3](#0-2) 

4. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` = pool and `sender` = router: [4](#0-3) 

A pool admin who allowlists the router address (the natural way to "permit swaps via the official router") sets `allowedSwapper[pool][router] = true`. Every subsequent call through `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` passes the check regardless of the originating EOA, because `sender` is always the router. No existing guard recovers the originating EOA; the pool has no `tx.origin` check, and the extension receives only what the pool passes.

## Impact Explanation
The `SwapAllowlistExtension`'s sole documented purpose is gating swaps by swapper address per pool. That invariant is broken for any pool that permits router-mediated swaps: pools intended to restrict trading to KYC'd, institutional, or otherwise vetted addresses are silently open to all router users. This is broken core pool functionality — the extension's primary access-control mechanism does not function as intended when the standard periphery router is used, matching the "Broken core pool functionality" and "Admin-boundary break via unprivileged path" allowed impact criteria.

## Likelihood Explanation
The router is the primary user-facing entry point for swaps. Any pool operator who deploys a `SwapAllowlistExtension` and allowlists the router will unknowingly expose the pool to all router users. The same bypass applies to all four router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`). The configuration is non-obvious, the extension emits no warning, and the `isAllowedToSwap` view function returns `true` for the router without indicating that this grants access to all router callers. [5](#0-4) 

## Recommendation
The router should accept a `swapper` parameter and forward it via `extensionData` for the extension to decode, or the extension should document explicitly that allowlisting a router address grants access to all users of that router and that only individual EOAs should be allowlisted. A more robust fix is for the pool to accept an explicit `swapper` argument in `swap()` (separate from `msg.sender`) that the router populates with `msg.sender` before calling the pool, and for the extension to check that field instead.

## Proof of Concept
```solidity
// 1. Deploy pool with SwapAllowlistExtension configured
// 2. Pool admin allowlists only the router:
swapExt.setAllowedToSwap(address(pool), address(router), true);
// 3. Non-allowlisted EOA (attacker) calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool), ...
}));
// 4. pool.swap(): msg.sender == router → _beforeSwap(router, ...)
// 5. ExtensionCalling encodes sender=router → extension.beforeSwap(sender=router, ...)
// 6. allowedSwapper[pool][router] == true → check passes
// 7. Swap executes for the non-allowlisted EOA — allowlist fully bypassed
```
