### Title
Non-Canonical Backref Path Encoding Accepted by `node_from_bytes_backrefs` but Rejected by `is_canonical_serialization` — (`src/serde/de_br.rs`, `src/serde/tools.rs`)

### Summary
`node_from_bytes_backrefs` accepts backref path atoms encoded with a redundant length prefix (e.g., `0x81 0x01` for path value `[0x01]`), while `is_canonical_serialization` rejects the same byte stream. Two distinct byte streams deserialize to the same CLVM tree, breaking the one-canonical-stream invariant.

### Finding Description

`parse_path` delegates to `parse_atom_ptr`, which performs no canonicality check on the path atom — it only decodes the length and reads the bytes. [1](#0-0) 

`parse_atom_ptr` accepts any valid length-prefixed encoding, including over-long ones: [2](#0-1) 

`is_canonical_serialization`, when it encounters `BACK_REFERENCE`, reads the next byte and passes it to `is_canonical_atom`: [3](#0-2) 

`is_canonical_atom` enforces that a 1-byte atom encoded with a 1-byte length prefix must have its high bit set (i.e., value ≥ `0x80`), otherwise it could be encoded as a bare single byte: [4](#0-3) 

For path byte `0x81 0x01` (value `[0x01]`, non-canonical):
- `node_from_bytes_backrefs` → `parse_path` → `parse_atom_ptr` → returns `&[0x01]`, resolves backref → **`Ok(node)`**
- `is_canonical_serialization` → `is_canonical_atom(0x81)` → reads `0x01`, `0x01 < 0x80` → **`false`**

For path byte `0x01` (value `[0x01]`, canonical):
- `node_from_bytes_backrefs` → **`Ok(same node)`**
- `is_canonical_serialization` → `0x01 ≤ MAX_SINGLE_BYTE` → **`true`**

### Impact Explanation
Any system that gates consensus acceptance on `is_canonical_serialization` before or instead of `node_from_bytes_backrefs` will split: one code path accepts the non-canonical byte stream and resolves it to a valid tree; the other rejects it. Two distinct byte streams map to the same consensus object (same tree hash), violating the one-canonical-stream invariant. This enables a mempool/consensus divergence attack where an attacker submits a spend bundle or block containing a non-canonical backref path, causing nodes that check canonicality to reject it while nodes that don't (or check after deserialization) accept it.

### Likelihood Explanation
The attacker only needs to control serialized bytes submitted to the API — a standard, supported entrypoint. The non-canonical encoding is trivially constructable: replace any single-byte path value `v` (where `v ≤ 0x7f`) in a backref with `0x81 v`. The divergence is deterministic and reproducible.

### Recommendation
Add a canonicality check inside `parse_path` (or `parse_atom_ptr` when called from the backref path). Specifically, after decoding the path atom, verify that the encoding is minimal: a 1-byte length-prefixed atom must have `value[0] >= 0x80`, and multi-byte length prefixes must encode a value that requires that prefix width. Alternatively, `parse_path` should call `is_canonical_atom` logic inline before returning the path slice, mirroring what `is_canonical_serialization` already enforces.

fn parse_atom_ptr<'a>(f: &'a mut Cursor<&[u8]>, first_byte: u8) -> Result<&'a [u8]> {
    let blob = if first_byte <= MAX_SINGLE_BYTE {
        let pos = f.position() as usize;
        &f.get_ref()[pos - 1..pos]
    } else {
        let blob_size = decode_size(f, first_byte)?;
        let pos = f.position() as usize;
        if f.get_ref().len() < pos + blob_size as usize {
            return Err(EvalErr::SerializationError);
        }
        f.seek(SeekFrom::Current(blob_size as i64))?;
        &f.get_ref()[pos..(pos + blob_size as usize)]
    };
    Ok(blob)
}

fn is_canonical_atom(f: &mut Cursor<&[u8]>, first_byte: u8) -> bool {
    if first_byte == 0x80 || first_byte <= MAX_SINGLE_BYTE {
        return true;
    }

    let Ok((prefix_len, atom_len)) = decode_size_with_offset(f, first_byte) else {
        return false;
    };

    let min_value = match prefix_len {
        1 => 1,
        2 => 1 << 6,
        3 => 1 << (5 + 8),
        4 => 1 << (4 + 8 + 8),
        5 => 1 << (4 + 8 + 8 + 8),
        6 => 1 << (4 + 8 + 8 + 8 + 8),
        _ => panic!("unexpected atom length prefix {prefix_len}"),
    };

    if atom_len == 1 {
        let mut value = [0_u8];
        if f.read_exact(&mut value).is_err() {
            return false;
        }
        if value[0] < 0x80 {
            return false;
        }
    } else if f.seek(SeekFrom::Current(atom_len as i64)).is_err() {
        return false;
    }
    atom_len >= min_value
}
