### Title
`AllResources`→`L1Gas` Variant Collapse in Protobuf Deserialization Causes Valid Declare Transactions to Be Rejected in Consensus Path — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero into the `L1Gas` variant. A downstream consensus-path converter for `RpcDeclareTransactionV3` explicitly rejects the `L1Gas` variant with a hard error. A valid declare transaction with non-zero L1 gas but zero L2/L1DataGas bounds is therefore accepted at the gateway, assigned a correct `AllResources` hash, propagated over P2P, and then permanently rejected by every receiving peer — preventing it from ever being sequenced.

---

### Finding Description

**Step 1 — Serialization (correct).**
`From<ValidResourceBounds> for protobuf::ResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` serializes an `AllResources` transaction faithfully, emitting all three resource-limit fields even when L2 and L1DataGas are zero:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),       // zero but present
        l1_data_gas: Some(l1_data_gas.into()), // zero but present
    },
``` [1](#0-0) 

**Step 2 — Deserialization (broken).**
`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a value-based heuristic: if both `l2_gas` and `l1_data_gas` are zero it returns `L1Gas`, discarding the `AllResources` variant:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [2](#0-1) 

This conversion is invoked for every `DeclareV3Common` and `DeployAccountV3` received over P2P: [3](#0-2) [4](#0-3) 

**Step 3 — Consensus-path rejection.**
`TryFrom<protobuf::DeclareV3WithClass> for RpcDeclareTransactionV3` (used in the consensus/RPC path) explicitly rejects the `L1Gas` variant:

```rust
resource_bounds: match common.resource_bounds {
    ValidResourceBounds::AllResources(resource_bounds) => resource_bounds,
    _ => {
        return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR);  // ← hard rejection
    }
},
``` [5](#0-4) 

**Step 4 — Hash domain divergence (secondary).**
`get_tip_resource_bounds_hash` produces structurally different preimages for the two variants. For `AllResources` it appends the L1DataGas resource-name felt even when the amount is zero; for `L1Gas` it does not. So if a hash were ever recomputed post-deserialization it would not match the gateway-computed hash:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // no L1DataGas felt
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // felt included
    }
});
``` [6](#0-5) 

The `ValidResourceBounds` enum and its two variants are defined here: [7](#0-6) 

---

### Impact Explanation

A valid V3 declare transaction whose `AllResourceBounds` has non-zero L1 gas but zero L2 gas and zero L1DataGas is accepted by the gateway, hashed correctly, and admitted to the mempool. When the proposing node serializes it into a `DeclareV3WithClass` consensus message and broadcasts it, every receiving peer's protobuf deserializer collapses the variant to `L1Gas`, and the subsequent `RpcDeclareTransactionV3` conversion returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction can never be included in a block. This matches the **High** impact scope: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The trigger is a user-controlled field combination (`l2_gas.max_amount = 0`, `l1_data_gas.max_amount = 0`, `l1_gas.max_amount > 0`) that is explicitly permitted by the gateway's `validate_resource_bounds` logic (any single non-zero resource bound is sufficient). No privileged access is required. The condition is reachable on every declare transaction that does not set L2 or L1DataGas bounds.

---

### Recommendation

Remove the value-based variant heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. The protobuf wire format already carries all three fields explicitly; the variant should be determined by the protocol version or a dedicated tag, not by whether the numeric values happen to be zero. One concrete fix: always deserialize into `AllResources` when all three fields are present (even if zero), and reserve `L1Gas` only for the legacy case where `l1_data_gas` is absent (`None`) — which is already handled by `unwrap_or_default` and the TODO comment on line 426.

---

### Proof of Concept

```rust
// 1. Construct a valid AllResources declare tx with zero L2/L1DataGas.
let original = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas:      ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) },
    l2_gas:      ResourceBounds::default(),   // zero
    l1_data_gas: ResourceBounds::default(),   // zero
});

// 2. Serialize to protobuf (correct — all three fields emitted).
let proto: protobuf::ResourceBounds = original.into();
assert!(proto.l1_data_gas.is_some());  // present, but zero

// 3. Deserialize — variant collapses to L1Gas.
let roundtripped = ValidResourceBounds::try_from(proto).unwrap();
assert!(matches!(roundtripped, ValidResourceBounds::L1Gas(_)));  // ← WRONG variant

// 4. Consensus-path conversion now fails.
// TryFrom<protobuf::DeclareV3WithClass> for RpcDeclareTransactionV3 returns
// Err(DEPRECATED_RESOURCE_BOUNDS_ERROR) because roundtripped is L1Gas, not AllResources.

// 5. Hash domain divergence: original hash includes L1DataGas felt; recomputed hash does not.
let hash_all  = get_tip_resource_bounds_hash(&original,     &Tip(0)).unwrap();
let hash_l1   = get_tip_resource_bounds_hash(&roundtripped, &Tip(0)).unwrap();
assert_ne!(hash_all, hash_l1);  // ← different hashes for the same logical transaction
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
