### Missing Intermediate `check_cost` Call in `op_bls_map_to_g2` After Message Byte Cost Accumulation - (File: `src/bls_ops.rs`)

---

### Summary

`op_bls_map_to_g2` (opcode 57) omits the `check_cost` call after accumulating the per-message-byte cost, while the structurally identical `op_bls_map_to_g1` (opcode 56) correctly calls `check_cost` at that intermediate point. An attacker supplying a large message atom can cause the cost to silently exceed `max_cost` before the guard fires, delaying enforcement until after the DST atom is also read and its cost accumulated.

---

### Finding Description

`op_bls_map_to_g1` follows a strict two-phase cost-check pattern:

```
cost += msg.len() * BLS_MAP_TO_G1_COST_PER_BYTE;
check_cost(cost, max_cost)?;          // ← present
cost += dst.len() * BLS_MAP_TO_G1_COST_PER_DST_BYTE;
check_cost(cost, max_cost)?;          // ← present
``` [1](#0-0) 

`op_bls_map_to_g2` is missing the first of those two guards:

```
cost += msg.len() * BLS_MAP_TO_G2_COST_PER_BYTE;
// ← check_cost MISSING here
cost += dst.len() * BLS_MAP_TO_G2_COST_PER_DST_BYTE;
check_cost(cost, max_cost)?;          // only guard
``` [2](#0-1) 

`check_cost` is a simple `cost > max_cost` comparison that returns `EvalErr::CostExceeded` on violation. [3](#0-2) 

Because `Cost` is `u64`, if `msg.len() * BLS_MAP_TO_G2_COST_PER_BYTE` (rate = 4) overflows the 64-bit accumulator, the wrapped value can be smaller than `max_cost`, causing the single surviving `check_cost` to pass. This requires a message atom of roughly `u64::MAX / 4 ≈ 4.6 × 10^18` bytes, which is not achievable in practice given allocator and serialization limits. The more realistic and concrete impact is that execution continues past the point where cost should have been rejected: the DST atom is read from the allocator (a second attacker-controlled atom when `argc == 2`) before the guard fires, meaning the interpreter performs additional work that should have been refused. [4](#0-3) 

---

### Impact Explanation

The broken invariant is: **every significant cost increment must be followed immediately by `check_cost`**. `op_bls_map_to_g2` violates this invariant by batching two cost increments before a single check. In the non-overflow case the final check still fires, so the cost limit is not bypassed — but the interpreter performs extra work (reading the DST atom) that should not have occurred. In the theoretical overflow case the cost limit is bypassed entirely, constituting undercharged execution and a consensus divergence between nodes that enforce cost limits differently.

---

### Likelihood Explanation

Low. The overflow path requires an impractically large atom. The delayed-check path (extra DST atom read before rejection) is reachable with any message atom large enough to exceed `max_cost / 4` bytes, which is bounded by the allocator heap and serialization size limits in practice. The structural inconsistency with `op_bls_map_to_g1` is concrete and the entry path (attacker-crafted CLVM bytes invoking opcode 57) is fully reachable.

---

### Recommendation

Insert the missing `check_cost` call in `op_bls_map_to_g2` immediately after the message-byte cost increment, mirroring `op_bls_map_to_g1`:

```rust
cost += msg.as_ref().len() as Cost * BLS_MAP_TO_G2_COST_PER_BYTE;
check_cost(cost, max_cost)?;   // add this line
``` [5](#0-4) 

---

### Proof of Concept

Craft a CLVM program that invokes opcode 57 (`g2_map`) with a two-argument call where the first argument is a message atom whose byte length satisfies:

```
msg.len() * BLS_MAP_TO_G2_COST_PER_BYTE  >  max_cost - BLS_MAP_TO_G2_BASE_COST
```

With `BLS_MAP_TO_G2_BASE_COST = 815000` and `BLS_MAP_TO_G2_COST_PER_BYTE = 4`, a message of `(max_cost - 815000) / 4 + 1` bytes should trigger rejection. Under the current code the rejection is delayed until after the DST atom (second argument) is also read and its cost added. Under the corrected code the rejection fires immediately after the message cost is accumulated, before the DST atom is touched. [6](#0-5)

pub fn op_bls_map_to_g2(
    a: &mut Allocator,
    input: NodePtr,
    max_cost: Cost,
    _flags: ClvmFlags,
) -> Response {
    let ([msg, dst], argc) = get_varargs::<2>(a, input, "g2_map")?;
    if !(1..=2).contains(&argc) {
        Err(EvalErr::InvalidOpArg(
            input,
            format!("g2_map takes exactly 1 or 2 arguments, got {argc}"),
        ))?;
    }
    let mut cost: Cost = BLS_MAP_TO_G2_BASE_COST;
    check_cost(cost, max_cost)?;

    let msg = atom(a, msg, "g2_map")?;
    cost += msg.as_ref().len() as Cost * BLS_MAP_TO_G2_COST_PER_BYTE;

    let dst = if argc == 2 {
        atom(a, dst, "g2_map")?
    } else {
        Atom::Borrowed(DST_G2.as_slice())
    };

    cost += dst.as_ref().len() as Cost * BLS_MAP_TO_G2_COST_PER_DST_BYTE;
    check_cost(cost, max_cost)?;

    let point = hash_to_g2_with_dst(msg.as_ref(), dst.as_ref());
    Ok(Reduction(
        cost + 96 * MALLOC_COST_PER_BYTE,
        a.new_g2(point)?,
    ))
}
