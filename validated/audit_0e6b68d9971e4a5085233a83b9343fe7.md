Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as Swapper, Allowing Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router — a necessary step for allowlisted users to use the standard periphery — every unprivileged address can bypass the curation gate by routing through the router.

## Finding Description
**Call path:**

1. `MetricOmmSimpleRouter.exactInputSingle` is called by any EOA (e.g., Bob, who is not allowlisted).
2. The router calls `IMetricOmmPoolActions(params.pool).swap(recipient, zeroForOne, amount, priceLimitX64, "", params.extensionData)` — the pool's `msg.sender` is the **router address**, not Bob.
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` passing the router as `sender`.
4. `ExtensionCalling._beforeSwap` dispatches `IMetricOmmExtensions.beforeSwap(sender=router, ...)` to all configured extensions.
5. `SwapAllowlistExtension.beforeSwap` evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. If `allowedSwapper[pool][router] == true`, the check passes for **any caller** regardless of individual allowlist status.

**Root cause:** The allowlist is keyed on the immediate caller of `pool.swap()`. The router is a transparent forwarder — it does not enforce per-user allowlist checks before calling the pool, and the original `msg.sender` (the end user) is never forwarded to the extension.

**Existing guards are insufficient:** The router stores the original payer in transient storage (`_setNextCallbackContext`) for payment purposes only; this value is never passed to the pool's `swap()` call or to any extension. There is no mechanism in the current code for the extension to recover the true initiating user.

## Impact Explanation
A pool admin deploying `SwapAllowlistExtension` intends to restrict trading to a curated set of addresses (e.g., KYC-gated, institutional-only, or partner-restricted). To allow those curated addresses to use the standard periphery, the admin must call `setAllowedToSwap(pool, router, true)`. This single configuration step silently grants universal swap access: any EOA can call `MetricOmmSimpleRouter.exactInputSingle` and the extension will pass because it sees the router address, not the individual user. This is an admin-boundary break — the pool admin's primary access-control mechanism is bypassed by an unprivileged, public path. In pools where the allowlist is the first line of defense against adversarial flow (e.g., paired with a stop-loss extension), bypassing it exposes LP funds to unintended counterparties.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router, which is a natural and expected configuration step — without it, allowlisted users cannot use the standard periphery at all. The admin is likely to make this choice without realizing it grants universal access. Once the router is allowlisted, the bypass is trivially reachable by any EOA with zero additional privileges or capital requirements.

## Recommendation
The allowlist must gate the end user, not the immediate caller. Options:

1. **Forward the original user through the router:** The router passes `msg.sender` as an additional field in `extensionData`; the extension decodes and checks it. Requires coordinated changes to the router and extension.
2. **Router registry with fallback:** If `sender` is a known router, the extension requires the actual user identity to be supplied in `extensionData` and verified (e.g., via a signed permit).
3. **Documentation + separate mechanism:** Document that allowlisting the router is equivalent to `allowAllSwappers = true`, and provide a router that enforces its own allowlist before calling the pool.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Pool admin: setAllowedToSwap(pool, alice, true)   // alice is curated
  - Pool admin: setAllowedToSwap(pool, router, true)  // router allowlisted so alice can use periphery

Attack:
  - Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(recipient, zeroForOne, amount, priceLimitX64, "", extensionData)
    → msg.sender to pool = router address
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension: allowedSwapper[pool][router] == true → check passes
  - Swap executes for Bob despite Bob not being on the allowlist.

Expected: revert NotAllowedToSwap
Actual:   swap succeeds
```

Confirmed code locations:
- `SwapAllowlistExtension.beforeSwap` check at [1](#0-0) 
- `MetricOmmPool.swap` passing `msg.sender` as `sender` to `_beforeSwap` at [2](#0-1) 
- `MetricOmmSimpleRouter.exactInputSingle` calling `pool.swap()` with the router as `msg.sender` at [3](#0-2) 
- Same pattern in multi-hop `exactInput` at [4](#0-3)
