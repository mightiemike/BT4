### Title
Bonus Reward Checkpoints Advance Without Payment When `ErrInsufficientBonusPool` Is Swallowed in `slashRedelegationPosition` — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

`slashRedelegationPosition` calls `processEventsAndClaimBonus` and deliberately swallows `ErrInsufficientBonusPool` to avoid a chain halt. However, `processEventsAndClaimBonus` mutates the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` fields **and** decrements event reference counts in the store **before** the pool-balance check. When the error is swallowed and the mutated position is persisted, those events are permanently skipped on all future claims, and the bonus owed for them is lost forever.

### Finding Description

**Step 1 — Entry point.**
`BeforeRedelegationSlashed` (a staking hook) calls `slashRedelegationPosition`. [1](#0-0) 

**Step 2 — Error is swallowed.**
`slashRedelegationPosition` calls `processEventsAndClaimBonus` and, on `ErrInsufficientBonusPool`, logs the error and continues execution with the already-mutated `pos`. [2](#0-1) 

**Step 3 — Checkpoints are advanced inside the loop, before the pool check.**
Inside `processEventsAndClaimBonus`, for every event in the loop:
- `pos.UpdateLastEventSeq(entry.Seq)` advances `LastEventSeq` in memory.
- `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` **writes to the store**, decrementing (and potentially deleting) the event.

Both happen before the `sufficientBonusPoolBalance` check at line 230. [3](#0-2) 

**Step 4 — `LastBonusAccrual` is also advanced before the pool check.**
`applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` are called at lines 215–217, still before the pool check. [4](#0-3) 

**Step 5 — Pool check fails, function returns error without paying.**
`sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`; `SendCoinsFromModuleToAccount` is never reached. [5](#0-4) [6](#0-5) 

**Step 6 — Mutated position is persisted.**
Back in `slashRedelegationPosition`, after swallowing the error, `setPositionWithState` is called with the mutated `pos`, persisting the advanced `LastEventSeq` and `LastBonusAccrual`. [7](#0-6) 

**Step 7 — Future `claimRewards` skips the consumed events.**
`getValidatorEventsSince` uses `StartExclusive(pos.LastEventSeq)`, so it only returns events with seq > the now-advanced value. The events for which bonus was never paid are permanently skipped. [8](#0-7) 

Additionally, `decrementEventRefCount` may have already deleted those events from the store entirely (when `ReferenceCount` reaches 1), making recovery impossible even if the pool is later refilled. [9](#0-8) 

### Impact Explanation

A position owner permanently loses all bonus rewards accrued up to the moment of a redelegation slash whenever the `RewardsPool` module account balance is zero at that instant. The lost amount equals the full bonus computed for the elapsed bonded segments since the position's last claim. This is a direct, irreversible economic loss to the position owner with no recovery path.

### Likelihood Explanation

The `RewardsPool` can legitimately be empty or near-empty between funding cycles. A redelegation slash is a normal on-chain event triggered by validator misbehavior. The two conditions can coincide without any attacker action — a validator double-sign while the pool is temporarily empty is sufficient. An adversary who controls a validator can also deliberately time the slash.

### Recommendation

`processEventsAndClaimBonus` must not advance any position checkpoints or decrement event reference counts until after the payment is confirmed to succeed. The fix is to perform the pool-balance check (and the `SendCoinsFromModuleToAccount` call) **before** mutating `pos.LastEventSeq`, `pos.LastBonusAccrual`, and `pos.LastKnownBonded`, and before calling `decrementEventRefCount`. Alternatively, `slashRedelegationPosition` should not persist the position when `ErrInsufficientBonusPool` is returned — it should leave all checkpoints at their pre-call values so the bonus can be claimed later.

### Proof of Concept

```
1. Create a tier position delegated to validator V.
2. Append several validator events (e.g., SLASH events) so pending bonus accrues.
3. Drain the RewardsPool to zero.
4. Trigger BeforeRedelegationSlashed for the position's unbonding ID.
   → slashRedelegationPosition calls processEventsAndClaimBonus.
   → Loop advances pos.LastEventSeq and decrements event ref counts.
   → applyBonusAccrualCheckpoint advances pos.LastBonusAccrual.
   → sufficientBonusPoolBalance returns ErrInsufficientBonusPool.
   → Error is swallowed; setPositionWithState persists the advanced pos.
5. Refill the RewardsPool with sufficient funds.
6. Call MsgClaimRewards for the position.
   → processEventsAndClaimBonus starts from the advanced LastEventSeq.
   → No events are returned (all already consumed).
   → Zero bonus is paid.
7. Assert: total bonus received == 0, expected > 0. Test fails, confirming the loss.
```

```go
func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
```

```go
func (k Keeper) getValidatorEventsSince(ctx context.Context, valAddr sdk.ValAddress, startSeq uint64) ([]EventEntry, error) {
	// Range from (valAddr, startSeq+1) to end of valAddr prefix.
	rng := collections.NewPrefixedPairRange[sdk.ValAddress, uint64](valAddr).
		StartExclusive(startSeq)

	var entries []EventEntry
	err := k.ValidatorEvents.Walk(ctx, rng, func(key collections.Pair[sdk.ValAddress, uint64], event types.ValidatorEvent) (bool, error) {
		entries = append(entries, EventEntry{Seq: key.K2(), Event: event})
		return false, nil
	})
	if err != nil {
		return nil, err
	}
	return entries, nil
}
```

```go
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}
```
