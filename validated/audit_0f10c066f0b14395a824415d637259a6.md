Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User, Allowing Any User to Bypass Per-Pool Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument against `allowedSwapper[pool][sender]`. The pool always passes `msg.sender` of `pool.swap()` as `sender`. When `MetricOmmSimpleRouter` is the direct caller of `pool.swap()`, the extension sees `sender = router`, not the end-user. If the pool admin allowlists the router — a natural operational decision to "permit all router users" — every non-allowlisted address can bypass the swap gate by routing through the router. If the router is not allowlisted, allowlisted users are blocked from using the router entirely.

## Finding Description

**Root cause — pool passes direct caller, not economic actor:**

In `MetricOmmPool.swap()`, `msg.sender` (the direct caller) is forwarded as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // always the direct caller of pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap()` encodes this value and passes it as the first argument to `IMetricOmmExtensions.beforeSwap()`.

**Extension checks the wrong identity:**

`SwapAllowlistExtension.beforeSwap()` uses `msg.sender` (the pool) as the pool key and `sender` (the direct caller of `pool.swap()`) as the swapper key:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

**Router is the direct caller of `pool.swap()`:**

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router `msg.sender` of that call:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The actual end-user (`msg.sender` of `exactInputSingle`) is stored only in transient callback context for payment purposes — it is never forwarded to the pool or extension as the swapper identity.

**Secondary issue — missing `onlyPool` in override:**

`BaseMetricExtension.beforeSwap()` carries the `onlyPool` modifier, but `SwapAllowlistExtension` overrides it without that modifier. The extension implicitly relies on `msg.sender` being a registered pool for the `allowedSwapper[msg.sender]` lookup to be meaningful. Without `onlyPool`, any address can call `beforeSwap` directly; however, since the mapping key would be the arbitrary caller's own address (not a pool), this does not produce a standalone bypass — the router bypass is the primary exploitable path.

**Broken invariant — no valid middle ground:**

| Router allowlisted? | Effect |
|---|---|
| Yes | Any non-allowlisted user bypasses the gate via the router |
| No | Every allowlisted user is blocked from using the router |

## Impact Explanation

A pool admin who deploys `SwapAllowlistExtension` to restrict swaps to KYC'd addresses, specific counterparties, or whitelisted market makers cannot achieve that goal if the router is allowlisted. Any address — including adversarial arbitrageurs — can call `router.exactInputSingle()` and the extension will see `sender = router`, which passes the check. LP providers in the restricted pool suffer adverse selection from actors the pool was explicitly designed to exclude. The pool's core access-control invariant — that only allowlisted addresses may swap — is broken by a public, unprivileged path through the official periphery router. This constitutes broken core pool functionality causing potential loss of LP assets through adverse selection.

## Likelihood Explanation

Pool admins who want to allow "all users going through the official router" will naturally add the router to the allowlist. The router is a trusted, public periphery contract and allowlisting it is a reasonable operational decision. Nothing in the extension's interface, NatSpec, `setAllowedToSwap`, or error messages warns that allowlisting the router opens the pool to everyone. The misconfiguration is therefore likely in any deployment that combines `SwapAllowlistExtension` with `MetricOmmSimpleRouter`. The exploit requires no special privileges — any EOA can call `router.exactInputSingle()`.

## Recommendation

Pass the economic actor — not the direct `msg.sender` — through the hook. Two options:

1. **Preferred**: Add a `swapper` field to `extensionData` that the router populates with `msg.sender` before calling the pool. Have the extension decode and verify this field against the allowlist, with the pool verifying the router is a trusted forwarder (e.g., via a factory-registered router registry).

2. **Simpler**: Add an on-chain guard in `setAllowedToSwap` that rejects the router address, and document that the router must never be allowlisted. This prevents the bypass but still leaves allowlisted users unable to use the router.

Additionally, restore the `onlyPool` modifier on `SwapAllowlistExtension.beforeSwap()` to match the base contract's intent.

## Proof of Concept

```solidity
// 1. Deploy pool with SwapAllowlistExtension
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Pool admin allowlists the router so "all router users" can swap
ext.setAllowedToSwap(pool, address(router), true);
// alice is NOT on the allowlist; bob IS on the allowlist

// 2. Alice bypasses the allowlist via the router
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: alice,
    zeroForOne: true,
    amountIn: 1e18,
    ...
}));
// pool.swap() called with msg.sender = router
// _beforeSwap(sender=router, ...)
// extension: allowedSwapper[pool][router] = true → passes
// Alice swaps successfully despite not being allowlisted

// 3. Verify: alice calling pool.swap() directly is blocked
pool.swap(alice, true, 1e18, ...);
// _beforeSwap(sender=alice, ...)
// extension: allowedSwapper[pool][alice] = false → NotAllowedToSwap revert
```
