### Title
Silent Truncating Cast of `ShardId` (u64) to `u16` Corrupts `CongestionInfoV1.allowed_shard` in Borsh-Serialized ChunkHeader — (`core/primitives/src/congestion_info.rs`, `core/primitives-core/src/types.rs`)

---

### Summary

`CongestionInfoV1.allowed_shard` is a `u16` field Borsh-serialized inside every `ChunkHeader`. The field is written by two production code paths — `finalize_allowed_shard` and `bootstrap_congestion_info` — both of which convert a `ShardId` (internally `u64`) to `u16` via the `Into<u16> for ShardId` impl, which performs an unchecked `self.0 as u16` truncation. If any live shard carries an ID whose lower 16 bits differ from the full 64-bit value (i.e., `shard_id > u16::MAX`), the wrong shard identity is silently written into the protocol field, breaking the congestion-control "allowed shard" invariant for every node that processes that chunk.

---

### Finding Description

`ShardId` is a newtype over `u64`:

```rust
pub struct ShardId(u64);
```

The `Into<u16>` conversion is a bare truncating cast with no overflow check:

```rust
impl Into<u16> for ShardId {
    fn into(self) -> u16 {
        self.0 as u16   // silent truncation if self.0 > 65535
    }
}
```

This conversion is invoked in two production paths that write into the Borsh-serialized protocol struct `CongestionInfoV1`:

**Path 1 — `finalize_allowed_shard`** (called every chunk to update the congestion header):

```rust
pub fn finalize_allowed_shard(
    &mut self,
    own_shard: ShardId,
    all_shards: &[ShardId],
    congestion_seed: u64,
) {
    let allowed_shard = Self::get_new_allowed_shard(own_shard, all_shards, congestion_seed);
    self.set_allowed_shard(allowed_shard.into()); // ShardId → u16, silent truncation
}
```

**Path 2 — `bootstrap_congestion_info`** (called during state sync / resharding bootstrap):

```rust
Ok(CongestionInfo::V1(CongestionInfoV1 {
    ...
    allowed_shard: shard_id.into(), // ShardId → u16, silent truncation
}))
```

The destination field is:

```rust
pub struct CongestionInfoV1 {
    pub delayed_receipts_gas: u128,
    pub buffered_receipts_gas: u128,
    pub receipt_bytes: u64,
    pub allowed_shard: u16,   // ← narrower than ShardId
}
```

`CongestionInfoV1` is `BorshSerialize + BorshDeserialize + ProtocolSchema` and is embedded in `ShardChunkHeaderInner` (V4 and V5), making it a canonical protocol field whose exact bytes are committed to by every validator.

The truncated value is later read back and compared against full `ShardId` values:

```rust
pub fn outgoing_gas_limit(&self, sender_shard: ShardId) -> Gas {
    ...
    if sender_shard == ShardId::from(self.info.allowed_shard()) {
        self.config.allowed_shard_outgoing_gas
    } else {
        Gas::ZERO
    }
}
```

`ShardId::from(u16)` widens back to `u64`, so if the stored `u16` was a truncated version of the real shard ID, the comparison will never match the intended shard.

---

### Impact Explanation

When a shard ID exceeds `u16::MAX` (65535), `finalize_allowed_shard` silently writes a wrong value into `CongestionInfoV1.allowed_shard` in the chunk header. Every validator independently computes the same wrong value (the truncation is deterministic), so consensus is not broken — but the congestion-control semantic is:

- In a fully-congested state, the "allowed shard" slot is assigned to a shard that does not exist or is the wrong shard. No shard matches the stored `allowed_shard`, so `outgoing_gas_limit` returns `Gas::ZERO` for every sender, including the one that should be allowed to make progress.
- This is a **liveness failure**: the congestion-control mechanism can no longer drain the delayed-receipt queue, permanently stalling the affected shard.
- The corrupted value is Borsh-serialized into the `ChunkHeader` and propagated to all peers, so the damage is network-wide and persistent across epochs.

**Scope:** Protocol-level congestion control correctness; shard liveness.

---

### Likelihood Explanation

`ShardId` is `u64` and the `ShardLayoutV2::derive` function allocates new shard IDs by incrementing from the current maximum:

```rust
let new_shards = vec![max_shard_id + 1, max_shard_id + 2];
```

Reaching `shard_id > 65535` requires ~32 768 resharding events, which is not realistic in the near term. However:

1. The `Into<u16>` impl is unconditional and carries no `debug_assert` or `TODO` guard.
2. The `ShardId` type is explicitly documented as a `u64` that can hold non-contiguous, non-sequential values.
3. No protocol-version gate or range check prevents a future shard layout from assigning large IDs.
4. The `Into<u32>` impl (`self.0 as u32`) used in `ShardUId::new` has the same pattern and a wider blast radius (DB key corruption for shard IDs > `u32::MAX`).

Likelihood is **Low** today but the invariant is structurally broken and will silently activate as resharding continues.

---

### Recommendation

Replace the unchecked `as` casts in both `Into<u16>` and `Into<u32>` for `ShardId` with checked conversions, and propagate errors at the two write sites:

```rust
// core/primitives-core/src/types.rs
// Remove or deprecate Into<u16> and Into<u32>; replace with TryInto.
impl TryFrom<ShardId> for u16 {
    type Error = TryFromIntError;
    fn try_from(id: ShardId) -> Result<Self, Self::Error> {
        u16::try_from(id.0)
    }
}
```

In `finalize_allowed_shard`, return an error or panic loudly if the shard ID does not fit:

```rust
let allowed_shard_u16 = u16::try_from(allowed_shard)
    .expect("ShardId exceeds u16::MAX; CongestionInfoV1.allowed_shard field must be widened");
self.set_allowed_shard(allowed_shard_u16);
```

Longer term, widen `CongestionInfoV1.allowed_shard` from `u16` to `u64` in a new `CongestionInfoV2` variant (the enum is already versioned for exactly this purpose), and gate the new variant behind a `ProtocolFeature`.

---

### Proof of Concept

```rust
use near_primitives_core::types::ShardId;

fn main() {
    // Simulate a shard ID that exceeds u16::MAX after many resharding rounds
    let shard_id = ShardId::new(0x0001_0001u64); // 65537, lower 16 bits = 1

    // This is what finalize_allowed_shard does today:
    let stored: u16 = shard_id.into(); // → 1, silently truncated from 65537

    // Reconstruction in outgoing_gas_limit:
    let reconstructed = ShardId::from(stored); // → ShardId(1)

    assert_ne!(shard_id, reconstructed);
    // shard_id = 65537, reconstructed = 1
    // The "allowed shard" comparison will never match shard 65537.
    // outgoing_gas_limit returns Gas::ZERO for every sender → liveness stall.
    println!("original={}, stored_u16={}, reconstructed={}", shard_id, stored, reconstructed);
}
