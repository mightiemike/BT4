Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, `sender` resolves to the router contract address, not the end user. If a pool admin allowlists the router to enable router-mediated swaps for permitted users, every unpermissioned user can bypass the allowlist by routing through the same public router contract.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` of that call — the original end-user's address is never forwarded to the pool: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) [6](#0-5) [7](#0-6) 

The extension has no mechanism to decode the real user from `extensionData` — the `bytes calldata` parameter is accepted but entirely ignored in `beforeSwap`: [3](#0-2) 

The pool admin faces an impossible choice: do not allowlist the router (permitted users cannot use the router at all), or allowlist the router (every unpermissioned user can swap through the router, completely bypassing the allowlist). The `DepositAllowlistExtension` does not share this flaw because it correctly gates `owner` (the position owner passed explicitly), not `sender`: [8](#0-7) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (e.g., KYC-verified counterparties, institutional LPs, or protocol-internal actors) can be fully bypassed by any unpermissioned user who calls any router entry point. The attacker receives oracle-priced execution on a pool intended to be private, draining LP value. This is a direct, unprivileged bypass of the pool's primary access-control guard with fund-impacting consequences for LPs — matching the "admin-boundary break bypassed by an unprivileged path" and "broken core pool functionality causing loss of funds" impact criteria.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless contract. Any user who observes that a pool has a swap allowlist can immediately attempt to route through the router. No special privileges, flash loans, or multi-step setup are required. The bypass is reachable in a single transaction and is repeatable indefinitely.

## Recommendation
`SwapAllowlistExtension` must gate the economically relevant actor, not the immediate caller of `pool.swap()`. The preferred fix is to have the router encode `msg.sender` into `extensionData` and have the extension decode and check it. Alternatively, gate at the router level by deploying a router wrapper that enforces the allowlist before forwarding to the pool, and only allowlist that wrapper. The pool-level extension then only needs to verify the wrapper is the caller.

## Proof of Concept
```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension as beforeSwap hook
  admin allowlists alice directly: allowedSwapper[pool][alice] = true
  admin also allowlists the router so alice can use it: allowedSwapper[pool][router] = true

Attack (bob, not allowlisted):
  bob calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(recipient, ...) → msg.sender = router
  pool calls _beforeSwap(sender=router, ...)
  extension checks allowedSwapper[pool][router] → true  ✓ (passes)
  bob's swap executes on the restricted pool

Result:
  bob bypasses the allowlist entirely by routing through the public router.
  The extension never sees bob's address.
```
