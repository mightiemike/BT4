Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End User, Allowing Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which equals the pool's `msg.sender` at the time of the `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The allowlist check therefore evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. Any pool that allowlists the router to support legitimate router-mediated swaps simultaneously opens the gate to every caller regardless of their individual allowlist status.

## Finding Description
`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct), and `sender` is the value forwarded by `ExtensionCalling._beforeSwap`:

```solidity
// metric-core/contracts/ExtensionCalling.sol L149-L177
function _beforeSwap(
    address sender,   // ← pool's msg.sender at swap() call time
    ...
) internal {
    _callExtensionsInOrder(
        BEFORE_SWAP_ORDER,
        abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
    );
}
```

When `MetricOmmSimpleRouter.exactInputSingle` is called, it calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-L80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
    );
```

The router is `msg.sender` to the pool, so the pool passes `sender = address(router)` into `_beforeSwap`, and the extension checks `allowedSwapper[pool][router]` — never the actual end user.

**Two broken outcomes:**

| Router allowlist status | Effect |
|---|---|
| Router **is** allowlisted | Every caller passes the check — allowlist fully bypassed |
| Router **is not** allowlisted | Allowlisted users cannot use the router — core swap path broken |

## Impact Explanation
This is a direct admin-boundary break. A pool deployer uses `SwapAllowlistExtension` to restrict swaps to KYC'd or otherwise vetted addresses. To allow those users to benefit from the router (slippage protection, multi-hop), the admin must allowlist the router. At that point the allowlist provides zero protection: any address can call the router and execute swaps on the restricted pool. LP funds are exposed to unrestricted trading, defeating the entire purpose of the permissioned pool design.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing entry point. Any pool deploying `SwapAllowlistExtension` that also wants router support must allowlist the router, automatically triggering the bypass. The attacker requires no special role, no privileged setup, and no non-standard token — only a public router call to `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`.

## Recommendation
The pool should pass the original end-user address as `sender` to extensions, not its own `msg.sender`. Two standard approaches:

1. **Explicit sender parameter**: Add `address sender` to `pool.swap(address sender, address recipient, ...)` so the router can forward `msg.sender` (the actual user). The pool uses this explicit value when calling `_beforeSwap`.
2. **Callback-derived identity**: Resolve the true sender inside the swap callback (similar to Uniswap v4's `hookData` pattern), where the router attests the real caller via signed or encoded data.

`SwapAllowlistExtension.beforeSwap` should then check the attested end-user address, not the intermediary router.

## Proof of Concept

```
Setup:
  pool P with SwapAllowlistExtension E
  allowedSwapper[P][router] = true   // admin must do this to allow router swaps
  allowedSwapper[P][alice]  = true   // alice is KYC'd
  allowedSwapper[P][bob]    = false  // bob is NOT allowlisted

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})
  → router calls P.swap(recipient=bob, ...)          // router is pool's msg.sender
  → pool passes sender = address(router) to _beforeSwap
  → extension checks allowedSwapper[P][router] == true  ✓
  → swap executes — bob bypasses the allowlist entirely
