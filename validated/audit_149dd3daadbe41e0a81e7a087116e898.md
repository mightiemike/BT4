Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Full Allowlist Bypass - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` — the immediate caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks the router's allowlist entry rather than the end user's. Any pool admin who allowlists the router (the required step to let their allowlisted users trade through the standard periphery) inadvertently opens the pool to every address in the system.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool (namespace key) and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router `msg.sender` to the pool: [4](#0-3) 

The same applies to `exactOutputSingle` (L136-137), `exactInput` (L104-112), `exactOutput` (L165-181), and the recursive `_exactOutputIterateCallback` path (L220-228): [5](#0-4) 

The extension therefore evaluates `allowedSwapper[pool][router]` for every user who routes through the router. If the pool admin has allowlisted the router — the necessary operational step to let their allowlisted users trade through the standard UX — the check passes for every caller regardless of their individual allowlist status. The `isAllowedToSwap` view function returns `false` for non-allowlisted addresses, giving a false sense of security: [6](#0-5) 

`DepositAllowlistExtension` does not share this flaw because it checks `owner` (the economic beneficiary of the position), which is passed correctly regardless of who calls `addLiquidity`: [7](#0-6) 

## Impact Explanation
A pool with `SwapAllowlistExtension` is designed to restrict trading to specific counterparties (e.g., KYC-verified addresses, institutional traders, or protocol-controlled addresses). Once the pool admin allowlists the router, the allowlist is fully bypassed for every address. Unauthorized users can trade on a pool whose LP positions were sized and priced assuming only specific, trusted counterparties, causing direct LP principal loss through adverse selection or arbitrage that the allowlist was intended to prevent. This constitutes an admin-boundary break and direct loss of LP assets.

## Likelihood Explanation
Likelihood is high. `MetricOmmSimpleRouter` is the primary supported swap interface; pool admins are expected to allowlist it so their allowlisted users can trade through the standard UX. The bypass requires no special privileges, no flash loans, and no multi-step setup — any address calls `exactInputSingle` on the router pointing at the curated pool. The pool admin has no on-chain signal that allowlisting the router opens the gate to all users.

## Recommendation
The extension must resolve the actual end user, not the immediate pool caller. Two sound approaches:

1. **Pass the original `msg.sender` through the router.** Add a `swapper` field to the router's swap parameters and forward it inside `extensionData`; the extension decodes it and verifies it against the allowlist. The pool's reentrancy guard and callback verification already authenticate the router, so the decoded address can be trusted within that call.

2. **Require the real user to be passed via `extensionData` and check that address.** The extension receives both `sender` (router) and the decoded real user from `extensionData`, then checks the real user against `allowedSwapper`. This keeps the extension stateless with respect to router identity.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension.
  - Pool admin allowlists Alice (0xAlice) and the router (0xRouter):
      setAllowedToSwap(pool, 0xAlice, true)
      setAllowedToSwap(pool, 0xRouter, true)   ← required for Alice to use the router
  - Eve (0xEve) is NOT allowlisted.

Attack:
  1. Eve calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
  2. Router calls pool.swap(recipient, ...) — msg.sender to pool = 0xRouter.
  3. Pool calls _beforeSwap(sender=0xRouter, ...).
  4. SwapAllowlistExtension checks allowedSwapper[pool][0xRouter] → true.
  5. Swap executes. Eve has bypassed the allowlist.

Direct call (control):
  1. Eve calls pool.swap(...) directly.
  2. Pool calls _beforeSwap(sender=0xEve, ...).
  3. SwapAllowlistExtension checks allowedSwapper[pool][0xEve] → false → revert NotAllowedToSwap.
```
