### Title
`ValidResourceBounds` protobuf round-trip silently downgrades `AllResources` to `L1Gas`, producing a divergent transaction hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a **value-based heuristic** — checking whether `l2_gas` and `l1_data_gas` are zero — to decide which enum variant to reconstruct. A gateway-accepted `AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}` transaction serializes to protobuf with those zero fields, then deserializes back as `L1Gas(X)`. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` packed felt only for `AllResources`, the two variants produce **different Poseidon hashes** over identical numeric values. Any node that receives the transaction via P2P block sync computes a hash that diverges from the hash committed by the originating sequencer.

---

### Finding Description

**Root cause — protobuf deserialization heuristic:** [1](#0-0) 

```rust
// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant
} else {
    ValidResourceBounds::AllResources(...)
})
```

The serializer always emits all three fields for both variants: [2](#0-1) 

So `AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}` serializes to `{l1_gas=X, l2_gas=0, l1_data_gas=0}` and deserializes back as `L1Gas(X)` — a silent type change.

**Hash divergence — `get_tip_resource_bounds_hash`:** [3](#0-2) 

For `L1Gas(X)` the hash chain is `poseidon(tip, concat(L1_GAS,X), concat(L2_GAS,0))` — two resource felts.  
For `AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}` it is `poseidon(tip, concat(L1_GAS,X), concat(L2_GAS,0), concat(L1_DATA_GAS,0))` — three resource felts.

`concat(L1_DATA_GAS, 0, 0)` is **not zero**: the resource-name bytes `b"L1_DATA"` occupy bits 56–112 of the packed felt: [4](#0-3) 

So the two hashes are distinct under Poseidon.

**Gateway accepts the triggering transaction:** [5](#0-4) 

`AllResourceBounds{l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0}` is a valid, accepted transaction.

**Conversion path that locks in `AllResources`:**

When the RPC transaction is ingested, `AllResourceBounds` is always wrapped as `ValidResourceBounds::AllResources`: [6](#0-5) 

The hash is computed from this `AllResources` variant at the gateway: [7](#0-6) 

The stored `InvokeTransactionV3` therefore carries `ValidResourceBounds::AllResources` and the hash H1 that includes the `L1_DATA_GAS` term. The P2P sync path serializes this to protobuf and the receiving node reconstructs `L1Gas`, computing H2 ≠ H1.

**Contrast with the correct JSON deserialization path:**

The JSON/storage path correctly preserves the variant by checking **key presence**, not value: [8](#0-7) 

The protobuf path lacks this distinction.

---

### Impact Explanation

**Impact: High — Transaction conversion binds the wrong hash.**

Any node that receives a block containing an `InvokeV3` with `AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}` via P2P protobuf sync will reconstruct the transaction as `L1Gas(X)`. When it recomputes the transaction hash for block validation, it obtains H2 ≠ H1 (the hash committed by the sequencer). This causes the block to be rejected by the syncing node, creating a persistent chain-split between the sequencer and all syncing peers for any block containing such a transaction.

Additionally, any RPC endpoint that re-derives the transaction hash from the stored (now-downgraded) `L1Gas` representation will return an authoritative-looking but wrong hash to callers.

---

### Likelihood Explanation

The triggering condition — an `InvokeV3` with only `l1_gas` set — is the **most common pre-0.13.3 transaction pattern** and is explicitly accepted by the gateway's stateless validator. No special privileges are required. Any user submitting a standard L1-gas-only V3 invoke triggers the divergence.

---

### Recommendation

Remove the value-based heuristic. The protobuf wire format should carry an explicit discriminant (e.g., a boolean `is_l1_gas_only` field, or a `oneof` wrapper) so the deserializer can reconstruct the exact variant without inspecting numeric values. Until the wire format is updated, the deserializer should default to `AllResources` whenever all three fields are present in the message, matching the behavior of the JSON path which treats presence of the `L1DataGas` key — not its value — as the discriminant.

---

### Proof of Concept

1. Submit `InvokeV3` with `resource_bounds = AllResourceBounds{l1_gas: GasAmount(1000)/GasPrice(1), l2_gas: 0, l1_data_gas: 0}`.
2. Gateway accepts it. `convert_rpc_tx_to_internal` wraps it as `ValidResourceBounds::AllResources(...)` and computes hash **H1** = `poseidon(INVOKE, 3, sender, poseidon(tip, concat(L1_GAS,1000,1), concat(L2_GAS,0,0), concat(L1_DATA_GAS,0,0)), ...)`.
3. Transaction is included in a block; H1 is recorded in the block's transaction list.
4. A syncing peer receives the block via P2P. The `InvokeTransactionV3` is deserialized from protobuf: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas(GasAmount(1000)/GasPrice(1))`.
5. Peer computes hash **H2** = `poseidon(INVOKE, 3, sender, poseidon(tip, concat(L1_GAS,1000,1), concat(L2_GAS,0,0)), ...)` — missing the `concat(L1_DATA_GAS,0,0)` element.
6. H1 ≠ H2. Block hash validation fails; the peer rejects the block.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

fn get_concat_resource(
    resource_bounds: &ResourceBounds,
    resource_name: &ResourceName,
) -> Result<Felt, StarknetApiError> {
    let max_amount = resource_bounds.max_amount.0.to_be_bytes();
    let max_price = resource_bounds.max_price_per_unit.0.to_be_bytes();
    let concat_bytes =
        [[0_u8].as_slice(), resource_name.as_slice(), max_amount.as_slice(), max_price.as_slice()]
            .concat();
    Ok(Felt::from_bytes_be(&concat_bytes.try_into().expect("Expect 32 bytes")))
}
