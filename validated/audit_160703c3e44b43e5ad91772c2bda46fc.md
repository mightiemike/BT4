Audit Report

## Title
Pool Admin Can Reduce Timelock to Zero Then Immediately Disable Stop-Loss Protection — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary
`OracleValueStopLossExtension` uses a timelock to give LPs a reaction window before parameter changes take effect, but the timelock value itself has no minimum floor and can be reduced to zero via the same timelocked flow. Once the timelock is zero, `_afterTimelock` returns exactly `block.timestamp`, allowing a pool admin to propose and execute `drawdown=0` in the same block with zero LP warning. This sets `drawdownE6 = 0`, which triggers an unconditional early return in `_afterSwapOracleStopLoss`, permanently disabling all stop-loss checks.

## Finding Description

**Phase 1 — Reduce timelock to zero:**

`proposeOracleStopLossTimelock` accepts any `uint32 newTimelock` with no minimum validation. [1](#0-0) 

After waiting the current timelock, `executeOracleStopLossTimelock` writes `oracleStopLossConfig[pool_].timelock = 0` with no floor check. [2](#0-1) 

**Phase 2 — Immediately disable drawdown:**

With `timelock = 0`, `_afterTimelock` returns exactly `block.timestamp`: [3](#0-2) 

`proposeOracleStopLossDrawdown(pool, 0)` passes `_validateDrawdown` because it only rejects values `> E6`, not `== 0`: [4](#0-3) 

This sets `pendingDrawdownExecuteAfter = block.timestamp`. `executeOracleStopLossDrawdown` is then called in the same block. `_requireElapsed` checks `block.timestamp < executeAfter`, which evaluates to `block.timestamp < block.timestamp` — false — so it does not revert: [5](#0-4) 

`drawdownE6` is written to 0. [6](#0-5) 

**Phase 3 — Stop-loss silently disabled:**

Every subsequent swap hits the early-return guard at the top of `_afterSwapOracleStopLoss`: [7](#0-6) 

All watermark breach checks are bypassed; swaps that would have reverted with `OracleStopLossTriggered` now execute freely.

## Impact Explanation
The stop-loss extension is the sole on-chain mechanism preventing per-bin value-per-share from falling below the configured floor. Disabling it allows value-draining swaps at unfavorable oracle prices to proceed without any revert, constituting direct LP principal loss. This is an admin-boundary break: the pool admin bypasses the LP protection window that the timelock is explicitly designed to enforce, exceeding the semi-trusted admin role boundary defined by the contest rules.

## Likelihood Explanation
Requires the pool admin to act maliciously or be compromised. Once the admin key is controlled, the two-phase sequence is fully permissionless — no factory owner, oracle admin, or external dependency is needed. The only cost is one timelock wait period (which LPs observe but cannot prevent if they do not exit in time). Likelihood is **Medium**.

## Recommendation
1. Add a `MIN_TIMELOCK` constant (e.g., `1 days`) and reject proposals below it in `proposeOracleStopLossTimelock`.
2. Alternatively, enforce a one-way ratchet: require `newTimelock >= oracleStopLossConfig[pool_].timelock` so the protection window can only grow.
3. Reject `drawdownE6 == 0` in `_validateDrawdown` (treat zero as "no protection" and require an explicit factory-level opt-out).

## Proof of Concept
```solidity
// 1. Initialize pool with 1-day timelock and 5% drawdown
extension.initialize(pool, abi.encode(uint32(50_000), uint32(58), uint32(86400)));

// 2. Pool admin proposes timelock=0; executeAfter = now + 86400
vm.prank(admin);
extension.proposeOracleStopLossTimelock(pool, 0);

// 3. Wait the current timelock
vm.warp(block.timestamp + 86400);

// 4. Execute — timelock is now 0
vm.prank(admin);
extension.executeOracleStopLossTimelock(pool);

// 5. Propose drawdown=0; executeAfter = block.timestamp (timelock=0)
vm.prank(admin);
extension.proposeOracleStopLossDrawdown(pool, 0);

// 6. Execute immediately in same block — drawdown is now 0
vm.prank(admin);
extension.executeOracleStopLossDrawdown(pool);

// 7. Confirm stop-loss is disabled
(uint32 dd,,,) = extension.oracleStopLossConfig(pool);
assertEq(dd, 0); // stop-loss silently bypassed; all swaps pass unchecked
```

  function executeOracleStopLossTimelock(address pool_) external onlyPoolAdmin(pool_) {
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    if (sched.pendingTimelockExecuteAfter == 0) revert OracleStopLossNoPendingTimelock(pool_);
    _requireElapsed(sched.pendingTimelockExecuteAfter);
    uint32 timelock = sched.pendingTimelock;
    oracleStopLossConfig[pool_].timelock = timelock;
    (sched.pendingTimelock, sched.pendingTimelockExecuteAfter) = (0, 0);
    emit OracleStopLossTimelockSet(pool_, timelock);
  }
