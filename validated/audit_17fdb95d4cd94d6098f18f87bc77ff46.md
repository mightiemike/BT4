### Title
`gas_prices_to_hash()` Omits `l2_gas_consumed` and `next_l2_gas_price` from Block Hash for Starknet Versions ≥ 0.14.0, Producing Wrong Committed Block Hash — (`File: crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

The `gas_prices_to_hash()` function and the `PartialBlockHashComponents` struct do not include `l2_gas_consumed` or `next_l2_gas_price` in the block hash preimage for any Starknet version, including V0_14.x. These two fee-market fields are present in every stored block header and are required by the V0_14.x protocol hash specification (as confirmed by the `echonet` CLI path and an explicit in-code TODO). Because `BlockHashVersion` has only two variants (`V0_13_2`, `V0_13_4`) and all V0_14.x versions collapse to `V0_13_4`, the sequencer permanently uses the pre-fee-market formula, producing a block hash that does not commit to the fee-market state. This is the direct analog of the external report's "in-flight assets not counted in `totalAssets()`": fee-market data that has "arrived" in the block header is silently excluded from the hash domain.

---

### Finding Description

**Version gate collapses all V0_14.x to V0_13_4**

`BlockHashVersion` is an enum with exactly two variants:

```rust
pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}
```

The `TryFrom<StarknetVersion>` implementation maps every version ≥ V0_13_4 — including all V0_14.x — to `V0_13_4`:

```rust
} else {
    Ok(Self::V0_13_4)   // V0_14.0, V0_14.1, … all land here
}
```

**`gas_prices_to_hash()` has no V0_14 branch**

The function has a single version gate at `V0_13_4`. For all V0_14.x blocks it executes the `V0_13_4` arm, which hashes only the six gas-price fields and omits `l2_gas_consumed` and `next_l2_gas_price`:

```rust
// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
pub fn gas_prices_to_hash(...) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())
                .chain(&l1_gas_price.price_in_fri.0.into())
                .chain(&l1_data_gas_price.price_in_wei.0.into())
                .chain(&l1_data_gas_price.price_in_fri.0.into())
                .chain(&l2_gas_price.price_in_wei.0.into())
                .chain(&l2_gas_price.price_in_fri.0.into())
                .get_poseidon_hash(),
        ]
    } else { ... }
}
```

**`PartialBlockHashComponents` does not carry the missing fields**

The struct fed into `calculate_block_hash()` has no `l2_gas_consumed` or `next_l2_gas_price` fields at all:

```rust
pub struct PartialBlockHashComponents {
    pub header_commitments: BlockHeaderCommitments,
    pub block_number: BlockNumber,
    pub l1_gas_price: GasPricePerToken,
    pub l1_data_gas_price: GasPricePerToken,
    pub l2_gas_price: GasPricePerToken,
    pub sequencer: SequencerContractAddress,
    pub timestamp: BlockTimestamp,
    pub starknet_version: StarknetVersion,
}
```

Yet `BlockHeaderWithoutHash` and `StorageBlockHeader` both carry these fields:

```rust
pub struct BlockHeaderWithoutHash {
    ...
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,
    ...
}
```

**The `echonet` CLI path confirms the correct formula includes these fields**

The `echonet/echo_center.py` `_compute_block_hash()` method passes both fields to the block-hash CLI and then compares the result against mainnet:

```python
"l2_gas_consumed": fee_market_info["l2_gas_consumed"],
"next_l2_gas_price": fee_market_info["next_l2_gas_price"],
```

The comparison function logs a mismatch when the sequencer-computed hash diverges from mainnet, confirming the CLI formula is canonical.

---

### Impact Explanation

Every V0_14.x block produced or synced by the sequencer receives a block hash that does not commit to `l2_gas_consumed` or `next_l2_gas_price`. Concretely:

1. **Wrong value returned by RPC**: `starknet_getBlockWithTxHashes`, `starknet_getBlockWithTxs`, `starknet_getBlockWithReceipts`, and `starknet_getStateUpdate` all return the stored block hash. For any V0_14.x block this hash is computed with the wrong formula and will not match the hash an independent verifier computes using the correct V0_14.x specification.

2. **Hash does not bind fee-market state**: Two blocks that differ only in `l2_gas_consumed` or `next_l2_gas_price` produce identical block hashes. The fee-market fields that drive EIP-1559 L2 gas pricing are outside the hash domain, so the hash provides no integrity guarantee over them.

3. **`PartialBlockHash` used in consensus is also wrong**: `PartialBlockHash::from_partial_block_hash_components()` calls `calculate_block_hash()` with the same incomplete `PartialBlockHashComponents`, so the partial hash committed during consensus also omits these fields.

The matching allowed impact is: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

The sequencer is already producing V0_14.x blocks (test fixtures `block_post_0_14_0.json`, `block_post_0_14_2.json`, `block_post_0_14_3.json`, `block_post_0_14_4.json` all exist in the repo). The `BlockHashVersion` enum has no V0_14 variant and the TODO has not been resolved. Every V0_14.x block hash computed by the Rust path is affected. No special transaction or attacker action is required; the wrong hash is produced unconditionally for every block at the current network version.

---

### Recommendation

1. Add a `V0_14_0` (or later) variant to `BlockHashVersion` and extend `TryFrom<StarknetVersion>` to map V0_14.x versions to it.
2. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` fields to `PartialBlockHashComponents` and populate them in `PartialBlockHashComponents::new()` from `BlockInfo` (or pass them separately).
3. Add a new arm in `gas_prices_to_hash()` for `block_hash_version >= V0_14_0` that chains `l2_gas_consumed` and `next_l2_gas_price` into the `STARKNET_GAS_PRICES0` Poseidon hash, matching the formula used by the `echonet` CLI.
4. Add regression tests that fix the expected hash for a V0_14.x block and assert it changes when `l2_gas_consumed` or `next_l2_gas_price` is modified (mirroring the existing `test_hash_changes!` macro pattern).

---

#[derive(Clone, Debug, PartialEq, Eq, PartialOrd)]
pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}

impl From<BlockHashVersion> for StarknetVersion {
    fn from(value: BlockHashVersion) -> Self {
        match value {
            BlockHashVersion::V0_13_2 => StarknetVersion::V0_13_2,
            BlockHashVersion::V0_13_4 => StarknetVersion::V0_13_4,
        }
    }
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    type Error = StarknetApiError;

    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            // Starknet versions 0.13.2 and 0.13.3 both have the same block hash mechanism.
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }
