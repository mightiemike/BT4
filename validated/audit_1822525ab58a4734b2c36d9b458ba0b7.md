Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End User, Allowing Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which the pool always sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract. The extension therefore checks whether the **router** is allowlisted, not whether the **end user** is allowlisted. Any user can bypass a per-user swap allowlist by calling the router instead of the pool directly.

## Finding Description

**Root cause — pool hardcodes `msg.sender` as `sender`:**

`MetricOmmPool.swap` always passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

**Extension checks `sender` (the router) against the allowlist:**

`SwapAllowlistExtension.beforeSwap` receives that value as `sender` and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

**Router calls `pool.swap` directly, making itself `msg.sender` in the pool:**

`exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` from the router contract, so the pool sees the router as `msg.sender`: [3](#0-2) 

The same pattern applies to `exactInput` (L104–112), `exactOutputSingle` (L136–137), and `exactOutput` (L165–181).

**Contrast with `DepositAllowlistExtension`**, which correctly gates `owner` (the position owner), an explicit parameter the pool does not overwrite with `msg.sender`: [4](#0-3) 

**Exploit call path:**
```
User (bob, not allowlisted)
  → MetricOmmSimpleRouter.exactInputSingle(...)
    → pool.swap(recipient, ...)   [msg.sender = router]
      → _beforeSwap(sender=router, ...)
        → SwapAllowlistExtension.beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router]  ← true (router allowlisted for alice)
          → passes; bob's address never checked
```

## Impact Explanation

Two mutually exclusive fund-impacting failure modes:

1. **Allowlist bypass (primary):** The pool admin must allowlist the router for any legitimate user to perform multi-hop swaps. Once the router is allowlisted, every address on the network can call `exactInputSingle`/`exactInput`/`exactOutputSingle`/`exactOutput` and swap on the pool regardless of individual allowlist status. Pools intended for KYC'd or restricted participants are open to all — a broken core invariant with direct fund-flow impact.

2. **Legitimate users blocked (secondary):** If the admin does not allowlist the router, individually allowlisted users who need multi-hop routing are blocked even though they are permitted. Core swap functionality is broken for those users.

## Likelihood Explanation

- `MetricOmmSimpleRouter` is the primary production swap entry point; virtually all non-technical users route through it.
- No special privileges, flash loans, or unusual conditions are required — any wallet can call the router.
- The pool admin has no way to configure the extension to check the real user identity; `sender` is hardcoded to `msg.sender` in the pool and cannot be overridden by the caller.
- The `multicall` path uses `delegatecall` (preserving `msg.sender` within the router context) but still calls `pool.swap` from the router, so the pool still sees the router as `msg.sender`. [5](#0-4) 

## Recommendation

The extension must gate on the **original user** identity, not the immediate pool caller. The cleanest fix: the router always appends `abi.encode(msg.sender)` to `extensionData` before forwarding to the pool. `SwapAllowlistExtension.beforeSwap` decodes this value when `sender` is a known trusted router and gates on the decoded address instead. Trust is already enforced by `onlyPool` on the extension side. Alternatively, a dedicated trusted-forwarder registry can map router addresses to "check decoded user" behavior.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only alice is allowlisted
swapExtension.setAllowedToSwap(address(pool), alice, true);
// router is NOT allowlisted, bob is NOT allowlisted

// Direct swap by bob → correctly reverts
vm.prank(bob);
pool.swap(bob, false, 1000, type(uint128).max, "", "");
// → NotAllowedToSwap ✓

// Admin allowlists the router so alice can use multi-hop routing
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Bob calls router — pool sees router as sender, allowlist passes
vm.prank(bob);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token1),
    recipient: bob,
    amountIn: 1000,
    amountOutMinimum: 0,
    zeroForOne: false,
    priceLimitX64: type(uint128).max,
    deadline: block.timestamp,
    extensionData: ""
}));
// → succeeds; bob swapped despite not being allowlisted ✗
// allowedSwapper[pool][router] == true → check passes, bob's address never inspected
``` [6](#0-5)
