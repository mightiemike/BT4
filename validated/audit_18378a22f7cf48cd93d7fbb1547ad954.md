### Title
P2P Protobuf `ValidResourceBounds` Deserialization Silently Mutates Variant, Producing Wrong Transaction Hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in `crates/apollo_protobuf/src/converters/transaction.rs` silently converts `AllResources{l2_gas=0, l1_data_gas=0}` to `L1Gas`. However, `get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` produces **structurally different hash preimages** for these two variants: `AllResources` includes a third `l1_data_gas` element in the Poseidon chain while `L1Gas` does not. A V3 transaction originally committed with `AllResources` bounds where both `l2_gas` and `l1_data_gas` are zero will be reconstructed by any P2P-syncing node as `L1Gas`, causing the recomputed transaction hash to diverge from the canonical hash stored by the sequencer. This is a hash-domain canonicalization failure at the P2P serialization boundary.

---

### Finding Description

**Two representations, one protobuf encoding, two different hashes.**

`ValidResourceBounds` has two variants:

```
L1Gas(ResourceBounds)          // pre-0.13.3 format
AllResources(AllResourceBounds) // 0.13.3+ format
```

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` branches on the variant to decide how many elements enter the Poseidon hash chain:

```rust
// L1Gas path: hashes [tip, l1_gas_concat, l2_gas_concat(zero)]  — 2 resource elements
ValidResourceBounds::L1Gas(_) => vec![],
// AllResources path: hashes [tip, l1_gas_concat, l2_gas_concat(zero), l1_data_gas_concat(zero)] — 3 resource elements
ValidResourceBounds::AllResources(all_resources) => {
    vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
}
``` [1](#0-0) 

The serializer `From<ValidResourceBounds> for protobuf::ResourceBounds` emits **identical bytes** for both variants when `l2_gas=0` and `l1_data_gas=0`:

- `L1Gas(X)` → `{l1_gas=X, l2_gas=0, l1_data_gas=0}`
- `AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}` → `{l1_gas=X, l2_gas=0, l1_data_gas=0}` [2](#0-1) 

The deserializer then unconditionally collapses any protobuf message with zero `l2_gas` and zero `l1_data_gas` back to `L1Gas`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← always chosen for old-format AllResources
} else {
    ValidResourceBounds::AllResources(...)
})
``` [3](#0-2) 

The round-trip is therefore **not an identity** for `AllResources{l2_gas=0, l1_data_gas=0}`:

```
AllResources{l1_gas=X, l2_gas=0, l1_data_gas=0}
  → serialize → {l1_gas=X, l2_gas=0, l1_data_gas=0}
  → deserialize → L1Gas(X)
```

Because `get_tip_resource_bounds_hash` includes `l1_data_gas` in the hash chain only for `AllResources`, the two variants produce different `tip_resource_bounds_hash` values, and therefore different transaction hashes, even though the numeric field values are identical. [4](#0-3) 

The same silent collapse also occurs in the JSON/serde path: `ValidResourceBounds::Serialize` emits `L1_DATA_GAS` only for `AllResources`, so a JSON round-trip of `AllResources{l2_gas=0, l1_data_gas=0}` also loses the variant tag and deserializes back as `L1Gas`. [5](#0-4) 

The `GasVectorComputationMode` also diverges: `L1Gas` returns `NoL2Gas` while `AllResources` returns `All`, so fee pre-validation, gas bound checks, and initial Sierra gas allocation all behave differently for the mutated transaction. [6](#0-5) 

---

### Impact Explanation

A syncing node that receives a block via P2P containing a V3 transaction originally committed as `AllResources{l2_gas=0, l1_data_gas=0}` will:

1. Deserialize the transaction's `resource_bounds` as `L1Gas` (wrong variant).
2. Recompute the transaction hash using the `L1Gas` hash preimage (2 resource elements instead of 3), producing a hash that differs from the canonical hash committed by the sequencer.
3. Store the transaction under the wrong hash, causing `starknet_getTransactionByHash` and `starknet_getTransactionReceipt` to return "not found" for the canonical hash.
4. Use `GasVectorComputationMode::NoL2Gas` instead of `All` for any subsequent re-execution (RPC simulation, tracing, fee estimation), producing wrong gas accounting results.

The P2P server sends the canonical hash alongside the transaction data (via `get_block_transaction_hashes`), but the client reconstructs the `ValidResourceBounds` variant from the protobuf payload, not from the stored hash. Any code path that recomputes the hash from the deserialized `Transaction` object — including `validate_transaction_hash` — will compute the wrong value. [7](#0-6) 

This matches the **"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"** and **"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value"** impact categories.

---

### Likelihood Explanation

Pre-0.13.3 V3 transactions with `AllResources{l2_gas=0, l1_data_gas=0}` exist on Starknet mainnet and testnet. The comment in `get_tip_resource_bounds_hash` explicitly acknowledges this: *"Old V3 txs always have L2 gas bounds of zero, but they exist."* Any node that syncs historical blocks via P2P will encounter such transactions. No special attacker capability is required; the trigger is ordinary P2P block sync of existing mainnet data. [8](#0-7) 

---

### Recommendation

The protobuf deserializer must preserve the original variant intent. The correct fix is to require an explicit discriminator field in the protobuf schema (e.g., a `bounds_version` enum), or to always deserialize into `AllResources` when all three gas fields are present in the wire message, regardless of their values:

```rust
// In TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
// Remove the zero-value collapse. Always produce AllResources when
// l1_data_gas is present on the wire.
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

The `L1Gas` variant should only be produced when the wire message explicitly signals the pre-0.13.3 format (e.g., `l1_data_gas` field is absent/`None`), not when it is present but zero. The same fix must be applied to the JSON `Deserialize` implementation for `ValidResourceBounds`. [9](#0-8)

### Proof of Concept

```
1. Construct a V3 InvokeTransaction with:
     resource_bounds = AllResources { l1_gas = X, l2_gas = 0, l1_data_gas = 0 }
   Compute its canonical hash H_all using get_tip_resource_bounds_hash(AllResources{...}).
   H_all includes the l1_data_gas element in the Poseidon chain (3 elements).

2. Serialize the transaction to protobuf via From<ValidResourceBounds> for protobuf::ResourceBounds.
   Wire bytes: { l1_gas=X, l2_gas=0, l1_data_gas=0 }

3. Deserialize on the receiving node via TryFrom<protobuf::ResourceBounds> for ValidResourceBounds.
   Result: L1Gas(X)  ← variant has changed

4. Recompute the hash H_l1 using get_tip_resource_bounds_hash(L1Gas(X)).
   H_l1 omits the l1_data_gas element (2 elements).

5. Assert H_all != H_l1.
   The receiving node stores the transaction under H_l1, not H_all.
   starknet_getTransactionByHash(H_all) returns "not found".
   starknet_simulateTransactions uses GasVectorComputationMode::NoL2Gas instead of All.
``` [10](#0-9) [3](#0-2)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {

        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
