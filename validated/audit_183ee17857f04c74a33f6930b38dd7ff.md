Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Enabling Full Per-User Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps using the `sender` argument passed by the pool, which is the pool's own `msg.sender` — the direct caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, the pool's `msg.sender` is the router contract, so the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actual_EOA]`. This either locks allowlisted users out of the standard periphery path, or — if the admin adds the router to unblock them — grants every unprivileged caller unrestricted access to a curated pool.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly without passing the originating EOA anywhere in the call: [4](#0-3) 

The router stores the real payer in transient storage via `_setNextCallbackContext` for payment purposes, but this context is never surfaced to the extension. The pool receives `msg.sender = router`, passes `router` as `sender` to the extension, and the extension evaluates `allowedSwapper[pool][router]` — a single shared address — rather than the individual EOA. The same flaw applies to `exactInput` (line 103–112), `exactOutputSingle` (line 135–137), and `exactOutput` (line 165–181). [5](#0-4) 

## Impact Explanation

Two fund-impacting outcomes are directly reachable:

**Scenario A — Router not allowlisted:** Allowlisted EOAs cannot use `MetricOmmSimpleRouter` because the extension sees the router address and reverts with `NotAllowedToSwap`. Core swap functionality is broken for legitimate users on any pool using this extension.

**Scenario B — Router allowlisted to restore access:** The pool admin adds the router to the allowlist. Every unprivileged EOA on the network can now call `exactInputSingle` through the router and pass the check, because `allowedSwapper[pool][router] == true`. The per-user curation is completely nullified — any caller can trade on a pool explicitly restricted to specific counterparties, draining LP value or executing trades the pool admin prohibited.

Both outcomes satisfy the "Broken core pool functionality causing loss of funds or unusable swap flows" and "Admin-boundary break: pool admin's access control bypassed by an unprivileged path" impact criteria.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary documented swap entrypoint shipped with the protocol. Pool admins configuring `SwapAllowlistExtension` have no reason to expect that routing through the standard periphery changes the identity the extension evaluates. The bypass requires only a normal `exactInputSingle` call — no flash loans, no reentrancy, no privileged access. Any unprivileged EOA can trigger Scenario B whenever the admin has added the router to the allowlist.

## Recommendation

Thread the originating caller through the pool's `swap` signature so extensions receive the economically relevant actor. Add a `sender` parameter to `IMetricOmmPoolActions.swap` and have the router pass `msg.sender` explicitly:

```solidity
// MetricOmmSimpleRouter.exactInputSingle
IMetricOmmPoolActions(params.pool).swap(
    msg.sender,       // sender — the actual user
    params.recipient,
    params.zeroForOne,
    ...
);
```

The pool then forwards this value to `_beforeSwap`, and `SwapAllowlistExtension` evaluates `allowedSwapper[pool][actual_user]` as intended. Alternatively, the extension can read the payer from the router's transient storage context (analogous to `_getPayer()` used for payment), but threading the real caller through the `swap` signature is the cleanest and most general fix.

## Proof of Concept

```solidity
// Setup: pool deployed with SwapAllowlistExtension.
// Admin allowlists trustedUser and also the router (forced to unblock trustedUser).
swapAllowlist.setAllowedToSwap(pool, trustedUser, true);
swapAllowlist.setAllowedToSwap(pool, address(router), true);

// Attacker (not individually allowlisted) calls the router.
// Extension sees sender = address(router), allowedSwapper[pool][router] == true → passes.
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: attacker,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));

// Attacker received token1 despite not being individually allowlisted.
assertGt(token1.balanceOf(attacker), 0);
```
