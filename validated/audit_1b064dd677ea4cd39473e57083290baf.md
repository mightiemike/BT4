### Title
Unvalidated `fee_proposal_fri` in P2P-synced `SignedBlockHeader` corrupts the fee proposals window used to compute `fee_actual` — (`crates/apollo_p2p_sync/src/client/header.rs`, `crates/apollo_protobuf/src/converters/header.rs`)

---

### Summary

The `fee_proposal_fri` field in the p2p sync `SignedBlockHeader` protobuf message is explicitly **not part of the block hash** (proto WARNING comment confirms this). Any p2p peer can therefore send a `SignedBlockHeader` with a forged `fee_proposal_fri` value that passes all existing checks, gets stored verbatim, and is later loaded into the `fee_proposals_window` used to compute `fee_actual`. A corrupted `fee_actual` shifts the geometric bounds used to validate and generate fee proposals in consensus, causing the affected node to reject valid proposals or produce proposals that honest peers reject.

---

### Finding Description

**Step 1 — Deserialization: no validation**

In `crates/apollo_protobuf/src/converters/header.rs` line 179, `fee_proposal_fri` is decoded with a plain `map`:

```rust
let fee_proposal_fri = value.fee_proposal_fri.map(|v| GasPrice(u128::from(v)));
```

Any `u128` value is accepted. [1](#0-0) 

**Step 2 — Proto WARNING: field is outside the block hash**

The proto definition explicitly warns:

```
// WARNING: this field is currently not part of the block hash, so the value must be trusted.
optional Uint128 fee_proposal_fri = 22;
```

Because `fee_proposal_fri` is absent from the block hash, the `BlockSignature` carried in the same message does **not** cover it. A peer can forge any value without invalidating the signature. [2](#0-1) 

**Step 3 — P2P sync stores the forged value without additional checks**

`parse_data_for_block` in `crates/apollo_p2p_sync/src/client/header.rs` only verifies block number ordering and signature-vector length. It performs no validation of `fee_proposal_fri`. [3](#0-2) 

`write_to_storage` then calls `append_header` directly, which stores `fee_proposal_fri` verbatim into `StorageBlockHeader`. [4](#0-3) 

The storage layer faithfully persists whatever value arrived from the wire: [5](#0-4) 

**Step 4 — Fee proposals window is seeded from stored headers**

Before consensus starts, `initialize_fee_proposals_window` reads `fee_proposal_fri` from every block in the sliding window via `state_sync_client.get_block()` and inserts it into `fee_proposals_window`:

```rust
Ok(block) => self.record_fee_proposal(
    block_number,
    block.block_header_without_hash.fee_proposal_fri,
),
``` [6](#0-5) 

If any of those blocks were p2p-synced with a forged `fee_proposal_fri`, the window is poisoned.

**Step 5 — Corrupted window produces wrong `fee_actual`**

`compute_fee_actual` takes the median of the window. A forged entry shifts the median. [7](#0-6) 

**Step 6 — Wrong `fee_actual` corrupts proposal validation and generation**

`is_proposal_init_valid` derives the allowed `[lower_bound, upper_bound]` for incoming `fee_proposal_fri` directly from `fee_actual`:

```rust
let (lower_bound, upper_bound) = fee_proposal_bounds(
    fee_actual,
    VersionedConstants::latest_constants().fee_proposal_margin_ppt,
);
if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
    return Err(ValidateProposalError::InvalidProposalInit(...));
}
``` [8](#0-7) 

`compute_fee_proposal` also uses `fee_actual` as the anchor for the local node's own proposal. [9](#0-8) 

---

### Impact Explanation

A node whose `fee_proposals_window` has been poisoned will compute a wrong `fee_actual`. This causes it to:

1. **Reject valid proposals** from honest proposers whose `fee_proposal_fri` is within the correct range but outside the corrupted bounds.
2. **Emit proposals** with a `fee_proposal_fri` anchored to the wrong `fee_actual`, which honest peers will reject.
3. **Propagate the error** — once the corrupted block is committed, its `fee_proposal_fri` enters the windows of all peers that sync from this node, spreading the corruption.

This maps to the allowed impact: *"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value"* and *"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

---

### Likelihood Explanation

Any peer on the p2p network can send `SignedBlockHeader` messages. The attacker only needs to be a peer that the victim node syncs from (e.g., during initial sync or after a reorg). No operator/admin privileges are required. The forged value passes all existing deserialization and storage checks because the only protection (`BlockSignature`) does not cover `fee_proposal_fri`.

---

### Recommendation

1. **Short-term**: In `parse_data_for_block` (or in the storage write path), validate that `fee_proposal_fri` is `None` for pre-V0_14_3 blocks and `Some(v)` where `v` is within `fee_proposal_bounds(fee_actual_from_previous_window, margin_ppt)` for V0_14_3+ blocks. Reject the peer on violation.
2. **Medium-term**: Include `fee_proposal_fri` in the block hash (or in a separate commitment that is signed), removing the need for out-of-band trust. The `proposal_commitment_from` function already does this for consensus messages — the same binding should apply to the p2p sync wire format.
3. **Operational**: Until fixed, nodes should only sync from trusted peers when `fee_proposal_fri` is relevant (V0_14_3+).

---

### Proof of Concept

```rust
// Pseudocode sketch (production path, no test infrastructure)
// 1. Craft a SignedBlockHeader with a valid block_hash/signature but forged fee_proposal_fri.
let mut forged_header = honest_signed_block_header.clone();
forged_header.block_header.block_header_without_hash.fee_proposal_fri =
    Some(GasPrice(u128::MAX)); // extreme forged value

// 2. Feed it through the protobuf round-trip (no error expected).
let proto: protobuf::SignedBlockHeader = (forged_header.block_header.clone(),
    forged_header.signatures.clone()).into();
let decoded = SignedBlockHeader::try_from(proto).unwrap();
assert_eq!(decoded.block_header.block_header_without_hash.fee_proposal_fri,
           Some(GasPrice(u128::MAX))); // passes

// 3. Store it.
writer.begin_rw_txn().unwrap()
    .append_header(block_number, &decoded.block_header).unwrap()
    .commit().unwrap();

// 4. Initialize fee proposals window — reads the forged value.
ctx.initialize_fee_proposals_window(start_height).await.unwrap();

// 5. Compute fee_actual — median is now dominated by the forged entry.
let fee_actual = compute_fee_actual(&ctx.fee_proposals_window, next_height, window_size);
assert_ne!(fee_actual, honest_fee_actual); // divergent value confirmed
```

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

pub fn compute_fee_proposal(
    fee_target: Option<GasPrice>,
    fee_actual: GasPrice,
    margin_ppt: u128,
) -> GasPrice {
    let Some(fee_target) = fee_target else {
        return fee_actual;
    };
    let (lower, upper) = fee_proposal_bounds(fee_actual, margin_ppt);
    GasPrice(fee_target.0.clamp(lower, upper))
}
