### Title
Unconditional `LastKnownBonded = true` in `TierRedelegate` Causes Bonus Reward Overpayment for Unbonded Validator Periods — (`x/tieredrewards/keeper/msg_server.go`)

### Summary

`TierRedelegate` resets the position's bonus-accrual checkpoint with `LastKnownBonded = true` unconditionally, regardless of the destination validator's actual bonded state. If the destination validator is currently unbonded (its latest recorded event is an UNBOND), the persisted checkpoint is wrong. When the validator later re-bonds, `processEventsAndClaimBonus` treats the entire gap from the redelegate block to the re-bond event as a bonded segment and pays out bonus rewards from the `RewardsPoolName` module account for a period during which no bonded delegation existed.

---

### Finding Description

In `TierRedelegate` (`msg_server.go` lines 257–265):

```go
latestSeq, err := ms.getValidatorEventLatestSeq(ctx, dstValAddr)
...
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)   // ← always true
```

`UpdateBonusCheckpoints` writes three fields atomically:

```go
// x/tieredrewards/types/position.go:65-69
func (p *Position) UpdateBonusCheckpoints(lastEventSeq uint64, t time.Time, lastKnownBonded bool) {
    p.LastEventSeq     = lastEventSeq
    p.LastBonusAccrual = t
    p.LastKnownBonded  = lastKnownBonded
}
```

`LastKnownBonded` is the persisted bonded-state seed used by `processEventsAndClaimBonus`:

```go
// x/tieredrewards/keeper/claim_rewards.go:164-165
bonded      := pos.LastKnownBonded
segmentStart := pos.LastBonusAccrual
```

The loop walks validator events since `LastEventSeq` and pays bonus for every segment where `bonded == true`:

```go
// claim_rewards.go:175-179
if bonded {
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
    totalBonus = totalBonus.Add(bonus)
}
```

**Concrete trigger:** Suppose the destination validator's event log ends with an UNBOND event at seq `N` (the validator is currently unbonded). `getValidatorEventLatestSeq` returns `N`. `UpdateBonusCheckpoints(N, blockTime, true)` stores `LastKnownBonded = true` and `LastEventSeq = N`. When the validator re-bonds (BOND event at seq `N+1`), `processEventsAndClaimBonus` fetches `[N+1: BOND]`, enters the loop with `bonded = true`, and computes bonus for the segment `[redelegateBlockTime, bondEventTime]` — the entire unbonded gap — before processing the BOND event. The rewards are immediately sent from the module pool to the position owner.

The correct value to pass is the bonded state implied by the destination validator's latest event (false if the latest event is UNBOND), mirroring exactly how the position is initialized in `createDelegatedPosition` where `lastKnownBonded` is derived from the validator's live state.

---

### Impact Explanation

The `RewardsPoolName` module account is drained by the overpaid bonus. The exact corrupted value is the `LastKnownBonded` field of the `Position` protobuf stored in `k.Positions`. The financial impact is:

```
overpayment ≈ shares × tokensPerShare × tier.BonusApy × unbondedDuration / SecondsPerYear
```

For a large position and a validator that was unbonded for days or weeks, this can be a material drain. If the pool is exhausted, all subsequent legitimate bonus claims revert with `ErrInsufficientBonusPool`, denying rewards to honest stakers.

---

### Likelihood Explanation

Validators are routinely jailed and unjailed on production networks. An attacker needs only to:
1. Identify a validator whose latest event is UNBOND (publicly observable on-chain).
2. Call `TierRedelegate` to that validator with an existing position.
3. Wait for the validator to re-bond (or control a validator and trigger the re-bond themselves).
4. Call `ClaimTierRewards`.

No privileged access is required. The entry path is a standard `MsgTierRedelegate` transaction signed by the position owner.

---

### Recommendation

Derive `lastKnownBonded` from the destination validator's actual current bonded state before calling `UpdateBonusCheckpoints`, the same way `createDelegatedPosition` initialises a new position. Concretely, after fetching `latestSeq`, also fetch the validator and check `val.IsBonded()`:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
...
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

This mirrors the existing defensive check already present at the end of `processEventsAndClaimBonus` (`claim_rewards.go` line 206: `if bonded && val.IsBonded()`), making the initial checkpoint consistent with that guard.

---

### Proof of Concept

1. Validator `V` has event history: `[seq=1: BOND, seq=2: UNBOND]`. `V` is currently unbonded.
2. Attacker owns position `P` (large delegation, high-APY tier) currently delegated to validator `W`.
3. Attacker submits `MsgTierRedelegate{PositionId: P, DstValidator: V}`.
   - `claimRewards` settles all pending rewards against `W`. ✓
   - `getValidatorEventLatestSeq(V)` → `2`.
   - `UpdateBonusCheckpoints(2, T_redelegate, true)` persists `LastKnownBonded=true`, `LastEventSeq=2`. ✗
4. `V` re-bonds at time `T_bond` (seq=3: BOND event recorded).
5. Attacker submits `MsgClaimTierRewards{PositionIds: [P]}`.
   - `processEventsAndClaimBonus`: `bonded=true`, `segmentStart=T_redelegate`.
   - Events since seq 2: `[seq=3: BOND at T_bond]`.
   - Loop iteration: `bonded=true` → computes bonus for `[T_redelegate, T_bond]` (unbonded gap). **Overpayment sent.**
   - After BOND event: `bonded=true`, computes bonus for `[T_bond, now]`. Legitimate.
6. Attacker receives bonus rewards for the full unbonded period `[T_redelegate, T_bond]` from `RewardsPoolName`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
