### Title
Spread Health Adjustment Uses Wrong Weight for Short Leg, Corrupting Health Calculation — (`core/contracts/Clearinghouse.sol`, `core/contracts/libraries/RiskHelper.sol`)

---

### Summary

The spread health bonus/penalty in `Clearinghouse.getHealth` is computed using `existingWeight = (spotCoreRisk.longWeight + perpCoreRisk.longWeight) / 2`, and `_getSpreadWeightX18` similarly uses `perpCoreRisk.longWeight` or `spotCoreRisk.longWeight` unconditionally. Because `getCoreRisk` always returns the **long weight** (by passing a hardcoded `1` to `_getWeightX18`), the short leg of every spread position is evaluated with the wrong weight. For a long-spot / short-perp spread the perp's short weight (>1) is replaced by its long weight (<1), and vice versa for a short-spot / long-perp spread. This makes the spread health adjustment directionally wrong, allowing over-leveraged positions in one direction and causing unfair liquidations in the other.

---

### Finding Description

`getCoreRisk` in `BaseEngine.sol` always calls `_getWeightX18(risk, 1, healthType)` — the literal `1` forces the function to return the **long** weight regardless of the subaccount's actual position direction: [1](#0-0) 

`_getWeightX18` branches on the sign of `amount`; passing `1` always selects `longWeightInitialX18` / `longWeightMaintenanceX18`: [2](#0-1) 

The `CoreRisk` struct therefore carries only a `longWeight` field: [3](#0-2) 

In `Clearinghouse.getHealth`, `existingWeight` averages the two `longWeight` values unconditionally: [4](#0-3) 

`_getSpreadWeightX18` also uses `perpCoreRisk.longWeight` when the spot is long (perp is short) and `spotCoreRisk.longWeight` when the spot is short (perp is long): [5](#0-4) 

For a **long-spot / short-perp** spread the perp's actual health contribution in `_calculateProductHealth` uses `shortWeightX18 > 1` (because `amount < 0`): [6](#0-5) 

But `existingWeight` uses `perpCoreRisk.longWeight < 1` instead of `shortWeightX18 > 1`. The result is that `existingWeight` is materially lower than the true average weight already applied, making `spreadWeight − existingWeight` larger than it should be, and the spread health bonus is inflated.

**Concrete example** (long ETH spot / short ETH perp, both with `longWeight = 0.9`, `shortWeight = 1.1`):

| | With bug | Correct |
|---|---|---|
| `existingWeight` | `(0.9 + 0.9) / 2 = 0.90` | `(0.9 + 1.1) / 2 = 1.00` |
| `spreadWeight` | `1 − (1−0.9)/5 = 0.98` | `1 − (1−1.1)/5 = 1.02` → capped at `0.99` |
| `spreadWeight − existingWeight` | **+0.08** | **−0.01** |

With the bug the spread position receives a **positive** health bonus; the correct formula would apply a small **penalty**. The sign is inverted.

---

### Impact Explanation

- **Long-spot / short-perp spreads**: health is over-estimated. Users can open and hold spread positions at leverage levels the protocol did not intend to permit, creating bad-debt risk for the insurance fund and NLP holders.
- **Short-spot / long-perp spreads**: `basisAmount < 0`, so the inflated `spreadWeight − existingWeight` subtracts too much health, causing premature liquidations and direct loss of funds for those users.

Both outcomes match the external report's stated impact: *"unfair liquidations or over-leveraged positions depending on price movements."*

---

### Likelihood Explanation

Any subaccount that holds a registered spread pair (encoded in `spreads` in `ClearinghouseStorage`) is affected every time `getHealth` is called. Liquidation eligibility (`isUnderMaintenance`) and the post-liquidation health check (`isAboveInitial`) both flow through `getHealth`: [7](#0-6) [8](#0-7) 

No special privilege is required. Any trader who opens a spread position through the normal `Endpoint` flow is affected.

---

### Recommendation

`getCoreRisk` should return the weight that corresponds to the **actual position direction**, not always the long weight. Either:

1. Pass `amount` instead of `1` to `_getWeightX18` in `getCoreRisk`:
   ```solidity
   RiskHelper._getWeightX18(risk, amount, healthType)
   ```
   and rename the `CoreRisk` field from `longWeight` to `weight`; or

2. Store both `longWeight` and `shortWeight` in `CoreRisk` and select the correct one in `_getSpreadWeightX18` and `existingWeight` based on `spotCoreRisk.amount` / `perpCoreRisk.amount`.

`existingWeight` in `Clearinghouse.getHealth` must then use the actual weight applied to each leg:
```solidity
int128 existingWeight = (
    RiskHelper._getWeightX18(spotRisk, spotCoreRisk.amount, healthType) +
    RiskHelper._getWeightX18(perpRisk, perpCoreRisk.amount, healthType)
) / 2;
```

---

### Proof of Concept

1. Register a spread pair `(spotId, perpId)` in `spreads`.
2. Open a long-spot / short-perp position of equal notional size.
3. Call `getHealth(subaccount, INITIAL)`. Observe that the spread adjustment adds positive health (`spreadWeight − existingWeight > 0`).
4. Compute the correct health using `shortWeightX18` for the perp leg. Observe that the correct adjustment is negative.
5. The difference allows the subaccount to borrow more quote than the protocol's risk parameters permit, or to avoid liquidation when it should be liquidatable.

    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }
```

    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
