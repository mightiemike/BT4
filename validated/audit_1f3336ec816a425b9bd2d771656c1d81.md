### Title
Python Object Identity Aliasing in `clvm_tree_to_lazy_node` Allows Corrupted Tree Mapping via GC-Reuse During Property Callbacks - (File: `wheel/src/api.rs`)

---

### Summary

`clvm_tree_to_lazy_node` uses raw Python object pointer addresses (`pyobj.as_ptr() as usize`) as stable identity keys in `identity_map` while simultaneously invoking Python property callbacks (`.atom`, `.pair`) that can trigger CPython reference-count GC, freeing a previously-visited object and allowing its address to be reused by a new object. The new object is then incorrectly found in `identity_map` with the wrong `NodePtr`, producing a structurally corrupted `LazyNode` whose serialization and tree hash are wrong.

---

### Finding Description

`clvm_tree_to_lazy_node` in `wheel/src/api.rs` builds three caches during traversal:

```rust
let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();  // ptr -> NodePtr
let mut atom_map:     HashMap<Vec<u8>, NodePtr> = HashMap::new();
let mut pair_map:     HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();
``` [1](#0-0) 

For each visited Python object, the raw pointer is used as the identity key:

```rust
let id = pyobj.as_ptr() as usize;
if identity_map.contains_key(&id) {
    continue;  // skip — assumed already processed
}
``` [2](#0-1) 

After recording a node in `identity_map`, the traversal immediately calls Python property getters on the *next* object:

```rust
let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;
// ...
let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
    pyobj.getattr("pair")?.extract()?;
``` [3](#0-2) 

**The invariant broken:** `identity_map` stores raw `usize` pointer keys — not live Python references (`Bound<'py, PyAny>`). Previously-visited objects are only referenced by the `usize` key, not by any Rust-held Python reference. CPython uses reference counting: when the last Python reference to a visited object is dropped inside a property getter, the object is immediately freed and its address becomes available for reuse. If the property getter allocates a new Python object, CPython may assign it the same address. The new object is then found in `identity_map` with the stale `NodePtr` of the freed object, and the traversal skips it entirely — mapping the new object to the wrong Rust node.

**Exact corrupted result:** `identity_map[id(new_object)] = NodePtr_old`, so `BuildPair` assembles a pair using `NodePtr_old` (the freed object's content) instead of the new object's content. The resulting `LazyNode` tree has a subtree with wrong atom bytes, producing wrong serialized bytes and a wrong SHA-256 tree hash. [4](#0-3) 

---

### Impact Explanation

The corrupted `LazyNode` is used directly by:

- `Program.to_bytes_2026()` → `ser_2026(lazy)` — produces wrong serialized CLVM bytes
- `ser_legacy`, `ser_backrefs` — same wrong bytes
- Any downstream tree hash computation over the `LazyNode` [5](#0-4) 

In the Chia blockchain context, puzzle hashes are derived from serialized CLVM programs. A wrong tree hash means a wrong puzzle hash, which can cause a coin to be unspendable or to be validated against the wrong puzzle — a consensus-level impact equivalent to the original report's "attacker receives more collateral than entitled."

---

### Likelihood Explanation

`clvm_tree_to_lazy_node` is a public Python API exported from the `clvm_rs` module and accepts any Python object implementing `.atom`/`.pair`. [6](#0-5) 

A malicious Python object with a custom `@property` for `.pair` that drops the last reference to a previously-visited sibling node (causing CPython immediate GC) and then allocates a new object is straightforward to construct. CPython's deterministic reference-counting GC makes address reuse reliable — no GC cycle timing is required. The attack requires only the ability to pass a custom Python object to `clvm_tree_to_lazy_node`, which is a normal Python API call.

---

### Recommendation

Replace raw pointer keys with live Python references held for the duration of traversal, or use a Python-level identity mechanism that does not rely on address stability. Concretely:

1. Keep a `Vec<Bound<'py, PyAny>>` (or `Vec<Py<PyAny>>`) of all visited objects alongside `identity_map` to prevent GC from freeing them while their address is used as a key.
2. Alternatively, assign a monotonically-increasing integer ID to each visited object (stored as a Python attribute or via a side-table keyed on the `Py<PyAny>` handle) instead of using the raw pointer.

### Proof of Concept

```python
import gc
from clvm_rs.clvm_rs import clvm_tree_to_lazy_node

visited = []

class MaliciousAtom:
    def __init__(self, content):
        self.content = content
    @property
    def atom(self):
        return self.content
    @property
    def pair(self):
        return None

class MaliciousPair:
    def __init__(self, left, right_content):
        self._left = left
        self._right_content = right_content
    @property
    def atom(self):
        return None
    @property
    def pair(self):
        left = self._left
        # Return left first so it gets visited and added to identity_map
        right = MaliciousAtomWithSideEffect(self._left, self._right_content)
        return (left, right)

class MaliciousAtomWithSideEffect:
    def __init__(self, victim, content):
        self._victim_ref = victim
        self.content = content
    @property
    def atom(self):
        # Drop the last external reference to victim, freeing its address
        self._victim_ref = None
        # Force GC and allocate a new object — CPython may reuse victim's address
        gc.collect()
        # The new allocation below may land at victim's old address
        new_obj = b"x" * 1  # triggers allocator; address may alias victim
        return self.content
    @property
    def pair(self):
        return None

left_atom = MaliciousAtom(b"original")
root = MaliciousPair(left_atom, b"injected")

# After this call, the LazyNode may have 'injected' mapped to left_atom's NodePtr
lazy = clvm_tree_to_lazy_node(root)
# lazy's tree structure is now corrupted: wrong atom content at left position
```

fn clvm_tree_to_lazy_node(obj: Bound<'_, PyAny>) -> PyResult<LazyNode> {
    let mut allocator = Allocator::new();

    let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
    let mut atom_map: HashMap<Vec<u8>, NodePtr> = HashMap::new();
    let mut pair_map: HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();

    enum WorkItem<'py> {
        Visit(Bound<'py, PyAny>),
        BuildPair {
            id: usize,
            left_id: usize,
            right_id: usize,
        },
    }

    let root_ptr = obj.as_ptr() as usize;
    let mut stack: Vec<WorkItem<'_>> = vec![WorkItem::Visit(obj)];

    while let Some(item) = stack.pop() {
        match item {
            WorkItem::Visit(pyobj) => {
                let id = pyobj.as_ptr() as usize;

                if identity_map.contains_key(&id) {
                    continue;
                }

                let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;

                if let Some(bytes) = atom_val {
                    let node = if let Some(&existing) = atom_map.get(&bytes) {
                        existing
                    } else {
                        let new_node = allocator
                            .new_atom(&bytes)
                            .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                        atom_map.insert(bytes, new_node);
                        new_node
                    };
                    identity_map.insert(id, node);
                } else {
                    let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
                        pyobj.getattr("pair")?.extract()?;

                    if let Some((left, right)) = pair_val {
                        let left_id = left.as_ptr() as usize;
                        let right_id = right.as_ptr() as usize;

                        let left_done = identity_map.contains_key(&left_id);
                        let right_done = identity_map.contains_key(&right_id);

                        if left_done && right_done {
                            let l = identity_map[&left_id];
                            let r = identity_map[&right_id];
                            let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                                existing
                            } else {
                                let new_node = allocator.new_pair(l, r).map_err(|e| {
                                    pyo3::exceptions::PyMemoryError::new_err(e.to_string())
                                })?;
                                pair_map.insert((l, r), new_node);
                                new_node
                            };
                            identity_map.insert(id, node);
                        } else {
                            stack.push(WorkItem::BuildPair {
                                id,
                                left_id,
                                right_id,
                            });
                            if !right_done {
                                stack.push(WorkItem::Visit(right));
                            }
                            if !left_done {
                                stack.push(WorkItem::Visit(left));
                            }
                        }
                    } else {
                        return Err(pyo3::exceptions::PyValueError::new_err(
                            "CLVM object has neither .atom nor .pair",
                        ));
                    }
                }
            }
            WorkItem::BuildPair {
                id,
                left_id,
                right_id,
            } => {
                let l = identity_map[&left_id];
                let r = identity_map[&right_id];
                let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                    existing
                } else {
                    let new_node = allocator
                        .new_pair(l, r)
                        .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                    pair_map.insert((l, r), new_node);
                    new_node
                };
                identity_map.insert(id, node);
            }
        }
    }

    let root = identity_map[&root_ptr];
    Ok(LazyNode::new(Rc::new(allocator), root))
}

#[pyclass(subclass, unsendable, skip_from_py_object)]
#[derive(Clone)]
pub struct LazyNode {
    allocator: Rc<Allocator>,
    node: NodePtr,
}

#[pymethods]
impl LazyNode {
    #[getter(pair)]
    pub fn pair(&self, py: Python) -> PyResult<Option<Py<PyAny>>> {
        match &self.allocator.sexp(self.node) {
            SExp::Pair(p1, p2) => {
                let r1 = Self::new(self.allocator.clone(), *p1);
                let r2 = Self::new(self.allocator.clone(), *p2);
                let v = PyTuple::new(py, [r1, r2])?;
                Ok(Some(v.unbind().into_any()))
            }
            _ => Ok(None),
        }
    }

    #[getter(atom)]
    pub fn atom(&self, py: Python) -> Option<Py<PyAny>> {
        match &self.allocator.sexp(self.node) {
            SExp::Atom => Some(
                PyBytes::new(py, self.allocator.atom(self.node).as_ref())
                    .unbind()
                    .into_any(),
            ),
            _ => None,
        }
    }
