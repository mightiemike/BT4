### Title
Unauthorized NFT Unescrow via Crafted `ClassId` Prefix in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` uses `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)` to decide whether to mint vouchers or unescrow native NFTs. A malicious counterparty chain whose IBC channel ID happens to be `channel-0` can craft a packet with `data.ClassId = "nft/channel-0/nativeClass"` to make `IsAwayFromOrigin` return `false`, causing the receiver chain to unescrow a previously-escrowed native NFT to an attacker-controlled address.

---

### Finding Description

**Step 1 — `IsAwayFromOrigin` logic** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

`IsAwayFromOrigin` returns `false` (i.e., "moving back to origin") whenever `fullClassPath` starts with `sourcePort/sourceChannel/`. The `sourcePort` and `sourceChannel` here are taken directly from the **incoming packet's source fields** — i.e., chain B's port and channel ID.

**Step 2 — `processReceivedPacket` unescrow branch** [2](#0-1) 

```go

escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

if isAwayFromOrigin {
    // mint vouchers ...
} else {
    unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

        if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver); err != nil {

        }
```

**Step 3 — Concrete exploit trace**

Preconditions:
- Chain A has port `nft`, channel `channel-0` connecting to chain B.
- Chain B **also** has port `nft`, channel `channel-0` connecting to chain A (both sides independently assigned `channel-0`; this is common for the first channel on each chain).
- Chain A previously escrowed NFT `(nativeClass, tokenID)` to `GetEscrowAddress("nft", "channel-0")` when sending to chain B.

Attacker (controlling chain B) sends a crafted `NonFungibleTokenPacketData`:
- `data.ClassId = "nft/channel-0/nativeClass"`
- `data.TokenIds = ["tokenID"]`
- `data.Receiver = attackerAddress`

On chain A's `OnRecvPacket`:
1. `packet.GetSourcePort() = "nft"`, `packet.GetSourceChannel() = "channel-0"` (chain B's side, validated by IBC core against the registered counterparty).
2. `IsAwayFromOrigin("nft", "channel-0", "nft/channel-0/nativeClass")` → `prefixClassID = "nft/channel-0/"` → `HasPrefix = true` → returns **`false`**.
3. `RemoveClassPrefix("nft", "channel-0", "nft/channel-0/nativeClass")` → `"nativeClass"`.
4. `ParseClassTrace("nativeClass").IBCClassID()` → `"nativeClass"` (empty path, returns base class ID directly). [3](#0-2) 

5. `escrowAddress = GetEscrowAddress("nft", "channel-0")` — **the same address that holds the escrowed NFT**. [4](#0-3) 

6. `TransferOwner(ctx, "nativeClass", tokenID, escrowAddress, attackerAddress)` — the escrow address owns the NFT (per precondition), so the transfer succeeds. The attacker receives the native NFT.

**Why `ValidateBasic` does not block this:** [5](#0-4) 

`ValidateBasic` only checks that `ClassId` is non-blank, `TokenIds` is non-empty, lengths match, and addresses are valid bech32. It does not validate the class trace path or check whether the classID is consistent with the channel's actual history.

---

### Impact Explanation

An attacker controlling a counterparty chain whose IBC channel ID is `channel-0` can drain any native NFT that chain A has escrowed for that channel, transferring it to an arbitrary receiver. This violates the core ICS-721 invariant: escrowed NFTs may only be unescrow to the original sender on a valid return packet. The impact is **unauthorized unescrow of native NFTs** — permanent loss of NFT ownership for the original sender.

---

### Likelihood Explanation

The exploit requires that chain B's channel ID for the channel connecting to chain A is `channel-0`. Since channel IDs are assigned sequentially starting from `channel-0`, this is the common case when the nft-transfer channel is the first channel established on chain B. This is realistic in production deployments, especially for new chains or dedicated IBC connections.

---

### Recommendation

The direction check must use the **destination** port/channel (chain A's own side), not the source port/channel from the incoming packet. The correct invariant is: if `data.ClassId` is prefixed with `destPort/destChannel/`, the NFT is returning to origin (chain A previously added that prefix when it sent the NFT). Replace:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress    := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
```

with:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetDestPort(), packet.GetDestChannel(), data.ClassId)
escrowAddress    := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
```

This mirrors the correct logic in `createOutgoingPacket`, which uses `sourcePort`/`sourceChannel` (chain A's own outgoing port/channel) for the direction check. [6](#0-5)

### Proof of Concept

```
Setup:
  Chain A: port=nft, channel=channel-0, counterparty=(nft, channel-0) on chain B
  Chain B: port=nft, channel=channel-0, counterparty=(nft, channel-0) on chain A

1. Alice on chain A calls SendTransfer(nft, channel-0, "nativeClass", ["token1"], alice, bobOnB, ...)
   → createOutgoingPacket: IsAwayFromOrigin("nft","channel-0","nativeClass") = true
   → TransferOwner("nativeClass","token1", alice, escrow(nft,channel-0))
   → NFT now owned by escrow address

2. Attacker on chain B crafts and submits a packet:
   data.ClassId  = "nft/channel-0/nativeClass"
   data.TokenIds = ["token1"]
   data.Receiver = attackerOnA
   data.Sender   = anyValidAddress

3. Chain A's OnRecvPacket → processReceivedPacket:
   IsAwayFromOrigin("nft","channel-0","nft/channel-0/nativeClass") = false
   unprefixedClassID = "nativeClass"
   voucherClassID    = "nativeClass"
   escrowAddress     = GetEscrowAddress("nft","channel-0")  ← owns token1
   TransferOwner("nativeClass","token1", escrowAddress, attackerOnA) → SUCCESS

4. Assert: owner of (nativeClass, token1) == attackerOnA  ← exploit confirmed
   Alice's NFT is stolen; the original send packet's ack/timeout no longer has an NFT to refund.
```

```go
func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}
```

```go
func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```
