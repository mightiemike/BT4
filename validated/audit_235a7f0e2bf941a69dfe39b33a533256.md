Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router intermediary address instead of originating user, rendering per-pool swap allowlist inoperative for all router-mediated swaps - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is `msg.sender` of `pool.swap()` — the direct caller of the pool, not the originating user. When swaps are routed through `MetricOmmSimpleRouter`, `sender` equals the router address. If the pool admin allowlists the router (the only way to permit any router-based swap), the allowlist check passes unconditionally for every caller of that router, regardless of individual authorization.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes this value as `sender` into the extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the pool see `msg.sender = router`: [4](#0-3) 

The `recipient` parameter (second argument to `beforeSwap`, currently unnamed/ignored) carries the actual end-user address but is never checked. By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly checks `owner` (the second parameter — the actual end-user) rather than the intermediary: [5](#0-4) 

The structural asymmetry is the root cause: the deposit extension reads the user-identifying field; the swap extension reads the intermediary-identifying field.

## Impact Explanation

A pool deploying `SwapAllowlistExtension` to restrict trading to a curated set (KYC-gated, institutional-only, regulatory-compliance) cannot simultaneously permit router-based swaps and enforce per-user restrictions. Any address can call `router.exactInputSingle / exactInput / exactOutputSingle / exactOutput`, causing the pool to see `sender = router`. If the router is allowlisted, the guard passes unconditionally, allowing unauthorized principals to trade against LP capital in a pool explicitly configured to exclude them. This constitutes a broken core pool access-control mechanism with direct fund-impact potential (unauthorized parties executing swaps against restricted LP positions).

## Likelihood Explanation

- `MetricOmmSimpleRouter` is the standard user-facing entry point; most swaps are expected to flow through it.
- A pool admin who wants any user to swap via the router must allowlist the router address, which simultaneously opens the bypass to every caller.
- No special privilege, flash loan, or oracle manipulation is required — any EOA or contract can call the router.
- The bypass is reachable through the standard production periphery path with zero additional preconditions beyond the router being allowlisted.

## Recommendation

Check `recipient` (the second parameter, currently ignored) instead of `sender`, consistent with how `DepositAllowlistExtension` handles the analogous case:

```solidity
function beforeSwap(address, address recipient, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][recipient]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

This works because `MetricOmmSimpleRouter` always sets `recipient` to the user-controlled address (`params.recipient`) in all swap variants, and the pool passes it unchanged as the second argument to the extension.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]  = true   // alice is KYC'd
  allowedSwapper[pool][router] = true   // admin must set this for any router swap to work
  bob is NOT in the allowlist

Attack:
  bob calls router.exactInputSingle({pool: pool, recipient: bob, ...})
  → router calls pool.swap(bob /*recipient*/, ...)          [MetricOmmSimpleRouter.sol L72-80]
  → pool calls _beforeSwap(msg.sender=router, recipient=bob, ...)  [MetricOmmPool.sol L230-240]
  → extension checks allowedSwapper[pool][router] == true  ✓
  → swap executes — bob trades in a pool he is not authorized to access

Foundry test sketch:
  1. Deploy pool with SwapAllowlistExtension.
  2. setAllowedToSwap(pool, alice, true); setAllowedToSwap(pool, router, true).
  3. Call router.exactInputSingle from bob (not allowlisted).
  4. Assert swap succeeds (no NotAllowedToSwap revert).
  5. Confirm allowedSwapper[pool][bob] == false throughout.
```
