### Title
`ValidResourceBounds` Protobuf Deserialization Boundary Silently Downcasts `AllResources` to `L1Gas`, Causing P2P Rejection of Gateway-Accepted Transactions - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

A zero-value boundary condition in the protobuf-to-`ValidResourceBounds` converter silently switches the resource-bounds variant from `AllResources` to `L1Gas` whenever `l2_gas` and `l1_data_gas` are both zero. Because the `RpcInvokeTransactionV3` type requires `AllResources`, any peer that receives such a transaction over P2P immediately rejects it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, even though the gateway accepted and hashed the transaction as a valid `AllResources` V3 invoke.

---

### Finding Description

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` applies a threshold test:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

This is the exact analog of the external report's "dust" reset: just as reserves ≤ 100 wei caused the pool to reset its ratio, `l2_gas == 0 && l1_data_gas == 0` causes the resource-bounds type to reset from `AllResources` to `L1Gas`.

The gateway accepts `RpcTransaction`, which always carries `AllResourceBounds` (never `ValidResourceBounds`). A V3 invoke with `AllResourceBounds { l1_gas: non-zero, l2_gas: zero, l1_data_gas: zero }` passes the stateless validator because `max_possible_fee` is non-zero (driven by `l1_gas`):

```rust
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
}
``` [2](#0-1) 

The transaction hash is then computed via `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`. For `AllResources`, this hashes **four** elements: `[tip, l1_gas, l2_gas, l1_data_gas]`. For `L1Gas`, it hashes only **three**: `[tip, l1_gas, l2_gas(=zero)]` — the `l1_data_gas` term is omitted:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
``` [3](#0-2) 

When the accepted transaction is serialized to protobuf for P2P propagation (consensus or mempool), `From<ValidResourceBounds> for protobuf::ResourceBounds` faithfully encodes all three fields with their zero values:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),
    },
``` [4](#0-3) 

On the receiving peer, deserialization triggers the zero-boundary condition and produces `ValidResourceBounds::L1Gas`. The conversion chain then reaches:

```rust
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
``` [5](#0-4) 

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` requires `AllResources`:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange {
            string: "resource_bounds".to_string(),
        });
    }
},
``` [6](#0-5) 

The peer rejects the transaction. The gateway-computed hash (4-element `AllResources` hash) is also irreconcilable with the 3-element `L1Gas` hash a peer would compute if it could reconstruct the transaction.

---

### Impact Explanation

A valid V3 invoke transaction accepted and hashed by the gateway is permanently undeliverable over P2P. Any node that receives it via consensus or mempool propagation rejects it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction can never be sequenced. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

and

> **High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

---

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a V3 invoke with `l2_gas = 0` and `l1_data_gas = 0` but non-zero `l1_gas`. This is a natural configuration for users who only want to pay L1 gas (e.g., pre-0.13.3 style transactions submitted via the V3 API). The gateway accepts it; every peer rejects it. No special privileges or knowledge of internals are required.

---

### Recommendation

**Short term:** Remove the `L1Gas`/`AllResources` type-switching heuristic from the protobuf deserializer. When a `ConsensusTransaction` or `MempoolTransaction` is received, always deserialize `ResourceBounds` as `AllResources` (matching the `AllResourceBounds` converter in `crates/apollo_protobuf/src/converters/rpc_transaction.rs` lines 212–223 which already does this correctly):

```rust
impl TryFrom<protobuf::ResourceBounds> for AllResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        Ok(Self {
            l1_gas: value.l1_gas.ok_or(missing("ResourceBounds::l1_gas"))?.try_into()?,
            l2_gas: value.l2_gas.ok_or(missing("ResourceBounds::l2_gas"))?.try_into()?,
            l1_data_gas: value.l1_data_gas.ok_or(missing("ResourceBounds::l1_data_gas"))?.try_into()?,
        })
    }
}
``` [7](#0-6) 

The `ValidResourceBounds` converter (used for historical sync of old blocks) should remain separate and not be used in the consensus/mempool deserialization path.

**Long term:** Enforce a strict invariant: any transaction that enters the gateway as `AllResources` must be deserialized as `AllResources` at every subsequent boundary. Add a round-trip test: serialize an `AllResources` transaction with zero `l2_gas`/`l1_data_gas` to protobuf and assert the deserialized type is still `AllResources`.

---

### Proof of Concept

1. Craft a V3 invoke transaction:
   ```
   AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
   }
   ```
2. Submit via `starknet_addInvokeTransaction`. The gateway's stateless validator passes (l1_gas fee is non-zero). The hash is computed as `AllResources` (4-element poseidon chain including zero `l1_data_gas`). `InternalRpcTransaction` is stored with this hash.
3. The transaction is serialized to `protobuf::InvokeV3WithProof` for P2P propagation. All three resource-bound fields are present with their values (l2_gas=0, l1_data_gas=0).
4. A peer deserializes the protobuf. `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` fires the `l1_data_gas.is_zero() && l2_gas.is_zero()` branch → produces `ValidResourceBounds::L1Gas`.
5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` fails → `DEPRECATED_RESOURCE_BOUNDS_ERROR`.
6. The peer rejects the transaction. It is never sequenced.
