Audit Report

## Title
`SwapAllowlistExtension` evaluates the router's address instead of the originating user, allowing any caller to bypass the per-pool swap allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` — the router contract — as the `sender` argument to `_beforeSwap`, which forwards it verbatim to `SwapAllowlistExtension.beforeSwap()`. The extension checks `allowedSwapper[pool][sender]`, so it evaluates the router's identity rather than the originating user's identity. Any pool admin who allowlists the router to enable router-mediated access for their intended users inadvertently opens the pool to every unprivileged caller of the router.

## Finding Description
`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)` at L230–231. When invoked via `MetricOmmSimpleRouter.exactInputSingle()` (L72–80), the pool's `msg.sender` is the router contract. `ExtensionCalling._beforeSwap()` (L149–176) passes this value verbatim as `sender` to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`. `SwapAllowlistExtension.beforeSwap()` (L37) then evaluates `allowedSwapper[msg.sender][sender]`, i.e., `allowedSwapper[pool][router]`. A pool admin who wants their allowlisted users to reach the pool through the standard router must allowlist the router address. Once the router is allowlisted, the check passes for every caller of the router — the extension has no mechanism to distinguish which end user invoked it. For multi-hop `exactInput`, intermediate hops use `address(this)` (the router) as payer (L103), so the router is always the `sender` seen by every intermediate pool's extension, compounding the bypass across all hops.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, institutional LPs, whitelisted market makers) is fully open to any user who routes through `MetricOmmSimpleRouter`. The unauthorized caller receives the same swap execution — at oracle-derived prices — as an allowlisted user, enabling unauthorized swaps against LP capital. This is a direct, fund-impacting breach of the pool's intended access-control invariant.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router, which is the natural operational step for any restricted pool that intends to support router-mediated access for its allowlisted users. The admin's mental model ("the router is a trusted intermediary for my users") does not match the extension's actual behavior. Once the router is allowlisted, the bypass is available to any unprivileged address with no special setup, no privileged role, and no additional preconditions.

## Recommendation
The pool's `swap()` interface should accept an explicit `originator` parameter that the router fills with `msg.sender` (the end user), and the pool should pass that value — not its own `msg.sender` — to extension hooks. Alternatively, `MetricOmmSimpleRouter` should forward the end user's address in `extensionData` and `SwapAllowlistExtension` should decode and verify it (requiring the extension to trust the router). Using `tx.origin` is not recommended for general use. Until fixed, pool admins must not allowlist the router; allowlisted users must call the pool directly, which breaks the intended UX.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension`. Admin calls `setAllowedToSwap(pool, alice, true)` and `setAllowedToSwap(pool, router, true)` so that `alice` can use the router.
2. `bob` (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: restrictedPool, ...})`.
3. The router calls `pool.swap(...)`. The pool's `msg.sender` is the router.
4. `_beforeSwap` is called with `sender = router`.
5. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][router]` → `true`.
6. The swap executes. `bob` receives output tokens from the restricted pool without ever being allowlisted.

Foundry test: deploy pool + extension, allowlist `alice` and the router, impersonate `bob`, call `exactInputSingle`, assert the swap succeeds and `bob` receives tokens — demonstrating the allowlist is fully bypassed.
