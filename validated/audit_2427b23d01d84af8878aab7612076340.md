Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual Swapper, Enabling Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which the pool unconditionally sets to `msg.sender` of the `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of the pool call, so the allowlist checks the router address rather than the end user. Any pool admin who allowlists the router to support legitimate users inadvertently grants every unprivileged address the ability to bypass the curated-pool restriction by calling the public router.

## Finding Description

In `MetricOmmPool.swap()`, the pool passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router contract `msg.sender` of that call: [2](#0-1) 

The same pattern applies to `exactInput` (multi-hop): [3](#0-2) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router — never the end user: [4](#0-3) 

There is no mechanism in the pool, the router, or the extension to propagate the original end-user identity through the router hop. The `extensionData` field is caller-supplied and unverified, so it cannot serve as a trusted identity channel without a coordinated, authenticated convention.

## Impact Explanation

**Outcome A — Allowlist bypass (High):** The pool admin must allowlist the router address for any legitimate user to trade through the supported periphery path. Once `allowedSwapper[pool][router] = true`, every caller of the router — including addresses the admin explicitly excluded — passes the guard. The curated-pool invariant is fully defeated; non-KYC'd or otherwise restricted addresses can execute swaps on the pool with a single `exactInputSingle` call.

**Outcome B — Broken core swap flow (Medium):** If the admin does not allowlist the router, every allowlisted user who attempts to swap through the router is blocked (`sender = router`, not in allowlist), forcing them to implement `IMetricOmmSwapCallback` themselves and call the pool directly. The supported periphery path becomes unusable for any pool deploying `SwapAllowlistExtension`.

Both outcomes constitute broken core pool functionality and/or direct loss of the curated-pool access-control guarantee, meeting Sherlock High/Medium thresholds.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entry point. Any pool combining `SwapAllowlistExtension` with the router — the natural production configuration — immediately exhibits the wrong-actor binding. No special timing, oracle manipulation, or privileged access is required; a single `exactInputSingle` call from a non-allowlisted address is sufficient to demonstrate the bypass.

## Recommendation

The extension must verify the identity of the economic actor, not the immediate pool caller. Viable approaches:

1. **Authenticated `extensionData` channel:** The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. Requires a coordinated, authenticated convention between the router and the extension (e.g., the extension only trusts `extensionData`-supplied identity when `sender` is a registered router address).

2. **Router registry with fallback identity:** The pool admin maintains a separate router registry in the extension; when `sender` is a registered router, the extension reads the actual swapper from `extensionData` instead.

3. **Document incompatibility and enforce it:** Add a revert in `beforeSwap` when `sender` is a known router (or a contract), and document that `SwapAllowlistExtension` only supports direct pool calls.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension (beforeSwap hook active)
  pool admin sets: allowedSwapper[pool][alice] = true
                   allowedSwapper[pool][router] = true  ← required for alice to use router

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: pool, ..., recipient: bob})
      → pool.swap(bob, ...)  [msg.sender = router]
        → _beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] == true  → PASSES

Result:
  bob executes a swap on a curated pool that was supposed to block him.
  The allowlist check never evaluated bob's address.

Verification (direct call, correctly blocked):
  bob calls pool.swap(...) directly
    → _beforeSwap(sender=bob, ...)
      → allowedSwapper[pool][bob] == false → REVERTS NotAllowedToSwap ✓
```
