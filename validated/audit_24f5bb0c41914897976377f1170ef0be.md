Audit Report

## Title
`SwapAllowlistExtension` gates on router address instead of end-user identity, allowing any user to bypass per-swapper restrictions via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` from the pool's perspective — the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router's address, not the end user's. A pool admin who allowlists the router to enable router-mediated swaps inadvertently grants every user — including non-allowlisted ones — the ability to bypass the guard.

## Finding Description
**Pool passes `msg.sender` (the router) as `sender` to the extension:**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

**Extension checks `sender` (the router), not the end user:**

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router. The original user's address is never visible to the extension: [2](#0-1) 

**Router does not forward the original user to the pool:**

`MetricOmmSimpleRouter.exactInputSingle` stores `msg.sender` only in transient callback context (for payment), but calls `pool.swap()` directly without encoding the original user into `extensionData` or any other argument: [3](#0-2) 

**The `extensionData` parameter is ignored by the extension:**

The `bytes calldata` last parameter in `beforeSwap` has no variable name and is not decoded, so there is no existing convention to pass the original user through it: [4](#0-3) 

**Forced dilemma:** `setAllowedToSwap` maps `allowedSwapper[pool][swapper]`. To permit any router-mediated swap, the admin must call `setAllowedToSwap(pool, router, true)`. Once set, `allowedSwapper[pool][router] == true` for every call arriving through the router, regardless of which end user initiated it: [5](#0-4) 

## Impact Explanation
`SwapAllowlistExtension` is the sole on-chain mechanism for pool admins to restrict who may trade. Once bypassed, any address can swap in a pool configured to be private (e.g., KYC-gated, institutional-only, or whitelist-only). Unauthorized swaps execute at oracle price, which is economically fair per swap but violates the pool's intended access model and exposes LPs to counterparties they explicitly excluded. This constitutes an admin-boundary break triggered by an unprivileged path — an unprivileged trader bypasses a pool admin's intended per-swapper access control.

## Likelihood Explanation
Any pool admin who deploys a `SwapAllowlistExtension` and also wants users to access the pool through `MetricOmmSimpleRouter` will naturally allowlist the router, as it is the primary user-facing entry point. The bypass is reachable on every such pool without any privileged action by the attacker — a non-allowlisted user simply calls `exactInputSingle` (or any other router swap function) targeting the restricted pool.

## Recommendation
The extension must gate on the actual end-user identity, not the intermediary. Two viable approaches:

1. **Pass the original user through `extensionData`:** The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a coordinated convention between router and extension.
2. **Add a `swapper` field to the pool's `beforeSwap` signature:** Expose both `sender` (direct caller) and `swapper` (original initiator) so extensions can choose which identity to gate. This is a core interface change but is the cleanest fix.

Until fixed, pool admins must not simultaneously allowlist the router and restrict individual swappers — the two settings are mutually exclusive under the current design.

## Proof of Concept
```
1. Pool admin deploys pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin allowlists the router (necessary for any router-mediated swap):
       swapExtension.setAllowedToSwap(pool, address(router), true);
3. Pool admin does NOT allowlist Bob:
       allowedSwapper[pool][bob] == false
4. Bob calls router.exactInputSingle({pool: pool, ...}).
5. Router calls pool.swap(recipient, ...) — msg.sender in pool = router.
6. Pool calls extension.beforeSwap(router, ...).
7. Extension evaluates:
       allowAllSwappers[pool]       → false
       allowedSwapper[pool][router] → true  ← router is allowlisted
   → check passes, swap executes.
8. Bob successfully swaps in a pool he was never individually permitted to access.
```
