### Title
Missing Declaration Check for Cairo 0 Classes in `ExecutionStateReader::get_compiled_class()` Allows Reverted/Undeclared Classes to Be Used in RPC Execution - (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class()` correctly gates Cairo 1 (V1) classes behind an `is_contract_class_declared()` check before returning them from the class manager. For Cairo 0 (V0/deprecated) classes, this check is explicitly absent — acknowledged by a TODO comment — allowing the class manager to serve a class that is not actually declared at the queried `state_number`. The same asymmetry exists in `StateReaderAndContractManager::get_compiled_from_class_manager()`.

### Finding Description

In `ExecutionStateReader::get_compiled_class()`, when `class_manager_handle` is `Some`, the code fetches the class from the class manager and branches on its type:

```rust
// crates/apollo_rpc_execution/src/state_reader.rs, lines 121–141
return match contract_class {
    ContractClass::V1(casm_contract_class) => {
        let is_declared = is_contract_class_declared(
            &self.storage_reader.begin_ro_txn()...,
            &class_hash,
            self.state_number,
        )?;
        if is_declared {
            Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
        } else {
            Err(StateError::UndeclaredClassHash(class_hash))
        }
    }
    // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
    ContractClass::V0(deprecated_contract_class) => {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    }
};
``` [1](#0-0) 

For `ContractClass::V1`, `is_contract_class_declared()` is called with `self.state_number` to verify the class is actually declared at the queried state. For `ContractClass::V0`, **no such check is performed**. The class is returned unconditionally from the class manager.

The same asymmetry exists in `StateReaderAndContractManager::get_compiled_from_class_manager()`:

```rust
// crates/blockifier/src/state/state_reader_and_contract_manager.rs, lines 70–87
match &runnable_class {
    RunnableCompiledClass::V0(_) => {}  // ← no is_declared check
    _ => {
        // The Cairo1 class is cached; verify it is declared,
        // since existence in the cache does not guarantee that
        // (it might contain a declared class from a reverted block, for example).
        if !self.state_reader.is_declared(class_hash)? {
            return Err(StateError::UndeclaredClassHash(class_hash));
        }
    }
}
``` [2](#0-1) 

The comment for the V1 branch explicitly states the reason for the check: *"it might contain a declared class from a reverted block"*. This exact risk applies equally to V0 classes, but the check is absent.

The class manager is a persistent, independent component. When a Cairo 0 declare transaction is processed at the gateway, the class is added to the class manager via `add_class()`. If the block containing that declaration is later reverted (e.g., due to a reorg or sequencer revert), the class manager does **not** remove the class. The `is_contract_class_declared()` / `is_declared()` check is the only gate that prevents the class manager from serving a class that no longer exists at the queried state. [3](#0-2) 

### Impact Explanation

**RPC execution path (High):** `ExecutionStateReader` with `class_manager_handle` is used for RPC fee estimation, simulation, tracing, and pending-block execution. If a Cairo 0 class was declared in a reverted block, RPC calls querying a state number before or after the revert would receive the class from the class manager and execute against it, producing authoritative-looking but wrong fee estimates, simulation results, and traces.

**Blockifier execution path (Critical):** `StateReaderAndContractManager` is used during actual block execution. A Cairo 0 class present in the contract class manager cache but not declared at the current state would be returned to the blockifier, causing a transaction that should revert with `UndeclaredClassHash` to instead execute successfully. This produces wrong state transitions, wrong receipts, and wrong events.

### Likelihood Explanation

The trigger requires a Cairo 0 class to be present in the class manager but not declared at the queried `state_number`. This occurs naturally during:
- Block reverts / reorgs (the class manager retains the class after the block is rolled back)
- RPC queries against historical state numbers that predate the declaration block

The class manager is populated at gateway ingestion time, independently of block finality. Any node that processes a Cairo 0 declare transaction and then experiences a revert is in the vulnerable state. The TODO comment confirms the developers are aware the check is missing and deferred it.

### Recommendation

Apply the same `is_contract_class_declared()` / `is_declared()` guard to the `ContractClass::V0` / `RunnableCompiledClass::V0` branches in both locations:

1. In `ExecutionStateReader::get_compiled_class()` (`crates/apollo_rpc_execution/src/state_reader.rs`, lines 136–140): call `is_contract_class_declared()` for `ContractClass::V0` and return `Err(StateError::UndeclaredClassHash(class_hash))` if the class is not declared at `self.state_number`.

2. In `StateReaderAndContractManager::get_compiled_from_class_manager()` (`crates/blockifier/src/state/state_reader_and_contract_manager.rs`, lines 73–74): call `self.state_reader.is_declared(class_hash)` for `RunnableCompiledClass::V0(_)` and return `Err(StateError::UndeclaredClassHash(class_hash))` if false.

The prerequisite fix mentioned in the TODO — resolving `get_class_definition_block_number` for deprecated classes — is already available via `get_deprecated_class_definition_block_number()` in `crates/apollo_storage/src/state/mod.rs`. [4](#0-3) 

### Proof of Concept

1. Submit a Cairo 0 declare transaction to the gateway. The class is added to the class manager.
2. The block containing the declaration is committed, then reverted (e.e., via `revert_state_diff`).
3. The class manager still holds the class (no eviction on revert).
4. Issue an RPC `starknet_call` / `starknet_estimateFee` / `starknet_simulateTransactions` against the latest block (post-revert state), calling a contract that invokes the reverted Cairo 0 class hash.
5. `ExecutionStateReader::get_compiled_class()` is called with `class_manager_handle = Some(...)`. The class manager returns `ContractClass::V0(...)`. The missing check means the class is returned without verifying it is declared at the current `state_number`.
6. The call executes successfully and returns a result, when the correct behavior is `StateError::UndeclaredClassHash` → RPC error. [5](#0-4) [6](#0-5)

impl<S: FetchCompiledClasses> StateReaderAndContractManager<S> {
    fn get_compiled_from_class_manager(
        &self,
        class_hash: ClassHash,
    ) -> StateResult<RunnableCompiledClass> {
        if let Some(runnable_class) =
            self.contract_class_manager.get_runnable(&class_hash, &self.native_classes_whitelist)
        {
            match &runnable_class {
                RunnableCompiledClass::V0(_) => {}
                _ => {
                    // The Cairo1 class is cached; verify it is declared,
                    // since existence in the cache does not guarantee that
                    // (it might contain a declared class from a reverted block, for example).
                    if !self.state_reader.is_declared(class_hash)? {
                        return Err(StateError::UndeclaredClassHash(class_hash));
                    }
                }
            }
            self.increment_cache_hit_metric();
            self.update_native_metrics(&runnable_class);
            return Ok(runnable_class);
        }
        self.increment_cache_miss_metric();

        let compiled_class = self.state_reader.get_compiled_classes(class_hash)?;
        self.contract_class_manager.set_and_compile(class_hash, compiled_class.clone());
        // Access the cache again in case the class was compiled.
        let runnable_class = self
            .contract_class_manager
            .get_runnable(&class_hash, &self.native_classes_whitelist)
            .unwrap_or_else(|| {
                // Edge case that should not be happen if the cache size is big enough.
                // TODO(Yoni): consider having an atomic set-and-get.
                log::error!("Class is missing immediately after being cached.");
                compiled_class.to_runnable()
            });
        self.update_native_metrics(&runnable_class);
        Ok(runnable_class)
    }
