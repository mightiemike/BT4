### Title
Borrow Normalization Rounds Toward Zero on Partial Repayment, Understating Borrower Debt - (File: `core/contracts/SpotEngineState.sol`)

### Summary
In `SpotEngineState._updateBalanceNormalized`, when a user partially repays a spot borrow (negative balance), the new normalized borrow amount is computed via `newAmount.div(cumulativeMultiplierX18)`. Because `newAmount` is negative and `cumulativeBorrowsMultiplierX18` is positive, Solidity's fixed-point division truncates toward zero, producing a result that is slightly less negative than the true value. This understates the borrower's remaining debt by a dust amount on every partial repayment, which accumulates as a loss for depositors over time.

### Finding Description
The normalized balance accounting in `SpotEngineState` stores each user's position as `amountNormalized`, which is scaled by a cumulative multiplier to derive the actual balance. When a user partially repays a borrow (deposits collateral that reduces but does not eliminate a negative spot balance), `_updateBalanceNormalized` executes:

```solidity
// SpotEngineState.sol line 33-43
int128 newAmount = balance.amountNormalized.mul(
    cumulativeMultiplierX18
) + balanceDelta;

if (newAmount > 0) {
    cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
} else {
    cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
}

balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);
```

After a partial repayment, `newAmount` is still negative (remaining borrow). `MathSD21x18.div` performs fixed-point division as `(a * ONE) / b`. For negative `a` and positive `b`, Solidity integer division truncates toward zero, yielding a result that is less negative than the exact quotient. The stored `amountNormalized` is therefore slightly less negative than it should be — the borrower's debt is understated by a dust amount.

When interest accrues and `cumulativeBorrowsMultiplierX18` grows, the actual balance reconstructed as `amountNormalized.mul(cumulativeBorrowsMultiplierX18)` is slightly smaller in magnitude than the true debt. The shortfall is never recovered, and depositors receive correspondingly less.

### Impact Explanation
Each partial repayment shaves a dust amount off the borrower's recorded debt. Over many repayments across many accounts, the aggregate shortfall grows and is borne by depositors (lenders), who receive less interest than they are owed. The impact per event is low (sub-unit dust), but the cumulative effect is a persistent, irrecoverable loss for the depositor pool.

**Impact: Low**

### Likelihood Explanation
Every user who partially repays a spot borrow triggers this path. Partial repayments are the normal operating mode for any account that incrementally deposits collateral or reduces leverage. No special conditions or privileges are required — any user interacting through `depositCollateral` or any `updateBalance` call that partially covers a negative spot balance will trigger the rounding error.

**Likelihood: High**

### Recommendation
When computing the new normalized borrow amount for a negative `newAmount`, round away from zero (i.e., round down in the negative direction) rather than toward zero. This ensures the borrower's recorded debt is never understated. Concretely, after the division, if the product `balance.amountNormalized * cumulativeBorrowsMultiplierX18` does not exactly equal `newAmount`, decrement `amountNormalized` by one unit (making it more negative) to round up the magnitude of the debt.

### Proof of Concept
1. Alice has a spot borrow: `amountNormalized = -1000`, `cumulativeBorrowsMultiplierX18 = 1.5e18` → actual debt = `1500`.
2. Alice deposits `700` (partial repayment). `newAmount = -1000 * 1.5 + 700 = -800`.
3. New normalized borrow: `(-800e18) / (1.5e18)`. Exact value = `-533.333...`. Solidity truncates toward zero → `-533`.
4. Stored: `amountNormalized = -533`. Reconstructed debt = `533 * 1.5 = 799.5`, but Alice actually owes `800`.
5. The `0.5` unit shortfall is never recovered. Repeated across thousands of partial repayments, depositors lose a meaningful aggregate amount. [1](#0-0) [2](#0-1)

    function _updateBalanceNormalized(
        State memory state,
        BalanceNormalized memory balance,
        int128 balanceDelta
    ) internal pure {
        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized -= balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized += balance.amountNormalized;
        }

        int128 cumulativeMultiplierX18;
        if (balance.amountNormalized > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        int128 newAmount = balance.amountNormalized.mul(
            cumulativeMultiplierX18
        ) + balanceDelta;

        if (newAmount > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);

        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized += balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized -= balance.amountNormalized;
        }
    }
