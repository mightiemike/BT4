### Title
Retroactive BonusApy Application on Governance Tier Update Enables Front-Running for Excess Bonus Extraction — (`File: x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

When governance executes a `SetTier` update that raises a tier's `BonusApy`, the `processEventsAndClaimBonus` function applies the new rate retroactively to all unclaimed bonus segments for every existing position in that tier. Because `BonusApy` is never snapshotted per time-segment (unlike `TokensPerShare`), an unprivileged attacker who observes the governance proposal in the voting period can front-run by creating a large position, wait for execution, and then claim rewards at the elevated rate for the entire period since position creation — extracting more from the `RewardsPool` than they are entitled to.

---

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` fetches the tier **once at claim time** and passes it to `computeSegmentBonus` for every historical bonded segment:

```go
tier, err := k.getTier(ctx, pos.TierId)   // current tier, post-governance
...
for _, entry := range events {
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
    ...
}
// final open segment
bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
``` [1](#0-0) 

Inside `computeSegmentBonus`, the rate used is `tier.BonusApy` — the value stored in the tier at the moment of the call, not the value that was in effect when each segment accrued:

```go
return tokens.
    Mul(tier.BonusApy).
    MulInt64(durationSeconds).
    QuoInt64(types.SecondsPerYear).
    TruncateInt()
``` [2](#0-1) 

By contrast, `TokensPerShare` **is** snapshotted per segment via validator events, so slashing is accounted for historically. `BonusApy` has no equivalent snapshot mechanism. `SetTier`, which is explicitly documented as the governance update path, overwrites the stored tier in place:

```go
// SetTier writes a tier after validation. Used by governance messages, genesis, and chain upgrades.
func (k Keeper) SetTier(ctx context.Context, tier types.Tier) error {
    if err := tier.Validate(); err != nil { return err }
    if err := k.Tiers.Set(ctx, tier.Id, tier); err != nil { ... }
    return nil
}
``` [3](#0-2) 

There is no historical record of what `BonusApy` was during any past segment. Once the tier is updated, every future call to `processEventsAndClaimBonus` for any position in that tier — regardless of when the position was created — uses the new rate for all unclaimed history.

---

### Impact Explanation

The exact corrupted value is the `RewardsPoolName` module account balance. When governance raises `BonusApy` from, say, 5 % to 20 %, every existing position with unclaimed bonus receives 20 % APY for the entire unclaimed period, not just from the governance execution block. The attacker who front-ran with a large position extracts a disproportionate share of the pool. Other legitimate holders who claim later may find the pool partially or fully drained, causing `ErrInsufficientBonusPool` failures for them. The protocol's intended reward budget is violated. [4](#0-3) 

---

### Likelihood Explanation

Cosmos SDK governance proposals are fully public on-chain during the deposit and voting periods (typically days to weeks). Any observer can submit `MsgLockTier` or `MsgCommitDelegationToTier` before the proposal passes — no special privilege is required. The attacker only needs to hold enough tokens to meet the tier's `MinLockAmount`. The longer the voting period and the larger the BonusApy increase, the greater the extractable profit. [5](#0-4) 

---

### Recommendation

Snapshot `BonusApy` per time-segment the same way `TokensPerShare` is snapshotted via validator events. Concretely: when `SetTier` is called with a changed `BonusApy`, emit a new validator event (or a dedicated tier-rate-change event) for every validator that has active positions in that tier, recording the old `BonusApy` and the block timestamp. `processEventsAndClaimBonus` should then use the snapshotted rate for each historical segment rather than the current tier value. Alternatively, force-claim all outstanding bonus for every position in the tier atomically inside `SetTier` before writing the new rate, so no historical accrual is ever computed at the new rate. [6](#0-5) 

---

### Proof of Concept

1. Governance proposal is submitted to raise Tier 1 `BonusApy` from 5 % to 25 %. Voting period begins (e.g., 14 days).

2. Attacker observes the proposal on-chain and immediately submits `MsgLockTier` with the maximum allowed amount for Tier 1. Position is created; `LastBonusAccrual` is set to the current block time.

3. Governance voting period elapses; proposal passes and `SetTier` overwrites Tier 1's `BonusApy` to 25 % in the KV store.

4. Attacker immediately submits `MsgClaimTierRewards`. `processEventsAndClaimBonus` fetches the tier — now with `BonusApy = 0.25` — and computes bonus for the entire 14-day accrual window at 25 % APY.

5. Legitimate users who held positions for the same 14 days expecting 5 % APY also receive 25 % APY on their unclaimed history, but the attacker's position was sized to maximize extraction. If the pool is insufficient, subsequent claimants receive `ErrInsufficientBonusPool`. [7](#0-6) [8](#0-7)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (ms msgServer) LockTier(ctx context.Context, msg *types.MsgLockTier) (*types.MsgLockTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, msg.Id)
	if err != nil {
		return nil, err
	}

	if err := ms.validateNewPosition(ctx, msg.Owner, msg.Amount, tier); err != nil {
		return nil, err
	}

	valAddr, err := sdk.ValAddressFromBech32(msg.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(msg.Owner)
	if err != nil {
		return nil, err
	}

	id, err := ms.NextPositionId.Peek(ctx)
	if err != nil {
		return nil, err
	}
	delAddr, err := ms.createPositionDelegatorAccount(ctx, ownerAddr, id)
	if err != nil {
		return nil, err
	}

	if err := ms.lockFunds(ctx, ownerAddr, delAddr, msg.Amount); err != nil {
		return nil, err
	}

	if _, err := ms.delegate(ctx, delAddr, valAddr, msg.Amount); err != nil {
		return nil, err
	}

	pos, err := ms.createDelegatedPosition(ctx, msg.Owner, tier, valAddr, delAddr, msg.TriggerExitImmediately)
	if err != nil {
		return nil, err
	}

	// Defensive, but should not happen since transactions are sequential
	if pos.Id != id {
		return nil, errorsmod.Wrapf(types.ErrInvalidPositionID, "position id mismatch: peeked %d, created %d", id, pos.Id)
	}

	if err := ms.setPosition(ctx, pos, &ValidatorTransition{PreviousAddress: ""}); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventPositionCreated{
		Position: pos,
	}); err != nil {
		return nil, err
	}

	return &types.MsgLockTierResponse{PositionId: pos.Id}, nil
}
