### Title
`ValidResourceBounds::AllResources` with zero `l2_gas`/`l1_data_gas` is silently downgraded to `L1Gas` in protobuf round-trip, producing a divergent transaction hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf serializer for `ValidResourceBounds` always emits a non-`None` `l1_data_gas` field (set to all-zeros) for the `L1Gas` variant. The deserializer then uses a zero-value heuristic to reconstruct the variant. A valid post-0.13.3 `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` transaction is accepted by the gateway and hashed with the `AllResources` path. After a P2P block-sync round-trip, the same transaction is reconstructed as `L1Gas(X)`, and any subsequent hash recomputation omits the `L1_DATA` element from the preimage, producing a different hash. This breaks the hash-binding invariant across the P2P sync boundary.
