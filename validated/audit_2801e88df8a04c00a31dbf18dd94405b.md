### Title
`AllResources`→`L1Gas` Silent Normalization in Protobuf Deserialization Produces Wrong Transaction Hash During P2P Block Sync — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion in the P2P block-sync path silently normalizes an `AllResources` bound whose `l2_gas` and `l1_data_gas` are both zero into `L1Gas`. Because `get_tip_resource_bounds_hash` hashes a **different number of elements** for `L1Gas` (3 elements) versus `AllResources` (4 elements), a transaction whose hash was computed at ingestion time under `AllResources` will produce a **different hash** when the same transaction is deserialized during block sync. This is the direct sequencer analog of the Wormhole dust-trimming bug: a normalization applied at one layer (protobuf deserialization) is not accounted for at the hash-computation layer, producing a permanent, silent divergence.

---

### Finding Description

**Layer 1 — Hash computation at ingestion (correct variant)**

When a V3 transaction is submitted via the gateway, `RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`. The converter calls `calculate_transaction_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`:

```
// crates/starknet_api/src/transaction_hash.rs  lines 188-210
resource_felts = [l1_gas_packed, l2_gas_packed];
// AllResources branch: appends l1_data_gas_packed
resource_felts.push(l1_data_gas_packed);
hash = Poseidon(tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed)  // 4 elements
``` [1](#0-0) 

For a transaction with `l2_gas = 0` and `l1_data_gas = 0`, the hash is still computed over **4 elements** because the variant is `AllResources`.

**Layer 2 — Protobuf deserialization during block sync (normalizing variant)**

When the same block is received by a syncing peer, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` is invoked:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← normalization: drops to L1Gas
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

When both fields are zero, the variant is silently changed to `L1Gas`.

**Layer 3 — Hash recomputed on the receiving side (wrong variant)**

`get_tip_resource_bounds_hash` for `L1Gas` omits the `l1_data_gas` element:

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],          // ← no l1_data_gas
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
``` [3](#0-2) 

The receiving node computes:
```
hash = Poseidon(tip, l1_gas_packed, l2_gas_packed)  // 3 elements — DIFFERENT
```

The Poseidon hash over 3 elements is cryptographically distinct from the Poseidon hash over 4 elements even when the 4th element is zero. The transaction hash stored in the block and the hash recomputed by the syncing node diverge.

**Concrete divergence**

| Side | `ValidResourceBounds` variant | Hash preimage elements |
|---|---|---|
| Originating sequencer | `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` | `[tip, l1_gas_packed, 0, 0]` — 4 |
| Syncing peer | `L1Gas(l1_gas: X)` | `[tip, l1_gas_packed, 0]` — 3 |

**Triggering condition**

A V3 transaction with `AllResourceBounds { l1_gas: { max_amount > 0, max_price_per_unit > 0 }, l2_gas: ZERO, l1_data_gas: ZERO }` passes the gateway's stateless resource-bounds check because `max_possible_fee = l1_gas.max_amount × l1_gas.max_price_per_unit > 0`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs  lines 64-68
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(ZeroResourceBounds { resource_bounds });
}
``` [4](#0-3) 

The stateless validator also checks `l2_gas.max_price_per_unit >= min_gas_price`; with `min_gas_price = 0` (a valid configuration) this passes. The stateful `validate_tx_l2_gas_price_within_threshold` similarly passes when `min_gas_price_percentage = 0` or when the network l2 gas price is zero (e.g., early network bootstrap). [5](#0-4) 

---

### Impact Explanation

Once the transaction is included in a block and the block is synced:

1. **Wrong transaction hash** — the syncing node recomputes a hash that differs from the one committed in the block's transaction commitment tree, causing the block hash computed by the syncing node to diverge from the canonical block hash.
2. **Wrong `GasVectorComputationMode`** — `L1Gas` selects `NoL2Gas` mode while `AllResources` selects `All` mode, changing fee computation, pre-validation resource checks, and execution gas accounting for the replayed transaction.
3. **Wrong receipt/state** — downstream execution of the deserialized transaction uses incorrect resource bounds, producing wrong actual-fee, wrong revert decisions, and wrong state diffs.

This matches the allowed impact: *"Critical. Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"* and *"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

The trigger requires a transaction with non-zero `l1_gas` bounds and zero `l2_gas`/`l1_data_gas` bounds to be accepted and included in a block. This is possible when:
- `min_gas_price = 0` in the stateless validator config, or
- `min_gas_price_percentage = 0` in the stateful validator config, or
- The network l2 gas price is zero (bootstrap phase).

These are realistic configurations in test networks, staging environments, and early mainnet phases. The normalization in the protobuf deserializer is unconditional and requires no special attacker capability — any conforming V3 transaction with the described bounds triggers it.

---

### Recommendation

Remove the silent normalization from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Either:

1. **Always deserialize as `AllResources`** when all three fields are present (matching the serialization side which always emits all three fields), or
2. **Preserve the variant faithfully** by adding an explicit discriminator field to the protobuf message so the deserializer knows which variant was originally used.

The serialization side (`From<ValidResourceBounds> for protobuf::ResourceBounds`) already emits all three fields for both variants:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 471-489
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),
    l1_data_gas: Some(ResourceBounds::default().into()),  // always emitted
},
``` [6](#0-5) 

The round-trip is already lossy: `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` serializes identically to `L1Gas(X)`. The fix must either add a discriminator or always deserialize as `AllResources` when all three fields are present, matching the hash-computation invariant.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, GasAmount, GasPrice, ResourceBounds, ValidResourceBounds,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

fn main() {
    let l1_gas = ResourceBounds {
        max_amount: GasAmount(100),
        max_price_per_unit: GasPrice(1),
    };
    let zero = ResourceBounds::default();

    // Variant produced at ingestion time (AllResources)
    let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas,
        l2_gas: zero,
        l1_data_gas: zero,
    });

    // Variant produced after protobuf round-trip (L1Gas — the normalization)
    let l1_gas_only = ValidResourceBounds::L1Gas(l1_gas);

    let tip = Tip(0);
    let hash_all = get_tip_resource_bounds_hash(&all_resources, &tip).unwrap();
    let hash_l1  = get_tip_resource_bounds_hash(&l1_gas_only,  &tip).unwrap();

    // These are different despite identical numeric values
    assert_ne!(hash_all, hash_l1,
        "Hash mismatch: AllResources({:?}) != L1Gas({:?})", hash_all, hash_l1);
}
```

The `assert_ne!` passes, demonstrating that the same numeric resource bounds produce different transaction hash preimages depending solely on which `ValidResourceBounds` variant is used — and the protobuf deserializer silently selects the wrong variant for any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are zero.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {

        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });
