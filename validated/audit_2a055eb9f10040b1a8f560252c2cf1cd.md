### Title
Missing `SetWithdrawAddr` in `backfillDelegatorAddress` Permanently Misdirects Base Rewards for All Legacy Positions After v8 Migration — (`x/tieredrewards/migrations/v2/migrate.go`)

---

### Summary

`backfillDelegatorAddress` writes `pos.DelegatorAddress = LegacyDelegatorAddress(pos.Id)` for every pre-v8 position but never calls `distributionKeeper.SetWithdrawAddr(posDelAddr, ownerAddr)`. After migration, every call to `claimBaseRewards` on a legacy position invokes `distributionKeeper.WithdrawDelegationRewards(posDelAddr, valAddr)`, which — because no withdraw address was ever registered — sends rewards to `posDelAddr` itself: a module-derived address with no private key. The funds are permanently inaccessible.

---

### Finding Description

**New position path (post-v8, correct):**

`MsgLockTier` → `createDelegatedPosition` calls `routeBaseRewardsToOwner(posDelAddr, ownerAddr)` which calls `distributionKeeper.SetWithdrawAddr(posDelAddr, ownerAddr)`. [1](#0-0) 

**Migration path (broken):**

`Migrate1to2` → `v2.Migrate` → `backfillDelegatorAddress` sets `pos.DelegatorAddress` and calls `positions.Set(ctx, pos.Id, pos)`. No `SetWithdrawAddr` call is made anywhere in the migration for non-vesting positions. [2](#0-1) 

**Claim path (relies on invariant that was never established):**

`claimBaseRewards` explicitly documents its assumption: *"This assumes that the delegation withdrawAddress has been set to the position's owner address."* It then calls `distributionKeeper.WithdrawDelegationRewards(posDelAddr, valAddr)`. [3](#0-2) 

In Cosmos SDK's distribution module, when no withdraw address has been registered for a delegator, `WithdrawDelegationRewards` sends rewards to the delegator address itself — here `posDelAddr`.

**`posDelAddr` is unspendable:**

`LegacyDelegatorAddress` derives the address as `authtypes.NewModuleAddress("tieredrewards/position/<id>")`. This is a deterministic hash with no corresponding private key. Any funds sent there are permanently locked. [4](#0-3) 

**Vesting positions are also affected during migration:**

`exitVestedAccountsPositions` calls `ForceFullExitWithDelegation`, which calls `claimRewards` → `claimBaseRewards` before deleting the position. The base rewards accumulated pre-migration for vesting positions are also sent to `posDelAddr` and lost. [5](#0-4) 

---

### Impact Explanation

Every non-vesting legacy position permanently loses all base rewards claimed via `MsgClaimTierRewards` after the v8 upgrade. The rewards are sent to `posDelAddr` — a keyless module address — on every claim. The loss is ongoing and cumulative: each `ClaimTierRewards` call silently drains base rewards into an unrecoverable address. The owner's bank balance never increases from base rewards. This matches the **Critical** scope: fund loss via migration/upgrade path, permanent and affecting all legacy positions.

---

### Likelihood Explanation

The precondition is simply "chain running pre-v8 with at least one non-vesting position," which is the normal production state. The upgrade executes automatically. No attacker action is required — the loss occurs on the first `MsgClaimTierRewards` submitted by any position owner after the upgrade. The existing migration test `TestMigrate1to2_BackfillsPositionsWithDelegatorAddress` only asserts that `DelegatorAddress` is set; it does not assert that `GetDelegatorWithdrawAddr(posDelAddr) == ownerAddr`, so the gap is untested. [6](#0-5) 

---

### Recommendation

In `backfillDelegatorAddress`, after setting `pos.DelegatorAddress`, call `distributionKeeper.SetWithdrawAddr(posDelAddr, ownerAddr)` for every non-vesting position that has an active delegation. The `v2.Migrate` function signature must be extended to accept a `DistributionKeeper` interface (mirroring the pattern already used in `position.go`). The migration test should be extended to assert `GetDelegatorWithdrawAddr(posDelAddr) == ownerAddr` after `Migrate1to2`. [7](#0-6) 

---

### Proof of Concept

```
1. Pre-upgrade: chain has position id=1, owner=Alice, DelegatorAddress="" (v1 state),
   delegation at posDelAddr=LegacyDelegatorAddress(1), no SetWithdrawAddr ever called.

2. v8 upgrade executes Migrate1to2:
   - backfillDelegatorAddress sets pos.DelegatorAddress = LegacyDelegatorAddress(1)
   - positions.Set(ctx, 1, pos)  ← no SetWithdrawAddr call

3. Distribution module state: GetDelegatorWithdrawAddr(posDelAddr) → posDelAddr (default)

4. Alice submits MsgClaimTierRewards{position_ids: [1]}:
   - claimBaseRewards(pos) is called
   - distributionKeeper.WithdrawDelegationRewards(posDelAddr, valAddr)
   - SDK distribution: withdraw addr for posDelAddr = posDelAddr (no override set)
   - Rewards R sent to posDelAddr
   - Alice's bank balance: unchanged
   - posDelAddr balance: +R (permanently locked, no private key)

5. Assert: Alice.balance.bondDenom == 0 (expected > 0)
   Assert: posDelAddr.balance.bondDenom == R (funds locked forever)
```

func backfillDelegatorAddress(ctx context.Context, positions collections.Map[uint64, types.Position]) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	sdkCtx.Logger().Info("v8 migration: backfill delegator address")
	iter, err := positions.Iterate(ctx, nil)
	if err != nil {
		return err
	}
	defer iter.Close()

	kvs, err := iter.KeyValues()
	if err != nil {
		return err
	}

	for _, kv := range kvs {
		pos := kv.Value
		pos.DelegatorAddress = LegacyDelegatorAddress(pos.Id)
		if _, err := sdk.AccAddressFromBech32(pos.DelegatorAddress); err != nil {
			return fmt.Errorf("backfill produced invalid delegator address for position %d: %w", pos.Id, err)
		}
		if err := positions.Set(ctx, pos.Id, pos); err != nil {
			return err
		}
	}
	sdkCtx.Logger().Info("v8 migration: delegator address backfilled", "count", len(kvs))
	return nil
}

func (k Keeper) claimBaseRewards(ctx context.Context, pos types.PositionState) (sdk.Coins, error) {
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}
	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}
	posDelAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	rewards, err := k.distributionKeeper.WithdrawDelegationRewards(ctx, posDelAddr, valAddr)
