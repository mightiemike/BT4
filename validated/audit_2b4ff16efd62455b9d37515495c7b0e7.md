### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` Blocks Finalization of the Last Perp Market Participant — (File: `core/contracts/PerpEngine.sol`)

---

### Summary

When the last active participant in a perp market is being finalized through `ClearinghouseLiq._finalizeSubaccount`, the call to `PerpEngine.socializeSubaccount` will panic with a division-by-zero if the subaccount still carries negative `vQuoteBalance` that exceeds insurance coverage, because `state.openInterest` is zero once all positions in that market have been closed.

---

### Finding Description

`ClearinghouseLiq._finalizeSubaccount` is the settlement path invoked when a liquidator submits `productId == type(uint32).max`. Before calling `perpEngine.socializeSubaccount`, it first asserts that every perp position on the liquidatee has already been reduced to zero:

```solidity
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [1](#0-0) 

Only after that check passes does it call:

```solidity
v.insurance = perpEngine.socializeSubaccount(txn.liquidatee, v.insurance);
``` [2](#0-1) 

Inside `PerpEngine.socializeSubaccount`, when insurance is insufficient to cover the remaining negative `vQuoteBalance`, the residual loss is spread across all other open-interest holders via:

```solidity
int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
``` [3](#0-2) 

The invariant assumed here is that `state.openInterest > 0` — i.e., there are other participants to absorb the loss. However, if the subaccount being finalized was the **last** participant in that perp market, closing their position (required before finalization) reduces `openInterest` to zero. The subsequent `div(0)` causes a Solidity panic (error code `0x12`), reverting the entire finalization transaction.

This is structurally identical to the Bima analog: in Bima, `_redistributeDebtAndColl` divides by `totalStakes` which is zero when the last trove is liquidated; in Nado, `socializeSubaccount` divides by `openInterest` which is zero when the last perp position holder is finalized.

---

### Impact Explanation

A subaccount that is the sole remaining participant in a perp market — or the last one after all others have exited — **cannot be finalized** if its negative `vQuoteBalance` exceeds the insurance fund. The finalization transaction always reverts with a panic. The protocol is left holding an insolvent, permanently un-closeable subaccount. Any collateral locked in that subaccount is irrecoverable, and the bad debt cannot be socialized, corrupting the accounting state of the perp market indefinitely.

---

### Likelihood Explanation

This is reachable in any perp market that has low participation or is winding down. Concretely:

1. A user opens a position in a new or low-liquidity perp market (counterparty is `X_ACCOUNT` or a small set of traders who later exit).
2. The price moves adversely; the user accumulates a large negative `vQuoteBalance`.
3. A liquidator closes the user's position (regular liquidation step), reducing `openInterest` to zero.
4. The liquidator then attempts to finalize the subaccount (`productId = type(uint32).max`).
5. `socializeSubaccount` panics on `div(0)`.

No privileged access is required. Any liquidator can trigger this path. The scenario is especially likely during market deprecation or in early-stage markets with a single dominant participant.

---

### Recommendation

Guard the socialization division against a zero denominator in `PerpEngine.socializeSubaccount`:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No other participants; absorb residual loss into the insurance fund
        // or write it off directly without spreading.
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

The protocol must also decide the accounting policy for the unabsorbed loss (e.g., charge it to the insurance fund, write it off, or revert with a meaningful error rather than a panic).

---

### Proof of Concept

1. Deploy the protocol with a single perp product.
2. Have `userA` open a long position against `X_ACCOUNT` (short). `openInterest` = `|userA.amount|`.
3. Drop the oracle price so `userA` is under maintenance margin.
4. Liquidator calls `liquidateSubaccountImpl` with the perp `productId` to close `userA`'s position. `userA.amount` → 0; `openInterest` → 0. `userA.vQuoteBalance` is now a large negative number.
5. Liquidator calls `liquidateSubaccountImpl` with `productId = type(uint32).max` (finalize).
6. `_finalizeSubaccount` passes the `balance.amount == 0` check, then calls `perpEngine.socializeSubaccount`.
7. Inside `socializeSubaccount`: `balance.vQuoteBalance < 0`, insurance is insufficient, execution reaches `fundingPerShare = -balance.vQuoteBalance.div(state.openInterest)` where `state.openInterest == 0` → **panic revert**.
8. `userA` can never be finalized; the bad debt is permanently stuck. [4](#0-3) [5](#0-4)

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }

    function socializeSubaccount(bytes32 subaccount, int128 insurance)
        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough
                if (balance.vQuoteBalance < 0) {
                    // socialize across all other participants
                    int128 fundingPerShare = -balance.vQuoteBalance.div(
                        state.openInterest
                    );
                    state.cumulativeFundingLongX18 += fundingPerShare;
                    state.cumulativeFundingShortX18 -= fundingPerShare;
                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;
    }
