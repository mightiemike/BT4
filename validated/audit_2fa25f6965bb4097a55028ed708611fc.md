Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Economic Actor, Allowing Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the user. If the pool admin allowlists the router (a necessary step to permit any router-mediated swaps for allowed users), every user—including disallowed ones—can bypass the allowlist by routing through the router. The allowlist policy is therefore unenforceable on the supported public periphery path.

## Finding Description

**Root cause — wrong actor checked:**

`ExtensionCalling._beforeSwap` passes `msg.sender` of `MetricOmmPool.swap()` as the `sender` argument to every extension:

```solidity
// MetricOmmPool.sol::swap (line 230-240)
_beforeSwap(
    msg.sender,   // <-- whoever called pool.swap()
    recipient, ...
);

// ExtensionCalling.sol::_beforeSwap (line 162-176)
abi.encodeCall(IMetricOmmExtensions.beforeSwap,
    (sender, recipient, ...))
```

`SwapAllowlistExtension.beforeSwap` then checks that exact value:

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct for namespacing), and `sender` is whoever called `pool.swap()`.

**Router path:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol line 72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
```

The router does **not** forward the original `msg.sender` (the user) to the pool. The pool sees `msg.sender = router`. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**Exploit flow:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured, intending to restrict swaps to a curated set of addresses.
2. Pool admin allowlists the router address so that allowed users can reach the pool via the standard periphery path.
3. A disallowed user calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
4. The router calls `pool.swap(recipient, ...)` — pool sees `msg.sender = router`.
5. `_beforeSwap(router, ...)` is dispatched; extension checks `allowedSwapper[pool][router]` → `true`.
6. The disallowed user's swap executes successfully, bypassing the allowlist entirely.

**Existing guards are insufficient:**

The `_requireFactoryPool` check in the router only validates that the pool is a factory-registered pool; it does not propagate the original caller identity. There is no mechanism in the pool or extension framework to recover the original EOA once the router is the immediate caller.

## Impact Explanation
A curated pool's swap allowlist is completely bypassed for any user who routes through `MetricOmmSimpleRouter`. Disallowed actors can execute swaps on pools that are supposed to be restricted, violating the pool's access-control policy. This constitutes a broken core pool functionality / admin-boundary break: the pool admin's configured guard fails to gate the economically relevant actor, and any disallowed user can trade on a pool they should be excluded from.

## Likelihood Explanation
The bypass requires only that the router is allowlisted for the pool—a condition the pool admin must satisfy to make the pool usable via the standard periphery for any allowed user. The attack is permissionless, requires no special capital or timing, and is repeatable on every swap. Any unprivileged user can execute it by calling the public router.

## Recommendation
The extension should check the original user identity rather than the immediate pool caller. Two viable approaches:

1. **Pass the original caller through the pool:** Add an `originator` field to the swap call or extension data so the pool can forward the true initiating address to extensions. The extension then checks `allowedSwapper[pool][originator]`.
2. **Require direct pool interaction for allowlisted pools:** Document and enforce that pools using `SwapAllowlistExtension` must not allowlist the router; instead, allowed users must call `pool.swap()` directly. This is a documentation/configuration fix but does not change the code-level vulnerability.

The cleanest fix is option 1: thread the original `msg.sender` through the router into the pool's extension context so the allowlist can gate the true economic actor.

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlistBypassViaRouter() public {
    // 1. Deploy pool with SwapAllowlistExtension
    SwapAllowlistExtension ext = new SwapAllowlistExtension(address(factory));
    MetricOmmPool pool = _deployPoolWithSwapAllowlist(ext);

    address allowedUser   = makeAddr("allowed");
    address disallowedUser = makeAddr("disallowed");

    // 2. Admin allowlists the router (required for periphery use) and one user
    ext.setAllowedToSwap(address(pool), address(router), true);
    ext.setAllowedToSwap(address(pool), allowedUser, true);
    // disallowedUser is NOT allowlisted

    // 3. Direct swap by disallowedUser reverts correctly
    vm.prank(disallowedUser);
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    pool.swap(disallowedUser, true, 1000, type(uint128).max, "", "");

    // 4. Router-mediated swap by disallowedUser succeeds — bypass confirmed
    _dealAndApprove(disallowedUser, address(token0), 1e18);
    vm.prank(disallowedUser);
    // Should revert but does NOT
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        recipient: disallowedUser,
        deadline: block.timestamp + 1,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: true,
        priceLimitX64: 0,
        extensionData: ""
    }));
}
```
