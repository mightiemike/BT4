### Title
Crafted ClassId in IBC Packet Bypasses `IsAwayFromOrigin` Direction Check, Enabling Unauthorized Unescrow of NFTs - (`x/nft-transfer/keeper/packet.go`)

---

### Summary

An attacker controlling an IBC counterparty chain can craft a `NonFungibleTokenPacketData` with a `ClassId` prefixed by their own source port/channel (e.g., `nft/channel-X/realClass`). This causes `IsAwayFromOrigin` in `processReceivedPacket` to return `false` (sink/return-to-origin direction), triggering the unescrow branch. The unescrow branch then calls `TransferOwner` from the victim chain's escrow address (keyed on the **destination** port/channel) to the attacker's receiver — stealing any NFT of that class held by that escrow address, even though no corresponding `SendTransfer` was ever initiated by the attacker.

---

### Finding Description

**Root cause — `IsAwayFromOrigin` uses attacker-controlled `data.ClassId` against the packet's source port/channel:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

In `processReceivedPacket`, this is called as: [2](#0-1) 

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
```

The direction check compares `data.ClassId` (fully attacker-controlled packet payload) against the **source** port/channel (the attacker's own port/channel). The attacker simply sets `data.ClassId = "nft/channel-X/realClass"` where `nft/channel-X` is their own source port/channel. `HasPrefix` returns `true`, so `IsAwayFromOrigin` returns `false`.

**The unescrow branch then executes:** [3](#0-2)

The `escrowAddress` is `GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())` — the victim chain's escrow for its own channel-Y. If `realClass/T1` was legitimately escrowed there from a prior `SendTransfer`, `TransferOwner` succeeds.

**`TransferOwner` only checks that `srcOwner` is the actual current owner:** [4](#0-3) 

It will succeed if the escrow address holds the NFT — which it does under the stated precondition.

**`ValidateBasic` imposes no constraint on `ClassId` format relative to the packet's source port/channel:** [5](#0-4) 

Only a non-blank check is performed. Any `ClassId` string passes.

---

### Impact Explanation

An attacker controlling a counterparty IBC chain can steal any NFT of a native class that is currently held in the victim chain's escrow address for a given channel. The escrow address accumulates NFTs whenever users on the victim chain send native NFTs outbound via `SendTransfer`. The attacker can drain all such escrowed NFTs by crafting packets with `ClassId = "<attacker_src_port>/<attacker_src_channel>/<native_class>"` and `TokenIds` matching the escrowed tokens.

---

### Likelihood Explanation

The precondition — that the victim chain's escrow address holds at least one native NFT — is satisfied any time a user has sent a native NFT outbound via IBC and the transfer has not been returned. This is the normal operating state of any chain using IBC NFT transfer. The attacker only needs to control a chain with an established IBC channel to the victim (a standard IBC connection), and can observe on-chain state to enumerate escrowed NFTs and their token IDs.

---

### Recommendation

In `processReceivedPacket`, validate that when `isAwayFromOrigin == false`, the `data.ClassId` must begin with the **destination** port/channel prefix (i.e., `GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())`), not the source. The ICS-721 specification requires that a "return to origin" packet carries a ClassId prefixed with the receiving chain's own port/channel (the destination), not the sender's. Add an explicit check:

```go
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if !strings.HasPrefix(data.ClassId, destPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classId %s does not have expected prefix %s for return-to-origin transfer",
        data.ClassId, destPrefix)
}
```

Additionally, consider recording a commitment (e.g., a set of `(classID, tokenID)` pairs) at escrow time in `createOutgoingPacket` and verifying it at unescrow time in `processReceivedPacket`.

---

### Proof of Concept

**Setup:**
- Victim chain V has native class `realClass`, token `T1`, owned by user Alice.
- Alice calls `SendTransfer` on V via channel-Y to chain B. `createOutgoingPacket` escrows `realClass/T1` at `GetEscrowAddress("nft", "channel-Y")` on V.

**Attack:**
- Attacker controls chain A, connected to V via channel-Y (V's dest channel) / channel-X (A's source channel).
- Attacker crafts and submits an IBC packet to V:
  - `SourcePort = "nft"`, `SourceChannel = "channel-X"`
  - `DestPort = "nft"`, `DestChannel = "channel-Y"`
  - `data.ClassId = "nft/channel-X/realClass"`
  - `data.TokenIds = ["T1"]`
  - `data.TokenUris = [""]`
  - `data.Receiver = <attacker_address_on_V>`

**Execution on V:**
1. `OnRecvPacket` → `data.ValidateBasic()` passes (ClassId is non-blank, addresses valid). [6](#0-5) 
2. `processReceivedPacket` → `IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/realClass")` = `false`. [7](#0-6) 
3. `escrowAddress = GetEscrowAddress("nft", "channel-Y")` — holds `realClass/T1`. [8](#0-7) 
4. `unprefixedClassID = "realClass"`, `voucherClassID = "realClass"`. [9](#0-8) 
5. `TransferOwner(ctx, "realClass", "T1", escrowAddress, attacker)` succeeds — `IsOwner` confirms escrow address owns `T1`. [10](#0-9) 

**Result:** `realClass/T1` is now owned by the attacker on V. Alice's NFT is stolen.

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```
