### Title
Incomplete `min_gas_price` Enforcement in Gateway Stateless Validator Admits Transactions That Always Fail Blockifier Pre-Validation - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a minimum price floor (`min_gas_price`) only on `l2_gas.max_price_per_unit`, leaving `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` unchecked. A transaction with zero prices for L1 gas and L1 data gas passes gateway admission but is deterministically rejected by the blockifier's pre-validation, which checks all three resource prices against actual block prices.

### Finding Description

In `validate_resource_bounds` (lines 56–88 of `crates/apollo_gateway/src/stateless_transaction_validator.rs`), the gateway performs two resource-bound checks:

1. **Zero-fee check** (line 66): Verifies that the sum of all resource bounds is non-zero. A transaction with only `l2_gas` non-zero satisfies this.
2. **Minimum price check** (line 71): Verifies only `l2_gas.max_price_per_unit >= config.min_gas_price`. No analogous check exists for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. [1](#0-0) 

A transaction with the following resource bounds passes all stateless checks:
```
l1_gas      = { max_amount: 1, max_price_per_unit: 0 }
l2_gas      = { max_amount: 1, max_price_per_unit: min_gas_price }
l1_data_gas = { max_amount: 1, max_price_per_unit: 0 }
```

- Zero-fee check: total fee = `0 + min_gas_price + 0 > 0` → passes.
- Min-price check: `l2_gas.max_price_per_unit >= min_gas_price` → passes.
- No check on `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`.

The stateful validator also only checks `l2_gas.max_price_per_unit` against the previous block's L2 gas price: [2](#0-1) 

However, the blockifier's `check_resource_bounds` (pre-validation) compares **all three** resource prices against actual block prices: [3](#0-2) 

In production, L1 gas and L1 data gas prices are always non-zero. A transaction with `l1_gas.max_price_per_unit = 0` will therefore always fail with `ResourceBoundsError::MaxGasPriceTooLow` for `L1Gas` during blockifier pre-validation.

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

An unprivileged user can craft `AllResourceBounds` transactions with zero `l1_gas.max_price_per_unit` and/or `l1_data_gas.max_price_per_unit` that:
- Pass `StatelessTransactionValidator::validate_resource_bounds` (gateway admission).
- Pass `StatefulTransactionValidator::validate_resource_bounds` (stateful admission).
- Are accepted into the mempool.
- Are fetched by the batcher and attempted for execution.
- Are deterministically rejected by the blockifier pre-validation with `InsufficientResourceBounds`.

This allows an attacker to flood the mempool with transactions that consume sequencer resources (mempool slots, batcher cycles) but never produce valid state transitions.

### Likelihood Explanation

**Medium.** The `AllResourceBounds` variant (post-0.13.3) is the standard transaction format. Any user can submit a transaction with zero L1 gas or L1 data gas price via the public RPC endpoint. The gateway's `validate_resource_bounds` flag is `true` by default in production config, but the gap in coverage for L1 resource prices is not guarded by any other admission check. [4](#0-3) 

### Recommendation

Extend `validate_resource_bounds` to enforce `min_gas_price` on all three resource price fields when `validate_resource_bounds` is `true`:

```rust
for (price, name) in [
    (resource_bounds.l1_gas.max_price_per_unit.0, "l1_gas"),
    (resource_bounds.l2_gas.max_price_per_unit.0, "l2_gas"),
    (resource_bounds.l1_data_gas.max_price_per_unit.0, "l1_data_gas"),
] {
    if price < self.config.min_gas_price {
        return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
    }
}
```

Alternatively, introduce per-resource minimum price config fields (e.g., `min_l1_gas_price`, `min_l1_data_gas_price`) to allow independent tuning, mirroring the existing `min_gas_price` for L2 gas.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1, max_price_per_unit: 0 },
       l2_gas:      { max_amount: 1, max_price_per_unit: 8_000_000_000 },  // = default min_gas_price
       l1_data_gas: { max_amount: 1, max_price_per_unit: 0 },
   }
   ```
2. Submit to the gateway. `StatelessTransactionValidator::validate_resource_bounds` passes: total fee > 0, `l2_gas` price meets `min_gas_price`.
3. The stateful validator also passes (only checks `l2_gas` price against block price).
4. The transaction enters the mempool and is fetched by the batcher.
5. The blockifier's `check_resource_bounds` compares `l1_gas.max_price_per_unit = 0` against the actual block L1 gas price (non-zero in production) and returns `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, ... }`.
6. The transaction is rejected at execution, wasting sequencer resources. [5](#0-4) [6](#0-5)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
