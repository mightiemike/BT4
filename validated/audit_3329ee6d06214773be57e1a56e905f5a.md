Audit Report

## Title
SwapAllowlistExtension gates on the router address instead of the end-user, allowing any user to bypass the swap allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which equals `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of that call, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any pool that allowlists the router to support normal periphery usage simultaneously grants every user — including explicitly banned ones — unrestricted swap access.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as `sender` to every configured extension via `abi.encodeCall`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` argument against the per-pool allowlist, where `msg.sender` inside the extension is the pool address: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router contract `msg.sender` of that call: [4](#0-3) 

The same pattern applies to `exactInput` and `exactOutputSingle`: [5](#0-4) [6](#0-5) 

The result is that `allowedSwapper[pool][router]` is the only check performed for all router-mediated swaps, regardless of which end-user initiated the call. The pool admin faces an impossible choice: allowlist the router (granting all users access) or do not allowlist it (blocking individually-allowed users from using the periphery entirely). No existing guard in the extension, pool, or router resolves this ambiguity.

## Impact Explanation
Any user explicitly blocked on a curated pool can bypass the restriction unconditionally by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, or `exactOutputSingle`. The pool settles the swap normally; the allowlist guard never evaluates the real user. LP funds are exposed to counterparties the pool admin explicitly excluded. This is a direct, policy-level bypass of a core access-control feature, matching the "admin-boundary break bypassed by an unprivileged path" impact category.

## Likelihood Explanation
The router is a public, permissionless contract — any address can call it. The only precondition is that the pool admin has allowlisted the router, which is operationally required for the pool to be usable through the standard periphery. No privileged access, malicious setup, non-standard token, or special timing is needed. The bypass is unconditional and repeatable on every affected pool.

## Recommendation
Key the allowlist check to the economically relevant actor, not the intermediary. Two viable approaches:

1. **Encode the real user in `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it (requires a trusted-router registry or a signature scheme).
2. **Fallback logic in the extension**: When `sender` is a known/registered router, read the actual user from a verified field in `extensionData`; fall back to `sender` for direct pool calls.

A third option is to document and enforce at the factory level that `SwapAllowlistExtension` is incompatible with any router-facing pool configuration, rejecting such combinations at deployment time.

## Proof of Concept
```
1. Pool P is deployed with SwapAllowlistExtension E in BEFORE_SWAP_ORDER.
2. Pool admin calls E.setAllowedToSwap(P, router, true)   // router allowlisted for normal use
3. Pool admin calls E.setAllowedToSwap(P, alice, true)    // alice individually allowed
4. Pool admin does NOT call E.setAllowedToSwap(P, bob, true) // bob is disallowed

5. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})
   → router calls P.swap(recipient, zeroForOne, amount, limit, "", extensionData)
   → msg.sender of P.swap = router (MetricOmmSimpleRouter)
   → P._beforeSwap(sender=router, ...)
   → E.beforeSwap(sender=router, ...) checks allowedSwapper[P][router] == true
   → guard passes; bob's swap executes on the curated pool
```

Foundry test outline: deploy pool with `SwapAllowlistExtension`, allowlist only the router, confirm a non-allowlisted EOA is rejected on direct `pool.swap` but succeeds via `MetricOmmSimpleRouter.exactInputSingle`.
