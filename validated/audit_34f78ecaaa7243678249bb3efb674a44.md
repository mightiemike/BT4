### Title
Missing Event Emission in `UpdateParams` Governance Handlers Silences Critical Parameter Changes - (File: x/tieredrewards/keeper/msg_server_auth.go, x/inflation/keeper/msg_server.go)

### Summary
Two governance-executed `UpdateParams` message handlers modify critical on-chain parameters without emitting any event. Off-chain clients, indexers, and monitoring systems that rely on the event stream have no way to detect when these parameters change. In the `tieredrewards` module this is a stark inconsistency: every other governance handler in the same file (`AddTier`, `UpdateTier`, `DeleteTier`) emits a typed event, but `UpdateParams` does not. The same omission exists in the `inflation` module.

### Finding Description

**Instance 1 — `x/tieredrewards/keeper/msg_server_auth.go`, `UpdateParams` (lines 21–31)**

`UpdateParams` stores the new `TargetBaseRewardsRate` and returns immediately with no event:

```go
func (ms msgServer) UpdateParams(ctx context.Context, msg *types.MsgUpdateParams) (*types.MsgUpdateParamsResponse, error) {
    if err := ms.requireAuthority(msg.Authority); err != nil {
        return nil, err
    }
    if err := ms.SetParams(ctx, msg.Params); err != nil {
        return nil, err
    }
    return &types.MsgUpdateParamsResponse{}, nil   // ← no EmitTypedEvent
}
```

Every other governance handler in the same file calls `emitTierChangedEvent`:

```go
func (ms msgServer) AddTier(...)    { ... ms.emitTierChangedEvent(...) }
func (ms msgServer) UpdateTier(...) { ... ms.emitTierChangedEvent(...) }
func (ms msgServer) DeleteTier(...) { ... ms.emitTierChangedEvent(...) }
```

`TargetBaseRewardsRate` is the sole parameter that drives `topUpBaseRewards` in `x/tieredrewards/keeper/abci.go`, which transfers bond tokens from the community pool into the rewards pool every block. A governance-approved change to this rate is therefore a direct, silent modification to the per-block token flow for every active tiered position.

**Instance 2 — `x/inflation/keeper/msg_server.go`, `UpdateParams` (lines 28–38)**

The inflation module's handler is identical in structure — it calls `SetParams` and returns with no event:

```go
func (k msgServer) UpdateParams(goCtx context.Context, msg *types.MsgUpdateParams) (*types.MsgUpdateParamsResponse, error) {
    ...
    if err := k.SetParams(ctx, msg.Params); err != nil {
        return nil, err
    }
    return &types.MsgUpdateParamsResponse{}, nil   // ← no EmitTypedEvent
}
```

`Params` here contains `MaxSupply` (the hard supply cap whose breach halts the chain), `BurnedAddresses` (addresses excluded from circulating supply), and `DecayRate` (monthly inflation decay factor). All three fields affect every token holder on the chain.

### Impact Explanation

Off-chain clients — block explorers, indexers, alerting systems, delegator dashboards — that subscribe to the event stream receive no signal when either of these parameter sets changes. A governance proposal that silently doubles `TargetBaseRewardsRate` or zeroes `DecayRate` is indistinguishable from a no-op proposal in the event log. Delegators and tiered-position holders cannot detect the change without polling the query endpoint on every block. This is the exact observability gap the original report identified: sensitive state changes that are invisible to off-chain consumers.

### Likelihood Explanation

Both handlers are reachable via the standard governance proposal flow (`MsgSubmitProposal` → vote passes → `MsgUpdateParams` executed by the governance module account). Any token holder with sufficient deposit can submit such a proposal. The omission is present in the current production code and requires no special conditions to trigger.

### Recommendation

Emit a typed event at the end of each `UpdateParams` handler, mirroring the pattern already used by `AddTier`/`UpdateTier`/`DeleteTier`.

For `x/tieredrewards`:
```go
sdkCtx := sdk.UnwrapSDKContext(ctx)
if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventParamsUpdated{
    Params: msg.Params,
}); err != nil {
    return nil, err
}
```

For `x/inflation`:
```go
ctx := sdk.UnwrapSDKContext(goCtx)
if err := ctx.EventManager().EmitTypedEvent(&types.EventParamsUpdated{
    Params: msg.Params,
}); err != nil {
    return nil, err
}
```

Define the corresponding proto message in each module's `events.proto` and regenerate.

### Proof of Concept

1. Submit a governance proposal containing `MsgUpdateParams` for `x/tieredrewards` changing `TargetBaseRewardsRate`.
2. Vote passes; the governance module executes the message.
3. Query the block's transaction events: no `chainmain.tieredrewards.v1.EventParamsUpdated` (or any tieredrewards event) appears.
4. Contrast with `MsgAddTier` / `MsgUpdateTier` / `MsgDeleteTier` executed in the same block: those produce `chainmain.tieredrewards.v1.EventTierChanged` entries.
5. Repeat for `x/inflation` `MsgUpdateParams`: no inflation event appears in the block result.
