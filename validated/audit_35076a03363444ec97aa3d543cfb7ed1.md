### Title
Deprecated `verify_transaction_inclusion` Accepts Internal Merkle Nodes as Valid Transaction Proofs (64-Byte Forgery) — (File: `contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` does not validate that the caller-supplied `tx_id` is a leaf node in the Merkle tree. An unprivileged NEAR caller can supply an internal Merkle node as `tx_id` together with a shortened proof path that correctly reconstructs the Merkle root, causing the function to return `true` for a transaction that was never included in the block. Any recipient contract that gates fund releases on this result is directly exploitable.

---

### Finding Description

`verify_transaction_inclusion` delegates all proof checking to `compute_root_from_merkle_proof`, which simply walks the supplied proof hashes upward from whatever starting hash the caller provides: [1](#0-0) 

The function accepts any `transaction_hash` as its starting point and iterates over the proof array, combining hashes left or right according to `transaction_position % 2` at each level. It has no mechanism to distinguish a leaf hash from an internal node hash.

`verify_transaction_inclusion` adds only one guard against degenerate input: [2](#0-1) 

The non-empty check prevents a zero-length proof but does nothing to prevent a proof that starts from an internal node. If an attacker supplies:

- `tx_id` = an internal node `N` at tree level `L` (where `L ≥ 1`)
- `tx_index` = the position of `N` at level `L` (e.g., `0` for the left child of the root)
- `merkle_proof` = the `(tree_depth − L)` sibling hashes from `N` up to the root

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
