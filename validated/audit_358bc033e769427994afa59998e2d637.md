Audit Report

## Title
SwapAllowlistExtension gates the router address instead of the actual user, enabling full allowlist bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is always `msg.sender` of the `pool.swap()` call. When a user swaps through `MetricOmmSimpleRouter`, `msg.sender` is the router contract, not the end user. The extension therefore checks `allowedSwapper[pool][router]` rather than the actual user's address. If the router is allowlisted on a curated pool (a realistic operational choice), every unprivileged user can bypass the per-user allowlist by routing through the router.

## Finding Description

**Call path:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)`.
2. Router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — `msg.sender` inside the pool is the **router**.
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, recipient, ...)`, forwarding the router address as `sender`.
4. `ExtensionCalling._beforeSwap` encodes and dispatches `IMetricOmmExtensions.beforeSwap(sender=router, ...)` to every configured extension.
5. `SwapAllowlistExtension.beforeSwap` evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` = pool, `sender` = **router**. The check is `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`.

**Root cause:** The pool passes `msg.sender` (the immediate caller) as `sender` to extensions. The router never forwards the originating user's address into the pool call, so the extension has no way to identify the real swapper.

**Existing guards are insufficient:** `BaseMetricExtension.onlyPool` (via `msg.sender == pool`) only verifies the pool is the caller; it does not validate that `sender` is the actual end user. The `_requireExpectedCallbackCaller` check in the router only validates the callback direction, not the swap initiator identity.

**Exploit flow:**
- Pool admin deploys a curated pool with `SwapAllowlistExtension` and allowlists the router address (e.g., to let users access the pool via the official periphery).
- Any non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle` targeting that pool.
- The extension sees `sender = router`, which is allowlisted → swap proceeds.
- The per-user allowlist is completely bypassed.

**Exact wrong value:** `allowedSwapper[pool][router]` is evaluated instead of `allowedSwapper[pool][actualUser]`.

## Impact Explanation
A curated pool (e.g., KYC-gated, institution-only, or regulatory-restricted) that allowlists the router to support standard periphery access inadvertently grants swap rights to every unprivileged address. This is a direct allowlist bypass matching the "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" and "allowlist bypass" impact categories. The pool's curation invariant is broken: unauthorized users can trade against LP funds, potentially causing adverse selection losses to LPs and violating the pool's access policy.

## Likelihood Explanation
The scenario is realistic: a pool admin who wants users to access the pool through the official router will allowlist the router address. The admin is not expected to know that doing so grants swap rights to all users. Any unprivileged user can then exploit this with a single `exactInputSingle` call. No special privileges, flash loans, or multi-block setup are required. The attack is repeatable every block.

## Recommendation
The router should pass the originating user's address explicitly so extensions can gate on the real swapper. One approach: add a `swapper` field to the pool's `swap` signature (or a dedicated transient-storage slot set by the router before calling the pool) and have `ExtensionCalling._beforeSwap` forward that value as `sender` instead of `msg.sender`. Alternatively, `SwapAllowlistExtension` should read the real initiator from a trusted transient context rather than relying on the `sender` argument, which is always the immediate pool caller.

## Proof of Concept

// 2. Pool admin allowlists the router (intending to support periphery access)
swapExtension.setAllowedToSwap(address(pool), address(router), true);

    pool: address(pool),
    tokenIn: address(token0),
    recipient: attacker,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// Swap succeeds — allowlist bypassed because sender == router (allowlisted)
