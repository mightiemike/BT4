### Title
Fork-Block Difficulty Computed Against Mainchain Ancestor Instead of Fork Ancestor — (`contract/src/dogecoin.rs`, `contract/src/litecoin.rs`, `contract/src/bitcoin.rs`)

---

### Summary

`get_header_by_height()` always resolves a block by height from the **mainchain** index (`mainchain_height_to_header`). When this function is used inside difficulty-retarget logic to fetch the first block of a difficulty interval, it returns the mainchain block at that height — not the fork's actual ancestor at that height. For fork submissions, this is the wrong data source, producing an incorrect `first_block_time` and therefore an incorrect expected `bits` value. The code itself acknowledges the concern with a `// TODO` comment at the exact call site.

---

### Finding Description

**Vulnerability class:** Cross-module desynchronization — wrong data source used for a specific entity type (fork block vs. mainchain block), analogous to using `connection.getBalance()` for WSOL instead of the SPL token account path.

**Root cause — `get_header_by_height` always reads the mainchain index:** [1](#0-0) 

```rust
fn get_header_by_height(&self, height: u64) -> ExtendedHeader {
    self.mainchain_height_to_header          // ← always mainchain
        .get(&height)
        .and_then(|hash| self.headers_pool.get(&hash))
        .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST))
}
```

**Call site in Dogecoin difficulty retarget (with developer TODO acknowledging the problem):** [2](#0-1)

**Same pattern in Bitcoin:** [4](#0-3) 

**How fork submission works:** `submit_block_header_inner` routes a block to the fork path when its parent is not the current mainchain tip. The fork block is stored in `headers_pool` but **not** in `mainchain_height_to_header`. [5](#0-4) 

When `check_pow` → `check_target` → `get_next_work_required` is called for a fork block, `get_header_by_height(height_first)` returns the **mainchain** block at `height_first`, not the fork's ancestor at that height. For Dogecoin with `new_difficulty_protocol` (height ≥ 145,000), `difficulty_adjustment_interval = 1`, so `height_first = prev_block_height − 1`. Any fork block at depth ≥ 2 from the fork point will have a different ancestor at `height_first` than the mainchain block there. [6](#0-5) 

The `first_block_time` fed into `calculate_next_work_required` is therefore wrong, producing an incorrect `modulated_timespan` and an incorrect expected `bits`. [7](#0-6) 

---

### Impact Explanation

An attacker who controls fork block timestamps (within MTP and future-time bounds) can exploit the discrepancy between the mainchain block's timestamp at `height_first` and the fork's actual ancestor timestamp. By choosing fork block timestamps that, when differenced against the fixed mainchain timestamp, produce a `modulated_timespan` at the clamped minimum, the attacker drives the computed target upward (lower difficulty). The contract then enforces this artificially relaxed `bits` value: [8](#0-7) 

A fork chain accepted with under-difficulty blocks can accumulate enough `chain_work` to trigger a reorg: [9](#0-8) 

Once the reorg succeeds, `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` will confirm transactions in the fraudulent fork as if they were on the canonical chain, corrupting all downstream consumers of the light client's proof results. [10](#0-9) 

---

### Likelihood Explanation

For **Dogecoin** (the primary affected chain): every block is a retarget block under `new_difficulty_protocol`. Any fork block at depth ≥ 2 triggers the bug. A relayer submitting a two-block fork (which is the minimum needed to trigger a reorg attempt) immediately hits the incorrect ancestor lookup on the second fork block. The attacker only needs to stake to become a trusted relayer and submit two crafted headers.

For **Litecoin/Bitcoin**: the bug fires only at 2016-block retarget boundaries, requiring a deep fork, making exploitation significantly harder.

---

### Recommendation

Replace `get_header_by_height(height_first)` with a backward traversal from `prev_block_header` through `get_prev_header` until the target height is reached. This mirrors how `get_prev_header` correctly follows the fork chain via `prev_block_hash` links rather than the mainchain index: [11](#0-10) 

```rust
// Correct: walk the fork chain backward to height_first
let mut cursor = prev_block_header.clone();
while cursor.block_height > height_first {
    cursor = blocks_getter.get_prev_header(&cursor.block_header);
}
let first_block_time = cursor.block_header.time;
```

### Proof of Concept

**Setup (Dogecoin, height ≥ 145,000):**

1. Mainchain tip is at height H. Mainchain block at H−1 has `time = T_main`.
2. Attacker submits fork block `F_H` at height H (different from mainchain block at H), with `time = T_fork_H`.
3. Attacker submits fork block `F_{H+1}` at height H+1, with `time = T_fork_H1`.

**Difficulty calculation for `F_{H+1}`:**
- `prev_block_header` = `F_H` (fork block at H)
- `height_first = H − 1`
- `get_header_by_height(H−1)` → **mainchain block at H−1**, `time = T_main`
- `modulated_timespan = T_fork_H − T_main`

**Correct calculation should use:**
- Fork ancestor at H−1 = mainchain block at H−1 (same here, fork diverged at H)

**For `F_{H+2}` (depth 3):**
- `prev_block_header` = `F_{H+1}`
- `height_first = H`
- `get_header_by_height(H)` → **mainchain block at H**, `time = T_main_H`
- But fork's ancestor at H = `F_H`, `time = T_fork_H` ≠ `T_main_H`
- `modulated_timespan = T_fork_H1 − T_main_H` (wrong)

Attacker sets `T_fork_H1` just above MTP (minimum allowed), making `modulated_timespan` very small → clamped to `min_timespan` → difficulty drops to minimum retarget step. The contract enforces this reduced `bits` for `F_{H+2}`, accepting it with far less PoW than the honest chain requires. Repeating this for subsequent fork blocks accumulates chainwork sufficient to trigger `reorg_chain`. [12](#0-11)

```rust
    fn reorg_chain(&mut self, fork_tip_header: ExtendedHeader, last_main_chain_block_height: u64) {
        let fork_tip_height = fork_tip_header.block_height;
        if last_main_chain_block_height > fork_tip_height {
            // If we see that main chain is longer than fork we first garbage collect
            // outstanding main chain blocks:
            //
            //      [m1] - [m2] - [m3] - [m4] <- We should remove [m4]
            //     /
            // [m0]
            //     \
            //      [f1] - [f2] - [f3]
            for height in (fork_tip_height + 1)..=last_main_chain_block_height {
                let current_main_chain_blockhash = self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str("cannot get a block"));
                self.remove_block_header(&current_main_chain_blockhash);
                self.mainchain_height_to_header.remove(&height);
            }
        }

        // Now we are in a situation where mainchain is equivalent to fork size:
        //
        //      [m1] - [m2] - [m3] - [m4] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip
        //
        //
        // Or in a situation where it is shorter:
        //
        //      [m1] - [m2] - [m3] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip

        let fork_tip_hash = fork_tip_header.block_hash.clone();
        let mut fork_header_cursor = fork_tip_header;

        while !self
            .mainchain_header_to_height
            .contains_key(&fork_header_cursor.block_hash)
        {
            let prev_block_hash = fork_header_cursor.block_header.prev_block_hash;
            let current_block_hash = fork_header_cursor.block_hash;
            let current_height = fork_header_cursor.block_height;

            // Inserting the fork block into the main chain, if some mainchain block is occupying
            // this height let's save its hashcode
            let main_chain_block = self
                .mainchain_height_to_header
                .insert(&current_height, &current_block_hash);
            self.mainchain_header_to_height
                .insert(&current_block_hash, &current_height);

            // If we found a mainchain block at the current height than remove this block from the
            // header pool and from the header -> height map
            if let Some(current_main_chain_blockhash) = main_chain_block {
                self.remove_block_header(&current_main_chain_blockhash);
            }

            // Switch iterator cursor to the previous block in fork
            fork_header_cursor = self
                .headers_pool
                .get(&prev_block_hash)
                .unwrap_or_else(|| env::panic_str("previous fork block should be there"));
        }

        // Updating tip of the new main chain
        self.mainchain_tip_blockhash = fork_tip_hash;
    }
```

```rust
fn calculate_next_work_required(
    config: &DogecoinConfig,
    prev_block_header: &ExtendedHeader,
    first_block_time: i64,
) -> u32 {
    let retarget_timespan = config.pow_target_timespan;
    let modulated_timespan = i64::from(prev_block_header.block_header.time) - first_block_time;

    let mut modulated_timespan = retarget_timespan + (modulated_timespan - retarget_timespan) / 8;

    let min_timespan = retarget_timespan - (retarget_timespan / 4);
    let max_timespan = retarget_timespan + (retarget_timespan / 2);

    if modulated_timespan < min_timespan {
        modulated_timespan = min_timespan;
    } else if modulated_timespan > max_timespan {
        modulated_timespan = max_timespan;
    }

    let new_target = target_from_bits(prev_block_header.block_header.bits);

    let (mut new_target, new_target_overflow) =
        new_target.overflowing_mul(<i64 as TryInto<u64>>::try_into(modulated_timespan).unwrap());
    require!(!new_target_overflow, "new target overflow");
    new_target =
        new_target / U256::from(<i64 as TryInto<u64>>::try_into(retarget_timespan).unwrap());

    if new_target > config.pow_limit {
        new_target = config.pow_limit;
    }

    new_target.target_to_bits()
```
