### Title
Python Object Pointer Address Reuse in `identity_map` Enables Incorrect CLVM Tree Construction - (File: `wheel/src/api.rs`)

### Summary

`clvm_tree_to_lazy_node` registers Python object pointer addresses as permanent identity keys in `identity_map`. When a child Python object is dropped without a live reference being retained on the work stack, CPython can immediately free and reallocate that address for a structurally different node. The stale `identity_map` entry then silently maps the new object to the old `NodePtr`, producing a `LazyNode` with wrong tree structure.

### Finding Description

In `clvm_tree_to_lazy_node` (`wheel/src/api.rs`), three maps are maintained:

```rust
let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();  // keyed by raw pointer address
let mut atom_map:     HashMap<Vec<u8>, NodePtr> = HashMap::new();
let mut pair_map:     HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();
``` [1](#0-0) 

When a pair node is visited and one child is already recorded (`left_done = true`), the code pushes only `WorkItem::BuildPair` (storing raw `left_id: usize`) and `WorkItem::Visit(right)`, but does **not** push `WorkItem::Visit(left)`:

```rust
stack.push(WorkItem::BuildPair { id, left_id, right_id });
if !right_done { stack.push(WorkItem::Visit(right)); }
if !left_done  { stack.push(WorkItem::Visit(left));  }
``` [2](#0-1) 

The `Bound<'py, PyAny>` for `left` is dropped at the end of the `if let Some((left, right)) = pair_val` block. If that was the last Python reference to the object, CPython's reference-counting GC frees it immediately. The address `left_id` is now on CPython's free-list.

Later, when `getattr("pair")` is called on another node (a Python call that can trigger allocation), CPython may reuse `left_id` for a completely different `CLVMStorage` object. When that new object is encountered during traversal, the check:

```rust
if identity_map.contains_key(&id) { continue; }
``` [3](#0-2) 

returns `true` because the stale entry for the old object is still present. The new object is silently skipped, and when `BuildPair` is later resolved:

```rust
let l = identity_map[&left_id];
let r = identity_map[&right_id];
``` [4](#0-3) 

it retrieves the `NodePtr` for the **original** (now-freed) object, not the new one. The resulting `LazyNode` encodes the wrong subtree at that position.

The `identity_map` is never pruned — entries for freed addresses persist for the entire call, exactly mirroring the vault registry that retains no record of removed names. [5](#0-4) 

### Impact Explanation

`clvm_tree_to_lazy_node` is the sole conversion path used by `Program.to_bytes_2026()`:

```python
def to_bytes_2026(self) -> bytes:
    lazy = clvm_tree_to_lazy_node(self)
    return ser_2026(lazy)
``` [6](#0-5) 

A corrupted `LazyNode` propagates directly into `ser_2026`, producing serialized CLVM bytes that encode a different program than intended. If those bytes are later deserialized and executed via `run_serialized_chia_program`, the evaluator runs the wrong program, producing a wrong result, wrong cost, or wrong condition set — a concrete API-equivalence divergence. The corrupted `NodePtr` is the exact identifiable artifact.

### Likelihood Explanation

The trigger requires a `CLVMStorage`-conforming Python object whose `.pair` property returns **fresh** (uncached) child objects on every call — so that `left_done` can be `true` for a previously-seen address while the underlying Python object has since been freed. `LazyNode` itself does this:

```rust
pub fn pair(&self, py: Python) -> PyResult<Option<Py<PyAny>>> {
    // creates new LazyNode wrappers on every call
    let r1 = Self::new(self.allocator.clone(), *p1);
    let r2 = Self::new(self.allocator.clone(), *p2);
``` [7](#0-6) 

However, for `LazyNode` input `left_done` is always `false` (fresh addresses), so the specific branch is not reached. For `Program` input, children are cached and reference counts stay positive. The realistic trigger is a custom `CLVMStorage` implementation with ephemeral children — possible but not the common case. Likelihood is **low**.

### Recommendation

Retain a live `Bound<'py, PyAny>` reference for every address stored in `identity_map` for the duration of the traversal (e.g., push all extracted children onto a `Vec<Bound<'py, PyAny>>` held alongside the work stack). Alternatively, replace pointer-address identity with a Python-level unique ID (e.g., `id()` called once and stored as an attribute), or switch to content-only deduplication via `atom_map` / `pair_map` for all nodes, eliminating `identity_map` entirely.

### Proof of Concept

```python
import gc
from clvm_rs.clvm_rs import clvm_tree_to_lazy_node

class EphemeralPair:
    """Returns brand-new child objects on every .pair access."""
    atom = None

    @property
    def pair(self):
        # Each call allocates fresh objects; CPython may reuse freed addresses.
        left  = Atom(b"LEFT_ORIGINAL")
        right = Atom(b"RIGHT")
        return (left, right)

class Atom:
    def __init__(self, data): self.atom = data
    pair = None

# Step 1: visit EphemeralPair once so left's address enters identity_map
root = EphemeralPair()
# Step 2: force GC to reclaim the transient left child
gc.collect()
# Step 3: a second traversal of the same root may allocate a *different* Atom
#         at the recycled address; identity_map returns the stale NodePtr.
lazy = clvm_tree_to_lazy_node(root)
# lazy.pair[0] may now point to the wrong CLVM atom.
```

The corrupted `NodePtr` inside `lazy` would cause `ser_2026(lazy)` to emit bytes for the wrong subtree, and any subsequent `run_serialized_chia_program` call on those bytes would evaluate a different program than the caller intended.

fn clvm_tree_to_lazy_node(obj: Bound<'_, PyAny>) -> PyResult<LazyNode> {
    let mut allocator = Allocator::new();

    let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
    let mut atom_map: HashMap<Vec<u8>, NodePtr> = HashMap::new();
    let mut pair_map: HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();

    enum WorkItem<'py> {
        Visit(Bound<'py, PyAny>),
        BuildPair {
            id: usize,
            left_id: usize,
            right_id: usize,
        },
    }

    let root_ptr = obj.as_ptr() as usize;
    let mut stack: Vec<WorkItem<'_>> = vec![WorkItem::Visit(obj)];

    while let Some(item) = stack.pop() {
        match item {
            WorkItem::Visit(pyobj) => {
                let id = pyobj.as_ptr() as usize;

                if identity_map.contains_key(&id) {
                    continue;
                }

                let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;

                if let Some(bytes) = atom_val {
                    let node = if let Some(&existing) = atom_map.get(&bytes) {
                        existing
                    } else {
                        let new_node = allocator
                            .new_atom(&bytes)
                            .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                        atom_map.insert(bytes, new_node);
                        new_node
                    };
                    identity_map.insert(id, node);
                } else {
                    let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
                        pyobj.getattr("pair")?.extract()?;

                    if let Some((left, right)) = pair_val {
                        let left_id = left.as_ptr() as usize;
                        let right_id = right.as_ptr() as usize;

                        let left_done = identity_map.contains_key(&left_id);
                        let right_done = identity_map.contains_key(&right_id);

                        if left_done && right_done {
                            let l = identity_map[&left_id];
                            let r = identity_map[&right_id];
                            let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                                existing
                            } else {
                                let new_node = allocator.new_pair(l, r).map_err(|e| {
                                    pyo3::exceptions::PyMemoryError::new_err(e.to_string())
                                })?;
                                pair_map.insert((l, r), new_node);
                                new_node
                            };
                            identity_map.insert(id, node);
                        } else {
                            stack.push(WorkItem::BuildPair {
                                id,
                                left_id,
                                right_id,
                            });
                            if !right_done {
                                stack.push(WorkItem::Visit(right));
                            }
                            if !left_done {
                                stack.push(WorkItem::Visit(left));
                            }
                        }
                    } else {
                        return Err(pyo3::exceptions::PyValueError::new_err(
                            "CLVM object has neither .atom nor .pair",
                        ));
                    }
                }
            }
            WorkItem::BuildPair {
                id,
                left_id,
                right_id,
            } => {
                let l = identity_map[&left_id];
                let r = identity_map[&right_id];
                let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                    existing
                } else {
                    let new_node = allocator
                        .new_pair(l, r)
                        .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                    pair_map.insert((l, r), new_node);
                    new_node
                };
                identity_map.insert(id, node);
            }
        }
    }

    let root = identity_map[&root_ptr];
    Ok(LazyNode::new(Rc::new(allocator), root))
}
