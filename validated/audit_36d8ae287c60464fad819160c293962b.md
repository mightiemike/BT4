### Title
`verify_transaction_inclusion` Is Publicly Callable, Bypassing the Coinbase Proof Guard Against 64-Byte Merkle Forgery — (`File: contract/src/lib.rs`)

### Summary
`verify_transaction_inclusion` (v1) is a public NEAR contract method that any unprivileged caller can invoke directly. It performs no coinbase-proof validation. `verify_transaction_inclusion_v2` was introduced specifically to add that guard, but because v1 remains a live public entrypoint (not restricted to internal calls), any caller can bypass the guard entirely and obtain a `true` inclusion result for a fabricated transaction ID that is actually an internal Merkle-tree node.

### Finding Description
`verify_transaction_inclusion_v2` was added to close the 64-byte transaction Merkle-proof forgery vulnerability (https://www.bitmex.com/blog/64-Byte-Transactions). Its guard works by requiring the caller to also supply a valid coinbase Merkle proof for the same block: [1](#0-0) 

Only after that check passes does it delegate to the v1 function: [2](#0-1) 

However, `verify_transaction_inclusion` (v1) is itself a `pub` NEAR method with only a `#[pause]` attribute and no access-control restriction: [3](#0-2) 

The `#[deprecated]` annotation is a Rust compiler hint; it does **not** remove the method from the compiled WASM ABI. Any NEAR account can call `verify_transaction_inclusion` directly, skipping the coinbase proof check entirely.

The v1 function's only Merkle-related checks are: [4](#0-3) 

`compute_root_from_merkle_proof` simply walks the supplied proof hashes and returns whatever root they produce: [5](#0-4) 

There is no check that `tx_id` is a leaf (i.e., a real transaction hash) rather than an internal node. An attacker who knows the Merkle tree of a real Bitcoin block can pick any internal node as `tx_id`, supply the correct sibling hashes to reconstruct the Merkle root, and `verify_transaction_inclusion` returns `true`.

### Impact Explanation
Any downstream NEAR contract that calls `verify_transaction_inclusion` to gate a cross-chain action (e.g., releasing funds, minting tokens, recording a payment) will accept a forged proof as valid. The attacker does not need to control any privileged role, submit any block headers, or perform any on-chain state mutation beyond calling the public view/call method with crafted arguments. The corrupted value is the boolean proof result returned to the consuming contract.

### Likelihood Explanation
The attack requires only knowledge of a real Bitcoin block's Merkle tree (publicly available) and the ability to call a NEAR contract method (open to any account). No special permissions, staked tokens, or prior state setup are needed. The deprecated v1 function is permanently reachable as long as the current WASM is deployed.

### Recommendation
**Short term:** Restrict `verify_transaction_inclusion` (v1) so it is no longer callable as a public NEAR entrypoint. Change it from `pub fn` to `fn` (private), or add an `#[private]` attribute so only the contract itself can call it. All external callers must use `verify_transaction_inclusion_v2`.

**Long term:** Enforce the invariant at the API boundary: any function that performs Merkle proof verification must require the coinbase proof. Remove the deprecated public entrypoint entirely in the next contract upgrade to eliminate the bypass path.

### Proof of Concept
1. Identify a real Bitcoin block `B` whose header is in the contract's main chain.
2. Obtain `B`'s full transaction list and compute its Merkle tree.
3. Pick any internal node `N` at depth `d` (a 32-byte value that is the hash of two child nodes).
4. Compute the sibling path from `N` up to the Merkle root — this is a valid `merkle_proof` of length `d`.
5. Call `verify_transaction_inclusion` directly with:
   - `tx_id = N`
   - `tx_block_blockhash = hash(B)`
   - `tx_index` = the position of `N` at depth `d` (even or odd determines left/right)
   - `merkle_proof` = the sibling path computed in step 4
   - `confirmations = 1`
6. The function returns `true` because `compute_root_from_merkle_proof(N, tx_index, proof) == B.merkle_root`, even though `N` is not a transaction.
7. Any contract that trusted this result would treat a non-existent transaction as confirmed. [6](#0-5)

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
