Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypassed via Router Intermediary — Any User Can Swap Through Allowlisted `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is the direct `msg.sender` of `pool.swap()`. When a pool admin allowlists the `MetricOmmSimpleRouter` to enable router-mediated swaps, any unprivileged user can bypass the allowlist by routing through the public, permissionless router — because the extension sees `sender = router` (allowlisted) rather than the actual end-user. This silently breaks the access-control invariant of every permissioned pool that allowlists the router.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs its identity check as:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the pool calls the extension via `_callExtensionsInOrder`), and `sender` is the address the pool forwards — which is `msg.sender` of the pool's own `swap()` call:

```solidity
_beforeSwap(
    msg.sender,   // <-- this becomes `sender` in the extension
    recipient,
    ...
);
``` [2](#0-1) 

The pool passes `msg.sender` directly as `sender` to `_beforeSwap`, which then encodes it into the extension call: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and all other router entry points) call `pool.swap()` directly with no caller-identity forwarding:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
``` [4](#0-3) 

The router is a public, permissionless contract with no caller restrictions on `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`. When the router calls `pool.swap()`, `msg.sender` inside the pool is the router address. The extension therefore receives `sender = router`.

If the pool admin has called `setAllowedToSwap(pool, router, true)` — a natural step to let approved users benefit from router UX — then `allowedSwapper[pool][router] = true`, and the allowlist check passes unconditionally for every caller of the public router, regardless of whether the actual end-user is on the allowlist. [5](#0-4) 

## Impact Explanation
Any user — including those explicitly excluded from the allowlist — can swap in a pool configured to be restricted to specific counterparties. In institutional or permissioned pool deployments, this allows unauthorized parties to trade against favorable liquidity, drain LP positions, or manipulate pool state. The broken invariant is: `allowedSwapper[pool][Alice] == false`, yet Alice successfully swaps by routing through the allowlisted router. This constitutes a broken core access-control invariant with direct potential for loss of LP principal, qualifying as Medium/High under Sherlock thresholds.

## Likelihood Explanation
Medium. The trigger is a pool admin allowlisting the `MetricOmmSimpleRouter` — a routine, non-malicious action taken to let approved users benefit from router UX (multi-hop, slippage helpers, etc.). The admin has no reason to suspect this opens the allowlist to all users. The `MetricOmmSimpleRouter` is a public, permissionless periphery contract with no caller restrictions; once the router address is allowlisted, the bypass is available to any EOA or contract with no further preconditions and is repeatable indefinitely.

## Recommendation
The extension must gate the originating user, not the intermediary. Two viable approaches:

1. **Extension-data forwarding**: The router encodes the original `msg.sender` into `extensionData`; `SwapAllowlistExtension.beforeSwap` decodes and checks that address instead of `sender`. The pool already forwards `extensionData` to every extension hook.
2. **No router allowlisting**: Document and enforce that the router must never be added to the allowlist; individual user addresses must be allowlisted directly. Optionally enforce this with a registry check in `setAllowedToSwap` that rejects known periphery contract addresses.

## Proof of Concept
**Setup:**
- Pool deployed with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
- Pool admin calls `setAllowedToSwap(pool, router, true)` to let approved users access the pool via the router.
- Alice (`0xAlice`) is **not** on the allowlist (`allowedSwapper[pool][Alice] == false`).

**Attack:**
1. Alice calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: Alice, ...})`.
2. Router calls `pool.swap(recipient=Alice, ...)` — `msg.sender` inside the pool is `router`.
3. Pool calls `_beforeSwap(sender=router, ...)`.
4. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][router]` → `true`.
5. Swap executes. Alice receives output tokens despite never being allowlisted.

**Foundry test plan:**
```solidity
// Deploy pool with SwapAllowlistExtension as beforeSwap hook
// Admin allowlists router: swapExtension.setAllowedToSwap(pool, address(router), true)
// Alice is NOT allowlisted
// Alice calls router.exactInputSingle(...)
// Assert: swap succeeds (no revert) despite allowedSwapper[pool][alice] == false
// Assert: direct pool.swap() from Alice still reverts with NotAllowedToSwap
```
