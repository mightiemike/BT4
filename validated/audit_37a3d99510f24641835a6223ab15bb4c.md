### Title
Spread Health Adjustment Can Be Negative When Both Leg Weights Are At Maximum — (`core/contracts/Clearinghouse.sol`)

### Summary

In `Clearinghouse.getHealth`, the spread loop computes a health bonus as `basisAmount.mul(spotCoreRisk.price + perpCoreRisk.price).mul(spreadWeight - existingWeight)`. The protocol intends `spreadWeight >= existingWeight` always, but when both legs have `longWeightInitial = 10^9` (the maximum allowed), the `maxSpreadWeight` cap in `_getSpreadWeightX18` forces `spreadWeight` below `existingWeight`, making the adjustment negative and reducing the subaccount's health below what it would be without the spread.

---

### Finding Description

**`getCoreRisk`** always passes `amount = 1` to `_getWeightX18`, so `CoreRisk.longWeight` is always the product's long weight regardless of actual position direction: [1](#0-0) 

The constraint in `_addOrUpdateProduct` allows `longWeightInitial` up to `10^9` (= `ONE` in 1e18 scale): [2](#0-1) 

In `_getSpreadWeightX18`, the uncapped spread weight for the long-spot/short-perp case is:

```
spreadWeight = ONE - (ONE - perpLongWeight) / 5
             = (4*ONE + perpLongWeight) / 5
```

Then it is capped: [3](#0-2) 

Back in `getHealth`, `existingWeight` is the simple average of both long weights: [4](#0-3) 

**Concrete numeric example (INITIAL health, both weights = `10^9`):**

| Variable | Value |
|---|---|
| `spotCoreRisk.longWeight` | `1e18` (= ONE) |
| `perpCoreRisk.longWeight` | `1e18` (= ONE) |
| `existingWeight` | `(1e18 + 1e18) / 2 = 1e18` |
| Uncapped `spreadWeight` | `(4e18 + 1e18) / 5 = 1e18` |
| `maxSpreadWeight` (INITIAL) | `0.99e18` |
| Capped `spreadWeight` | `0.99e18` |
| **`spreadWeight - existingWeight`** | **`-0.01e18` (negative!)** |

The health delta becomes:
```
basisAmount * (spotPrice + perpPrice) * (-0.01e18)  <  0
```

Since `basisAmount > 0` always (shown by the `min`/`max` logic) and prices are positive, the spread contribution is **negative**, reducing health below the non-spread baseline. [5](#0-4) 

---

### Impact Explanation

A subaccount holding a valid spread position (long spot + short perp, or vice versa) with both products configured at maximum long weight has its `getHealth` return value **reduced** by the spread loop rather than increased. This makes the account appear unhealthier than it truly is. Any downstream check that calls `getHealth` — including liquidation eligibility — will see a falsely depressed health value, enabling premature liquidation and extraction of value from the subaccount.

---

### Likelihood Explanation

The trigger requires both the spot and perp product to have `longWeightInitial` close enough to `ONE` that their average exceeds `maxSpreadWeight` (0.99 for INITIAL, 0.994 for MAINTENANCE). This is achievable with any low-volatility or stablecoin-like asset pair where the protocol owner sets `longWeightInitial = 10^9`. The constraint validation in `_addOrUpdateProduct` explicitly permits this value. No attacker-controlled input is needed — the vulnerability is latent in any such product configuration and affects all users with spread positions in those products.

---

### Recommendation

In `getHealth`, clamp the spread adjustment to be non-negative before applying it:

```solidity
int128 weightDelta = spreadWeight - existingWeight;
if (weightDelta > 0) {
    health += basisAmount
        .mul(spotCoreRisk.price + perpCoreRisk.price)
        .mul(weightDelta);
}
```

Alternatively, enforce in `_getSpreadWeightX18` that the returned value is always `>= existingWeight` by computing `existingWeight` there and taking the max. A protocol-level fix would also add a validation in `_addOrUpdateProduct` that rejects configurations where `(longWeightInitial + longWeightInitial) / 2 > maxSpreadWeight` for any registered spread pair.

---

```text
    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
