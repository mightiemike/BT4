Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates the router address instead of the real end-user, enabling full allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is the `msg.sender` of the pool's `swap()` call. When swaps are routed through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the actual end-user. Any pool admin who allowlists the router — the only way to let approved users use the standard periphery path — simultaneously opens the pool to every non-allowlisted address that routes through the same contract.

## Finding Description

**Root cause — `SwapAllowlistExtension.beforeSwap`:**

The extension checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the first argument forwarded by the pool. [1](#0-0) 

**Pool wires its own `msg.sender` as `sender`:**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, so `sender` in the extension is whoever called `pool.swap()` — not the originating user. [2](#0-1) 

**Router calls `pool.swap()` as itself:**

`exactInputSingle` (and all other router entry points) call `pool.swap(...)` directly, making the router the `msg.sender` of that call. The pool therefore passes `sender = router` to `_beforeSwap`, and the extension evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][actual_user]`. [3](#0-2) 

**The inescapable dilemma:**

The allowlist mapping is keyed by `(pool, swapper)` where `swapper` is the immediate caller of `pool.swap()`. There is no secondary key for the real originator, and `extensionData` is present in the hook signature but unused by the extension. [4](#0-3) 

| Admin choice | Effect |
|---|---|
| Do **not** allowlist the router | Allowlisted users cannot use `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` at all |
| Allowlist the router | Every non-allowlisted user bypasses the guard by routing through the same contract |

## Impact Explanation
A pool admin who deploys a curated pool with `SwapAllowlistExtension` and allowlists the router inadvertently opens the pool to every address on-chain. Non-allowlisted users can execute swaps at oracle-derived prices on a pool explicitly configured to exclude them. This breaks the core allowlist invariant — the admin-configured guard is bypassed by an unprivileged path — and constitutes an admin-boundary break and broken core pool functionality causing loss of the access-control guarantee the pool was configured to enforce.

## Likelihood Explanation
Medium. The bypass requires the pool admin to allowlist the router. However, allowlisting the router is the only way to let approved users trade through the standard periphery path. Any pool admin who wants their allowlisted users to use the router will trigger the vulnerability. The router is a public, permissionless contract, so once allowlisted, any address can exploit the bypass without further precondition.

## Recommendation
The extension must gate the real end-user, not the immediate caller of `pool.swap()`. Two viable approaches:

1. **Pass the real caller via `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap()`. The extension decodes and verifies it against the allowlist. The pool admin must also configure the extension to trust the router as a forwarder.

2. **Signed identity in `extensionData`**: The extension accepts either a directly allowlisted `sender` or a signed proof (e.g., EIP-712 permit) that the real originator is allowlisted, forwarded through `extensionData`.

Either approach requires coordinated changes to both the router and the extension.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` configured in `BEFORE_SWAP_ORDER`.
2. Pool admin allowlists Alice: `setAllowedToSwap(pool, alice, true)`.
3. Pool admin allowlists the router so Alice can use the standard periphery: `setAllowedToSwap(pool, router, true)`.
4. Bob (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, ...)` — `msg.sender` of `pool.swap` is the router.
6. Pool calls `_beforeSwap(sender=router, ...)`.
7. Extension evaluates `allowedSwapper[pool][router] == true` → passes without checking Bob.
8. Bob's swap executes at oracle price on the curated pool, bypassing the allowlist entirely.
