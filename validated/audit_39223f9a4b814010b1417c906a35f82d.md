Audit Report

## Title
`SwapAllowlistExtension` gates on the router address instead of the originating user, allowing any caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `sender` is the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router contract becomes `msg.sender` to the pool, so `sender` is the router address, not the originating user. Any pool admin who allowlists the router to enable legitimate router-based trading simultaneously grants every unprivileged user the ability to bypass the allowlist by routing through the same public contract.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to `_beforeSwap`:**

`MetricOmmPool.swap` captures `msg.sender` and forwards it as the `sender` argument: [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` unchanged to all extensions:** [2](#0-1) 

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`:** [3](#0-2) 

The check at line 37 is `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. There is no mechanism to recover the original EOA from the call.

**Step 4 — Every `MetricOmmSimpleRouter` entry point calls `pool.swap()` directly, making the router `msg.sender` to the pool:** [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

**Root cause:** The extension checks the proximate caller of `pool.swap()`, not the originating user. The router is a public, permissionless contract that any address can call. Allowlisting the router is the only way to let legitimate users reach the pool through the router, but doing so makes the router entry the effective gate — not the per-user allowlist entries.

**Existing guards are insufficient:** `allowedSwapper` and `allowAllSwappers` are the only guards in `beforeSwap`. There is no inspection of `extensionData` for an authenticated user identity, no check that `sender` is an EOA, and no secondary verification path. [8](#0-7) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (e.g., KYC-verified counterparties, institutional LPs, or protocol-internal actors) can be fully bypassed by any unprivileged user who calls any of the four public router entry points targeting that pool. The unauthorized user executes real swaps, receives real token output settled at oracle price, and the pool's LP funds cover the trade — identical economic outcome to an authorized swap. This constitutes broken core pool functionality (access control bypass) with direct loss exposure to LP funds and violation of the pool admin's intended access restriction. [9](#0-8) 

## Likelihood Explanation
- `MetricOmmSimpleRouter` is a public, permissionless periphery contract with no caller restrictions.
- The only precondition is that the pool admin has allowlisted the router — a natural and expected configuration for any pool intending to support router-based trading for its allowlisted users.
- No privileged access, special token, flash loan, or admin cooperation is required from the attacker.
- All four router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are affected.
- The attack is repeatable indefinitely with no on-chain record distinguishing the bypass from an authorized trade. [10](#0-9) 

## Recommendation
The extension must gate on the **originating user**, not the immediate pool caller. Two sound approaches:

1. **Encode the real user in `extensionData`:** Have the router encode `msg.sender` into `extensionData`; have the extension decode and verify it when `sender` is a known, allowlisted router. The router itself must be allowlisted as a trusted forwarder, and the extension must verify the decoded user is allowlisted.

2. **Require direct pool calls for allowlisted pools:** Remove the router from the allowlist and require all allowlisted users to call `pool.swap()` directly until the extension is updated to forward user identity. This is the simplest safe fix with no code changes required to the extension.

The simplest safe fix is option 2: do not allowlist the router on pools using `SwapAllowlistExtension`, accepting that the router is incompatible with the allowlist model in its current form. [11](#0-10) 

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true   // alice is the only intended swapper
  allowedSwapper[pool][router] = true  // admin adds router so alice can use it

Attack (executed by bob, who is NOT allowlisted):
  bob calls MetricOmmSimpleRouter.exactInputSingle({
      pool:      <curated pool>,
      recipient: bob,
      zeroForOne: true,
      amountIn:  X,
      ...
  })

  Router calls pool.swap(bob_recipient, true, X, ..., extensionData)
    → msg.sender to pool = router address
    → _beforeSwap(sender=router, ...)                          [ExtensionCalling.sol L160]
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)    [SwapAllowlistExtension.sol L31]
    → allowedSwapper[pool][router] == true  → PASSES           [SwapAllowlistExtension.sol L37]
    → swap executes, bob receives token output

Result:
  bob, a non-allowlisted address, completes a real swap in a curated pool.
  alice's allowlist entry is irrelevant; the router entry is the effective gate.
  LP funds settle the trade at oracle price with no revert or slippage penalty.
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
