Audit Report

## Title
Swap Allowlist Bypassed by Any User via `MetricOmmSimpleRouter` When Router Is Allowlisted - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates the call, `sender` is the router address. If the pool admin allowlists the router so legitimate users can swap through it, every user — including those not on the allowlist — can bypass the guard by routing through the same public router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` and checks it against the per-pool allowlist: [1](#0-0) 

`MetricOmmPool.swap` passes `msg.sender` (the immediate caller of `pool.swap()`) as the `sender` argument to the extension: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` seen by the pool. The actual end user's address is stored only in transient storage via `_setNextCallbackContext` for payment callback purposes and is never forwarded to the pool or the extension: [3](#0-2) 

The same applies to `exactInput` (L103-112), `exactOutputSingle` (L135-137), and `exactOutput` (L165-181) — all router entry points call `pool.swap()` with `msg.sender = router`. [4](#0-3) 

**Attack path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` as the `beforeSwap` hook.
2. Admin calls `setAllowedToSwap(pool, alice, true)` (KYC'd user).
3. Admin calls `setAllowedToSwap(pool, router, true)` so Alice can use the router.
4. Charlie (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(...)` with `msg.sender = router`.
6. `_beforeSwap(router, ...)` is called; extension sees `sender = router`.
7. `allowedSwapper[pool][router] == true` → check passes.
8. Charlie's swap executes despite never being allowlisted.

## Impact Explanation

`SwapAllowlistExtension` is the production mechanism for restricting pool access to specific counterparties (e.g., KYC-gated or institutional-only pools). When bypassed, any unprivileged user can execute swaps against a pool intended to be restricted. LP assets are exposed to unrestricted trading, defeating the admin's access-control intent and potentially exposing LPs to adversarial flow they explicitly opted out of. This constitutes broken core pool functionality with direct fund-impacting consequences.

## Likelihood Explanation

The bypass requires the admin to allowlist the router — a natural and expected operational step if any legitimate user needs to swap through the router rather than calling the pool directly. The router is the standard user-facing entry point in the periphery. Any pool using `SwapAllowlistExtension` that also wants router access for its allowlisted users will inevitably trigger this condition. The attacker needs no special privileges, no malicious setup, and no non-standard tokens.

## Recommendation

The extension must check the end user, not the immediate caller. Two sound approaches:

1. **Pass the end user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a trusted router or a signed payload.
2. **Router registry + end-user field**: The extension recognizes known routers and reads the actual trader from a standardized field in `extensionData`.

The cleanest fix is option 1: require the router to forward the originating user address in `extensionData` and have the extension verify it against the allowlist.

## Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension as beforeSwap hook
  admin calls setAllowedToSwap(pool, alice, true)   // Alice is KYC'd
  admin calls setAllowedToSwap(pool, router, true)  // so Alice can use the router

Attack:
  charlie (not allowlisted) calls:
    router.exactInputSingle({
      pool: pool,
      recipient: charlie,
      zeroForOne: true,
      amountIn: X,
      ...
    })

  Execution trace:
    router.exactInputSingle()
      → _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, charlie, tokenIn)
      → pool.swap(charlie, true, X, ...) [msg.sender = router]
        → _beforeSwap(router, charlie, ...)
          → SwapAllowlistExtension.beforeSwap(sender=router, ...)
            → allowedSwapper[pool][router] == true → PASSES
        → swap executes, charlie receives output tokens

Result: Charlie swaps successfully despite not being on the allowlist.
  Direct pool call by Charlie would revert (allowedSwapper[pool][charlie] == false).
```
