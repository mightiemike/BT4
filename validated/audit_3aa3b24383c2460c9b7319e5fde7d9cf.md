Audit Report

## Title
SwapAllowlistExtension checks the router intermediary instead of the originating user, allowing full allowlist bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the direct caller of `pool.swap()` — the router — not the originating EOA. Because `MetricOmmSimpleRouter.exactInputSingle` is a public, permissionless function, any user who routes through it will pass the allowlist check once the pool admin allowlists the router address. There is no mechanism to simultaneously permit router-mediated swaps for allowlisted users and block non-allowlisted users from using the same router.

## Finding Description
**Root cause — wrong actor checked**

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the direct caller of `pool.swap()`: [3](#0-2) 

**The bypass path**

`MetricOmmSimpleRouter.exactInputSingle` is a public, permissionless function with no access control on the caller: [4](#0-3) 

When the router calls `pool.swap(...)`, the pool receives `msg.sender = router`. The extension therefore checks `allowedSwapper[pool][router]`, not the originating EOA. The originating attacker's address is never seen by the guard.

**The structural trap**

If a pool admin deploys a `SwapAllowlistExtension`-protected pool and wants allowlisted users to swap through the canonical router, the admin must call `setAllowedToSwap(pool, address(router), true)`. Once the router is allowlisted, every user on the network can call `exactInputSingle` and the extension will pass, because the pool always sees the router as `sender`. There is no way to simultaneously allow router-mediated swaps for allowlisted users and block non-allowlisted users from using the same router. [5](#0-4) 

## Impact Explanation
Any user — regardless of whether they appear in the allowlist — can execute swaps on a curated pool by routing through `MetricOmmSimpleRouter`, provided the pool admin has allowlisted the router. The swap allowlist, which is the pool's primary curation mechanism, is rendered entirely ineffective. Unauthorized parties can interact with KYC-gated, institutional-only, or rate-limited pools, draining arbitrage value, front-running allowlisted LPs, or accessing pools they were explicitly excluded from. This constitutes broken core pool functionality with direct fund-impacting consequences.

## Likelihood Explanation
Any pool admin who deploys a `SwapAllowlistExtension`-protected pool and also wants to support the standard router path will allowlist the router. The design gives no indication that doing so opens the gate to all users. The trap is structural, not a configuration error requiring malicious intent. The router is the canonical periphery contract, making this the expected operational path. Any unprivileged user can exploit this with a single call to `exactInputSingle`.

## Recommendation
The allowlist hook must check the originating user, not the intermediary. Two viable approaches:

1. **Pass the real caller through `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap`; the extension decodes and checks it. This requires the router to be trusted to supply honest data, which is acceptable for a canonical periphery contract.

2. **Trusted forwarder registry**: The extension maintains a registry of trusted forwarders; when `sender` is a forwarder, it reads the real caller from `extensionData`; otherwise it checks `sender` directly.

Additionally, document clearly that allowlisting the router address is equivalent to `allowAllSwappers = true` for any user who can reach the router.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  pool admin calls setAllowedToSwap(pool, address(router), true)
    → intent: allow router-mediated swaps for allowlisted users
    → actual effect: any user can now swap

Attack:
  attacker = address not in allowedSwapper[pool]
  attacker calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...) with msg.sender = router
    → pool calls _beforeSwap(router, ...)
    → SwapAllowlistExtension.beforeSwap receives sender = router
    → checks allowedSwapper[pool][router] → true → passes
    → swap executes; attacker receives output tokens

The allowlist check never sees the attacker's address. The guard is fully bypassed.
```
