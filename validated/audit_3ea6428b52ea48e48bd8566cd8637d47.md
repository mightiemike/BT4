Audit Report

## Title
Swap Allowlist Bypassed via Router Intermediary: `sender` Checked Is Router, Not Original User - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the originating user. If the router is allowlisted (a natural admin action to enable normal periphery flows), every non-allowlisted user can bypass the swap allowlist by routing through the router, defeating the entire curation policy.

## Finding Description

**Root cause — wrong actor checked:**

`SwapAllowlistExtension.beforeSwap` (line 37) checks the `sender` argument against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:31-41
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`msg.sender` here is the pool (the extension caller). `sender` is the first argument forwarded by the pool.

**Pool passes `msg.sender` as `sender`:**

In `MetricOmmPool.swap` (line 231), the pool calls:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged as the `sender` argument to the extension (line 165).

**Router replaces the user as `sender`:**

In `MetricOmmSimpleRouter.exactInputSingle` (line 72-80), the router calls:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...
);
```

`msg.sender` of this `pool.swap()` call is the router contract (`address(MetricOmmSimpleRouter)`), not the originating user. The extension therefore sees `sender = router`, not the user.

**Exploit flow:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict trading to approved addresses.
2. Pool admin allowlists `MetricOmmSimpleRouter` (a natural step to enable standard periphery usage).
3. A non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
4. The router calls `pool.swap(...)` with `msg.sender = router`.
5. The pool calls `_beforeSwap(sender=router, ...)`.
6. The extension evaluates `allowedSwapper[pool][router]` → `true`.
7. The non-allowlisted user's swap executes successfully, bypassing the allowlist.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` router entry points.

**Contrast with deposit allowlist (not vulnerable):**

`DepositAllowlistExtension.beforeAddLiquidity` ignores the `sender` argument entirely and checks `owner` (the economic position owner), which is correctly preserved regardless of who calls `pool.addLiquidity()`. The swap extension lacks this design.

## Impact Explanation
A pool admin configuring `SwapAllowlistExtension` intends to restrict which addresses may trade. Any non-allowlisted user can bypass this restriction by routing through the public `MetricOmmSimpleRouter`. This is a direct admin-boundary break: the allowlist policy is rendered ineffective for all router-mediated swaps. On pools where the allowlist is used to enforce KYC, regulatory compliance, or exclusive LP arrangements, unauthorized users can drain LP value through unrestricted swaps. This meets the "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" impact criterion at High severity.

## Likelihood Explanation
The `MetricOmmSimpleRouter` is the standard public swap interface documented and expected to be used by all users. A pool admin who wants to allow normal periphery usage must allowlist the router. Once the router is allowlisted, the bypass is trivially reachable by any address with no special privileges, no capital requirements beyond the swap amount, and is repeatable on every block. No malicious setup is required beyond using the standard router.

## Recommendation
Change `SwapAllowlistExtension.beforeSwap` to check the original user rather than the direct pool caller. The simplest fix is to check `recipient` if that represents the economic beneficiary, or — more robustly — require the router to forward the originating user's address as an explicit `extensionData` payload that the extension verifies. Alternatively, mirror the deposit allowlist design: gate on a parameter that is always the economic actor regardless of intermediary (e.g., require the router to pass the original `msg.sender` in `extensionData` and verify it in the extension). The pool's `sender` argument is insufficient for allowlist enforcement when public router intermediaries are involved.

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted, user not allowlisted
    swapExtension.setAllowedToSwap(address(pool), address(router), true);
    // user is NOT allowlisted

    // Direct swap by user reverts correctly
    vm.prank(user);
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    pool.swap(user, true, 1000, type(uint128).max, "", "");

    // Router-mediated swap by same user succeeds (bypass)
    vm.prank(user);
    // router.exactInputSingle calls pool.swap with msg.sender=router
    uint256 out = router.exactInputSingle(ExactInputSingleParams({
        pool: address(pool),
        recipient: user,
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: type(uint128).max,
        tokenIn: address(token0),
        deadline: block.timestamp,
        extensionData: ""
    }));
    // out > 0: non-allowlisted user successfully swapped
    assertGt(out, 0);
}
```
