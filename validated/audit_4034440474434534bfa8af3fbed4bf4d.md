### Title
`min_sierra_version_for_sierra_gas` Override Without Gas-Limit Compensation Causes Execution Divergence Between Normal Sequencing and Virtual Block Proving - (File: `crates/starknet_transaction_prover/src/running/virtual_block_executor.rs`)

---

### Summary

`BaseBlockInfo::new()` in the transaction prover unconditionally overrides `min_sierra_version_for_sierra_gas` to `SierraVersion::new(0, 0, 1)` — forcing every Cairo 1 contract into `SierraGas` mode — but does **not** raise `execute_max_sierra_gas` / `validate_max_sierra_gas` to compensate. The companion `blockifier_reexecution` path explicitly documents and fixes this exact omission. The result is a version-constant boundary where a transaction that executed successfully on-chain (under `CairoSteps` mode with effectively infinite gas) is re-executed by the prover under a capped `SierraGas` budget and reverts, producing a divergent execution result for accepted input.

---

### Finding Description

**Normal sequencing path** (blockifier, batcher):

For a block at Starknet version ≤ 0.13.x, `VersionedConstants::get(&block_info.starknet_version)` returns constants with `min_sierra_version_for_sierra_gas = "100.0.0"`. Any Cairo 1 contract whose compiled Sierra version is below `100.0.0` (i.e., every real contract) is assigned `TrackedResource::CairoSteps` and receives `default_initial_gas_cost = 100_000_000 × 100 = 10_000_000_000` gas — effectively unlimited. [1](#0-0) 

**Virtual block executor path** (prover):

`BaseBlockInfo::new()` selects the latest versioned constants (default `use_latest_versioned_constants = true`, which gives 0.14.4 constants with `execute_max_sierra_gas = 1_110_000_000` and `validate_max_sierra_gas = 100_000_000`), then unconditionally writes:

```rust
versioned_constants.min_sierra_version_for_sierra_gas = SierraVersion::new(0, 0, 1);
``` [2](#0-1) 

This flips every Cairo 1 contract to `TrackedResource::SierraGas` with the 0.14.4 caps still in place. The gas limits are **not** raised.

**The fix exists in the sibling crate but is absent here:**

`blockifier_reexecution/src/state_reader/rpc_state_reader.rs` contains an explicit comment and correction:

```rust
// When overriding min_sierra_version, contracts that normally run in CairoSteps
// mode (with effectively infinite gas) are forced into SierraGas mode, where gas
// is capped by execute/validate_max_sierra_gas. Raise these limits to match
// default_initial_gas_cost so these contracts don't hit out-of-gas errors.
let default_gas = new_os_constants.default_initial_gas_cost;
new_os_constants.execute_max_sierra_gas = default_gas;
new_os_constants.validate_max_sierra_gas = default_gas;
``` [3](#0-2) 

The virtual block executor omits this correction entirely.

**Concrete divergence values:**

| Context | `min_sierra_version_for_sierra_gas` | `execute_max_sierra_gas` | `validate_max_sierra_gas` |
|---|---|---|---|
| Normal execution (0.13.x block) | `100.0.0` → CairoSteps | N/A (infinite) | N/A (infinite) |
| Virtual block executor (latest VC) | `0.0.1` → SierraGas | **1,110,000,000** | **100,000,000** |
| Reexecution path (corrected) | `0.0.1` → SierraGas | 10,000,000,000 | 10,000,000,000 | [4](#0-3) [5](#0-4) 

---

### Impact Explanation

The virtual block executor re-executes accepted, on-chain transactions to generate OS input for proving. When a Cairo 1 contract with Sierra version < 1.7.0 consumed between 1,110,000,001 and 10,000,000,000 gas during normal execution (CairoSteps mode), the prover re-executes it under a 1,110,000,000-gas cap and receives an out-of-gas revert. The executor then returns `VirtualBlockExecutorError::TransactionReverted`, aborting proof generation for the entire block. [6](#0-5) 

This matches: **Critical — wrong revert result from blockifier/execution logic for accepted input**, and **Critical — wrong compiled class / execution mode selected for execution**.

---

### Likelihood Explanation

- The default configuration (`use_latest_versioned_constants = true`) is always active in production.
- Cairo 1 contracts compiled with Sierra < 1.7.0 are common on mainnet (all contracts deployed before Starknet 0.14.0).
- Any such contract whose `__execute__` or `__validate__` consumes more than ~11M steps (1,110,000,000 ÷ 100 gas/step) triggers the divergence. Complex account contracts or contracts performing heavy computation are realistic candidates.
- The trigger requires no privileged access — any user transaction invoking such a contract suffices.

---

### Recommendation

Mirror the `blockifier_reexecution` fix inside `BaseBlockInfo::new()` immediately after overriding `min_sierra_version_for_sierra_gas`:

```rust
versioned_constants.min_sierra_version_for_sierra_gas = SierraVersion::new(0, 0, 1);
// Raise gas caps to match default_initial_gas_cost so contracts previously
// running in CairoSteps mode (infinite gas) do not hit out-of-gas in SierraGas mode.
let mut new_os_constants = (*versioned_constants.os_constants).clone();
let default_gas = new_os_constants.default_initial_gas_cost;
new_os_constants.execute_max_sierra_gas = default_gas;
new_os_constants.validate_max_sierra_gas = default_gas;
versioned_constants.os_constants = Arc::new(new_os_constants);
``` [3](#0-2)

### Proof of Concept

1. Deploy a Cairo 1 contract compiled with Sierra version 1.5.0 (below the 1.7.0 threshold in 0.14.x constants).
2. Submit an invoke transaction whose `__execute__` consumes ~12,000,000 Cairo steps (1,200,000,000 Sierra gas units — above `execute_max_sierra_gas = 1,110,000,000` but well below `default_initial_gas_cost = 10,000,000,000`).
3. The sequencer executes the transaction in `CairoSteps` mode (infinite gas); it succeeds and is committed to a block.
4. The prover's `RpcVirtualBlockExecutor` calls `BaseBlockInfo::new(…, use_latest_versioned_constants=true)`, which sets `min_sierra_version_for_sierra_gas = SierraVersion(0,0,1)` without raising gas caps.
5. `transaction_executor.execute_txs` re-executes the transaction in `SierraGas` mode; the contract exhausts its 1,110,000,000-gas budget and reverts.
6. `VirtualBlockExecutorError::TransactionReverted` is returned; proof generation for the block fails. [7](#0-6) [8](#0-7)

        mut header: BlockHeader,
        chain_info: ChainInfo,
        use_latest_versioned_constants: bool,
    ) -> Result<Self, VirtualBlockExecutorError> {
        let base_block_hash = header.block_hash;
        let prev_base_block_hash = header.parent_hash;
        let base_block_header_commitments = BlockHeaderCommitments {
            transaction_commitment: header.transaction_commitment,
            event_commitment: header.event_commitment,
            receipt_commitment: header.receipt_commitment,
            state_diff_commitment: header.state_diff_commitment,
            concatenated_counts: concat_counts(
                header.transaction_count,
                header.event_count,
                header.state_diff_length,
                header.l1_da_mode,
            ),
        };

        let raw_starknet_version = header.starknet_version.clone();
        // Versions newer than the latest known one are executed with the latest known semantics;
        // the raw version string is kept separately so the OS recomputes the on-chain block hash
        // from the exact version string.
        header.starknet_version = starknet_version_or_latest(&raw_starknet_version)
            .map_err(|e| {
                VirtualBlockExecutorError::TransactionExecutionError(format!(
                    "Unsupported starknet version in block header: {e}"
                ))
            })?
            .to_string();
        let block_info: BlockInfo = header.try_into().map_err(|e| {
            VirtualBlockExecutorError::TransactionExecutionError(format!(
                "Failed to convert block header to block info: {e}"
            ))
        })?;
        let mut versioned_constants = if use_latest_versioned_constants {
            VersionedConstants::latest_constants().clone()
        } else {
            VersionedConstants::get(&block_info.starknet_version)
                .map_err(|e| {
                    VirtualBlockExecutorError::TransactionExecutionError(format!(
                        "Failed to get versioned constants: {e}"
                    ))
                })?
                .clone()
        };
        // Disable casm hash migration for virtual block execution.
        versioned_constants.disable_casm_hash_migration();
        // Enable Sierra gas for all Cairo 1 contracts.
        // Version (0, 0, 0) is reserved for Cairo 0, so set to (0, 0, 1) for Cairo 1.
        versioned_constants.min_sierra_version_for_sierra_gas = SierraVersion::new(0, 0, 1);

        let block_context = BlockContext::new(
            block_info,
            chain_info,
            versioned_constants.clone(),
            BouncerConfig::default(),
        );

        Ok(BaseBlockInfo {
            block_context,
            raw_starknet_version,
            base_block_hash,
            base_block_header_commitments,
            prev_base_block_hash,
        })
    }
