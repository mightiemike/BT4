### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables Free Drain of usdcE from DirectDepositV1 Accounts — (`File: core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` is a publicly callable function that is intended to swap usdcE held in a `DirectDepositV1` (DDA) address for USDC provided by the caller. The function calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the return value. If this call returns `false` instead of reverting (e.g., due to insufficient allowance or balance), execution continues and the caller still receives the usdcE, paying nothing.

---

### Finding Description

`ContractOwner` declares `using ERC20Helper for IERC20Base` at line 24, making the checked `safeTransferFrom` wrapper available on all `IERC20Base` instances. Despite this, `replaceUsdcEWithUsdc` calls the raw `transferFrom` directly on line 616:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance); // return value ignored
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));       // pulls usdcE to ContractOwner
IERC20Base(usdcE).safeTransfer(msg.sender, balance);                 // sends usdcE to caller
```

The outbound transfer on line 618 correctly uses `safeTransfer`, but the inbound payment on line 616 does not. If the USDC `transferFrom` silently returns `false`, the function proceeds to withdraw usdcE from the DDA and deliver it to the caller at no cost.

The function's only access control is `require(block.chainid == 57073, ERR_UNAUTHORIZED)`, making it callable by any unprivileged address on that chain. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

An attacker with zero USDC (or zero USDC allowance granted to `ContractOwner`) can call `replaceUsdcEWithUsdc` for any subaccount whose DDA holds usdcE. If the USDC `transferFrom` returns `false` rather than reverting, the attacker receives the full usdcE balance of that DDA for free. This directly drains user-deposited collateral (usdcE) from DDA addresses, corrupting the asset balance of the affected subaccounts. [4](#0-3) 

---

### Likelihood Explanation

The function is externally callable by any address on chain 57073 (Ink). The USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on that chain may not revert on a failed `transferFrom` (e.g., if it is a non-standard or bridged implementation). Even for a standard USDC that does revert, the inconsistency with the rest of the codebase — which uniformly uses `safeTransfer`/`safeTransferFrom` — represents a latent integration risk that becomes critical if the token behavior differs from mainnet USDC. Any DDA with a non-zero usdcE balance is a target. [5](#0-4) [6](#0-5) 

---

### Recommendation

Replace the raw `transferFrom` call with the `safeTransferFrom` wrapper already available via `ERC20Helper`:

```solidity
// Before (line 616):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After:
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

This is consistent with how all other token transfers in the codebase are handled. [3](#0-2) 

---

### Proof of Concept

1. Identify a subaccount whose `directDepositV1Address` holds a non-zero usdcE balance on chain 57073.
2. Call `ContractOwner.replaceUsdcEWithUsdc(subaccount)` from an address with **no USDC balance and no USDC allowance** granted to `ContractOwner`.
3. `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` returns `false` (no revert).
4. `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` transfers the full usdcE balance from the DDA to `ContractOwner`.
5. `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` delivers the usdcE to the attacker.
6. Attacker receives `balance` usdcE tokens, paying 0 USDC. [7](#0-6)

```text
    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
