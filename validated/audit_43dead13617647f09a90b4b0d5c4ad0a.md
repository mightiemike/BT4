### Title
`Treehasher` Domain-Separator Parameters Silently Ignored in Hash Computation — (`File: wheel/python/clvm_rs/tree_hash.py`)

---

### Summary

The `Treehasher` class in the Python wheel exposes `atom_prefix` and `pair_prefix` as configurable domain-separator parameters, but the inner functions that perform the actual hashing (`handle_obj` and `handle_pair`) call module-level `shatree_atom` / `shatree_pair` — which are permanently bound to the singleton `CHIA_TREEHASHER` instance — instead of `self.shatree_atom` / `self.shatree_pair`. As a result, any `Treehasher` constructed with non-standard prefixes silently produces the standard Chia tree hash, making the prefix parameters completely inert. This is a direct analog of the "parameter not included in signed hash" class: a value that is supposed to govern the hash domain is accepted by the API but never fed into the digest.

---

### Finding Description

In `wheel/python/clvm_rs/tree_hash.py`, the `Treehasher` class stores `atom_prefix` and `pair_prefix` as instance attributes and exposes correct instance methods `shatree_atom` / `shatree_pair` that use them:

```python
def shatree_atom(self, atom: bytes) -> bytes:
    s = sha256()
    s.update(self.atom_prefix)   # uses self.atom_prefix
    s.update(atom)
    return s.digest()
```

However, the `sha256_treehash` method defines two inner closures, `handle_obj` and `handle_pair`, that call the **module-level** names `shatree_atom` and `shatree_pair` instead of `self.shatree_atom` and `self.shatree_pair`:

```python
# handle_obj — line 63
r = shatree_atom(obj.atom)          # module-level, NOT self.shatree_atom

# handle_pair — line 86
r = shatree_pair(p0, p1)            # module-level, NOT self.shatree_pair
```

At module load time, those module-level names are bound to the singleton `CHIA_TREEHASHER`'s methods:

```python
CHIA_TREEHASHER = Treehasher(CHIA_TREE_HASH_ATOM_PREFIX, CHIA_TREE_HASH_PAIR_PREFIX)
shatree_atom = CHIA_TREEHASHER.shatree_atom   # bound to CHIA_TREEHASHER
shatree_pair = CHIA_TREEHASHER.shatree_pair   # bound to CHIA_TREEHASHER
```

Therefore, every `Treehasher` instance — regardless of the `atom_prefix` / `pair_prefix` passed to its constructor — will always hash with `\x01` (atom prefix) and `\x02` (pair prefix) from `CHIA_TREEHASHER`. The constructor parameters are stored but never consulted during hashing. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

---

### Impact Explanation

The `Treehasher` API is explicitly designed to allow callers to supply custom domain-separator prefixes. Any caller that constructs a `Treehasher` with non-standard prefixes (e.g., a different `atom_prefix` / `pair_prefix` for a separate network, testnet, or application-level domain) will receive tree hashes that are **identical** to standard Chia mainnet puzzle hashes, not the intended custom-domain hashes.

Concretely:
- **Cross-domain hash collision**: Two logically distinct CLVM trees intended to have different hashes under different prefix schemes will hash to the same value, because the prefix is never applied.
- **Replay / substitution**: A puzzle hash computed for one domain is indistinguishable from the same tree's hash in any other domain. An attacker who can present a valid spend for a coin on one domain can replay it on another domain that was supposed to use a different prefix for isolation.
- **Silent correctness failure**: The API gives no error or warning; callers have no way to detect that their custom prefix was ignored. [6](#0-5) 

---

### Likelihood Explanation

The `Treehasher` class is part of the public Python wheel API and is parameterized precisely to support multiple hash domains. Any downstream library or application that instantiates `Treehasher` with non-default prefixes — a use case the class explicitly advertises — will silently receive wrong hashes. The standard `CHIA_TREEHASHER` instance is unaffected, so the bug is invisible in the default path and is unlikely to be caught by existing tests that only exercise the standard prefixes. [5](#0-4) 

---

### Recommendation

Replace the bare module-level calls inside the closures with calls through `self`:

```python
# handle_obj — replace:
r = shatree_atom(obj.atom)
# with:
r = self.shatree_atom(obj.atom)

# handle_pair — replace:
r = shatree_pair(p0, p1)
# with:
r = self.shatree_pair(p0, p1)
```

Additionally, the cached value stored via `setattr(obj, "_cached_sha256_treehash", r)` should be keyed per-`Treehasher` instance (or the cache should be disabled for non-standard hashers), since the current cache stores the standard-prefix hash and would return it even for a corrected custom-prefix hasher on a previously seen object. [3](#0-2) [7](#0-6)

### Proof of Concept

```python
from hashlib import sha256
from clvm_rs.tree_hash import Treehasher, CHIA_TREE_HASH_ATOM_PREFIX, CHIA_TREE_HASH_PAIR_PREFIX

# Custom hasher with different prefixes
custom = Treehasher(atom_prefix=b'\xAA', pair_prefix=b'\xBB')

# A simple atom object
class FakeAtom:
    atom = b'hello'
    pair = None

obj = FakeAtom()

# Expected: sha256(b'\xAA' + b'hello')
expected = sha256(b'\xAA' + b'hello').digest()

# Actual: sha256(b'\x01' + b'hello')  — standard Chia prefix used instead
actual = custom.sha256_treehash(obj)

assert actual != expected, "Bug confirmed: custom prefix was silently ignored"
# actual == sha256(b'\x01hello').digest()  — identical to CHIA_TREEHASHER output
``` [8](#0-7)

class Treehasher:
    """
    `Treehasher` performs the standard sha256tree hashing in a non-recursive
    way so that extremely large objects don't blow out the python stack.

    We also force a `_cached_sha256_treehash` into the hashed sub-objects
    whenever possible so that taking the hash of the same sub-tree is
    more efficient in future.
    """

    atom_prefix: bytes
    pair_prefix: bytes
    cache_hits: int

    def __init__(self, atom_prefix: bytes, pair_prefix: bytes):
        self.atom_prefix = atom_prefix
        self.pair_prefix = pair_prefix
        self.cache_hits = 0

    def shatree_atom(self, atom: bytes) -> bytes:
        s = sha256()
        s.update(self.atom_prefix)
        s.update(atom)
        return s.digest()

    def shatree_pair(self, left_hash: bytes, right_hash: bytes) -> bytes:
        s = sha256()
        s.update(self.pair_prefix)
        s.update(left_hash)
        s.update(right_hash)
        return s.digest()
```

    def sha256_treehash(self, clvm_storage: CLVMStorage) -> bytes:
        def handle_obj(
            obj_stack: List[CLVMStorage],
            hash_stack: List[bytes],
            op_stack: List[OP_STACK_F],
        ) -> None:
            obj = obj_stack.pop()
            r = getattr(obj, "_cached_sha256_treehash", None)
            if r is not None:
                self.cache_hits += 1
                hash_stack.append(r)
                return
            elif obj.atom is not None:
                r = shatree_atom(obj.atom)
                hash_stack.append(r)
                try:
                    setattr(obj, "_cached_sha256_treehash", r)
                except AttributeError:
                    pass
            else:
                pair = cast(Tuple[CLVMStorage, CLVMStorage], obj.pair)
                p0, p1 = pair
                obj_stack.append(obj)
                obj_stack.append(p0)
                obj_stack.append(p1)
                op_stack.append(handle_pair)
                op_stack.append(handle_obj)
                op_stack.append(handle_obj)

        def handle_pair(
            obj_stack: List[CLVMStorage],
            hash_stack: List[bytes],
            op_stack: List[OP_STACK_F],
        ) -> None:
            p0 = hash_stack.pop()
            p1 = hash_stack.pop()
            r = shatree_pair(p0, p1)
            hash_stack.append(r)
            obj = obj_stack.pop()
            try:
                setattr(obj, "_cached_sha256_treehash", r)
            except AttributeError:
                pass

        obj_stack: List[CLVMStorage] = [clvm_storage]
        op_stack: List[OP_STACK_F] = [handle_obj]
        hash_stack: List[bytes] = []
        while len(op_stack) > 0:
            op: OP_STACK_F = op_stack.pop()
            op(obj_stack, hash_stack, op_stack)
        return hash_stack[0]
