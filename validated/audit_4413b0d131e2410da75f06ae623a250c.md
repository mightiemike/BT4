Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating User, Enabling Full Per-User Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which is the direct caller of `MetricOmmPool.swap`. When `MetricOmmSimpleRouter` intermediates a swap, `sender` is the router's address rather than the originating user. Any pool that allowlists the router (required for router-mediated swaps to function) inadvertently grants every user — including those never individually allowlisted — the ability to bypass the per-user swap restriction.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim as `sender` to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` enforces the allowlist exclusively against that `sender`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` all call `pool.swap(...)` directly, making the router the `msg.sender` to the pool: [4](#0-3) [5](#0-4) 

The extension therefore evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. For the router to be usable at all on an allowlisted pool, the pool admin must call `setAllowedToSwap(pool, router, true)`. Once that entry is set, every user routing through the router passes the check regardless of their individual allowlist status.

`DepositAllowlistExtension` does not share this flaw because `beforeAddLiquidity` gates on the explicit `owner` parameter rather than `sender`, and `addLiquidity` accepts `owner` as a caller-supplied argument that the router can set to the actual user: [6](#0-5) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` for compliance, KYC, or access-control purposes has its entire per-user swap gate nullified for any swap routed through `MetricOmmSimpleRouter`. Non-allowlisted addresses can execute swaps against a pool explicitly configured to restrict them, constituting broken core pool access-control functionality with direct fund-flow consequences: unauthorized parties can drain or manipulate restricted liquidity pools.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary public-facing swap entry point. Any pool that enables `SwapAllowlistExtension` and supports router-based swaps (the normal operational mode) must allowlist the router, triggering the bypass automatically. No special privileges, flash loans, or exotic tokens are required — a standard router call suffices. The bypass is repeatable by any address on every such pool.

## Recommendation
Pass the originating user's address through the router to the pool and have the pool forward it to extensions as a verified `originSender`. Concrete options: (1) the router encodes the real user address into `extensionData`, and the extension reads it from there after verifying `msg.sender` is a trusted router; (2) the pool accepts an explicit `swapper` parameter (analogous to `owner` in `addLiquidity`) that the router sets to `msg.sender` before calling the pool, and extensions gate on that field rather than the pool-level `msg.sender`.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` as the `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` — necessary so router-mediated swaps are not universally blocked.
3. Pool admin does **not** call `setAllowedToSwap(pool, alice, true)`.
4. Alice (non-allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
5. Router calls `pool.swap(recipient, ...)` — `msg.sender` inside the pool is the router.
6. Pool calls `_beforeSwap(router, ...)` → extension checks `allowedSwapper[pool][router] = true` → passes.
7. Swap executes successfully despite Alice never being allowlisted.
8. Verification: Alice calls `pool.swap(...)` directly → extension checks `allowedSwapper[pool][alice] = false` → reverts `NotAllowedToSwap`.
