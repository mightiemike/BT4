Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user identity for router-mediated swaps - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of the `pool.swap()` call. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The extension therefore checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actual_user]`, making the per-user allowlist completely ineffective for all router-mediated swaps.

## Finding Description

**Call chain:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)` — `msg.sender = user`
2. Router calls `IMetricOmmPoolActions(params.pool).swap(recipient, ...)` — pool sees `msg.sender = router`
   - `MetricOmmPool.sol` line 231: `_beforeSwap(msg.sender, ...)` passes the router as `sender`
3. `ExtensionCalling._beforeSwap` (line 160–176) encodes `sender = router` and calls the extension
4. `SwapAllowlistExtension.beforeSwap` (line 37) evaluates:
   ```solidity
   allowedSwapper[msg.sender][sender]
   // = allowedSwapper[pool][router]   ← NOT the end user
   ```

**Root cause:** The `sender` argument to `beforeSwap` is the direct caller of `pool.swap()` (the router), not the originating user. The extension has no mechanism to unwrap the router intermediary.

**Bypass path:**
- Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to KYC'd addresses.
- Admin allowlists the router (or the router is allowlisted by default to enable router-based trading).
- Any non-KYC'd user calls `MetricOmmSimpleRouter.exactInputSingle` → pool sees `sender = router` → allowlist passes → swap executes.

**Blocking path (secondary):**
- Admin allowlists individual user addresses but not the router.
- All router-mediated swaps revert for every user, including allowlisted ones, because `allowedSwapper[pool][router]` is false.

**Existing guards are insufficient:** The `BaseMetricExtension` only validates that `msg.sender` (the pool) is a registered pool; it does not recover the originating user from the router's transient storage or any other source.

## Impact Explanation
The swap allowlist — a core access-control mechanism — is rendered ineffective for all swaps routed through `MetricOmmSimpleRouter`. An unprivileged, non-allowlisted trader can execute swaps on a pool the admin intended to restrict, bypassing compliance or risk controls. This constitutes an admin-boundary break: the pool admin's configured allowlist is bypassed by an unprivileged path (the public router). Severity: **High** — the security invariant "only allowlisted addresses may swap" is fully violated for the primary user-facing entry point.

## Likelihood Explanation
The router is the standard, documented entry point for swaps. Any user who calls `MetricOmmSimpleRouter.exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on a restricted pool triggers this bypass automatically, with no special setup. The condition is met whenever the router address is allowlisted (the natural configuration for a pool that supports router trading). Repeatability: every swap through the router bypasses the check.

## Recommendation
In `SwapAllowlistExtension.beforeSwap`, check the `recipient` argument (the intended beneficiary) or, preferably, require the pool to pass the originating user. The cleanest fix is to have the router store the originating `msg.sender` in `extensionData` and have the extension decode and verify it. Alternatively, gate on `tx.origin` as a stopgap (with documented trust assumptions), or require direct pool interaction (no router) for allowlisted pools. The extension's NatSpec comment "Gates `swap` by swapper address" must match the actual checked identity.

function test_nonAllowlistedUserBypassesViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router is allowlisted, attacker is not

    // attacker is NOT allowlisted

    vm.prank(attacker);
    router.exactInputSingle(ExactInputSingleParams({

        tokenIn: token0,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: true,
        priceLimitX64: 0,
        deadline: block.timestamp + 1,
        extensionData: ""
    }));
    // Swap succeeds — allowlist bypassed
}
