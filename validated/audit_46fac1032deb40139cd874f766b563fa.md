### Title
NLP Burn Silently Destroys User Tokens When Computed Quote Amount Falls Below Minimum Fee Threshold — (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

In `Clearinghouse.burnNlp`, when the computed quote value of the NLP tokens being burned is less than the hardcoded minimum burn fee (`ONE = 1e18`), the resulting `quoteAmount` is clamped to zero. The function then burns the user's NLP tokens unconditionally but skips the quote credit entirely, causing a permanent, silent loss of the user's NLP position with zero compensation.

---

### Finding Description

The `burnNlp` function computes the quote amount a user should receive and applies a burn fee:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [1](#0-0) 

`ONE = 1e18` is the hardcoded minimum fee floor. [2](#0-1) 

When `nlpAmount.mul(oraclePriceX18) < ONE` — i.e., the total quote value of the burn is less than 1 unit in 18-decimal representation — the subtraction `quoteAmount - burnFee` goes negative, and `MathHelper.max(0, ...)` clamps it to zero. [3](#0-2) 

The NLP balance is then unconditionally decremented:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);
``` [4](#0-3) 

But the quote credit is gated behind `if (quoteAmount > 0)`, which is never entered:

```solidity
if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
``` [5](#0-4) 

There is no `require(quoteAmount > 0)` guard before the NLP burn. The transaction succeeds, the NLP tokens are permanently transferred to `N_ACCOUNT`, and the user receives nothing.

---

### Impact Explanation

A user who submits a `BurnNlp` transaction under conditions where `nlpAmount * oraclePriceX18 / 1e18 < 1e18` (i.e., the total redemption value is below $1 in protocol-internal units) will have their NLP tokens permanently destroyed with zero quote tokens returned. This is a direct, irreversible loss of principal — not a revert, not a temporary block. The NLP balance is credited to `N_ACCOUNT` and the user's position is zeroed.

**Concrete example:**
- User holds 0.5 NLP tokens (`nlpAmount = 0.5e18`)
- Oracle price = $0.80 (`oraclePriceX18 = 0.8e18`)
- `quoteAmount = 0.5e18 * 0.8e18 / 1e18 = 0.4e18`
- `burnFee = max(1e18, 0.4e18/1000) = 1e18`
- `quoteAmount = max(0, 0.4e18 - 1e18) = 0`
- User loses 0.5 NLP tokens worth $0.40 and receives $0.

This is analogous to the external report: in both cases, a user's redemption of a liquidity position yields zero assets due to a state condition (zero WETH collateral there; fee floor exceeding computed value here), and the user suffers a total loss of the redeemed position.

---

### Likelihood Explanation

The condition `nlpAmount * oraclePriceX18 < ONE` is reachable under two realistic scenarios:

1. **Low NLP price**: If the NLP oracle price drops below $1 (e.g., during a market downturn or protocol stress), any user burning fewer than `1 / price` NLP tokens will trigger this path.
2. **Small burn amounts**: Even at a healthy NLP price of $10, a user burning fewer than 0.1 NLP tokens (worth $1) hits the floor.

The NLP lock period is 4 days (`NLP_LOCK_PERIOD = 4 * 24 * 60 * 60`), meaning users who deposited during a higher-price period and attempt to exit during a lower-price period are particularly exposed. [6](#0-5) 

The sequencer processes the burn with the current oracle price, which is a legitimate market price — no sequencer compromise is required. The user is the unprivileged initiator of the `BurnNlp` transaction.

---

### Recommendation

Add a guard before the NLP balance is decremented to revert when `quoteAmount == 0`, preventing silent destruction of user tokens:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

require(quoteAmount > 0, "ZERO_QUOTE_AMOUNT");
```

Alternatively, apply the fee only up to the available quote amount so that users always receive at least a partial redemption:

```solidity
burnFee = MathHelper.min(burnFee, quoteAmount);
quoteAmount = quoteAmount - burnFee;
```

This mirrors the external report's recommended fix: ensure that as long as there is any redeemable value, the user can exit their position.

---

### Proof of Concept

1. User mints NLP tokens and waits for the 4-day lock period to expire.
2. NLP oracle price drops to $0.80 (`oraclePriceX18 = 0.8e18`).
3. User submits a `BurnNlp` transaction for 0.5 NLP tokens (`nlpAmount = 0.5e18`).
4. Sequencer calls `Clearinghouse.burnNlp` with `oraclePriceX18 = 0.8e18`.
5. `quoteAmount = 0.5e18 * 0.8e18 / 1e18 = 0.4e18`.
6. `burnFee = max(1e18, 0.4e18/1000) = 1e18`.
7. `quoteAmount = max(0, 0.4e18 - 1e18) = 0`.
8. `spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -0.5e18)` executes — NLP burned.
9. `if (quoteAmount > 0)` is false — no quote credited.
10. Health checks pass (NLP balance is now 0, which is `>= 0`).
11. Transaction succeeds. User has lost 0.5 NLP tokens worth $0.40 and received $0. [7](#0-6)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
```
