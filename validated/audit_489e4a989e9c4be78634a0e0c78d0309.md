### Title
`updateTierFeeRates()` with `QUOTE_PRODUCT_ID` Does Not Propagate to Products Added After the Call — (`OffchainExchange.sol`)

---

### Summary

When `updateTierFeeRates()` is called with `productId == QUOTE_PRODUCT_ID` to apply a fee rate to all products for a given tier, it only iterates over products that exist at call time. The `nonDefaultFeeTierMask` bit for that tier is set globally, but any product added after this call will have uninitialized (zero) fee rates for that tier. `getTierFeeRateX18()` then returns `FeeRates({makerRateX18: 0, takerRateX18: 0})` for those new products, causing all traders in the affected tier to pay zero fees.

---

### Finding Description

`updateTierFeeRates()` in `OffchainExchange.sol` handles the `QUOTE_PRODUCT_ID` case by iterating over all currently registered spot and perp products and writing `feeRates[tier][productId]` for each:

```solidity
// OffchainExchange.sol lines ~962–991
function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
    external
    onlyEndpoint
{
    if (txn.productId == QUOTE_PRODUCT_ID) {
        uint32[] memory spotProductIds = spotEngine.getProductIds();
        uint32[] memory perpProductIds = perpEngine.getProductIds();
        for (uint32 i = 0; i < spotProductIds.length; i++) {
            ...
            feeRates[txn.tier][spotProductIds[i]] = FeeRates(...);
        }
        for (uint32 i = 0; i < perpProductIds.length; i++) {
            feeRates[txn.tier][perpProductIds[i]] = FeeRates(...);
        }
    } else {
        feeRates[txn.tier][txn.productId] = FeeRates(...);
    }
    nonDefaultFeeTierMask |= uint128(1) << txn.tier;  // <-- set globally for the tier
}
``` [1](#0-0) 

The `nonDefaultFeeTierMask` bit is set for the tier unconditionally. `getTierFeeRateX18()` uses this mask to decide whether to return the stored per-product rate or the hardcoded default:

```solidity
// OffchainExchange.sol lines ~933–946
function getTierFeeRateX18(uint32 tier, uint32 productId)
    public view returns (FeeRates memory)
{
    if (nonDefaultFeeTierMask & (1 << tier) != 0) {
        return feeRates[tier][productId];   // returns zero for new products
    }
    return FeeRates({makerRateX18: 0, takerRateX18: 200_000_000_000_000});
}
``` [2](#0-1) 

Once the bit is set for a tier, `getTierFeeRateX18()` always reads from `feeRates[tier][productId]`. For any product added after `updateTierFeeRates()` was called, `feeRates[tier][newProductId]` is the zero value of the mapping — `FeeRates({makerRateX18: 0, takerRateX18: 0})`. The new product is never written into `feeRates` during `addOrUpdateProduct()`:

```solidity
// SpotEngine.sol lines ~68–97
function addOrUpdateProduct(...) public onlyOwner {
    bool isNewProduct = _addOrUpdateProduct(...);
    configs[productId] = config;
    if (isNewProduct) {
        _setState(productId, State({...}));
    }
    // no updateTierFeeRates call
}
``` [3](#0-2) 

`_addOrUpdateProduct()` in `BaseEngine` only calls `_exchange().updateMarket()` (which sets `minSize`/`sizeIncrement`) and does not touch fee rates: [4](#0-3) 

The fee is applied during `matchOrders()` via `applyFee()` → `getUserFeeRateWithBuilder()` → `getTierFeeRateX18()`: [5](#0-4) 

---

### Impact Explanation

For every tier whose bit is set in `nonDefaultFeeTierMask`, any product added after the `updateTierFeeRates(QUOTE_PRODUCT_ID)` call will have `takerRateX18 = 0`. Takers trading on those products pay zero fees. If tier 0 (the default tier, used by all users whose `feeTiers[user]` was never explicitly set) has been customized, this affects every trader on every newly listed product. The protocol loses all taker fee revenue on new listings until the admin manually calls `updateTierFeeRates` again for each new product.

---

### Likelihood Explanation

This is a realistic operational sequence: the protocol customizes fee tiers at launch (calling `updateTierFeeRates` with `QUOTE_PRODUCT_ID`), then lists new trading pairs over time via `addOrUpdateProduct`. Any trader can immediately exploit zero fees on new products simply by submitting orders through the normal `matchOrders` path — no special privileges required. The sequencer processes the trade and `applyFee` silently computes a zero fee.

---

### Recommendation

When `addOrUpdateProduct` registers a new product, iterate over all tiers present in `nonDefaultFeeTierMask` and initialize `feeRates[tier][newProductId]` to a sensible default (e.g., copy from an existing product or from a stored "global default" per tier). Alternatively, store a per-tier default rate separately and fall back to it when `feeRates[tier][productId]` is zero for a non-default tier, rather than returning zero unconditionally.

---

### Proof of Concept

1. Admin calls `updateTierFeeRates({tier: 0, productId: QUOTE_PRODUCT_ID, makerRateX18: 0, takerRateX18: 1.5bps})`.
   - All existing products get `feeRates[0][existingProduct] = {0, 1.5bps}`.
   - `nonDefaultFeeTierMask |= 1` (tier 0 is now non-default).
2. Admin calls `SpotEngine.addOrUpdateProduct(newProductId, ...)` to list a new token.
   - `feeRates[0][newProductId]` remains `{0, 0}` (uninitialized).
3. Trader (tier 0, the default for all users) submits a taker order on `newProductId`.
   - `getTierFeeRateX18(0, newProductId)` → `nonDefaultFeeTierMask & 1 != 0` → returns `feeRates[0][newProductId]` = `{0, 0}`.
   - `applyFee` computes `feeRate = 0`, `fee = 0`.
   - Trader pays zero taker fees on the new product indefinitely.

        uint32 productId,
        OrderInfo memory orderInfo,
        MarketInfo memory market,
        int128 alreadyMatched, // in quote

        // X account is passthrough for trading and incurs
        // no fees
        if (orderInfo.sender == X_ACCOUNT) {
            return;
        }
        int128 matchQuote = orderInfo.quoteDelta;
        int128 meteredQuote = 0;
        if (taker) {
            // flat minimum fee
            if (alreadyMatched == 0) {
                meteredQuote += market.minSize;
                if (matchQuote < 0) {
                    meteredQuote = -meteredQuote;
                }
            }

            // exclude the portion on [0, self.min_size) for match_quote and
            // add to metered_quote
            // fee is only applied on [minSize, quote_amount)
            int128 feeApplied = MathHelper.abs(alreadyMatched + matchQuote) -
                market.minSize;
            feeApplied = MathHelper.min(feeApplied, matchQuote.abs());
            if (feeApplied > 0) {
                if (matchQuote < 0) {
                    feeApplied = -feeApplied;
                }
                meteredQuote += feeApplied;
            }
        } else {
            // for maker rebates things stay the same
            meteredQuote += matchQuote;

        FeeInfo memory feeInfo = getUserFeeRateWithBuilder(
            orderInfo.sender,
            productId,
            appendix,
            taker

        int128 newMeteredQuote = (meteredQuote > 0)
            ? meteredQuote.mul(keepRateX18)
            : meteredQuote.div(keepRateX18);
        orderInfo.fee = meteredQuote - newMeteredQuote;
        orderInfo.builderFee = matchQuote.abs().mul(feeInfo.builderFeeRate);

    function _addOrUpdateProduct(
        uint32 productId,
        uint32 quoteId,
        int128 sizeIncrement,
        int128 minSize,
        RiskHelper.RiskStore memory riskStore
    ) internal returns (bool isNewProduct) {
        require(
            riskStore.longWeightInitial <= riskStore.longWeightMaintenance &&
                riskStore.longWeightMaintenance <= 10**9 &&
                riskStore.shortWeightInitial >=
                riskStore.shortWeightMaintenance &&
                riskStore.shortWeightMaintenance >= 10**9,
            ERR_BAD_PRODUCT_CONFIG
        );

        // product id is in ascending order
        if (
            productIds.length == 0 ||
            productId > productIds[productIds.length - 1]
        ) {
            productIds.push(productId);
            _clearinghouse.registerProduct(productId);
            isNewProduct = true;
        }

        if (isNewProduct) {
            IEndpoint(getEndpoint()).setInitialPrice(
                productId,
                riskStore.priceX18
            );
        } else {
            riskStore.priceX18 = _risk().value[productId].priceX18;
        }
        _risk().value[productId] = riskStore;
        _exchange().updateMarket(productId, quoteId, sizeIncrement, minSize);

        emit AddOrUpdateProduct(productId);
    }
