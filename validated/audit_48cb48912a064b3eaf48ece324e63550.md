### Title
`ExecutionStateReader::get_compiled_class_hash` reads from the declaration-block state diff instead of the most-recent compiled-class-hash table, causing RPC simulation and fee estimation to re-trigger CASM hash migration that has already been committed to state - (`File: crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

After the CASM hash migration from V1 (Poseidon) to V2 (Blake) is committed to the chain (enabled in Starknet ≥ 0.14.3), the `ExecutionStateReader` used by all RPC execution paths (`estimate_fee`, `simulate_transactions`, `starknet_call`) still returns the old V1 compiled class hash for migrated classes. This is because it reads from the state diff at the class's *declaration* block rather than from the most-recent compiled-class-hash table. The sequencer's own `ApolloReader` correctly uses `get_compiled_class_hash_at`, which finds the most-recent entry and returns V2 after migration. The divergence causes `should_migrate` to fire again inside every RPC simulation, producing wrong state diffs and inflated fee estimates.

### Finding Description

**Root cause — `ExecutionStateReader::get_compiled_class_hash`** [1](#0-0) 

The function locates the block at which the class was declared, then reads the state diff at *that specific block*:

```rust
let maybe_block_number = self.storage_reader...get_class_definition_block_number(&class_hash)?;
let state_diff = self.storage_reader...get_state_diff(block_number)?;
let compiled_class_hash = state_diff.class_hash_to_compiled_class_hash.get(&class_hash)...;
```

The declaration-block state diff is written once and never updated. When the migration runs in a later block, it writes the V2 hash into a *new* state diff entry for the migration block. The declaration-block entry is left unchanged with the V1 hash.

**Correct implementation — `ApolloReader::get_compiled_class_hash`** [2](#0-1) 

`ApolloReader` calls `get_compiled_class_hash_at(state_number, &class_hash)`, which uses a B-tree cursor to find the *most-recent* entry at or before the requested block: [3](#0-2) 

After migration, the cursor finds the V2 entry written by the migration block and returns it correctly.

**Migration mechanism**

Migration is enabled in `VersionedConstants` starting with Starknet 0.14.3: [4](#0-3) 

The `should_migrate` function compares `get_compiled_class_hash` (V1 from declaration block via `ExecutionStateReader`) with `get_compiled_class_hash_v2` (V2 from class manager): [5](#0-4) 

Because `ExecutionStateReader` always returns V1, `should_migrate` always returns `Some(...)` for migrated classes, even after migration has been committed.

**Migration gas is added to every RPC simulation**

`CasmHashMigrationData::from_state` calls `should_migrate` and accumulates migration resources: [6](#0-5) 

This is called from `get_tx_weights`, which is invoked by `TransactionExecutor::execute_txs` — the same executor used by `estimate_fee` and `simulate_transactions`.

**`get_compiled_class_hash_v2` is correct in `ExecutionStateReader`** [7](#0-6) 

It reads from the class manager or the `executable_class_hash_v2` storage column, which is populated correctly. The divergence is only in `get_compiled_class_hash`.

### Impact Explanation

Every `estimate_fee` or `simulate_transactions` call that touches a class migrated from V1 to V2 will:

1. Have `should_migrate` return `Some(...)` (V1 ≠ V2 in the RPC state reader).
2. Include migration gas in the bouncer weights, inflating the fee estimate by the full migration cost (measured at ~111 M sierra-gas per class in tests).
3. Produce a simulated state diff that contains `class_hash → compiled_class_hash_v2` entries that are already present in the committed state, making the simulation diverge from what the sequencer would actually produce.

This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

### Likelihood Explanation

- `enable_casm_hash_migration = true` is the production setting for all Starknet versions ≥ 0.14.3.
- Any Cairo 1 class declared before 0.14.3 (i.e., with a V1 Poseidon hash in state) and executed after migration is affected.
- No special privileges are required; any caller of `estimate_fee` or `simulate_transactions` triggers the wrong path.
- The divergence is permanent: the declaration-block state diff is immutable, so the bug persists for the lifetime of every pre-migration class.

### Recommendation

Replace the declaration-block state-diff lookup in `ExecutionStateReader::get_compiled_class_hash` with the same `get_compiled_class_hash_at(state_number, &class_hash)` call used by `ApolloReader`. This ensures the RPC execution path sees the most-recent compiled class hash, including any migration updates, and eliminates the divergence between the two state readers.

```rust
// In ExecutionStateReader::get_compiled_class_hash
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    // ... pending data check unchanged ...

    let state_number = self.state_number; // use the reader's own state number
    let compiled_class_hash = self.storage_reader
        .begin_ro_txn()
        .map_err(storage_err_to_state_err)?
        .get_state_reader()
        .map_err(storage_err_to_state_err)?
        .get_compiled_class_hash_at(state_number, &class_hash)
        .map_err(storage_err_to_state_err)?;

    Ok(compiled_class_hash.unwrap_or_default())
}
```

### Proof of Concept

1. Deploy a Cairo 1 contract on a node running Starknet 0.14.2 (`enable_casm_hash_migration = false`). The state diff records `class_hash → compiled_class_hash_v1` (Poseidon hash).
2. Upgrade the node to Starknet 0.14.3 (`enable_casm_hash_migration = true`). The sequencer executes a block that touches the class; `CasmHashMigrationData::from_state` via `ApolloReader` detects V1 ≠ V2 and writes `class_hash → compiled_class_hash_v2` into the new block's state diff.
3. Call `starknet_estimateFee` for any transaction that invokes the migrated class.
4. Observe that `ExecutionStateReader::get_compiled_class_hash` returns the V1 hash (from the declaration block's state diff), while `get_compiled_class_hash_v2` returns V2.
5. `should_migrate` returns `Some(...)` → migration gas (~111 M sierra-gas) is added to the estimate.
6. Call `starknet_simulateTransactions` and inspect the returned state diff: it contains `class_hash → compiled_class_hash_v2` even though that mapping is already committed to state, diverging from the actual sequencer output.

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }
}

pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }

    fn from_state<S: StateReader>(
        state_reader: &S,
        executed_class_hashes: &HashSet<ClassHash>,
        versioned_constants: &VersionedConstants,
    ) -> TransactionExecutionResult<Self> {
        if !versioned_constants.enable_casm_hash_migration {
            return Ok(Self::empty());

        executed_class_hashes.iter().try_fold(Self::empty(), |mut migration_data, &class_hash| {

                let class = state_reader.get_compiled_class(class_hash)?;
                migration_data.resources +=
                    &class.estimate_compiled_class_hash_migration_resources();
            }
            Ok(migration_data)
        })
