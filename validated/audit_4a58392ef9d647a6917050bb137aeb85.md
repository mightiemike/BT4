Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Allowing Any Caller to Bypass the Swap Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` performs its allowlist check keyed on the `sender` argument, which is `msg.sender` inside `MetricOmmPool.swap`. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating EOA. The extension therefore evaluates whether the router is allowlisted rather than the actual trader. If the pool admin allowlists the router to permit legitimate users to trade, every unprivileged address on the internet can bypass the curated allowlist by routing through the public router.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // <-- direct caller of pool.swap()
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

`SwapAllowlistExtension.beforeSwap` then performs its allowlist lookup keyed on `(msg.sender=pool, sender=direct_caller_of_pool)`:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    ...
    params.extensionData
  );
```

At that point `msg.sender` inside the pool is the **router contract address**, not the originating EOA. The extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][bob]`.

**Exploit path:**
1. Pool P is deployed with `SwapAllowlistExtension`. Pool admin sets `allowedSwapper[P][alice] = true` and also sets `allowedSwapper[P][router] = true` so that alice can use the router.
2. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: P, recipient: bob, ...})`.
3. Router calls `pool.swap(recipient=bob, ...)` with `msg.sender = router`.
4. Pool calls `_beforeSwap(sender=router, ...)`.
5. Extension checks `allowedSwapper[P][router]` → `true` → swap proceeds.
6. Bob receives tokens from a pool he was never supposed to access.

Alternatively, if the pool admin does NOT allowlist the router, then even allowlisted users like alice cannot use the router at all — the allowlist is broken in both directions.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (KYC'd counterparties, institutional LPs, whitelisted market makers) provides **zero effective access control** for router-mediated swaps once the router is allowlisted. Any non-allowlisted address can call `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` on the public router and execute swaps against the restricted pool. This constitutes a direct loss of LP principal and a broken core pool invariant — the allowlist policy the pool was explicitly configured to enforce is completely bypassed. This meets the "Broken core pool functionality causing loss of funds" and "Admin-boundary break bypassed by an unprivileged path" impact criteria.

## Likelihood Explanation

Likelihood is **high**. `MetricOmmSimpleRouter` is a public, permissionless periphery contract. No special privilege, token balance, or setup is required beyond knowing the pool address. The bypass is a single function call. Any actor denied direct pool access will immediately discover that routing through the router succeeds if the router is allowlisted. The pool admin is incentivized to allowlist the router to allow legitimate users to trade conveniently, which simultaneously opens the bypass to all users.

## Recommendation

The `SwapAllowlistExtension` must gate the **originating user**, not the intermediary router. Two sound approaches:

1. **Check `recipient` instead of `sender`**: For most swap use cases the recipient is the economic beneficiary; allowlisting on recipient closes the router bypass for the common case.
2. **Require the router to attest the real caller via `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension trusts only known routers and reads the attested address from the payload, then checks `allowedSwapper[pool][attestedCaller]`.

  Pool P deployed with SwapAllowlistExtension.

Attack:
  Bob (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({
        pool: P,
        recipient: bob,
        zeroForOne: true,
        amountIn: X,
        ...
    })

  Router calls pool.swap(recipient=bob, ...) with msg.sender = router.
  Pool calls _beforeSwap(sender=router, recipient=bob, ...).
  SwapAllowlistExtension.beforeSwap checks allowedSwapper[P][router] → true.
  Swap proceeds. Bob receives tokens from a pool he was never supposed to access.

Foundry test outline:
  1. Deploy SwapAllowlistExtension, pool, and MetricOmmSimpleRouter.
  2. setAllowedToSwap(pool, alice, true); setAllowedToSwap(pool, router, true).
  3. vm.prank(bob); router.exactInputSingle({pool: pool, recipient: bob, ...}).
  4. Assert swap succeeded for bob despite bob not being in the allowlist.
  5. Assert allowedSwapper[pool][bob] == false (confirming bypass).
