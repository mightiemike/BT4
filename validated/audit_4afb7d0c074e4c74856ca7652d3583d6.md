Audit Report

## Title
Router-Mediated Swaps Check Router Address Instead of End User in SwapAllowlistExtension — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` as seen by `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the hook checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. This breaks the allowlist invariant in both directions: allowlisting the router grants access to all users regardless of individual authorization, and allowlisting specific EOAs blocks those users from using the router entirely.

## Finding Description

**Call chain (confirmed against production code):**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)` — `msg.sender` = User
2. Router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — pool sees `msg.sender` = **Router** [1](#0-0) 
3. `MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, passing **Router** as `sender` [2](#0-1) 
4. `ExtensionCalling._beforeSwap` encodes `sender` (= Router) and dispatches to the extension [3](#0-2) 
5. `SwapAllowlistExtension.beforeSwap` receives `sender` = **Router**, `msg.sender` = **Pool**, and evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [4](#0-3) 

The check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. The router stores `msg.sender` (the user) only in transient callback context for payment purposes via `_setNextCallbackContext`, never forwarding it to the pool's `swap()` identity parameter. [5](#0-4) 

**Two concrete failure modes:**

**A — Allowlist bypass:** If the pool admin allowlists the router address (a natural action for "allow all router users"), any unprivileged user bypasses the per-user allowlist by routing through `MetricOmmSimpleRouter`. The hook sees `allowedSwapper[pool][router] = true` and passes for every user.

**B — Allowlist denial:** If the pool admin allowlists specific EOAs but not the router, those allowlisted users cannot use the router at all — the hook sees `allowedSwapper[pool][router] = false` and reverts even though the user is individually authorized.

No existing guard in the extension, pool, or router prevents either failure mode.

## Impact Explanation

Scenario A is a direct policy bypass on curated/permissioned pools. The pool admin intended to gate specific addresses; any user circumvents this by routing through the public router. This breaks the core access-control invariant of `SwapAllowlistExtension` and constitutes broken core pool functionality for allowlisted pools — a swap that should be blocked proceeds, or a swap that should succeed is blocked. The exact corrupted value is `allowedSwapper[pool][sender]` being evaluated against the router address rather than the economic actor.

## Likelihood Explanation

Allowlisting the router is a natural and expected admin action for pools that want to allow "all router users" while blocking raw pool calls. The bypass is single-transaction, requires no special privileges, no front-running, and no two-transaction setup. Any unprivileged user with knowledge of the router address can exploit it immediately.

## Recommendation

The `beforeSwap` hook must be keyed to the economic actor, not the transport layer:
- The pool/extension design should distinguish between `sender` (transport) and `originator` (economic actor). The router would need to be a trusted forwarder that attests the originator identity in a verifiable way (e.g., via `extensionData` with a router-signed attestation).
- Alternatively, document explicitly that `SwapAllowlistExtension` only supports direct pool calls and add a revert if `sender` is a known router address.
- Most robustly: redesign the hook interface to carry an `originator` field populated by the pool from a trusted forwarder registry, separate from `sender`.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   (admin allowlists the router)
  allowedSwapper[pool][attacker] = false (attacker is NOT allowlisted)

Attack:
  attacker calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  → router calls pool.swap(recipient, ...)  [msg.sender = router]
  → pool calls _beforeSwap(router, ...)
  → extension checks allowedSwapper[pool][router] == true  ✓
  → swap proceeds for attacker despite attacker not being allowlisted

Result: allowlist bypassed; attacker swaps on a curated pool they were not authorized to access.
```
