### Title
Missing Blocked-Address Guard in `processReceivedPacket` Allows Permanent NFT Loss to Module Accounts — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` decodes `data.Receiver` as a bech32 address and passes it directly to `MintNFT` or `TransferOwner` with no check against blocked/module accounts. A counterparty chain (or any user sending an IBC NFT back to the origin chain) can set `Receiver` to a module account address (e.g., the `nft-transfer` module account), causing the NFT to be permanently locked in an account that can never sign a transaction to recover it.

---

### Finding Description

`OnRecvPacket` in `ibc_module.go` calls `keeper.OnRecvPacket`, which calls `data.ValidateBasic()` then `processReceivedPacket`.

`ValidateBasic()` only checks that `Receiver` is a non-blank, syntactically valid bech32 address: [1](#0-0) 

A module account address (e.g., the `nft-transfer` module account) is a perfectly valid bech32 address and passes this check.

`processReceivedPacket` then decodes the receiver and calls either `MintNFT` (away-from-origin path) or `TransferOwner` (back-to-origin path) with no blocked-address guard: [2](#0-1) [3](#0-2) [4](#0-3) 

The `Keeper` struct holds an `authKeeper` but it is only used for `SetEscrowAddress`; there is no `BankKeeper` or any interface that exposes `BlockedAddr`: [5](#0-4) 

The NFT module's own `expected_keepers.go` defines `AccountKeeper` with only `GetAccount` and `BankKeeper` with only balance methods — no blocked-address capability exists anywhere in the stack: [6](#0-5) 

By contrast, the ICS-20 fungible token transfer module in ibc-go explicitly calls `k.bankKeeper.BlockedAddr(receiver)` and returns an error acknowledgement before any mint or unescrow. No equivalent guard exists here.

---

### Impact Explanation

- **Away-from-origin (mint path):** A counterparty chain sends a packet with `Receiver` = module account address. The destination chain mints a new IBC voucher NFT directly into the module account. Module accounts have no private key; the NFT is permanently unrecoverable.
- **Back-to-origin (unescrow path):** A user on chain B holds an IBC voucher for an NFT originally escrowed on chain A. They submit `MsgTransfer` with `Receiver` = a module account address on chain A. Chain A unescrows the original NFT to the module account. The original NFT is permanently lost; the user's voucher is burned.

In both cases the NFT ownership record is written to a module account address with no recovery path, satisfying the "loss of NFTs" criterion in the target scope.

---

### Likelihood Explanation

- The back-to-origin path requires only a normal user transaction (`MsgTransfer`) on the counterparty chain with `Receiver` set to a known module account address (module account addresses are deterministic and publicly derivable). No privileged access is needed.
- The away-from-origin path requires a malicious or misconfigured counterparty chain, which is a higher bar.
- Accidental loss (user error) is also realistic since there is no UI-level or protocol-level rejection.

---

### Recommendation

Add a blocked-address check in `processReceivedPacket` immediately after decoding the receiver, mirroring the ICS-20 pattern:

```go
// in processReceivedPacket, after line 146
if k.bankKeeper.BlockedAddr(receiver) {
    return sdkerrors.Wrapf(sdkerrors.ErrUnauthorized,
        "receiver address %s is a blocked address", receiver)
}
```

This requires adding a `BankKeeper` (or a minimal `BlockedAddrKeeper`) interface to the nft-transfer `Keeper` and wiring it at app construction. The `types/expected_keepers.go` for the nft-transfer module should define:

```go
type BlockedAddrKeeper interface {
    BlockedAddr(addr sdk.AccAddress) bool
}
```

---

### Proof of Concept

1. Deploy a two-chain local testnet with the nft-transfer module enabled.
2. On chain A, issue a denom and mint an NFT to a user address.
3. User on chain A sends the NFT to chain B via `MsgTransfer` (escrow path). NFT is now escrowed on chain A; chain B holds a voucher.
4. From chain B, submit `MsgTransfer` back to chain A with `Receiver` = `cosmos1jv65s3grqf6v6jl3dp4t6c9t9rk99cd88lyufl` (the `distribution` module account, or compute the `nft-transfer` module account address).
5. Observe that chain A's `processReceivedPacket` calls `TransferOwner(ctx, classID, tokenID, escrowAddress, moduleAccAddr)` without error.
6. Query NFT ownership on chain A: the NFT is now owned by the module account.
7. Attempt any recovery transaction — none is possible since the module account has no signing key. [7](#0-6)

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}

	return nil
```
