### Title
Unprivileged Caller Can Permanently Prune Mainchain Headers via `run_mainchain_gc()` - (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc()` is a state-mutating public function that permanently deletes mainchain block headers from contract storage. Unlike `submit_blocks()`, which is gated by `#[trusted_relayer]`, `run_mainchain_gc()` carries only a `#[pause]` guard — meaning any unprivileged NEAR account can call it freely when the contract is not paused. An attacker can supply a large `batch_size` to prune all stored mainchain headers, permanently destroying the SPV verification capability of the light client.

### Finding Description
`submit_blocks()` is protected by `#[trusted_relayer]`, restricting it to registered, staked relayers: [1](#0-0) 

`run_mainchain_gc()` carries only a pause guard with no equivalent caller restriction: [2](#0-1) 

The `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute only controls access when the contract is **paused** — it grants `UnrestrictedRunGC` holders the ability to bypass the pause. When the contract is running normally, the function is callable by any NEAR account with no role check at all.

The function permanently removes the oldest mainchain headers from `headers_pool` and `mainchain_height_to_header`, then advances `mainchain_initial_blockhash`: [3](#0-2) 

The existence of the `UnrestrictedRunGC` role — mirroring the `UnrestrictedSubmitBlocks` role that bypasses the pause for `submit_blocks` — makes the design intent clear: `run_mainchain_gc` was meant to be a privileged operation, but the caller-restriction guard was never added. [4](#0-3) 

### Impact Explanation
An unprivileged attacker calls `run_mainchain_gc(u64::MAX)`. The function computes `selected_amount_to_remove = min(total_to_remove, batch_size)` and deletes every mainchain header older than the current `gc_threshold` window: [5](#0-4) 

Concrete consequences:
1. **SPV proofs permanently broken**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up `args.tx_block_blockhash` in `mainchain_header_to_height`. After GC, any block in the pruned range returns `"block does not belong to the current main chain"`, making all historical SPV proofs permanently unverifiable.
2. **Chain reorg permanently broken**: `reorg_chain()` walks back through `headers_pool` to find the common ancestor. If the fork point falls in the pruned range, the contract panics with `PrevBlockNotFound`, permanently breaking the ability to process any reorg that touches pruned heights.
3. **Irreversible**: deleted entries cannot be restored without a full contract re-initialization.

### Likelihood Explanation
The entry point is a standard NEAR contract call requiring no tokens, no role, and no special setup — any NEAR account can execute it in a single transaction. The `batch_size` parameter is uncapped, so a single call is sufficient to prune the entire stored history. The attack is cheap, deterministic, and requires no privileged access.

### Recommendation
Add a caller-restriction guard to `run_mainchain_gc()` equivalent to the one on `submit_blocks()`. The simplest approach is to introduce a dedicated `GCManager` role (or reuse `RelayerManager`/`DAO`) and require the caller to hold it, mirroring the `#[trusted_relayer]` pattern:

```rust
// Option A: reuse the existing role hierarchy
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::RelayerManager, &env::predecessor_account_id())
            || self.acl_has_role(Role::DAO, &env::predecessor_account_id()),
        "Unauthorized: caller lacks GC permission"
    );
    // ... existing body
}
```

Alternatively, add `#[trusted_relayer]` so only active registered relayers can trigger GC, consistent with the model used for `submit_blocks`.

### Proof of Concept
```rust
// Any unprivileged NEAR account can execute this:
let unauthorized = sandbox.dev_create_account().await?;

// Prune the entire mainchain history in one call
let outcome = unauthorized
    .call(contract.id(), "run_mainchain_gc")
    .args_json(json!({"batch_size": u64::MAX}))
    .max_gas()
    .transact()
    .await?;

assert!(outcome.is_success()); // succeeds with no role check

// SPV proof for any pruned block now fails permanently
let proof_result = unauthorized
    .view(contract.id(), "verify_transaction_inclusion_v2")
    .args_borsh(valid_proof_args_for_pruned_block)
    .await;
assert!(proof_result.is_err()); // "block does not belong to the current main chain"
```

The existing integration test at `contract/tests/test_basics.rs` already demonstrates that `run_mainchain_gc` succeeds when called by an unprivileged `user_account` with no role grants: [6](#0-5)
