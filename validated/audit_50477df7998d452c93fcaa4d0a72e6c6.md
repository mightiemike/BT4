### Title
Protobuf `ValidResourceBounds` Round-Trip Collapses `AllResources` to `L1Gas` When l2_gas and l1_data_gas Are Zero, Producing a Wrong Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf serialization of `ValidResourceBounds::AllResources` with zero `l2_gas` and `l1_data_gas` is byte-identical to the serialization of `ValidResourceBounds::L1Gas`. On deserialization the converter always reconstructs `L1Gas` for this case. Because `get_tip_resource_bounds_hash` hashes a different number of resource elements depending on the variant (`L1Gas` → 2 resources; `AllResources` → 3 resources), the Poseidon hash computed after the round-trip differs from the hash the sender signed. Any node that re-derives the transaction hash from the deserialized form (e.g. during state sync or RPC tracing) will produce and serve a wrong value.

### Finding Description

**Serialization side** (`From<ValidResourceBounds> for protobuf::ResourceBounds`):

```
ValidResourceBounds::L1Gas(l1_gas)  →  { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
ValidResourceBounds::AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
                                    →  { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
```

Both variants produce **identical** protobuf bytes. [1](#0-0) 

**Deserialization side** (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`):

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← always chosen for the ambiguous case
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

**Hash divergence** (`get_tip_resource_bounds_hash`):

For `L1Gas` the hash input is `[tip, L1_GAS_packed, L2_GAS_packed(0)]` — **3 Poseidon elements**.
For `AllResources` the hash input is `[tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)]` — **4 Poseidon elements**. [3](#0-2) 

These two inputs produce different Poseidon digests, so the transaction hash computed before serialization (`AllResources` path, 4 elements) differs from the hash computed after deserialization (`L1Gas` path, 3 elements).

The `ValidResourceBounds` enum comment itself documents the version boundary: `L1Gas` is pre-0.13.3; `AllResources` is 0.13.3+. A V3 transaction submitted via the RPC gateway always carries `AllResourceBounds` (and is therefore always wrapped in `AllResources`), so any V3 transaction with zero `l2_gas` and `l1_data_gas` is a valid, gateway-accepted input that triggers the collapse. [4](#0-3) 

### Impact Explanation

A node that re-derives a transaction hash from the protobuf-deserialized form — during state sync, RPC `get_transaction`, `get_transaction_receipt`, fee estimation, or tracing — will compute and return a hash that differs from the hash the sender signed and that the sequencer stored. This satisfies:

- **High**: "RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."
- **High**: "Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."

A syncing node that calls `validate_transaction_hash` on the deserialized transaction will compute the wrong hash and reject a valid block, breaking state synchronization for any block containing such a transaction. [5](#0-4) 

### Likelihood Explanation

Any user who submits a V3 Invoke transaction with `l2_gas = ResourceBounds::default()` and `l1_data_gas = ResourceBounds::default()` (both fields zero) triggers the bug. The gateway stateless validator does not require non-zero `l2_gas` or `l1_data_gas`; the test suite explicitly exercises `AllResourceBounds` with individual resources set to zero. No special privilege is required. [6](#0-5) 

### Recommendation

The deserialization must preserve the original variant. The simplest fix is to encode the variant explicitly — for example, by using the presence/absence of the `l1_data_gas` field as the discriminant (the proto comment already notes `l1_data_gas` is `optional` for pre-0.14.0 compatibility):

```rust
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Alternatively, the serialization of `L1Gas` should omit `l1_data_gas` entirely (leave it `None`) so the round-trip is injective. Either way, the invariant that must hold is:

```
hash(serialize(deserialize(bytes))) == hash(deserialize(bytes))
``` [7](#0-6) 

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, ValidResourceBounds, GasAmount, GasPrice,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;
use apollo_protobuf::protobuf;

// 1. Construct AllResources with zero l2_gas and l1_data_gas.
let l1_gas = ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) };
let original = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas,
    l2_gas: ResourceBounds::default(),      // zero
    l1_data_gas: ResourceBounds::default(), // zero
});

// 2. Compute hash before round-trip (AllResources path → 4 Poseidon elements).
let tip = Tip(0);
let hash_before = get_tip_resource_bounds_hash(&original, &tip).unwrap();

// 3. Serialize to protobuf and deserialize back.
let proto: protobuf::ResourceBounds = original.into();
// proto.l1_data_gas == Some(zero), proto.l2_gas == Some(zero)
let recovered = ValidResourceBounds::try_from(proto).unwrap();
// recovered == ValidResourceBounds::L1Gas(l1_gas)  ← WRONG variant

// 4. Compute hash after round-trip (L1Gas path → 3 Poseidon elements).
let hash_after = get_tip_resource_bounds_hash(&recovered, &tip).unwrap();

// 5. Hashes differ → any node recomputing the hash from the deserialized form
//    will produce a wrong value.
assert_ne!(hash_before, hash_after); // PASSES — demonstrates the bug
``` [1](#0-0) [8](#0-7)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
