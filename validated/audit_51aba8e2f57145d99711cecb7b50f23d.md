Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any Caller to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` inside `MetricOmmPool.swap`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` in the pool is the router contract, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. Any pool admin who allowlists the router — required for legitimate router-mediated swaps — inadvertently opens the pool to every user, completely defeating the allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap` (L230–231). `ExtensionCalling._beforeSwap` forwards this value unchanged to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))` (L160–176). `SwapAllowlistExtension.beforeSwap` then checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap(...)` directly (L72–80), making the router the `sender` seen by the extension. The actual end-user's address is never evaluated. A pool admin who allowlists the router (necessary for any router-mediated swap to succeed) simultaneously grants swap access to every address on the network that can call the router. The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

## Impact Explanation
The `SwapAllowlistExtension` is the sole on-chain mechanism preventing unauthorized swaps on curated pools. Once the router is allowlisted, non-allowlisted users can execute swaps freely, draining LP value through price impact, extracting arbitrage, or violating compliance constraints the pool admin intended to enforce. This constitutes a direct loss of LP principal and a broken core pool invariant (allowlist bypass enabling unauthorized swap execution).

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing entry point for the protocol. Any pool admin who deploys a `SwapAllowlistExtension`-gated pool and wants their allowlisted users to use the router must allowlist the router address — this is the natural and expected operational pattern. The bypass is triggered by normal, correct administrative setup and requires no privileged access or special conditions from the attacker. Any EOA can call `exactInputSingle` on the router.

## Recommendation
The extension must resolve the true end-user identity rather than trusting the `sender` argument when the caller is a known periphery contract. The most robust fix is to introduce a separate `originator` field that periphery contracts populate via transient storage (similar to how the router already stores payer context via `_setNextCallbackContext`), and pass it as a distinct argument to extension hooks so guards can always key on the economic actor. Alternatively, the router can include the original `msg.sender` inside `extensionData` under a coordinated convention that the extension decodes and verifies.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin allowlists alice: setAllowedToSwap(pool, alice, true)
  - Pool admin allowlists router: setAllowedToSwap(pool, router, true)
    (required so alice can use the router)

Attack:
  - bob (not allowlisted) calls:
      router.exactInputSingle({pool: pool, ..., recipient: bob, ...})
  - Router calls pool.swap(bob, ...) → msg.sender in pool = router
  - Pool calls _beforeSwap(router, bob, ...)
  - Extension checks allowedSwapper[pool][router] → true ✓
  - Swap executes for bob despite bob not being allowlisted

Result:
  - bob swaps successfully on a pool that should have blocked him
  - SwapAllowlistExtension invariant broken
```
