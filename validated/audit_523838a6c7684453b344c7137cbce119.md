### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double-Claiming of Bonus Rewards - (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgClaimTierRewards` accepts a caller-supplied list of position IDs. The handler loads all positions into an in-memory slice **before** any rewards are processed. Because there is no deduplication of `msg.PositionIds`, supplying the same position ID twice causes `processEventsAndClaimBonus` to re-process the same validator events and re-pay the same bonus from the `RewardsPoolName` module account a second time within a single transaction.

---

### Finding Description

In `ClaimTierRewards` the handler first builds a `positions` slice by loading each requested position from state:

```go
// x/tieredrewards/keeper/msg_server.go  lines 434-446
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
``` [1](#0-0) 

It then passes the entire slice to `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go line 448
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [2](#0-1) 

Inside `claimRewardsAndUpdatesPositions`, each element of the slice is processed independently using its own in-memory copy of the position state:

```go
// x/tieredrewards/keeper/claim_rewards.go  lines 110-132
for i := range positions {
    pos := &positions[i]
    ...
    base, err := k.claimBaseRewards(ctx, *pos)
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [3](#0-2) 

If position ID `N` appears twice in `msg.PositionIds`, both `positions[0]` and `positions[1]` are loaded from state **before** any claim occurs and therefore carry identical stale checkpoints (`LastEventSeq = S0`, `LastBonusAccrual = T0`).

When `positions[0]` is processed:
- `processEventsAndClaimBonus` walks all validator events since `S0`, computes bonus `B`, transfers `B` from `RewardsPoolName` to the owner, and advances the in-memory checkpoints of `positions[0]` to `(S_current, T_current)`.
- `setPosition` persists the updated position to state.

When `positions[1]` is processed next:
- Its in-memory checkpoints are still `(S0, T0)` — the stale pre-claim values — because it is a **separate copy** in the slice.
- `processEventsAndClaimBonus` re-walks the same events from `S0`, re-computes the same bonus `B`, and transfers `B` from `RewardsPoolName` to the owner **a second time**.
- `decrementEventRefCount` is also called a second time for each event, corrupting the event reference-count bookkeeping. [4](#0-3) 

The `sufficientBonusPoolBalance` guard only checks whether the pool has enough balance for the current iteration's bonus; it does not prevent the same bonus from being paid twice across two iterations. [5](#0-4) 

Base rewards (`claimBaseRewards` → `WithdrawDelegationRewards`) are **not** double-claimable this way because the distribution module returns zero on the second call within the same block. The vulnerability is specific to the **bonus reward** path.

---

### Impact Explanation

A position owner can drain the `RewardsPoolName` module account by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated `k` times. Each repetition re-pays the full accrued bonus for that position, multiplying the actual payout by `k`. The `RewardsPoolName` pool holds the protocol's bonus reward reserve; draining it prevents all other participants from ever claiming their legitimate bonus rewards.

Corrupted value: the `RewardsPoolName` module account balance (bond denom) and the per-event reference counts stored in the validator event index.

---

### Likelihood Explanation

The entry path is a standard, unprivileged Cosmos SDK transaction (`MsgClaimTierRewards`) signed by any position owner. No special role, leaked key, or social engineering is required. The attacker only needs to own at least one delegated tier position with accrued bonus rewards. The attack is repeatable across blocks.

---

### Recommendation

Deduplicate `msg.PositionIds` before building the `positions` slice, or enforce uniqueness in `MsgClaimTierRewards.Validate()`. A simple approach:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, dup := seen[id]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest,
            "duplicate position id %d in ClaimTierRewards", id)
    }
    seen[id] = struct{}{}
}
```

Alternatively, re-load each position from state inside `claimRewardsAndUpdatesPositions` (after the previous position has been persisted) rather than operating on a pre-loaded snapshot slice.

---

### Proof of Concept

1. Alice owns position ID `42` with accrued bonus `B` (e.g., 1000 basecro).
2. Alice submits `MsgClaimTierRewards{Owner: alice, PositionIds: [42, 42, 42, 42, 42]}`.
3. The handler loads five copies of position 42 into `positions[0..4]`, all with `LastEventSeq = S0` and `LastBonusAccrual = T0`.
4. `claimRewardsAndUpdatesPositions` iterates five times; each iteration finds the same stale checkpoints and pays `B` from `RewardsPoolName`.
5. Alice receives `5 × B = 5000 basecro` instead of `B = 1000 basecro`.
6. The `RewardsPoolName` balance is reduced by `5B`; all other position owners' future bonus claims are underfunded. [6](#0-5) [7](#0-6)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
