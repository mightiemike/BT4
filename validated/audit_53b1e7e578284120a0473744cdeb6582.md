### Title
`ValidResourceBounds` type collapses from `AllResources` to `L1Gas` on protobuf round-trip, causing valid transactions to be rejected by peers — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a zero-value heuristic to decide whether to reconstruct `L1Gas` or `AllResources`. A valid `RpcInvokeTransactionV3` carrying `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the gateway and assigned a hash computed under the `AllResources` branch. When the same transaction is serialized to protobuf and received by any peer (mempool P2P or consensus P2P), the deserializer silently reconstructs `ValidResourceBounds::L1Gas` instead of `AllResources`. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion then hard-fails with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, permanently dropping the transaction. Even if that conversion were bypassed, the `get_tip_resource_bounds_hash` function produces a structurally different hash for `L1Gas` vs `AllResources` (the `l1_data_gas` field is omitted from the hash for `L1Gas`), so the stored hash would not match any recomputed hash on the receiving side.

---

### Finding Description

**Root cause — protobuf deserializer (`crates/apollo_protobuf/src/converters/transaction.rs:431-435`):**

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

The deserializer uses the *values* of `l2_gas` and `l1_data_gas` to infer the *type* of the original `ValidResourceBounds`. This is a lossy heuristic: `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` and `L1Gas(X)` are semantically distinct (they produce different transaction hashes), but the heuristic maps both to `L1Gas` after a round-trip.

**Serialization path — `RpcInvokeTransactionV3` → protobuf (`crates/apollo_protobuf/src/converters/rpc_transaction.rs:134-143`):**

```rust
impl From<RpcInvokeTransactionV3> for protobuf::InvokeV3WithProof {
    fn from(mut value: RpcInvokeTransactionV3) -> Self {
        let snapi_invoke: InvokeTransactionV3 = value.into();
        Self { invoke: Some(snapi_invoke.into()), proof }
    }
}
```

`RpcInvokeTransactionV3.resource_bounds` is typed `AllResourceBounds` (always `AllResources`). The conversion to `InvokeTransactionV3` wraps it as `ValidResourceBounds::AllResources(...)`. The protobuf serializer faithfully encodes `l2_gas = 0` and `l1_data_gas = 0`.

**Deserialization path — protobuf → `RpcInvokeTransactionV3` (`crates/apollo_protobuf/src/converters/rpc_transaction.rs:115-131`):**

```rust
let snapi_invoke: InvokeTransactionV3 = value.invoke...try_into()?;
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
```

`snapi_invoke.resource_bounds` is now `ValidResourceBounds::L1Gas` (from the heuristic). `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` (`crates/starknet_api/src/rpc_transaction.rs:591-597`) rejects any non-`AllResources` variant:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => return Err(StarknetApiError::OutOfRange { ... }),
},
```

The conversion fails and the transaction is dropped.

**Hash domain divergence — `get_tip_resource_bounds_hash` (`crates/starknet_api/src/transaction_hash.rs:203-208`):**

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // l1_data_gas omitted
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

`AllResources` hashes `[tip, l1_gas, l2_gas=0, l1_data_gas=0]`; `L1Gas` hashes `[tip, l1_gas, l2_gas=0]`. The two produce different Poseidon digests, so the `tx_hash` stored in `InternalRpcTransaction` at the gateway is irreconcilable with any hash a peer would compute after deserialization.

---

### Impact Explanation

A user submits a valid `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it, computes a hash under `AllResources`, and stores it in the mempool. When the transaction is broadcast to peers via mempool P2P (`RpcTransactionBatch`) or included in a consensus block proposal, every receiving node fails to deserialize it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction can never be sequenced. This matches the **High** impact scope: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

Any user who submits an invoke V3 transaction with `l2_gas.max_amount = 0` and `l1_data_gas.max_amount = 0` (e.g., a legacy-style V3 transaction that sets only L1 gas bounds but uses the newer `AllResourceBounds` wire format) triggers this path. No privileged access is required. The gateway performs no check that `l2_gas` or `l1_data_gas` must be non-zero for `AllResourceBounds` transactions.

---

### Recommendation

The protobuf deserializer must not infer the `ValidResourceBounds` variant from field values. The protobuf schema should carry an explicit discriminant (e.g., a `bounds_type` enum field or a `oneof`) so that `AllResources` with zero sub-fields round-trips faithfully. Until the schema is updated, the deserializer should default to `AllResources` whenever all three resource fields are present in the wire message, reserving `L1Gas` only for messages that explicitly omit `l2_gas` and `l1_data_gas` (i.e., `None`, not zero-valued):

```rust
// Distinguish "field absent" (legacy L1Gas) from "field present but zero" (AllResources with zeros)
Ok(if value.l1_data_gas.is_none() && value.l2_gas_is_absent {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
