### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Enabling 64-Byte Internal Merkle Node Forgery — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` is marked `#[deprecated]` but is still a live, unpermissioned NEAR contract method. Any caller can supply an internal Merkle tree node hash as `tx_id` and receive `true`, falsely proving inclusion of a fabricated transaction.

---

### Finding Description

`#[deprecated]` in Rust is a **compile-time lint only** — it does not remove or restrict the function at runtime. The method remains fully exported as a NEAR contract call, gated only by `#[pause]` (which is inactive by default). [1](#0-0) 

The function's own doc comment explicitly acknowledges the vulnerability: [2](#0-1) 

The core check is a bare Merkle root comparison with no leaf-node validation: [3](#0-2) 

`compute_root_from_merkle_proof` accepts any 32-byte value as `transaction_hash` and walks up the tree without distinguishing leaves from internal nodes: [4](#0-3) 

`verify_transaction_inclusion_v2` closes this gap by first verifying a coinbase proof (which anchors the tree depth), then delegating to the deprecated function: [5](#0-4) 

But because `verify_transaction_inclusion` is still independently callable, the coinbase guard in `v2` is entirely bypassable.

There is no `#[private]`, no `trusted_relayer`, and no role-based access control on `verify_transaction_inclusion`: [6](#0-5) 

---

### Impact Explanation

An attacker can call `verify_transaction_inclusion` directly with:
- `tx_id` = an internal Merkle node hash (e.g., `hash(tx[0] || tx[1])`) from a known canonical block
- `tx_index` = the position of that node at its level in the tree
- `merkle_proof` = the path from that internal node up to the root (one element shorter than a leaf proof)
- `tx_block_blockhash` = any confirmed canonical block hash

`compute_root_from_merkle_proof` will correctly recompute the Merkle root from the internal node, and the function returns `true` — falsely asserting that a fabricated, non-existent transaction is included in the block.

Any downstream system (bridge, oracle, settlement layer) that calls `verify_transaction_inclusion` to gate asset releases or state transitions can be deceived into accepting a proof for a transaction that never existed.

---

### Likelihood Explanation

- The attack requires no privileges, no leaked keys, and no social engineering.
- The attacker only needs knowledge of any canonical block's transaction set (public on-chain data).
- The internal node hash and a valid shortened proof can be computed entirely off-chain.
- The contract is not paused by default; the method is callable immediately after deployment.

---

### Recommendation

Remove `verify_transaction_inclusion` from the public contract interface entirely, or add `#[private]` to prevent external calls. The `#[deprecated]` attribute provides zero runtime protection. All external callers must be migrated to `verify_transaction_inclusion_v2`.

---

### Proof of Concept

```
Given a canonical block B with transactions [tx0, tx1, tx2, tx3]:
  Level 0 (leaves): [H(tx0), H(tx1), H(tx2), H(tx3)]
  Level 1 (internal): [H(H(tx0)||H(tx1)), H(H(tx2)||H(tx3))]
  Root: H(level1[0] || level1[1])

internal_node = H(H(tx0) || H(tx1))   // 32-byte hash, position 0 at level 1

Call verify_transaction_inclusion({
  tx_id:              internal_node,
  tx_block_blockhash: B.hash,
  tx_index:           0,               // position 0 at level 1
  merkle_proof:       [level1[1]],     // only one sibling needed (not two)
  confirmations:      0,
})
// Returns: true
// Actual result: no transaction with id=internal_node exists
```

#[must_use]
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
