### Title
Unvalidated IBC Packet `tokenIds` Bypass Format Constraints, Minting Permanently Locked NFTs — (`x/nft-transfer/types/packet.go`, `x/nft-transfer/keeper/packet.go`)

---

### Summary

`NonFungibleTokenPacketData.ValidateBasic()` only checks that each `tokenId` is non-blank. It does not enforce the format constraints (`[a-z0-9]+`, length 3–64, must begin with `[a-z]`) that `ValidateTokenID` enforces on every local `MsgMintNFT` transaction. A counterparty chain can craft an IBC-721 packet with malformed `tokenIds` (e.g., containing uppercase letters, `/`, control characters, or lengths outside `[3, 64]`). `processReceivedPacket` passes these directly to `nftKeeper.MintNFT` → `MintNFTUnverified`, which stores them without any format check. The resulting NFTs are permanently locked: every subsequent operation (`MsgTransferNFT`, `MsgBurnNFT`, `MsgEditNFT`, outbound IBC re-transfer) calls `ValidateTokenID` and rejects the malformed ID.

---

### Finding Description

`NonFungibleTokenPacketData.ValidateBasic()` validates `tokenIds` only for emptiness:

```go
for _, tokenID := range nftpd.TokenIds {
    if strings.TrimSpace(tokenID) == "" {
        return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
    }
}
``` [1](#0-0) 

No character-set, length, or prefix check is performed. The local `ValidateTokenID` enforces `[a-z0-9]+`, length `[3, 64]`, and a leading `[a-z]`:

```go
func ValidateTokenID(tokenID string) error {
    if len(tokenID) < MinDenomLen || len(tokenID) > MaxDenomLen { ... }
    if !IsBeginWithAlpha(tokenID) || !IsAlphaNumeric(tokenID) { ... }
    return nil
}
``` [2](#0-1) 

`OnRecvPacket` calls `data.ValidateBasic()` and then immediately calls `processReceivedPacket`: [3](#0-2) 

Inside `processReceivedPacket`, the unvalidated `tokenID` values from the packet are passed directly to `nftKeeper.MintNFT`:

```go
for i, tokenID := range data.TokenIds {
    if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
        return err
    }
}
``` [4](#0-3) 

`MintNFT` calls `MintNFTUnverified`, which stores the NFT directly with no tokenID format check: [5](#0-4) 

A secondary issue exists in `ClassTrace.Validate()`, which explicitly skips `BaseClassId` validation (`// NOTE: no base class validation`), and `processReceivedPacket` never calls `classTrace.Validate()` before storing the trace: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

NFTs minted with malformed `tokenIds` (e.g., `"ABC"`, `"ab/cd"`, `"x"`, a 200-character string) are permanently locked on the receiving chain. Every operation that would allow the receiver to move or destroy them calls `ValidateTokenID`:

- `MsgTransferNFT.ValidateBasic()` → `ValidateTokenID` [8](#0-7) 
- `MsgBurnNFT.ValidateBasic()` → `ValidateTokenID` [9](#0-8) 
- `MsgEditNFT.ValidateBasic()` → `ValidateTokenID` [10](#0-9) 
- Outbound IBC re-transfer via `MsgTransfer.ValidateBasic()` checks tokenIDs for blank only, but `createOutgoingPacket` calls `nftKeeper.GetNFT` which would succeed — however the ante handler `ValidateMsgTransferDecorator` enforces `MaxTokenIDLength = 2048` [11](#0-10)  — a tokenID of length > 2048 would be rejected at CheckTx, but not at DeliverTx (the decorator skips non-CheckTx). More critically, a tokenID with `/` or uppercase characters cannot be referenced by any local transaction at all.

The corrupted value is: **NFT ownership record** — `nft.Owner` is set to the receiver, `k.setOwner` records the mapping, and `k.increaseSupply` increments the denom supply counter. All three are permanently inconsistent with the chain's own validation rules.

---

### Likelihood Explanation

The entry path requires a counterparty IBC-721 chain to send a crafted packet. This is reachable by:
1. Any chain that has an open IBC-721 channel with this chain and is itself compromised or malicious.
2. A relayer that substitutes packet data (if the IBC light-client proof is not enforced — but IBC does enforce packet commitment proofs, so the counterparty chain must actually commit the malformed packet).

The realistic attacker is a malicious or compromised counterparty chain. Given that ICS-721 is a cross-chain protocol, this is a standard threat model for IBC applications. The `ValidateMsgTransferDecorator` ante handler only runs at CheckTx and only checks length, not character set, so it provides no protection at the consensus layer. [12](#0-11) 

---

### Recommendation

In `NonFungibleTokenPacketData.ValidateBasic()`, add format validation for each `tokenId` that mirrors `ValidateTokenID` (or a relaxed IBC-compatible variant that at minimum rejects control characters and `/`). Similarly, validate `ClassId` against the ICS-721 class identifier rules. Call `classTrace.Validate()` after `ParseClassTrace` in `processReceivedPacket` before storing the trace.

---

### Proof of Concept

1. On a counterparty chain, mint an NFT with a valid local ID (e.g., `"validtoken"`).
2. Craft an IBC-721 `NonFungibleTokenPacketData` packet with `token_ids: ["INVALID/TOKEN\x00"]` and a valid `class_id`, `sender`, `receiver`, and matching `token_uris`.
3. Submit the packet commitment on the counterparty chain and relay it to the Cronos POS Chain.
4. `OnRecvPacket` → `OnRecvPacket (keeper)` → `data.ValidateBasic()` passes (non-blank).
5. `processReceivedPacket` → `nftKeeper.MintNFT(ctx, "ibc/{hash}", "INVALID/TOKEN\x00", ...)` succeeds.
6. The receiver now holds an NFT with ID `"INVALID/TOKEN\x00"`.
7. Attempt `MsgTransferNFT` with `Id: "INVALID/TOKEN\x00"` → `ValidateTokenID` rejects it.
8. Attempt `MsgBurnNFT` → same rejection.
9. The NFT is permanently locked; the denom supply counter is permanently inflated by 1. [13](#0-12) [14](#0-13) [5](#0-4)

func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil

func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err

// MintNFTUnverified mints an NFT without verifying if the owner is the creator of denom
// Needed during genesis initialization
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil

const (
	// MaxClassIDLength values chosen arbitrarily
	MaxClassIDLength      = 2048
	MaxTokenIds           = 256
	MaxTokenIDLength      = 2048
	MaximumReceiverLength = 2048
)

// ValidateMsgTransferDecorator is a temporary decorator that limit the field length of MsgTransfer message.
type ValidateMsgTransferDecorator struct{}

func NewValidateMsgTransferDecorator() ValidateMsgTransferDecorator {
	return ValidateMsgTransferDecorator{}
}

func (vtd ValidateMsgTransferDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (sdk.Context, error) {
	// avoid breaking consensus
	if !ctx.IsCheckTx() {
		return next(ctx, tx, simulate)
	}

	msgs := tx.GetMsgs()
	for _, msg := range msgs {
		transfer, ok := msg.(*nfttypes.MsgTransfer)
		if !ok {
			continue
		}

		if len(transfer.ClassId) > MaxClassIDLength {
			return ctx, newsdkerrors.Wrapf(sdkerrors.ErrInvalidRequest, "class id length must be less than %d", MaxClassIDLength)
		}

		if len(transfer.TokenIds) > MaxTokenIds {
			return ctx, newsdkerrors.Wrapf(sdkerrors.ErrInvalidRequest, "token id length must be less than %d", MaxTokenIds)
		}

		for _, tokenID := range transfer.TokenIds {
			if len(tokenID) > MaxTokenIDLength {
				return ctx, newsdkerrors.Wrapf(sdkerrors.ErrInvalidRequest, "token id length must be less than %d", MaxTokenIDLength)
			}
		}

		if len(transfer.Receiver) > MaximumReceiverLength {
			return ctx, newsdkerrors.Wrapf(sdkerrors.ErrInvalidRequest, "receiver length must be less than %d", MaximumReceiverLength)
		}
	}
