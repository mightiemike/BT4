### Title
NFT Permanently Locked When Minted to Module Account — No Recovery Path - (`x/nft/keeper/keeper.go`, `x/nft/types/msgs.go`)

### Summary

`MsgMintNFT.ValidateBasic()` only validates that `Recipient` is a syntactically valid bech32 address. Neither the message server nor the keeper checks whether the recipient is a blocked module account. Once an NFT is minted to a module account address (e.g., `distribution`, `staking`, `gov`), it is permanently locked: no `MsgTransferNFT` or `MsgBurnNFT` can recover it because both require the sender to be the current owner, and module accounts cannot sign transactions.

---

### Finding Description

**Step 1 — No blocked-address guard in `ValidateBasic`**

`MsgMintNFT.ValidateBasic()` accepts any bech32-valid address as `Recipient`: [1](#0-0) 

There is no call to `BlockedAddr`, `IsModuleAccount`, or any equivalent check.

**Step 2 — Keeper `MintNFT` only checks denom creator, not recipient**

`Keeper.MintNFT` verifies the sender is the denom creator, then delegates unconditionally to `MintNFTUnverified`: [2](#0-1) 

**Step 3 — `MintNFTUnverified` writes module account as owner**

`MintNFTUnverified` calls `setOwner` with the caller-supplied `owner` address, which may be a module account: [3](#0-2) 

`setOwner` writes directly to the KV store with no address-type validation: [4](#0-3) 

**Step 4 — No recovery path exists**

`BurnNFT` requires the caller to pass `IsOwner` before it can proceed: [5](#0-4) 

`TransferOwner` likewise requires `IsOwner(srcOwner)`: [6](#0-5) 

Because module accounts cannot sign transactions, neither `MsgBurnNFT` nor `MsgTransferNFT` can ever be submitted with `sender = moduleAccAddr`. The NFT is irrecoverably locked.

---

### Impact Explanation

Any denom creator — an unprivileged user — can mint an NFT directly to a module account address. The NFT's ownership record is permanently set to that address in the KV store, and no on-chain message can change it. If the NFT carries economic value (marketplace listing, game asset, IBC-bridged token), it is destroyed with no recourse. The `BankKeeper` interface exposed in `expected_keepers.go` does not include `BlockedAddr`, confirming the module has no mechanism to detect or reject module account recipients. [7](#0-6) 

---

### Likelihood Explanation

The attack requires only that the sender be the denom creator (any user who previously called `MsgIssueDenom`). The recipient address is fully attacker-controlled. The path is reachable via a standard signed Cosmos SDK transaction with no special permissions, governance approval, or operator involvement.

---

### Recommendation

1. Extend `BankKeeper` in `expected_keepers.go` to include `BlockedAddr(addr sdk.AccAddress) bool`.
2. In `msgServer.MintNFT` (or `Keeper.MintNFT`), reject the transaction if `bankKeeper.BlockedAddr(recipient)` returns `true`.
3. Apply the same guard to `TransferOwner` to prevent post-mint transfers to module accounts.

---

### Proof of Concept

```go
// keeper_test.go (illustrative)
func TestMintNFTToModuleAccount(t *testing.T) {
    ctx, keeper := setupKeeper(t)

    creator := sdk.AccAddress([]byte("creator"))
    distModuleAddr := authtypes.NewModuleAddress(distrtypes.ModuleName)

    // Issue denom
    _ = keeper.IssueDenom(ctx, "testdenom", "Test", "", "", creator)

    // Mint NFT to distribution module account — succeeds with no error
    err := keeper.MintNFT(ctx, "testdenom", "token1", "Token1", "", "", creator, distModuleAddr)
    require.NoError(t, err) // passes — no blocked-address check

    // Confirm ownership
    nft, err := keeper.GetNFT(ctx, "testdenom", "token1")
    require.NoError(t, err)
    require.Equal(t, distModuleAddr.String(), nft.GetOwner().String())

    // Attempt burn as denom creator — fails: creator is not the owner
    err = keeper.BurnNFT(ctx, "testdenom", "token1", creator)
    require.Error(t, err) // ErrUnauthorized: creator is not owner

    // Attempt transfer from module account — impossible: no signer
    // NFT is permanently locked.
}
```

func (msg MsgMintNFT) ValidateBasic() error {
	if _, err := sdk.AccAddressFromBech32(msg.Sender); err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid sender address (%s)", err)
	}
	if _, err := sdk.AccAddressFromBech32(msg.Recipient); err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid receipt address (%s)", err)
	}
	if err := ValidateDenomID(msg.DenomId); err != nil {
		return err
	}
	if err := ValidateTokenURI(msg.URI); err != nil {
		return err
	}
	return ValidateTokenID(msg.Id)
}

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
