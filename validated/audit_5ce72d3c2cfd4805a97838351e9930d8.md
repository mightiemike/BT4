### Title
`feeX18` Excluded from EIP-712 Digest Allows Sequencer to Charge Arbitrary Withdrawal Fee — (`core/contracts/Verifier.sol`, `core/contracts/EndpointTx.sol`)

---

### Summary

`SignedWithdrawCollateralV2.feeX18` is not included in the EIP-712 type string or digest. The sequencer constructs the submitted transaction bytes and can freely set `feeX18` to any value up to `currentFeeX18`. The user's compact signature covers only the `WithdrawCollateralV2` inner struct fields, so the signature remains valid regardless of what `feeX18` is set to. The protocol then charges the user exactly `signedTx.feeX18`, which the user never authorized.

---

### Finding Description

**Struct layout:** `SignedWithdrawCollateralV2` places `feeX18` as a field *outside* the signed `tx` struct:

```solidity
struct SignedWithdrawCollateralV2 {
    WithdrawCollateralV2 tx;   // signed
    CompactSignature signature;
    int128 feeX18;             // NOT signed
}
``` [1](#0-0) 

**EIP-712 type string** for `WithdrawCollateralV2` contains only the inner struct fields — `feeX18` is absent:

```
"WithdrawCollateralV2(bytes32 sender,uint32 productId,uint128 amount,uint64 nonce,address sendTo,uint128 appendix)"
``` [2](#0-1) 

**Digest computation** in `Verifier.computeDigest` encodes only those six fields, confirming `feeX18` is never hashed: [3](#0-2) 

**Execution path** in `EndpointTx.processTransactionImpl`:
1. `validateSignedTx` verifies the compact signature against the digest — passes regardless of `feeX18`.
2. `require(signedTx.feeX18 <= currentFeeX18)` only caps the fee at the protocol maximum; it does not enforce what the user intended.
3. `chargeFee(signedTx.tx.sender, signedTx.feeX18, ...)` deducts the sequencer-supplied value from the user's collateral balance. [4](#0-3) 

---

### Impact Explanation

A user signs a `WithdrawCollateralV2` expecting zero or minimal fees. The sequencer submits the same valid signature with `feeX18 = currentFeeX18`. The signature check passes, and the user is charged up to `currentFeeX18` more quote than they authorized. The user receives `amount` tokens but loses `amount + currentFeeX18` from their collateral balance — a direct, unilateral asset extraction with no on-chain recourse.

---

### Likelihood Explanation

The sequencer is the sole entity that calls `processTransactionImpl` and constructs the raw transaction bytes. No additional compromise is required; the sequencer simply sets the `feeX18` field in the ABI-encoded payload before submission. The scope explicitly lists "sequencer" as a relevant attack vector for Critical findings, and the broken invariant (signature must commit to the exact fee) is directly reachable through the normal withdrawal flow.

---

### Recommendation

Include `feeX18` in the EIP-712 type string and digest computation:

```
"WithdrawCollateralV2(bytes32 sender,uint32 productId,uint128 amount,uint64 nonce,address sendTo,uint128 appendix,int128 feeX18)"
```

And in `Verifier.computeDigest`, add `signedTx.feeX18` as the final argument to `abi.encode`. This binds the user's signature to the exact fee that will be charged, preventing the sequencer from substituting a higher value post-signing. [5](#0-4) 

---

### Proof of Concept

1. Deploy the protocol with `currentFeeX18 = 1e18` (1 unit of quote).
2. Victim signs `WithdrawCollateralV2{sender, productId=0, amount=100e18, nonce=0, sendTo=address(0), appendix=0}` — no `feeX18` field exists in what they sign.
3. Sequencer encodes `SignedWithdrawCollateralV2{tx: <victim's struct>, signature: <victim's compact sig>, feeX18: 1e18}` and calls `processTransactionImpl`.
4. `validateSignedTx` recovers the victim's address from the digest (which does not include `feeX18`) — **passes**.
5. `require(1e18 <= 1e18)` — **passes**.
6. `chargeFee(victim, 1e18, 0)` deducts 1e18 quote from victim's spot balance.
7. `clearinghouse.withdrawCollateral(victim, 0, 100e18, ...)` sends 100e18 tokens out.
8. Assert: victim's spot balance decreased by `100e18 + 1e18`, not `100e18`. The extra `1e18` was never authorized by the victim's signature.
