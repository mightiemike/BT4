Audit Report

## Title
SwapAllowlistExtension Gates on `sender` (Router) Instead of `recipient` (Economic Actor), Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument — the direct `msg.sender` of `pool.swap()` — against the per-pool allowlist. Because `MetricOmmSimpleRouter` is the `msg.sender` when it calls `pool.swap`, allowlisting the router (a necessary step for any permitted user to route) grants every unpermitted user the ability to bypass the allowlist entirely by routing through that same router.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that value verbatim as the first positional argument of the `beforeSwap` call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `sender` is the direct caller of `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` — the router is `msg.sender`, so `sender` in the extension is the router's address, not the end user: [4](#0-3) 

The same pattern holds for `exactInput`, `exactOutputSingle`, and `exactOutput`. For any permitted user to route through `MetricOmmSimpleRouter`, the admin must allowlist the router. Once the router is allowlisted, `allowedSwapper[pool][router] == true` and every user who calls the router passes the check, regardless of individual permission.

`DepositAllowlistExtension` correctly gates on `owner` (the economic actor whose position is credited), not on `sender` (the intermediary), confirming the swap check is bound to the wrong actor: [5](#0-4) 

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to specific counterparties loses that restriction entirely the moment the pool admin allowlists the shared periphery router. Every unpermitted address can execute swaps against the pool at oracle-derived prices, constituting direct unauthorized extraction of value from LP positions and a broken core pool invariant (the allowlist restriction). This maps to the allowed impact category of broken core pool functionality causing loss of funds and admin-boundary break via an unprivileged path.

## Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard periphery entry point for multi-step swaps. Any pool admin who wants even one permitted user to route multi-hop swaps must allowlist the router. The extension's NatSpec states it "Gates `swap` by swapper address" without clarifying that the swapper is the intermediary, not the end user, so the admin has no on-chain signal that this action opens the gate for all users. The trigger is a routine, well-intentioned admin action with no visible warning. [6](#0-5) 

## Recommendation

Replace the `sender` check with a check on `recipient` — the address that receives the output tokens and is the true economic beneficiary of the swap:

```solidity
function beforeSwap(
    address,          // sender — ignore
    address recipient,
    bool, int128, uint128, uint256, uint128, uint128,
    bytes calldata
) external view override returns (bytes4) {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][recipient]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

Also align the NatSpec and admin documentation to make explicit which actor is being gated.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured as `beforeSwap`.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is permitted.
3. Admin calls `setAllowedToSwap(pool, router, true)` — allowlists `MetricOmmSimpleRouter` so Alice can route.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. The router calls `pool.swap(recipient = bob, ...)` with `msg.sender = router`.
6. `beforeSwap` receives `sender = router`; `allowedSwapper[pool][router] == true` → check passes.
7. Bob's swap executes at oracle prices against LP liquidity — allowlist fully bypassed. [7](#0-6)
