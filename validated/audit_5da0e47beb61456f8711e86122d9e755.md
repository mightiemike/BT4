### Title
Protobuf `ResourceBounds` deserialization silently downgrades `AllResources` to `L1Gas`, causing valid transactions to be rejected during consensus propagation — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter silently reclassifies an `AllResources` transaction as `L1Gas` whenever `l2_gas` and `l1_data_gas` are both zero. The gateway accepts such transactions as valid, but the receiving validator's protobuf deserialization path then fails with `DEPRECATED_RESOURCE_BOUNDS_ERROR` when it tries to reconstruct the `RpcInvokeTransactionV3`, causing the entire consensus proposal to be rejected.

### Finding Description

**Conversion boundary — serialization side**

`From<ValidResourceBounds> for protobuf::ResourceBounds` faithfully serializes `AllResources` with all three fields present, even when `l2_gas` and `l1_data_gas` are zero: [1](#0-0) 

**Conversion boundary — deserialization side**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a value-based heuristic: if both `l2_gas` and `l1_data_gas` are zero it returns `ValidResourceBounds::L1Gas`, discarding the `AllResources` variant: [2](#0-1) 

**Hash preimage divergence**

`get_tip_resource_bounds_hash` produces a different hash preimage for `L1Gas` (2 resource entries) versus `AllResources` (3 resource entries, including `L1_DATA_GAS`): [3](#0-2) 

The proposer computes the transaction hash with `AllResources` (3 entries). After the protobuf round-trip the validator holds `L1Gas` (2 entries), so any hash recomputation would also diverge.

**Downstream rejection**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()` to convert `InvokeTransactionV3 → RpcInvokeTransactionV3`. That conversion (same pattern as `TryFrom<DeployAccountTransactionV3> for RpcDeployAccountTransactionV3`) rejects anything that is not `AllResources`: [4](#0-3) [5](#0-4) 

The comment on line 129 of `rpc_transaction.rs` confirms: *"This conversion can fail only if the resource_bounds are not AllResources."*

**Proposal-level rejection**

When the validator's `validate_proposal` loop calls `convert_consensus_tx_to_internal_consensus_tx` and the deserialization of the `ConsensusTransaction` fails, the entire proposal is marked `Failed`: [6](#0-5) 

**Gateway accepts the triggering transaction**

The gateway's stateless validator explicitly marks `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }` as valid: [7](#0-6) 

### Impact Explanation

A transaction accepted by the gateway and placed in the mempool can be included in a block proposal by the proposer. Every validator that receives the proposal will fail to deserialize the transaction from protobuf, causing the proposal to be rejected. An attacker can deliberately craft such transactions to repeatedly invalidate proposals, stalling consensus.

**Impact class**: High — RPC/gateway admission accepts a transaction that is then rejected during consensus propagation, and the conversion error causes the entire proposal to be marked `Failed`.

### Likelihood Explanation

The trigger requires only a single valid RPC call: submit an `InvokeV3` with `AllResourceBounds` where `l2_gas = 0` and `l1_data_gas = 0`. No privileged access, no malformed bytes, and no special network position are needed. The gateway's own test suite confirms this input is accepted.

### Recommendation

1. **Fix the deserialization heuristic**: In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, always return `AllResources` when all three fields are present in the protobuf message, regardless of whether `l2_gas` and `l1_data_gas` are zero. Reserve the `L1Gas` variant only for messages that genuinely omit `l1_data_gas` (the pre-0.13.2 wire format).

2. **Alternatively, add gateway rejection**: Reject `AllResourceBounds` where both `l2_gas` and `l1_data_gas` are zero at the gateway, forcing callers to use the canonical `L1Gas` variant for pre-0.13.3 transactions.

### Proof of Concept

```
1. POST /gateway/add_transaction with:
   type: INVOKE
   version: 0x3
   resource_bounds: {
     l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
     l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
     l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 }
   }
   ... (valid signature, nonce, etc.)

2. Gateway accepts → transaction enters mempool with hash H computed
   via AllResources (3-entry preimage).

3. Proposer includes the transaction in a block proposal and serializes
   it to protobuf::InvokeV3WithProof.
   Wire bytes: l2_gas = 0, l1_data_gas = 0.

4. Validator deserializes:
   TryFrom<protobuf::ResourceBounds>:
     l1_data_gas.is_zero() && l2_gas.is_zero()  →  ValidResourceBounds::L1Gas(l1_gas)

5. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3:
     resource_bounds is L1Gas, not AllResources  →  DEPRECATED_RESOURCE_BOUNDS_ERROR

6. TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction returns Err.

7. validate_proposal returns HandledProposalPart::Failed(…).
   Proposal is rejected; consensus round fails.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
