Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Any User to Bypass Curated Pool Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, so when a swap is routed through `MetricOmmSimpleRouter`, the `sender` received by `SwapAllowlistExtension.beforeSwap` is the router's address, not the end user's. If the router is allowlisted (a necessary operational step for allowlisted users to use the standard periphery), every user — including those the pool admin explicitly excluded — can bypass the allowlist by calling the public router.

## Finding Description

In `MetricOmmPool.swap`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // <-- this is the router when called via MetricOmmSimpleRouter
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L162-165
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, ...)  // sender == router address
)
```

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```

Here `msg.sender` is the pool (correct) and `sender` is the router (wrong — should be the end user). The check evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`.

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly with no mechanism to pass the originating user's address as `sender`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
    );
```

The pool's `swap` function has no parameter for an explicit sender override — it always uses `msg.sender` (the router). The same flaw applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

## Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` intends to restrict swapping to a specific set of addresses. To allow those allowlisted users to use the standard periphery, the admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the guard is completely bypassed for **all** users who route through `MetricOmmSimpleRouter` — including addresses the admin explicitly never allowlisted. Any non-allowlisted address can drain liquidity, front-run, or otherwise interact with a pool designed to be access-controlled. This is a direct policy bypass with fund-impacting consequences for LP positions in the curated pool.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the standard documented periphery entry point. Any user who encounters a `NotAllowedToSwap` revert on a direct pool call will naturally try the router as an alternative. No privileged access, special tokens, or malicious setup is required — only a public router call. The likelihood is high whenever a pool uses `SwapAllowlistExtension` and the router is allowlisted (which is required for the allowlist to be usable at all via periphery).

## Recommendation

The pool must gate the economically relevant actor, not the immediate `msg.sender`. Two approaches:

1. **Pass the real user through the call chain**: Add an explicit `sender` parameter to `IMetricOmmPoolActions.swap` that the router populates with `msg.sender` (the end user). The pool forwards this value — not its own `msg.sender` — into `_beforeSwap`.

2. **Extension-level router awareness**: The extension checks whether `sender` is a known trusted router and, if so, falls back to checking an authenticated caller address embedded in `extensionData` (signed or passed by the router from its own `msg.sender`).

The invariant that must hold: `allowedSwapper[pool][X]` must be evaluated where `X` is the address that controls and economically benefits from the swap, regardless of which periphery contract relays the call.

## Proof of Concept

1. Pool admin deploys a curated pool with `SwapAllowlistExtension` attached.
2. Admin calls `setAllowedToSwap(pool, router, true)` so allowlisted users can use the standard periphery.
3. Admin calls `setAllowedToSwap(pool, alice, true)` and intentionally does **not** allowlist `bob`.
4. `bob` calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the curated pool.
5. The router calls `pool.swap(...)`. At the pool level, `msg.sender` is the router.
6. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender /*=router*/, ...)`. [1](#0-0) 
7. `ExtensionCalling._beforeSwap` encodes `sender = router` and calls `extension.beforeSwap(router, ...)`. [2](#0-1) 
8. The extension evaluates `allowedSwapper[pool][router]` → `true` → no revert. [3](#0-2) 
9. `bob`'s swap executes successfully despite never being allowlisted.

Foundry test plan: deploy pool with `SwapAllowlistExtension`, allowlist the router but not `bob`, call `MetricOmmSimpleRouter.exactInputSingle` as `bob`, assert no revert occurs and swap settles — demonstrating the bypass.
