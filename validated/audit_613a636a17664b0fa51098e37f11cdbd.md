### Title
Quote Token Price Hardcoded to $1 Causes Solvency Accounting Corruption During Stablecoin Depeg — (`File: core/contracts/SpotEngine.sol`)

---

### Summary

The Nado protocol hardcodes the quote token (USDC) price to exactly `ONE` (1e18 = $1) in `SpotEngine.initialize()` and never updates it via oracle. All health calculations, liquidation eligibility checks, and insurance fund valuations treat USDC as perpetually worth $1. During a stablecoin depeg event, every subaccount's health is overstated, insolvent accounts evade liquidation, and the insurance fund's real purchasing power is less than the protocol assumes — directly mirroring the Salty.IO finding that "the collateralization ratio is calculated assuming that the price of USDS is $1."

---

### Finding Description

In `SpotEngine.initialize()`, the `QUOTE_PRODUCT_ID` risk parameters are set once and never touched again:

```solidity
_risk().value[QUOTE_PRODUCT_ID] = RiskHelper.RiskStore({
    longWeightInitial:       1e9,
    shortWeightInitial:      1e9,
    longWeightMaintenance:   1e9,
    shortWeightMaintenance:  1e9,
    priceX18:                ONE   // ← hardcoded $1, no oracle feed
});
``` [1](#0-0) 

`ONE` is `10**18`, representing exactly $1.00 in the protocol's fixed-point system. [2](#0-1) 

The `RiskHelper._getWeightX18` function uses `risk.priceX18` when computing each product's contribution to a subaccount's health. For the quote token, this is always `ONE`, so the health contribution of a USDC balance is always `quoteBalance.amount × 1 × 1 = quoteBalance.amount` — regardless of USDC's actual market price. [3](#0-2) 

The `isUnderMaintenance` check in `ClearinghouseLiq` gates every liquidation on this health value: [4](#0-3) 

The insurance fund (`int128 internal insurance`) is also denominated in USDC and its value is used at face value (1:1 with dollars) when covering bad debt: [5](#0-4) [6](#0-5) 

---

### Impact Explanation

If USDC depegs to price `P < 1`:

1. **Liquidation suppression**: A subaccount holding `Q` USDC and owing `L` in liabilities has on-chain health `Q - L`. If `Q = 100`, `L = 95`, health = `+5` (healthy). At `P = 0.87`, real health = `87 - 95 = -8` (insolvent). The protocol never triggers liquidation because `isUnderMaintenance` returns `false`.

2. **Insurance fund devaluation**: The `insurance` variable stores a nominal USDC amount. If USDC is worth $0.87, the fund covers only 87% of the bad debt it is assumed to cover. The socialization step in `_finalizeSubaccount` will under-cover losses, pushing more bad debt onto all open-interest holders.

3. **Liquidation price miscalculation**: `getLiqPriceX18` computes the liquidation price of non-quote assets in USDC terms. If USDC is depegged, the real USD value transferred is less than the protocol's accounting records, locking in losses at the moment of liquidation. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

USDC has depegged in production before (March 2023, briefly to ~$0.87). The protocol has no mechanism to update `priceX18` for `QUOTE_PRODUCT_ID` — the only place it is set is in `initialize()`, which runs once at deployment. Any future depeg event of any magnitude will trigger this condition. No privileged access or attacker action is required; the market event alone is sufficient.

---

### Recommendation

**Short term**: Add an oracle feed for the quote token and update `priceX18` for `QUOTE_PRODUCT_ID` in the same `updateStates` loop that refreshes all other product prices. Apply a conservative weight (e.g., `longWeightMaintenance < 1`) to the quote token to create a buffer against depeg risk.

**Long term**: Create a design specification that explicitly documents the assumption that the quote token is $1, identifies all code paths that depend on it, and defines circuit-breaker conditions (e.g., pause liquidations or deposits if an on-chain USDC/USD oracle deviates beyond a threshold).

---

### Proof of Concept

1. USDC depegs to $0.87 on-chain (as occurred in March 2023).
2. Alice holds 100 USDC (`quoteBalance.amount = 100e18`) and has a 95 USDC liability from a short spot position.
3. `getHealth(alice, MAINTENANCE)` computes: `100e18 × ONE × 1e9 - 95e18 × ONE × 1e9 = +5e18 > 0`.
4. `isUnderMaintenance(alice)` returns `false`; no liquidation is possible.
5. Alice's real collateral value is `100 × 0.87 = $87`, against a $95 liability — she is insolvent by $8.
6. The sequencer submits a `LiquidateSubaccount` transaction; `liquidateSubaccountImpl` reverts with `ERR_NOT_LIQUIDATABLE` at line 603.
7. Alice's insolvency accumulates. When USDC re-pegs, the socialization mechanism distributes the unrecovered bad debt across all open-interest holders. [9](#0-8) [1](#0-0)

    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }

    function getLiqPriceX18(uint32 productId, int128 amount)
        internal
        returns (int128, int128)
    {
        RiskHelper.Risk memory risk = IProductEngine(productToEngine[productId])
            .getRisk(productId);
        int128 penaltyX18 = (RiskHelper._getWeightX18(
            risk,
            amount,
            IProductEngine.HealthType.MAINTENANCE
        ) - ONE) / 5;
        if (penaltyX18.abs() < MIN_NON_SPREAD_LIQ_PENALTY_X18) {
            if (penaltyX18 < 0) {
                penaltyX18 = -MIN_NON_SPREAD_LIQ_PENALTY_X18;
            } else {
                penaltyX18 = MIN_NON_SPREAD_LIQ_PENALTY_X18;
            }
        }
        return (risk.priceX18.mul(ONE + penaltyX18), risk.priceX18);
    }
