### Title
Division-by-Zero in `socializeSubaccount` When `openInterest == 0` Permanently Locks Liquidatee Bad Debt — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when a liquidatee's `vQuoteBalance` remains negative after insurance is exhausted. No guard exists for `state.openInterest == 0`. The `MathSD21x18.div` library explicitly reverts with `ERR_DIV_BY_ZERO` in this case. Because `_finalizeSubaccount` calls `socializeSubaccount` as a mandatory step, any finalization attempt for an underwater account in a zero-open-interest market permanently reverts, locking bad debt in the protocol forever.

---

### Finding Description

**Root cause — `PerpEngine.socializeSubaccount` lines 164–168:**

```solidity
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← no zero-check; reverts via ERR_DIV_BY_ZERO
    );
``` [1](#0-0) 

`MathSD21x18.div` explicitly enforces `require(y != 0, ERR_DIV_BY_ZERO)`: [2](#0-1) 

**How `openInterest` reaches zero:**

`openInterest` is the sum of `|balance.amount|` across all participants, maintained in `_updateBalance`: [3](#0-2) 

When every participant's `amount` is 0, `openInterest == 0`. The liquidatee's `amount` is already required to be 0 before finalization (line 319 of `ClearinghouseLiq.sol`): [4](#0-3) 

A non-zero `vQuoteBalance` with `amount == 0` is a normal state — it represents unsettled PnL from a closed position. `_setBalanceAndUpdateBitmap` explicitly preserves the balance bit when `vQuoteBalance != 0`: [5](#0-4) 

**Concrete reachable path:**

1. Liquidatee opens a long position at price P1; counterparty opens a matching short.
2. Price drops; liquidatee becomes underwater.
3. Counterparty closes their short → `openInterest` halves.
4. Liquidatee's position is liquidated to the liquidator (via `_handleLiquidationPayment`) → `openInterest` stays the same (liquidator now holds the long).
5. Liquidator closes their position → `openInterest == 0`.
6. Liquidatee still has `balance.amount == 0`, `balance.vQuoteBalance < 0`.
7. A liquidator calls `liquidateSubaccountImpl` with `txn.productId == type(uint32).max` to trigger `_finalizeSubaccount`.
8. `_finalizeSubaccount` calls `perpEngine.socializeSubaccount` (line 386): [6](#0-5) 

9. Inside `socializeSubaccount`, insurance is insufficient, `balance.vQuoteBalance` remains negative, and the division by `state.openInterest == 0` reverts.

**The protocol itself already knows `openInterest == 0` is a valid state** — `updateStates` guards against it with `if (state.openInterest == 0) { continue; }` — but `socializeSubaccount` has no equivalent guard: [7](#0-6) 

---

### Impact Explanation

- `_finalizeSubaccount` always reverts for any liquidatee with `vQuoteBalance < 0` in a zero-open-interest market when insurance is insufficient.
- The liquidatee's bad debt is permanently locked; it can never be socialized or written off.
- Insurance is never applied to cover the loss.
- The invariant that finalization must always complete for any valid underwater account is broken.

---

### Likelihood Explanation

This is reachable in any low-liquidity perp market or a market that is winding down. It does not require any privileged access, governance action, or oracle manipulation. Any liquidator can trigger the revert by calling `liquidateSubaccountImpl` with `productId == type(uint32).max` after the market's `openInterest` reaches zero. The scenario is especially likely for newly listed or thinly traded perp products.

---

### Recommendation

Add a zero-check for `state.openInterest` before the division in `socializeSubaccount`, mirroring the existing guard in `updateStates`. When `openInterest == 0`, there are no other participants to socialize the loss to; the remaining negative `vQuoteBalance` should be absorbed by insurance or written off directly:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest != 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    // If openInterest == 0, no one to socialize to; loss is already absorbed
    balance.vQuoteBalance = 0;
}
``` [8](#0-7)

### Proof of Concept

```solidity
// Hardhat unit test (pseudocode)
it("socializeSubaccount reverts when openInterest == 0 and vQuoteBalance < 0", async () => {
    // 1. Deploy PerpEngine, set up product
    // 2. Open long for liquidatee (amount=100, vQuoteBalance=-1000e18)
    //    Open short for counterparty (amount=-100)
    //    → openInterest = 200
    // 3. Close counterparty short → openInterest = 100
    // 4. Liquidate liquidatee's position to liquidator → openInterest = 100
    //    (liquidatee: amount=0, vQuoteBalance=-1500e18 after liquidation loss)
    // 5. Close liquidator's long → openInterest = 0
    // 6. Verify: states[productId].openInterest == 0
    //            balances[productId][liquidatee].vQuoteBalance < 0
    // 7. Call perpEngine.socializeSubaccount(liquidatee, 0)
    //    → REVERTS with "DBZ"
    await expect(
        perpEngine.socializeSubaccount(liquidatee, 0)
    ).to.be.revertedWith("DBZ");
});
```

The revert is deterministic: `MathSD21x18.div` with `y == 0` always emits `ERR_DIV_BY_ZERO`. [9](#0-8)
