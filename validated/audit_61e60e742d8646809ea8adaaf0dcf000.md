### Title
NLP Burn With Value ≤ $1 Silently Destroys User's NLP Without Crediting Any Quote — (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

In `Clearinghouse.burnNlp()`, a hardcoded minimum burn fee of `ONE = 1e18` ($1 in X18 format) can equal or exceed the entire quote value when a user burns a small NLP position. The NLP balance is **always** decremented unconditionally, but the quote credit is **conditional** on `quoteAmount > 0`. When the NLP being burned is worth ≤ $1, the user loses their NLP tokens and receives zero quote in return — a direct, silent asset loss with no revert or warning.

---

### Finding Description

In `Clearinghouse.burnNlp()`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [1](#0-0) 

`ONE` is defined as `10**18` = $1 in X18 format. [2](#0-1) 

The fee formula `MathHelper.max(ONE, quoteAmount / 1000)` sets a **minimum fee of $1**, regardless of how small the burn is. When `quoteAmount ≤ ONE` (i.e., the NLP being burned is worth ≤ $1), `quoteAmount - burnFee ≤ 0`, so `quoteAmount` is clamped to 0.

The NLP debit is unconditional:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);
``` [3](#0-2) 

But the quote credit is gated:

```solidity
if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
``` [4](#0-3) 

When `quoteAmount = 0`, the NLP is burned and the user receives nothing. There is **no minimum `nlpAmount` check** in `burnNlp()` — the only guards are an overflow check and a balance sufficiency check:

```solidity
require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
int128 nlpAmount = int128(txn.nlpAmount);
require(
    spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
    ERR_UNLOCKED_NLP_INSUFFICIENT
);
``` [5](#0-4) 

By contrast, `depositCollateral` enforces a minimum deposit of `MIN_DEPOSIT_AMOUNT = ONE / 10` ($0.10): [6](#0-5) 

No equivalent minimum is enforced on the burn side.

---

### Impact Explanation

A user who burns NLP with a quote value ≤ $1 loses their entire NLP position and receives zero quote. The state delta is:

- `NLP balance of sender`: decremented by `nlpAmount` (permanent loss)
- `QUOTE balance of sender`: unchanged (no credit)
- `NLP balance of N_ACCOUNT`: incremented by `nlpAmount` (protocol gains the NLP)

This is a direct, unrecoverable asset loss for the user. The transaction does not revert; it silently succeeds with the user holding less NLP and the same quote balance.

---

### Likelihood Explanation

This is reachable by any unprivileged user who:

1. Holds a small unlocked NLP balance worth ≤ $1 (e.g., accumulated through rounding from prior mints, partial rebalances, or profit-share distributions), **or**
2. Deliberately submits a `BurnNlp` transaction with a small `nlpAmount`.

The entry path is: user signs a `BurnNlp` transaction → sequencer submits it via `EndpointTx.submitSlowModeTransactionImpl` → `clearinghouse.burnNlp()` is called. [7](#0-6) 

The sequencer processes the user's signed intent faithfully; no sequencer compromise is required. The on-chain contract has no floor on `nlpAmount`.

---

### Recommendation

Add a minimum quote value check before proceeding with the burn, analogous to the minimum deposit check used in `depositCollateral`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
require(quoteAmount > ONE, "burn value too small"); // must exceed minimum fee
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
```

This ensures the user always receives a non-zero quote credit and mirrors the `MIN_DEPOSIT_AMOUNT` guard on the mint side.

---

### Proof of Concept

**Scenario**: User holds 0.5 NLP, NLP oracle price = $1.00 (i.e., `oraclePriceX18 = 1e18`).

1. User signs `BurnNlp { nlpAmount: 5e17 }` (0.5 NLP in X18).
2. `quoteAmount = nlpAmount.mul(oraclePriceX18) = (5e17 * 1e18) / 1e18 = 5e17` ($0.50 in X18).
3. `burnFee = MathHelper.max(ONE, 5e17 / 1000) = max(1e18, 5e14) = 1e18` ($1.00).
4. `quoteAmount = MathHelper.max(0, 5e17 - 1e18) = max(0, -5e17) = 0`.
5. `spotEngine.updateBalance(NLP_PRODUCT_ID, sender, -5e17)` — NLP burned. ✓
6. `if (quoteAmount > 0)` — **false**, quote credit skipped.
7. User loses 0.5 NLP ($0.50) and receives $0.00 quote. Net loss: $0.50.
