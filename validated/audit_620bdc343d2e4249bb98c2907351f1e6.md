Audit Report

## Title
`SwapAllowlistExtension` checks the immediate pool caller (router) instead of the end-user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of the pool's `swap()` call — the immediate caller, not the originating user. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract. A pool admin who allowlists the router to enable router-mediated swaps for permitted users simultaneously opens the allowlist bypass to every unpermitted user who routes through the same router.

## Finding Description

**Root cause — `SwapAllowlistExtension.beforeSwap()`:** [1](#0-0) 

The check `allowedSwapper[msg.sender][sender]` uses `sender` — the first argument forwarded by the pool — as the identity being gated. `msg.sender` here is the pool (the extension is called by the pool via `CallExtension.callExtension`).

**`MetricOmmPool.swap()` sets `sender = msg.sender` of the pool call:** [2](#0-1) 

**`ExtensionCalling._beforeSwap()` passes it verbatim:** [3](#0-2) 

**`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router the `msg.sender`:** [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

**Call path analysis:**
- **Direct**: `user → pool.swap()` → extension sees `sender = user` → `allowedSwapper[pool][user]` checked ✓
- **Router**: `user → router → pool.swap()` → extension sees `sender = router` → `allowedSwapper[pool][router]` checked ✗

For a pool admin to allow any allowlisted user to swap via the router, they must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, `allowedSwapper[pool][router] == true` passes for **every** user who routes through it, regardless of individual allowlist status.

**Contrast with `DepositAllowlistExtension`**, which correctly checks `owner` (the LP position owner, a user-supplied argument preserved through the call path) rather than `sender`: [5](#0-4) 

The swap extension has no equivalent mechanism to recover the true end-user identity.

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of counterparties loses that protection entirely for any user who routes through `MetricOmmSimpleRouter` once the router is allowlisted. An unpermitted user can execute swaps at the oracle-anchored bid/ask price, draining LP principal at the configured price. This is a direct loss of LP assets and a broken core pool invariant — the allowlist guard fails open. Severity: High.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the documented, supported periphery entrypoint for swaps. [6](#0-5) 

Any pool admin who enables router-mediated swaps for their allowlisted users must allowlist the router address, which simultaneously opens the bypass for all other users. The trigger requires no privileged access — any unpermitted user can call the public router functions. The condition is a natural and expected admin action, making exploitation highly likely once a pool with this extension is deployed.

## Recommendation

The extension must gate on the actual end-user identity, not the immediate pool caller. Two sound approaches:

1. **Transient-storage forwarding**: Have the router write the originating user address into transient storage before calling the pool, and have the extension read it. The pool's `nonReentrant` guard already uses transient storage (`TSTORE`/`TLOAD`), so the pattern is established in this codebase.

2. **Extension-data identity**: Require the router to embed the originating user address in `extensionData`, and have the extension decode and verify it against a factory-registered router whitelist (to prevent spoofing by arbitrary callers).

Either approach must ensure the identity cannot be spoofed by an arbitrary caller.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Admin calls setAllowedToSwap(pool, alice, true)       // alice is permitted
  - Admin calls setAllowedToSwap(pool, router, true)      // router allowlisted for periphery support
  - bob is NOT on the allowlist

Attack:
  1. bob calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  2. Router calls pool.swap(recipient=bob, ...)            // router is msg.sender
  3. pool.swap() calls _beforeSwap(msg.sender=router, ...)
  4. Extension checks allowedSwapper[pool][router] == true → passes
  5. bob's swap executes at oracle price, draining LP funds

Expected: revert NotAllowedToSwap
Actual:   swap succeeds
```
