Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as Swapper, Allowing Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If a pool admin allowlists the router to enable standard periphery usage, every user — including those not individually allowlisted — can bypass the curation gate by routing through the router.

## Finding Description
**Call path:**

1. User (Bob, not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle()`.
2. The router calls `IMetricOmmPoolActions(params.pool).swap(...)` — `msg.sender` to the pool is the **router address** (`metric-periphery/contracts/MetricOmmSimpleRouter.sol`, L72-80).
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` where `msg.sender` is the router (`metric-core/contracts/MetricOmmPool.sol`, L230-231).
4. `ExtensionCalling._beforeSwap` forwards `sender = router` to every configured extension.
5. `SwapAllowlistExtension.beforeSwap` evaluates:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `msg.sender` is the pool and `sender` is the router. If `allowedSwapper[pool][router]` is `true`, the check passes for **any caller**.

The router performs no per-user allowlist enforcement before calling the pool. The same bypass applies to `exactInput` (all hops see `sender = router`), `exactOutputSingle`, and `exactOutput` (intermediate hops called from within the router's callback also see `msg.sender = router`).

**Root cause:** The allowlist is keyed on the immediate caller of `pool.swap()`, not the economic actor initiating the trade. The router is a transparent forwarder with no per-user access control.

## Impact Explanation
A pool admin deploying `SwapAllowlistExtension` intends to restrict trading to a curated set of addresses (e.g., KYC-gated, institutional-only, or partner-restricted). To allow those addresses to use the standard periphery, the admin must allowlist the router — a natural and expected configuration step. Doing so silently opens the pool to all callers. This is an admin-boundary break: the pool admin's access-control boundary is bypassed by an unprivileged path (the public router). Unauthorized users can trade on pools designed to be curated, and in pools where the allowlist is the primary defense (e.g., paired with a stop-loss extension), bypassing it removes the first line of protection and exposes LP funds to unintended counterparties.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router. This is a natural and expected configuration step — without it, allowlisted users cannot use the standard periphery at all. The admin is likely to make this configuration choice without realizing it grants universal access. The router is a public, permissionless contract, so once allowlisted, the bypass is trivially reachable by any EOA with no special privileges.

## Recommendation
The allowlist must gate the end user, not the immediate caller. Options:

1. **Forward the original user through the router:** The router passes `msg.sender` as an additional field in `extensionData`; the extension decodes and checks it. Requires coordinated change to router and extension.
2. **Router registry fallback:** If `sender` is a known router, the extension requires the actual user identity to be supplied in `extensionData` and verified.
3. **Document the equivalence:** Document that allowlisting the router is equivalent to `allowAllSwappers = true` and provide a separate per-user gating mechanism through the router.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension.
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is allowed
  - Pool admin calls setAllowedToSwap(pool, router, true)  // router allowlisted so alice can use periphery

Attack:
  - Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)
    → msg.sender to pool = router
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] → true
  - Swap executes successfully for Bob despite Bob not being on the allowlist.

Expected: revert NotAllowedToSwap
Actual:   swap succeeds
```
