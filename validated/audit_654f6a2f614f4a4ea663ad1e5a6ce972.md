Audit Report

## Title
`SwapAllowlistExtension` Gates on Router Address Instead of End-User, Allowing Any User to Bypass Curated-Pool Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` (its direct caller) as `sender` to `_beforeSwap`, which forwards it unchanged to `SwapAllowlistExtension.beforeSwap`. When swaps are routed through `MetricOmmSimpleRouter`, the direct caller of `pool.swap` is the router contract, not the end user. A pool admin who allowlists the router — the only way to permit any router-mediated swap — simultaneously opens the gate to every user on the internet, rendering the allowlist ineffective against router-routed calls.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender`:**
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` at line 231. [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` unchanged:**
The value is encoded directly into the extension call with no transformation. [2](#0-1) 

**Step 3 — Extension gates on `allowedSwapper[pool][sender]`:**
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. [3](#0-2) 

**Step 4 — Router calls `pool.swap` directly, making itself `msg.sender`:**
`MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` with no originator forwarding. The end user's address is stored only in transient storage (`_setNextCallbackContext(..., msg.sender, ...)`) for payment settlement, never forwarded to the extension. [4](#0-3) 

**Root cause:** The extension receives `sender = router_address`. The pool admin faces an inescapable dilemma:
- **Router NOT allowlisted**: all allowlisted users are also blocked from using the router.
- **Router IS allowlisted** (required for any router-based swap): `allowedSwapper[pool][router] = true`, so the check passes for **every** caller who routes through the router, including completely non-allowlisted users.

The same vulnerability applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) [6](#0-5) 

## Impact Explanation

Any non-allowlisted user can bypass the swap allowlist on a curated pool by calling `MetricOmmSimpleRouter.exactInputSingle` (or any `exact*` variant). This breaks the core pool functionality of the `SwapAllowlistExtension` — the extension's entire purpose is to restrict swaps to approved addresses, and that invariant is violated for all router-routed calls. Pools intended to be KYC-gated, institution-only, or compliance-restricted are fully open to any user who routes through the canonical router, causing direct policy violation and potential regulatory or financial harm to the pool operator and its LPs.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool admin who wants allowlisted users to use the standard router **must** allowlist the router address — this is a normal, expected operational step with no alternative. Once taken, the bypass is trivially reachable by any user with no special privileges, no flash loan, and no multi-transaction setup. A single `exactInputSingle` call suffices.

## Recommendation

The extension must gate on the economically relevant actor — the end user — not the intermediary. Two complementary fixes:

1. **Pass the original `msg.sender` through the router**: Add an `originator` field to the swap call that the pool always sets to `msg.sender` at the outermost entry, and have the router forward `msg.sender` (the end user) in that field. Extensions can then check `originator` instead of `sender`.

2. **Encode the real user in `extensionData`**: The router could encode `msg.sender` into `extensionData` and the extension could decode and verify it (with appropriate trust checks on the router).

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]  = true   // alice is KYC'd
  allowedSwapper[pool][router] = true   // admin must set this for alice to use the router

Attack (bob, not allowlisted):
  bob calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)
    → msg.sender inside pool = router
    → _beforeSwap(sender=router, ...)
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → allowedSwapper[pool][router] == true  ✓  (check passes)
  swap executes — bob trades on a pool he should be barred from

Direct pool call by bob (no router):
  bob calls pool.swap(...)
    → msg.sender inside pool = bob
    → allowedSwapper[pool][bob] == false  ✗  (correctly reverts)
```

Foundry test plan: deploy pool with `SwapAllowlistExtension`, allowlist alice and the router, call `router.exactInputSingle` as bob (not allowlisted), assert the swap succeeds instead of reverting with `NotAllowedToSwap`.

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
