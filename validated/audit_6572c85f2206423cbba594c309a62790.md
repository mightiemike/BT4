### Title
Exit-Timer Reset via `ClearPosition` Before Lock Expiry Extends Bonus Accrual Window Indefinitely — (`x/tieredrewards/keeper/msg_validate.go`, `x/tieredrewards/types/position_state.go`)

---

### Summary

`validateClearPosition` only enforces the delegation/unbonding guards when `CompletedExitLockDuration` is **true**. When the exit lock has **not** yet elapsed, the function imposes no restriction on resetting the exit state. An attacker can therefore loop: trigger exit → clear exit (before lock expires) → re-trigger exit, pushing `ExitUnlockAt` forward by one block each cycle and extending bonus accrual indefinitely beyond what the protocol intends.

---

### Finding Description

**Step 1 — Trigger exit at time T.**

`MsgTriggerExitFromTier` → `validateTriggerExit` passes (owner check + `HasTriggeredExit()=false`) → `TriggerExit(T, ExitDuration)` sets `ExitTriggeredAt=T`, `ExitUnlockAt=T+ExitDuration`. [1](#0-0) 

`computeSegmentBonus` now caps every future bonus segment at `ExitUnlockAt`: [2](#0-1) 

**Step 2 — Call `MsgClearPosition` at time T+ε (lock not yet elapsed).**

`validateClearPosition` checks owner, then enters the `CompletedExitLockDuration` branch **only when the lock has elapsed**. Because T+ε < T+ExitDuration, the branch is skipped entirely — no delegation check, no unbonding check, no guard against resetting an in-flight exit: [3](#0-2) 

Back in `ClearPosition`, `HasTriggeredExit()=true` so the early-return is bypassed, `claimRewards` is called (collecting bonus up to T+ε, correctly capped because T+ε < ExitUnlockAt), then `ClearExit(T+ε)` zeroes both `ExitTriggeredAt` and `ExitUnlockAt` and resets `LastBonusAccrual` to the current block time: [4](#0-3) [5](#0-4) 

**Step 3 — Re-trigger exit at T+ε.**

`validateTriggerExit` passes again because `HasTriggeredExit()` is now false: [6](#0-5) 

`TriggerExit(T+ε, ExitDuration)` sets a fresh `ExitUnlockAt = T+ε+ExitDuration`. Bonus accrual is now capped at the new, later deadline.

**Repeat N times** → `ExitUnlockAt` advances by N·ε, and the attacker accrues bonus for `ExitDuration + N·ε` instead of the intended `ExitDuration`.

---

### Impact Explanation

`computeSegmentBonus` caps each segment at `ExitUnlockAt`: [7](#0-6) 

`applyBonusAccrualCheckpoint` also caps `LastBonusAccrual` at `ExitUnlockAt` only when the lock has **completed**: [8](#0-7) 

Both caps are defeated when `ExitUnlockAt` is reset and pushed forward. The attacker drains extra tokens from `RewardsPoolName` equal to:

```
shares × tokensPerShare × BonusApy × (N·ε) / SecondsPerYear
```

for each reset cycle N. With a 6-second block time and a 21-day `ExitDuration`, running the loop once per block for the full duration yields roughly **2× the intended bonus payout** for that position, at direct cost to the shared bonus pool and all other legitimate stakers.

---

### Likelihood Explanation

The attack requires only two standard user-signed transactions per cycle (`MsgClearPosition` + `MsgTriggerExitFromTier`), no special permissions, no governance, and no external dependencies. Any position owner on a non-close-only tier can execute it. The loop is cheap (two Cosmos SDK messages per block) and the gain scales linearly with position size and repetition count.

---

### Recommendation

In `validateClearPosition`, add an explicit guard that blocks clearing an in-flight exit before the lock duration has elapsed:

```go
if pos.HasTriggeredExit() && !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
    return types.ErrExitLockDurationNotReached
}
``` [3](#0-2) 

Alternatively, if mid-lock cancellation is intentional, `ClearExit` must set `LastBonusAccrual` to the **original** `ExitUnlockAt` (not `blockTime`) so that re-triggering exit cannot extend the accrual window beyond the first trigger's deadline. [5](#0-4)

func (k Keeper) validateClearPosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
		if err != nil {
			return err
		}
		if isUnbonding {
			return types.ErrPositionUnbonding
		}

		if !pos.IsDelegated() {
			return types.ErrPositionNotDelegated
		}

	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	return nil
}
