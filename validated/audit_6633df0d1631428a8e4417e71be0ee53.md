### Title
Slow Mode `WithdrawCollateral` Bypasses `withdrawFeeX18`, Depriving Sequencer of Fee Revenue - (File: `core/contracts/EndpointTx.sol`)

---

### Summary

`EndpointTx.sol` contains two separate code paths for processing a `WithdrawCollateral` transaction. The sequencer-executed path (`processTransactionImpl`) charges the per-product `withdrawFeeX18` fee before executing the withdrawal. The slow mode path (`processSlowModeTransactionImpl`) executes the identical withdrawal without charging `withdrawFeeX18` at all. Any user can route their withdrawal through slow mode to avoid this fee entirely.

---

### Finding Description

When the sequencer processes a `WithdrawCollateral` transaction via `processTransactionImpl`, it explicitly charges the configured withdrawal fee before calling `clearinghouse.withdrawCollateral()`: [1](#0-0) 

```solidity
chargeFee(
    signedTx.tx.sender,
    spotEngine.getConfig(signedTx.tx.productId).withdrawFeeX18,
    signedTx.tx.productId
);
clearinghouse.withdrawCollateral(...);
```

However, when the same `WithdrawCollateral` transaction type is processed through the slow mode path in `processSlowModeTransactionImpl`, no `withdrawFeeX18` is charged: [2](#0-1) 

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(...);
    validateSender(txn.sender, sender);
    clearinghouse.withdrawCollateral(
        txn.sender, txn.productId, txn.amount, address(0), nSubmissions
    );
}
```

The slow mode submission path is publicly accessible to any user via `submitSlowModeTransaction()`: [3](#0-2) 

The only cost to use slow mode is a flat `SLOW_MODE_FEE` of `$1` (in quote token, paid from the user's external wallet), not from the subaccount balance: [4](#0-3) [5](#0-4) 

The `withdrawFeeX18` is a per-product fee stored in `SpotEngine` config and is deducted from the user's subaccount balance via `chargeFee`, which credits `sequencerFee[productId]`: [6](#0-5) 

By using slow mode, the user avoids this subaccount deduction entirely. The slow mode transaction executes after a 3-day delay (`SLOW_MODE_TX_DELAY`): [7](#0-6) 

---

### Impact Explanation

The `withdrawFeeX18` fee is never charged when a user withdraws via slow mode. The `sequencerFee[productId]` accumulator is never incremented for these withdrawals, meaning the sequencer/protocol permanently loses the fee revenue that would have been collected. For any withdrawal where `withdrawFeeX18 > SLOW_MODE_FEE` ($1), the user profits by routing through slow mode. The corrupted state is `sequencerFee[productId]` — it is understated by the sum of all `withdrawFeeX18` amounts that should have been charged on slow mode withdrawals.

---

### Likelihood Explanation

High. The entry point `submitSlowModeTransaction()` is a public, permissionless function callable by any user. No special role, signature from a privileged party, or protocol state is required beyond having a registered subaccount and $1 in quote tokens. The 3-day delay is the only friction, which is acceptable for users making large withdrawals where the fee savings exceed $1.

---

### Recommendation

Apply the same `chargeFee` call for `withdrawFeeX18` inside `processSlowModeTransactionImpl` for the `WithdrawCollateral` case, mirroring the logic in `processTransactionImpl`. Since slow mode transactions do not carry a nonce-validated signature over the fee amount, the fee should be read directly from `spotEngine.getConfig(txn.productId).withdrawFeeX18` at execution time, identical to the V1 sequencer path.

---

### Proof of Concept

1. User has a subaccount with a balance in product `P`, where `spotEngine.getConfig(P).withdrawFeeX18 > SLOW_MODE_FEE` (e.g., fee is $5).
2. User encodes a `WithdrawCollateral{sender, productId: P, amount, nonce}` transaction (no signature required for slow mode sender validation — only `validateSender` checks `msg.sender` matches the address embedded in `sender`).
3. User calls `Endpoint.submitSlowModeTransaction(transaction)`, paying $1 flat fee from their wallet.
4. After 3 days, anyone calls `Endpoint.executeSlowModeTransaction()`.
5. `processSlowModeTransactionImpl` executes the `WithdrawCollateral` branch: `clearinghouse.withdrawCollateral(...)` is called with no prior `chargeFee(sender, withdrawFeeX18, productId)`.
6. User receives full withdrawal amount; `sequencerFee[P]` is not incremented by `withdrawFeeX18`. The $5 fee is permanently lost to the protocol.
