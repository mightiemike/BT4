Audit Report

## Title
`SwapAllowlistExtension` gates the direct pool caller (`msg.sender`) as `sender`, permanently locking allowlisted EOAs out of swapping — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` (the direct pool caller) as `sender` to `_beforeSwap()`, which forwards it unchanged to `SwapAllowlistExtension.beforeSwap`. When an EOA routes through `MetricOmmSimpleRouter` — the only viable path, since EOAs cannot implement `IMetricOmmSwapCallback` — the router becomes `msg.sender` on the pool call, so `sender = router address`. The router is not in the per-user allowlist, causing every swap by an allowlisted EOA to revert with `NotAllowedToSwap`.

## Finding Description

**Step 1:** `MetricOmmPool.swap()` passes `msg.sender` as `sender` to `_beforeSwap()`: [1](#0-0) 

**Step 2:** `ExtensionCalling._beforeSwap()` forwards `sender` unchanged to the extension: [2](#0-1) 

**Step 3:** `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the direct pool caller: [3](#0-2) 

**Step 4:** `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` with the router as `msg.sender`, so `sender = router address` inside the extension: [4](#0-3) 

**Step 5:** `pool.swap()` immediately calls back `IMetricOmmSwapCallback(msg.sender)`, making direct EOA calls impossible: [5](#0-4) 

The two paths available to an allowlisted EOA:

| Path | `sender` seen by extension | Result |
|---|---|---|
| Direct `pool.swap()` | EOA address (allowlisted ✓) | Reverts — EOA has no `metricOmmSwapCallback` |
| Via `MetricOmmSimpleRouter` | Router address (not allowlisted ✗) | Reverts with `NotAllowedToSwap` |

No reachable path exists through which an allowlisted EOA can successfully swap. The `allowedSwapper` mapping is keyed on the direct pool caller, not the economic actor the admin intended to gate.

## Impact Explanation

Any pool deploying `SwapAllowlistExtension` with per-address allowlisting renders the swap flow completely unusable for allowlisted EOAs. The swap function is the core revenue-generating action of the protocol. The pool admin cannot remedy this without either allowlisting the router (defeating the allowlist entirely) or requiring users to deploy custom callback contracts. This constitutes broken core pool functionality causing an unusable swap flow, matching the allowed impact gate.

## Likelihood Explanation

Every pool using `SwapAllowlistExtension` with individual address allowlisting — the primary documented use case — exhibits this behavior for every allowlisted EOA. No special attacker action is required; the breakage is structural and triggered by the normal user flow through the supported periphery (`MetricOmmSimpleRouter`). The condition is deterministic and repeatable.

## Recommendation

The extension must gate the economically relevant actor, not the direct pool caller. The recommended fix is to have the router encode the original `msg.sender` into `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and check that value when `extensionData` is non-empty. Alternatively, the pool admin can allowlist the router address and rely on the router's own access controls, though this weakens the per-user guarantee. At minimum, documentation must warn that allowlisted addresses must be direct pool callers implementing `IMetricOmmSwapCallback`, not EOAs.

## Proof of Concept

```solidity
// Pool is configured with SwapAllowlistExtension.
// Pool admin allowlists Alice (an EOA).
swapExtension.setAllowedToSwap(address(pool), alice, true);

// Alice tries to swap through the router (the only path available to an EOA).
vm.prank(alice);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    recipient: alice,
    tokenIn: token1,
    zeroForOne: false,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// Reverts: NotAllowedToSwap
// pool.swap() is called by the router → sender = router address
// allowedSwapper[pool][router] == false → revert

// Alice cannot call pool.swap() directly:
// pool.swap() calls IMetricOmmSwapCallback(alice).metricOmmSwapCallback(...)
// Alice is an EOA → no code → revert
// Alice is permanently locked out.
```
