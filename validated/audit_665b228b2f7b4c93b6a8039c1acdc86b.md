### Title
Gas Key Host Functions Use Borsh Wire Length Instead of Trie-Identifier Length for ML-DSA-65 Keys, Breaking Exec-Fee Parity Invariant — (`runtime/near-vm-runner/src/logic/logic.rs`, `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`)

---

### Summary

The gas-key host functions (`promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`) compute both send and exec fees using `public_key_len as usize` — the raw borsh wire length of the public key as passed by the WASM contract. For ML-DSA-65 keys this is **1953 bytes** (1 tag + 1952-byte raw pubkey). The actual on-trie identifier is **33 bytes** (1 tag + 32-byte SHA3-256 hash). The transaction path correctly uses `action.public_key.trie_id_len()` (= 33) for the same fee functions. The divergence is ~1920 bytes, producing a ~59× overcharge in exec gas for any WASM contract that calls these host functions with an ML-DSA-65 gas key.

---

### Finding Description

`PublicKey::trie_id_len()` and `PublicKey::len()` are identical for ED25519 (33) and SECP256K1 (65), but diverge for ML-DSA-65: `len()` returns 1953 (borsh wire form) while `trie_id_len()` returns 33 (SHA3-256 hash form stored in the trie). The design document explicitly states: *"Every storage-stake and trie-byte-priced fee path was updated to call `trie_id_len()`."*

**Transaction path** (`runtime/runtime/src/config.rs`, line 115) correctly uses `trie_id_len()`:

```rust
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len()).total()
}
```

**Host function path** (`runtime/near-vm-runner/src/logic/logic.rs`, lines 3091–3095) uses the raw wire length:

```rust
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← 1953 for ML-DSA-65, should be 33
);
```

The same pattern appears in `promise_batch_action_add_gas_key_with_full_access` (lines 3155–3159) and `promise_batch_action_add_gas_key_with_function_call` (lines 3226–3230), and is mirrored in the wasmtime runner at lines 3323–3325, 3395–3399, and 3499–3503.

`gas_key_transfer_exec_fee` passes `public_key_len` directly into `access_key_key_len(account_id_len, public_key_len)`, which is supposed to compute the actual trie key length. For ML-DSA-65 this inflates the trie key length by ~1920 bytes. `gas_key_add_key_exec_fee` similarly inflates the per-nonce key length:

```rust
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
```

The public key is already parsed before the fee call (`self.get_public_key(public_key_ptr, public_key_len)?`), so the key type is known and `public_key.decode()?.trie_id_len()` is available — it is simply not used.

The fee-parity test (`test_gas_key_fee_parity_full_access`) only exercises ED25519 keys, where `len() == trie_id_len() == 33`, so the discrepancy is invisible in the test suite.

---

### Impact Explanation

Any WASM contract invoking `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key will be charged exec gas computed against a 1953-byte trie key instead of the actual 33-byte trie key — a ~59× overcharge on the per-byte component. Concretely, `gas_key_byte` exec gas is multiplied by `access_key_key_len(account_id_len, 1953)` instead of `access_key_key_len(account_id_len, 33)`. This breaks the protocol invariant that the exec fee reflects the actual trie write cost, breaks fee parity between the transaction and host-function paths for the same action, and can cause legitimate contract calls to exhaust their gas budget and fail.

---

### Likelihood Explanation

Both `ProtocolFeature::GasKeys` and `ProtocolFeature::PostQuantumSignatures` must be active. Once both are live, any unprivileged WASM contract that manages ML-DSA-65 gas keys via host functions triggers the overcharge. The contract author controls the public key bytes passed to the host function; no privileged role is required.

---

### Recommendation

Replace `public_key_len as usize` with the decoded key's `trie_id_len()` in all three host functions, for both the send-fee and exec-fee calls. After `get_public_key` succeeds, the key type is known:

```rust
let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
let decoded_key = public_key.decode()?;
let trie_len = decoded_key.trie_id_len();

let send = gas_key_transfer_send_fee(&self.fees_config, sir, trie_len);
let exec = gas_key_transfer_exec_fee(&self.fees_config, receiver_id.len(), trie_len);
// ... then use decoded_key directly in append_action_transfer_to_gas_key
```

Apply the same fix to `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`, and to the wasmtime runner counterparts. Extend `test_gas_key_fee_parity_full_access` to cover ML-DSA-65 keys.

---

### Proof of Concept

**Divergent values for ML-DSA-65:**

| Path | Value passed to fee functions | Source |
|---|---|---|
| Transaction (`total_send_fees`) | `action.public_key.trie_id_len()` = **33** | `runtime/runtime/src/config.rs:115` |
| Host function (`promise_batch_action_transfer_to_gas_key`) | `public_key_len as usize` = **1953** | `runtime/near-vm-runner/src/logic/logic.rs:3091` |

`gas_key_transfer_exec_fee` with `public_key_len = 1953` computes:
```
trie_key_len = access_key_key_len(account_id_len, 1953)
             = 1 + account_id_len + 1953          // ~1920 bytes too large
```
versus the correct:
```
trie_key_len = access_key_key_len(account_id_len, 33)
             = 1 + account_id_len + 33
```

The per-byte exec gas charged is `gas_key_byte × (trie_key_len + min_gas_key_borsh_len)`. With a typical account id of 10 bytes, the inflated trie key is 1964 bytes vs the correct 44 bytes — a **~44.6× overcharge** on the per-byte exec component alone. For `gas_key_add_key_exec_fee` with `num_nonces` nonces, the overcharge compounds across every nonce entry. pub fn gas_key_transfer_exec_fee(
```rust
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
```

```rust
pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }
```

```rust
    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }
```

```markdown
### 5. Storage usage and fee plumbing

The storage-stake calculation
(`runtime/runtime/src/access_keys.rs::access_key_storage_usage`) and the

trie-byte-priced fee path was updated to call `trie_id_len()`.
```
