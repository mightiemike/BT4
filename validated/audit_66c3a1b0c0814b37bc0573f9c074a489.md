Audit Report

## Title
SwapAllowlistExtension Gates Router Address Instead of Actual Swapper, Allowing Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the router becomes `msg.sender` of `pool.swap()`, so the extension checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any pool admin who allowlists the router (a routine step to let legitimate users trade via the standard periphery) inadvertently grants every unpermissioned user the ability to bypass the allowlist.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no user identity forwarding, making the router the `msg.sender` of that call: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. Once the router is in `allowedSwapper[pool]`, the check `allowedSwapper[pool][router] == true` passes for every caller of the router regardless of individual permission. The `allowAllSwappers` short-circuit does not mitigate this: in restrictive mode (`allowAllSwappers[pool] == false`), the bypass still works as long as the router address itself is allowlisted. [5](#0-4) 

## Impact Explanation

A pool configured with `SwapAllowlistExtension` is a gated pool — only approved counterparties are supposed to trade against its liquidity. Once the router is allowlisted (a necessary operational step for legitimate users to trade via the standard periphery), any address can execute swaps against the pool's oracle-anchored bins. An unpermissioned user can drain value from the pool at the oracle mid-price, causing direct loss of LP principal. This breaks the core security invariant of the allowlist extension and constitutes a Medium-to-High direct loss of user principal.

## Likelihood Explanation

The trigger requires the router to be in the allowlist, which is a normal operational action any pool admin would take to let their legitimate users trade via the standard periphery. The bypass itself requires no privilege — any EOA or contract can call `MetricOmmSimpleRouter`. The combination of a routine admin action and a zero-privilege attacker path makes this reliably reachable.

## Recommendation

The extension must resolve the actual end-user identity rather than the immediate caller. Two sound approaches:

1. **Pass the original user through the router.** Have `MetricOmmSimpleRouter` encode the real `msg.sender` in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and gate on that value (with the pool verifying the router is the trusted forwarder).
2. **Gate on `recipient` instead of `sender`.** If the pool's design guarantees that the economic beneficiary of a swap is always `recipient`, gate on that field. This is simpler but must be verified against the pool's settlement logic.

## Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension, allowAllSwappers[pool] = false
  allowedSwapper[pool][alice] = true          // alice is a legitimate user
  allowedSwapper[pool][router] = true         // admin adds router so alice can use it
  bob is NOT in allowedSwapper

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=bob, ...)           // router is msg.sender
    → pool calls _beforeSwap(sender=router, ...)
    → extension checks: allowedSwapper[pool][router] == true  ✓
    → swap executes, bob receives token output
    → LP funds reduced without bob being an approved swapper
```

Bob successfully swaps against a pool he is not allowlisted for, extracting value from LPs who expected only approved counterparties.
