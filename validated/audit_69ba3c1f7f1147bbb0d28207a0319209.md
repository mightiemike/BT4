### Title
`ValidResourceBounds` Protobuf Round-Trip Loses `AllResources` Variant When `l2_gas` and `l1_data_gas` Are Both Zero, Causing Transaction Hash Divergence Between Gateway and P2P Sync Paths - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf serialization of `ValidResourceBounds` is lossy: `AllResources { l1_gas: X, l2_gas: ZERO, l1_data_gas: ZERO }` and `L1Gas(X)` both serialize to the same protobuf bytes. On deserialization, the receiving node always reconstructs `L1Gas`. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` field in the hash preimage for `AllResources` but omits it for `L1Gas`, the transaction hash computed at the gateway (H1) differs from the hash recomputed by any node that receives the transaction over p2p sync (H2). The OS/prover recomputes the hash from stored transaction data and will produce H2 ‚Č† H1, invalidating the proof.

### Finding Description

**Step 1 ‚Äď Lossy serialization.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes both `AllResources { l1_gas: X, l2_gas: ZERO, l1_data_gas: ZERO }` and `L1Gas(X)` identically:

```
l1_gas = X, l2_gas = ZERO, l1_data_gas = ZERO
``` [1](#0-0) 

**Step 2 ‚Äď Deserialization always produces `L1Gas` for the zero case.**

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ‚Üź always chosen
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

**Step 3 ‚Äď Hash preimage differs between the two variants.**

`get_tip_resource_bounds_hash` appends the `L1_DATA_GAS` field only for `AllResources`:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // no L1_DATA_GAS
    ValidResourceBounds::AllResources(all) =>
        vec![get_concat_resource(&all.l1_data_gas, L1_DATA_GAS)?],   // L1_DATA_GAS appended
});
``` [3](#0-2) 

So for the same transaction:
- Gateway path: `resource_bounds = AllResources(...)` ‚Üí hash preimage contains `[tip, L1_GAS, L2_GAS(0), L1_DATA_GAS(0)]` ‚Üí **H1**
- P2P sync path: deserialized as `L1Gas(...)` ‚Üí hash preimage contains `[tip, L1_GAS, L2_GAS(0)]` ‚Üí **H2 ‚Č† H1**

**Step 4 ‚Äď The gateway accepts such transactions when `min_gas_price = 0` or `validate_resource_bounds = false`.**

The stateless validator only rejects zero-l2_gas-price transactions when `min_gas_price > 0`:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [4](#0-3) 

The test suite explicitly confirms that `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: ZERO, l1_data_gas: ZERO }` passes validation with the default testing config (which has `min_gas_price = 0`): [5](#0-4) 

**Step 5 ‚Äď The OS Cairo code recomputes the hash from transaction data.**

The OS `compute_invoke_transaction_hash` function recomputes the hash from the `CommonTxFields` (which includes the resource bounds). If the stored transaction data has `L1Gas` instead of `AllResources`, the OS computes H2 ‚Č† H1, producing an invalid proof. [6](#0-5) 

### Impact Explanation

A transaction accepted by the gateway with hash H1 is stored with `AllResources` bounds. When synced to another node via p2p, the protobuf round-trip silently converts the bounds to `L1Gas`. The receiving node stores the transaction with the correct hash H1 (from the `FullTransaction` wire message) but with structurally wrong `L1Gas` bounds. When the OS/prover later recomputes the hash from the stored transaction data, it produces H2 ‚Č† H1, making the proof invalid. This matches the impact category: **Transaction conversion or signature/hash logic binds the wrong hash/type** (High) and **wrong receipt/revert result from execution logic for accepted input** (Critical path through proof generation).

### Likelihood Explanation

The trigger requires `min_gas_price = 0` (the default in testing configurations) or `validate_resource_bounds = false`. Both are plausible in non-mainnet deployments, bootstrap phases, or nodes configured with `validate_resource_bounds: false`. The bug is latent in all deployments and activates whenever a transaction with only `l1_gas` non-zero reaches the sequencer.

### Recommendation

Fix the deserialization in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` to preserve the `AllResources` variant when `l1_data_gas` is explicitly present in the protobuf message (even if zero), rather than collapsing it to `L1Gas`. One approach: use the presence of the `l1_data_gas` field (not its value) to distinguish the two variants, consistent with the proto comment `optional ResourceLimits l1_data_gas = 2`. [7](#0-6) 

Alternatively, add a gateway-level check that rejects any `AllResourceBounds` transaction where both `l2_gas` and `l1_data_gas` are entirely zero (both `max_amount` and `max_price_per_unit`), since such a transaction is indistinguishable from a legacy `L1Gas` transaction after protobuf round-trip.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, ValidResourceBounds,
};
use apollo_protobuf::proto::p2p::proto as protobuf;

// Construct AllResources with zero l2_gas and l1_data_gas
let original = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: ResourceBounds { max_amount: 100.into(), max_price_per_unit: 1.into() },
    l2_gas: ResourceBounds::default(),      // zero
    l1_data_gas: ResourceBounds::default(), // zero
});

// Serialize to protobuf (gateway ‚Üí wire)
let proto: protobuf::ResourceBounds = original.into();

// Deserialize on receiving node (wire ‚Üí storage)
let deserialized = ValidResourceBounds::try_from(proto).unwrap();

// Variant has changed: AllResources ‚Üí L1Gas
assert!(matches!(deserialized, ValidResourceBounds::L1Gas(_)));  // passes

// Hash divergence
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

let h1 = get_tip_resource_bounds_hash(&original, &Tip(0)).unwrap();
let h2 = get_tip_resource_bounds_hash(&deserialized, &Tip(0)).unwrap();
assert_ne!(h1, h2);  // passes ‚ÄĒ hash divergence confirmed
```

The `get_tip_resource_bounds_hash` call for `original` (`AllResources`) appends `L1_DATA_GAS(0)` to the hash preimage; the call for `deserialized` (`L1Gas`) does not, producing a different Poseidon digest. [8](#0-7) [9](#0-8)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
