### Title
Bonus Reward Checkpoints Advanced and Event Reference Counts Decremented Without Paying Bonus — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, by the time the error is returned, the function has already (1) decremented validator event reference counts in persistent state and (2) advanced the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` checkpoints in memory. The caller then persists those updated checkpoints via `setPositionWithState`. The net result is that the accounting records the bonus as settled, but the bonus tokens are never transferred to the position owner — a permanent, irrecoverable loss.

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` follows this sequence:

1. **Loop over events** — for each event, compute segment bonus and call `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` (line 196). This is a persistent state write; when the count reaches zero the event is pruned.
2. **Advance checkpoints in memory** — `applyBonusAccrualCheckpoint` (line 215) and `pos.UpdateLastKnownBonded` (line 217) mutate the `*PositionState` passed by pointer.
3. **Check pool balance** — `k.sufficientBonusPoolBalance(ctx, bonusCoins)` (line 230). If the pool is insufficient, the function returns `ErrInsufficientBonusPool` **after** steps 1–2 have already executed. [1](#0-0) [2](#0-1) 

In `slashRedelegationPosition`, the error from `processEventsAndClaimBonus` is caught and swallowed for `ErrInsufficientBonusPool`:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [3](#0-2) 

Execution then continues to `setPositionWithState`, which persists the in-memory checkpoint updates that `processEventsAndClaimBonus` wrote before failing: [4](#0-3) 

After this call:
- The validator events' reference counts have been decremented (events may be pruned from state).
- The position's `LastEventSeq` and `LastBonusAccrual` are advanced past those events.
- The bonus tokens have **not** been sent to the owner.

Because the checkpoints are now ahead of the pruned events, the position can never re-process those events to claim the owed bonus. The loss is permanent.

### Impact Explanation

A position owner whose validator's redelegation is slashed at a moment when the bonus pool balance is below the owed bonus permanently loses all bonus rewards accrued up to that point. The `underlyingBalance` analog here is the bonus pool accounting: the system records the bonus as consumed (checkpoints advanced, events pruned) without the corresponding transfer. Any future `claimRewards` call on the position will start from the advanced checkpoint and will never recover the lost amount.

### Likelihood Explanation

The trigger requires two concurrent conditions: (a) a validator slash on a redelegation entry that maps to a tiered-rewards position, and (b) the `RewardsPoolName` module account balance being below the computed bonus. Condition (a) is a normal protocol event (double-sign or downtime slash). Condition (b) can arise naturally as the pool is depleted by many concurrent claims or if it was never sufficiently funded. Neither condition requires privileged access; any delegator whose validator is slashed while the pool is low is affected.

### Recommendation

Before decrementing event reference counts or advancing checkpoints, verify that the bonus pool has sufficient balance. If the pool is insufficient, either:
- Skip the entire `processEventsAndClaimBonus` call without mutating any state (preserve events and checkpoints for a future claim), or
- Record the unpaid bonus as a debt to be settled when the pool is replenished.

The fix must ensure that persistent state (ref count decrements) and in-memory checkpoint advances are only committed when the corresponding token transfer succeeds.

### Proof of Concept

1. Alice creates a tiered-rewards position delegated to validator V.
2. Alice accrues bonus rewards over several blocks (validator events accumulate).
3. Alice redelegates from V to V2 — a redelegation entry is created.
4. The bonus pool is drained (e.g., many other positions claim rewards).
5. V2 is slashed; `BeforeRedelegationSlashed` fires → `slashRedelegationPosition` is called.
6. `processEventsAndClaimBonus` decrements ref counts for all of Alice's pending events and advances her checkpoints, then returns `ErrInsufficientBonusPool`.
7. The error is swallowed; `setPositionWithState` persists Alice's advanced checkpoints.
8. Alice calls `ClaimRewards` — her `LastEventSeq` is already past all accrued events; zero bonus is returned. The bonus is permanently lost. [5](#0-4) [6](#0-5)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
