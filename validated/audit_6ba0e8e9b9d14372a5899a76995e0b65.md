Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, enabling full allowlist bypass for any caller routing through `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the guard checks `allowedSwapper[pool][router]`. Any pool admin who allowlists the router to support normal UX simultaneously grants every address on-chain the ability to swap in that pool, completely neutralizing the allowlist.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the extension.**

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← whoever called pool.swap
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards that value verbatim as the `sender` argument to every configured extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
abi.encodeCall(
  IMetricOmmExtensions.beforeSwap,
  (sender, recipient, zeroForOne, ...)
)
```

**Step 2 — The router is `msg.sender` to the pool, not the end user.**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly. The pool therefore sees `msg.sender = router`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ...);
```

The same applies to `exactOutputSingle` (L136-137), `exactInput` (L104-112, all hops), and `exactOutput` (L165-181, all recursive hops via `_exactOutputIterateCallback` L220-228).

**Step 3 — The extension checks the router address, not the user.**

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router address. The lookup becomes `allowedSwapper[pool][router]`. Every user who routes through the same router contract presents the identical identity to the guard.

**Step 4 — The pool admin faces an impossible choice.**

| Admin configuration | Allowlisted users via router | Disallowed users via router |
|---|---|---|
| Router NOT allowlisted | Blocked (broken UX) | Blocked |
| Router allowlisted | Allowed | Also allowed — **bypass** |

There is no configuration that simultaneously permits router-mediated swaps for allowlisted users and blocks them for disallowed users. The router stores the original `msg.sender` only in transient storage for the payment callback context (`_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn)` at L71), but this value is never forwarded to the pool or extension as the identity to check.

## Impact Explanation

A pool admin who allowlists the router address — the natural configuration for a curated pool expected to be used through the standard periphery — grants every address on-chain the ability to swap in that pool. The swap allowlist guard is completely neutralized. Any disallowed user can trade in the curated pool, receiving output tokens at the oracle-derived bid/ask price. This constitutes a direct admin-boundary break: the pool's curation invariant is broken and the LP's exposure is no longer limited to the intended counterparty set, with direct fund-impacting consequences.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entry point. A pool admin who deploys a curated pool and wants allowlisted users to have normal UX will allowlist the router. The bypass is then immediately reachable by any unprivileged address with no preconditions beyond calling `router.exactInputSingle` (or any other router entry point). No special tokens, no malicious setup, and no privileged access are required. The bypass is repeatable on every swap.

## Recommendation

The extension must check the economically relevant actor — the end user — not the intermediary router. Two sound approaches:

1. **Pass the original caller through the router.** The router already stores the original `msg.sender` in transient storage for the payer context. The router could embed the user address in a standardized prefix of `extensionData`, and the extension could decode and check that value when the pool-level `sender` is a known router.

2. **Trusted-forwarder pattern in the extension.** The extension can inspect whether `sender` is a factory-registered router and, if so, require the user address to be embedded in `extensionData` and check that address instead.

The simplest safe fix is to have the router embed the original `msg.sender` in a standardized prefix of `extensionData` and have the extension decode and check that value when the pool-level `sender` is a known router.

## Proof of Concept

```
1. Deploy a pool with SwapAllowlistExtension configured.
2. Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is allowed
3. Pool admin calls setAllowedToSwap(pool, router, true)  // router allowlisted for UX
4. Bob (disallowed) calls router.exactInputSingle({pool: pool, ...})
   → router calls pool.swap(recipient, ...)
   → pool calls _beforeSwap(msg.sender=router, ...)
   → extension checks allowedSwapper[pool][router] → TRUE
   → swap executes for Bob with no revert
5. Bob receives output tokens from the curated pool despite being explicitly disallowed.
```

Root cause locations:
- `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol` L37: checks `sender` (router) instead of end user
- `metric-core/contracts/MetricOmmPool.sol` L231: passes `msg.sender` (router) as `sender`
- `metric-periphery/contracts/MetricOmmSimpleRouter.sol` L72-80: calls `pool.swap` directly without forwarding end user identity to the extension layer
