Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address as `sender` instead of the originating user, nullifying per-pool swap allowlists when the router is allowlisted ‚Äî (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` ‚Äî the router contract when users enter through `MetricOmmSimpleRouter`. A pool admin who allowlists the router so that their curated users can swap through the periphery inadvertently grants every user on the network the ability to bypass the per-pool allowlist, completely nullifying the access control the extension is designed to enforce.

## Finding Description

**Step 1 ‚Äî Pool passes `msg.sender` (the router) as `sender` to the extension.**

`MetricOmmPool.swap` calls `_beforeSwap` with `msg.sender` as the first argument: [1](#0-0) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle` or `exactInput`, the router calls `pool.swap()` directly, so `msg.sender` inside the pool is the router contract, not the end user. [2](#0-1) [3](#0-2) 

**Step 2 ‚Äî `ExtensionCalling._beforeSwap` forwards that `sender` to the extension.**

`_beforeSwap` in `ExtensionCalling` encodes and forwards the `sender` argument (the router address) to every configured extension: [4](#0-3) 

**Step 3 ‚Äî The extension checks the router address, not the end user.**

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool (correct) and `sender` is the router address (wrong ‚Äî it should be the originating user): [5](#0-4) 

**Step 4 ‚Äî Allowlisting the router opens the pool to all users.**

`setAllowedToSwap` stores `allowedSwapper[pool][swapper] = true`. When the admin calls this with `swapper = address(router)`, the check at line 37 passes for every user who routes through the router ‚Äî the per-user allowlist is completely nullified: [6](#0-5) 

**Contrast with `DepositAllowlistExtension`.**

The deposit extension correctly checks `owner` (the position holder), not `sender` (the direct caller), providing the correct separation between the routing intermediary and the economically relevant actor. The swap extension has no equivalent field: [7](#0-6) 

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to a specific set of addresses loses that protection entirely for any user routing through `MetricOmmSimpleRouter`. Unauthorized users can execute swaps at oracle-derived prices, draining LP value or front-running allowlisted participants on pools designed to be closed to the public. This is a direct loss of LP principal and a broken core pool invariant (the allowlist). The impact qualifies as broken core pool functionality causing loss of funds.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entrypoint in the periphery. A pool admin who configures a swap allowlist and wants their allowlisted users to use the router will naturally allowlist the router address ‚Äî there is no documentation warning against this, and the deposit extension's correct `owner`-based check creates a false sense of symmetry. The trigger requires only a standard admin configuration action (allowlisting the router) followed by any public user calling the router. No special privileges or unusual conditions are required for the attacker.

## Recommendation
Pass the originating user's address through the extension mechanism so the allowlist can gate the economically relevant actor:

1. **Preferred**: Have the router encode the originating user in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check that address when `sender` is a known router.
2. **Alternative**: Add a `recipient` check in addition to `sender`, since `recipient` is the address that receives output tokens and is set by the user in `exactInputSingle`/`exactOutputSingle`.
3. **Structural**: Mirror the deposit extension's design ‚Äî expose a separate `originator` field in the swap hook signature so extensions always have access to the true initiating address regardless of routing depth.

## Proof of Concept

```solidity
// Pool admin sets up a curated pool with SwapAllowlistExtension.
// Only `allowedUser` is supposed to be able to swap.
extension.setAllowedToSwap(pool, allowedUser, true);

// Admin also allowlists the router so allowedUser can use it conveniently.
extension.setAllowedToSwap(pool, address(router), true);

// Now ANY user bypasses the allowlist:
// bannedUser calls the router ‚Äî pool sees msg.sender = router, check passes.
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: bannedUser,
    zeroForOne: true,
    amountIn: 1_000e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    extensionData: "",
    deadline: block.timestamp
}));
// Swap executes successfully despite bannedUser not being on the allowlist.
```

The call chain is: `bannedUser` ‚Üí `MetricOmmSimpleRouter.exactInputSingle` ‚Üí `MetricOmmPool.swap` (with `msg.sender = router`) ‚Üí `_beforeSwap(router, ...)` ‚Üí `SwapAllowlistExtension.beforeSwap(router, ...)` ‚Üí `allowedSwapper[pool][router] == true` ‚Üí check passes.
