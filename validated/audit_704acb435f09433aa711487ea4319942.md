### Title
Duplicate `CreateIsolatedSubaccount` Orders for Same `productId` Pool Margins Into a Single Isolated Subaccount — (`core/contracts/OffchainExchange.sol`)

### Summary

`createIsolatedSubaccount` reuses an existing isolated subaccount when one already exists for the same `productId`, then unconditionally transfers the new order's margin into it. Two distinct orders (different digests) for the same `productId` both have their margins transferred to the same isolated subaccount, breaking the one-order-per-isolated-subaccount invariant and creating a desync between `digestToMargin` and the actual subaccount balance.

---

### Finding Description

In `OffchainExchange.createIsolatedSubaccount` (lines 999–1090), the replay guard at lines 1009–1011 only prevents the **same digest** from being processed twice: [1](#0-0) 

It does **not** prevent a second, distinct order (different nonce/price/amount → different digest) for the same `productId` from entering the rest of the function.

The loop at lines 1025–1038 then searches for an existing isolated subaccount for the same `productId` and, if found, reuses it: [2](#0-1) 

After that, regardless of whether a new or existing subaccount was selected, line 1072 maps the new digest to it, and lines 1074–1087 unconditionally transfer the new order's margin into it: [3](#0-2) 

**Concrete call sequence:**

1. `createIsolatedSubaccount(parent, productId, margin=M1)` with `digest1`
   - No existing isolated subaccount for `productId` → new one created
   - `digestToSubaccount[digest1] = iso`, `digestToMargin[digest1] = M1`
   - `iso.quoteBalance += M1`

2. `createIsolatedSubaccount(parent, productId, margin=M2)` with `digest2`
   - `digestToSubaccount[digest2] == 0` → passes replay guard
   - Existing isolated subaccount found for same `productId` → reused
   - `digestToSubaccount[digest2] = iso` (same subaccount), `digestToMargin[digest2] = M2`
   - `iso.quoteBalance += M2` → actual balance is now `M1 + M2`

Both `digest1` and `digest2` are then routed to the same `iso` subaccount in `matchOrders` (lines 673–678): [4](#0-3) 

---

### Impact Explanation

- **Margin accounting desync:** `digestToMargin[digest1] = M1` and `digestToMargin[digest2] = M2`, but the actual isolated subaccount balance is `M1 + M2`. The per-digest margin records no longer correspond to separate isolated positions.
- **Broken isolation invariant:** Both orders trade from the same isolated subaccount. Losses from one order's fill reduce the margin available to the other, and the combined margin `M1 + M2` backs both positions simultaneously — violating the design guarantee that each isolated order has its own ring-fenced margin.
- **Leverage amplification:** A user can submit two isolated orders for the same `productId`, each with margin `M`, and both will be matched against a single subaccount holding `2M`. The effective position size can exceed what either individual margin was intended to support.
- **`digestToMargin` unusable for per-order margin recovery:** Any protocol logic that relies on `digestToMargin[digest]` to determine how much margin belongs to a specific order will produce incorrect results, since multiple digests share one balance pool.

---

### Likelihood Explanation

The path is fully reachable via the sequencer's `submitTransactions` → `processTransaction` → `createIsolatedSubaccount` flow. No special privileges are required beyond owning the parent subaccount. Any user can submit two `CreateIsolatedSubaccount` transactions for the same `productId` with different order parameters (different nonce is sufficient to produce a different digest). [5](#0-4) 

---

### Recommendation

Add a guard in `createIsolatedSubaccount` that rejects a new order when an active isolated subaccount already exists for the same `productId`, rather than silently reusing it:

```solidity
// After the loop at line 1038:
require(
    newIsolatedSubaccount == bytes32(0),
    "Isolated subaccount already exists for this productId"
);
```

Alternatively, if reuse is intentional, the margin transfer must be gated so that it only occurs when a **new** subaccount is created, and the function must revert (not silently succeed) when a second order attempts to add margin to an existing isolated subaccount for the same `productId`.

---

// Pseudocode fuzz test
function testDuplicateIsolatedSubaccountMarginDesync() public {
    bytes32 parent = ...; // funded parent subaccount
    uint32 productId = 1;
    int128 M1 = 100e18;
    int128 M2 = 200e18;

    // Order 1: nonce=1, margin=M1
    IEndpoint.CreateIsolatedSubaccount memory txn1 = buildOrder(parent, productId, M1, nonce=1);
    bytes32 iso1 = offchainExchange.createIsolatedSubaccount(txn1, linkedSigner);

    // Order 2: nonce=2 (different digest), margin=M2, same productId
    IEndpoint.CreateIsolatedSubaccount memory txn2 = buildOrder(parent, productId, M2, nonce=2);
    bytes32 iso2 = offchainExchange.createIsolatedSubaccount(txn2, linkedSigner);

    // Both map to the same isolated subaccount
    assertEq(iso1, iso2);

    // Actual balance is M1+M2, but digestToMargin records them separately
    int128 actualBalance = spotEngine.getBalance(QUOTE_PRODUCT_ID, iso1).amount;
    assertEq(actualBalance, M1 + M2); // passes — desync confirmed
}

    function createIsolatedSubaccount(
        IEndpoint.CreateIsolatedSubaccount memory txn,
        address linkedSigner
    ) external onlyEndpoint returns (bytes32) {
        require(
            !RiskHelper.isIsolatedSubaccount(txn.order.sender),
            ERR_UNAUTHORIZED
        );
        require(_isIsolated(txn.order.appendix), ERR_UNAUTHORIZED);
        bytes32 digest = getDigest(txn.productId, txn.order);
        if (digestToSubaccount[digest] != bytes32(0)) {
            return digestToSubaccount[digest];
        }
        require(
            _checkSignature(
                txn.order.sender,
                digest,
                linkedSigner,
                txn.signature
            ),
            ERR_INVALID_SIGNATURE
        );

        address senderAddress = address(uint160(bytes20(txn.order.sender)));
        uint256 mask = isolatedSubaccountsMask[senderAddress];
        bytes32 newIsolatedSubaccount = bytes32(0);
        for (uint256 id = 0; (1 << id) <= mask; id += 1) {
            if (mask & (1 << id) != 0) {
                bytes32 subaccount = isolatedSubaccounts[txn.order.sender][id];
                if (subaccount != bytes32(0)) {
                    uint32 productId = RiskHelper.getIsolatedProductId(
                        subaccount
                    );
                    if (productId == txn.productId) {
                        newIsolatedSubaccount = subaccount;
                        break;
                    }
                }
            }
        }

        if (newIsolatedSubaccount == bytes32(0)) {
            require(
                !_isReduceOnly(txn.order.appendix),
                "Reduce-only order cannot create isolated subaccount"
            );
            require(
                mask != (1 << MAX_ISOLATED_SUBACCOUNTS_PER_ADDRESS) - 1,
                "Too many isolated subaccounts"
            );
            uint8 id = 0;
            while (mask & 1 != 0) {
                mask >>= 1;
                id += 1;
            }

            // |  address | reserved | productId |   id   |  'iso'  |
            // | 20 bytes |  6 bytes |  2 bytes  | 1 byte | 3 bytes |
            newIsolatedSubaccount = bytes32(
                (uint256(uint160(senderAddress)) << 96) |
                    (uint256(txn.productId) << 32) |
                    (uint256(id) << 24) |
                    6910831
            );
            isolatedSubaccountsMask[senderAddress] |= 1 << id;
            parentSubaccounts[newIsolatedSubaccount] = txn.order.sender;
            isolatedSubaccounts[txn.order.sender][id] = newIsolatedSubaccount;
            _onCreateIsolatedSubaccount(
                newIsolatedSubaccount,
                txn.order.sender
            );
        }

        digestToSubaccount[digest] = newIsolatedSubaccount;

        int128 margin = int128(_isolatedMargin(txn.order.appendix));
        if (margin > 0) {
            digestToMargin[digest] = margin;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.order.sender,
                -margin
            );
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                newIsolatedSubaccount,
                margin
            );
        }

        return newIsolatedSubaccount;
    }
