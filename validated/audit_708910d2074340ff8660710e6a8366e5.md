### Title
`burnNlp` Uses Weaker `MAINTENANCE` Health Check While `withdrawCollateral` Enforces `INITIAL` Health, Enabling Withdrawal Bypass - (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

`burnNlp` gates execution with a `MAINTENANCE` health check, while `withdrawCollateral` enforces the stricter `INITIAL` health check. A user whose INITIAL health is negative (withdrawal-blocked) but MAINTENANCE health is non-negative (not yet liquidatable) can burn NLP tokens to convert a discounted-weight asset into full-weight quote, improving their INITIAL health enough to then pass `withdrawCollateral`'s check and extract funds that should have been frozen.

---

### Finding Description

`withdrawCollateral` enforces `INITIAL` health:

```solidity
IProductEngine.HealthType healthType = sender == X_ACCOUNT
    ? IProductEngine.HealthType.PNL
    : IProductEngine.HealthType.INITIAL;

require(getHealth(sender, healthType) >= 0, ERR_SUBACCT_HEALTH);
``` [1](#0-0) 

`burnNlp` enforces only `MAINTENANCE` health:

```solidity
require(
    getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
    ERR_SUBACCT_HEALTH
);
``` [2](#0-1) 

`INITIAL` and `MAINTENANCE` health use different risk weights. `_getWeightX18` selects `longWeightInitialX18` for `INITIAL` and `longWeightMaintenanceX18` for `MAINTENANCE`: [3](#0-2) 

For any non-quote asset (including NLP), `longWeightInitialX18 < longWeightMaintenanceX18 < 1.0`. Quote always carries weight `1.0` for both health types: [4](#0-3) 

When a user burns NLP and receives quote, the health delta is:

```
ΔINITIAL_health = quote_received × 1.0 − nlp_burned × longWeightInitialX18
```

Because `longWeightInitialX18 < 1.0`, this delta is **positive** for any burn where the burn fee is smaller than `(1 − longWeightInitialX18) × nlp_value`. The burn fee is `max(1e18, quoteAmount / 1000)` — at most 0.1% of value — while the weight gap is typically 10–20%, so the net INITIAL health improvement is large and positive. [5](#0-4) 

---

### Impact Explanation

A user in the zone `MAINTENANCE_health ≥ 0 > INITIAL_health` is supposed to be blocked from withdrawing collateral. By calling `burnNlp`, they convert NLP (discounted INITIAL weight) into quote (full INITIAL weight), lifting their INITIAL health above zero. They can then call `withdrawCollateral` and extract funds that the protocol intended to freeze. This is a direct collateral extraction bypass, identical in class to the referenced report: a detection/gating function uses a less strict threshold than the actual execution logic, allowing users to withdraw more than they should.

---

### Likelihood Explanation

Any user holding unlocked NLP tokens whose subaccount has drifted into the INITIAL-negative / MAINTENANCE-positive band (a normal intermediate state for leveraged positions) can execute this two-step sequence permissionlessly through the sequencer. No privileged access, oracle manipulation, or governance capture is required. The NLP lock period (`NLP_LOCK_PERIOD`) delays but does not prevent the attack once tokens are unlocked. [6](#0-5) 

---

### Recommendation

Replace the `MAINTENANCE` health check in `burnNlp` with an `INITIAL` health check, consistent with `withdrawCollateral` and `transferQuote`:

```solidity
require(
    getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
    ERR_SUBACCT_HEALTH
);
```

This ensures that a user who is already below the INITIAL health threshold cannot use NLP burning as a stepping stone to unlock withdrawals.

---

### Proof of Concept

1. Alice holds 1000 NLP (price = $1, `longWeightInitialX18` = 0.80, `longWeightMaintenanceX18` = 0.90) and has open perp positions such that her INITIAL health = −$15 and MAINTENANCE health = +$85.
2. Alice calls `burnNlp` for 1000 NLP. Burn fee = max($1e18, $1000 / 1000) = $1. She receives $999 quote.
3. INITIAL health delta = $999 × 1.0 − $1000 × 0.80 = $999 − $800 = +$199. New INITIAL health = −$15 + $199 = +$184 ≥ 0. MAINTENANCE check: $999 × 1.0 − $1000 × 0.90 = +$99 > 0 ✓.
4. Alice calls `withdrawCollateral` for $999 quote. INITIAL health check passes. Funds leave the protocol.
5. Without `burnNlp`, Alice could not have called `withdrawCollateral` at step 4 because her INITIAL health was −$15. [7](#0-6) [8](#0-7)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
