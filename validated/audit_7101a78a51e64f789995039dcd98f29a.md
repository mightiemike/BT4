Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is always `msg.sender` of the direct `pool.swap()` caller. When `MetricOmmSimpleRouter` is used, the pool sees the router as `sender`, not the end user. If the pool admin allowlists the router to permit allowlisted users to use the standard periphery, any unprivileged user can bypass the allowlist entirely by routing through the router with a single call.

## Finding Description
`MetricOmmPool.swap` captures `msg.sender` and passes it as `sender` to `_beforeSwap` (L231). `ExtensionCalling._beforeSwap` forwards this value unchanged to all configured extensions via `_callExtensionsInOrder` (L160–176). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]` (L37), where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. `MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap()` directly (L72–80), making the router the `sender` seen by the extension. There is no mechanism in the router to encode or forward the original end-user address. The admin faces an inescapable dilemma: not allowlisting the router blocks allowlisted users from using the standard periphery; allowlisting the router opens the gate to every user. `DepositAllowlistExtension` avoids this because it checks `owner` — a parameter that represents the position owner and is preserved correctly regardless of who the payer/caller is (L32–42).

## Impact Explanation
A pool admin deploying a curated pool (e.g., KYC-only, institutional-only) with `SwapAllowlistExtension` and who allowlists the router to support normal UX for approved users will have the allowlist completely neutralized. Any unprivileged address can execute swaps on the restricted pool via `router.exactInputSingle`. This is a direct admin-boundary break: an access-control invariant configured by the pool admin is bypassed by an unprivileged path through a supported periphery contract.

## Likelihood Explanation
The router is the canonical swap entry point. A pool admin who wants allowlisted users to have standard UX will allowlist the router. The bypass requires no special privileges, no flash loans, and no multi-transaction setup — a single `exactInputSingle` call from any address with token approval suffices. The condition is a natural and expected admin configuration.

## Recommendation
The extension must gate on the original end-user identity, not the intermediary. The most direct fix matching the `DepositAllowlistExtension` pattern: require the router to encode `msg.sender` into `extensionData` for every swap hop, and update `SwapAllowlistExtension.beforeSwap` to decode and check that address when `sender` is a known router. Alternatively, document and enforce that `SwapAllowlistExtension` is incompatible with router-mediated swaps by reverting when `sender` is not itself allowlisted, forcing allowlisted users to call the pool directly.

## Proof of Concept
1. Deploy pool with `SwapAllowlistExtension` configured (`beforeSwapOrder = 1`).
2. Admin calls `swapExtension.setAllowedToSwap(pool, alice, true)` — Alice is the only allowlisted swapper.
3. Admin calls `swapExtension.setAllowedToSwap(pool, router, true)` — required so Alice can use the router.
4. Add liquidity so the pool has depth.
5. Bob (not allowlisted) calls `router.exactInputSingle(ExactInputSingleParams({ pool: pool, recipient: bob, zeroForOne: true, amountIn: 1000, extensionData: "" }))`.
6. Router calls `pool.swap(bob, true, 1000, ...)` — `msg.sender` seen by pool = router.
7. Pool calls `_beforeSwap(sender=router, ...)`.
8. `SwapAllowlistExtension` checks `allowedSwapper[pool][router] == true` → passes.
9. Bob's swap executes on the allowlisted pool despite never being allowlisted.

Expected: revert `NotAllowedToSwap`. Actual: swap succeeds.
