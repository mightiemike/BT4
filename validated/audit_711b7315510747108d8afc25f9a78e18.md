### Title
Sequencer-Supplied `oraclePriceX18` Is Not Committed to in User's EIP-712 Signature for `BurnNlp`/`MintNlp`, Enabling Arbitrary NLP Price Manipulation — (`core/contracts/Verifier.sol`, `core/contracts/Clearinghouse.sol`, `core/contracts/interfaces/IEndpoint.sol`)

---

### Summary

The `oraclePriceX18` field embedded in `SignedBurnNlp` and `SignedMintNlp` is excluded from the EIP-712 digest that the user signs. Because the user's signature only commits to `(sender, nlpAmount, nonce)` or `(sender, quoteAmount, nonce)`, any party that constructs or relays the transaction — including a user submitting via slow mode — can freely set `oraclePriceX18` to an arbitrary value. `burnNlp` uses this price directly to compute the quote payout to the user, and `mintNlp` uses it to compute the NLP tokens minted. No on-chain bound or staleness check constrains the value.

---

### Finding Description

**Root cause — missing field in EIP-712 digest:**

In `Verifier.sol`, the `computeDigest` function for `BurnNlp` hashes only the inner `BurnNlp` struct fields:

```solidity
// BURN_NLP_SIGNATURE = "BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce)"
digest = keccak256(abi.encode(
    keccak256(bytes(BURN_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.nlpAmount,
    signedTx.tx.nonce
));
``` [1](#0-0) 

The `SignedBurnNlp` struct, however, carries two additional fields that are **not** signed:

```solidity
struct SignedBurnNlp {
    BurnNlp tx;
    bytes signature;
    int128 oraclePriceX18;       // ← not in digest
    int128[] nlpPoolRebalanceX18; // ← not in digest
}
``` [2](#0-1) 

The same omission applies to `SignedMintNlp`: [3](#0-2) [4](#0-3) 

**How the price is consumed:**

In `Clearinghouse.burnNlp`, `oraclePriceX18` is used to:
1. Overwrite the stored NLP price in the risk system.
2. Compute the quote payout: `quoteAmount = nlpAmount.mul(oraclePriceX18)`.

```solidity
spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);
// ...
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [5](#0-4) 

The attacker also controls `nlpPoolRebalanceX18`, which is validated only for internal consistency (sum must equal `-quoteAmount`), not for economic soundness: [6](#0-5) 

**Health check is insufficient:**

After the burn, the health check uses `MAINTENANCE` type (less strict than `INITIAL`). Because `spotEngine.updatePrice` was just called with the attacker-supplied inflated price, any remaining NLP balance is also valued at the inflated price, making the health check trivially pass: [7](#0-6) 

**Attacker-controlled entry path:**

`EndpointTx` processes `BurnNlp` transactions submitted via the sequencer path or via `submitSlowModeTransaction` (a public function). In either path, `validateSignedTx` verifies only the EIP-712 digest — which excludes `oraclePriceX18`. A user who submits a `BurnNlp` slow-mode transaction constructs the full `SignedBurnNlp` payload themselves, including `oraclePriceX18`: [8](#0-7) 

---

### Impact Explanation

An attacker holding any amount of NLP tokens can:

1. Sign a valid `BurnNlp(sender, nlpAmount, nonce)` — the signature is valid regardless of `oraclePriceX18`.
2. Construct `SignedBurnNlp` with `oraclePriceX18` set to an arbitrarily large value (e.g., 1000× market price) and `nlpPoolRebalanceX18` set to drain the NLP pool quote balances by the inflated amount.
3. Submit via slow mode (or have the sequencer include it).
4. Receive `nlpAmount × 1000 × marketPrice` in quote tokens instead of `nlpAmount × marketPrice`.
5. The NLP pool subaccounts are driven to deeply negative quote balances with no utilization guard (unlike `withdrawCollateral`, `burnNlp` does not call `assertUtilization`).

The corrupted state delta: NLP pool quote balances are drained by `(inflatedPrice - truePrice) × nlpAmount`, directly reducing the solvency of the NLP system and all LPs.

The symmetric attack on `mintNlp` (deflated price → attacker receives more NLP per quote deposited) also applies.

---

### Likelihood Explanation

Any user holding NLP tokens can trigger this via slow mode without any privileged access. The slow-mode path is specifically designed for censorship resistance and is reachable by any EOA. The only precondition is holding NLP tokens, which is a normal user action. The attack is deterministic and requires no external market conditions.

---

### Recommendation

Include `oraclePriceX18` (and `nlpPoolRebalanceX18` or its hash) in the EIP-712 type hash and digest for both `MintNlp` and `BurnNlp`:

```solidity
string internal constant BURN_NLP_SIGNATURE =
    "BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce,int128 oraclePriceX18)";
```

And in `computeDigest`:
```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(BURN_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.nlpAmount,
    signedTx.tx.nonce,
    signedTx.oraclePriceX18   // ← add this
));
```

This forces the user to commit to the price at signing time, preventing post-signing substitution. Additionally, consider adding an on-chain price bound check (e.g., `oraclePriceX18` must be within X% of the last accepted price) to limit sequencer-side manipulation.

### Proof of Concept

1. Alice holds 100 NLP tokens. True NLP price = 1 USDC. Alice's NLP is worth 100 USDC.
2. Alice signs `BurnNlp(sender=Alice, nlpAmount=100e18, nonce=N)`. The EIP-712 digest covers only these three fields.
3. Alice constructs `SignedBurnNlp` with `oraclePriceX18 = 1000e18` (1000 USDC) and `nlpPoolRebalanceX18 = [-100000e18]` (draining 100,000 USDC from the NLP pool).
4. Alice submits via `submitSlowModeTransaction`.
5. `validateSignedTx` passes — the signature is valid (it was signed over the correct digest).
6. `burnNlp` executes: `quoteAmount = 100e18 * 1000e18 / 1e18 = 100,000e18`. Alice receives 100,000 USDC.
7. The NLP pool's quote balance is reduced by 100,000 USDC (from e.g. 500,000 to 400,000).
8. Health check: Alice's quote balance is +100,000 USDC, NLP balance is 0. `getHealth` returns positive. Check passes.
9. Alice repeats with fresh NLP tokens (minted cheaply using the deflated-price `mintNlp` variant) until the NLP pools are empty.
