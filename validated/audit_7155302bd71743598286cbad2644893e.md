### Title
Hash Preimage Divergence for `AllResources` Transactions with Zero L2/L1DataGas Across P2P Protobuf Boundary — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently reclassifies an `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` produces a structurally different hash preimage for `L1Gas` (2 resource elements) versus `AllResources` (3 resource elements, including an explicit L1_DATA_GAS slot), the transaction hash computed after crossing the P2P boundary is different from the hash computed at submission time. This breaks the canonicalization invariant that every representation of the same transaction must hash to the same value.

---

### Finding Description

**Step 1 — Submission path always uses `AllResources`.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (never `ValidResourceBounds`). [1](#0-0) 

When converted to `InternalRpcInvokeTransactionV3`, the field is preserved as `AllResourceBounds`. [2](#0-1) 

The `InvokeTransactionV3Trait` implementation wraps it unconditionally as `ValidResourceBounds::AllResources`: [3](#0-2) 

The same wrapping occurs when converting to the executable `InvokeTransactionV3`: [4](#0-3) 

**Step 2 — Hash computation is variant-sensitive.**

`get_tip_resource_bounds_hash` includes L1_DATA_GAS in the Poseidon preimage only for `AllResources`, not for `L1Gas`: [5](#0-4) 

Concretely:
- `AllResources` → preimage = `[tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]` (4 felts)
- `L1Gas` → preimage = `[tip, L1_GAS_packed, L2_GAS_packed]` (3 felts)

These produce different Poseidon hashes even when `l2_gas = 0` and `l1_data_gas = 0`.

**Step 3 — P2P deserialization reclassifies the variant.**

When `InvokeTransactionV3` (which carries `ValidResourceBounds`) is serialized to protobuf and deserialized on a peer, the converter applies a zero-value heuristic: [6](#0-5) 

If `l1_data_gas.is_zero() && l2_gas.is_zero()`, the result is `ValidResourceBounds::L1Gas(l1_gas)` — even though the originating transaction was `AllResources`.

**Step 4 — The stateless validator accepts the triggering input.**

A transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` passes stateless validation: [7](#0-6) 

**Divergence summary:**

| Stage | Variant | Hash preimage length | Hash |
|---|---|---|---|
| Gateway / batcher | `AllResources` | 4 felts | H |
| Syncing peer (after protobuf round-trip) | `L1Gas` | 3 felts | H′ ≠ H |

---

### Impact Explanation

When a block containing such a transaction is propagated via P2P sync, the receiving node deserializes the transaction as `L1Gas` and recomputes a transaction hash H′ that does not match the committed hash H. The transaction commitment in the block header (which is derived from transaction hashes) therefore cannot be verified, causing the receiving node to reject a legitimately produced block. This constitutes:

- **Wrong hash bound to the transaction** — matching "Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" (High).
- Potential **wrong state / receipt** if the receiving node stores H′ instead of H (Critical boundary).

---

### Likelihood Explanation

The trigger requires only a standard V3 invoke transaction with non-zero `l1_gas` and zero `l2_gas`/`l1_data_gas`. This is explicitly tested as a valid input by the gateway's own stateless validator test suite. No privileged access, no malformed bytes, and no special network position are required — any user can submit such a transaction.

---

### Recommendation

1. **Remove the zero-value heuristic** from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. The variant (`L1Gas` vs `AllResources`) should be determined by the transaction version or an explicit tag, not by whether the numeric values happen to be zero.

2. **Alternatively**, make `get_tip_resource_bounds_hash` version-gated rather than variant-gated, so that all V3 transactions (regardless of how `ValidResourceBounds` was reconstructed) always include the L1_DATA_GAS slot in the preimage.

3. Add a round-trip hash consistency test: serialize an `AllResources` transaction with zero `l2_gas`/`l1_data_gas` to protobuf, deserialize it, recompute the hash, and assert it equals the original hash.

---

### Proof of Concept

```
1. Craft RpcInvokeTransactionV3 with:
       resource_bounds = AllResourceBounds {
           l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      ResourceBounds::default(),   // zero
           l1_data_gas: ResourceBounds::default(),   // zero
       }

2. Gateway accepts it (stateless validator passes).
   InternalRpcInvokeTransactionV3::resource_bounds() returns AllResources.
   get_tip_resource_bounds_hash → 4-felt preimage → hash H.
   Transaction stored in block with hash H.

3. Block is propagated to a syncing peer via P2P protobuf.

4. Peer deserializes InvokeTransactionV3:
       TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
           l1_data_gas.is_zero() && l2_gas.is_zero()  →  L1Gas(l1_gas)

5. Peer recomputes transaction hash:
   get_tip_resource_bounds_hash with L1Gas → 3-felt preimage → hash H′.

6. H ≠ H′.
   Block transaction commitment verification fails.
   Peer rejects the block.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
