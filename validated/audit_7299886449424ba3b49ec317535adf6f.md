### Title
Interest Rate Config Update Without Prior State Settlement Causes Retroactive Rate Application - (File: `core/contracts/SpotEngine.sol`)

---

### Summary

`SpotEngine.addOrUpdateProduct()` overwrites the interest rate configuration (`configs[productId]`) without first calling `_updateState()` to settle accumulated interest under the old parameters. The next sequencer-triggered `updateStates(dt)` call then applies the new rates retroactively across the entire elapsed `dt` window, corrupting borrow and deposit interest accrual for all users of that product.

---

### Finding Description

`SpotEngineState._updateState()` computes the borrow rate multiplier and deposit rate multiplier using `configs[productId]` — specifically `interestFloorX18`, `interestSmallCapX18`, `interestLargeCapX18`, `interestInflectionUtilX18`, and `minDepositRateX18`: [1](#0-0) 

These multipliers are accumulated into `state.cumulativeBorrowsMultiplierX18` and `state.cumulativeDepositsMultiplierX18`: [2](#0-1) 

`updateStates(uint128 dt)` is the only function that calls `_updateState()`, and it is `onlyEndpoint` — meaning only the sequencer can invoke it: [3](#0-2) 

`addOrUpdateProduct()` in `SpotEngine.sol` directly overwrites `configs[productId]` with no prior settlement call: [4](#0-3) 

The critical line is: [5](#0-4) 

Because `updateStates` is `onlyEndpoint`, the admin has no mechanism to settle the state before updating the config even if they wanted to. The next sequencer `updateStates(dt)` call will compute interest for the full `dt` window — which spans time both before and after the config change — entirely using the new config parameters.

---

### Impact Explanation

When the admin updates interest rate parameters for a spot product (e.g., raises `interestFloorX18` or `interestLargeCapX18`), the next `updateStates(dt)` call applies the new rates to the entire elapsed period since the last update. This corrupts `cumulativeBorrowsMultiplierX18` and `cumulativeDepositsMultiplierX18`, which are the basis for all user balance calculations via `_updateBalanceNormalized`. Borrowers are charged incorrect interest and depositors receive incorrect yield for the pre-change window. The magnitude scales with the size of the rate change and the elapsed `dt`. [6](#0-5) 

---

### Likelihood Explanation

Any legitimate admin invocation of `addOrUpdateProduct()` to tune interest rate parameters triggers this issue. Since `updateStates` is called by the sequencer on a regular cadence (bounded by `dt < 7 * SECONDS_PER_DAY`), there will always be a non-zero elapsed window between the config change and the next settlement. This is not a theoretical edge case — it fires on every config update.

---

### Recommendation

Before overwriting `configs[productId]`, `addOrUpdateProduct()` must settle the current state. Since `_updateState()` requires a `dt` value and `updateStates` is `onlyEndpoint`, the recommended fix is to expose an internal settlement path or require the sequencer to submit an `updateStates` transaction immediately before any config change is applied. Alternatively, record the timestamp of the last settlement and compute `dt` inline within `addOrUpdateProduct()` using `block.timestamp`, then call `_updateState()` and `_setState()` before writing the new config.

---

### Proof of Concept

1. At time `T0`, sequencer calls `updateStates(dt)` — state is settled with old config `C_old` (e.g., `interestFloorX18 = 1e16`).
2. At time `T1 = T0 + 3600` (1 hour later), admin calls `addOrUpdateProduct(productId, ..., newConfig)` where `newConfig.interestFloorX18 = 5e16` (5× increase). `configs[productId]` is overwritten immediately. No settlement occurs.
3. At time `T2 = T1 + 3600` (1 hour after config change), sequencer calls `updateStates(dt)` with `dt = T2 - T0 = 7200` seconds.
4. `_updateState()` reads `configs[productId]` — now `C_new` — and computes the borrow rate multiplier for the full 7200-second window using the elevated rate.
5. The 3600-second window `[T0, T1]` that accrued under `C_old` is retroactively repriced at `C_new`, inflating `cumulativeBorrowsMultiplierX18` beyond the correct value.
6. All borrowers' effective debt (computed as `amountNormalized * cumulativeBorrowsMultiplierX18`) is overstated for that window; depositors' yield is correspondingly distorted. [7](#0-6) [8](#0-7)

    function _updateState(
        uint32 productId,
        State memory state,
        uint128 dt
    ) internal {
        int128 borrowRateMultiplierX18;
        int128 totalDeposits = state.totalDepositsNormalized.mul(
            state.cumulativeDepositsMultiplierX18
        );
        int128 totalBorrows = state.totalBorrowsNormalized.mul(
            state.cumulativeBorrowsMultiplierX18
        );
        int128 utilizationRatioX18 = totalBorrows.div(totalDeposits);
        int128 minDepositRateX18;
        {
            Config memory config = configs[productId];

            // annualized borrower rate
            int128 borrowerRateX18 = config.interestFloorX18;
            if (utilizationRatioX18 == 0) {
                // setting borrowerRateX18 to 0 here has the property that
                // adding a product at the beginning of time and not using it until time T
                // results in the same state as adding the product at time T
                borrowerRateX18 = 0;
            } else if (utilizationRatioX18 < config.interestInflectionUtilX18) {
                borrowerRateX18 += config
                    .interestSmallCapX18
                    .mul(utilizationRatioX18)
                    .div(config.interestInflectionUtilX18);
            } else {
                borrowerRateX18 +=
                    config.interestSmallCapX18 +
                    config.interestLargeCapX18.mul(
                        (
                            (utilizationRatioX18 -
                                config.interestInflectionUtilX18).div(
                                    ONE - config.interestInflectionUtilX18
                                )
                        )
                    );
            }

            // convert to per second
            borrowerRateX18 = borrowerRateX18.div(
                MathSD21x18.fromInt(31536000)
            );
            borrowRateMultiplierX18 = (ONE + borrowerRateX18).pow(int128(dt));
            minDepositRateX18 = config.minDepositRateX18;
        }

    function updateStates(uint128 dt) external onlyEndpoint {
        State memory quoteState;
        require(dt < 7 * SECONDS_PER_DAY, ERR_INVALID_TIME);
        for (uint32 i = 0; i < productIds.length; i++) {
            uint32 productId = productIds[i];
            if (productId == NLP_PRODUCT_ID) {
                continue;
            }
            State memory state = states[productId];
            if (productId == QUOTE_PRODUCT_ID) {
                quoteState = state;
            }
            if (state.totalDepositsNormalized == 0) {
                continue;
            }
            _updateState(productId, state, dt);
            _setState(productId, state);
        }
    }
