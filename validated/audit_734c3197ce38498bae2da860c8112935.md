### Title
`ValidResourceBounds` protobuf round-trip silently coerces `AllResources(zero_l2, zero_l1data)` to `L1Gas`, producing a divergent transaction hash preimage — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter uses a zero-value heuristic to reconstruct the `ValidResourceBounds` variant. It cannot distinguish a genuine `L1Gas` transaction from an `AllResources` transaction whose l2_gas and l1_data_gas bounds are both zero. After a protobuf round-trip the latter is silently coerced to `L1Gas`, changing the hash preimage and producing a different transaction hash on the receiving node.

---

### Finding Description

**Serialization path** (`From<ValidResourceBounds> for protobuf::ResourceBounds`): [1](#0-0) 

An `AllResources` value whose `l2_gas` and `l1_data_gas` are both zero is serialized identically to an `L1Gas` value: all three protobuf fields are present but the last two carry zero `ResourceLimits`.

**Deserialization path** (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`): [2](#0-1) 

The only discriminator is `l1_data_gas.is_zero() && l2_gas.is_zero()`. When both are zero the decoder unconditionally returns `ValidResourceBounds::L1Gas`, even if the sender originally held `AllResources`.

**Hash divergence** — `get_tip_resource_bounds_hash` produces structurally different Poseidon inputs for the two variants: [3](#0-2) 

- `L1Gas` → Poseidon hashes **2** resource felts: `[tip, l1_gas_felt, l2_gas_felt]`
- `AllResources` (even with zero l2/l1_data) → Poseidon hashes **3** resource felts: `[tip, l1_gas_felt, l2_gas_felt, l1_data_gas_felt]`

These produce different digests, so the transaction hash computed after deserialization (H2) differs from the hash computed at ingestion (H1).

**Trigger path**:

1. A user submits a V3 invoke with `AllResourceBounds { l1_gas: non_zero, l2_gas: zero, l1_data_gas: zero }`. The gateway accepts it because `RpcInvokeTransactionV3` carries `AllResourceBounds` directly and no validation rejects zero l2/l1_data bounds. [4](#0-3) 

2. The gateway converter calls `get_invoke_transaction_v3_hash` via `InvokeTransactionV3Trait::resource_bounds()` which returns `ValidResourceBounds::AllResources(...)`, producing H1 with 3 resource felts. [5](#0-4) 

3. The transaction is stored as `InvokeTransactionV3 { resource_bounds: ValidResourceBounds::AllResources(...) }` and propagated via P2P state sync.

4. The receiving node deserializes the protobuf `ResourceBounds` message. Because `l2_gas` and `l1_data_gas` are zero, the decoder returns `ValidResourceBounds::L1Gas(l1_gas)`.

5. Any subsequent hash recomputation on the receiving node (RPC simulation, fee estimation, re-execution, `validate_transaction_hash`) calls `get_tip_resource_bounds_hash` with `L1Gas`, producing H2 ≠ H1. [6](#0-5) 

Additionally, the coercion changes `get_gas_vector_computation_mode()` from `All` to `NoL2Gas`: [7](#0-6) 

This alters fee pre-validation logic on the syncing node, potentially accepting or rejecting the transaction differently than the sequencer did.

---

### Impact Explanation

**High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

A syncing node that recomputes the transaction hash from the deserialized body returns H2 ≠ H1 for any RPC call that re-derives the hash (e.g., `starknet_simulateTransactions`, `starknet_estimateFee`, re-execution in `blockifier_reexecution`). The node presents this wrong hash as authoritative. Additionally, the changed `GasVectorComputationMode` causes different fee-bound validation behavior between the sequencer and the syncing node for the same transaction.

---

### Likelihood Explanation

Any user can submit a V3 transaction with zero l2_gas and l1_data_gas bounds. The gateway stateless validator does not reject zero bounds for individual resources: [8](#0-7) 

The condition is therefore reachable by any unprivileged sender with a valid account.

---

### Recommendation

Add an explicit variant discriminator to the protobuf `ResourceBounds` message (e.g., a boolean `is_l1_gas_only` field), so the decoder does not rely on zero-value inference. Alternatively, mandate that all V3 transactions always serialize as `AllResources` in the P2P path and reject `L1Gas` deserialization for V3 transaction types, since `InternalRpcInvokeTransactionV3` and `InternalRpcDeclareTransactionV3` already unconditionally wrap their bounds in `ValidResourceBounds::AllResources`: [5](#0-4) [9](#0-8) 

---

### Proof of Concept

```
1. Submit RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway computes H1:
     get_tip_resource_bounds_hash(AllResources(...), tip)
     = Poseidon(tip_felt, l1_gas_felt, l2_gas_felt=0, l1_data_gas_felt=0)
     → 4-element Poseidon input

3. Transaction stored as InvokeTransactionV3 {
     resource_bounds: ValidResourceBounds::AllResources(...)
   }

4. P2P state-sync serializes to protobuf::ResourceBounds {
     l1_gas:      Some({ max_amount: 1000, max_price_per_unit: 1 }),
     l2_gas:      Some({ max_amount: 0,    max_price_per_unit: 0 }),
     l1_data_gas: Some({ max_amount: 0,    max_price_per_unit: 0 }),
   }

5. Syncing node deserializes:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas(l1_gas)   ← WRONG VARIANT

6. Syncing node recomputes hash:
     get_tip_resource_bounds_hash(L1Gas(...), tip)
     = Poseidon(tip_felt, l1_gas_felt, l2_gas_felt=0)
     → 3-element Poseidon input → H2 ≠ H1

7. starknet_simulateTransactions / starknet_estimateFee on syncing node
   uses H2 and GasVectorComputationMode::NoL2Gas instead of H1 and All,
   returning an authoritative-looking wrong value.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
