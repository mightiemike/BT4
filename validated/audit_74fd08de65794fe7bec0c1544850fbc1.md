### Title
`slashRedelegationPosition` Advances Bonus Checkpoints Without Paying Bonus When Pool Is Insufficient, Permanently Losing Accrued Rewards - (`File: x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is intentionally swallowed to prevent a chain halt. However, `processEventsAndClaimBonus` mutates the position's reward checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) **before** it checks the pool balance. The partially-mutated position is then persisted via `setPositionWithState`. This permanently skips the accrued bonus for the entire period up to the slash block, mirroring the `previouslyVested` update bug in the reference report.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` performs all checkpoint mutations in-memory on the `*PositionState` pointer before it checks whether the pool can cover the computed bonus:

```
// claim_rewards.go
for _, entry := range events {
    ...
    pos.UpdateLastEventSeq(entry.Seq)          // line 193 — checkpoint advanced
    k.decrementEventRefCount(ctx, valAddr, ...) // line 196 — store mutation
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime) // line 215 — LastBonusAccrual = blockTime
pos.UpdateLastKnownBonded(bonded)                     // line 217 — bonded state updated

if totalBonus.IsZero() { return sdk.NewCoins(), nil }

bonusCoins := sdk.NewCoins(...)
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // line 230 — error returned AFTER all mutations
}
```

In `slashRedelegationPosition`, the `ErrInsufficientBonusPool` error is swallowed:

```
// slash.go lines 54-64
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool ...")
        // error swallowed — pos already mutated
    } else {
        return err
    }
}
```

For a **partial** redelegation slash (`sharesToUnbond < pos.Delegation.Shares`), the code falls through to:

```
// slash.go lines 76-77
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)
```

`pos` now carries the advanced `LastBonusAccrual` (set to `blockTime`) and the advanced `LastEventSeq`, but zero bonus was paid. The next call to `processEventsAndClaimBonus` for this position will start its segment computation from `blockTime`, permanently skipping the entire accrued period.

For a **full** slash, `pos.ClearBonusCheckpoints()` is called at line 70, which resets the checkpoints — so the lost bonus is irrelevant since the position is effectively terminated. The vulnerability only manifests on partial slashes.

---

### Impact Explanation

The corrupted values are `pos.LastBonusAccrual` and `pos.LastEventSeq` stored in the `Position` proto. After the erroneous persist:

- The position owner can never recover the bonus that accrued from `pos.LastBonusAccrual` (before the slash) up to `blockTime` (the slash block). That time window is permanently skipped.
- Event reference counts are also decremented in the store (line 196) even though the bonus was not paid, which can cause validator events to be garbage-collected before other positions have processed them — though each position has its own ref-count slot, so this secondary effect is limited to the affected position's events.

The direct impact is **permanent loss of accrued bonus rewards** for the position owner, with no recourse. The amount lost equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the skipped period.

---

### Likelihood Explanation

The trigger requires two concurrent conditions:

1. **A redelegation slash fires** — `BeforeRedelegationSlashed` is invoked whenever a validator is slashed (double-sign evidence or downtime) while an active redelegation from that validator exists. This is a normal protocol event requiring no privileged actor; any validator misbehaviour or downtime triggers it.
2. **The bonus pool is insufficient** — The `RewardsPoolName` module account can be drained by normal bonus claims from other positions, or it may simply not have been replenished. An empty or low pool is a realistic operational state, especially early in the chain's life or after a large batch of claims.

Both conditions can occur independently and simultaneously without any attacker action. A position owner who redelegated from a validator that later gets slashed while the pool is low will silently lose their accrued bonus.

---

### Recommendation

Move all checkpoint mutations (`applyBonusAccrualCheckpoint`, `UpdateLastKnownBonded`, `UpdateLastEventSeq`) to **after** the successful pool transfer, or — for the slash hook path specifically — avoid persisting the mutated `pos` when the pool check failed. One approach:

In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, reload a fresh copy of the position (or snapshot the pre-call state) before calling `setPositionWithState`, so that the checkpoints are not advanced when no bonus was paid:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)
        // Re-load pos to discard in-memory checkpoint mutations
        pos, err = k.getPositionState(ctx, positionId)
        if err != nil { return err }
    } else {
        return err
    }
}
```

Alternatively, restructure `processEventsAndClaimBonus` to perform the pool check before mutating any checkpoint fields, so that a failed pool check leaves `pos` unmodified.

---

### Proof of Concept

1. Alice creates a tier position and calls `MsgTierRedelegate` to move her delegation from validator V1 to V2. A `RedelegationMapping` entry is created.
2. The bonus pool is drained to near-zero by other users claiming rewards.
3. V1 is slashed (double-sign evidence submitted on-chain). The Cosmos SDK staking module calls `SlashRedelegation`, which fires `BeforeRedelegationSlashed`.
4. `slashRedelegationPosition` is invoked. `processEventsAndClaimBonus` computes Alice's accrued bonus (e.g., 30 days of accrual), advances `LastBonusAccrual` to `blockTime` and `LastEventSeq` to the latest event, then returns `ErrInsufficientBonusPool`.
5. The error is swallowed. Since `sharesToUnbond < pos.Delegation.Shares` (partial slash), `setPositionWithState` is called with the mutated `pos`.
6. Alice's `LastBonusAccrual` is now `blockTime`. Her next `MsgClaimTierRewards` computes bonus only from `blockTime` forward — the 30-day accrual is gone.

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
