Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User, Allowing Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which is `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, `sender` equals the router address, not the end-user. Any non-allowlisted user can bypass the per-user curation policy by routing through `MetricOmmSimpleRouter` if the router itself is allowlisted — the only configuration that permits any router-mediated swap.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever was passed — the router address when called via the router: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router itself `msg.sender` of that call: [3](#0-2) 

The same holds for `exactInput` (all hops): [4](#0-3) 

And for `exactOutputSingle` and `exactOutput` recursive hops via `_exactOutputIterateCallback`: [5](#0-4) 

The extension always evaluates `allowedSwapper[pool][router]`, never `allowedSwapper[pool][actualUser]`. The pool admin faces an impossible choice: if the router is not allowlisted, no user can swap through it even if individually allowlisted; if the router is allowlisted, every user can swap through it regardless of individual allowlist status.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly checks the `owner` parameter (the position beneficiary explicitly passed by the liquidity adder), not the `sender`: [6](#0-5) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to KYC'd or otherwise curated addresses is completely open to any user routing through `MetricOmmSimpleRouter`. Non-allowlisted addresses execute live swaps at oracle prices against LP capital, receiving real token output and paying real token input. This is a direct, unprivileged bypass of an access-control guard with fund-flow consequences — unauthorized trades execute against LP capital, defeating the pool admin's curation policy.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard periphery for swapping. A pool admin who deploys `SwapAllowlistExtension` and wants any user to swap through the router must add the router to the allowlist — the natural and expected configuration. Once the router is allowlisted, the bypass is trivially reachable by any address with a standard `exactInputSingle` or `exactInput` call, requiring no special privileges or setup.

## Recommendation
The `beforeSwap` hook must receive the original end-user's address. The cleanest fix is for the router to encode `msg.sender` (the actual user) into `extensionData` before calling `pool.swap()`, and for `SwapAllowlistExtension` to decode and check that address instead of the `sender` (router) parameter. Alternatively, the pool's `_beforeSwap` hook signature could be extended to carry an explicit originator field, but that requires a core change.

## Proof of Concept
```solidity
// Pool deployed with SwapAllowlistExtension as beforeSwap extension.
// Pool admin allowlists alice and the router (required for any router swap).
extension.setAllowedToSwap(address(pool), alice, true);
extension.setAllowedToSwap(address(pool), address(router), true);

// charlie is NOT on the allowlist.
vm.prank(charlie);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token1),
        recipient: charlie,
        deadline: block.timestamp + 1,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: false,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// Succeeds: pool calls extension.beforeSwap(router, charlie, ...).
// Extension evaluates allowedSwapper[pool][router] → true and passes.
// charlie's address is never inspected; charlie receives token0 output.
```
