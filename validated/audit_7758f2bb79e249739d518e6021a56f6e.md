### Title
Missing Liquidator Health Check in Finalization Path Allows Unbounded Negative Quote Balance — (`core/contracts/ClearinghouseLiq.sol`)

---

### Summary

`liquidateSubaccountImpl` has two execution branches. The normal liquidation branch terminates in `_handleLiquidationPayment`, which enforces a post-action health check on the liquidator. The finalization branch (`txn.productId == type(uint32).max`) terminates in `_finalizeSubaccount`, which settles all positive perp PnL against the liquidator but **never checks the liquidator's health afterward**. A liquidator with minimal quote balance can finalize a subaccount carrying large positive `vQuoteBalance` across many perp products, driving the liquidator's QUOTE balance arbitrarily negative with no protocol guard.

---

### Finding Description

**Entry point:** `liquidateSubaccountImpl` with `txn.productId = type(uint32).max`.

**Branch split:**

In `liquidateSubaccountImpl`, after the initial guards (lines 601–611), the function calls `_finalizeSubaccount`. If it returns `true`, the function returns immediately: [1](#0-0) 

The normal (non-finalization) path continues to `_handleLiquidationPayment`, which contains the only liquidator health check in the contract: [2](#0-1) 

This check is **never reached** when `_finalizeSubaccount` returns `true`.

**PnL settlement inside `_finalizeSubaccount`:**

The function loops over all perp products and, for each with `balance.vQuoteBalance > 0`, calls `_settlePnlAgainstLiquidator`: [3](#0-2) 

`_settlePnlAgainstLiquidator` unconditionally debits the full PnL from the liquidator's QUOTE balance: [4](#0-3) 

There is no floor, no cap, and no health check on `txn.sender` anywhere inside `_finalizeSubaccount`. [5](#0-4) 

---

### Impact Explanation

After the loop completes, the liquidator's QUOTE balance can be an arbitrarily large negative number. The liquidator is now insolvent. Because `spotEngine.updateBalance` accepts negative values without a floor, the negative balance is recorded as a real liability in the spot engine. When the liquidator is subsequently liquidated, the insurance fund must cover the shortfall; if the fund is insufficient, `socializeSubaccount` distributes the loss across all depositors. This is direct bad-debt creation and loss socialization — a Critical-scope impact.

---

### Likelihood Explanation

The path is reachable by any non-isolated subaccount that submits a signed `LiquidateSubaccount` order with `productId = type(uint32).max` against a liquidatee whose perp positions are all closed but whose `vQuoteBalance` is positive. This is a normal end-state for a liquidatee that has been progressively liquidated. No admin access, no governance capture, and no special privileges are required. The liquidator can be a freshly created subaccount with near-zero quote.

---

### Recommendation

After `_finalizeSubaccount` returns `true`, apply the same liquidator health check that exists in `_handleLiquidationPayment`:

```solidity
if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {
    require(
        txn.sender == N_ACCOUNT || !isUnderInitial(txn.sender),
        ERR_SUBACCT_HEALTH
    );
    if (RiskHelper.isIsolatedSubaccount(txn.liquidatee)) { ... }
    return;
}
```

Alternatively, inside `_finalizeSubaccount`, before settling positive PnL, verify that the liquidator holds sufficient QUOTE to absorb the total positive `vQuoteBalance` being transferred.

---

### Proof of Concept

1. Create liquidatee with open positions across N perp products; let each accumulate a large positive `vQuoteBalance` (e.g., 100,000 USDC each across 10 products = 1,000,000 USDC total PnL). Close all perp `amount` positions so `balance.amount == 0` for each, satisfying the `ERR_NOT_FINALIZABLE_SUBACCOUNT` checks at lines 313–320. Ensure liquidatee's maintenance health is negative (e.g., via a large negative QUOTE balance).
2. Create liquidator with 1 USDC quote balance.
3. Submit `LiquidateSubaccount` with `productId = type(uint32).max`, `sender = liquidator`, `liquidatee = liquidatee`.
4. `_finalizeSubaccount` runs the positive-PnL loop (lines 322–338), calling `_settlePnlAgainstLiquidator` 10 times, each time subtracting 100,000 from the liquidator's QUOTE.
5. Liquidator's QUOTE balance is now `1 - 1,000,000 = -999,999 USDC`.
6. `liquidateSubaccountImpl` returns at line 626 — no health check fires.
7. Assert `getHealth(liquidator, INITIAL) < 0` — invariant broken, bad debt created.

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }
