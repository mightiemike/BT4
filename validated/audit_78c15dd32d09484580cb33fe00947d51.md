Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as Swapper Instead of Actual User, Enabling Complete Allowlist Bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates access using the `sender` argument forwarded by the pool, which is the pool's own `msg.sender` — the direct caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the router is that direct caller. A pool admin who allowlists the router to enable standard-periphery access inadvertently grants unrestricted swap access to every address on-chain, completely voiding the allowlist.

## Finding Description

**Root cause — wrong actor bound in the extension check:**

`SwapAllowlistExtension.beforeSwap()` evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the extension is called by the pool) and `sender` is the first argument the pool passes — which is the pool's own `msg.sender` at the time `swap()` was called.

**How the pool populates `sender`:**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, passing its own `msg.sender` as the `sender` argument. `ExtensionCalling._beforeSwap()` then encodes this value verbatim into the `abi.encodeCall` forwarded to the extension.

**How the router calls the pool:**

`MetricOmmSimpleRouter.exactInputSingle()` calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` directly. The router is therefore `msg.sender` of that pool call. The pool forwards `msg.sender` (= router) as `sender` to every `beforeSwap` hook.

**Full call chain:**

```
User → router.exactInputSingle()
     → pool.swap()          [router is msg.sender]
     → _beforeSwap(msg.sender=router, ...)
     → ExtensionCalling encodes sender=router
     → extension.beforeSwap(sender=router, ...)
     → checks allowedSwapper[pool][router]   ← user identity lost
```

**The two broken states:**

| Pool admin action | Consequence |
|---|---|
| Allowlists the router to support standard periphery | Every unprivileged user can swap by routing through the router — allowlist is completely void |
| Does not allowlist the router | Individually allowlisted users cannot use the router at all — standard periphery is broken for the curated pool |

## Impact Explanation

A curated pool (KYC-gated, institutional, or partner-only) that deploys `SwapAllowlistExtension` and allowlists `MetricOmmSimpleRouter` to support normal user flows inadvertently opens swap access to every address on-chain. Any actor — including sanctioned addresses, competitors, or bots — can call `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on the router and successfully execute swaps on the restricted pool. This is a confirmed admin-boundary break: an unprivileged path (`MetricOmmSimpleRouter`) bypasses the pool admin's configured access control.

## Likelihood Explanation

High. `MetricOmmSimpleRouter` is the protocol's own production periphery. Any pool admin who wants their curated pool to be usable by end users through the standard interface must allowlist the router. The wrong-actor binding is invisible from the admin's perspective — `setAllowedToSwap(pool, router, true)` reads as "allow router-mediated swaps," but it silently means "allow all swaps." No special attacker capability is required beyond calling the public router.

## Recommendation

The extension must gate the economic actor (the end user), not the intermediary. The cleanest fix is to have the router encode `msg.sender` into `extensionData` and have the extension decode and verify it. This requires a documented encoding convention so pool admins know which router versions are compatible with the allowlist extension. Alternatively, a purpose-built router that appends a verified user identity to `extensionData` can be required for allowlisted pools, rather than the generic `MetricOmmSimpleRouter`.

## Proof of Concept

```
Setup:
  - Pool deployed with SwapAllowlistExtension as beforeSwap hook
  - Pool admin calls: setAllowedToSwap(pool, address(router), true)
    (intending to enable standard-periphery access for their allowlisted users)
  - Attacker address is NOT individually allowlisted

Attack:
  1. Attacker calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, ...) — router is msg.sender
  3. Pool calls _beforeSwap(msg.sender=router, ...)
  4. ExtensionCalling encodes sender=router and calls extension.beforeSwap(router, ...)
  5. Extension evaluates: allowedSwapper[pool][router] == true  ✓
  6. Swap executes — attacker trades on the curated pool with zero allowlist friction

Result: The allowlist is completely bypassed. Any user can swap on the
        restricted pool by routing through MetricOmmSimpleRouter.
```
