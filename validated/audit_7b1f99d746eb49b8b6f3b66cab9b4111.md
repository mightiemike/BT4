### Title
Gateway `validate_resource_bounds` Unconditionally Checks L2 Gas Price Against `min_gas_price`, Rejecting Valid Transactions With Only L1/L1-Data Gas Bounds — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` first accepts a transaction whose combined `max_possible_fee` is non-zero (i.e., at least one of L1 gas, L2 gas, or L1-data gas is non-zero), then unconditionally checks `l2_gas.max_price_per_unit` against the configured `min_gas_price`. In production `min_gas_price = 8_000_000_000`. A transaction that carries only L1 gas or L1-data gas bounds (with `l2_gas.max_price_per_unit = 0`) passes the first guard but is always rejected by the second, even though it is a structurally valid, fee-paying transaction.

### Finding Description

`validate_resource_bounds` in the stateless validator runs two sequential checks:

```rust
// Check 1 – passes if ANY resource bound is non-zero
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
}

// Check 2 – unconditionally tests L2 gas price alone
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
        gas_price: resource_bounds.l2_gas.max_price_per_unit,
        min_gas_price: self.config.min_gas_price,
    });
}
``` [1](#0-0) 

Check 1 uses `max_possible_fee`, which sums all three resource components:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    l1_gas.max_amount.saturating_mul(l1_gas.max_price_per_unit)
    .saturating_add(l2_gas.max_amount.saturating_mul(...))
    .saturating_add(l1_data_gas.max_amount.saturating_mul(...))
``` [2](#0-1) 

So a transaction with `l1_gas = {max_amount: 1, max_price_per_unit: 1}` and `l2_gas = Default::default()` has `max_possible_fee = 1 > 0` and passes Check 1. But `l2_gas.max_price_per_unit = 0 < 8_000_000_000`, so Check 2 rejects it.

The production node config confirms `min_gas_price = 8_000_000_000`: [3](#0-2) 

The existing test suite explicitly validates that a transaction with only L1 gas bounds should be accepted when `validate_resource_bounds: true`, but it uses `min_gas_price: 0` (the test-only default), masking the production failure: [4](#0-3) [5](#0-4) 

The same single-component check is repeated in the stateful validator's `validate_tx_l2_gas_price_within_threshold`, which also inspects only `l2_gas.max_price_per_unit` for every `AllResources` transaction regardless of whether L2 gas bounds are zero: [6](#0-5) 

### Impact Explanation

Any transaction whose fee is carried entirely in L1 gas or L1-data gas (with `l2_gas.max_price_per_unit = 0`) is permanently rejected at the gateway admission layer. The blockifier's own `enforce_fee` logic correctly treats such a transaction as fee-paying (because `max_possible_fee > 0`), so the rejection is inconsistent with the execution layer's semantics. This maps to the **High** impact category: *mempool/gateway admission rejects valid transactions before sequencing*.

### Likelihood Explanation

The `AllResourceBounds` struct allows any combination of zero and non-zero resource fields. The `ZeroResourceBounds` guard explicitly permits transactions with only one non-zero resource. Any client that constructs a V3 transaction with non-zero L1 gas but omits L2 gas (e.g., a legacy-style transaction migrated to the V3 format, or a transaction that intentionally sets only L1 bounds) will be silently rejected in production while passing in test environments. The discrepancy between the test config (`min_gas_price: 0`) and the production config (`min_gas_price: 8_000_000_000`) means this path is never exercised by the existing test suite under production parameters.

### Recommendation

Guard the `min_gas_price` check with a condition that only applies it when L2 gas bounds are actually non-zero:

```rust
if resource_bounds.l2_gas.max_amount > GasAmount(0)
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Apply the same fix to `validate_tx_l2_gas_price_within_threshold` in the stateful validator. Add a test case that exercises `valid_l1_gas` with the production `min_gas_price` value to prevent regression.

### Proof of Concept

1. Configure the gateway with production defaults (`min_gas_price = 8_000_000_000`, `validate_resource_bounds = true`).
2. Submit a V3 invoke transaction with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1_000, max_price_per_unit: 10_000_000_000 },
       l2_gas:      { max_amount: 0,     max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,     max_price_per_unit: 0 },
   }
   ```
3. **Check 1** (`ZeroResourceBounds`): `max_possible_fee = 1_000 * 10_000_000_000 = 10^13 > 0` → passes.
4. **Check 2** (`MaxGasPriceTooLow`): `0 < 8_000_000_000` → **rejected** with `MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 }`.
5. The blockifier's `enforce_fee` for the same transaction returns `true` (fee is enforced), confirming the transaction is valid at the execution layer but blocked at the gateway. [7](#0-6)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
