### Title
`get_protocol_config` Fails to Propagate `num_chunk_producer_seats`, `num_chunk_validator_seats`, and `chunk_producer_assignment_changes_limit` from `EpochConfig` to `GenesisConfig` — (`File: chain/chain/src/runtime/mod.rs`)

---

### Summary

`NightshadeRuntime::get_protocol_config` manually copies a subset of `EpochConfig` fields into a cloned `GenesisConfig`, but omits three fields that have changed across protocol upgrades: `num_chunk_producer_seats`, `num_chunk_validator_seats`, and `chunk_producer_assignment_changes_limit`. The `EXPERIMENTAL_protocol_config` RPC endpoint therefore returns stale genesis-time values for these fields regardless of the current epoch's actual configuration.

---

### Finding Description

`get_protocol_config` in `chain/chain/src/runtime/mod.rs` constructs a `ProtocolConfig` by cloning `self.genesis_config` and then overwriting it field-by-field with values from the live `EpochConfig`: [1](#0-0) 

The function copies 17 fields from `epoch_config` to `genesis_config`, but three fields present in both `EpochConfig` and `GenesisConfig` are never written:

- `epoch_config.num_chunk_producer_seats` → `genesis_config.num_chunk_producer_seats` — **missing**
- `epoch_config.num_chunk_validator_seats` → `genesis_config.num_chunk_validator_seats` — **missing**
- `epoch_config.chunk_producer_assignment_changes_limit` → `genesis_config.chunk_producer_assignment_changes_limit` — **missing**

`EpochConfig` defines all three fields: [2](#0-1) 

`GenesisConfig` defines the same three fields: [3](#0-2) 

The `ProtocolConfig` produced by `get_protocol_config` is converted directly to `ProtocolConfigView` for the public `EXPERIMENTAL_protocol_config` RPC endpoint: [4](#0-3) 

`ProtocolConfigView` does not include `num_chunk_producer_seats`, `num_chunk_validator_seats`, or `chunk_producer_assignment_changes_limit` as top-level fields, so the stale values from `genesis_config` are silently embedded in the returned `GenesisConfig`-derived fields. The `From<ProtocolConfig> for ProtocolConfigView` conversion reads directly from `genesis_config`: [4](#0-3) 

The RPC handler calls `get_protocol_config` and returns the result verbatim: [5](#0-4) 

---

### Impact Explanation

`num_chunk_validator_seats` changed from 300 to 500 in mainnet protocol version 85: [6](#0-5) 

Any caller of `EXPERIMENTAL_protocol_config` after that upgrade receives `num_chunk_validator_seats = 300` (the genesis default) instead of `500`. Similarly, `num_chunk_producer_seats` and `chunk_producer_assignment_changes_limit` are frozen at genesis values. Tools, indexers, and bridges that use this endpoint to reason about the current validator set size or assignment parameters receive incorrect data. The divergent value is exact and deterministic: the genesis-time integer vs. the current epoch's integer.

---

### Likelihood Explanation

The `EXPERIMENTAL_protocol_config` endpoint is publicly accessible to any RPC caller without authentication. The discrepancy is permanent for any epoch after a protocol upgrade that changes these fields. Any user or tool querying the endpoint for post-upgrade epochs will observe the stale values. [7](#0-6) 

---

### Recommendation

Add the three missing assignments inside `get_protocol_config`, immediately after the existing field copies:

```rust
genesis_config.num_chunk_producer_seats = epoch_config.num_chunk_producer_seats;
genesis_config.num_chunk_validator_seats = epoch_config.num_chunk_validator_seats;
genesis_config.chunk_producer_assignment_changes_limit =
    epoch_config.chunk_producer_assignment_changes_limit;
``` [8](#0-7) 

Additionally, add a compile-time or test-time exhaustiveness check that enumerates every field shared between `EpochConfig` and `GenesisConfig` to prevent future omissions.

---

### Proof of Concept

1. Start a node on mainnet or any chain that has a protocol version ≥ 85 epoch config (where `num_chunk_validator_seats = 500`).
2. Call `EXPERIMENTAL_protocol_config` for any block in that epoch.
3. Observe that the returned `num_chunk_validator_seats` is `300` (genesis default) rather than `500`.

The divergence is confirmed by comparing the epoch config file: [6](#0-5) 

against the genesis default: [9](#0-8)
