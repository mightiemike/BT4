Audit Report

## Title
`addLiquidity` Missing `whenNotPaused` Guard Allows New Principal Deposit Into Paused/Compromised Pool — (`metric-core/contracts/MetricOmmPool.sol`)

## Summary
`MetricOmmPool` exposes a `pauseLevel` state variable and a `whenNotPaused` modifier that reverts when `pauseLevel != 0`. The `swap` function correctly applies this guard, but `addLiquidity` and `removeLiquidity` do not. When the pool is paused in response to a critical event, any address can still call `addLiquidity` to deposit new principal into the compromised pool, and all four extension hooks (`_beforeAddLiquidity`, `_afterAddLiquidity`, `_beforeRemoveLiquidity`, `_afterRemoveLiquidity`) continue to execute against potentially vulnerable extension logic.

## Finding Description
`pauseLevel` is declared at line 72 and `_checkNotPaused()` reverts when `pauseLevel != 0` (line 644). The `whenNotPaused` modifier wraps this check (lines 174–177). `swap` applies it at line 224:

```solidity
) external whenNotPaused nonReentrant(PoolActions.SWAP) returns (int128, int128) {
```

`addLiquidity` (lines 182–196) and `removeLiquidity` (lines 199–212) carry only `nonReentrant` — `whenNotPaused` is absent from both signatures. Both functions unconditionally invoke extension hooks via `_callExtensionsInOrder` (ExtensionCalling.sol lines 75–86), which dispatches into configured extension contracts (`EXTENSION_1`–`EXTENSION_7`). There is no guard or skip path for the paused state inside those hooks.

Exploit path:
1. An exploit or active bug is discovered; admin calls `factory.setPause(poolAddress, 1)`.
2. `swap()` reverts with `PoolPaused` — the swap drain path is blocked.
3. Any address calls `pool.addLiquidity(...)` — succeeds because `whenNotPaused` is absent.
4. `_beforeAddLiquidity` fires extension hooks on the still-live extension contracts.
5. `LiquidityLib.addLiquidity` credits shares and updates `binTotals.scaledToken0`/`scaledToken1`.
6. `_afterAddLiquidity` fires extension hooks again.
7. Deposited token0/token1 are now tracked in `binTotals` and subject to the broken invariant.
8. When the pool is unpaused, the exploit can be triggered against the enlarged pool, draining the newly deposited principal.

## Impact Explanation
Direct loss of user principal. New LP deposits made while the pool is paused are immediately credited to `binTotals` and `_binStates`, making them subject to whatever invariant is broken. Extension hooks continue to execute against potentially vulnerable extension logic, widening the attack surface. The pause mechanism's core purpose — preventing new exposure during a critical event — is defeated for the liquidity path. This meets the "direct loss of user principal" and "broken core pool functionality causing loss of funds" impact gates.

## Likelihood Explanation
No special privilege is required. Any address can call `addLiquidity`; any position owner can call `removeLiquidity`. The trigger condition is an active pause event, which is infrequent but explicitly designed into the protocol. A MEV bot or informed attacker can monitor the mempool for the `setPause` transaction and front-run it with `addLiquidity` to seed a position before the pause takes effect, or ordinary users unaware of the pause continue to add liquidity after it. Likelihood: Medium. Impact: High. Overall: Medium (Sherlock threshold met).

## Recommendation
Apply `whenNotPaused` to `addLiquidity` mirroring the pattern on `swap`:

```solidity
function addLiquidity(
  address owner, uint80 salt, LiquidityDelta calldata deltas,
  bytes calldata callbackData, bytes calldata extensionData
) external whenNotPaused nonReentrant(PoolActions.ADD_LIQUIDITY) returns (...) {
```

For `removeLiquidity`, if the protocol intentionally allows LP exits during a pause, this must be explicitly documented and extension hooks must be skipped during a paused state to prevent hook-level exploitation. Otherwise, apply `whenNotPaused` identically.

## Proof of Concept
```
1. Deploy pool with any extension (e.g., DepositAllowlistExtension).
2. Admin calls factory.setPause(poolAddress, 1) — pauseLevel = 1.
3. Call pool.swap(...) → reverts PoolPaused. ✓
4. Call pool.addLiquidity(owner, salt, deltas, callbackData, extensionData):
   - No whenNotPaused check → proceeds.
   - _beforeAddLiquidity fires extension hooks.
   - LiquidityLib.addLiquidity credits shares, updates binTotals.scaledToken0/scaledToken1.
   - _afterAddLiquidity fires extension hooks.
   - Returns (amount0Added, amount1Added) > 0. ✓ (succeeds while paused)
5. Deposited tokens are now tracked in binTotals and at risk from the broken invariant.
6. On unpause, exploit the broken invariant against the enlarged pool.
```
