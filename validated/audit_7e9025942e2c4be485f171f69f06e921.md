Audit Report

## Title
SwapAllowlistExtension Evaluates the Router Address Instead of the Actual User, Allowing Any Address to Bypass the Swap Allowlist â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`. When a user trades through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, which resolves to `allowedSwapper[pool][router]`. Any pool admin who allowlists the router so that legitimate users can trade through it simultaneously opens the gate to every unprivileged address, completely defeating the allowlist.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` directly as the `sender` argument to every extension hook: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` value against the per-pool allowlist: [2](#0-1) 

Every public entry point in `MetricOmmSimpleRouter` â€” `exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` â€” calls `pool.swap()` directly from the router contract, making the router the immediate `msg.sender` of every pool call: [3](#0-2) [4](#0-3) 

Because the router is always the immediate caller of `pool.swap()`, the `sender` seen by `beforeSwap` is always the **router address**, never the end user. The guard therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

This creates an irresolvable dilemma for any pool admin who wants to restrict swaps to a specific set of users while also allowing those users to trade through the public router. To satisfy router compatibility, the admin must execute `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the access control is void: any address can call `router.exactInputSingle()` and the extension will pass because `allowedSwapper[pool][router] == true`. No existing guard in the extension or the pool checks the original initiating user. [5](#0-4) 

## Impact Explanation

Any pool that deploys `SwapAllowlistExtension` to gate access to a curated set of traders is rendered ineffective the moment the router is allowlisted. An unprivileged, non-allowlisted address can execute a full swap against the pool â€” receiving output tokens and draining pool liquidity â€” simply by calling the public router. This constitutes an admin-boundary break where an unprivileged path bypasses a configured access control, and broken core pool functionality (the allowlist guard) causing potential loss of funds to pool LPs whose liquidity is exposed to unrestricted trading.

## Likelihood Explanation

- `MetricOmmSimpleRouter` is a public, permissionless contract requiring no special role, token balance, or prior interaction.
- The bypass requires a single standard router call (`exactInputSingle`).
- Any pool that uses `SwapAllowlistExtension` for access control and also needs router compatibility is affected by construction â€” the admin has no correct configuration option.
- The condition is triggered automatically whenever the router is allowlisted, which is the expected operational setup.

## Recommendation

The pool should pass the original initiating user as `sender` to extension hooks, not the immediate `msg.sender`. Two concrete approaches:

1. **Router-side**: Store the original `msg.sender` in transient storage alongside the payer (already done for payment via `_setNextCallbackContext`) and expose it via a `getOriginalSender()` view. The pool reads this during the swap and passes it as `sender` to extensions when the immediate caller is a known periphery contract.

2. **Extension-side**: Change `SwapAllowlistExtension.beforeSwap` to read an explicit `originalSender` field from the `extensionData` bytes argument, and require the router to encode the real user address there. The extension then validates that field rather than the `sender` argument.

Option 1 is preferable because it does not require callers to self-report their identity.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]  = true   // intended: only alice may swap
  allowedSwapper[pool][router] = true   // needed: so alice can use the router

Attack:
  vm.prank(eve);                        // eve is NOT allowlisted
  router.exactInputSingle(ExactInputSingleParams({
      pool:             address(pool),
      tokenIn:          token0,
      tokenOut:         token1,
      zeroForOne:       true,
      amountIn:         1_000e18,
      amountOutMinimum: 0,
      recipient:        eve,
      deadline:         block.timestamp + 1,
      priceLimitX64:    0,
      extensionData:    ""
  }));
  // pool.swap() is called with msg.sender = router
  // _beforeSwap receives sender = router
  // allowedSwapper[pool][router] == true â†’ guard passes
  // eve receives token1 output; allowlist is bypassed
```
