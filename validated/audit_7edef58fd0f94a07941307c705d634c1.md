### Title
Unauthenticated Class Declaration via Undeployed Bootstrap Address Bypasses Signature Validation — (File: `crates/starknet_api/src/executable_transaction.rs`)

---

### Summary

The `bootstrap_address` (`0x424f4f545354524150`, ASCII "BOOTSTRAP") is a hardcoded, permanently undeployed address — the Sequencer-native analog of an unverified EOA — that carries special execution privileges. Any unprivileged user can craft a `DeclareV3` transaction with zero resource *amounts* (but a valid L2 gas *price*) to satisfy the `is_bootstrap_declare` predicate, bypassing signature validation, fee enforcement, and nonce management, and thereby declaring arbitrary classes into the chain state without authorization.

---

### Finding Description

**The privileged predicate — `is_bootstrap_declare`** [1](#0-0) 

Three conditions must hold simultaneously:

| Condition | Attacker control |
|---|---|
| `sender_address == bootstrap_address()` (`0x424f4f545354524150`) | Freely settable in any `DeclareV3` |
| `nonce == Nonce(Felt::ZERO)` | Bootstrap address is never deployed → on-chain nonce is always 0 |
| `!charge_fee` | Derived from `enforce_fee`; returns `false` when all resource `max_amount` values are zero |

**How `charge_fee` is set in the gateway** [2](#0-1) 

`enforce_fee` is called on the `ExecutableTransaction` before the blockifier validator runs: [3](#0-2) 

The test confirms the invariant: `enforce_fee = (l1_gas_amount + l2_gas_amount + l1_data_gas_amount) > 0`. [4](#0-3) 

Setting all three `max_amount` fields to `0` makes `enforce_fee` return `false`, so `charge_fee = false`.

**The gateway's resource-bounds check does not block zero amounts** [5](#0-4) 

`validate_resource_bounds` only checks `l2_gas.max_price_per_unit` against a threshold — it never inspects `max_amount`. An attacker sets `max_amount = 0` for every resource (making `enforce_fee = false`) while setting `l2_gas.max_price_per_unit ≥ threshold` (passing the price check). Both conditions are satisfied simultaneously.

**Nonce check does not block the attack** [6](#0-5) 

The bootstrap address is never deployed, so its on-chain nonce is permanently `0`. A transaction with `nonce = 0` passes both the `reject_future_declare_txs = true` path (exact match) and the `_` path (range check `0 ≤ 0 ≤ max_allowed`).

**The blockifier executes the bootstrap path unconditionally** [7](#0-6) 

When `is_bootstrap_declare(charge_fee=false)` is `true`, `execute_raw` skips `perform_pre_validation_stage` (nonce/fee checks), skips `__validate_declare__` (signature verification), and returns `TransactionExecutionInfo::default()` — the class is written to state with no account-related side effects.

The identical gate exists in the Cairo OS program: [8](#0-7) 

**The bootstrap address is an undeployed address (EOA analog)** [9](#0-8) 

It is a raw felt constant, never deployed as a contract, never holding a class hash or nonce in state. There is no on-chain or off-chain mechanism that restricts who may use it as a `sender_address`.

---

### Impact Explanation

**Critical — Wrong state (class hash) from blockifier/execution logic for accepted input.**

An unprivileged attacker can:
1. Declare arbitrary Sierra classes into the chain state without holding a deployed account or providing a valid signature.
2. Front-run any legitimate `Declare` transaction: since `prev_value=0` is enforced, the first declaration wins; the legitimate user's subsequent transaction is rejected as a redeclaration.
3. Flood the class registry with attacker-controlled classes at zero cost (no fee, no nonce consumed).

---

### Likelihood Explanation

**High.** The bootstrap address value is public (`0x424f4f545354524150`). The three trigger conditions are trivially satisfiable from any RPC client. No privileged access, no special key material, and no knowledge of internal state is required. The attack works in both bootstrap mode (`validate_resource_bounds = false`) and normal production mode (`validate_resource_bounds = true`, bypassed by setting a valid price with zero amount).

---

### Recommendation

1. **Gate bootstrap declares on a sequencer-controlled flag.** Add a node-level config boolean (e.g., `allow_bootstrap_declares`) that is `false` by default in production. The gateway should reject any `DeclareV3` from `bootstrap_address` unless this flag is set.
2. **Enforce the bootstrap path only at block 0 (or genesis).** Check `block_number == 0` (or a configurable genesis range) before entering the bootstrap path in `execute_raw` and in the Cairo OS program.
3. **Treat zero-amount resource bounds as fee-enforced for bootstrap-address senders.** Alternatively, add an explicit check in `is_bootstrap_declare` that the transaction was submitted through a trusted internal path, not via the public RPC gateway.

---

### Proof of Concept

```
DeclareV3 {
    sender_address:  0x424f4f545354524150,   // "BOOTSTRAP"
    nonce:           0,
    resource_bounds: AllResources {
        l1_gas:      { max_amount: 0, max_price_per_unit: 0 },
        l2_gas:      { max_amount: 0, max_price_per_unit: <threshold+1> },
        l1_data_gas: { max_amount: 0, max_price_per_unit: 0 },
    },
    contract_class:       <any valid Sierra class>,
    compiled_class_hash:  <matching CASM hash>,
    signature:            [],   // ignored — __validate_declare__ is never called
}
```

**Step-by-step:**

1. Submit the transaction to the gateway RPC endpoint.
2. `validate_resource_bounds`: `l2_gas.max_price_per_unit ≥ threshold` → **passes**.
3. `validate_nonce`: bootstrap address nonce = 0, incoming nonce = 0 → **passes**.
4. `enforce_fee(tx, false)`: `0 + 0 + 0 > 0` = `false` → `charge_fee = false`.
5. `StatefulValidator::perform_validations` → `execute(tx)` → `execute_raw`:
   - `is_bootstrap_declare(false)` = `true` → bootstrap path taken.
   - `__validate_declare__` is **never called** (no signature check).
   - Class is written to `contract_class_changes` / `compiled_class_hashes`.
   - Returns `TransactionExecutionInfo::default()` (no fee, no nonce bump).
6. The class is now declared in the chain state without any authorization.

    // Returns whether the declare transaction is for bootstrapping.
    // In this case, no account-related actions should be made besides the declaration.
    pub fn is_bootstrap_declare(&self, charge_fee: bool) -> bool {
        if let crate::transaction::DeclareTransaction::V3(tx) = &self.tx {
            return tx.sender_address == Self::bootstrap_address()
                && tx.nonce == Nonce(Felt::ZERO)
                && !charge_fee;
        }
        false
    }

    /// Returns the address of the bootstrap contract.

        ContractAddress::from(0x424f4f545354524150_u128)
    }

    fn validate_nonce(
        &self,
        executable_tx: &ExecutableTransaction,
        account_nonce: Nonce,
    ) -> StatefulTransactionValidatorResult<()> {
        let incoming_tx_nonce = executable_tx.nonce();

        let create_error = |message: String| {
            debug!("{message}");
            StarknetError {
                code: StarknetErrorCode::KnownErrorCode(
                    KnownStarknetErrorCode::InvalidTransactionNonce,
                ),
                message,
            }
        };

        match executable_tx {
            // Declare transactions must have the same nonce as the account nonce.
            ExecutableTransaction::Declare(_) if self.config.reject_future_declare_txs => {
                if incoming_tx_nonce != account_nonce {
                    return Err(create_error(format!(
                        "Invalid transaction nonce. Expected: nonce = {account_nonce}, got: \
                         {incoming_tx_nonce}."
                    )));
                }
            }
            // Deploy account transactions must have nonce 0.
            ExecutableTransaction::DeployAccount(_) => {
                if account_nonce != Nonce(Felt::ZERO) {
                    return Err(create_error(format!(
                        "Invalid deploy account transaction. Account is already deployed \
                         (nonce={account_nonce})."
                    )));
                }
                if incoming_tx_nonce != Nonce(Felt::ZERO) {
                    return Err(create_error(format!(
                        "Invalid transaction nonce. Expected: nonce = 0, got: {incoming_tx_nonce}."
                    )));
                }
            }
            // Other transactions must be within the allowed nonce range.
            _ => {
                let max_allowed_nonce =
                    Nonce(account_nonce.0 + Felt::from(self.config.max_allowed_nonce_gap));
                if !(account_nonce <= incoming_tx_nonce && incoming_tx_nonce <= max_allowed_nonce) {
                    return Err(create_error(format!(
                        "Invalid transaction nonce. Expected: {account_nonce} <= nonce <= \
                         {max_allowed_nonce}, got: {incoming_tx_nonce}."
                    )));
                }
            }
        }

        Ok(())
    }

        // meet the following conditions.
        // This flow is used for the sequencer to bootstrap a new system.
        // Note: The absence of any account-related action leads to some unintuitive but expected
        // behavior:
        // - After the transaction is executed successfully, the batcher does not notify the mempool
        //   about its inclusion in a block. As a result, the transaction remains in the mempool.
        // - When the next block is produced, the mempool will propose the same transaction again.
        // - This time, execution will fail because the contract has already been declared.
        // - The transaction will then be marked as rejected, the mempool will be notified, and the
        //   transaction will be removed from the mempool.
        if let Transaction::Declare(tx) = &self.tx {
            if tx.is_bootstrap_declare(self.execution_flags.charge_fee) {
                let mut context = EntryPointExecutionContext::new_invoke(
                    tx_context.clone(),
                    self.execution_flags.charge_fee,
                    SierraGasRevertTracker::new(GasAmount::default()),
                );
                let mut remaining_gas = 0;
                let res = tx.run_execute(state, &mut context, &mut remaining_gas)?;
                assert!(res.is_none(), "Declare execute should not result in a CallInfo.");

                return Ok(TransactionExecutionInfo::default());
            }
