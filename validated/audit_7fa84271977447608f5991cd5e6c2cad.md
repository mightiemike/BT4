Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of actual user, enabling full allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` gates swaps by checking `sender`, which resolves to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap()`, not the actual user. Any pool admin who allowlists the router to enable router-mediated swaps for their permitted users simultaneously opens the gate to every unpermissioned user, completely defeating the allowlist.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as `sender` to `_beforeSwap()`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router the `sender` the extension sees: [4](#0-3) 

The same substitution applies to `exactInput` (intermediate hops use `address(this)` as payer), `exactOutputSingle`, and `exactOutput`: [5](#0-4) [6](#0-5) 

The result: the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. Any admin who calls `setAllowedToSwap(pool, router, true)` — the natural step to let their KYC'd users use the standard router — simultaneously grants every unpermissioned user access.

`DepositAllowlistExtension` does not share this flaw because it checks `owner`, the position owner explicitly supplied by the caller, which `MetricOmmPoolLiquidityAdder` always sets to the actual depositor: [7](#0-6) [8](#0-7) 

## Impact Explanation
A pool admin deploying a permissioned pool (KYC-only, institutional-only) who allowlists the router to enable router-mediated swaps for their permitted users inadvertently grants every user — including those not on the allowlist — full access to the restricted pool's liquidity. The allowlist access-control invariant is completely broken: unauthorized users can trade against restricted LP capital, violating the pool's intended access policy and exposing LP positions to unintended counterparties. This constitutes a broken core pool functionality (access control) causing unauthorized access to restricted pool liquidity, meeting the admin-boundary break criterion.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router, which is the natural and expected configuration for any admin who wants their allowlisted users to use the standard public router. The configuration mistake is subtle: the admin has no indication that allowlisting the router opens the gate to all users. The `MetricOmmSimpleRouter` is a public, permissionless contract callable by anyone. The hazard is undocumented. Likelihood is **medium**.

## Recommendation
The `beforeSwap` hook must gate the economically relevant actor, not the transport layer. Two viable fixes:

1. **Require the router to forward the original `msg.sender` in `extensionData`**: the router encodes `msg.sender` into `extensionData`; the extension decodes and verifies that address against the allowlist. This is the cleanest fix with no interface changes to the pool.

2. **Mirror the deposit pattern**: introduce an explicit `swapper` parameter (analogous to `owner` in `addLiquidity`) that the router sets to `msg.sender` before calling `pool.swap()`, and have the pool forward it to the extension instead of its own `msg.sender`.

Option 1 is the minimal safe fix without protocol-wide interface changes.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  - setAllowedToSwap(pool, router, true)  // admin enables router for alice

Attack:
  - bob (not KYC'd) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(recipient=bob, ...) → msg.sender of pool.swap() = router
  - pool.swap() calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] == true → PASS
  - Bob's swap executes on the restricted pool

Result:
  - Bob bypasses the KYC allowlist entirely via the public router
  - The allowlist guard is rendered ineffective for all router-mediated swaps
  - All restricted LP capital is exposed to unintended counterparties
```
