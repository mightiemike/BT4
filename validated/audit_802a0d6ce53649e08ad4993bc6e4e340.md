Audit Report

## Title
`SwapAllowlistExtension` Gates the Router Address Instead of the End User, Allowing Any Caller to Bypass the Swap Allowlist via the Router - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` from the pool's perspective â€” the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` to the pool, so the extension evaluates `allowedSwapper[pool][router]` rather than the end user's address. A pool admin who allowlists the router to enable router-mediated swaps for their curated users silently voids the allowlist for every user on the network.

## Finding Description
`MetricOmmPool.swap` captures `msg.sender` and passes it as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // direct caller of pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension via `abi.encodeCall`:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router `msg.sender` to the pool:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
```

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. When any of these are called, the pool receives `sender = router`. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. Once the router is allowlisted, the check passes for every caller regardless of identity. There is no existing guard that recovers the real end user's address from `extensionData` or any other source.

## Impact Explanation
A pool deploying `SwapAllowlistExtension` to restrict swaps to a curated set of counterparties (e.g., a private institutional pool, a KYC-gated pool, or a pool with a limited market-maker whitelist) loses that restriction entirely once the router is allowlisted. Any unpermissioned user can call `MetricOmmSimpleRouter.exactInputSingle` targeting the restricted pool and execute swaps as if they were allowlisted. This constitutes broken core pool functionality â€” the allowlist extension is completely voided â€” and enables unauthorized users to extract value from LP positions in a pool calibrated for a specific, trusted set of counterparties, constituting direct loss of LP principal.

## Likelihood Explanation
Likelihood is high. The router is the standard, documented entry point for end users. A pool admin who configures a swap allowlist and also wants to support router-mediated swaps for their allowlisted users will naturally add the router to the allowlist. The bypass requires no special permissions, no flash loans, and no unusual token behavior â€” any user with knowledge of the router address can exploit it immediately after the router is allowlisted.

## Recommendation
Gate on the end user's identity rather than the direct caller. The simplest fix is to pass the end user's address through the router as part of `extensionData` and have the extension decode it when `sender` is a known router, then check `allowedSwapper[pool][realSwapper]` instead. Alternatively, add a dedicated `swapperOverride` field to the swap parameters. At minimum, the extension must document that allowlisting the router opens the pool to all users, and pool admins must never allowlist the router on a pool intended to restrict individual swappers.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured in `beforeSwap`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps.
3. Pool admin does **not** call `setAllowedToSwap(pool, alice, true)` â€” Alice is not allowlisted.
4. Alice calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. The router calls `pool.swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)` with `msg.sender = router`.
6. `MetricOmmPool.swap` passes `msg.sender` (router) as `sender` to `_beforeSwap` (L230-231).
7. `ExtensionCalling._beforeSwap` encodes `sender = router` and calls the extension (L160-176).
8. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][router]` â†’ `true` â†’ swap proceeds (L37).
