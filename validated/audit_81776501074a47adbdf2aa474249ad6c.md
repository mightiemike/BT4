### Title
`CurryTreehasher.uncurry()` Always Returns `(sexp, None)` for `LazyNode` Inputs Due to Missing `__eq__` — (`wheel/python/clvm_rs/curry_and_treehash.py`)

### Summary

`CurryTreehasher.uncurry()` compares `CLVMStorage` nodes returned by `at()` directly against `bytes` keyword constants from the dialect. When the input is a `LazyNode` (the pyo3-backed type returned by deserialization), these comparisons always evaluate as unequal because `LazyNode` has no `__eq__`/`__richcmp__` defined, causing `uncurry` to unconditionally return `(sexp, None)` — the "not a curry" sentinel — for every `LazyNode` input regardless of actual program structure.

### Finding Description

**Root cause — no `__eq__` on `LazyNode`:**

`LazyNode` is defined in `wheel/src/lazy_node.rs` with a `#[pymethods]` block that exposes only `pair` and `atom` getters. There is no `__richcmp__` method anywhere in the Rust codebase (confirmed: zero matches for `__richcmp__`, `__eq__`, `richcmp` in all `.rs` files). [1](#0-0) [2](#0-1) 

Without `__richcmp__`, pyo3 falls back to Python's `object.__eq__`, which is pure identity comparison.

**What `at()` returns:**

`at()` traverses `pair` tuples and returns the node object itself — a `LazyNode` — not its `.atom` bytes. [3](#0-2) 

**The broken comparisons in `uncurry()`:**

```python
if (
    at(sexp, "f") != dialect.A_KW      # LazyNode != b'\x02'  → always True
    or at(sexp, "rff") != dialect.Q_KW  # LazyNode != b'\x01'  → always True
    or at(sexp, "rrr") != dialect.NULL  # LazyNode != b''       → always True
):
    return sexp, None
``` [4](#0-3) 

`dialect.A_KW` is `bytes.fromhex("02")`: [5](#0-4) 

Python's comparison chain for `LazyNode != bytes`:
1. `LazyNode.__ne__(bytes_obj)` → delegates to `object.__eq__` → identity check → `False` (different objects) → `__ne__` returns `True`
2. Short-circuit: the `or` chain fires immediately on the first condition

The first `!=` is always `True`, so `uncurry` returns `(sexp, None)` before examining any further structure.

### Impact Explanation

Any caller that deserializes a CLVM program via `LazyNode` (e.g., via `deser_auto()` or the serde path in `wheel/python/clvm_rs/serde.py`) and then calls `CurryTreehasher.uncurry()` on the result will never receive a valid `(mod, args)` decomposition. Wallet or mempool code relying on `uncurry` to identify curried puzzle types will misclassify every deserialized curried puzzle as "not a curry", causing valid curried puzzles to be rejected or handled by a wrong code path.

### Likelihood Explanation

`LazyNode` is the natural output of deserialization and is a valid `CLVMStorage` implementor (it has `.atom` and `.pair`). Passing it to `uncurry` is a supported, documented API use. The failure is silent — no exception is raised, just a wrong return value — making it easy to miss in testing if tests use `Program` wrappers (which may have their own `__eq__`) rather than raw `LazyNode`.

### Recommendation

Add `__richcmp__` to `LazyNode`'s `#[pymethods]` block that compares atom bytes when both sides are atoms, or alternatively rewrite the comparisons in `uncurry()` to extract `.atom` before comparing:

```python
# Instead of:
at(sexp, "f") != dialect.A_KW
# Use:
(lambda n: n is None or n.atom != dialect.A_KW)(at(sexp, "f"))
```

#[pymethods]
impl LazyNode {
    #[getter(pair)]
    pub fn pair(&self, py: Python) -> PyResult<Option<Py<PyAny>>> {
        match &self.allocator.sexp(self.node) {
            SExp::Pair(p1, p2) => {
                let r1 = Self::new(self.allocator.clone(), *p1);
                let r2 = Self::new(self.allocator.clone(), *p2);
                let v = PyTuple::new(py, [r1, r2])?;
                Ok(Some(v.unbind().into_any()))
            }
            _ => Ok(None),
        }
    }

    #[getter(atom)]
    pub fn atom(&self, py: Python) -> Option<Py<PyAny>> {
        match &self.allocator.sexp(self.node) {
            SExp::Atom => Some(
                PyBytes::new(py, self.allocator.atom(self.node).as_ref())
                    .unbind()
                    .into_any(),
            ),
            _ => None,
        }
    }

```python
    def uncurry(
        self, sexp: CLVMStorage
    ) -> Tuple[CLVMStorage, Optional[List[CLVMStorage]]]:
        """
        uncurry the given program

        returns `mod, [arg1, arg2, ...]`

        if the program is not a valid curry, return `sexp, NULL`

        This distinguishes it from the case of a valid curry of 0 arguments
        (which is rather pointless but possible), which returns `sexp, []`
        """

        dialect = self.dialect
        if (
            at(sexp, "f") != dialect.A_KW
            or at(sexp, "rff") != dialect.Q_KW
            or at(sexp, "rrr") != dialect.NULL
        ):
            return sexp, None
        # since "rff" is not None, neither is "rfr"
        uncurried_function = at(sexp, "rfr")
        assert uncurried_function is not None
        core_items = []

        # since "rrr" is not None, neither is rrf
        core = at(sexp, "rrf")
        while core != dialect.ONE:
            assert core is not None
            if (
                at(core, "f") != dialect.C_KW
                or at(core, "rff") != dialect.Q_KW
                or at(core, "rrr") != dialect.NULL
            ):
                return sexp, None
            # since "rff" is not None, neither is "rfr"
            new_item = at(core, "rfr")
            assert new_item is not None
            core_items.append(new_item)
            # since "rrr" is not None, neither is rrf
            core = at(core, "rrf")
        return uncurried_function, core_items
