Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypass via Router — Extension Gates on Router Address Instead of Actual Swapper - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` equal to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the allowlist checks the router's address rather than the actual user's address. If the pool admin allowlists the router — a necessary operational step for any router-mediated swap to succeed for legitimate users — every unprivileged address on the network can bypass the per-user restriction by calling the public router.

## Finding Description

**Root cause — `MetricOmmPool.swap` passes `msg.sender` (the router) as `sender` to `_beforeSwap`:**

`MetricOmmPool.sol` line 230–231:
```solidity
_beforeSwap(
  msg.sender,   // ← router address when called via router
  recipient,
  ...
```

**`ExtensionCalling._beforeSwap` forwards `sender` unchanged to every configured extension:**

`ExtensionCalling.sol` lines 162–165:
```solidity
abi.encodeCall(
  IMetricOmmExtensions.beforeSwap,
  (sender, recipient, ...)
```

**`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `sender` is the router:**

`SwapAllowlistExtension.sol` line 37:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```
Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()` — the router, not the user.

**The router stores the actual user only in transient callback context, never forwarding it to extensions:**

`MetricOmmSimpleRouter.sol` lines 71–80:
```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    ...
  );
```
`msg.sender` (the actual user) is stored only for the payment callback; it is never passed to `pool.swap()` in a way that reaches the extension.

**Contrast with `DepositAllowlistExtension`**, which correctly gates on `owner` (the economically relevant depositor):

`DepositAllowlistExtension.sol` line 38:
```solidity
if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
```
`owner` is an explicit parameter of `addLiquidity` that the caller supplies directly, so it survives router indirection. No equivalent `swapper` parameter exists in `pool.swap()`.

**Full exploit call chain:**
```
bob (non-allowlisted) → MetricOmmSimpleRouter.exactInputSingle()
  → pool.swap(recipient=bob, ...) [msg.sender = router]
    → _beforeSwap(sender=router, ...)
      → SwapAllowlistExtension.beforeSwap(sender=router, ...)
        → allowedSwapper[pool][router] → true  ← bob's swap passes
```

The same bypass is reachable through `exactInput`, `exactOutputSingle`, and `exactOutput`.

## Impact Explanation
A pool admin who deploys a pool with `SwapAllowlistExtension` to restrict swaps to a specific set of addresses (e.g., KYC-verified counterparties, whitelisted market makers) faces an irresolvable dilemma: if the router is not allowlisted, no allowlisted user can use the router at all; if the router is allowlisted (the only way to enable router-mediated swaps for legitimate users), every address on the network can bypass the per-user restriction. The swap settles fully — the attacker receives real token output and pays real token input — so LPs in a restricted pool are exposed to counterparties the pool admin explicitly intended to exclude. This constitutes a broken core access-control invariant with direct fund-exposure impact on LP assets.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless contract. No special role, token balance, or prior interaction is required beyond holding the input token. The only precondition — the router being allowlisted — is a necessary operational step for any pool that wants to support router-mediated swaps for its legitimate users, making it a near-certain real-world condition. The bypass is reachable on every swap entry point the router exposes.

## Recommendation
Pass the original user through the swap path so the extension can gate on the economically relevant actor, mirroring how the deposit allowlist uses `owner`:

1. **Add a `swapper` parameter to `pool.swap()`** (analogous to `owner` in `addLiquidity`) that the router fills with `msg.sender` before calling the pool. The pool forwards this value as `sender` to extensions instead of its own `msg.sender`. This is architecturally consistent with `DepositAllowlistExtension`.

2. **Alternatively, check `recipient` instead of `sender` in `SwapAllowlistExtension`** — the router always sets `recipient` to the user-supplied address. This is a lower-effort fix but conflates output recipient with access-control identity.

Option 1 is the correct architectural fix.

## Proof of Concept

```solidity
// Setup: pool admin deploys pool with SwapAllowlistExtension,
//        allowlists alice and the router (required for router-mediated swaps).
extension.setAllowedToSwap(address(pool), alice, true);
extension.setAllowedToSwap(address(pool), address(router), true);
// bob is NOT allowlisted

// Attack: bob calls the public router
vm.prank(bob);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000,
        amountOutMinimum: 0,
        recipient:        bob,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// pool.swap() fires with msg.sender = router
// SwapAllowlistExtension checks allowedSwapper[pool][router] → true
// bob's swap succeeds despite not being allowlisted
```

The `beforeSwap` hook fires with `sender = address(router)`, which is allowlisted, so the guard passes and bob receives token output from a pool that was supposed to be restricted to alice only.
