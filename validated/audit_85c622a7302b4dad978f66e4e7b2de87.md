Audit Report

## Title
SwapAllowlistExtension Checks Router Address as `sender` Instead of Actual User, Allowing Any User to Bypass the Per-User Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the original user. If the router is allowlisted (the natural admin step to enable router-based swaps), every unprivileged address can bypass the per-user allowlist entirely; conversely, if only individual users are allowlisted, those users cannot swap via the router at all.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap` (line 231). When called through `MetricOmmSimpleRouter.exactInputSingle`, `msg.sender` at the pool level is the router contract address, not the originating EOA. `ExtensionCalling._beforeSwap` forwards this value unchanged to every configured extension via `abi.encodeCall` (lines 162–176). `SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. The check therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. The router passes `""` as `callbackData` and the user-supplied `params.extensionData` as `extensionData`; the real caller identity is never encoded into `extensionData` by the router, so the extension has no way to recover it. Two mutually exclusive failure modes result: (1) if the admin allowlists the router to enable router-based swaps, every address on the internet passes the check; (2) if the admin allowlists only individual users, those users' router calls revert with `NotAllowedToSwap` even though they are individually permitted.

## Impact Explanation
The allowlist path invariant — that `SwapAllowlistExtension` gates the exact actor the pool designer intended — is broken. In failure mode 1, any unprivileged EOA can swap on a pool intended to be restricted (e.g., KYC-gated or private market-making pools), draining LP value at oracle prices without restriction. In failure mode 2, the primary swap UX path (router) is permanently broken for all individually allowlisted users, constituting an unusable core swap flow. Both outcomes satisfy the allowed impact gate: allowlist bypass by an unprivileged path and broken core pool swap functionality.

## Likelihood Explanation
The trigger is fully unprivileged: any EOA calls `MetricOmmSimpleRouter.exactInputSingle`. The router is a public, deployed periphery contract with no access control. Allowlisting the router is the expected operational step to enable router-based swaps; the bypass is therefore the default outcome of normal configuration. No special timing, flash loan, or oracle manipulation is required. The condition is repeatable on every swap through the router.

## Recommendation
The extension must recover the original user identity rather than trusting the pool-forwarded `sender`. The cleanest correct fix is for the router to encode `msg.sender` into `extensionData` (e.g., as a leading 20-byte prefix), and for `SwapAllowlistExtension.beforeSwap` to decode and verify the real caller when `sender` is a recognized router address. Alternatively, the extension can maintain a registry of trusted routers and, when `sender` is a trusted router, decode the real user from `extensionData`. Using `tx.origin` is a simpler fallback but is incompatible with smart-contract wallets and introduces its own risks.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
  alice (not individually allowlisted) calls:
    router.exactInputSingle({pool: pool, ...})
      → pool.swap(msg.sender=router, ...)           // MetricOmmPool.sol:231
        → _beforeSwap(sender=router, ...)           // ExtensionCalling.sol:160-176
          → SwapAllowlistExtension.beforeSwap(sender=router)
            → allowedSwapper[pool][router] == true  → PASSES
  Alice's swap executes despite never being individually allowlisted.

Alternatively:
  admin calls setAllowedToSwap(pool, alice, true)    // allowlist alice directly
  alice calls router.exactInputSingle({pool: pool, ...})
    → pool.swap(msg.sender=router, ...)
      → SwapAllowlistExtension.beforeSwap(sender=router)
        → allowedSwapper[pool][router] == false → REVERTS NotAllowedToSwap
  Alice cannot use the router even though she is allowlisted.
```
