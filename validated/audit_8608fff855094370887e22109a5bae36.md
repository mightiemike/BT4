Audit Report

## Title
Swap Allowlist Bypassed via Router: `sender` Identity Mismatch Lets Any User Swap on Restricted Pools — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` to the pool. If the pool admin allowlists the router address to support router-mediated swaps for legitimate users, every user — including those explicitly excluded — can bypass the allowlist by routing through the public router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as the first argument and checks it against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-231
_beforeSwap(
    msg.sender,   // <-- direct caller of pool.swap()
    recipient,
    ...
);
``` [2](#0-1) 

`ExtensionCalling._beforeSwap` forwards this `sender` verbatim to the extension: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` and `exactInput` call `pool.swap()` directly, making the router the `msg.sender` to the pool:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
    );
``` [4](#0-3) [5](#0-4) 

The `sender` passed to the extension is therefore the **router address**, not the end user. If the admin allowlists the router (the only way to allow whitelisted users to use the router), the extension check `allowedSwapper[pool][router]` returns `true` for every caller regardless of their individual allowlist status.

## Impact Explanation

This is a direct admin-boundary break. The `SwapAllowlistExtension` is the primary access-control mechanism for restricting who can trade against a pool. A bypass allows unauthorized users to swap against pools intended for private or institutional use, exposing LP positions to unauthorized counterparties and undermining the pool admin's security model. The admin has no way to prevent this without also blocking all router-mediated swaps for legitimate users — a forced choice between broken access control and broken UX.

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract. Any user who discovers the allowlist restriction can trivially route through it. The scenario where a pool admin allowlists the router while maintaining a user-level allowlist is the natural operational configuration (required to support router UX for whitelisted users), making this reachable in normal deployment without any privileged access.

## Recommendation

The extension should check the end user's identity, not the intermediary's. The cleanest fix is to check the `recipient` parameter (the economic beneficiary of the swap) rather than `sender` (the mechanical caller), since `recipient` is always the end user regardless of routing path. Alternatively, the router can write the original `msg.sender` into a transient storage slot before calling `pool.swap()`, and the extension can read it from there. A third option is to move the allowlist check into the router itself.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - allowedSwapper[pool][router] = true   (admin enables router for whitelisted users)
  - allowedSwapper[pool][alice] = true    (alice is whitelisted)
  - allowedSwapper[pool][bob] = false     (bob is NOT whitelisted)

Direct swap by bob:
  bob → pool.swap(...)
  extension checks allowedSwapper[pool][bob] → false → REVERT ✓

Router swap by bob:
  bob → router.exactInputSingle(pool, recipient=bob, ...)
  router → pool.swap(recipient=bob, ...)   // router is msg.sender
  extension checks allowedSwapper[pool][router] → true → PASSES ✗

Result: bob bypasses the allowlist by routing through the public MetricOmmSimpleRouter.
```

A Foundry integration test can confirm this by deploying a pool with `SwapAllowlistExtension`, setting `allowedSwapper[pool][router] = true` and `allowedSwapper[pool][bob] = false`, then calling `router.exactInputSingle` from `bob` and asserting the swap succeeds rather than reverting with `NotAllowedToSwap`.
