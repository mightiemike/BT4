Audit Report

## Title
SwapAllowlistExtension checks router address instead of end-user address, allowing any user to bypass the swap allowlist via the public router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the router is allowlisted (required for any router-mediated swap to work on the pool), every unprivileged user can bypass the per-user allowlist by calling through the router.

## Finding Description

**Root cause — wrong actor identity in `beforeSwap`:**

`SwapAllowlistExtension.beforeSwap` (line 37):
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `msg.sender` is the pool (the extension's caller) and `sender` is the first argument forwarded by the pool.

**How `sender` is bound — `MetricOmmPool.swap` (line 230-240):**
```solidity
_beforeSwap(
  msg.sender,   // ← whoever called pool.swap()
  recipient,
  ...
);
```
`msg.sender` inside `pool.swap()` is the immediate caller. When the user goes through `MetricOmmSimpleRouter.exactInputSingle`, that caller is the router contract, not the end user.

**Router call site — `MetricOmmSimpleRouter.exactInputSingle` (line 72-80):**
```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData
  );
```
The router passes no user-identity information to the pool. The pool receives `msg.sender == address(router)`.

**Exploit flow:**
1. Pool admin deploys pool with `SwapAllowlistExtension` and intends to restrict swaps to a curated set of addresses.
2. Pool admin calls `setAllowedToSwap(pool, routerAddress, true)` so that router-mediated swaps work for allowlisted users.
3. Any non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
4. The pool calls `_beforeSwap(msg.sender=router, ...)`, the extension checks `allowedSwapper[pool][router] == true`, and the swap proceeds — the per-user gate is completely bypassed.

**Existing guards are insufficient:** The extension has no fallback to inspect `extensionData` for a user identity, and the router provides no signed or verified user-identity field. The `onlyPool` guard in `BaseMetricExtension` only ensures the extension is called by a registered pool, not that the correct actor is being checked.

## Impact Explanation
Any unprivileged user can trade on a pool that the admin intended to restrict to a curated allowlist. This constitutes a broken core pool functionality (allowlist curation failure) and, depending on the pool's purpose (e.g., institutional-only, KYC-gated, or rate-limited pools), can result in direct loss of LP assets or protocol fees through unauthorized toxic flow that the allowlist was designed to prevent. Severity: **High**.

## Likelihood Explanation
The bypass requires only that the pool admin has allowlisted the router (the natural configuration for any pool that supports router-mediated swaps). The attacker needs no special privileges, no capital beyond the swap amount, and no multi-transaction setup — a single `exactInputSingle` call suffices. The condition is repeatable on every block.

## Recommendation
The extension must gate the economic actor, not the immediate `pool.swap()` caller. Two sound approaches:

1. **Pass original user through `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify it (requires a trusted router registry or a signed payload).
2. **Check `extensionData` for a verified payer field:** The router already stores the payer in transient storage; expose it as a verified field in the extension payload so the extension can check the true end user.
3. **Document that the allowlist gates the direct caller only**, and require pools using this extension to prohibit router access entirely (i.e., do not allowlist the router).

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted, attacker not allowlisted
    swapExtension.setAllowedToSwap(address(pool), address(router), true);
    // attacker address is NOT in allowedSwapper

    address attacker = makeAddr("attacker");
    deal(address(token0), attacker, 1e18);
    vm.startPrank(attacker);
    token0.approve(address(router), type(uint256).max);

    // Direct pool.swap() would revert NotAllowedToSwap for attacker
    // But via router, sender == router == allowlisted → succeeds
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(token0),
            recipient: attacker,
            deadline: block.timestamp + 1,
            amountIn: 1000,
            amountOutMinimum: 0,
            zeroForOne: true,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    // Swap succeeds — allowlist bypassed
    vm.stopPrank();
}
```
