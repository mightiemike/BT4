Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the real user, allowing any user to bypass per-pool swap allowlists via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension` is documented as gating "swap by swapper address, per pool," but its `beforeSwap` hook checks `sender` — which is `msg.sender` of `pool.swap()`, i.e., the router contract — rather than the actual end user. When `MetricOmmSimpleRouter` is allowlisted (the required production configuration for normal operation), every user of the router passes the allowlist check regardless of whether their own address is permitted. A pool admin who configures a per-user allowlist for KYC-gated or permissioned trading achieves no restriction for any user who routes through `MetricOmmSimpleRouter`.

## Finding Description
`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the extension is called by the pool), and `sender` is the first argument passed by the pool — which is `msg.sender` of the pool's `swap()` call.

`MetricOmmPool.swap()` passes `msg.sender` directly as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // whoever called pool.swap() — the router, not the user
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
        params.extensionData
    );
```

So the allowlist lookup resolves to `allowedSwapper[pool][router]`. The `recipient` parameter (the actual output receiver / real user identity) is silently discarded in the `beforeSwap` signature (`address,` with no name). There is no path to recover the real user identity inside the extension.

The router must be allowlisted for the pool to be usable at all, so `allowedSwapper[pool][router] == true` is the default production state. Any user calling `router.exactInputSingle()` passes the check unconditionally.

## Impact Explanation
This is an admin-boundary break: the pool admin's per-user access control is bypassed by an unprivileged path (the standard periphery router). A pool configured for permissioned trading (e.g., institutional or KYC-gated pools) is fully open to any address that uses the router. Unauthorized users can execute swaps, collecting output tokens they are not entitled to receive. The pool admin's allowlist configuration is silently ineffective — no on-chain error or event signals the bypass.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing entry point for swaps; virtually all non-technical users interact through it. The router must be allowlisted for the pool to be usable at all, so the bypass condition is the default production state. No special knowledge or privilege is required — any user can call `router.exactInputSingle()`.

## Recommendation
The extension must identify the real user, not the intermediary. The cleanest fix: when `sender` is a known router, decode the real initiator from `extensionData` (which the router already accepts as a pass-through parameter) and apply the allowlist check to that address. Alternatively, gate on `recipient` for pools where the intent is to control who receives output. A third option is to document that the allowlist only works for direct pool interaction and remove the router from the allowlist for permissioned pools, but this breaks normal usability.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` and adds only `alice` to the allowlist via `setAllowedToSwap(pool, alice, true)`.
2. Pool admin also adds `MetricOmmSimpleRouter` to the allowlist via `setAllowedToSwap(pool, router, true)` — required for normal operation.
3. `bob` (not on the allowlist) calls `router.exactInputSingle({pool: pool, recipient: bob, ...})`.
4. Router calls `pool.swap(bob, ...)` — pool calls `extension.beforeSwap(sender=router, recipient=bob, ...)`.
5. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
6. `bob` receives output tokens despite not being on the allowlist.
7. Contrast: direct call `pool.swap(bob, ...)` from `bob`'s EOA → `allowedSwapper[pool][bob]` → `false` → reverts `NotAllowedToSwap`.

The bypass is unconditional whenever the router is allowlisted, which is the required production configuration.
