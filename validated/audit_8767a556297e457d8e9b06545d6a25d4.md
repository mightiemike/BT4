Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Allowing Any User to Bypass a Curated Pool's Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` parameter, which `MetricOmmPool.swap()` sets to `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap()`, so the allowlist evaluates the router's address rather than the actual end user. Any user can bypass the per-user swap restriction by calling the router, provided the router is allowlisted — a natural and expected admin action.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to `IMetricOmmExtensions.beforeSwap`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` gates on that `sender` value — checking `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly from the router contract, making the router `msg.sender` inside `pool.swap()`: [4](#0-3) 

The same structural mismatch exists for `exactInput`, `exactOutputSingle`, and `exactOutput` — all call `pool.swap()` from the router: [5](#0-4) 

When a pool admin allowlists the router (to enable router-based swaps for legitimate users), `allowedSwapper[pool][router] == true`. From that point, every call through the router passes the check regardless of who the actual end user is, because `sender` is always the router address. The per-user curation is completely defeated.

`DepositAllowlistExtension` does not share this flaw because it gates on the `owner` parameter — the explicit position owner passed through `addLiquidity` — which `MetricOmmPoolLiquidityAdder` correctly sets to the user-supplied `owner` address rather than to `msg.sender`: [6](#0-5) 

## Impact Explanation
Any user can trade on a curated pool that is supposed to restrict swaps to a specific set of addresses. The attacker receives pool output tokens at oracle-anchored prices, draining the pool's inventory in directions the pool admin intended to block. This constitutes a direct loss of LP principal and a complete failure of the allowlist invariant — a broken core pool functionality causing loss of funds and an admin-boundary break where an unprivileged caller bypasses a configured access control.

## Likelihood Explanation
The trigger requires the pool admin to have allowlisted the router — a natural and expected configuration step for any pool that wants to support the official periphery. No privileged attacker capability is needed beyond calling the public, permissionless router. Once the router is allowlisted, the bypass is unconditional and repeatable by any address on the network.

## Recommendation
The `beforeSwap` hook must identify the actual end user, not the direct caller of `pool.swap()`. The cleanest fix is to have the router encode `abi.encode(msg.sender)` into `extensionData` for allowlisted pools, and have `SwapAllowlistExtension.beforeSwap` decode and check that address instead of the raw `sender` parameter. This requires the router and extension to agree on an encoding convention. Alternatively, the pool's `swap()` interface could be extended to carry an explicit `originator` field, but that is a larger breaking change.

## Proof of Concept
```
1. Deploy a pool with SwapAllowlistExtension configured.
2. Pool admin calls setAllowedToSwap(pool, alice, true)
   // alice is the only allowed swapper
3. Pool admin calls setAllowedToSwap(pool, router, true)
   // enable router-based swaps for alice
4. Bob (not allowlisted) calls router.exactInputSingle({pool, ...})
5. Router calls pool.swap(recipient, ...) — msg.sender inside pool.swap() = router
6. beforeSwap receives sender = router; allowedSwapper[pool][router] == true → passes
7. Bob's swap executes at oracle price, receiving pool tokens he was not authorized to receive.
```
Step 3 is the realistic admin action that opens the bypass. Without it, allowlisted users cannot use the router at all, which is a broken-functionality impact in the other direction.
