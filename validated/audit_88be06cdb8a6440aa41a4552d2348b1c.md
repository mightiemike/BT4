### Title
Denom Name Whitespace Collision Bypasses Uniqueness Check, Enabling NFT Fraud — (`x/nft/types/validation.go`, `x/nft/keeper/denom.go`)

### Summary

`ValidateDenomName` trims whitespace only in a local copy before checking non-emptiness, but never normalises the value that is ultimately stored. `SetDenom` and `HasDenomNm` operate on the raw, un-trimmed name. Because `KeyDenomName` encodes the raw bytes, `'foo'` and `'foo '` produce distinct KVStore keys, so the uniqueness guard is silently bypassed. Any unprivileged user can register a second denom whose display name is visually identical to a legitimate one and mint NFTs that users mistake for the real collection.

---

### Finding Description

**`ValidateDenomName` — trims locally, stores nothing** [1](#0-0) 

The function assigns the trimmed string to a local variable `denomName` and checks only that it is non-empty. The caller's value is never modified; the raw, possibly-padded name is what flows into storage.

**`SetDenom` — uniqueness check and storage both use the raw name** [2](#0-1) 

`HasDenomNm` looks up `KeyDenomName(denom.Name)` with the raw bytes. If `denom.Name` is `'foo '`, the key is different from the key for `'foo'`, so the guard returns `false` and the second denom is stored.

**`KeyDenomName` — raw bytes, no normalisation** [3](#0-2) 

**Runtime entry point — no privileges required**

`MsgIssueDenom.ValidateBasic` calls `ValidateDenomName(msg.Name)`, which passes for any non-blank-only string including `'foo '`. [4](#0-3) 

`msgServer.IssueDenom` forwards `msg.Name` verbatim to `keeper.IssueDenom` → `SetDenom`. [5](#0-4) 

The same gap exists in the genesis path: `ValidateGenesis` calls `ValidateDenomName` per collection but does not normalise before `InitGenesis` calls `SetDenom`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

An attacker registers a denom with `Name:'foo '` after a legitimate denom `Name:'foo'` already exists. Both pass validation; both are stored under different KVStore keys. The attacker controls the second denom and can mint arbitrary NFTs under it. Because most UIs render the name after trimming whitespace, users see two collections both named `foo` and cannot distinguish them, enabling NFT fraud (users purchase attacker-minted tokens believing them to be from the legitimate collection).

---

### Likelihood Explanation

The exploit requires only a standard `MsgIssueDenom` transaction — no governance vote, no operator access, no leaked keys. The cost is a single transaction fee. The attack is repeatable with any Unicode whitespace character accepted by `strings.TrimSpace` (space, tab, non-breaking space U+00A0, etc.), multiplying the number of collision variants.

---

### Recommendation

Normalise the denom name **before** any storage or uniqueness check. The simplest fix is to trim and store the trimmed value in `SetDenom` (or in `IssueDenom`), and to apply the same normalisation in `ValidateGenesis` before calling `SetDenom`:

```go
// In SetDenom (or keeper.IssueDenom):
denom.Name = strings.TrimSpace(denom.Name)
if k.HasDenomNm(ctx, denom.Name) { ... }
```

Alternatively, reject names that differ from their trimmed form in `ValidateDenomName` (i.e., return an error if `denomName != strings.TrimSpace(denomName)`), so the raw and normalised values are always identical.

---

### Proof of Concept

```go
// keeper test — no special setup required
func (suite *KeeperSuite) TestWhitespaceNameCollision() {
    err := suite.keeper.IssueDenom(suite.ctx, "denom1", "foo", schema, "", address)
    suite.NoError(err)

    // 'foo ' differs only by trailing space — should be rejected
    err = suite.keeper.IssueDenom(suite.ctx, "denom2", "foo ", schema, "", address2)
    suite.Error(err, "expected ErrInvalidDenom: name collision via whitespace padding")
}
```

With the current code this test **fails** (the second `IssueDenom` succeeds), confirming the invariant is broken. After the fix the second call must return `ErrInvalidDenom`.

```go
func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil
```
