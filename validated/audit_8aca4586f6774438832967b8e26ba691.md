The vulnerability is real. Here is the analysis:

---

### Title
Wrong PoW Target in AuxPoW Parent Block Verification — (`contract/src/dogecoin.rs`)

### Summary

`check_aux` validates the parent block's proof-of-work hash against the **Dogecoin** block's difficulty target (`block_header.bits`) instead of the **parent chain's** difficulty target (`aux_data.parent_block.bits`). Additionally, `aux_data.parent_block.bits` is never validated against any expected parent-chain difficulty. Together, these allow an attacker to submit a crafted AuxPoW submission whose parent block satisfies only Dogecoin's (easier) difficulty, not Bitcoin's/Litecoin's (harder) difficulty, causing the contract to accept a Dogecoin block backed by a fake parent block that does not exist on the parent chain.

### Finding Description

The critical lines in `check_aux`:

```rust
let pow_hash = aux_data.parent_block.block_hash_pow();
require!(
    self.skip_pow_verification
        || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
    format!("block should have correct pow")
);
``` [1](#0-0) 

`block_header` here is the **Dogecoin** block, so `block_header.bits` is the Dogecoin difficulty. The parent block's hash is checked against the Dogecoin target, not against `target_from_bits(aux_data.parent_block.bits)` (the parent chain's own declared difficulty). Furthermore, `aux_data.parent_block.bits` is never range-checked or validated against any expected parent-chain difficulty anywhere in the codebase.

The `AuxData` struct exposes `parent_block` as a fully attacker-controlled `Header`: [2](#0-1) 

The `submit_block_header` flow calls `check_target` (which validates the Dogecoin block's `bits` against expected Dogecoin difficulty) and then `check_aux` (which validates the parent block's hash against the Dogecoin target — not the parent chain's target): [3](#0-2) 

The existing test comment at line 259 even documents this behavior as expected: *"Litecoin bits 0x19309265 satisfies the easier DOGE target 0x1a00a52e ✓"* — confirming the parent block's hash is checked against the Dogecoin target, not the Litecoin target. [4](#0-3) 

### Impact Explanation

An attacker can:

1. Construct a valid Dogecoin block header with correct `bits` (passes `check_target`/`check_pow`).
2. Set `aux_data.parent_block.bits = 0x1e0fffff` (easiest target) — never validated.
3. Mine a fake parent block whose hash satisfies only the Dogecoin target (much easier than Bitcoin's/Litecoin's actual difficulty).
4. Construct a valid coinbase transaction with the merged mining commitment and valid merkle proofs (all attacker-controlled data).
5. Call `submit_blocks` with this crafted `(Header, Some(AuxData))`.

The contract accepts the submission. The fake Dogecoin block is stored, `chain_work` is updated via `work_from_bits(block_header.bits)`, and `mainchain_tip_blockhash` can be promoted if the attacker accumulates enough chain work — all without a real Bitcoin/Litecoin parent block ever existing. [5](#0-4) 

### Likelihood Explanation

The `submit_blocks` entrypoint is publicly callable (no `#[private]` or trusted-relayer access control gate). The attacker only needs to perform mining work equivalent to Dogecoin's difficulty (for the fake parent block hash), not Bitcoin's or Litecoin's difficulty. Dogecoin's difficulty is orders of magnitude lower than Bitcoin's. The rest of the proof construction (coinbase tx, merkle proofs) is pure data manipulation with no additional PoW requirement.

### Recommendation

Replace `block_header.bits` with `aux_data.parent_block.bits` in the PoW check inside `check_aux`:

```rust
// WRONG (current):
U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits)

// CORRECT:
U256::from_le_bytes(&pow_hash.0) <= target_from_bits(aux_data.parent_block.bits)
```

Additionally, validate that `aux_data.parent_block.bits` does not exceed the parent chain's `proof_of_work_limit_bits`, mirroring the same guard applied to the Dogecoin block's bits in `check_pow`.

### Proof of Concept

A cargo test can:
1. Initialize the contract with a known Dogecoin chain tip.
2. Construct a Dogecoin block header with correct `bits` matching the expected Dogecoin difficulty.
3. Set `parent_block.bits = 0x1e0fffff`.
4. Mine (or brute-force) a `parent_block` nonce such that `block_hash_pow()` satisfies `target_from_bits(dogecoin_bits)` but **not** `target_from_bits(0x1e0fffff)` — i.e., the hash is between the two targets.
5. Construct a coinbase transaction embedding the chain merkle root and valid merkle proofs.
6. Call `check_aux` directly and assert it does not panic.
7. Confirm the block is accepted into contract state despite the parent block failing its own declared PoW target. [6](#0-5)

        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
