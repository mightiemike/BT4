### Title
Missing Parent Health Check After Margin Deduction Enables Collateral Extraction and Forced Liquidation — (`core/contracts/OffchainExchange.sol`)

---

### Summary

`createIsolatedSubaccount` deducts quote margin from the parent subaccount via `spotEngine.updateBalance` but never verifies the parent's health afterward. The analogous quote-transfer path (`transferQuote`) enforces `require(_isAboveInitial(txn.sender))` after the same deduction. The omission allows a user to drain the parent's entire quote balance into an isolated subaccount, rendering the parent immediately liquidatable, while the isolated subaccount's funds are shielded from that liquidation.

---

### Finding Description

`_isolatedMargin` extracts the upper 64 bits of the user-controlled `appendix` field and scales by `1e12`:

```solidity
// OffchainExchange.sol:358-360
function _isolatedMargin(uint128 appendix) internal pure returns (uint128) {
    return (appendix >> 64) * (10**12);
}
```

The user signs the `CreateIsolatedSubaccount` order and sets `appendix >> 64` to any value they choose. Inside `createIsolatedSubaccount`, the margin is deducted from the parent with no post-deduction health gate:

```solidity
// OffchainExchange.sol:1074-1087
int128 margin = int128(_isolatedMargin(txn.order.appendix));
if (margin > 0) {
    digestToMargin[digest] = margin;
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.order.sender, -margin);   // parent loses quote
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, newIsolatedSubaccount, margin); // isolated gains quote
}
// ← no health check on txn.order.sender
```

Compare with `transferQuote`, which performs the identical quote movement but enforces the invariant:

```solidity
// Clearinghouse.sol:247-249
spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -toTransfer);
spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.recipient, toTransfer);
require(_isAboveInitial(txn.sender), ERR_SUBACCT_HEALTH);   // ← guard present here
```

The `CreateIsolatedSubaccount` transaction type is processed by the sequencer through `EndpointTx.sol` lines 620–631 with no additional health gate at that layer either. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

An attacker can:

1. Hold a parent subaccount with quote collateral `Q` and an open perp position with unrealized losses `L` where `Q > L` (parent is healthy).
2. Sign a `CreateIsolatedSubaccount` order with `appendix >> 64` set so that `margin = Q` (the full quote balance).
3. After the sequencer processes the transaction, the parent's quote balance is `0` and its health is negative (driven by the unhedged perp loss).
4. A liquidator immediately liquidates the parent's perp position. Because the parent has no quote, the insurance fund absorbs the residual loss.
5. The attacker retains `Q` in the isolated subaccount, which is not reachable by the parent's liquidation.

Net effect: the attacker extracts `Q` of collateral while socializing `L` of losses onto the insurance fund. This is a direct protocol asset loss, not merely a self-harm scenario. [5](#0-4) 

---

### Likelihood Explanation

The path is fully user-controlled: the attacker signs a valid `CreateIsolatedSubaccount` order with a crafted `appendix`. The sequencer is expected to process all validly signed transactions of this type. No admin privilege, oracle manipulation, or external dependency is required. The attack is executable in a single sequencer batch. [4](#0-3) 

---

### Recommendation

Add a parent health check immediately after the margin deduction in `createIsolatedSubaccount`, mirroring the guard already present in `transferQuote`:

```solidity
// After the spotEngine.updateBalance calls at lines 1077-1086:
if (margin > 0) {
    ...
    require(
        clearinghouse.getHealth(txn.order.sender, IProductEngine.HealthType.INITIAL) >= 0,
        ERR_SUBACCT_HEALTH
    );
}
``` [6](#0-5) [7](#0-6)

// Hardhat test (unmodified contracts)
it("createIsolatedSubaccount drains parent collateral below initial health", async () => {
    // 1. Parent deposits 100e18 USDC quote
    await endpoint.depositCollateral(subaccountName, QUOTE_PRODUCT_ID, 100e6);

    // 2. Parent opens a perp position such that health is positive but
    //    would go negative if quote is removed (simulate via sequencer match).

    // 3. Build CreateIsolatedSubaccount order with appendix >> 64 = 100e6
    //    (scaled by 1e12 inside _isolatedMargin → margin = 100e18)
    const appendix = buildAppendix({ isolatedBit: 1, marginUnits: 100e6 });
    const order = { sender: parentSubaccount, ..., appendix };
    const sig = await signer.signTypedData(...);

    // 4. Sequencer submits the transaction
    await endpoint.submitTransactions([encodeCreateIsolatedSubaccount(order, sig)]);

    // 5. Assert parent health is now negative
    const health = await clearinghouse.getHealth(parentSubaccount, INITIAL);
    expect(health).to.be.lt(0);   // ← fails without the fix, passes with it

    // 6. Confirm isolated subaccount holds the full margin
    const isoBalance = await spotEngine.getBalance(QUOTE_PRODUCT_ID, isoSubaccount);
    expect(isoBalance.amount).to.equal(100e18);
});
