### Title
Gateway stateless and stateful validators enforce a minimum price only for L2 gas, leaving L1 and L1-data gas prices unchecked — (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

The `validate_resource_bounds` function in the stateless transaction validator enforces a minimum price floor only for `l2_gas.max_price_per_unit`. No analogous lower-bound check exists for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The same gap is present in the stateful validator. A transaction with arbitrarily low L1/L1-data gas prices passes both gateway admission stages and enters the mempool, only to be rejected later by the blockifier's pre-validation.

---

### Finding Description

**Stateless validator — partial range check**

In `validate_resource_bounds` the validator performs exactly two checks on resource bounds:

1. At least one bound is non-zero: `max_possible_fee(Tip::ZERO) > 0`
2. L2 gas price meets the static floor: `l2_gas.max_price_per_unit >= config.min_gas_price` [1](#0-0) 

No equivalent minimum is enforced for `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The `StatelessTransactionValidatorConfig` struct exposes only a single `min_gas_price` field, which is wired exclusively to the L2 check. [2](#0-1) 

**Stateful validator — same gap, acknowledged with a TODO**

`validate_tx_l2_gas_price_within_threshold` applies a dynamic threshold (a percentage of the previous block's L2 gas price) only to `l2_gas.max_price_per_unit`. The function comment reads:

> `// TODO(Arni): Consider running this validation for all gas prices.` [3](#0-2) 

**Blockifier pre-validation — the only full check, but too late**

The blockifier's `check_resource_bounds` iterates over all three resources (L1 gas, L1 data gas, L2 gas) and rejects any transaction whose `max_price_per_unit` is below the block's actual gas price for that resource. [4](#0-3) 

Because this check runs inside the batcher, not at the gateway, the transaction has already been admitted to the mempool and consumed gateway resources.

**Concrete path**

A transaction with `l1_gas.max_price_per_unit = 1`, `l2_gas.max_price_per_unit = min_gas_price` (e.g. 8 Gwei), and `l1_data_gas.max_price_per_unit = 1`:

- Passes the stateless check: `max_possible_fee > 0` ✓, `l2_gas_price >= min_gas_price` ✓
- Passes the stateful check: `l2_gas_price >= threshold` ✓
- Is rejected by the blockifier with `ResourceBoundsError::MaxGasPriceTooLow` for L1 gas

The gateway has admitted an invalid transaction — the direct analog of the Chainlink oracle returning a clamped price that the consuming code accepts because it only checks `price > 0` rather than `minPrice <= price <= maxPrice`.

---

### Impact Explanation

The gateway is the first line of defense. Admitting transactions whose L1/L1-data gas prices are below the current block price allows any unprivileged caller to flood the mempool with transactions that will fail at execution time. This wastes mempool slots, batcher CPU, and state-reader I/O, and constitutes a gateway admission of invalid transactions — matching the **High** impact category: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The trigger requires no privilege. Any external caller can craft a V3 `invoke`/`declare`/`deploy_account` transaction with `l1_gas.max_price_per_unit = 1` and `l2_gas.max_price_per_unit = min_gas_price`. The production default `min_gas_price = 8_000_000_000` is publicly known from `config_schema.json`. [5](#0-4) 

---

### Recommendation

Mirror the existing L2 gas price floor check for L1 and L1-data gas prices in both validators:

```rust
// stateless validator — add alongside the existing L2 check
if resource_bounds.l1_gas.max_price_per_unit.0 < self.config.min_l1_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
if resource_bounds.l1_data_gas.max_price_per_unit.0 < self.config.min_l1_data_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Add `min_l1_gas_price` and `min_l1_data_gas_price` fields to `StatelessTransactionValidatorConfig` and wire them through `config_schema.json` / `CONFIG_POINTERS` so they stay synchronized with the consensus-layer min/max bounds already present in `ContextDynamicConfig`. [6](#0-5) 

The stateful validator should resolve the TODO and apply the same percentage-of-previous-block threshold to L1 and L1-data gas prices.

---

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
