Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the end user, allowing any caller to bypass a curated pool's swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` routes a swap, it becomes the pool's `msg.sender`, so `sender` is the router address — not the end user. If the pool admin allowlists the router (the only way to permit router-mediated swaps for legitimate users), every unprivileged address can bypass the per-user allowlist by routing through the same public router, completely defeating the access-control invariant of `SwapAllowlistExtension`.

## Finding Description

**Step 1 — Pool passes `msg.sender` (the router) as `sender` to `_beforeSwap`.**

In `MetricOmmPool.swap()`, `msg.sender` is unconditionally forwarded as the `sender` argument: [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap` encodes that same `sender` and dispatches it to every configured extension.** [2](#0-1) 

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router.** [3](#0-2) 

**Step 4 — `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making itself the pool's `msg.sender`.** [4](#0-3) 

The pool admin faces an impossible choice: not allowlisting the router blocks all legitimate router users; allowlisting the router opens the allowlist to every address on-chain. There is no configuration that simultaneously permits router-mediated swaps for some users while blocking others, because the identity checked is always the router's address, never the end user's.

The contrast with `DepositAllowlistExtension.beforeAddLiquidity` is direct: it explicitly ignores the `sender` parameter (unnamed first argument) and checks `owner` — the economically relevant actor — instead. [5](#0-4) 

## Impact Explanation
A curated pool's swap allowlist is completely defeated for any user who routes through `MetricOmmSimpleRouter`. The attacker requires no special privilege — the router is a public, permissionless contract. Unauthorized parties can execute directional trades or arbitrage on pools designed to be restricted, causing direct loss of LP principal and protocol fees through unauthorized trading on restricted pools. This breaks the core access-control invariant of `SwapAllowlistExtension` and constitutes a fund-impacting policy bypass.

## Likelihood Explanation
The trigger is unprivileged and reachable by any address. The only precondition is that the pool admin has allowlisted the router — which is the expected operational state for any pool that intends to support router-mediated swaps. `MetricOmmSimpleRouter` is the canonical public swap entry point in the periphery layer, so this precondition is routinely satisfied in production. The attack is repeatable with no cooldown or resource cost.

## Recommendation
The allowlist check must gate the **economic actor** (the end user), not the immediate caller of `pool.swap()`. Two complementary fixes:

1. **In `SwapAllowlistExtension.beforeSwap`**: check `recipient` (the address that receives output tokens and is economically attributed to the trade) rather than `sender`, mirroring the correct pattern in `DepositAllowlistExtension` which checks `owner` and ignores `sender`.

2. **In `MetricOmmSimpleRouter`**: pass the original `msg.sender` (the end user) as the `recipient` when the caller intends to receive the output, ensuring `recipient` always identifies the economic actor. Alternatively, encode the user address in `extensionData` so the extension can recover it.

## Proof of Concept

**Setup**: Pool P is configured with `SwapAllowlistExtension`. Pool admin calls `setAllowedToSwap(P, Alice, true)` and `setAllowedToSwap(P, router, true)` so Alice can use the router.

**Attack**:
1. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({ pool: P, recipient: Bob, ... })`.
2. Router calls `P.swap(Bob, zeroForOne, amountIn, ...)` — pool's `msg.sender` = router address.
3. Pool calls `_beforeSwap(router, Bob, ...)`.
4. `ExtensionCalling` encodes `sender = router` and calls `SwapAllowlistExtension.beforeSwap(router, Bob, ...)`.
5. Extension checks `allowedSwapper[P][router]` → `true`. No revert.
6. Swap executes. Bob trades on the restricted pool without being allowlisted.

**Expected**: revert `NotAllowedToSwap`.
**Actual**: swap succeeds because the router's address passes the allowlist check.
