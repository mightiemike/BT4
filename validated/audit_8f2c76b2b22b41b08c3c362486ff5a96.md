### Title
Removed Validator Causes Permanent `ErrNotFound` in `processEventsAndClaimBonus`, Blocking All Reward Claims — (`x/tieredrewards/keeper/claim_rewards.go`)

### Summary

`processEventsAndClaimBonus` unconditionally calls `stakingKeeper.GetValidator` after processing historical events. When a validator is tombstoned and its unbonding period completes, the validator is deleted from the staking store. `AfterValidatorRemoved` intentionally skips cleanup when pending events exist, leaving those events in place. Any subsequent call to `claimRewards` for a position on that validator then hits `GetValidator` → `ErrNotFound` → hard error, permanently blocking all reward claims, undelegations, exits, and force-exits for those positions.

### Finding Description

**Reachable sequence:**

1. A validator with tieredrewards positions begins unbonding → `AfterValidatorBeginUnbonding` fires → an `UNBOND` event is appended with `ReferenceCount = N` (number of positions). [7](#0-6) 

2. During the unbonding period, the validator double-signs → tombstoned → `AfterValidatorRemoved` fires. `hasValidatorEvents` returns `true` (the UNBOND event still has `ReferenceCount > 0`), so the hook skips all cleanup and returns `nil`. [1](#0-0) 

3. The unbonding period completes; the validator is fully deleted from the staking store.

4. A position owner submits `MsgClaimTierRewards` (or any other message that calls `claimRewards`). Execution reaches `processEventsAndClaimBonus`. The historical events are iterated and reference counts decremented. Then, unconditionally: [2](#0-1) 

   ```go
   val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
   if err != nil {
       return nil, err   // ErrNotFound propagates
   }
   ```

5. The error propagates through every caller (`claimRewards`, `claimRewardsAndUpdatesPositions`, `claimRewardsAndUpdateTierPositions`, `ForceFullExitWithDelegation`). There is no alternative code path that bypasses this call. The position is permanently stuck.

### Impact Explanation

All pending bonus rewards for every position on the removed validator become permanently unclaimable. Additionally, because `claimRewards` is called as a prerequisite inside `TierUndelegate`, `ExitTierWithDelegation`, `AddToTierPosition`, `ClearPosition`, and `ForceFullExitWithDelegation`, position owners also cannot exit or recover their locked principal. The economic loss is the full accrued bonus plus the inability to exit the position.

### Likelihood Explanation

Validator tombstoning (double-sign slashing) is a normal, supported chain event. Any validator with active tieredrewards positions that gets tombstoned while those positions have unprocessed events (e.g., an UNBOND event from a prior jailing) triggers this bug. No privileged access or governance action is required — the attacker is the chain's own slashing logic.

### Recommendation

In `processEventsAndClaimBonus`, handle `ErrNotFound` from `GetValidator` gracefully. If the validator no longer exists, treat it as unbonded and skip the tail-segment computation:

```go
val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
if errors.Is(err, collections.ErrNotFound) || errors.Is(err, stakingtypes.ErrNoValidatorFound) {
    // Validator removed; no tail segment to compute.
    applyBonusAccrualCheckpoint(&pos.Position, blockTime)
    pos.UpdateLastKnownBonded(false)
    // ... return any already-computed totalBonus
} else if err != nil {
    return nil, err
}
```

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
