### Title
Unchecked u64 Multiplication Overflow in `op_unknown` Produces Fraudulently Low Cost — (`File: src/more_ops.rs`)

---

### Summary

`op_unknown` in `src/more_ops.rs` computes a final cost by multiplying a base cost (bounded by `max_cost`) by an attacker-controlled `cost_multiplier + 1`. This multiplication is performed on `u64` values without overflow protection. In Rust release builds, integer overflow wraps silently. A crafted unknown operator with `cost_multiplier` near `u32::MAX` and arguments sized to push the base cost to a specific value can cause the product to wrap around `u64::MAX`, yielding a fraudulently small (or zero) cost that passes the subsequent `u32::MAX` guard and is returned to the interpreter as the true cost of the operation.

---

### Finding Description

In `op_unknown` (`src/more_ops.rs`), the cost multiplier is extracted from the operator bytes:

```rust
let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
    Some(v) => v as u64,
    None => { return Err(EvalErr::Invalid(o))?; }
};
```

For a 5-byte operator, `op[0..4]` is 4 bytes, so `cost_multiplier` can reach `u32::MAX = 4_294_967_295`, making `cost_multiplier + 1 = 4_294_967_296` (fits in u64 without overflow).

The base cost is then computed (for `cost_function` 1, 2, or 3) from the arguments, checked against `max_cost`, and then multiplied:

```rust
check_cost(cost, max_cost)?;      // line 260 — ensures cost ≤ max_cost
cost *= cost_multiplier + 1;      // line 261 — UNCHECKED multiplication
if cost > u32::MAX as u64 {       // line 262
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

The `check_cost` at line 260 only guarantees `cost ≤ max_cost`. It does **not** guarantee that `cost * (cost_multiplier + 1)` fits in `u64`. With Chia's production `max_cost ≈ 11_000_000_000` and `cost_multiplier + 1 = 4_294_967_296`:

```
11_000_000_000 × 4_294_967_296 = 47_244_640_256_000_000_000
u64::MAX                        = 18_446_744_073_709_551_615
```

The product exceeds `u64::MAX` by a factor of ~2.56, so it wraps. The wrapped value is:

```
47_244_640_256_000_000_000 mod 2^64 ≈ 10_351_152_108_580_896_768
```

For the specific case where `cost` is chosen to be an exact multiple of `2^32` (e.g., `cost = 4_294_967_296`), the product `cost × 2^32 mod 2^64 = 0`, which is `≤ u32::MAX`, so the guard at line 262 passes and the function returns `Ok(Reduction(0, allocator.nil()))` — a cost of **zero**.

The constants confirm the base cost can reach the required threshold:

- `ARITH_BASE_COST = 99`, `ARITH_COST_PER_BYTE = 3` → reaching `cost = 4_294_967_296` requires `total_bytes ≈ 1.43 billion`, within the `u32::MAX`-byte heap limit.
- `CONCAT_COST_PER_BYTE = 3` — same analysis applies for `cost_function = 3`. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

An attacker who can submit CLVM programs in consensus mode (where `NO_UNKNOWN_OPS` is not set) can craft a 5-byte unknown operator with `cost_multiplier = u32::MAX` and supply arguments whose total byte count is tuned to make the base cost a multiple of `2^32`. The multiplication wraps to zero (or another small value ≤ `u32::MAX`), and `op_unknown` returns that fraudulently low cost to `run_program`. The interpreter accumulates this undercharged cost, allowing the attacker to execute arbitrarily expensive unknown-op sequences while consuming negligible reported cost — bypassing the cost ceiling that is the primary DoS defence of the CLVM. [4](#0-3) [5](#0-4) [6](#0-5) 

---

### Likelihood Explanation

- Unknown operators are permitted in consensus mode (the default `ChiaDialect` without `NO_UNKNOWN_OPS`). Mempool mode sets `NO_UNKNOWN_OPS` and would reject the operator before reaching `op_unknown`, but consensus validation does not.
- The attacker controls the operator bytes (5 bytes, fully attacker-chosen) and the argument list (atom sizes). Both are delivered as raw CLVM bytes — a direct attacker-controlled entry path.
- The required base cost (`≥ 2^32 ≈ 4.3 billion`) demands large argument atoms, but the heap limit is `u32::MAX` bytes, so it is within the allocator's design envelope.
- The specific alignment condition (`cost mod 2^32 = 0`) is achievable by tuning argument sizes, since `ARITH_COST_PER_BYTE = 3` and `CONCAT_COST_PER_BYTE = 3` allow fine-grained control of the base cost. [7](#0-6) [8](#0-7) 

---

### Recommendation

Replace the unchecked multiplication with a checked variant and treat overflow as an invalid operator:

```rust
// Before (line 261):
cost *= cost_multiplier + 1;

// After:
cost = cost
    .checked_mul(cost_multiplier + 1)
    .ok_or(EvalErr::Invalid(o))?;
```

This mirrors the pattern already used in `op_add`'s fast path (`checked_add`) and is consistent with the existing `if cost > u32::MAX` guard that already treats oversized costs as invalid. [1](#0-0) [9](#0-8) 

---

### Proof of Concept

Operator bytes (5 bytes): `[0xff, 0xff, 0xff, 0xff, 0x40]`
- `cost_multiplier = u32_from_u8([0xff, 0xff, 0xff, 0xff]) = 4_294_967_295`
- `cost_function = (0x40 & 0b11000000) >> 6 = 1` (arithmetic-like)
- `cost_multiplier + 1 = 4_294_967_296 = 2^32`

Argument list: a single atom of `N` bytes where `N` is chosen so that:
```
ARITH_BASE_COST + ARITH_COST_PER_ARG + ARITH_COST_PER_BYTE * N ≡ 0 (mod 2^32)
99 + 320 + 3*N ≡ 0 (mod 2^32)
3*N ≡ -419 (mod 2^32)
N ≡ (-419) * modular_inverse(3, 2^32) (mod 2^32)
```

The modular inverse of 3 mod 2^32 is `2_863_311_531`. So:
```
N ≡ (-419) * 2_863_311_531 mod 2^32
  ≡ 4_294_966_877 * 2_863_311_531 mod 2^32
  ≡ 1_431_655_765 (mod 2^32)   [≈ 1.43 billion bytes]
```

With this atom size, `cost = 4_294_967_296 = 2^32` after the loop. Then:
```
cost *= 4_294_967_296   →   2^32 * 2^32 = 2^64 ≡ 0 (mod 2^64)
```

The guard `if cost > u32::MAX` evaluates `0 > 4_294_967_295` → false, so the function returns `Ok(Reduction(0, nil))` — a cost of zero for an operation that consumed 1.43 GB of argument data. [10](#0-9) [11](#0-10)

pub fn op_unknown(
    allocator: &mut Allocator,
    o: NodePtr,
    mut args: NodePtr,
    max_cost: Cost,
) -> Response {
    // unknown opcode in lenient mode
    // unknown ops are reserved if they start with 0xffff
    // otherwise, unknown ops are no-ops, but they have costs. The cost is computed
    // like this:

    // byte index (reverse):
    // | 4 | 3 | 2 | 1 | 0          |
    // +---+---+---+---+------------+
    // | multiplier    |XX | XXXXXX |
    // +---+---+---+---+---+--------+
    //  ^               ^    ^
    //  |               |    + 6 bits ignored when computing cost
    // cost_multiplier  |
    // (up to 4 bytes)  + 2 bits
    //                    cost_function

    // 1 is always added to the multiplier before using it to multiply the cost, this
    // is since cost may not be 0.

    // cost_function is 2 bits and defines how cost is computed based on arguments:
    // 0: constant, cost is 1 * (multiplier + 1)
    // 1: computed like operator add, multiplied by (multiplier + 1)
    // 2: computed like operator mul, multiplied by (multiplier + 1)
    // 3: computed like operator concat, multiplied by (multiplier + 1)

    // this means that unknown ops where cost_function is 1, 2, or 3, may still be
    // fatal errors if the arguments passed are not atoms.

    let op_atom = allocator.atom(o);
    let op = op_atom.as_ref();

    if op.is_empty() || (op.len() >= 2 && op[0] == 0xff && op[1] == 0xff) {
        Err(EvalErr::Reserved(o))?;
    }

    let cost_function = (op[op.len() - 1] & 0b11000000) >> 6;
    let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
        Some(v) => v as u64,
        None => {
            return Err(EvalErr::Invalid(o))?;
        }
    };

    let mut cost = match cost_function {
        0 => 1,
        1 => {
            let mut cost = ARITH_BASE_COST;
            let mut byte_count: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                cost += ARITH_COST_PER_ARG;
                let len = atom_len(allocator, arg, "unknown op")?;
                byte_count += len as u64;
                check_cost(cost + (byte_count as Cost * ARITH_COST_PER_BYTE), max_cost)?;
            }
            cost + (byte_count * ARITH_COST_PER_BYTE)
        }
        2 => {
            let mut cost = MUL_BASE_COST;
            let mut first_iter: bool = true;
            let mut l0: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                if first_iter {
                    l0 = len as u64;
                    first_iter = false;
                    continue;
                }
                let l1 = len as u64;
                cost += MUL_COST_PER_OP;
                cost += (l0 + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                l0 += l1;
                check_cost(cost, max_cost)?;
            }
            cost
        }
        3 => {
            let mut cost = CONCAT_BASE_COST;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                cost += CONCAT_COST_PER_ARG;
                cost += CONCAT_COST_PER_BYTE * (len as Cost);
                check_cost(cost, max_cost)?;
            }
            cost
        }
        _ => 1,
    };

    assert!(cost > 0);

    check_cost(cost, max_cost)?;
    cost *= cost_multiplier + 1;
    if cost > u32::MAX as u64 {
        Err(EvalErr::Invalid(o))?
    } else {
        Ok(Reduction(cost as Cost, allocator.nil()))
    }
}
