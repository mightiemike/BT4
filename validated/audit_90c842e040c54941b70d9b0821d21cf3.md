### Title
Committed-Nonce Retention Window Expiry Silently Re-Admits Already-Used-Nonce Transactions to the Mempool — (`crates/apollo_mempool/src/mempool.rs`)

### Summary

`MempoolState` tracks committed nonces in a finite sliding window (`CommitHistory`). After `committed_nonce_retention_block_count` blocks the committed nonce for an account is **evicted**. Once evicted, `validate_incoming_tx` falls back entirely to the gateway-supplied `incoming_account_nonce`. If the gateway's state reader is stale (behind by more than the retention window), a transaction whose nonce was already consumed on-chain is accepted by the mempool without error.

### Finding Description

`CommitHistory` is a fixed-capacity `VecDeque<AddressToNonce>` initialised with `committed_nonce_retention_block_count` empty maps.

```rust
// crates/apollo_mempool/src/mempool.rs
fn push(&mut self, commit: AddressToNonce) -> AddressToNonce {
    let removed_commit = self.commits.pop_front();   // oldest entry evicted
    self.commits.push_back(commit);
    removed_commit.expect("...")
}
```

Every `commit_block` call pushes a new entry and pops the oldest. The popped entry is then used to **delete** the address from `MempoolState::committed` if the stored nonce matches the evicted one:

```rust
// crates/apollo_mempool/src/mempool.rs  lines 148-157
let removed_commit = self.commit_history.push(address_to_nonce);
for (address, removed_nonce) in removed_commit {
    let last_committed_nonce = *self.committed.get(&address)...;
    if last_committed_nonce == removed_nonce {
        self.committed.remove(&address);   // ← nonce tracking permanently lost
    }
}
```

After eviction, `resolve_nonce` has no committed or staged entry for the address and falls back to `incoming_account_nonce`:

```rust
// crates/apollo_mempool/src/mempool.rs  lines 107-113
fn resolve_nonce(&self, address: ContractAddress, incoming_account_nonce: Nonce) -> Nonce {
    self.staged.get(&address)
        .or_else(|| self.committed.get(&address))
        .copied()
        .unwrap_or(incoming_account_nonce)   // ← falls back to gateway-supplied value
}
```

`validate_incoming_tx` then checks only `tx_nonce < account_nonce` where `account_nonce` is whatever the gateway passed:

```rust
// crates/apollo_mempool/src/mempool.rs  lines 162-174
fn validate_incoming_tx(&self, tx_reference: TransactionReference,
                         incoming_account_nonce: Nonce) -> MempoolResult<()> {
    let account_nonce = self.resolve_nonce(address, incoming_account_nonce);
    if tx_nonce < account_nonce {
        return Err(MempoolError::NonceTooOld { ... });
    }
    Ok(())
}
```

The gateway reads the on-chain nonce via `gateway_fixed_block_state_reader.get_nonce()` and passes it as `account_nonce` to `validate_by_mempool`. If that state reader is stale (e.g. during sync catch-up, a lagging fixed-block snapshot, or a race between block commitment and state-reader refresh), it returns an old nonce. The mempool's committed-nonce tracking was the only guard against this staleness window — but it expires after `committed_nonce_retention_block_count` blocks (default 100, production config also 100).
