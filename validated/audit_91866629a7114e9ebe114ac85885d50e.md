### Title
Unprivileged Caller Can Aggressively Prune Historical Block Headers via `run_mainchain_gc` - (File: contract/src/lib.rs)

### Summary
`run_mainchain_gc` is a public, state-mutating function on `BtcLightClient` that permanently deletes block headers from contract storage and advances `mainchain_initial_blockhash`. It carries no caller restriction — any unprivileged NEAR account can call it with an arbitrarily large `batch_size`, forcing the maximum possible GC sweep in a single transaction. This permanently destroys block headers that downstream consumers rely on for SPV proof verification, causing `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` to panic for any pruned block.

### Finding Description

`run_mainchain_gc` is declared as a fully public `#[near]` method: [1](#0-0) 

The only decorator it carries is `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. That attribute controls whether the function is callable when the contract is **paused** — it does not restrict who may call it when the contract is running normally. There is no `#[trusted_relayer]`, no `#[private]`, and no explicit `require!` guard on `env::predecessor_account_id()`.

The GC logic removes up to `min(total_amount_to_remove, batch_size)` headers, where `total_amount_to_remove = stored_count − gc_threshold`: [2](#0-1) 

Passing `batch_size = u64::MAX` causes the function to remove every header beyond `gc_threshold` in a single call — the maximum sweep the contract's own invariant permits. Each removed header is permanently deleted from `headers_pool` and `mainchain_height_to_header`: [3](#0-2) 

By contrast, the intended internal caller — `submit_blocks` — passes only `num_of_headers` (the count of newly submitted headers) as `batch_size`, producing a gradual, proportional prune: [4](#0-3) 

`submit_blocks` itself is protected by `#[trusted_relayer]`, meaning only staked/approved relayers may call it. The public `run_mainchain_gc` path bypasses that gate entirely.

### Impact Explanation

Both SPV verification entry points look up the target block in `headers_pool` and panic if it is absent: [5](#0-4) [6](#0-5) 

`mainchain_header_to_height` is also cleared for pruned blocks: [7](#0-6) 

So after an attacker calls `run_mainchain_gc(u64::MAX)`:

- Every block header older than `gc_threshold` blocks from the current tip is permanently erased from `headers_pool` and both index maps.
- Any call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a transaction in a pruned block panics with `"cannot find requested transaction block"` or `"block does not belong to the current main chain"`.
- `mainchain_initial_blockhash` is advanced irreversibly; the deleted headers cannot be recovered without a privileged relayer resubmitting them.
- Cross-chain bridges, custody systems, or any consumer that depends on SPV proofs for recently-confirmed (but now pruned) blocks will have their verification permanently broken.

### Likelihood Explanation

The entry point is a plain NEAR contract call requiring no deposit, no role, and no special account. Any NEAR account — including a freshly created one with minimal balance — can invoke it. The call is cheap (gas-bounded by the loop), and the attacker needs no knowledge beyond the contract's account ID. A griefing attacker targeting a bridge or settlement layer has clear economic motivation to prevent SPV proofs from succeeding.

### Recommendation

Restrict `run_mainchain_gc` to trusted callers using the same `#[trusted_relayer]` macro already applied to `submit_blocks`, or add an explicit role check (e.g., `Role::DAO` or a dedicated `GCManager` role) at the top of the function body:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
#[trusted_relayer]          // add this, or an equivalent acl_require! guard
pub fn run_mainchain_gc(&mut self, batch_size: u64) { … }
```

Alternatively, make the function `pub(crate)` and remove it from the public ABI entirely, since `submit_blocks` already calls it internally.

### Proof of Concept

```
# Attacker account (no special role required)
near call <btc_light_client_account> run_mainchain_gc \
  '{"batch_size": 18446744073709551615}' \
  --accountId attacker.near \
  --gas 300000000000000

# Immediately after, any SPV verification for a block older than gc_threshold panics:
near call <btc_light_client_account> verify_transaction_inclusion_v2 \
  '<borsh-encoded ProofArgsV2 for a now-pruned block>' \
  --accountId victim.near
# → panics: "cannot find requested transaction block"
```

The attacker calls `run_mainchain_gc` with `batch_size = u64::MAX`. The contract computes `selected_amount_to_remove = min(stored_count − gc_threshold, u64::MAX)` and deletes every header beyond the threshold in one sweep. All subsequent SPV proof calls for those blocks fail permanently. [8](#0-7)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
