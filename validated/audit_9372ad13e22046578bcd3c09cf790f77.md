### Title
Unsafe `transferFrom` in `replaceUsdcEWithUsdc` Allows usdcE Drain Without Paying USDC — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(...)` as a raw interface call whose boolean return value is silently discarded. The protocol already provides `ERC20Helper.safeTransferFrom` (imported and bound via `using ERC20Helper for IERC20Base`) and uses `safeTransfer` for the other leg of the same swap. If the USDC `transferFrom` returns `false` without reverting, execution continues, and the caller receives usdcE from the DDA without having paid any USDC.

---

### Finding Description

`replaceUsdcEWithUsdc` is a permissionless function (only gated by `block.chainid == 57073`) that is intended to atomically swap usdcE held in a `DirectDepositV1` address for USDC supplied by the caller:

```
1. Pull `balance` USDC from msg.sender → directDepositV1   (line 616 — raw transferFrom)
2. Withdraw `balance` usdcE from directDepositV1 → ContractOwner  (line 617)
3. Send `balance` usdcE from ContractOwner → msg.sender    (line 618 — safeTransfer)
```

Step 1 uses a raw `transferFrom` call:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
```

The return value is never checked. [1](#0-0) 

Steps 2 and 3 execute unconditionally after step 1. If the USDC token returns `false` instead of reverting (e.g., due to insufficient allowance, insufficient balance, or a non-reverting ERC20 implementation), the function continues and the caller receives usdcE for free.

The `ERC20Helper` library — already imported and bound with `using ERC20Helper for IERC20Base` — provides `safeTransferFrom`, which performs a low-level call and requires `success && (data.length == 0 || abi.decode(data, (bool)))`. [2](#0-1) 

The same function already uses `safeTransfer` for the usdcE leg, making the omission on the USDC leg a clear inconsistency. [3](#0-2) 

---

### Impact Explanation

An attacker who calls `replaceUsdcEWithUsdc` for any subaccount whose DDA holds usdcE, while holding zero USDC allowance, will:

- Receive the full usdcE balance of that DDA at no cost if the USDC token's `transferFrom` returns `false` rather than reverting.
- Drain usdcE from every DDA on chain 57073 that has a non-zero usdcE balance.

The corrupted asset delta is: usdcE balance of `directDepositV1` → attacker, with no corresponding USDC credit to `directDepositV1`. This is a direct token theft.

---

### Likelihood Explanation

The function is callable by any unprivileged address on chain 57073 (Ink). The only precondition is that a DDA exists with a non-zero usdcE balance. The USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on Ink is a bridged/wrapped deployment whose exact revert-on-failure behavior may differ from mainnet USDC. Non-reverting ERC20 tokens (returning `false` on failure) are a well-documented class; the protocol's own `ERC20Helper` was written specifically to handle them. [4](#0-3) 

---

### Recommendation

Replace the raw `transferFrom` call with `safeTransferFrom`, consistent with how the rest of the codebase handles token transfers:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`safeTransferFrom` is already available on `IERC20Base` via `using ERC20Helper for IERC20Base` declared at line 24 of `ContractOwner.sol`. [5](#0-4) 

---

### Proof of Concept

1. A DDA for some `subaccount` on chain 57073 holds `N` usdcE tokens.
2. Attacker calls `replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance granted to `ContractOwner`.
3. `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, N)` returns `false` (no revert).
4. `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` transfers `N` usdcE to `ContractOwner`. [6](#0-5) 
5. `IERC20Base(usdcE).safeTransfer(msg.sender, N)` sends `N` usdcE to the attacker.
6. Attacker receives `N` usdcE; `directDepositV1` receives 0 USDC. Net: usdcE drained for free.

```text
// @dev Adapted from https://github.com/Uniswap/v3-core/blob/main/contracts/libraries/TransferHelper.sol
library ERC20Helper {
    function safeTransfer(
        IERC20Base self,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(IERC20Base.transfer.selector, to, amount)
        );
        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
