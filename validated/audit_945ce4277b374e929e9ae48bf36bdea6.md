Audit Report

## Title
`SwapAllowlistExtension` gates on the router address instead of the economic actor, allowing any user to bypass per-user swap allowlist via `MetricOmmSimpleRouter` — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` checks the `sender` parameter, which `MetricOmmPool.swap()` sets to `msg.sender` — the immediate caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` inside the pool, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actual_user]`. Any pool admin who allowlists the router to permit router-based swaps inadvertently grants every address — including those explicitly excluded — the ability to execute live swaps with real token transfers.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap()`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` forwards that value unchanged as the first argument to each configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` then checks whether that `sender` is in the allowlist: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router contract the `msg.sender` inside the pool: [4](#0-3) 

The same binding applies to `exactOutputSingle`, `exactInput`, and `exactOutput` — every router entry point calls `pool.swap()` directly: [5](#0-4) [6](#0-5) 

The `DepositAllowlistExtension` does **not** share this flaw because it ignores the `sender` parameter entirely and checks the `owner` parameter (the position owner explicitly supplied to `addLiquidity()`): [7](#0-6) 

`MetricOmmPoolLiquidityAdder` correctly sets `owner` to `msg.sender` or an explicit owner, so the deposit allowlist correctly gates on the economic actor: [8](#0-7) 

The exact wrong value is `allowedSwapper[pool][router]` being evaluated instead of `allowedSwapper[pool][actual_user]`, causing the extension's access-control decision to be bound to the wrong actor.

## Impact Explanation
A pool admin who deploys a curated pool with `SwapAllowlistExtension` intends to restrict trading to a specific set of addresses. Once the router is allowlisted (the natural operational step to support standard periphery usage), the allowlist is completely defeated: any EOA can call any `MetricOmmSimpleRouter` entry point and execute a live swap with real token transfers, receiving output tokens and draining LP-owned liquidity at oracle prices. This constitutes a direct loss of LP principal and a broken core pool invariant (curated access control). The impact matches "Broken core pool functionality causing loss of funds" and "direct loss of user principal."

## Likelihood Explanation
`MetricOmmSimpleRouter` is the protocol's standard and only supported swap periphery. Any pool admin who wants end-users to interact via the router — the normal operational assumption — must allowlist the router address. The bypass requires no special privilege, no malicious setup, and no non-standard token behavior: any EOA calls a public router function. The combination of a natural admin action (allowlisting the router) and a public router call is the expected usage pattern, making exploitation trivially reachable by any unprivileged trader.

## Recommendation
The `sender` parameter passed to `beforeSwap` represents the immediate caller of `pool.swap()`, which is the router when periphery is used. The extension must gate on the economic actor — the address that initiated the trade and will pay for it. Two complementary fixes:

1. **In the router**: pass the original `msg.sender` (the user) through `extensionData` as a documented convention for allowlist extensions to decode and check. This requires no core interface change.
2. **Preferred — core interface change**: add an `initiator` field to the `beforeSwap` signature so the pool can pass both `msg.sender` (the immediate caller) and the original user address separately, letting extensions choose which actor to gate on.

Until fixed, pool admins should **not** allowlist the router address on pools that use `SwapAllowlistExtension` for per-user access control, and should instead require users to call `pool.swap()` directly.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension configured on beforeSwap.
  - Pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
    (to allow router-based swaps — the natural operational step).
  - Pool admin does NOT call setAllowedToSwap(pool, alice, true)
    (alice is intentionally excluded from the curated pool).

Attack:
  1. alice calls router.exactInputSingle({pool: pool, ...}).
  2. Router calls pool.swap(recipient, ...) — msg.sender in pool = router.
  3. Pool calls _beforeSwap(sender=router, ...).
  4. SwapAllowlistExtension checks allowedSwapper[pool][router] → true.
  5. Swap executes. Alice receives output tokens.

Expected: revert NotAllowedToSwap (alice is not allowlisted).
Actual:   swap succeeds because the router is allowlisted and the
          extension never sees alice's address.

Foundry test outline:
  - Deploy SwapAllowlistExtension, configure pool with it.
  - setAllowedToSwap(pool, address(router), true).
  - vm.prank(alice); router.exactInputSingle(...).
  - Assert swap succeeds (no revert) despite alice not being allowlisted.
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
