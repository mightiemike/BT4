### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas` When L2/L1DataGas Are Zero, Breaking Consensus Transaction Hash Binding — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter in the p2p/consensus path uses a zero-value heuristic to decide between `ValidResourceBounds::L1Gas` and `ValidResourceBounds::AllResources`. When a v0.13.3+ (`AllResources`) transaction carries `l2_gas = {0,0}` and `l1_data_gas = {0,0}`, the serializer and deserializer produce the same wire bytes as a pre-0.13.3 `L1Gas` transaction. The deserializer always resolves the ambiguity to `L1Gas`. This causes the consensus path to fail with `DEPRECATED_RESOURCE_BOUNDS_ERROR` when a validator tries to reconstruct the `RpcInvokeTransactionV3` from the received protobuf, and causes the p2p-sync path to compute a different transaction hash than the one originally committed, breaking hash binding.

### Finding Description

**Serialization is lossy — both variants produce identical wire bytes**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `AllResources { l2_gas: zero, l1_data_gas: zero }` and `L1Gas(l1_gas)` to the same protobuf:

```
l1_gas  = Some(l1_gas_limits)
l2_gas  = Some({ max_amount: 0, max_price_per_unit: 0 })
l1_data_gas = Some({ max_amount: 0, max_price_per_unit: 0 })
``` [1](#0-0) 

**Deserialization always resolves the ambiguity to `L1Gas`**

The reverse converter applies the heuristic `if l1_data_gas.is_zero() && l2_gas.is_zero() → L1Gas`: [2](#0-1) 

**Consensus path: hard rejection**

The consensus p2p path deserializes `ConsensusTransaction::RpcTransaction(Invoke)` through:

`protobuf::InvokeV3WithProof` → `InvokeTransactionV3` (uses the broken `ValidResourceBounds::try_from`) → `RpcInvokeTransactionV3`

The last step explicitly rejects anything that is not `AllResources`: [3](#0-2) 

`DEPRECATED_RESOURCE_BOUNDS_ERROR` is returned, so the validator drops the entire consensus transaction. [4](#0-3) 

**P2P-sync path: wrong hash**

`get_tip_resource_bounds_hash` includes the `l1_data_gas` field in the Poseidon preimage only for `AllResources`, not for `L1Gas`: [5](#0-4) 

After deserialization as `L1Gas`, the recomputed hash omits the `l1_data_gas` element, producing a different value than the hash that was originally committed and stored on-chain.

### Impact Explanation

**Consensus path (High — transaction conversion binds wrong type/hash):** A proposer that includes a valid `AllResources { l2_gas: 0, l1_data_gas: 0 }` invoke transaction in a block proposal will have that transaction rejected by every validator during `convert_consensus_tx_to_internal_consensus_tx`. The validator returns `DEPRECATED_RESOURCE_BOUNDS_ERROR` and cannot process the proposal, causing consensus to stall or the proposer's block to be permanently invalid.

**P2P-sync path (High — wrong hash):** A syncing node that receives a historical block containing such a transaction will recompute a different transaction hash than the one stored in the block header, causing `validate_transaction_hash` to fail and the block to be rejected, breaking state synchronization.

### Likelihood Explanation

A transaction with `AllResourceBounds { l1_gas: non-zero, l2_gas: {0,0}, l1_data_gas: {0,0} }` passes all current gateway stateless checks:

- `ZeroResourceBounds`: passes because `l1_gas` is non-zero, so `max_possible_fee > 0`.
- `MaxGasPriceTooLow`: only checks `l2_gas.max_price_per_unit >= min_gas_price`; when `min_gas_price = 0` (a valid configuration), `l2_gas.max_price_per_unit = 0` is accepted.
- `MaxGasAmountTooHigh`: only rejects values that are too *high*, not zero. [6](#0-5) 

Such a transaction would be accepted by the gateway, placed in the mempool, and included in a block proposal by the batcher — but then rejected by every validator during consensus deserialization.

### Recommendation

Replace the zero-value heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminator field in the protobuf schema (e.g., a `bool is_all_resources` or an enum tag), so that the variant is preserved losslessly across the wire. Alternatively, always deserialize to `AllResources` in the consensus/mempool path (as the `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` converter in `rpc_transaction.rs` already does) and reserve the `L1Gas` variant only for legacy p2p-sync of pre-0.13.3 blocks. [7](#0-6)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
