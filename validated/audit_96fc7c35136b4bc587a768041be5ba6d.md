Audit Report

## Title
SwapAllowlistExtension Gates Router Address Instead of Originating User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`, `metric-core/contracts/MetricOmmPool.sol`)

## Summary

`MetricOmmPool.swap` unconditionally passes `msg.sender` as `sender` to `_beforeSwap`, which forwards it to `SwapAllowlistExtension.beforeSwap`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the router contract address, not the originating user. `SwapAllowlistExtension` checks `allowedSwapper[pool][sender]` where `sender` is the router, creating an irresolvable dilemma: either the router is allowlisted (bypassing per-user restrictions for everyone) or it is not (breaking router-based swaps for all users including legitimate ones).

## Finding Description

**Root cause — `MetricOmmPool.swap` (L231):**

`msg.sender` is hardcoded as the `sender` argument passed to `_beforeSwap`: [1](#0-0) 

**Propagation — `ExtensionCalling._beforeSwap` (L149–177):**

`sender` is encoded and dispatched to every configured extension in order, including `SwapAllowlistExtension`: [2](#0-1) 

**Guard — `SwapAllowlistExtension.beforeSwap` (L37):**

The check is `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When the call originates from `MetricOmmSimpleRouter`, `sender = router address`: [3](#0-2) 

**Router never injects originating user — `MetricOmmSimpleRouter` (L72–80, L104–112):**

`exactInputSingle` and `exactInput` pass `params.extensionData` / `params.extensionDatas[i]` directly to the pool without prepending `msg.sender`. The extension's `beforeSwap` also ignores `extensionData` entirely (unnamed last parameter): [4](#0-3) [5](#0-4) 

**Call chain:**
```
bannedUser → MetricOmmSimpleRouter.exactInput(pool, ...)
               → MetricOmmPool.swap(recipient, ..., extensionData)
                    msg.sender = router
                    → _beforeSwap(sender=router, ...)
                         → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                              checks allowedSwapper[pool][router]  ← not the user
```

The actual user's address is never presented to the guard. The `extensionData` field is ignored by the extension.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (KYC'd users, whitelisted market makers, protocol-controlled accounts) is rendered structurally ineffective. Any unprivileged user can execute swaps against the restricted pool by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`. This constitutes broken core pool functionality: the configured allowlist guard is bypassed on every router-mediated swap. Pools whose LP positions are held by the protocol itself are exposed to unauthorized draining of liquidity. Severity: **Medium** (broken invariant with a reachable, unprivileged trigger; direct fund impact depends on pool configuration but the guard is structurally defeated).

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract — any user can call it with no special role, token balance, or admin action required. The bypass is automatic: any swap submitted via the router presents the router's address to the allowlist check, not the user's. Likelihood: **High**.

## Recommendation

`SwapAllowlistExtension` must check the originating user's identity, not the intermediary router's address. The cleanest fix: `MetricOmmSimpleRouter` appends `abi.encode(msg.sender)` to `extensionData` before calling `pool.swap`, and `SwapAllowlistExtension.beforeSwap` decodes and checks it when `sender` is a known/trusted router address. Alternatively, expose a `trueSender` field in the pool's swap interface that is validated against `msg.sender` or a trusted forwarder registry and passed to extensions instead of raw `msg.sender`.

## Proof of Concept

```solidity
// Pool deployed with SwapAllowlistExtension.
// Admin allowlists only trustedUser: allowedSwapper[pool][trustedUser] = true
// Router is NOT allowlisted: allowedSwapper[pool][router] = false

// Scenario A: bannedUser routes through router
vm.prank(bannedUser);
router.exactInputSingle(ExactInputSingleParams({pool: pool, ...}));
// pool.swap called with msg.sender = router
// SwapAllowlistExtension checks allowedSwapper[pool][router] = false → REVERTS
// But also: trustedUser cannot swap via router either → core functionality broken

// Scenario B: admin adds router to fix Scenario A
// allowedSwapper[pool][router] = true

vm.prank(bannedUser);
router.exactInputSingle(ExactInputSingleParams({pool: pool, ...}));
// sender = router → allowedSwapper[pool][router] = true → PASSES
// bannedUser successfully swaps against the restricted pool
```
