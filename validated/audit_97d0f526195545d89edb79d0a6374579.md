### Title
`ContractOwner` Cannot Receive ETH, Making `DirectDepositV1` Native ETH Permanently Unwithdrawable — (File: `core/contracts/ContractOwner.sol`)

---

### Summary
`ContractOwner` is the owner of every `DirectDepositV1` instance it deploys. The only path to recover native ETH from a `DirectDepositV1` is `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`, which routes ETH through `ContractOwner` itself. However, `ContractOwner` has no `receive()` or `fallback()` function, so it cannot accept ETH. The intermediate ETH transfer always reverts, making any native ETH that accumulates in `DirectDepositV1` permanently unrecoverable.

---

### Finding Description

`DirectDepositV1` has a `receive()` function that attempts to wrap incoming ETH to WETH: [1](#0-0) 

Its constructor also handles a pre-funded balance at the CREATE2 address, but uses a soft-failure path — if wrapping fails, it emits `NativeTokenTransferFailed` and continues, leaving ETH stranded: [2](#0-1) 

`DirectDepositV1` does expose a `withdrawNative()` function, but it is `onlyOwner`: [3](#0-2) 

The owner is always `ContractOwner`, because `ContractOwner.createDirectDepositV1` is the sole deployer: [4](#0-3) 

The intended recovery path in `ContractOwner` is `withdrawFromDirectDepositV1(subaccount, address(0))`: [5](#0-4) 

This calls `DirectDepositV1.withdrawNative()`, which executes `msg.sender.call{value: balance}("")` — where `msg.sender` is `ContractOwner`. Since `ContractOwner` has no `receive()` or `fallback()` function (confirmed: the only `receive()` functions in the codebase are in `DirectDepositV1` and `MockWrappedNative`), the ETH transfer fails. The `require(success, "Failed to transfer native token to owner")` inside `withdrawNative()` then reverts the entire call. [3](#0-2) 

---

### Impact Explanation

Any native ETH that ends up in a `DirectDepositV1` instance is permanently stuck. The `withdrawFromDirectDepositV1(subaccount, address(0))` function is the only owner-accessible ETH recovery path, and it always reverts. ETH can accumulate via:

1. **Constructor pre-funding failure**: If the CREATE2 address is pre-funded with ETH and the WETH wrapping fails at construction time, the constructor silently emits `NativeTokenTransferFailed` and leaves ETH stranded.
2. **`selfdestruct` force-send**: A third party can force ETH into `DirectDepositV1` by selfdestructing a contract with it as the beneficiary, bypassing `receive()` entirely.

**Impact: Medium** — results in permanently stuck ETH (gas refund value or accidental sends), mirroring the original report's impact classification.

---

### Likelihood Explanation

**Likelihood: Medium** — The constructor soft-failure path is reachable if `wrappedNative` is misconfigured or temporarily broken at deployment time. Force-sends via `selfdestruct` are always possible by any external actor. Both scenarios are realistic in production.

---

### Recommendation

Add a `receive()` function to `ContractOwner` so it can accept ETH forwarded from `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

Alternatively, refactor `DirectDepositV1.withdrawNative()` to accept a `recipient` address parameter so ETH can be sent directly to the final destination without routing through `ContractOwner`.

---

### Proof of Concept

1. Deploy `ContractOwner` and call `createDirectDepositV1(subaccount)`.
2. Force-send ETH to the deployed `DirectDepositV1` via `selfdestruct` (or pre-fund the CREATE2 address before deployment with a failing WETH wrapper).
3. Call `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))` as the owner.
4. Internally, `DirectDepositV1.withdrawNative()` executes `ContractOwner.call{value: balance}("")`.
5. `ContractOwner` has no `receive()` function → the call returns `success = false`.
6. `require(success, "Failed to transfer native token to owner")` reverts.
7. ETH remains permanently locked in `DirectDepositV1` with no recovery path. [6](#0-5) [3](#0-2)

    function withdrawFromDirectDepositV1(bytes32 subaccount, address token)
        external
        onlyOwner
    {
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""
            );
            require(success, "xfer");
        } else {
            uint256 preBalance = IERC20Base(token).balanceOf(address(this));
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(token));
            uint256 postBalance = IERC20Base(token).balanceOf(address(this));
            require(postBalance > preBalance, "empty");
            IERC20Base(token).safeTransfer(
                msg.sender,
                postBalance - preBalance
            );
        }
    }
