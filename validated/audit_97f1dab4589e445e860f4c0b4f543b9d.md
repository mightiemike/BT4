### Title
Gateway `virtual_os_config_hash` computed from stale `starknet_version` at `V0_14_3` cutover causes proof-facts admission mismatch — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful validator builds a `BlockContext` for proof-facts validation by reading the current block's `block_info` and incrementing only `block_number`, leaving `starknet_version` unchanged. `virtual_os_config_hash` is derived from `block_info.starknet_version`. At the `V0_14_3` cutover (`OS_CONFIG_HASH_V4_CUTOVER`), the hash algorithm switches from Pedersen (`StarknetOsConfig3`) to Blake (`StarknetOsConfig4`). If the batcher builds the next block under `V0_14_3` while the gateway still reads `V0_14_2` from the committed chain tip, the two components compute different `virtual_os_config_hash` values for the same chain configuration. The gateway then rejects valid proof-facts transactions (Blake config hash) and accepts invalid ones (Pedersen config hash), which the batcher subsequently rejects.
