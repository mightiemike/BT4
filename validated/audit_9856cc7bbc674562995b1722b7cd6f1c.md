### Title
Unauthenticated `ContractCodeResponse::V1` Accepted Without Source Verification Before `SignedContractCodeResponse` Activation — (File: `chain/client/src/stateless_validation/validate.rs`)

### Summary

Before `ProtocolFeature::SignedContractCodeResponse` is enabled, `ContractCodeResponse::V1` carries no signature and no authentication. `validate_contract_code_response` skips signature validation entirely for pre-feature epochs. Any unprivileged network peer can send a `ContractCodeResponse::V1` with arbitrary contract code for a valid `ChunkProductionKey`, and the receiving validator will accept and store it without any authentication check — an exact analog of the Solidity contract's missing "used-parameters" tracking.

### Finding Description

`ContractCodeResponseV1` has no `signature` field and no authentication mechanism: [1](#0-0) 

`ContractCodeResponse::verify_signature` explicitly returns `false` for V1 — there is nothing to verify: [2](#0-1) 

In `validate_contract_code_response`, the signature check is gated entirely on `ProtocolFeature::SignedContractCodeResponse`. Before that feature activates, the function returns `ChunkRelevance::Relevant` after only a staleness check, with no authentication of the sender: [3](#0-2) 

The handler then stores whatever contracts the response contains, keyed only by `ChunkProductionKey`: [4](#0-3) 

Contrast this with `VersionedPartialEncodedStateWitness`, which has an explicit version gate (`witness_version_mismatch`) that drops the wrong-version message before any processing, and whose inner types use distinct `signature_differentiator` strings to prevent cross-version grafting: [5](#0-4) [6](#0-5) 

`ContractCodeResponse` has no equivalent version gate and no differentiator in V1.

The `validate_witness_contract_code_response_signature` function does reject an unsigned response *after* the feature is enabled (by checking `response.responder()` returns `None`): [7](#0-6) 

But this gate is never reached before the feature activates, leaving the entire pre-feature window open.

### Impact Explanation

An unprivileged network peer that knows the `ChunkProductionKey` for an upcoming chunk (broadcast publicly) can race the legitimate chunk producer and deliver a `ContractCodeResponse::V1` containing arbitrary compressed contract bytes. The validator stores those bytes under the correct key. When it subsequently attempts to validate the chunk, it uses the injected code; the code hash will not match the hash recorded in the state witness, causing validation to fail and the validator to withhold its endorsement. Targeting a sufficient fraction of the chunk-validator set for a shard prevents the chunk from accumulating the required endorsements, stalling chunk production for that shard. This is a reachable, unprivileged-peer-triggered consensus-liveness failure.

### Likelihood Explanation

The `ChunkProductionKey` (`shard_id`, `epoch_id`, `height_created`) is public. Any peer connected to the network can observe it and send a spoofed response before the legitimate chunk producer does. No special keys or roles are required. The attack window is the entire period before `SignedContractCodeResponse` is activated on mainnet.

### Recommendation

1. Add a version gate in `validate_contract_code_response` that rejects `ContractCodeResponse::V1` once `SignedContractCodeResponse` is enabled, mirroring the `witness_version_mismatch` pattern used for `VersionedPartialEncodedStateWitness`.
2. Add a `signature_differentiator` field to `ContractCodeResponseV2Inner` (already present) and ensure V1 is structurally incapable of being confused with V2 at the Borsh level.
3. Consider verifying the code hash of each received contract against the set advertised in the corresponding `ChunkContractAccesses` message before storing, so injected code with a wrong hash is discarded immediately rather than stored and used.

### Proof of Concept

```
1. Attacker connects to a validator node as a normal network peer.
2. Attacker observes the ChunkProductionKey (shard_id, epoch_id, height_created)
   for an upcoming chunk — this is broadcast on the network.
3. Before SignedContractCodeResponse is enabled, attacker constructs:
     ContractCodeResponse::V1(ContractCodeResponseV1 {
         next_chunk: <valid ChunkProductionKey>,
         compressed_contracts: <compressed arbitrary bytes>,
     })
4. Attacker sends this message to the target validator.
5. validate_contract_code_response:
     - validate_chunk_relevant passes (chunk is current)
     - SignedContractCodeResponse is NOT enabled → signature check skipped
     - returns ChunkRelevance::Relevant
6. handle_contract_code_response calls
     partial_witness_tracker.store_accessed_contract_codes(key, injected_contracts)
7. Validator later attempts chunk validation using the injected code.
   Code hash mismatch → validation fails → endorsement withheld.
8. Repeat for enough validators in the assignment set → chunk stalls.
```

pub fn validate_contract_code_response(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
    store: &Store,
) -> Result<ChunkRelevance, Error> {
    let key = response.chunk_production_key();
    require_relevant!(validate_chunk_relevant(epoch_manager, key, store)?);
    let protocol_version = epoch_manager.get_epoch_protocol_version(&key.epoch_id)?;
    if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
        validate_witness_contract_code_response_signature(epoch_manager, response)?;
    }

    Ok(ChunkRelevance::Relevant)
}

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())

    /// Handles contract code responses message from chunk producer.
    fn handle_contract_code_response(&self, response: ContractCodeResponse) -> Result<(), Error> {
        if !validate_contract_code_response(
            self.epoch_manager.as_ref(),
            &response,
            self.runtime.store(),
        )?
        .is_relevant()
        {
            return Ok(());
        }
        let key = response.chunk_production_key().clone();
        let contracts = response.decompress_contracts()?;
        self.partial_witness_tracker.store_accessed_contract_codes(key, contracts)
    }
