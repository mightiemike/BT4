### Title
Proof Manager Stores Proofs Without Sender Binding, Allowing Any Sender to Bypass Proof Verification - (`crates/apollo_transaction_converter/src/transaction_converter.rs`)

### Summary

The `ProofFacts::hash()` function used as the proof manager's storage key does not include the `sender_address`. Combined with `run_proof_verification` checking `contains_proof` before calling `verify_proof`, any sender can submit a transaction with another sender's `proof_facts` and an invalid/garbage proof, and the gateway will accept it without verifying the proof.

### Finding Description

The `ProofFacts::hash()` function hashes only the proof facts content (proof version, program hash, block number, block hash, config hash) with no binding to the submitting account:

```rust
pub fn hash(&self) -> Felt {
    HashChain::new().chain_iter(self.0.iter()).get_poseidon_hash()
}
``` [1](#0-0) 

The proof manager stores and retrieves proofs keyed exclusively by this sender-agnostic hash:

```rust
pub async fn set_proof(&self, proof_facts: ProofFacts, proof: Proof) -> ... {
    if self.contains_proof(proof_facts.clone()).await? { return Ok(()); }
    let facts_hash = proof_facts.hash();
    self.proof_storage.set_proof(facts_hash, proof.clone()).await?;
    ...
}
``` [2](#0-1) 

The critical gate in `run_proof_verification` checks `contains_proof` **before** calling `verify_proof`. If the proof is already stored (by any sender), the cryptographic proof supplied by the current caller is never verified:

```rust
async fn run_proof_verification(proof_facts, proof, pmc) -> Result<bool, ...> {
    let contains_proof = pmc.contains_proof(proof_facts.clone()).await?;
    if contains_proof {
        return Ok(false);  // Skip verification entirely
    }
    verify_proof(proof_facts, proof)?;
    Ok(true)
}
``` [3](#0-2) 

This function is used in both the gateway path (`spawn_proof_verification`) and the consensus path (`spawn_verify_and_store_proof`): [4](#0-3) [5](#0-4) 

The transaction hash for invoke V3 does include both `sender_address` and `proof_facts_hash`, so the signature covers the proof_facts. However, the `proof` field itself is explicitly dropped during conversion to the internal representation:

```rust
impl From<RpcInvokeTransactionV3> for InternalRpcInvokeTransactionV3 {
    fn from(tx: RpcInvokeTransactionV3) -> Self {
        Self {
            ...
            proof_facts: tx.proof_facts,
            // Note: proof field is dropped
        }
    }
}
``` [6](#0-5) 

The `proof_facts` content (block_number, block_hash, program_hash, config_hash) is entirely public — it appears in committed blocks and is observable by any network participant: [7](#0-6) 

### Impact Explanation

An attacker (Bob) can:

1. Observe Alice's accepted transaction containing `proof_facts` P (these are public in committed blocks and mempool gossip).
2. Submit his own invoke V3 transaction with the same `proof_facts` P but a garbage/single-byte `proof` field.
3. The gateway's `run_proof_verification` calls `contains_proof(P)` → `true` → returns `Ok(false)` without ever calling `verify_proof`.
4. Bob's transaction passes gateway admission with an invalid proof.
5. The blockifier's `validate_proof_facts` only checks the proof_facts content (block_number, block_hash, config_hash, program_hash) — it never verifies the cryptographic proof. [8](#0-7) 

Bob's transaction is accepted and executed with the `proof_facts`

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        if !os_constants.allowed_proof_versions.contains(&snos_proof_facts.proof_version.as_felt())
        {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Proof version {} is not allowed under this protocol version.",
                snos_proof_facts.proof_version
            )));
        }

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let virtual_os_config_hash = block_context.virtual_os_config_hash();
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }
