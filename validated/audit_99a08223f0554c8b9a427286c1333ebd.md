The `BurnNFT` keeper function at lines 141–161 of `x/nft/keeper/keeper.go` contains a real, concrete vulnerability. Here is the full analysis:

---

### Title
NFT Owner Who Is Not Denom Creator Cannot Burn Their Own NFT, Causing Permanent Lock — (`x/nft/keeper/keeper.go`)

### Summary

`BurnNFT` requires the caller to simultaneously satisfy **both** `IsOwner` (current NFT owner) **and** `IsDenomCreator` (original denom creator). After a legitimate `MsgTransferNFT`, these two conditions can never be satisfied by the same address, making the NFT permanently unburnable by anyone.

### Finding Description

The keeper's `BurnNFT` function performs two sequential authorization checks:

```go
// x/nft/keeper/keeper.go lines 141-161
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    ...
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // must be current owner
    if err != nil {
        return err
    }

    _, err = k.IsDenomCreator(ctx, denomID, owner)         // must ALSO be denom creator
    if err != nil {
        return err
    }
    ...
}
``` [1](#0-0) 

`IsOwner` checks that `owner` equals `nft.GetOwner()`: [2](#0-1) 

`IsDenomCreator` checks that `address` equals `denom.Creator`: [3](#0-2) 

`TransferOwner` (called by `MsgTransferNFT`) only checks `IsOwner` — it does **not** require the sender to be the denom creator, so transfers to non-creators are fully permitted: [4](#0-3) 

Once Alice (denom creator) transfers an NFT to Bob (non-creator):
- **Bob** calls `BurnNFT` → passes `IsOwner`, **fails** `IsDenomCreator` → `ErrUnauthorized`
- **Alice** calls `BurnNFT` → **fails** `IsOwner` (she no longer owns it) → `ErrUnauthorized`

No address can satisfy both checks simultaneously. The NFT is permanently unburnable via `MsgBurnNFT`.

The only escape hatch is `BurnNFTUnverified`, which skips the `IsDenomCreator` check, but that function is **not** reachable from any `MsgServer` handler — it is only called internally for IBC escrow logic: [5](#0-4) 

### Impact Explanation

Any NFT transferred from the denom creator to a third party becomes permanently unburnable through the public `MsgBurnNFT` transaction path. The NFT is locked in the recipient's account with no on-chain mechanism to destroy it. This constitutes **permanent lock of an economically valuable NFT** — the exact Critical impact described in the scope.

### Likelihood Explanation

This is triggered by the ordinary, intended workflow: issue denom → mint NFT → transfer to another user → recipient tries to burn. No special privileges, governance, or key compromise are required. Any user who receives an NFT via `MsgTransferNFT` from the denom creator is immediately affected.

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Burning should only require proof of ownership (`IsOwner`). The denom creator restriction is appropriate for minting and editing, but not for destruction of property the caller legitimately owns. The corrected function should be:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

### Proof of Concept

A keeper test that proves the invariant breaks:

```go
// 1. Alice issues denom (Alice = denom creator)
k.IssueDenom(ctx, "testdenom", "Test", "", "", alice)

// 2. Alice mints NFT to herself
k.MintNFT(ctx, "testdenom", "token1", "T", "", "", alice, alice)

// 3. Alice transfers NFT to Bob
k.TransferOwner(ctx, "testdenom", "token1", alice, bob)

// 4. Bob (owner, non-creator) tries to burn → ErrUnauthorized (IsDenomCreator fails)
err := k.BurnNFT(ctx, "testdenom", "token1", bob)
require.ErrorContains(t, err, "is not the creator of")

// 5. Alice (creator, non-owner) tries to burn → ErrUnauthorized (IsOwner fails)
err = k.BurnNFT(ctx, "testdenom", "token1", alice)
require.ErrorContains(t, err, "is not the owner of")

// NFT still exists — permanently locked
require.True(t, k.HasNFT(ctx, "testdenom", "token1"))

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
