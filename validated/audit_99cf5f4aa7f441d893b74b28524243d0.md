Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of Original User, Allowing Full Allowlist Bypass via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router address, not the original user. Once the pool admin allowlists the router (a necessary operational step for any allowlisted user to trade via the router), the allowlist check passes for every caller regardless of whether they are individually authorized.

## Finding Description
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool (enforced by `onlyPool`) and `sender` is the first argument passed by the pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L31-41
function beforeSwap(address sender, ...)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap()
    recipient, ...
);
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the pool's `msg.sender`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
```

The router forwards `params.extensionData` verbatim from the caller — it never encodes the original `msg.sender` into it. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][original_user]`. The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput` paths. There is no existing guard that recovers the original user identity from `extensionData` or any other channel.

## Impact Explanation
A pool deploying `SwapAllowlistExtension` to restrict trading to a known set of addresses (KYC-verified counterparties, institutional LPs, whitelisted market makers) loses that restriction entirely for any user routing through `MetricOmmSimpleRouter`. Non-allowlisted users can execute swaps against the pool at oracle-anchored prices the pool admin intended to offer only to approved parties. This is a direct, repeatable loss of the curation guarantee and constitutes unauthorized fund flows out of the pool — a broken core pool functionality and admin-boundary break meeting Sherlock High/Critical thresholds.

## Likelihood Explanation
The bypass requires the router to be allowlisted. This is a realistic and expected operational step: without it, even individually allowlisted users cannot use the router, making the router useless for curated pools. The protocol provides no warning that allowlisting the router opens the pool to all users. Any pool admin enabling router-based trading for their allowlisted users will inadvertently enable it for everyone. The attack requires no special privileges — any unprivileged address can call `router.exactInputSingle()`.

## Recommendation
The extension must verify the original user, not the intermediary. The most robust fix is to have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` (or a dedicated field) and have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known router. Alternatively, the extension can maintain a registry of trusted routers and, when `sender` is a trusted router, require the `extensionData` payload to carry the original user's address. A documentation-only fix (never allowlist the router) is not a code-level fix and severely limits usability.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is individually authorized.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — router is allowlisted so Alice can use it.
4. Bob (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(...)` with `msg.sender = router`.
6. Pool calls `extension.beforeSwap(router, ...)` via `_beforeSwap(msg.sender, ...)`.
7. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
8. Bob successfully trades on a pool he was never authorized to access.

Foundry test plan: deploy `SwapAllowlistExtension`, configure a pool with it, allowlist only Alice and the router, then `vm.prank(bob); router.exactInputSingle(...)` and assert the swap succeeds (no `NotAllowedToSwap` revert). [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
