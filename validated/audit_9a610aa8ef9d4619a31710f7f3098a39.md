### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables usdcE Drain from Any DDA Without Providing USDC - (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` performs a raw `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` at line 616 without checking the return value or using the `safeTransferFrom` wrapper that the rest of the codebase consistently uses. The function also carries no `onlyOwner` guard — only a `block.chainid == 57073` check — making it callable by any unprivileged user on the Ink chain. If the `transferFrom` returns `false` instead of reverting, the subsequent `DirectDepositV1.withdraw(usdcE)` and `safeTransfer(usdcE, msg.sender, balance)` still execute, allowing the caller to receive usdcE from the DDA without having deposited any USDC.

---

### Finding Description

`replaceUsdcEWithUsdc` is a migration helper intended to swap usdcE held in a `DirectDepositV1` (DDA) contract for native USDC. Its logic is:

1. Read the usdcE balance of the DDA.
2. Pull USDC from `msg.sender` into the DDA to replace it.
3. Withdraw usdcE from the DDA back to `ContractOwner`.
4. Send usdcE to `msg.sender`.

Step 2 uses a bare `transferFrom`:

```solidity
// ContractOwner.sol L616
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
```

Every other token pull in the codebase goes through `ERC20Helper.safeTransferFrom`, which wraps the low-level call and `require`s a truthy return:

```solidity
// ERC20Helper.sol L29-41
(bool success, bytes memory data) = address(self).call(...);
require(
    success && (data.length == 0 || abi.decode(data, (bool))),
    ERR_TRANSFER_FAILED
);
```

The bare call at L616 discards the boolean return. If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on Ink returns `false` on failure (e.g., insufficient allowance, insufficient balance, or any future upgrade that adopts a non-reverting path), execution continues silently into:

```solidity
// L617-618
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
IERC20Base(usdcE).safeTransfer(msg.sender, balance);
```

`DirectDepositV1.withdraw` transfers the full usdcE balance to `ContractOwner`, and `safeTransfer` forwards it to `msg.sender` — all without the caller having provided any USDC.

The function has no `onlyOwner` modifier:

```solidity
// L608-609
function replaceUsdcEWithUsdc(bytes32 subaccount) external {
    require(block.chainid == 57073, ERR_UNAUTHORIZED);
```

Any address on chain 57073 can supply an arbitrary `subaccount` value.

---

### Impact Explanation

An attacker who calls `replaceUsdcEWithUsdc` with a subaccount whose DDA holds usdcE, while holding zero USDC allowance, receives the full usdcE balance of that DDA for free if the USDC `transferFrom` fails silently. The corrupted asset delta is: DDA usdcE balance → attacker wallet, with no USDC deposited in return. Every DDA on chain 57073 with a non-zero usdcE balance is at risk.

---

### Likelihood Explanation

The function is externally callable by any unprivileged user on chain 57073. The only precondition is that a DDA exists with a non-zero usdcE balance, which is the exact scenario the function is designed to handle. Exploitability is conditional on the USDC token returning `false` rather than reverting on a failed transfer; however, the absence of the return-value check is a latent vulnerability that violates the invariant the rest of the codebase enforces uniformly.

---

### Recommendation

Replace the bare `transferFrom` with the `safeTransferFrom` wrapper already used everywhere else in the protocol:

```solidity
// Before (L616)
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

Additionally, add an `onlyOwner` modifier to `replaceUsdcEWithUsdc` to restrict the migration path to the protocol owner, consistent with the intent of the function.

---

### Proof of Concept

1. A DDA for `subaccount` holds `N` usdcE on chain 57073.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance.
3. `IERC20Base(usdc).transferFrom(attacker, dda, N)` returns `false` — no USDC moves.
4. `DirectDepositV1(dda).withdraw(usdcE)` executes: `N` usdcE moves from DDA → ContractOwner.
5. `IERC20Base(usdcE).safeTransfer(attacker, N)` executes: `N` usdcE moves from ContractOwner → attacker.
6. Attacker holds `N` usdcE; DDA holds 0 usdcE; no USDC was ever deposited.

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
