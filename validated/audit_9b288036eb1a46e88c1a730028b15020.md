Audit Report

## Title
`SwapAllowlistExtension` allowlist fully bypassed when router is allowlisted — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool call — the direct caller, not the economic actor. When `MetricOmmSimpleRouter` is used, `sender` is always the router address. A pool admin who allowlists the router to permit their curated users to swap via the router inadvertently grants every user unrestricted swap access, completely defeating the allowlist.

## Finding Description

**Step 1 — Pool binds `msg.sender` as `sender`:**

`MetricOmmPool.swap` passes `msg.sender` (the direct pool caller) as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value unchanged into the ABI-encoded call to each extension: [2](#0-1) 

**Step 2 — Extension checks only `sender`:**

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` (pool → direct caller): [3](#0-2) 

The actual economic user's address is never visible to the extension.

**Step 3 — Router is always the pool's `msg.sender`:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**The broken invariant:**

A pool admin who wants allowlisted users to be able to use the router must call `setAllowedToSwap(pool, router, true)`. Once the router is in `allowedSwapper[pool]`, the extension evaluates `allowedSwapper[pool][router] == true` for every caller regardless of who the actual user is. There is no mechanism in the current design for the extension to recover the real user's address from the router call, so the allowlist is unconditionally bypassed for all router users. [6](#0-5) 

## Impact Explanation

The `SwapAllowlistExtension` is a core access-control hook for permissioned pools. When the router is allowlisted (the only way to let curated users route through it), the allowlist is rendered completely ineffective: any unprivileged, non-allowlisted user can call `exactInputSingle` on the router and swap in a pool that is supposed to be restricted. This breaks the core pool functionality of access-gated swaps and constitutes an admin-boundary break where an unprivileged path bypasses the pool admin's intended access control.

## Likelihood Explanation

The precondition — the pool admin having allowlisted the router — is a natural and expected operational step for any permissioned pool that also wants to support router-based UX. Once that step is taken, the bypass is trivially reachable by any user with no special capability: they simply call `MetricOmmSimpleRouter.exactInputSingle` with the target pool. The attack is repeatable and requires no privileged access.

## Recommendation

The extension needs the actual economic user's address, not the intermediary's. Two viable approaches:

1. **Pass the real user through `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify it when `sender` is a known router. This requires a trusted router registry in the extension.
2. **Dedicated router allowlist path:** Add a separate `allowedRouter` mapping. When `sender` is an allowlisted router, decode the real user from `extensionData` and check `allowedSwapper[pool][realUser]` instead.

Either approach must ensure the user address in `extensionData` cannot be spoofed by an arbitrary caller (i.e., only trusted routers should be permitted to assert a user identity).

## Proof of Concept

```solidity
// Setup: pool admin allowlists the router so curated users can use it
swapAllowlistExtension.setAllowedToSwap(pool, address(router), true);
// Alice is an allowlisted user
swapAllowlistExtension.setAllowedToSwap(pool, alice, true);

// Attack: Bob (not allowlisted) calls the router directly
vm.prank(bob); // bob is NOT in allowedSwapper[pool]
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: bob,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// Extension checks allowedSwapper[pool][router] == true → passes
// Bob successfully swaps despite not being allowlisted
```
