Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the end-user, making the allowlist bypassable or permanently broken for router-mediated swaps - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` validates the `sender` argument, which is `msg.sender` of the `pool.swap()` call. When a user swaps through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks whether the **router** is allowlisted rather than the actual end-user. This produces two mutually exclusive fund-impacting failure modes: if the router is allowlisted, every user bypasses the allowlist; if the router is not allowlisted, every individually-allowlisted user is blocked from using the only supported periphery swap path.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool (the extension's caller) and `sender` is the direct caller of `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly, making `msg.sender` to the pool the **router address**: [4](#0-3) 

The same mismatch applies to `exactInput` (line 104), `exactOutputSingle` (line 136), `exactOutput` (line 165), and the recursive hop inside `_exactOutputIterateCallback` where the pool is called again with `msg.sender` = router: [5](#0-4) 

The router stores the original `msg.sender` only in transient storage as the **payer** for the callback, but this value is never forwarded to the pool or the extension: [6](#0-5) 

No existing guard in `SwapAllowlistExtension` or `MetricOmmPool` compensates for this indirection. The extension has no visibility into who originally called the router.

## Impact Explanation

Two mutually exclusive failure modes, both fund-impacting:

**Mode A – Allowlist bypass (High):** The pool admin allowlists the router so that users can reach the pool through the supported periphery path. Because the extension checks `allowedSwapper[pool][router]`, every user who calls the router is treated as allowlisted regardless of their individual status. Any non-allowlisted address can swap on a curated pool by routing through `MetricOmmSimpleRouter`, draining liquidity that the pool admin intended to restrict.

**Mode B – Allowlisted users permanently blocked (Medium):** The pool admin does not allowlist the router (only individual users). Every individually-allowlisted user who calls the router is blocked because the extension sees the router address, which is not in the allowlist. The pool's swap functionality is broken for all router-mediated paths, which is the only supported periphery swap path.

Both outcomes break the core invariant that a curated pool enforces the same allowlist policy regardless of which supported public entrypoint reaches it, constituting broken core pool functionality causing loss of funds or unusable swap flows.

## Likelihood Explanation

The router is the primary and documented user-facing swap entrypoint. Any pool that deploys `SwapAllowlistExtension` and expects users to swap through the router will immediately encounter one of the two failure modes on the very first public `exactInputSingle` call. No special timing, privileged access, flash loans, or unusual token behavior is required. The bug is triggered by the normal, documented usage pattern.

## Recommendation

The extension must check the economically relevant actor — the address that initiated the swap and will pay for it — not the intermediate contract that called the pool. Two options:

1. **Pass the original initiator through the router.** Have the router store `msg.sender` in transient storage (analogous to how it already stores the payer via `_setNextCallbackContext`) and expose it so the extension can read it directly from the router's transient slot.

2. **Validate a caller-supplied field in `extensionData`.** Have the router populate `extensionData` with `msg.sender` and have the extension validate that field when `sender` is a known router address, requiring the router to be trusted for this path.

The simplest safe fix is option 1: store the original `msg.sender` in transient storage at router entry and have the extension read it, so the checked identity is always the address that economically initiated the swap.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, alice, true)   // alice is allowlisted
  admin does NOT allowlist bob
  admin calls setAllowedToSwap(pool, router, true)  // router allowlisted so users can swap

Attack (Mode A – bypass):
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...)         // msg.sender to pool = router
    → pool calls _beforeSwap(router, ...)
    → SwapAllowlistExtension.beforeSwap(router, ...)
    → checks allowedSwapper[pool][router] == true    ✓ passes
    → bob's swap executes despite not being allowlisted

Broken invariant (Mode B – alice blocked):
  admin does NOT allowlist router
  alice calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...)         // msg.sender to pool = router
    → pool calls _beforeSwap(router, ...)
    → SwapAllowlistExtension.beforeSwap(router, ...)
    → checks allowedSwapper[pool][router] == false   ✗ reverts NotAllowedToSwap
    → alice is blocked even though she is individually allowlisted

Foundry test plan:
  1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  2. Deploy MetricOmmSimpleRouter pointing to the same factory.
  3. allowlist alice directly; allowlist router for Mode A test.
  4. Call router.exactInputSingle as bob → assert swap succeeds (Mode A bypass).
  5. Remove router from allowlist; call router.exactInputSingle as alice → assert revert NotAllowedToSwap (Mode B block).
```
