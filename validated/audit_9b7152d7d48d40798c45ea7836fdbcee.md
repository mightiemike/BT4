Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypass via Router — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which is forwarded to `SwapAllowlistExtension.beforeSwap()`. When a user swaps through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router address, not the originating user. If the pool admin allowlists the router (required for any user to use the router on an allowlisted pool), every unprivileged user can bypass the per-user allowlist by routing through the public router.

## Finding Description
**Root cause — `sender` is the direct pool caller, not the originating user:**

In `MetricOmmPool.sol::swap()` (line 231), `msg.sender` is passed as `sender` to `_beforeSwap`:
```solidity
_beforeSwap(
    msg.sender,   // ← always the direct caller of pool.swap()
    recipient, ...
);
```

In `MetricOmmSimpleRouter.sol::exactInputSingle()` (lines 72–80), the router calls `pool.swap()` directly with no mechanism to forward the originating user:
```solidity
IMetricOmmPoolActions(params.pool).swap(
    params.recipient, params.zeroForOne, ..., params.extensionData
);
// msg.sender seen by the pool = address(router), not the EOA
```

`SwapAllowlistExtension.beforeSwap()` (line 37) checks:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert NotAllowedToSwap();
}
// msg.sender = pool, sender = router address (not the EOA)
```

**Exploit flow:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to a set of known addresses.
2. Pool admin allowlists the router address (`setAllowedToSwap(pool, router, true)`) so that legitimate allowlisted users can use the router.
3. Attacker (not individually allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle()`.
4. The router calls `pool.swap()`; the pool sees `msg.sender = router`.
5. `SwapAllowlistExtension.beforeSwap()` receives `sender = router`, finds it allowlisted, and passes.
6. Attacker's swap executes on a pool that was intended to be restricted.

**Existing guards are insufficient:** The `onlyPool` modifier on `BaseMetricExtension` only verifies the caller of `beforeSwap` is a known pool — it does not verify the identity of the originating user. There is no mechanism in the pool or the router to forward the real EOA as `sender`.

## Impact Explanation
The `SwapAllowlistExtension` is a core access-control gate for restricted pools. Its bypass allows any unprivileged user to execute swaps on pools that pool admins intended to restrict to a specific set of addresses. This constitutes broken core pool functionality (allowlist gate failure) and can result in unauthorized trading activity on pools designed for permissioned participants (e.g., KYC-gated, institutional, or protocol-internal pools). Severity: **Medium** (broken core functionality; requires router to be allowlisted as a precondition, but this is the only practical way to allow allowlisted users to use the router).

## Likelihood Explanation
Any user who knows the pool uses `SwapAllowlistExtension` and that the router is allowlisted can exploit this with a single public call to `MetricOmmSimpleRouter`. No special privileges, flash loans, or complex setup are required. The condition (router allowlisted) is a natural operational state whenever the pool admin wants allowlisted users to be able to use the router. Repeatable at will.

## Recommendation
Pass the originating user through the call chain. Options:
1. Add an `originator` parameter to `pool.swap()` and have the router pass `msg.sender` explicitly; validate it in the pool (e.g., require `originator == msg.sender` for direct calls).
2. Alternatively, have `SwapAllowlistExtension.beforeSwap()` check both `sender` (the direct caller) and a separately forwarded originator field in `extensionData`.
3. At minimum, document that allowlisting the router grants all users swap access, and provide a separate router-aware allowlist that checks the payer stored in transient callback context.

function test_swapAllowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, only `trustedUser` allowlisted
    // Router is allowlisted so trustedUser can use it
    extension.setAllowedToSwap(address(pool), address(router), true);

    // Attacker (not individually allowlisted) calls router
    vm.prank(attacker);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: token0,
        tokenOut: token1,
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        recipient: attacker,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    }));
    // Swap succeeds — allowlist bypassed
}
