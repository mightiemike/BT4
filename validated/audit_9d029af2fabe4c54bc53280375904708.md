### Title
Bonus Accrual Checkpoint Advances Without Payment When Pool Is Insufficient During Redelegation Slash — (`x/tieredrewards/keeper/slash.go`, `claim_rewards.go`)

### Summary

When `BeforeRedelegationSlashed` fires and the `RewardsPoolName` balance is below the computed bonus, `processEventsAndClaimBonus` advances `pos.LastBonusAccrual` to `blockTime` **before** the pool-sufficiency check. The `ErrInsufficientBonusPool` error is then swallowed in `slashRedelegationPosition`, and the position is persisted with the advanced checkpoint. All bonus rewards accrued up to the slash event are permanently lost for the position owner.

### Finding Description

**Step 1 — Entry point.**
`BeforeRedelegationSlashed` (hooks.go:128-130) delegates unconditionally to `slashRedelegationPosition`. [1](#0-0) 

**Step 2 — Pointer mutation before pool check.**
`processEventsAndClaimBonus` receives `&pos` (a pointer). At line 215 it calls `applyBonusAccrualCheckpoint`, which writes the new `LastBonusAccrual` directly into the pointed-to struct. Only *after* that, at line 230, does it check `sufficientBonusPoolBalance`. If the pool is short, it returns `nil, ErrInsufficientBonusPool` — but the in-memory mutation at line 215 has already happened. [2](#0-1) 

**Step 3 — Error swallowed.**
Back in `slashRedelegationPosition`, the `ErrInsufficientBonusPool` branch is explicitly caught and only logged; execution continues with the mutated `pos`. [3](#0-2) 

**Step 4 — Mutated state persisted.**
`setPositionWithState` is called on the now-mutated `pos`, writing the advanced `LastBonusAccrual` to the store with no bonus coins ever sent. [4](#0-3) 

**Step 5 — Future claims start from the advanced checkpoint.**
Any subsequent call to `processEventsAndClaimBonus` uses `pos.LastBonusAccrual` as `segmentStart` (claim_rewards.go:165). Because it was already advanced to `blockTime`, the entire accrual window before the slash is skipped permanently. [5](#0-4) 

### Impact Explanation

A position owner permanently loses all bonus rewards accrued from their previous `LastBonusAccrual` up to the block time of the slash event. The lost amount equals `shares × tokensPerShare × tier.BonusApy × duration / SecondsPerYear` for the entire unprocessed window. There is no recourse: the checkpoint cannot be rolled back, and the accrual window is gone. [6](#0-5) 

### Likelihood Explanation

The precondition — `RewardsPoolName` balance below the computed bonus — is realistic:
- The pool is funded externally and can be depleted by concurrent claims.
- A newly deployed or underfunded chain can have an empty pool.
- A redelegation slash is a normal chain event (double-sign or downtime on the source validator during the redelegation unbonding period).

No attacker privilege is required; the slash is triggered by the staking module's normal slash path.

### Recommendation

Move `applyBonusAccrualCheckpoint` (and `UpdateLastKnownBonded`) to **after** the successful `SendCoinsFromModuleToAccount` call, so the checkpoint only advances when coins are actually transferred. Alternatively, if the pool is insufficient, skip the checkpoint update entirely and leave `LastBonusAccrual` unchanged so the accrual can be retried once the pool is refunded. [7](#0-6) 

### Proof of Concept

```
1. Create a tiered-rewards position P with a redelegation (unbondingId U).
2. Drain RewardsPoolName to 0 (e.g., via concurrent ClaimTierRewards calls).
3. Trigger BeforeRedelegationSlashed(U, sharesToUnbond) via a validator slash.
   → slashRedelegationPosition → processEventsAndClaimBonus:
       line 215: pos.LastBonusAccrual = blockTime  (checkpoint advanced)
       line 230: sufficientBonusPoolBalance fails  (pool empty)
       returns ErrInsufficientBonusPool
   → error swallowed at slash.go:56-63
   → setPositionWithState persists pos with advanced LastBonusAccrual
4. Refund RewardsPoolName.
5. Call ClaimTierRewards for P.
   → processEventsAndClaimBonus: segmentStart = blockTime (already advanced)
   → bonus computed for [blockTime, now] only; pre-slash accrual is zero.
6. Assert: bonus received == 0 for the pre-slash period despite valid accrual.
```

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
