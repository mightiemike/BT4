### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Wrong Transaction Hash Preimage on Syncing Nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` classifies any transaction whose `l2_gas` and `l1_data_gas` fields are both zero as `ValidResourceBounds::L1Gas`, even when the original transaction was submitted and hashed as `AllResources`. Because `get_tip_resource_bounds_hash` includes a different number of resource felts depending on the variant (2 for `L1Gas`, 3 for `AllResources`), any hash recomputation from the deserialized form on a syncing node produces a value that diverges from the hash the user signed and the sequencer committed.

---

### Finding Description

**Conversion boundary — serialization path:**

`From<ValidResourceBounds> for protobuf::ResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` always emits all three fields for `AllResources`, including zero-valued ones: [1](#0-0) 

**Conversion boundary — deserialization path:**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in the same file collapses back to `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero, regardless of whether they were explicitly present: [2](#0-1) 

The round-trip is therefore **lossy**: `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` → protobuf → `L1Gas(l1_gas=X)`.

**Hash domain boundary:**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` includes a structurally different number of felts depending on the variant: [3](#0-2) 

- `L1Gas` → 2 resource felts hashed: `[tip, concat(L1Gas), concat(L2Gas=0)]`
- `AllResources` → 3 resource felts hashed: `[tip, concat(L1Gas), concat(L2Gas=0), concat(L1DataGas=0)]`

These produce distinct Poseidon digests for the same numeric field values.

**Origination path — sequencer node always uses `AllResources`:**

`InternalRpcInvokeTransactionV3` stores `AllResourceBounds` directly: [4](#0-3) 

The conversion to the storage/consensus type `InvokeTransactionV3` always wraps it as `AllResources`: [5](#0-4) 

So the sequencer computes and commits hash H1 (3-felt preimage). After p2p block sync, the receiving node holds the same transaction as `L1Gas`, and any recomputation yields H2 ≠ H1.

**Gateway accepts the triggering transaction:**

The stateless validator accepts `AllResourceBounds` with only `l1_gas` non-zero (l2_gas=0, l1_data_gas=0): [6](#0-5) 

---

### Impact Explanation

A syncing node that re-executes or re-hashes a block containing such a transaction will:

1. **Compute a wrong transaction hash** — the 2-felt `L1Gas` preimage diverges from the 3-felt `AllResources` preimage the sequencer signed and stored. Any block-hash verification that recomputes transaction hashes from stored `InvokeTransactionV3` data will fail or produce an incorrect block hash.

2. **Apply the wrong gas computation mode** — `ValidResourceBounds::L1Gas` returns `GasVectorComputationMode::NoL2Gas` while `AllResources` returns `All`. Re-execution under `NoL2Gas` converts L2 gas to L1 gas units via `sierra_gas_to_l1_gas_amount_round_up`, producing different fee, gas, and revert outcomes than the original execution.

These map to:
- **Critical**: Wrong state, receipt, event, or revert result from blockifier/execution logic for accepted input.
- **High**: RPC execution, fee estimation, tracing, or simulation on a syncing node returns an authoritative-looking wrong value.

---

### Likelihood Explanation

Any unprivileged user can craft a valid `AllResources` V3 invoke transaction with `l2_gas.max_amount = 0`, `l2_gas.max_price_per_unit = 0`, `l1_data_gas.max_amount = 0`, `l1_data_gas.max_price_per_unit = 0`. The gateway's stateless validator explicitly accepts this configuration. No special privilege or malformed bytes are required; the transaction passes all admission checks and is sequenced normally.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, preserve `AllResources` whenever all three fields are explicitly present in the protobuf message, even if their values are zero. Reserve the `L1Gas` downgrade only for the backward-compatibility case where `l1_data_gas` is absent (`None`), which is already handled by the `unwrap_or_default()` on line 427. Concretely, change the classification condition from:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})
```

to:

```rust
Ok(if value.l1_data_gas.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})
```

This matches the intent of the existing `TODO(Shahak)` comment on line 426 and aligns with the JSON deserialization path in `TryFrom<DeprecatedResourceBoundsMapping>`, which uses the presence of the `L1DataGas` key (not its value) to distinguish the two variants. [7](#0-6) 

---

### Proof of Concept

1. Submit via RPC:
   ```json
   { "type": "INVOKE", "version": "0x3",
     "resource_bounds": {
       "l1_gas":      { "max_amount": "0x64", "max_price_per_unit": "0x1" },
       "l2_gas":      { "max_amount": "0x0",  "max_price_per_unit": "0x0" },
       "l1_data_gas": { "max_amount": "0x0",  "max_price_per_unit": "0x0" }
     }, ... }
   ```
2. Gateway accepts; sequencer computes hash **H1** via `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `AllResources` → 3-felt preimage → Poseidon(tip, L1Gas_packed, L2Gas_packed=0, L1DataGas_packed=0).
3. Transaction is included in a block. `InvokeTransactionV3` stored with `ValidResourceBounds::AllResources`.
4. Block is synced via p2p. Serialized protobuf has `l1_gas=X`, `l2_gas=0`, `l1_data_gas=0` (all present).
5. Receiving node runs `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas(l1_gas=X)`.
6. Receiving node recomputes hash **H2** via `get_tip_resource_bounds_hash` with `L1Gas` → 2-felt preimage → Poseidon(tip, L1Gas_packed, L2Gas_packed=0).
7. **H1 ≠ H2**. Block hash verification on the syncing node fails; re-execution uses `NoL2Gas` mode, producing divergent fee/revert results.

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
```

```rust
    type Error = StarknetApiError;
    fn try_from(
        resource_bounds_mapping: DeprecatedResourceBoundsMapping,

        if let (Some(l1_bounds), Some(l2_bounds)) = (
```
