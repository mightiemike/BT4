### Title
Unprivileged Caller Can Aggressively Garbage-Collect Mainchain Headers, Breaking SPV Proofs and Reorgs — (File: `contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating function with **no caller access control**. Any unprivileged NEAR account can call it with an arbitrarily large `batch_size` to immediately remove all mainchain headers that are eligible for garbage collection. This can cause `verify_transaction_inclusion` to fail for transactions in those blocks and can break chain reorganizations whose common ancestor falls in the GC-eligible range — both of which would otherwise remain available for a significant time under normal relayer-driven GC.

---

### Finding Description

`submit_blocks` is correctly gated behind `#[trusted_relayer]`, restricting block submission to authorized relayers. However, `run_mainchain_gc` — which **deletes** mainchain headers — carries only `#[pause]`:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
``` [1](#0-0) 

When the contract is not paused, any NEAR account — with no role, no stake, and no skin in the game — can call this function. The `batch_size` parameter is fully attacker-controlled.

Inside the function, the only bound on removal is:

```rust
let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);
``` [2](#0-1) 

Passing `u64::MAX` as `batch_size` causes the function to remove every header that is currently eligible for GC in a single transaction — headers that the relayer's gradual GC (which removes only `num_of_headers` per `submit_blocks` call) would have left available for many future blocks. [3](#0-2) 

The removed headers are permanently deleted from all three maps:

```rust
self.remove_block_header(blockhash);
self.mainchain_height_to_header.remove(&height);
``` [4](#0-3) 

where `remove_block_header` also removes from `mainchain_header_to_height` and `headers_pool`. [5](#0-4) 

---

### Impact Explanation

**1. SPV proof breakage.** `verify_transaction_inclusion` looks up the target block in `mainchain_header_to_height`:

```rust
let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));
``` [6](#0-5) 

After an attacker-triggered aggressive GC, any block that was in the GC-eligible range is gone from this map. A downstream contract or user that calls `verify_transaction_inclusion` for a transaction in one of those blocks will receive a hard panic — permanently breaking SPV verification for that transaction.

**2. Chain reorg breakage.** The project's own documentation explicitly warns:

> If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound` when it cannot walk the chain back to the common ancestor. [7](#0-6) 

An attacker who calls `run_mainchain_gc(u64::MAX)` immediately after a large batch of headers is submitted can remove the common ancestor of a pending fork before the relayer has a chance to submit the fork's tip. The subsequent reorg attempt will panic and the fork can never be resolved.

---

### Likelihood Explanation

The entry path requires no privilege, no stake, and no special knowledge — only the ability to send a NEAR transaction. The function is callable by any account when the contract is not paused. The cost to the attacker is only NEAR gas fees (a few cents). The attack can be repeated after every relayer batch submission to keep the GC window maximally aggressive at all times.

---

### Recommendation

Apply the same `#[trusted_relayer]` guard (or at minimum a role check) to `run_mainchain_gc` that is applied to `submit_blocks`. Alternatively, remove the public entrypoint entirely and make GC an internal-only operation triggered exclusively from within `submit_blocks`. If a public GC endpoint is desired for operational reasons, it should require the caller to hold a privileged role (e.g., `UnrestrictedRunGC` or `DAO`) unconditionally, not only when the contract is paused.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and let the relayer submit 53704 headers (1000 over threshold).
2. A user initiates a `verify_transaction_inclusion` call for a transaction in block at height `tip - 52704 - 500` (500 blocks into the GC-eligible range). The block is still present because the relayer's gradual GC has only removed a few hundred headers so far.
3. Before the user's call is processed (or in a subsequent block), the attacker calls:
   ```
   run_mainchain_gc(batch_size: 18446744073709551615)
   ```
4. The function removes all 1000 GC-eligible headers in one shot, including the user's target block.
5. The user's `verify_transaction_inclusion` call now panics with `"block does not belong to the current main chain"` — the SPV proof is permanently broken for that transaction.

The same sequence applies to a pending fork reorg: if the common ancestor of the fork is in the GC-eligible range, the attacker's call removes it, and the subsequent `reorg_chain` walk panics with `PrevBlockNotFound`. [8](#0-7)

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
