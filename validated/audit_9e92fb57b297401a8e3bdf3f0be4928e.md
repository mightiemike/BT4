### Title
Withdrawal Fee Bypass via Slow Mode Path — (`File: core/contracts/EndpointTx.sol`)

---

### Summary

The `processSlowModeTransactionImpl` function handles `WithdrawCollateral` transactions without charging the per-product `withdrawFeeX18` fee, while the sequencer fast path in `processTransactionImpl` correctly charges it. Any unprivileged user can exploit this asymmetry to withdraw collateral while paying only the flat `SLOW_MODE_FEE` and avoiding the per-product withdrawal fee entirely.

---

### Finding Description

Two distinct code paths exist for processing a `WithdrawCollateral` transaction in `EndpointTx.sol`.

**Fast path** (`processTransactionImpl`, lines 413–436):

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.SignedWithdrawCollateral memory signedTx = abi.decode(...);
    validateSignedTx(signedTx.tx.sender, signedTx.tx.nonce, transaction, signedTx.signature, true);
    chargeFee(
        signedTx.tx.sender,
        spotEngine.getConfig(signedTx.tx.productId).withdrawFeeX18,
        signedTx.tx.productId
    );
    clearinghouse.withdrawCollateral(...);
}
```

**Slow mode path** (`processSlowModeTransactionImpl`, lines 217–229):

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(...);
    validateSender(txn.sender, sender);
    clearinghouse.withdrawCollateral(txn.sender, txn.productId, txn.amount, address(0), nSubmissions);
}
```

The slow mode path calls `clearinghouse.withdrawCollateral` identically but omits the `chargeFee(..., withdrawFeeX18, ...)` call entirely. The only cost to the user on the slow mode path is the flat `SLOW_MODE_FEE` charged at submission time in `submitSlowModeTransactionImpl` (line 370–371), which is a fixed amount unrelated to the withdrawal size or product.

This is structurally identical to the reported vulnerability: `userDeposit()` enforces the investment period check while `processDeposit()` and batch functions omit it. Here, `processTransactionImpl` enforces `withdrawFeeX18` while `processSlowModeTransactionImpl` omits it for the same operation.

---

### Impact Explanation

The `withdrawFeeX18` is a per-product fee configured in `SpotEngine` (initialized to `ONE = 1e18` in X18 format for the quote product). This fee accrues to `sequencerFee[productId]` and is ultimately claimed by the protocol via `claimSequencerFees`. By routing all withdrawals through the slow mode queue, a user pays only the flat `SLOW_MODE_FEE` regardless of withdrawal size, bypassing the per-product fee entirely. For large withdrawals, the economic difference is significant. The protocol's fee revenue from `sequencerFee` is systematically undercollected.

---

### Likelihood Explanation

The slow mode submission path (`submitSlowModeTransaction` → `executeSlowModeTransaction`) is publicly callable by any user with no access restrictions. The only friction is a 3-day delay (`SLOW_MODE_TX_DELAY`). Any user aware of the discrepancy can exploit it on every withdrawal. No special role, leaked key, or governance capture is required.

---

### Recommendation

Add the `chargeFee` call in `processSlowModeTransactionImpl` for `WithdrawCollateral`, consistent with the fast path:

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(
        transaction[1:], (IEndpoint.WithdrawCollateral)
    );
    validateSender(txn.sender, sender);
    chargeFee(
        txn.sender,
        spotEngine.getConfig(txn.productId).withdrawFeeX18,
        txn.productId
    );
    clearinghouse.withdrawCollateral(txn.sender, txn.productId, txn.amount, address(0), nSubmissions);
}
```

---

### Proof of Concept

1. User holds collateral in a subaccount and wants to withdraw without paying `withdrawFeeX18`.
2. User calls `Endpoint.submitSlowModeTransaction(withdrawCollateralTx)`. The flat `SLOW_MODE_FEE` is charged and the transaction is queued with `executableAt = block.timestamp + 3 days`.
3. After 3 days, user (or anyone) calls `Endpoint.executeSlowModeTransaction()`.
4. `_executeSlowModeTransaction` dispatches to `processSlowModeTransaction` → `processSlowModeTransactionImpl`.
5. The `WithdrawCollateral` branch executes `clearinghouse.withdrawCollateral(...)` with no `chargeFee` call.
6. The full withdrawal amount is transferred to the user. `sequencerFee[productId]` is never incremented. The per-product withdrawal fee is lost. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
