### Title
Missing Validation That At Least One Timeout Is Non-Zero in IBC NFT Transfer Allows Permanent NFT Lock - (File: x/nft-transfer/types/msgs.go)

### Summary
`MsgTransfer.ValidateBasic()` in the custom ICS-721 nft-transfer module does not enforce that at least one of `TimeoutHeight` or `TimeoutTimestamp` is non-zero. A user who submits a `MsgTransfer` with both timeouts set to zero creates an IBC packet that can never expire. If the packet is never relayed, the NFT is permanently locked in the escrow account with no recovery path.

### Finding Description
The `MsgTransfer` message in `x/nft-transfer/types/msgs.go` accepts `TimeoutHeight` and `TimeoutTimestamp` fields. The comment on line 51 explicitly states: *"NOTE: timeout height or timestamp values can be 0 to disable the timeout."* The `ValidateBasic()` function validates port, channel, classID, tokenIDs, sender, and receiver — but never checks whether both timeout fields are simultaneously zero. [1](#0-0) 

The CLI command in `x/nft-transfer/client/cli/tx.go` enforces a non-zero timestamp only in the **relative** timeout path (lines 84–86). When `--absolute-timeouts` is passed, no such guard exists — a user can explicitly pass `--packet-timeout-height 0-0 --packet-timeout-timestamp 0` and the message is accepted on-chain. [2](#0-1) 

When the message is processed, `createOutgoingPacket()` in `x/nft-transfer/keeper/packet.go` immediately escrows the NFT (or burns the voucher) before the packet is sent: [3](#0-2) 

The packet is then dispatched with both timeouts at zero via `SendPacket`: [4](#0-3) 

A packet with both timeouts at zero never expires. The IBC core layer will never call `OnTimeoutPacket`, so `refundPacketToken()` — the only recovery path — is never triggered: [5](#0-4) 

### Impact Explanation
If the packet is never relayed (counterparty chain halts, channel is closed, relayer stops), the NFT remains permanently locked in the escrow address with no on-chain mechanism to recover it. The corrupted value is the NFT owner field: the NFT is transferred from the sender to the escrow address (`types.GetEscrowAddress(sourcePort, sourceChannel)`) and can never be returned. This is a direct, irreversible user asset loss. [6](#0-5) 

### Likelihood Explanation
The `--absolute-timeouts` flag is a documented, supported CLI path. A user who intends to disable one timeout (e.g., sets `--packet-timeout-height 0-0`) and forgets or explicitly sets `--packet-timeout-timestamp 0` will produce a no-timeout packet. The CLI default for `--packet-timeout-timestamp` is `DefaultRelativePacketTimeoutTimestamp` (10 minutes), but this default only applies in relative mode; in absolute mode the user controls both values directly. The missing guard in `ValidateBasic()` means the chain accepts such messages without warning. [7](#0-6) [8](#0-7) 

### Recommendation
Add a check in `MsgTransfer.ValidateBasic()` that rejects messages where both `TimeoutHeight` and `TimeoutTimestamp` are zero:

```go
if msg.TimeoutHeight.IsZero() && msg.TimeoutTimestamp == 0 {
    return newsdkerrors.Wrap(ErrInvalidPacket, "timeout height or timestamp must be non-zero")
}
```

This mirrors the pattern used in ibc-go's own fungible token transfer module and ensures every IBC NFT packet has a finite lifetime, preserving the user's ability to reclaim their NFT via timeout if the packet is never delivered. [9](#0-8) 

### Proof of Concept
1. User owns NFT with `denomID=cryptoCat`, `tokenID=kitty` on chain A.
2. User submits:
   ```
   chain-maind tx nft-transfer transfer nft channel-0 <receiver> cryptoCat kitty \
     --absolute-timeouts \
     --packet-timeout-height 0-0 \
     --packet-timeout-timestamp 0 \
     --from <sender>
   ```
3. `MsgTransfer.ValidateBasic()` passes — no check for both-zero timeouts.
4. `createOutgoingPacket()` transfers the NFT to the escrow address on chain A.
5. `SendPacket` dispatches the IBC packet with `TimeoutHeight={0,0}` and `TimeoutTimestamp=0`.
6. If the relayer never relays the packet (or the counterparty chain is permanently halted), `OnTimeoutPacket` is never called, `refundPacketToken` never executes, and the NFT remains permanently locked in the escrow account with no recovery path. [10](#0-9) [11](#0-10)

// ValidateBasic performs a basic check of the MsgTransfer fields.
// NOTE: timeout height or timestamp values can be 0 to disable the timeout.
// NOTE: The recipient addresses format is not validated as the format defined by
// the chain is not known to IBC.
func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil
}

// refundPacketToken will unescrow and send back the tokens back to sender
// if the sending chain was the source chain. Otherwise, the sent tokens
// were burnt in the original send so new tokens are minted and sent to
// the sending address.
func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil

func (k Keeper) SendTransfer(
	ctx sdk.Context,
	sourcePort,
	sourceChannel,
	classID string,
	tokenIDs []string,
	sender sdk.AccAddress,
	receiver string,
	timeoutHeight clienttypes.Height,
	timeoutTimestamp uint64,
) error {
	if sourcePort != types.PortID {
		return sdkerrors.Wrapf(types.ErrInvalidSourcePort, "source port must be %q", types.PortID)
	}

	sourceChannelEnd, found := k.channelKeeper.GetChannel(ctx, sourcePort, sourceChannel)
	if !found {
		return sdkerrors.Wrapf(channeltypes.ErrChannelNotFound, "port ID (%s) channel ID (%s)", sourcePort, sourceChannel)
	}

	destinationPort := sourceChannelEnd.Counterparty.PortId
	destinationChannel := sourceChannelEnd.Counterparty.ChannelId

	// get the next sequence
	sequence, found := k.channelKeeper.GetNextSequenceSend(ctx, sourcePort, sourceChannel)
	if !found {
		return sdkerrors.Wrapf(
			channeltypes.ErrSequenceSendNotFound,
			"source port: %s, source channel: %s", sourcePort, sourceChannel,
		)
	}

	// See spec for this logic: https://github.com/cosmos/ibc/blob/master/spec/app/ics-721-nft-transfer/README.md#packet-relay
	packet, err := k.createOutgoingPacket(ctx,
		sourcePort,
		sourceChannel,
		destinationPort,
		destinationChannel,
		classID,
		tokenIDs,
		sender,
		receiver,
		sequence,
		timeoutHeight,
		timeoutTimestamp,
	)
	if err != nil {
		return err
	}

	if _, err := k.ics4Wrapper.SendPacket(ctx, sourcePort, sourceChannel, timeoutHeight, timeoutTimestamp, packet.GetData()); err != nil {
		return err
	}

	return nil
