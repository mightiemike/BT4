### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing Coinbase Proof Validation — (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` (v1) was deprecated in favour of `verify_transaction_inclusion_v2`, which adds a mandatory coinbase Merkle proof check to mitigate the known 64-byte transaction Merkle proof forgery vulnerability. However, the v1 function is still a live, publicly callable NEAR contract method. Any unprivileged NEAR caller can invoke v1 directly, skipping the coinbase proof gate entirely and obtaining a `true` result for a forged transaction inclusion proof.

### Finding Description

`verify_transaction_inclusion_v2` enforces a two-tier proof check: first it validates the coinbase Merkle proof against the block's `merkle_root`, then it delegates to v1 for the transaction proof:

```
verify_transaction_inclusion_v2
  └─ require!(coinbase proof == merkle_root)   ← stricter gate
  └─ self.verify_transaction_inclusion(...)    ← weaker inner check
```

The stricter gate exists specifically because v1 alone is vulnerable to the 64-byte internal-node forgery attack (documented in the code comments referencing the BitMEX blog post).

The `#[deprecated]` Rust attribute at line 283 only emits a compile-time warning for Rust callers. It does **not** remove the function from the compiled WASM binary or prevent external NEAR accounts from calling it via RPC. The function retains its `#[pause]` attribute (line 287), confirming it is still a live contract endpoint:

```rust
#[deprecated(since = "0.5.0", note = "Use `verify_transaction_inclusion_v2` instead.")]
#[pause]
pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
```

A caller who wants to forge a transaction inclusion proof can call v1 directly with a crafted 64-byte internal Merkle node as `tx_id`, bypassing the coinbase proof check entirely. v1 only checks:

1. Confirmations threshold
2. Block membership in the main chain
3. `merkle_proof` is non-empty
4. `compute_root_from_merkle_proof(tx_id, tx_index, proof) == merkle_root`

Condition 4 can be satisfied with an internal Merkle tree node as `tx_id` when the proof depth is chosen to match, which is the exact attack the coinbase proof in v2 was designed to prevent.

### Impact Explanation

A downstream contract or bridge that calls `verify_transaction_inclusion` (v1) directly — or that is tricked into doing so — will receive `true` for a forged transaction inclusion. This enables an attacker to claim a Bitcoin deposit, withdrawal, or cross-chain event that never occurred on the Bitcoin chain. The corrupted proof result is the canonical output consumed by integrators, making this a critical integrity failure in the trust model the light client provides.

### Likelihood Explanation

The function is publicly accessible with no role restriction — any NEAR account can call it. No special preconditions, leaked keys, or social engineering are required. The 64-byte Merkle forgery technique is well-documented and the construction of a valid forged proof is a known, reproducible attack. Any integrator that calls v1 (whether by mistake or because they are unaware of the deprecation) is immediately vulnerable.

### Recommendation

Remove `verify_transaction_inclusion` (v1) from the public contract interface entirely, or gate it with an access-control role that prevents external callers. A `#[private]` attribute would restrict it to self-calls only (which is the only legitimate use — called internally by v2). Alternatively, inline the v1 logic into v2 and delete the standalone v1 method so there is only one callable proof-verification endpoint.

### Proof of Concept

1. Deploy the contract with `skip_pow_verification = true` and submit a block containing two transactions (coinbase + one other).
2. Compute the Merkle root: `root = H(coinbase_hash || tx_hash)`.
3. Note that `root` itself is a 64-byte-encodable internal node. Craft `ProofArgs { tx_id: root, tx_index: 0, merkle_proof: [] }` — but `merkle_proof` must be non-empty per the check at line 315, so instead use the standard 64-byte attack: supply `tx_id = H(A || B)` as an internal node with a one-element proof that reconstructs the real `merkle_root`.
4. Call `verify_transaction_inclusion` (v1) directly via NEAR RPC with the crafted `ProofArgs`.
5. The function returns `true` despite no real transaction matching `tx_id` existing in the block.
6. Calling `verify_transaction_inclusion_v2` with the same forged `tx_id` (and any coinbase proof) would fail at the coinbase proof gate, demonstrating that v1 bypasses the protection. [1](#0-0) [2](#0-1)

    #[deprecated(
        since = "0.5.0",

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
