### Title
Unbounded `ExitDuration` in Tier Validation Allows Governance to Permanently Lock User Funds - (File: `x/tieredrewards/types/tier.go`)

### Summary
The `Tier.Validate()` function enforces only a lower bound (`ExitDuration > 0`) on the tier exit duration, with no upper bound. A governance proposal can set `ExitDuration` to an arbitrarily large value, making it impossible for any user who has triggered an exit to ever undelegate or withdraw their locked staking position.

### Finding Description
`Tier.ExitDuration` is the time a user must wait after calling `TriggerExitFromTier` before they are permitted to undelegate or withdraw their locked funds. The field is validated in `Tier.Validate()`: [1](#0-0) 

The check only rejects non-positive values. There is no ceiling. `SetTier`, which is the write path used by governance messages, genesis, and chain upgrades, calls `tier.Validate()` and then persists the tier unconditionally: [2](#0-1) 

When a user calls `TriggerExitFromTier`, the exit unlock timestamp is computed from the tier's `ExitDuration` at that moment: [3](#0-2) 

Both `TierUndelegate` and `WithdrawFromTier` gate fund access on `CompletedExitLockDuration`: [4](#0-3) [5](#0-4) 

If governance sets `ExitDuration` to `math.MaxInt64` nanoseconds (~292 years), any position that subsequently triggers an exit will have `ExitUnlockAt` set centuries in the future. The user's staked tokens remain locked in the position's delegator sub-account with no reachable withdrawal path.

### Impact Explanation
Users who lock funds via `LockTier` or `CommitDelegationToTier` and then call `TriggerExitFromTier` after a malicious governance update will have their staked tokens permanently inaccessible. The `TierUndelegate` and `WithdrawFromTier` paths both hard-reject the transaction with `ErrExitLockDurationNotReached`. The `ExitTierWithDelegation` path also requires `CompletedExitLockDuration`. The corrupted value is the `ExitUnlockAt` timestamp stored in each affected `Position`, which gates all fund-recovery paths.

### Likelihood Explanation
The attack requires a governance proposal to pass. Governance capture via a large token holder or coordinated coalition is a realistic threat model for live Cosmos chains. The missing upper bound means the protocol itself provides no safety net even if governance acts in bad faith or is captured. Any token holder can submit the proposal; execution requires a majority vote.

### Recommendation
Add an upper bound check inside `Tier.Validate()`, analogous to the fix applied in the referenced PaymentsERC20 report:

```go
// e.g., cap at 365 days expressed in nanoseconds
const MaxExitDuration = int64(365 * 24 * 3600 * 1e9)

if t.ExitDuration > MaxExitDuration {
    return fmt.Errorf("exit duration must not exceed %d nanoseconds", MaxExitDuration)
}
```

Because `SetTier` always calls `Validate()` before persisting, this single change covers the governance message path, genesis, and upgrade paths simultaneously. [6](#0-5) 

### Proof of Concept

1. Governance submits and passes a `MsgUpdateTier` (or equivalent) setting `ExitDuration = math.MaxInt64` for tier ID 1.
2. `SetTier` calls `tier.Validate()` — passes, because `math.MaxInt64 > 0`.
3. User Alice has an active position in tier 1 and calls `TriggerExitFromTier`.
4. `pos.TriggerExit(blockTime, tier.ExitDuration)` sets `ExitUnlockAt = blockTime + ~292 years`.
5. Alice calls `TierUndelegate` → `validateUndelegatePosition` → `CompletedExitLockDuration` returns `false` → `ErrExitLockDurationNotReached`.
6. Alice calls `WithdrawFromTier` → same gate → same rejection.
7. Alice calls `ExitTierWithDelegation` → same gate → same rejection.
8. Alice's staked tokens are permanently inaccessible through any normal transaction path.

```go
func (t Tier) Validate() error {
	if t.Id == 0 {
		return ErrInvalidTierID
	}

	if t.ExitDuration <= 0 {
		return fmt.Errorf("exit duration must be positive")
	}

	if t.BonusApy.IsNil() {
		return fmt.Errorf("bonus apy cannot be nil")
	}

	if t.BonusApy.IsNegative() {
		return fmt.Errorf("bonus apy cannot be negative: %s", t.BonusApy)
	}

	// Cap BonusApy at 100% to prevent governance from draining the rewards pool.
	if t.BonusApy.GT(math.LegacyOneDec()) {
		return fmt.Errorf("bonus apy must not exceed 1.0 (100%%): got %s", t.BonusApy)
	}

	if t.MinLockAmount.IsNil() {
		return fmt.Errorf("min lock amount cannot be nil")
	}

	if t.MinLockAmount.IsNegative() {
		return fmt.Errorf("min lock amount cannot be negative: %s", t.MinLockAmount)
	}

	return nil
}
```

```go
		return err
	}

		return errorsmod.Wrapf(err, "%s (tier id %d)", types.ErrTierStore.Error(), tier.Id)
	}
	return nil
}
```
