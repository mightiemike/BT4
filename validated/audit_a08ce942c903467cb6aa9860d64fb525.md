Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Enabling Full Allowlist Bypass or Lockout via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking `sender`, which the pool sets to its own `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. This causes the allowlist to check the router address rather than the economic actor, producing either a complete bypass (if the router is allowlisted) or a lockout of allowlisted users from all router-based swap paths.

## Finding Description

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, passing its own `msg.sender` as the `sender` argument forwarded to every extension:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // <-- direct caller of pool.swap()
  recipient,
  ...
);
```

`SwapAllowlistExtension.beforeSwap()` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When the call originates from `MetricOmmSimpleRouter.exactInputSingle()`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData
  );
```

the pool's `msg.sender` is `address(router)`, so `sender` forwarded to the extension is the router — not the end user. The allowlist evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][end_user]`.

This produces two mutually exclusive failure modes:

**Mode A – Bypass**: Admin allowlists the router so router-based swaps work. Every user on-chain can call `router.exactInputSingle()` and pass the check, because the router is allowlisted. Per-user restriction is completely defeated.

**Mode B – Lockout**: Admin allowlists individual user addresses. Those users can call `pool.swap()` directly, but any attempt to route through `MetricOmmSimpleRouter` (including multi-hop, exact-output, and native-ETH paths) reverts with `NotAllowedToSwap`, because the router is not on the list.

The asymmetry with `DepositAllowlistExtension` confirms the design gap. The deposit hook receives an explicit `owner` parameter (the beneficiary, supplied by the caller) and checks that:

```solidity
// metric-periphery/contracts/extensions/DepositAllowlistExtension.sol L32-42
function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
  external view override returns (bytes4)
{
  if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
    revert IMetricOmmPoolActions.NotAllowedToDeposit();
  }
  ...
}
```

`owner` survives router indirection because it is an explicit argument, not derived from `msg.sender`. The swap hook has no equivalent; it only has `sender` (the direct caller) and `recipient` (the output destination), neither of which reliably identifies the economic actor when a router is involved.

## Impact Explanation

**Mode A**: Any unprivileged user can execute swaps in a pool the admin intended to restrict to specific counterparties (e.g., KYC-verified addresses, whitelisted market makers). Unauthorized swaps against an oracle-priced pool expose LP funds to adverse selection or deliberate drain if the oracle price lags — a direct loss path for LP principal.

**Mode B**: Allowlisted users cannot access any router-based swap path (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, native-ETH wrapping). Core swap functionality is broken for the intended user set, matching the "broken core pool functionality causing unusable swap flows" impact criterion.

## Likelihood Explanation

Any pool deploying `SwapAllowlistExtension` with per-user restrictions is affected the moment a user routes through `MetricOmmSimpleRouter`. The router is the standard periphery entry point. A pool admin following the natural configuration path — allowlist the router so users can swap — triggers Mode A immediately. No special privileges, no malicious setup, and no non-standard tokens are required. The trigger is a normal `exactInputSingle` call by any on-chain address.

## Recommendation

Mirror the deposit pattern: introduce an explicit end-user identity that survives router indirection.

**Preferred**: Add a `swapper` parameter to the `beforeSwap` hook (analogous to `owner` in `beforeAddLiquidity`) that the pool populates from a caller-supplied argument rather than from `msg.sender`. The router would forward `msg.sender` as the swapper, and the allowlist would check that value.

**Minimal**: Document that `SwapAllowlistExtension` gates the direct caller of `pool.swap()`, not the end user, and that pool admins must allowlist each router/aggregator address rather than individual users — accepting that this grants all users of that router access.

## Proof of Concept

1. Pool is deployed with `SwapAllowlistExtension` configured in `beforeSwap` order.
2. Admin calls `swapExtension.setAllowedToSwap(pool, ALICE, true)`. Only `ALICE` is allowlisted.
3. **Mode A test**: Admin also calls `swapExtension.setAllowedToSwap(pool, router, true)` to enable router-based swaps. `BOB` (not allowlisted) calls `router.exactInputSingle({pool: pool, recipient: BOB, ...})`. Router calls `pool.swap(BOB, ...)` — pool's `msg.sender` is `address(router)`. Extension evaluates `allowedSwapper[pool][router]` → `true`. `BOB`'s swap executes — allowlist bypassed.
4. **Mode B test**: Admin does NOT allowlist the router. `ALICE` calls `router.exactInputSingle({pool: pool, recipient: ALICE, ...})`. Router calls `pool.swap(ALICE, ...)` — pool's `msg.sender` is `address(router)`. Extension evaluates `allowedSwapper[pool][router]` → `false`. Reverts `NotAllowedToSwap` — `ALICE` locked out of all router paths despite being individually allowlisted.
