### Title
Malicious Counterparty Chain Can Unescrow Arbitrary NFTs via Crafted ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` determines transfer direction solely by checking whether `data.ClassId` starts with `sourcePort/sourceChannel/`. A counterparty chain that controls packet data can craft a `ClassId` with that prefix to force the back-to-origin (unescrow) branch, draining any NFT currently held by the channel's escrow address — including NFTs escrowed by other users.

---

### Finding Description

**`IsAwayFromOrigin` is purely string-based:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

It returns `false` (back-to-origin) whenever `data.ClassId` starts with `"<sourcePort>/<sourceChannel>/"`. There is no on-chain record consulted to verify that the NFT was ever minted as a voucher on the sending chain.

**`processReceivedPacket` unescrow branch:** [2](#0-1) 

When `isAwayFromOrigin == false`:
1. `escrowAddress = GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())` — Chain B's own escrow address for this channel.
2. `unprefixedClassID = RemoveClassPrefix(sourcePort, sourceChannel, data.ClassId)` — strips the crafted prefix.
3. `TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver)` — transfers ownership from the escrow address to the attacker-supplied receiver.

**`TransferOwner` enforces ownership of `srcOwner`:** [3](#0-2) 

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
```

This check passes because the escrow address legitimately owns the NFT that was escrowed by an honest user.

**Escrow address collision:**

When userA on Chain B sends `(classID="nftClass", tokenID="token1")` to Chain A via `channel-0`, `createOutgoingPacket` escrows it at `GetEscrowAddress("nft", "channel-0")`. [4](#0-3) 

When the crafted packet arrives at Chain B, `processReceivedPacket` computes:

```go
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
// = GetEscrowAddress("nft", "channel-0")  ← identical address
``` [5](#0-4) 

The two escrow addresses are identical, so the ownership check in `TransferOwner` succeeds and the NFT is transferred to the attacker.

---

### Impact Explanation

A malicious counterparty chain submits a packet with:
- `ClassId = "nft/channel-0/nftClass"` (prefixed with Chain B's own port/channel)
- `TokenIds = ["token1"]`
- `Receiver = attacker_address`

Chain B's `processReceivedPacket` interprets this as a legitimate return of a voucher, strips the prefix, resolves `voucherClassID = "nftClass"`, and calls `TransferOwner` from the escrow address to the attacker. UserA's NFT — which was legitimately escrowed — is stolen. The invariant that "unescrow requires a prior escrow of the exact (classID, tokenID) pair by the same counterparty" is never enforced.

---

### Likelihood Explanation

Requires control of a counterparty chain (or the ability to submit IBC packets authenticated by its light client). In a permissionless multi-chain IBC environment this is a realistic threat: any chain that establishes an IBC channel with Cronos POS Chain can exploit this. The precondition — at least one NFT escrowed on the channel — is routinely satisfied in production.

---

### Recommendation

Maintain an on-chain mapping of `(channelID, classID, tokenID) → escrowed` and verify membership before calling `TransferOwner` in the back-to-origin branch. Only unescrow a token if it was recorded as escrowed on that specific channel. This mirrors the safeguard used in ICS-20 fungible token transfer via module account balance checks.

---

### Proof of Concept

```
1. Chain B: userA calls SendTransfer("nft", "channel-0", "nftClass", ["token1"], userA, chainA_addr)
   → createOutgoingPacket escrows token1 at GetEscrowAddress("nft","channel-0") on Chain B.

2. Chain A (attacker-controlled): craft IBC packet:
     SourcePort    = "nft"
     SourceChannel = "channel-0"   // Chain A's channel toward Chain B
     DestPort      = "nft"
     DestChannel   = "channel-0"   // Chain B's channel
     Data.ClassId  = "nft/channel-0/nftClass"
     Data.TokenIds = ["token1"]
     Data.Receiver = attacker_address

3. Chain B receives packet via OnRecvPacket → processReceivedPacket:
     IsAwayFromOrigin("nft","channel-0","nft/channel-0/nftClass") → false
     unprefixedClassID = "nftClass"
     voucherClassID    = "nftClass"
     escrowAddress     = GetEscrowAddress("nft","channel-0")  // owns token1
     TransferOwner(ctx, "nftClass", "token1", escrowAddress, attacker_address)
     → SUCCESS: token1 now owned by attacker_address

4. Assert: userA no longer owns token1; attacker_address owns token1.
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
