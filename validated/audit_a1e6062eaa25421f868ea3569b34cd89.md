### Title
`gas_key_storage_cost` uses a truncated nonce-key length, systematically undercharging storage stake for gas-key nonces - (`File: runtime/runtime/src/access_keys.rs`)

### Summary

`gas_key_storage_cost` in `runtime/runtime/src/access_keys.rs` computes `per_nonce_key_size` as only `public_key.trie_id_len() + size_of::<NonceIndex>()`. The actual trie key for a `GasKeyNonce` entry is `access_key_key_len(account_id_len, pk_trie_id_len) + size_of::<NonceIndex>()`, which expands to `COLUMN_PREFIX_LEN(1) + account_id_len + ACCESS_KEY_SEPARATOR_LEN(1) + pk_trie_id_len + size_of::<NonceIndex>()`. The function omits the `2 + account_id_len` bytes of the column prefix, account-id, and separator from every nonce entry's key-size contribution. This is the exact same class of bug as the external report: a value that is already a partial component of a larger quantity is passed directly where the full quantity is required.

### Finding Description

`gas_key_storage_cost` is the sole function that sets the storage-usage delta when a gas key is added (`add_gas_key`) and when it is deleted (`delete_gas_key`). It computes:

```rust
// runtime/runtime/src/access_keys.rs  lines 38-43
let per_nonce_value_size = borsh::object_length(&(0 as Nonce)).unwrap() as u64;
let per_nonce_key_size = public_key.trie_id_len() as u64 + size_of::<NonceIndex>() as u64;

num_nonces as u64
    * (per_nonce_key_size + per_nonce_value_size + storage_config.num_extra_bytes_record)
    + access_key_storage_usage(fee_config, public_key, access_key)
```

The canonical trie-key length for a `GasKeyNonce` entry is defined in `core/primitives/src/trie_key.rs`:

```rust
// core/primitives/src/trie_key.rs  lines 324-326
pub fn gas_key_nonce_key_len(account_id: &AccountId, key_handle: &PublicKeyHandle) -> usize {
    access_key_key_len(account_id.len(), key_handle.trie_id_len()) + size_of::<NonceIndex>()
}
```

where `access_key_key_len` is:

```rust
// core/primitives-core/src/trie_key.rs  lines 13-14
pub fn access_key_key_len(account_id_len: usize, public_key_len: usize) -> usize {
    COLUMN_PREFIX_LEN + account_id_len + ACCESS_KEY_SEPARATOR_LEN + public_key_len
}
```

`gas_key_storage_cost` uses only `pk_trie_id_len + sizeof(NonceIndex)` for `per_nonce_key_size`, silently dropping `COLUMN_PREFIX_LEN(1) + account_id_len + ACCESS_KEY_SEPARATOR_LEN(1)` bytes per nonce. The gas-fee counterpart `gas_key_add_key_exec_fee` in `core/parameters/src/cost.rs` correctly calls `access_key_key_len(account_id_len, public_key_len)`:

```rust
// core/parameters/src/cost.rs  lines 888-889
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
```

The same correct full-key length is used in `delete_gas_key` for compute-cost accounting:

```rust
// runtime/runtime/src/access_keys.rs  line 118
let nonce_key_len = gas_key_nonce_key_len(account_id, &public_key.into());
```

So the compute-cost path and the gas-fee path both use the full key length; only the storage-stake path uses the truncated length.

### Impact Explanation

**Impact: High.** Storage stake is the protocol's mechanism for ensuring that every byte of on-chain state is backed by locked NEAR. When `gas_key_storage_cost` underestimates the nonce key size by `(2 + account_id_len)` bytes per nonce, the account's `storage_usage` field is set too low. The storage-stake lock (`storage_amount_per_byte × storage_usage`) is therefore smaller than the actual trie bytes consumed. Any user who adds a gas key with `N` nonces and an account ID of length `L` pays `(2 + L) × N` fewer bytes of storage stake than the trie actually stores. For a 64-byte account ID and the maximum allowed nonce count, the undercharge is `66 × num_nonces` bytes of storage stake per gas key. Because `add_gas_key` and `delete_gas_key` both call `gas_key_storage_cost` symmetrically, the undercharge is permanent for the lifetime of the key and is never corrected on deletion.

### Likelihood Explanation

**Likelihood: High.** The `GasKeys` feature is assigned protocol version 85 (stable, not nightly). Every `AddKey` action that creates a gas key with at least one nonce triggers `add_gas_key` → `gas_key_storage_cost`. No privileged role is required; any account holder can submit such a transaction. The bug fires unconditionally on every such call.

### Recommendation

Replace the truncated `per_nonce_key_size` with the canonical helper, passing the account ID length:

```rust
// runtime/runtime/src/access_keys.rs
fn gas_key_storage_cost(
    fee_config: &RuntimeFeesConfig,
    account_id: &AccountId,          // add this parameter
    public_key: &PublicKey,
    access_key: &AccessKey,
    num_nonces: NonceIndex,
) -> StorageUsage {
    use near_primitives::trie_key::gas_key_nonce_key_len;
    use near_primitives_core::trie_key::access_key_key_len;

    let storage_config = &fee_config.storage_usage_config;
    let per_nonce_value_size = borsh::object_length(&(0 as Nonce)).unwrap() as u64;
    // Correct: full trie key = col_prefix + account_id + separator + pk_handle + NonceIndex
    let per_nonce_key_size =
        (access_key_key_len(account_id.len(), public_key.trie_id_len())
            + size_of::<NonceIndex>()) as u64;

    num_nonces as u64
        * (per_nonce_key_size + per_nonce_value_size + storage_config.num_extra_bytes_record)
        + access_key_storage_usage(fee_config, public_key, access_key)
}
```

All four call sites (`add_gas_key`, `delete_gas_key`, and the two test helpers) must be updated to pass `account_id`.

### Proof of Concept

For an account `"alice.near"` (10 bytes) adding a gas key with `num_nonces = 100`:

- **Actual per-nonce key bytes**: `1 + 10 + 1 + 33 + 4 = 49` (ED25519 key, `trie_id_len = 33`)
- **Computed `per_nonce_key_size`**: `33 + 4 = 37`
- **Underestimation per nonce**: `49 − 37 = 12` bytes
- **Total underestimation**: `12 × 100 = 1 200` bytes of storage stake not locked

The divergence is directly observable by comparing `account.storage_usage()` after `add_gas_key` against the sum of `TrieKey::gas_key_nonce(...).len()` for all nonces plus the access-key entry length.

fn add_gas_key(
    fee_config: &RuntimeFeesConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
    block_height: BlockHeight,
) -> Result<(), StorageError> {
    // For gas keys, nonce stored on access key is not used and should always be zero
    let mut access_key = access_key.clone();
    access_key.nonce = 0;
    set_access_key(state_update, account_id.clone(), public_key.clone(), &access_key);

    // Set up nonces for gas key
    let num_nonces = gas_key_info.num_nonces;
    let nonce = initial_nonce_value(block_height);
    for i in 0..num_nonces {
        set_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i, nonce);
    }

    account.set_storage_usage(
        account
            .storage_usage()
            .checked_add(gas_key_storage_cost(fee_config, public_key, &access_key, num_nonces))
            .ok_or_else(|| {
                StorageError::StorageInconsistentState(format!(
                    "Storage usage integer overflow for account {}",
                    account_id
                ))
            })?,
    );
    Ok(())
}

use std::mem::size_of;

/// Length of the column prefix byte in a trie key (a single `u8`).
const COLUMN_PREFIX_LEN: usize = size_of::<u8>();

/// Length of the access key separator byte in a trie key (a single `u8`).
const ACCESS_KEY_SEPARATOR_LEN: usize = size_of::<u8>();

/// Returns the length of the trie key for an access key.
///
/// The trie key format is: `[col_prefix] [account_id] [separator] [public_key]`
/// where the prefix and separator are each a single `u8` byte.
pub fn access_key_key_len(account_id_len: usize, public_key_len: usize) -> usize {
    COLUMN_PREFIX_LEN + account_id_len + ACCESS_KEY_SEPARATOR_LEN + public_key_len
}
