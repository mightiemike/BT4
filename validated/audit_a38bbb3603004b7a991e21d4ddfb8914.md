### Title
Whitespace-Collision Denom Name Bypass Allows NFT Impersonation — (`x/nft/types/validation.go`, `x/nft/keeper/denom.go`)

### Summary

`ValidateDenomName` trims whitespace only on a local copy for the emptiness check, but never normalises the name before it is stored. `SetDenom` and `HasDenomNm` use the raw, un-trimmed name as the KVStore key. An attacker can therefore register a denom whose name is `"foo "` (trailing space) after a legitimate denom named `"foo"` already exists, because the uniqueness guard compares different byte sequences. The result is two live denoms with visually identical display names but different IDs, one of which the attacker controls.

### Finding Description

**`ValidateDenomName`** (`x/nft/types/validation.go` lines 57–63):

```go
func ValidateDenomName(denomName string) error {
    denomName = strings.TrimSpace(denomName)   // trims a LOCAL copy only
    if len(denomName) == 0 {
        return sdkerrors.Wrapf(ErrInvalidDenomName, ...)
    }
    return nil
}
```

The trimmed value is never returned or stored; the caller's original string is unchanged. [1](#0-0) 

**`SetDenom`** (`x/nft/keeper/denom.go` lines 26–39) calls `HasDenomNm` with the raw name and stores the raw name as the KVStore key:

```go
if k.HasDenomNm(ctx, denom.Name) { ... }          // raw name lookup
store.Set(types.KeyDenomName(denom.Name), ...)     // raw name stored
``` [2](#0-1) 

**`KeyDenomName`** (`x/nft/types/keys.go` lines 125–128) appends the raw name bytes verbatim, so `"foo"` and `"foo "` produce distinct keys: [3](#0-2) 

**Reachable via two paths:**

1. **Normal `MsgIssueDenom` transaction** — `ValidateBasic` calls `ValidateDenomName(msg.Name)` which passes for `"foo "`, then `IssueDenom → SetDenom` stores it under a different key than `"foo"`. [4](#0-3) 

2. **Genesis `InitGenesis`** — `ValidateGenesis` calls `ValidateDenomName` for each collection (passes for both `"foo"` and `"foo "`), then `InitGenesis` calls `SetDenom` for each, both succeed. [5](#0-4) [6](#0-5) 

### Impact Explanation

The attacker controls `denom2` (name `"foo "`) and can mint NFTs under it. Wallets and explorers that display the denom name will show `"foo "` and `"foo"` identically (trailing space is invisible). Users querying `DenomByName("foo ")` receive the attacker's denom. NFTs minted under `denom2` are indistinguishable from those of the legitimate `denom1` to casual inspection, enabling NFT fraud and collection impersonation.

### Likelihood Explanation

The `MsgIssueDenom` path requires no special privilege — any account with enough gas can execute it. The attacker only needs to know the target denom name and append a space. The genesis path additionally requires influence over chain initialization, but the transaction path alone is sufficient and trivially executable.

### Recommendation

Normalize the denom name before any comparison or storage. In `ValidateDenomName`, return an error if the raw input differs from its trimmed form:

```go
func ValidateDenomName(denomName string) error {
    trimmed := strings.TrimSpace(denomName)
    if len(trimmed) == 0 {
        return sdkerrors.Wrapf(ErrInvalidDenomName, "denom name cannot be blank")
    }
    if trimmed != denomName {
        return sdkerrors.Wrapf(ErrInvalidDenomName, "denom name must not have leading/trailing whitespace")
    }
    return nil
}
```

Alternatively, normalize (trim) the name before passing it to `SetDenom` in both `IssueDenom` and `InitGenesis`.

### Proof of Concept

```go
// keeper test — no mocks, standard test setup
func TestWhitespaceNameCollision(t *testing.T) {
    ctx, keeper := setupKeeper(t)

    addr, _ := sdk.AccAddressFromBech32("cosmos1...")
    err := keeper.IssueDenom(ctx, "denom1", "foo", "", "", addr)
    require.NoError(t, err)

    // "foo " differs only by trailing space — should be rejected, but is not
    err = keeper.IssueDenom(ctx, "denom2", "foo ", "", "", addr)
    require.Error(t, err, "expected ErrInvalidDenom for whitespace-collision name")
}
```

With the current code the second `IssueDenom` succeeds, leaving two denoms with visually identical names in the store.

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
