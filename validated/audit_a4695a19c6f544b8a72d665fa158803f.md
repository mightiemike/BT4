### Title
Integer Overflow in `op_unknown` Cost Multiplier Produces Undercharged Execution — (`File: src/more_ops.rs`)

---

### Summary

`op_unknown` in `src/more_ops.rs` computes a final cost as `cost *= cost_multiplier + 1` using unchecked `u64` multiplication. Because `[profile.release]` in `Cargo.toml` does not set `overflow-checks = true`, release builds silently wrap on overflow. The post-multiplication guard `if cost > u32::MAX as u64` only inspects the already-wrapped value, not the true product. An attacker-controlled opcode and argument list can be crafted so the wrapped cost passes the guard and the program is accepted at a fraction of its true cost — a direct analog of the missing upper-bound check on `royaltyNumerator`.

---

### Finding Description

In `op_unknown`, after accumulating cost from arguments (bounded by `check_cost` to `<= max_cost`), the code performs:

```rust
// src/more_ops.rs lines 258-266
assert!(cost > 0);

check_cost(cost, max_cost)?;
cost *= cost_multiplier + 1;          // ← unchecked u64 multiply
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
``` [1](#0-0) 

`cost_multiplier` is decoded as a `u64` from up to 4 opcode bytes:

```rust
let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
    Some(v) => v as u64,
    None => { return Err(EvalErr::Invalid(o))?; }
};
``` [2](#0-1) 

`cost_multiplier` is therefore at most `u32::MAX = 4,294,967,295`, so `cost_multiplier + 1` is at most `4,294,967,296 = 2^32`. The product `cost * (cost_multiplier + 1)` can exceed `u64::MAX` when `cost` is large enough, wrapping to a small value that passes the `u32::MAX` guard.

**Concrete trigger** (choosing `cost_multiplier = 2^31 − 1 = 0x7fffffff`):

| Parameter | Value |
|---|---|
| Opcode bytes | `[0x7f, 0xff, 0xff, 0xff, 0x40]` |
| `cost_function` | 1 (ARITH-like, top 2 bits of last byte = `01`) |
| `cost_multiplier` | `0x7fffffff = 2,147,483,647` |
| `cost_multiplier + 1` | `2^31 = 2,147,483,648` |
| Required pre-multiply `cost` | `2^33 + 1 = 8,589,934,593` |
| True product | `(2^33 + 1) × 2^31 = 2^64 + 2^31` |
| Wrapped result (mod 2^64) | `2^31 = 2,147,483,648` |
| Guard `cost > u32::MAX`? | `2,147,483,648 > 4,294,967,295` → **false** |
| Returned cost | `2,147,483,648` instead of `~1.84 × 10^19` |

The opcode `[0x7f, 0xff, 0xff, 0xff, 0x40]` does not begin with `0xff, 0xff`, so the reserved-opcode check at line 197 does not fire:

```rust
if op.is_empty() || (op.len() >= 2 && op[0] == 0xff && op[1] == 0xff) {
    Err(EvalErr::Reserved(o))?;
}
``` [3](#0-2) 

The pre-multiply `cost = 8,589,934,593` is achievable with `cost_function = 1`:

```
cost = ARITH_BASE_COST + N × ARITH_COST_PER_ARG + byte_count × ARITH_COST_PER_BYTE
     = 99 + N × 320 + byte_count × 3
``` [4](#0-3) 

Since `gcd(320, 3) = 1`, any target cost is reachable by adjusting `N` and `byte_count`. The value `8,589,934,593` is below Chia's ~11 billion block cost limit, so `check_cost` does not reject it.

The `[profile.release]` section in `Cargo.toml` sets only `lto = "thin"` — no `overflow-checks = true` — confirming that release builds wrap silently:

```toml
[profile.release]
lto = "thin"
``` [5](#0-4) 

---

### Impact Explanation

The returned `Reduction` cost is `2,147,483,648` (~2.1 billion) instead of the true cost of `~1.84 × 10^19`. This cost is added to the running total in `run_program`:

```rust
cost += match op {
    Operation::Apply => self.apply_op(cost, effective_max_cost - cost)?,
    ...
``` [6](#0-5) 

A program that should be rejected for exceeding the block cost limit is instead accepted. This constitutes **undercharged execution**: the attacker's program consumes a cost slot orders of magnitude smaller than its true cost, allowing it to coexist with other transactions in a block that would otherwise be full. Nodes running debug builds (where overflow panics) would reject the block, causing a **consensus split** between debug and release nodes.

---

### Likelihood Explanation

- `op_unknown` is reachable in consensus mode (without `NO_UNKNOWN_OPS`). The `ChiaDialect` in consensus mode does not set this flag, so `unknown_operator` calls `op_unknown` directly.
- `MEMPOOL_MODE` sets `NO_UNKNOWN_OPS`, so mempool nodes reject the transaction before it reaches `op_unknown`. However, consensus-mode block validators do not.
- The attacker controls both the opcode bytes (setting `cost_multiplier`) and the argument atoms (setting the accumulated `cost`), giving full control over both operands of the overflow.
- No special privileges are required; any coin puzzle submitted to the network can use an unknown opcode. [7](#0-6) 

---

### Recommendation

Replace the unchecked multiplication with a checked variant and treat overflow as an invalid opcode:

```rust
let Some(new_cost) = cost.checked_mul(cost_multiplier + 1) else {
    return Err(EvalErr::Invalid(o))?;
};
cost = new_cost;
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

Additionally, add `overflow-checks = true` to `[profile.release]` in `Cargo.toml` as a defense-in-depth measure to surface any other latent overflow bugs during production builds.

---

### Proof of Concept

**Opcode construction:**
- Bytes: `[0x7f, 0xff, 0xff, 0xff, 0x40]`
- `cost_function = (0x40 & 0b11000000) >> 6 = 1` (ARITH-like)
- `cost_multiplier = u32_from_u8([0x7f, 0xff, 0xff, 0xff]) = 0x7fffffff = 2,147,483,647`

**Argument construction (cost_function = 1):**
- Supply `N` atom arguments with total byte count `B` such that:
  `99 + N × 320 + B × 3 = 8,589,934,593`
- Example: `N = 26,843,545`, `B = 1` → cost ≈ `8,589,934,499` (adjust B to hit exact target)

**Overflow arithmetic:**
```
cost_before = 8,589,934,593  (= 2^33 + 1)
multiplier  = 2,147,483,648  (= 2^31)
product     = 2^64 + 2^31
wrapped     = 2^31 = 2,147,483,648
guard check: 2,147,483,648 > 4,294,967,295 → false
returned cost: 2,147,483,648   (true cost: ~1.84 × 10^19)
```

The program executes and is accepted by a consensus-mode validator at ~8.6 billion times less than its true cost.
