Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is bound to `msg.sender` at the pool's `swap` entry point. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If a pool admin allowlists the router address to enable standard periphery UX, every user — including those the allowlist was meant to exclude — can bypass the per-user gate entirely by routing through the router.

## Finding Description

**Root cause: wrong-actor binding on the swap allowlist path.**

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(params.recipient, ...)` directly: [4](#0-3) 

At that point `msg.sender` inside the pool is the **router contract**, so `sender` delivered to the extension is the router address. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**No existing guard corrects this.** The extension has no mechanism to distinguish a direct caller from a router-mediated caller. The `allowAllSwappers` flag and `allowedSwapper` mapping both operate solely on the `sender` argument, which is structurally bound to the immediate pool caller, not the economic actor.

The same misbinding applies to `exactOutputSingle` and `exactOutput` (multi-hop), all of which call `pool.swap` with the router as `msg.sender`. [5](#0-4) 

## Impact Explanation
A curated pool relying on `SwapAllowlistExtension` as its sole swap gate is fully bypassed once the pool admin allowlists the router. Any unprivileged user can call `exactInputSingle` pointing at the restricted pool and execute swaps. The pool's token reserves are exposed to unauthorized traders, directly impacting LP principal and fee revenue. This constitutes a broken core pool functionality / admin-boundary break with direct loss of LP assets — matching the allowed impact gate for Critical/High severity.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entry point. A pool admin deploying a curated pool (KYC-gated, institutional, protocol-owned) who wants users to interact through the standard periphery will naturally allowlist the router — this is the intended operational pattern. The bypass requires no special privilege, no flash loan, and no unusual token behavior. Any EOA can call `exactInputSingle` on the router pointing at the restricted pool. The precondition (router allowlisted) is a routine admin action, making this highly likely in practice.

## Recommendation
The extension must check the **economic actor** (the end user), not the intermediary. The cleanest fix mirrors the existing `_getPayer()` / `_getTokenToPay()` transient storage pattern already used in the router for callback settlement:

1. Before calling `pool.swap`, the router stores the original `msg.sender` in transient storage.
2. The pool or extension reads that value via a trusted accessor on the router.
3. `SwapAllowlistExtension.beforeSwap` checks the transient-storage-sourced user identity instead of (or in addition to) `sender`.

Alternatively, if the pool design guarantees `recipient` is always the end user for single-hop swaps, the extension could gate on `recipient` — but this does not hold for multi-hop paths where intermediate recipients are the router itself. [6](#0-5) 

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // Alice is KYC'd
  - Pool admin calls setAllowedToSwap(pool, router, true)  // Enable standard periphery

Attack (Bob, not allowlisted):
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({
         pool: restrictedPool,
         recipient: bob,
         zeroForOne: true,
         amountIn: X,
         ...
     })
  2. Router calls pool.swap(bob, true, X, ...) — msg.sender inside pool = router
  3. Pool calls _beforeSwap(router, bob, ...)
  4. Extension checks allowedSwapper[pool][router] → true (router is allowlisted)
  5. Swap executes; Bob receives tokens from the restricted pool

Result:
  - Bob bypassed the per-user allowlist entirely
  - allowedSwapper[pool][bob] was never consulted
  - LP funds in the restricted pool are exposed to unauthorized traders
```

    );
