### Title
Governance `MsgUpdateTier` can set `CloseOnly` while positions are in the exit commitment period, permanently blocking `MsgClearPosition` - (File: `x/tieredrewards/keeper/msg_validate.go`)

---

### Summary

The `x/tieredrewards` module allows governance to set a tier's `CloseOnly` flag via `MsgUpdateTier` at any time, including while users have active positions in the exit commitment period. Once `CloseOnly` is set, `MsgClearPosition` is unconditionally blocked for all positions in that tier. Because the governance voting period can be shorter than the tier's `ExitDuration`, governance can strip users of their ability to cancel an in-progress exit before the exit commitment elapses — directly analogous to the NibblVault `BUYOUT_DURATION > UPDATE_TIME` mismatch.

---

### Finding Description

When a user calls `MsgTriggerExitFromTier`, the keeper reads the current `tier.ExitDuration` and stores a fixed `ExitUnlockAt = block_time + tier.ExitDuration` on the position:

```go
// x/tieredrewards/types/position.go
func (p *Position) TriggerExit(blockTime time.Time, duration time.Duration) {
    p.ExitTriggeredAt = blockTime
    p.ExitUnlockAt = blockTime.Add(duration)
}
``` [1](#0-0) 

During the exit commitment period (before `ExitUnlockAt`), the user can cancel via `MsgClearPosition` to re-lock and continue earning bonus rewards. The ADR explicitly documents this as a supported user action:

> "Cancel exit — `MsgClearPosition` can cancel exit at any point during or after the exit commitment, as long as the position is delegated." [2](#0-1) 

However, `validateClearPosition` unconditionally rejects the call if the tier is `CloseOnly`, with no exception for positions whose exit was triggered before the tier became `CloseOnly`:

```go
// x/tieredrewards/keeper/msg_validate.go
func (k Keeper) validateClearPosition(ctx context.Context, pos types.PositionState, owner string) error {
    ...
    tier, err := k.getTier(ctx, pos.TierId)
    ...
    if tier.IsCloseOnly() {
        return types.ErrTierIsCloseOnly
    }
    return nil
}
``` [3](#0-2) 

The `UpdateTier` handler applies the `CloseOnly` flag immediately with no guard for existing exiting positions:

```go
// x/tieredrewards/keeper/msg_server_auth.go
func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
    ...
    // Only BonusApy changes trigger reward settlement; CloseOnly change has no guard
    if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
        if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil { ... }
    }
    if err := ms.SetTier(ctx, msg.Tier); err != nil { ... }
    ...
}
``` [4](#0-3) 

The `Tier` struct stores `ExitDuration` as a governance-managed field with no minimum relative to the governance voting period: [5](#0-4) 

The ADR documents that `ExitDuration` is intended to be long (e.g., 1 year), while governance voting periods are typically much shorter (e.g., 14 days): [6](#0-5) 

**Broken invariant**: There is no enforcement that `GovernanceVotingPeriod >= ExitDuration`. Governance can pass a `CloseOnly` proposal in a time shorter than the exit commitment period, retroactively blocking `ClearPosition` for all positions that triggered exit before the proposal passed.

---

### Impact Explanation

A user who called `MsgTriggerExitFromTier` expecting to be able to cancel the exit (via `MsgClearPosition`) within the `ExitDuration` window loses that ability the moment governance passes a `CloseOnly` update. The user is forced to complete the exit, losing:

- The ability to re-lock and resume earning bonus APY rewards for the remainder of the exit commitment period.
- The ability to redelegate (`MsgTierRedelegate` is also blocked by `CloseOnly`).

The user's principal is recoverable (exit paths remain open), but the loss of future bonus rewards is a concrete economic harm — not merely "faith in the protocol." The corrupted invariant is the `Position.ExitUnlockAt` timestamp: it was set under the assumption that `ClearPosition` would remain available until that time, but governance can invalidate that assumption before `ExitUnlockAt` is reached. [7](#0-6) 

---

### Likelihood Explanation

The governance voting period is a chain parameter that is typically far shorter than a tier's `ExitDuration`. The integration test configuration uses `voting_period: '10s'` and `exit_duration: '5s'` (test values), but production tiers are designed for durations like 1 year. Any governance proposal to sunset a tier (set `CloseOnly = true`) that passes while users are in the exit commitment period triggers this condition. This is a realistic governance action (sunsetting a tier is an expected lifecycle event), and there is no on-chain mechanism to prevent it from occurring mid-exit. [8](#0-7) 

---

### Recommendation

One or more of the following mitigations should be applied:

1. **Allow `ClearPosition` for pre-CloseOnly exits**: In `validateClearPosition`, skip the `CloseOnly` check if `pos.ExitTriggeredAt` is non-zero and was set before the tier became `CloseOnly` (requires storing a `CloseOnlySetAt` timestamp on the tier).

2. **Enforce the duration invariant at governance time**: In `UpdateTier`, when setting `CloseOnly = true`, verify that no positions currently have `ExitTriggeredAt.IsZero() == false && ExitUnlockAt > block_time` (i.e., no positions are mid-exit), or alternatively require a grace period.

3. **Enforce `ExitDuration <= GovernanceVotingPeriod` as a tier validation rule**: Add a check in `Tier.Validate()` or in `UpdateTier` that `ExitDuration` does not exceed the current governance voting period, so the invariant `ExitDuration < VotingPeriod` holds and governance can always complete before any exit commitment. [9](#0-8) 

---

### Proof of Concept

1. Governance creates Tier 1 with `ExitDuration = 365 days`, `BonusApy = 10%`. Governance voting period is 14 days.
2. User A calls `MsgTriggerExitFromTier` on their position. `ExitUnlockAt = now + 365 days` is stored on the position.
3. After 7 days, governance submits `MsgUpdateTier` with `CloseOnly = true` for Tier 1.
4. After 14 days (voting period), the proposal passes. `tier.CloseOnly = true` is written to state.
5. User A, now 21 days into a 365-day exit commitment, calls `MsgClearPosition` to cancel and re-lock.
6. `validateClearPosition` reads `tier.IsCloseOnly() == true` and returns `ErrTierIsCloseOnly`. The call fails.
7. User A is permanently locked into the exit for the remaining ~344 days with no ability to cancel, losing all future bonus rewards they would have earned by re-locking. [10](#0-9) [11](#0-10) [12](#0-11)

func (k Keeper) validateClearPosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
		if err != nil {
			return err
		}
		if isUnbonding {
			return types.ErrPositionUnbonding
		}

		if !pos.IsDelegated() {
			return types.ErrPositionNotDelegated
		}

	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	return nil
}

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

func (t Tier) Validate() error {
	if t.Id == 0 {
		return ErrInvalidTierID
	}

	if t.ExitDuration <= 0 {
		return fmt.Errorf("exit duration must be positive")
	}

	if t.BonusApy.IsNil() {
		return fmt.Errorf("bonus apy cannot be nil")
	}

	if t.BonusApy.IsNegative() {
		return fmt.Errorf("bonus apy cannot be negative: %s", t.BonusApy)
	}

	// Cap BonusApy at 100% to prevent governance from draining the rewards pool.
	if t.BonusApy.GT(math.LegacyOneDec()) {
		return fmt.Errorf("bonus apy must not exceed 1.0 (100%%): got %s", t.BonusApy)
	}

	if t.MinLockAmount.IsNil() {
		return fmt.Errorf("min lock amount cannot be nil")
	}

	if t.MinLockAmount.IsNegative() {
		return fmt.Errorf("min lock amount cannot be negative: %s", t.MinLockAmount)
	}

	return nil
}

func (ms msgServer) TriggerExitFromTier(ctx context.Context, msg *types.MsgTriggerExitFromTier) (*types.MsgTriggerExitFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateTriggerExit(pos.Position, msg.Owner); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitTriggered{
		PositionId:   pos.Id,
		TierId:       pos.TierId,
		Owner:        pos.Owner,
		ExitUnlockAt: pos.ExitUnlockAt,
	}); err != nil {
		return nil, err
	}

	return &types.MsgTriggerExitFromTierResponse{
		ExitUnlockAt: pos.ExitUnlockAt,
		PositionId:   pos.Id,
	}, nil
}

func (ms msgServer) ClearPosition(ctx context.Context, msg *types.MsgClearPosition) (*types.MsgClearPositionResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateClearPosition(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	if !pos.HasTriggeredExit() {
		return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.ClearExit(sdkCtx.BlockTime())

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitCleared{
		PositionId: pos.Id,
		TierId:     pos.TierId,
		Owner:      pos.Owner,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
}
