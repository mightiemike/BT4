Audit Report

## Title
SwapAllowlistExtension Gates on Router Address Instead of Actual User, Allowing Any User to Bypass Curated Pool Restrictions via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument passed by the pool, which equals `msg.sender` of the `pool.swap()` call — the router address, not the actual user. When a pool admin allowlists the router to enable their approved users to trade conveniently, every unapproved user gains access to the curated pool by routing through `MetricOmmSimpleRouter`. The per-user allowlist invariant is completely bypassed.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    ...
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist, where `msg.sender` inside the extension is the pool address:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the pool's `msg.sender` the router:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The actual user's identity is stored only in transient storage for the payment callback (`_setNextCallbackContext`) and is never surfaced to the extension. The extension therefore evaluates `allowedSwapper[pool][router]`. Once the router is allowlisted — the only way for approved users to use the router on a curated pool — the check passes for every caller regardless of identity. The same flaw applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

## Impact Explanation
Any unprivileged user can trade on a KYC-gated, institution-only, or regulatory-restricted pool by routing through the public `MetricOmmSimpleRouter`. The allowlist invariant — "only approved addresses may swap" — is broken. LP funds are exposed to counterparties the pool was explicitly designed to exclude, and regulatory or compliance guarantees of the pool operator are violated. This constitutes a broken core pool access-control mechanism with direct fund-level consequences for LPs in curated pools.

## Likelihood Explanation
The bypass requires the router to be allowlisted. This is not an edge case: it is the only way allowlisted users can use the standard periphery router on a curated pool. Any operator who deploys a curated pool and wants their approved users to access it through `MetricOmmSimpleRouter` must allowlist the router. The configuration that triggers the vulnerability is the expected production configuration. The attack requires no special privileges, no capital beyond the swap amount, and is repeatable by any on-chain address.

## Recommendation
The extension must gate on the actual economic actor, not the immediate caller of `pool.swap()`. Two sound approaches:

1. **Router-forwarded identity via `extensionData`**: Require the router to ABI-encode the actual user's address into `extensionData`. The extension decodes and checks it, and rejects calls where `sender` is a known router but no valid user identity is present in `extensionData`.
2. **Separate allowlist variant for routed flows**: Provide a distinct extension that decodes user identity from `extensionData` for router-mediated flows, and document clearly that allowlisting the router in the base `SwapAllowlistExtension` opens the pool to all users.

Do not use `tx.origin` as it breaks contract-to-contract composability.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is approved.
3. Admin calls `setAllowedToSwap(pool, router, true)` — router is approved so Alice can use it.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. Router calls `pool.swap(bob, ...)` with `msg.sender = router`.
6. Pool calls `extension.beforeSwap(sender=router, ...)`.
7. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
8. Bob successfully trades on the curated pool despite never being allowlisted. [1](#0-0) [2](#0-1) [3](#0-2)
