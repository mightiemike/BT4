Audit Report

## Title
SwapAllowlistExtension checks the router address instead of the actual user, allowing allowlist bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. Allowlisting the router — the only way to let allowlisted users access the router — opens the pool to every user on-chain, rendering the per-user swap allowlist unenforceable.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` (the direct caller) as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly — making the router `msg.sender` to the pool: [3](#0-2) 

The same applies to `exactInput` (line 104), `exactOutputSingle` (line 136), and `exactOutput` (line 165) — all call `pool.swap` from the router contract. [4](#0-3) 

The result is a forced dilemma: if the pool admin allowlists only specific users, those users cannot use the router (router is blocked). If the pool admin allowlists the router, every user on-chain can bypass the allowlist by routing through it. There is no configuration that allows specific users to use the router while blocking others.

**Contrast with the deposit allowlist — which is correct.** `DepositAllowlistExtension.beforeAddLiquidity` checks the `owner` argument (the economic actor, second parameter), not `sender` (the technical caller, first parameter): [5](#0-4) 

The swap allowlist should analogously gate the actual user, but instead gates the technical caller.

## Impact Explanation

Any pool deploying `SwapAllowlistExtension` to restrict swaps to a curated set (KYC-gated, institutional-only, MEV-protected) is rendered unenforceable the moment the pool admin allowlists the router. Non-allowlisted users can swap freely, extracting value from LPs who expected a restricted counterparty set. This constitutes a direct loss of LP principal through trades the pool was explicitly configured to block — a broken core pool functionality causing loss of funds. Severity: High.

## Likelihood Explanation

The trigger is a pool admin allowlisting the router — a natural and expected action when allowlisted users need to access standard periphery. The admin is not acting maliciously; the misconfiguration is invisible because `setAllowedToSwap(pool, router, true)` gives no indication it opens the pool to all users. Any pool combining `SwapAllowlistExtension` with the public router is vulnerable once the router is added to the allowlist.

## Recommendation

Gate the actual user, not the technical caller. The simplest fix mirrors the deposit allowlist pattern: check `recipient` (the second parameter of `beforeSwap`, the address receiving output tokens) instead of `sender`. This is the economically relevant actor, analogous to `owner` in `beforeAddLiquidity`. Alternatively, have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` and have the extension decode and verify it when the caller is a known trusted router.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` as a `beforeSwap` hook.
2. Pool admin allowlists Alice: `setAllowedToSwap(pool, alice, true)`.
3. Alice wants to use the router, so pool admin also allowlists the router: `setAllowedToSwap(pool, router, true)`.
4. Non-allowlisted Bob calls `router.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. The router calls `pool.swap(bob, ...)` — `msg.sender` to the pool is the router.
6. `beforeSwap` checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
7. Bob successfully swaps despite never being allowlisted, bypassing the per-user restriction entirely.
