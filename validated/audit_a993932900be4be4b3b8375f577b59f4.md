Audit Report

## Title
Unguarded `refundETH()` allows any caller to drain stranded ETH from `MetricOmmSimpleRouter` across transactions — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.refundETH()` transfers the router's entire native ETH balance to `msg.sender` with no access control. ETH sent via a `payable` `multicall` that is not fully consumed by the swap (because `pay()` wraps only the exact amount needed) remains on the contract. Any subsequent unprivileged caller can invoke `refundETH()` in a separate transaction and receive all of it, causing direct loss of user ETH principal.

## Finding Description

`refundETH()` contains no caller restriction:

```solidity
// PeripheryPayments.sol L58-63
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // no depositor check
    }
}
``` [1](#0-0) 

ETH enters legitimately via `multicall`, which is `public payable`:

```solidity
// MetricOmmSimpleRouter.sol L39-44
function multicall(bytes[] calldata data) public payable override returns (bytes[] memory results) {
    results = new bytes[](data.length);
    for (uint256 i = 0; i < data.length; i++) {
        results[i] = Address.functionDelegateCall(address(this), data[i]);
    }
}
``` [2](#0-1) 

Inside `pay()`, when `token == WETH`, only the exact `value` required by the swap is deposited into WETH; any excess native ETH remains on the contract:

```solidity
// PeripheryPayments.sol L75-77
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
}
``` [3](#0-2) 

The `receive()` fallback only blocks *direct* ETH sends from non-WETH addresses; it does not prevent ETH from entering via payable function calls like `multicall` or `exactInputSingle`: [4](#0-3) 

**Cross-transaction attack path:**

1. User A calls `multicall{value: 1 ETH}([exactInputSingle_calldata])` — the swap consumes 0.5 ETH via `pay()`; 0.5 ETH remains on the router because the user omitted a trailing `refundETH()` call in their batch.
2. Attacker calls `router.refundETH()` in a separate transaction.
3. Attacker receives 0.5 ETH; User A's ETH is unrecoverable.

**Why the "same transaction" framing is wrong:** `multicall` uses `delegatecall`, so every sub-call shares the original `msg.sender`. An attacker cannot inject a `refundETH()` call into a victim's multicall batch. The exploit is strictly cross-transaction.

**Why existing guards are insufficient:** The `receive()` guard does not prevent ETH accumulation via payable entry points. There is no transient-storage depositor record, no per-caller balance accounting, and no restriction on who may call `refundETH()`.

## Impact Explanation

Direct loss of user ETH principal. Any ETH deposited via a payable multicall that is not fully consumed by the swap and not reclaimed within the same multicall batch is permanently accessible to any external caller. Loss is bounded only by the amount the victim sent. This satisfies the "direct loss of user principal" impact gate at Medium severity.

## Likelihood Explanation

Medium. The standard Uniswap-style UX pattern expects users to append `refundETH()` as the last call in their multicall batch. Users who omit it — through UI bugs, direct contract interaction, or integrator error — leave ETH exposed. MEV bots routinely monitor for stranded ETH on router contracts and will extract it within the same block.

## Recommendation

Track the depositor in transient storage at `multicall` entry and require `msg.sender == depositor` inside `refundETH()`. Alternatively, remove the standalone `refundETH()` external entrypoint and only allow ETH refunds within the multicall context where the original depositor is already `msg.sender`. A minimal fix:

```solidity
// at multicall entry, store msg.sender in a transient slot
// inside refundETH(), require msg.sender == tload(DEPOSITOR_SLOT)
```

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "metric-periphery/contracts/MetricOmmSimpleRouter.sol";

contract RefundETHDrainTest is Test {
    MetricOmmSimpleRouter router;
    address userA    = address(0xA);
    address attacker = address(0xB);

    function setUp() public {
        router = new MetricOmmSimpleRouter(address(weth), address(factory));
        vm.deal(userA, 1 ether);
    }

    function test_attackerDrainsUserETH() public {
        // Simulate: userA's multicall consumed 0.5 ETH; 0.5 ETH stranded on router
        vm.deal(address(router), 0.5 ether);

        uint256 before = attacker.balance;

        vm.prank(attacker);
        router.refundETH();

        assertEq(attacker.balance, before + 0.5 ether);
        assertEq(address(router).balance, 0);
    }
}
```
