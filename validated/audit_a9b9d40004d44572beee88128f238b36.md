Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Originating EOA, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is always `msg.sender` of `MetricOmmPool.swap()`. When users route through `MetricOmmSimpleRouter`, `msg.sender` is the router contract address, not the originating EOA. Any pool admin who adds the router to the allowlist (the only way to let allowlisted users trade via the router) simultaneously opens the pool to every user on the network, completely defeating the allowlist's curation guarantee.

## Finding Description
The full call chain is confirmed in production code:

**Step 1 — `MetricOmmPool.swap()` passes `msg.sender` (the router) as `sender` to `_beforeSwap`:** [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap()` forwards `sender` unchanged into the extension calldata:** [2](#0-1) 

**Step 3 — `SwapAllowlistExtension.beforeSwap()` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router:** [3](#0-2) 

**Step 4 — `MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` with the router as `msg.sender`; the originating EOA is never forwarded:** [4](#0-3) 

The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`, all of which call `pool.swap()` with `msg.sender = router`. [5](#0-4) [6](#0-5) 

**Root cause:** The `allowedSwapper` mapping is keyed by `[pool][swapper]`, where `swapper` is the immediate caller of `pool.swap()`. [7](#0-6) 

When a pool admin adds individual EOAs to the allowlist, those EOAs can only swap by calling `pool.swap()` directly. If they use the router, the check fails because `allowedSwapper[pool][eoa]` is set but `allowedSwapper[pool][router]` is not. To restore router usability for legitimate users, the admin must add `address(router)` to the allowlist — but this entry passes for **any** caller of the router, not just the intended EOAs. The originating EOA identity is never inspected anywhere in the call chain.

## Impact Explanation
This is a broken core pool functionality issue causing direct loss of the curation guarantee that LP depositors and the pool admin relied upon. Once the router is allowlisted:
- Any unprivileged user can execute swaps on a pool designed to be closed (e.g., KYC-gated, institutional-only).
- LP positions are exposed to toxic flow the allowlist was meant to exclude.
- Protocol fees accrue from unauthorized trades.
- The pool admin has no on-chain recourse short of removing the router from the allowlist entirely, which simultaneously breaks the UX for all legitimate allowlisted users.

This meets the "Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" and "Admin-boundary break" allowed impact criteria.

## Likelihood Explanation
High. `MetricOmmSimpleRouter` is the primary documented user-facing swap entrypoint. Any pool admin who wants allowlisted users to have a normal swap UX must add the router to the allowlist. The bypass requires no special knowledge, no privileged access, no malicious setup, and no non-standard tokens — any user who observes `allowedSwapper[pool][router] == true` and calls the router directly can exploit it. The precondition (router on allowlist) is the natural, good-faith operational state for any allowlist pool that supports router-based trading.

## Recommendation
The extension must gate the **originating user**, not the immediate caller of `pool.swap()`. Options:

1. **Preferred structural fix:** Have the pool record `tx.origin` (or a caller-override field) in transient storage at the start of `swap()` and expose it to extensions, so extensions always see the true initiator regardless of intermediaries.
2. **Router-cooperative fix:** Require the router to encode the originating `msg.sender` into `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and verify it. This requires the router to be trusted and the extension to validate the encoding.
3. **Short-term mitigation:** Document that adding the router to the allowlist opens the pool to all users, and require pool admins to allowlist individual EOAs only, accepting that those EOAs cannot use the router.

## Proof of Concept
```solidity
// Setup:
// - Pool configured with SwapAllowlistExtension
// - allowedSwapper[pool][allowedUser] = true  (legitimate user)
// - Pool admin adds router: allowedSwapper[pool][address(router)] = true
//   (so allowedUser can use the router)

// Attacker (not on allowlist) calls:
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(allowlistedPool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000e18,
        amountOutMinimum: 0,
        recipient:        attacker,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// pool.swap() is called with msg.sender = router
// _beforeSwap(sender=router, ...)
// SwapAllowlistExtension checks allowedSwapper[pool][router] → true
// Swap executes; attacker receives token1
// Allowlist is fully bypassed
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
