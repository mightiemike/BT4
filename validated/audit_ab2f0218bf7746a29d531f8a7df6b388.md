### Title
Unguarded `submitTransactionsCheckedWithGasLimit` Leaks Internal Gas Execution Data to Any Unprivileged Caller — (File: `core/contracts/Endpoint.sol`)

---

### Summary

`Endpoint.submitTransactionsCheckedWithGasLimit` is an `external` function with no caller authorization and no Schnorr signature check. It processes arbitrary caller-supplied transactions through the full `processTransaction` pipeline and then **unconditionally reverts** via `Verifier.revertGasInfo`, encoding the transaction index and cumulative gas consumed into the revert string. Any unprivileged address can call this function, probe the protocol's internal execution paths, and extract gas-cost telemetry from the revert data — a direct on-chain analog to the verbose error disclosure in the external report.

---

### Finding Description

`submitTransactionsChecked` (the production sequencer path) enforces two guards before processing any transaction:

```
require(msg.sender == sequencer);
verifier.requireValidSignature(digest, e, s, signerBitmask);
```

`submitTransactionsCheckedWithGasLimit` omits both: [1](#0-0) 

The function reads `nSubmissions` (a `public` storage variable) to satisfy `validateSubmissionIdx`, which any caller can read off-chain before calling: [2](#0-1) 

After processing each transaction through `processTransaction`, the function unconditionally calls:

```solidity
verifier.revertGasInfo(transactions.length, initialGas - gasleft());
```

`revertGasInfo` is declared `external pure` with no access control and always reverts with a human-readable string encoding `(i, gasUsed)`: [3](#0-2) 

It is also exposed in the `IVerifier` interface, confirming it is an intentional public surface: [4](#0-3) 

**Attacker-controlled entry path:**

1. Read `nSubmissions` from `Endpoint` (public).
2. Craft any array of `transactions` bytes (e.g., `DepositCollateral`, `WithdrawCollateral`, `LiquidateSubaccount`, `MatchOrders`).
3. Call `submitTransactionsCheckedWithGasLimit(nSubmissions, transactions, type(uint256).max)`.
4. `validateSubmissionIdx` passes; each transaction is dispatched through `processTransaction` → `_delegatecallEndpointTx` → full engine logic.
5. The function always terminates with a revert whose message is `"G <count> <gasUsed>"`, leaking precise gas consumption for the submitted batch.

Because the outer call reverts, all EVM state changes are rolled back. However, the revert data is observable by any caller via `eth_call` or by inspecting the failed transaction receipt.

---

### Impact Explanation

**Information disclosure — internal execution telemetry exposed to any caller.**

- An attacker can measure the exact gas cost of any transaction type (deposit, withdrawal, liquidation, order match) without sequencer authorization. This reveals internal loop counts, storage slot access patterns, and engine branching behavior.
- By varying transaction parameters and observing gas deltas, an attacker can infer internal protocol state: number of registered products, subaccount existence, funding-rate accumulator depth, and NLP pool iteration cost — none of which require a successful state-changing call.
- The gas profiling data can be used to identify the cheapest path to trigger specific engine branches, optimizing griefing or sandwich strategies.
- The function processes real engine logic (including `delegatecall` into `EndpointTx`) before reverting, meaning any revert-observable side channel (e.g., event emission order, storage read patterns) is also accessible.

**Impact: 3 / 5** — No direct asset loss per invocation, but the disclosed telemetry materially aids targeted follow-on attacks.

---

### Likelihood Explanation

- The function is `external` with no modifier.
- `nSubmissions` is `public`, satisfying the only input constraint.
- The call can be made via `eth_call` at zero cost to the attacker.
- No special role, deposit, or prior interaction is required.

**Likelihood: 4 / 5** — Trivially reachable by any address with knowledge of the ABI.

---

### Recommendation

Remove `submitTransactionsCheckedWithGasLimit` from the production deployment entirely, or restrict it with the same sequencer guard used in `submitTransactionsChecked`:

```solidity
require(msg.sender == sequencer, ERR_UNAUTHORIZED);
```

If gas profiling is needed off-chain, it should be performed via a dedicated view-simulation harness that is never deployed to mainnet, or restricted to `onlyOwner` / sequencer-only access.

### Proof of Concept

```solidity
// Any EOA or contract can execute this via eth_call or a live tx:
uint64 idx = IEndpoint(endpoint).nSubmissions(); // public getter

bytes[] memory txns = new bytes[](1);
// Craft a DepositCollateral transaction body for an existing product
txns[0] = abi.encodePacked(
    uint8(IEndpoint.TransactionType.DepositCollateral),
    abi.encode(IEndpoint.DepositCollateral({
        sender: targetSubaccount,
        productId: 0,
        amount: 1e18
    }))
);

// Call with no sequencer key, no Schnorr signature
try IEndpoint(endpoint).submitTransactionsCheckedWithGasLimit(
    idx,
    txns,
    type(uint256).max
) {} catch Error(string memory reason) {
    // reason == "G 1 <gasUsed>" — gas cost of processing one DepositCollateral
    // is now known to the attacker
    emit GasLeaked(reason);
}
```

The revert string `"G <txCount> <totalGasUsed>"` is returned unconditionally on every call, matching the verbose error disclosure pattern in the external report. [1](#0-0) [3](#0-2)

    function submitTransactionsCheckedWithGasLimit(
        uint64 idx,
        bytes[] calldata transactions,
        uint256 gasLimit
    ) external {
        uint256 initialGas = gasleft();
        validateSubmissionIdx(idx);
        for (uint256 i = 0; i < transactions.length; i++) {
            bytes calldata transaction = transactions[i];
            processTransaction(transaction);
            uint256 gasUsed = initialGas - gasleft();
            if (gasUsed > gasLimit) {
                verifier.revertGasInfo(i, gasUsed);
            }
        }
        verifier.revertGasInfo(transactions.length, initialGas - gasleft());
    }
