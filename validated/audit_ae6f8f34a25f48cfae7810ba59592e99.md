### Title
`ValidResourceBounds::L1Gas` vs `AllResources` Hash Preimage Divergence After Protobuf Round-Trip Causes P2P Propagation Failure and Block-Hash Inconsistency — (File: `crates/starknet_api/src/transaction_hash.rs`, `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

`get_tip_resource_bounds_hash` produces a **different Poseidon hash** for `ValidResourceBounds::L1Gas(X)` versus `ValidResourceBounds::AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` because the `AllResources` branch appends a third felt (`l1_data_gas_packed`) to the hash preimage while the `L1Gas` branch does not. The protobuf deserializer `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` silently collapses any `AllResources` with zero `l2_gas` and zero `l1_data_gas` back to `L1Gas`. A user-submitted `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the originating gateway, assigned hash H1 (3-element preimage), and stored in the mempool. When the transaction is serialized to protobuf and received by a peer, the deserialization produces `ValidResourceBounds::L1Gas(X)`, which (a) causes the `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion to fail with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, dropping the transaction from P2P propagation, and (b) causes any syncing node that recomputes the hash to derive H2 ≠ H1, breaking block-hash canonicalization.

---

### Finding Description

**Root cause — hash preimage length is variant-dependent:**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage conditionally:

```
L1Gas(X)                                  → [tip, l1_packed, l2_packed(0)]          (2 resource felts)
AllResources { l1_gas:X, l2_gas:0, l1_data_gas:0 } → [tip, l1_packed, l2_packed(0), l1_data_gas_packed(0)] (3 resource felts)
``` [1](#0-0) 

The two variants produce **different Poseidon digests** even when the numeric values of all three resource bounds are identical.

**Trigger — protobuf deserializer silently normalizes the variant:**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` converts any wire message with zero `l2_gas` and zero `l1_data_gas` to `ValidResourceBounds::L1Gas`, discarding the `AllResources` variant: [2](#0-1) 

**Serialization path preserves `AllResources`:**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` faithfully as three zero-valued fields: [3](#0-2) 

The wire bytes are identical to those produced by `L1Gas(X)`, so the deserializer cannot distinguish them and always produces `L1Gas`.

**Admission path always uses `AllResources`:**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` (never `ValidResourceBounds`). Its `InvokeTransactionV3Trait::resource_bounds()` implementation always wraps the value in `ValidResourceBounds::AllResources(...)`: [4](#0-3) 

So the hash computed at admission time always uses the 3-element preimage.

**P2P propagation failure:**

The mempool P2P path converts `RpcInvokeTransactionV3 → InvokeTransactionV3 → protobuf::InvokeV3WithProof` on the sender side, then `protobuf::InvokeV3WithProof → InvokeTransactionV3 → RpcInvokeTransactionV3` on the receiver side. The final step: [5](#0-4) 

requires `AllResources`. After protobuf round-trip the variant is `L1Gas`, so the conversion returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`: [6](#0-5) 

The receiving node silently drops the transaction.

**Block-hash inconsistency:**

The block sync path deserializes `InvokeTransactionV3` directly (not via `RpcInvokeTransactionV3`), so it does not error. However, `InvokeTransactionV3::calculate_transaction_hash` calls `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with the deserialized `L1Gas(X)` variant, producing H2 ≠ H1: [7](#0-6) 

Any node that validates the stored transaction hash against the recomputed hash will observe a mismatch.

---

### Impact Explanation

**High — P2P admission rejects valid transactions before sequencing.** A transaction accepted by the originating gateway is silently dropped by every peer that receives it via the mempool P2P propagation path (`MempoolTransaction` protobuf). The transaction never reaches other mempools, so if the originating node is not the current proposer, the transaction may never be sequenced.

**Critical — Wrong transaction hash / block-hash value for accepted input.** The hash H1 committed at admission time (3-element preimage, `AllResources`) differs from the hash H2 any syncing node computes from the stored `InvokeTransactionV3` (2-element preimage, `L1Gas`). This breaks the canonicalization invariant: the same transaction produces two different hashes depending on which representation is used, causing `validate_transaction_hash` to return `false` on syncing nodes and potentially invalidating block-hash commitments that include H1.

---

### Likelihood Explanation

The trigger condition — `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` with X > 0 — is reachable by any unprivileged user submitting a V3 invoke transaction through the standard JSON-RPC gateway. `RpcInvokeTransactionV3.resource_bounds` is typed as `AllResourceBounds` (not `ValidResourceBounds`), so no normalization occurs at the RPC layer. The gateway performs no check that `l2_gas` or `l1_data_gas` must be non-zero. The condition is therefore trivially constructible.

---

### Recommendation

**Option A — Normalize at admission.** In `convert_rpc_tx_to_internal` (or at the `RpcInvokeTransactionV3 → InternalRpcInvokeTransactionV3` conversion boundary), reject or normalize any `AllResourceBounds` with zero `l2_gas` and zero `l1_data_gas` to the `L1Gas` variant before computing the hash. This ensures the hash preimage is always canonical regardless of how the transaction was submitted.

**Option B — Normalize at deserialization.** Remove the `L1Gas` / `AllResources` branching from `get_tip_resource_bounds_hash` and always hash all three resource felts (including a zero `l1_data_gas` felt for old-style transactions). This makes the hash function independent of the Rust enum variant and eliminates the divergence.

**Option C — Normalize in the protobuf deserializer.** Change `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` to always produce `AllResources` (never `L1Gas`) for transactions received in the mempool/consensus context, matching the invariant that `InternalRpcInvokeTransactionV3` always uses `AllResources`.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, GasAmount, GasPrice, ValidResourceBounds, Tip,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;

fn main() {
    let l1_gas = ResourceBounds {
        max_amount: GasAmount(1000),
        max_price_per_unit: GasPrice(1),
    };
    let tip = Tip(0);

    // Variant 1: AllResources with zero l2_gas and l1_data_gas
    let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas,
        l2_gas: ResourceBounds::default(),    // zero
        l1_data_gas: ResourceBounds::default(), // zero
    });

    // Variant 2: L1Gas with the same l1_gas bounds
    let l1_gas_only = ValidResourceBounds::L1Gas(l1_gas);

    let hash_all = get_tip_resource_bounds_hash(&all_resources, &tip).unwrap();
    let hash_l1  = get_tip_resource_bounds_hash(&l1_gas_only,  &tip).unwrap();

    // These are NOT equal — different preimage lengths
    assert_ne!(hash_all, hash_l1,
        "AllResources hashes 3 felts; L1Gas hashes 2 felts — same numeric values, different digests");

    // Now simulate the protobuf round-trip:
    // AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
    //   → protobuf::ResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
    //   → ValidResourceBounds::L1Gas(X)   (because l1_data_gas.is_zero() && l2_gas.is_zero())
    //
    // The gateway computed hash_all (H1) at admission.
    // Any syncing node recomputes hash_l1 (H2).
    // H1 != H2 → validate_transaction_hash returns false → block rejected.
}

    transaction: &T,
    chain_id: &ChainId,

        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),

    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);

        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {

            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {

        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
