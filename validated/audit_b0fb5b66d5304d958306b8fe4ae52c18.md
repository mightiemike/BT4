### Title
`alignVestingDelegationTracking` Inflates `DelegatedVesting` to `OriginalVesting`, Zeroing `LockedCoins` and Making Permanently Locked Bank Coins Spendable — (`x/tieredrewards/keeper/force_exit.go`)

---

### Summary

`alignVestingDelegationTracking` calls `vacc.TrackDelegation(blockTime, deficit, deficit)` with a synthetic `balance = deficit` rather than the owner's actual bank balance. For a `PermanentLockedAccount` whose locked coins were never delegated (LockTier-origin position funded from spendable coins), this sets `DelegatedVesting = OriginalVesting`, which drives `LockedCoins = OriginalVesting − DV = 0`. The owner's bank balance — which still contains the original permanently locked coins — becomes fully spendable.

The question's specific sub-claim that `DV` can *exceed* `OriginalVesting` is **false**: the SDK's `BaseVestingAccount.TrackDelegation` caps `DV` at `OriginalVesting` via `x = min(OriginalVesting − DV, deficit)`. However, the consequence of `DV` *saturating* at `OriginalVesting` is the real vulnerability: `LockedCoins` drops to zero, unlocking the permanently locked bank balance.

---

### Finding Description

**Entry path:** v8 upgrade handler → `v2.Migrate` → `exitVestedAccountsPositions` → `ForceFullExitWithDelegation` → `alignVestingDelegationTracking`. [1](#0-0) 

`exitVestedAccountsPositions` walks every position whose owner is a `VestingAccount` and calls `ForceFullExitWithDelegation` on each. [2](#0-1) 

`ForceFullExitWithDelegation` transfers the position's delegation back to the owner (bypassing the bank-side `TrackDelegation` hook, so `DV`/`DF` are not updated), then calls `alignVestingDelegationTracking` to repair the gap. [3](#0-2) 

`alignVestingDelegationTracking` computes `deficit = actualDelegated − (DV + DF)` and calls:

```go
vacc.TrackDelegation(sdkCtx.BlockTime(), coins, coins)  // coins = deficit
```

The comment explicitly acknowledges that `balance = deficit` (not the real bank balance) is passed, claiming the bank balance is "irrelevant". This is the root cause.

**SDK `TrackDelegation` for `PermanentLockedAccount`:**

```
vestingAmt = OriginalVesting   (never vests)
x          = min(OriginalVesting − DV, deficit)
DV        += x                 // capped at OriginalVesting
DF        += deficit − x
```

`DV` cannot exceed `OriginalVesting` — the question's overflow claim is wrong. But when `deficit ≥ OriginalVesting − DV` (e.g., `DV = 0`, `deficit = OriginalVesting`), `DV` saturates at `OriginalVesting`, and:

```
LockedCoins = OriginalVesting − DV = 0
SpendableCoins = bank_balance − 0 = bank_balance
```

The permanently locked coins in the bank are now fully spendable.

**Concrete pre-upgrade state (confirmed by the integration test):** [4](#0-3) 

On v7.2.0, a `PermanentLockedAccount` with `OriginalVesting = X` could:
1. Receive `X` additional spendable coins (transfer/rewards).
2. Call `MsgLockTier` with `X` spendable coins → bank balance drops to `X` (locked coins only); `DV = 0`.

After v8 migration:
- `actualDelegated = X`, `tracked = 0`, `deficit = X`
- `TrackDelegation(t, X, X)` → `DV = X = OriginalVesting`
- `LockedCoins = 0`, `SpendableCoins = X`

The owner can now spend the `X` permanently locked coins.

**The existing test `TestForceFullExitWithDelegation_VestingOwner_LockOrigin` sets up exactly this scenario** (`balance = 2*lockedAmount`, `OriginalVesting = lockedAmount`, LockTier with `lockedAmount`) but only asserts `DV + DF = Σ delegations` — it never asserts `SpendableCoins == 0`, so the invariant break is not caught. [5](#0-4) 

---

### Impact Explanation

A `PermanentLockedAccount` owner who held a LockTier-origin position funded from spendable coins before the v8 upgrade can, after the migration, spend their permanently locked principal. The amount unlocked equals `min(OriginalVesting, LockTier_position_amount)`. This is a direct, irreversible economic loss: coins that were contractually locked forever become freely transferable.

---

### Likelihood Explanation

The integration test `_create_vesting_acc_owned_positions` explicitly creates this exact pre-upgrade state on v7.2.0 as part of the upgrade test suite, confirming the scenario is realistic and expected to exist on mainnet. Any vesting account holder who used `MsgLockTier` with spendable coins before v8 is affected. The migration runs automatically at upgrade height with no user action required.

---

### Recommendation

In `alignVestingDelegationTracking`, pass the owner's **actual bank balance** as the `balance` argument to `TrackDelegation`, not `deficit`:

```go
balance := k.bankKeeper.GetBalance(sdkCtx, ownerAddr, bondDenom)
balCoins := sdk.NewCoins(balance)
vacc.TrackDelegation(sdkCtx.BlockTime(), balCoins, coins)
```

This ensures the SDK correctly classifies the delegated coins as `DelegatedFree` (not `DelegatedVesting`) when the owner's bank balance does not contain enough locked coins to cover the delegation, preserving `LockedCoins > 0` for the permanently locked principal.

Additionally, add a post-condition assertion in the test:

```go
spendable := s.app.BankKeeper.SpendableCoins(s.ctx, owner)
s.Require().True(spendable.AmountOf(bondDenom).IsZero(),
    "permanently locked coins must not become spendable after force exit")
```

func exitVestedAccountsPositions(
	ctx context.Context,
	positions collections.Map[uint64, types.Position],
	ak AccountKeeper,
	pk PositionForceExiter,
) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)

	var toExit []uint64
	if err := positions.Walk(ctx, nil, func(posID uint64, pos types.Position) (bool, error) {
		ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
		if err != nil {
			return false, fmt.Errorf("parse owner address for position %d: %w", posID, err)
		}
		acc := ak.GetAccount(ctx, ownerAddr)
		if acc == nil {
			return false, fmt.Errorf("owner account not found for position %d: %s", posID, ownerAddr.String())
		}
		if _, ok := acc.(sdkvesting.VestingAccount); ok {
			toExit = append(toExit, posID)
		}
		return false, nil
	}); err != nil {
		return fmt.Errorf("walk positions: %w", err)
	}

	for _, posID := range toExit {
		sdkCtx.Logger().Info("v8 migration: force-exit vesting-owned position", "position_id", posID)
		if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
			return fmt.Errorf("force-exit position %d: %w", posID, err)
		}
	}

	sdkCtx.Logger().Info("v8 migration: vesting-owned positions exited", "count", len(toExit))
	return nil

    val_addr = get_node_validator_addr(cluster)

    tier_id = int(tiers[0]["id"])
    amount = int(tiers[0]["min_lock_amount"])

    lock_amount = amount

    # Create a permanent locked account with the commit amount

func (s *KeeperSuite) TestForceFullExitWithDelegation_VestingOwner_LockOrigin() {
	s.setupTier(1)
	vals, bondDenom := s.getStakingData()
	val := vals[0]
	valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	lockedAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	// OriginalVesting equals the locked amount; the account is funded with
	// 2*lockedAmount so that lockedAmount of it is spendable (balance −
	// LockedCoins). LockTier consumes the spendable portion via bank send,
	// without touching DelegatedVesting/DelegatedFree.
	owner := s.newVestingOwnerWithBalance(bondDenom, lockedAmount, lockedAmount.MulRaw(2))

	pos := s.createLockTierPositionV1(owner, valAddr, lockedAmount)

	// Pre-migration: DV/DF are zero (LockTier didn't touch them); owner has
	// no on-chain delegation.
	s.Require().True(s.delegatedVesting(owner).AmountOf(bondDenom).IsZero(),
		"DV must be zero for a LockTier-origin position before force exit")
	s.Require().True(s.delegatedFree(owner).AmountOf(bondDenom).IsZero(),
		"DF must be zero for a LockTier-origin position before force exit")
	s.Require().True(s.totalDelegated(owner).IsZero(),
		"owner has no on-chain delegation pre force-exit; position holds it")

	s.advanceForRewards(valAddr, bondDenom)

	s.Require().NoError(s.keeper.ForceFullExitWithDelegation(s.ctx, pos.Id))

	_, err := s.keeper.GetPosition(s.ctx, pos.Id)
	s.Require().Error(err)
	ownerDel := s.totalDelegated(owner)
	s.Require().Equal(lockedAmount, ownerDel,
		"owner must hold the returned delegation post force-exit")

	// Alignment must have topped up DV+DF by lockedAmount; otherwise a later
	// normal Undelegate would underflow vesting accounting.
	s.Require().Equal(lockedAmount, s.delegatedVesting(owner).AmountOf(bondDenom),
		"DV must saturate at OriginalVesting (= lockedAmount)")
	s.Require().True(s.delegatedFree(owner).AmountOf(bondDenom).IsZero(),
		"DF must be zero — the deficit fits entirely within OriginalVesting")
	s.Require().Equal(ownerDel, s.trackedTotal(owner, bondDenom),
		"alignment must satisfy DV + DF == Σ delegations for LockTier-origin positions")
}
