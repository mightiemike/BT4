Audit Report

## Title
SwapAllowlistExtension checks router address instead of originating EOA, allowing any caller to bypass per-pool swap gating via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from `MetricOmmPool.swap`, where `sender` is `msg.sender` of the pool call — the router, not the originating EOA. When a pool admin allowlists the router to enable router-based swaps, every EOA including explicitly blocked ones can swap freely through the router, defeating the extension's curation invariant entirely.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

When `MetricOmmSimpleRouter.exactOutputSingle` (or `exactInputSingle`, `exactInput`, `exactOutput`) calls `pool.swap(...)`, the router is `msg.sender` to the pool, so `sender = router` is forwarded to the extension. [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

Here `msg.sender` is the pool and `sender` is the router. The check resolves to `allowedSwapper[pool][router]`. The originating EOA is never consulted. If `allowedSwapper[pool][router] == true`, the check passes unconditionally for any EOA, including those for whom `allowedSwapper[pool][EOA] == false`.

The same flaw applies to all four router entry points: `exactInputSingle` (L71), `exactInput` (L103), `exactOutputSingle` (L135), and `exactOutput` (L165). [4](#0-3) [5](#0-4) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set of addresses provides no actual restriction when the router is allowlisted. Any EOA can call any router entry point and receive tokens from a pool intended to be restricted. This breaks the core swap-gating functionality of the extension: the exact wrong value is `allowedSwapper[pool][EOA]`, which is never read when the router intermediates. Unauthorized parties extract tokens from a curated pool, constituting broken core pool functionality with direct fund-flow impact.

## Likelihood Explanation
The router is the primary user-facing swap interface. Allowlisting the router is the expected operational pattern for any pool admin who wants to support router-based swaps while still curating direct callers. The bypass requires no special privileges, no malicious setup, no non-standard tokens — only a standard router call from any EOA. The condition is trivially reachable and repeatable.

## Recommendation
The extension must verify the true economic actor, not the intermediary. Concrete options:

1. **Pass the original initiator through the call chain**: Have the router write `msg.sender` into transient storage before calling `pool.swap`; expose a view on the router that the extension reads to identify the true swapper.
2. **Require direct pool interaction for allowlisted pools**: Document and enforce that pools using `SwapAllowlistExtension` must not allowlist the router; instead, allowlist individual EOAs who call the pool directly.
3. **Extension-level initiator recovery via `extensionData`**: Allow the extension to read an authenticated initiator from `extensionData` set by the router, with careful replay protection.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, EOA blocked
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// blockedEOA is NOT in allowedSwapper[pool][blockedEOA]

// Attack: blocked EOA calls exactOutputSingle through router
vm.prank(blockedEOA);
uint256 amountIn = router.exactOutputSingle(
    IMetricOmmSimpleRouter.ExactOutputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        tokenOut: address(token1),
        zeroForOne: true,
        amountOut: 1000,
        amountInMaximum: type(uint128).max,
        recipient: blockedEOA,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// Swap succeeds; blockedEOA receives exactly 1000 token1
// allowedSwapper[pool][blockedEOA] == false, but check was against router
assertEq(token1.balanceOf(blockedEOA), 1000);
```
