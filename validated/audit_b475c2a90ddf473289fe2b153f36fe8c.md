### Title
NFT Owner Cannot Burn or Edit Their Own NFT Due to Dual-Role AND Logic — (`File: x/nft/keeper/keeper.go`)

### Summary
`BurnNFT` and `EditNFT` in the NFT keeper require the caller to be **both** the current NFT owner **and** the denom creator simultaneously. Once an NFT is transferred away from the denom creator, neither the new owner nor the original creator can burn or edit it. The correct logic should allow **either** the NFT owner **or** the denom creator to perform these operations.

### Finding Description
In `x/nft/keeper/keeper.go`, both `BurnNFT` and `EditNFT` apply two sequential authorization checks:

```go
// EditNFT (lines 93–101)
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
if err != nil {
    return err
}
_, err = k.IsDenomCreator(ctx, denomID, owner)
if err != nil {
    return err
}
```

```go
// BurnNFT (lines 146–154)
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
if err != nil {
    return err
}
_, err = k.IsDenomCreator(ctx, denomID, owner)
if err != nil {
    return err
}
```

`IsOwner` returns an error if the caller is not the current NFT owner. `IsDenomCreator` returns an error if the caller is not the denom creator. Because both checks are applied sequentially with early-return on error, the effective gate is `IsOwner(caller) && IsDenomCreator(caller)` — the caller must hold **both** roles simultaneously.

This is the direct analog of the `onlyAdminOrOperator()` bug: the intended semantics are `||` (either role suffices), but the implementation enforces `&&` (both roles required).

By contrast, `MintNFT` only checks `IsDenomCreator`, and `TransferNFT` only checks `IsOwner`, confirming that the module treats these as independent, sufficient authorities for their respective operations. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation
Once an NFT is transferred from the denom creator to any other address, the NFT becomes permanently unburnable and uneditable by anyone:

- The **new owner** fails `IsDenomCreator` — they are not the collection creator.
- The **denom creator** fails `IsOwner` — they no longer hold the token.
- No third party can satisfy both checks either.

For `BurnNFT`: NFT owners lose the ability to destroy their own assets. NFTs accumulate in the system with no recourse for the holder. This corrupts the NFT ownership invariant — owning an asset should grant the right to destroy it.

For `EditNFT`: NFT metadata (name, URI, data) becomes permanently frozen once the token leaves the creator's wallet, even if the owner legitimately needs to update it. [5](#0-4) [6](#0-5) 

### Likelihood Explanation
The trigger is a standard `MsgBurnNFT` or `MsgEditNFT` transaction. Any NFT that has been transferred from its denom creator to another address is affected. This is a normal, expected usage pattern — minting NFTs to recipients is the primary use case of the module (`MintNFT` explicitly accepts a separate `recipient` parameter). Every such minted-and-transferred NFT is permanently locked against burn/edit by its owner. [7](#0-6) 

### Recommendation
Replace the sequential AND-logic with OR-logic in both `EditNFT` and `BurnNFT`. The caller should be authorized if they are **either** the current NFT owner **or** the denom creator:

```go
// Proposed fix for BurnNFT / EditNFT
_, ownerErr := k.IsOwner(ctx, denomID, tokenID, owner)
_, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
if ownerErr != nil && creatorErr != nil {
    return sdkerrors.Wrapf(types.ErrUnauthorized,
        "%s is neither the owner nor the creator of %s/%s", owner, denomID, tokenID)
}
``` [1](#0-0) 

### Proof of Concept

1. Alice calls `MsgIssueDenom` → Alice is the denom creator of `denom-A`.
2. Alice calls `MsgMintNFT` with `recipient = Bob` → Bob is the owner of `nft-1` in `denom-A`.
3. Bob calls `MsgBurnNFT(sender=Bob, denomId="denom-A", id="nft-1")`.
4. Keeper calls `IsOwner(ctx, "denom-A", "nft-1", Bob)` → **passes** (Bob is the owner).
5. Keeper calls `IsDenomCreator(ctx, "denom-A", Bob)` → **fails** (Alice is the creator, not Bob).
6. `BurnNFT` returns `ErrUnauthorized`. Bob cannot burn his own NFT.
7. Alice calls `MsgBurnNFT(sender=Alice, denomId="denom-A", id="nft-1")`.
8. Keeper calls `IsOwner(ctx, "denom-A", "nft-1", Alice)` → **fails** (Bob is the owner, not Alice).
9. Alice cannot burn it either. The NFT is permanently locked. [2](#0-1) [3](#0-2) [4](#0-3)

// EditNFT updates an already existing NFT
func (k Keeper) EditNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, owner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	if types.Modified(tokenNm) {
		nft.Name = tokenNm
	}

	if types.Modified(tokenURI) {
		nft.URI = tokenURI
	}

	if types.Modified(tokenData) {
		nft.Data = tokenData
	}

	k.setNFT(ctx, denomID, nft)

	return nil
}

// IsOwner checks if the sender is the owner of the given NFT
// Return the NFT if true, an error otherwise
func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

// IsDenomCreator checks if address is the creator of Denom
// Return the Denom if true, an error otherwise
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
```

		return nil, err
	}

	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
