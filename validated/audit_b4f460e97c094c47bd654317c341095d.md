### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` Permanently Blocks Subaccount Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when a subaccount's `vQuoteBalance` remains negative after insurance coverage. If `state.openInterest == 0` at that moment — a reachable state — `MathSD21x18.div` reverts with `"DBZ"`, permanently blocking finalization and locking any remaining collateral in the subaccount.

---

### Finding Description

The division occurs at `PerpEngine.socializeSubaccount`, lines 164–168:

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← reverts if == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` explicitly enforces `require(y != 0, ERR_DIV_BY_ZERO)`: [2](#0-1) 

`state.openInterest` is computed by `_updateBalance`. When `balance.amount == 0`, both the pre-update subtraction and post-update subtraction operate on zero, leaving `state.openInterest` equal to the stored `states[productId].openInterest` unchanged: [3](#0-2) 

So if `states[productId].openInterest == 0` in storage, `getStateAndBalance` returns `state.openInterest == 0` for a subaccount with `balance.amount == 0`.

The protocol itself already guards against zero OI in `updateStates` (line 111), but no equivalent guard exists in `socializeSubaccount`: [4](#0-3) 

---

### Reachable State Construction

`_finalizeSubaccount` requires `balance.amount == 0` for every perp product before calling `socializeSubaccount`: [5](#0-4) 

The negative-PnL settlement loop only runs if `quoteBalance.amount > 0`: [6](#0-5) 

If the liquidatee has no USDC, this loop is skipped entirely, leaving `balance.vQuoteBalance < 0` intact when `socializeSubaccount` is called at line 386: [7](#0-6) 

**Concrete path to trigger:**

1. Subaccount A opens a long perp position in product P — it is the sole holder of open interest (`state.openInterest = A.amount`).
2. A's perp position is liquidated: `perpEngine.updateBalance(P, liquidatee, -amount, liquidationPayment)` zeroes A's `amount` and adds `liquidationPayment` to `vQuoteBalance`. If `liquidationPayment < |original_vQuoteBalance|`, A's `vQuoteBalance` remains negative. The liquidator now holds the position, so `state.openInterest` is still non-zero.
3. The liquidator (or any other holder) closes their position in product P. `state.openInterest` drops to 0.
4. A has no USDC (`quoteBalance.amount <= 0`).
5. Liquidator submits a finalization transaction (`txn.productId = type(uint32).max`).
6. `_finalizeSubaccount` → `socializeSubaccount` → `div(state.openInterest)` → **revert "DBZ"**.

This is also reachable without a prior liquidation: a subaccount that was the sole OI holder closes its position at a realized loss (amount → 0, vQuoteBalance < 0, OI → 0), then becomes unhealthy.

---

### Impact Explanation

Every subsequent finalization attempt reverts. The subaccount can never be finalized. Any remaining collateral (spot balances, quote balance) is permanently locked inside the subaccount. The negative `vQuoteBalance` is never socialized, leaving the protocol's accounting in a broken state. This satisfies the Critical scope criterion of **permanent lock of collateral and protocol-controlled assets**.

---

### Likelihood Explanation

The state is reachable through normal trading and the standard liquidation flow. No admin access, sequencer compromise, or special privileges are required. A liquidator can deliberately trigger it by closing their own position before submitting the finalization transaction, making this a reliable griefing vector. It can also occur naturally in low-liquidity markets where a single subaccount is the sole OI holder.

---

### Recommendation

Add a zero-OI guard before the division in `socializeSubaccount`. If `state.openInterest == 0` and `balance.vQuoteBalance < 0`, there are no other participants to socialize against; the loss should be absorbed entirely by the insurance fund (already attempted above) or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; write off residual loss
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```
