### Title
Malicious IBC Counterparty Can Inject Double-Prefixed ClassID to Mint Unredeemable Spurious NFT Vouchers — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` determines the transfer direction using `IsAwayFromOrigin`, which checks only whether `data.ClassId` starts with the **source** port/channel prefix. There is no guard that rejects or detects a classID that already embeds the **destination** port/channel prefix. A malicious IBC counterparty can craft a packet whose `ClassId` already contains the destination prefix, causing the receiver to prepend it a second time, store a fabricated `ClassTrace`, and mint NFT vouchers that can never be redeemed.

---

### Finding Description

**Entrypoint:** `OnRecvPacket` → `OnRecvPacket` (keeper) → `processReceivedPacket` [1](#0-0) 

The direction decision is made at line 148:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
```

`IsAwayFromOrigin` is:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [2](#0-1) 

It only checks whether the classID starts with `sourcePort/sourceChannel/`. It does **not** check whether the classID already starts with `destPort/destChannel/`.

When `isAwayFromOrigin == true`, the code unconditionally prepends the destination prefix:

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
prefixedClassID := classPrefix + data.ClassId   // double-prefix if data.ClassId already had it
classTrace := types.ParseClassTrace(prefixedClassID)
if !k.HasClassTrace(ctx, classTrace.Hash()) {
    k.SetClassTrace(ctx, classTrace)
}
voucherClassID := classTrace.IBCClassID()
// ... MintNFT under voucherClassID
``` [3](#0-2) 

**Concrete attack trace:**

| Variable | Value |
|---|---|
| Dest port/channel (victim) | `nft` / `channel-0` |
| Source port/channel (attacker) | `nft` / `channel-1` |
| Malicious `data.ClassId` | `nft/channel-0/nft/channel-0/nftClass` |

1. `IsAwayFromOrigin("nft", "channel-1", "nft/channel-0/nft/channel-0/nftClass")` → `"nft/channel-0/..."` does **not** start with `"nft/channel-1/"` → returns `true`.
2. `classPrefix = "nft/channel-0/"`.
3. `prefixedClassID = "nft/channel-0/nft/channel-0/nft/channel-0/nftClass"` — triple-prefixed.
4. `ClassTrace{Path: "nft/channel-0/nft/channel-0/nft/channel-0", BaseClassId: "nftClass"}` is stored.
5. `voucherClassID = ibc/{sha256("nft/channel-0/nft/channel-0/nft/channel-0/nftClass")}`.
6. NFTs are minted to the receiver under this spurious classID.

**No guard prevents this.** `ValidateBasic()` on `NonFungibleTokenPacketData` only checks that `ClassId` is non-blank, that `TokenIds`/`TokenUris` lengths match, and that addresses are valid — it performs no structural validation of the classID path. [4](#0-3) 

**Why the minted NFTs are permanently unredeemable:**

If the receiver attempts to send the voucher back, `createOutgoingPacket` resolves the full class path to `nft/channel-0/nft/channel-0/nft/channel-0/nftClass`. `IsAwayFromOrigin("nft", "channel-0", ...)` returns `false` (starts with `nft/channel-0/`), so the voucher is **burned** and a packet is sent to the counterparty with classID `nft/channel-0/nft/channel-0/nftClass`. The counterparty never escrowed any NFT under that classID, so the unescrow fails. The receiver's voucher is destroyed with no recovery. [5](#0-4) 

---

### Impact Explanation

- A spurious `ClassTrace` is written to the KV store that does not correspond to any legitimate hop history, violating the invariant that every stored trace accurately represents an NFT's provenance.
- NFT vouchers are minted to an attacker-chosen receiver under a fabricated `voucherClassID`.
- Those vouchers are permanently unredeemable: attempting to return them burns the voucher on the victim chain but produces an unescrow failure on the counterparty, resulting in net loss of the NFT for any user who received or traded the voucher.
- The malicious counterparty can repeat this for arbitrary classIDs and token IDs, polluting the class-trace store and minting unbacked vouchers at will.

---

### Likelihood Explanation

The attacker must control a chain that has an established IBC channel with the victim chain. This is a realistic scenario: IBC channels are permissionlessly relayed, and a chain with even a small validator set can craft arbitrary packet data. The IBC core layer authenticates the packet commitment but does not inspect application-level classID content, so no additional privilege is required beyond operating the counterparty chain.

---

### Recommendation

In `processReceivedPacket`, before prepending the destination prefix, add an explicit check that `data.ClassId` does **not** already start with `destPort/destChannel/`. If it does, the packet should either be rejected (error acknowledgement) or routed to the unescrow path, consistent with the ICS-721 specification's intent that the destination prefix signals "returning to origin."

```go
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, destPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classID already contains destination prefix %s", destPrefix)
}
```

This guard should be applied before the `isAwayFromOrigin` branch is entered.

### Proof of Concept

```go
func TestDoublePrefix_SpuriousClassTrace(t *testing.T) {
    // Setup: victim chain dest port="nft", dest channel="channel-0"
    //        malicious source port="nft", source channel="channel-1"
    destPort, destChannel := "nft", "channel-0"
    srcPort, srcChannel   := "nft", "channel-1"

    // Attacker pre-prefixes the classID with the destination prefix
    maliciousClassID := fmt.Sprintf("%s/%s/nft/channel-0/nftClass", destPort, destChannel)
    // = "nft/channel-0/nft/channel-0/nftClass"

    // Simulate processReceivedPacket logic
    isAway := types.IsAwayFromOrigin(srcPort, srcChannel, maliciousClassID)
    require.True(t, isAway) // passes: does not start with "nft/channel-1/"

    classPrefix    := types.GetClassPrefix(destPort, destChannel)
    prefixedClassID := classPrefix + maliciousClassID
    // = "nft/channel-0/nft/channel-0/nft/channel-0/nftClass"

    classTrace := types.ParseClassTrace(prefixedClassID)
    // Path = "nft/channel-0/nft/channel-0/nft/channel-0", BaseClassId = "nftClass"

    // Assert: stored path is exactly one prefix longer than the incoming classID
    expectedPath := fmt.Sprintf("%s/%s/%s", destPort, destChannel, maliciousClassID[:len(maliciousClassID)-len("/nftClass")])
    require.Equal(t, expectedPath, classTrace.Path)
    // Confirms spurious triple-hop trace is stored for a single-hop packet
}

```go
// GetClassPrefix returns the receiving class prefix
func GetClassPrefix(portID, channelID string) string {
	return fmt.Sprintf("%s/%s/", portID, channelID)
}

// RemoveClassPrefix returns the unprefixed classID.
// After the receiving chain receives the packet,if isAwayFromOrigin=false, it means that nft is moving
// in the direction of the original chain, and the portID/channelID prefix of the sending chain
// in trace.path needs to be removed
func RemoveClassPrefix(portID, channelID, classID string) string {
	classPrefix := GetClassPrefix(portID, channelID)
	return classID[len(classPrefix):]
}

// IsAwayFromOrigin determine if non-fungible token is moving away from
// the origin chain (the chain issued by the native nft).
// Note that fullClassPath refers to the full path of the unencoded classID.
// The longer the fullClassPath, the farther it is from the origin chain
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
	prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
	return !strings.HasPrefix(fullClassPath, prefixClassID)
}

func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
