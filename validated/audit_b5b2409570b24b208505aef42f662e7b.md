### Title
Silent Bonus Forfeit with Permanent Checkpoint Advancement During Redelegation Slash When Pool Is Insufficient — (`x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, `processEventsAndClaimBonus` advances the position's accrual checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) **before** it checks the pool balance. The caller then persists the position with those advanced checkpoints via `setPositionWithState`, permanently destroying the user's accrued bonus without paying it.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` performs checkpoint advancement unconditionally before the pool balance check:

```
applyBonusAccrualCheckpoint(&pos.Position, blockTime)   // line 215 — advances LastBonusAccrual
pos.UpdateLastKnownBonded(bonded)                        // line 217
...
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // returns error, but pos already has advanced checkpoints
}
```

Inside the event loop (line 193), `pos.UpdateLastEventSeq(entry.Seq)` is also called before the pool check, so `LastEventSeq` is already advanced past all processed events when the error is returned. [1](#0-0) [2](#0-1) [3](#0-2) 

In `slashRedelegationPosition`, the `ErrInsufficientBonusPool` error is caught and swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // error swallowed — pos already has advanced checkpoints
    } else {
        return err
    }
}
// ...
return k.setPositionWithState(ctx, pos, ...)  // persists pos with advanced checkpoints
``` [4](#0-3) [5](#0-4) 

Because `processEventsAndClaimBonus` takes `&pos` (a pointer), all in-memory checkpoint mutations are visible to the caller even after the error return. `setPositionWithState` then writes those advanced checkpoints to the store. The next time the user calls `ClaimTierRewards`, `LastBonusAccrual` and `LastEventSeq` are already at the current block, so the previously accrued bonus segment is gone forever — no payment was made, but the accounting window was closed. [6](#0-5) 

---

### Impact Explanation

A user whose tier position is in an active redelegation at the time a validator slash fires, and whose accrued bonus exceeds the current pool balance, permanently loses all bonus rewards accrued up to that block. The loss is silent: no error is surfaced to the user, no event records the forfeited amount, and the position's checkpoints are advanced as if payment occurred. The corrupted state values are `pos.LastBonusAccrual`, `pos.LastEventSeq`, and `pos.LastKnownBonded`. [4](#0-3) [7](#0-6) 

---

### Likelihood Explanation

Two independent conditions must coincide: (1) the bonus pool is depleted or under-funded, and (2) a validator slash fires while a position is in a redelegation. Pool depletion is a realistic operational state (the pool is funded externally and can run dry). Validator slashing during redelegation is an uncommon but fully supported Cosmos SDK path (double-sign evidence or downtime). The combination is low-frequency but entirely reachable by an unprivileged user whose validator happens to be slashed at the wrong time. [8](#0-7) [9](#0-8) 

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the successful `SendCoinsFromModuleToAccount` call, so checkpoints are only advanced when payment actually succeeds. In `slashRedelegationPosition`, if the pool is insufficient, either (a) skip the checkpoint advance entirely (leaving the bonus claimable once the pool is replenished), or (b) record the forfeited amount in an on-chain debt entry so it can be paid retroactively. Option (a) is the minimal fix and mirrors the atomic-rollback behaviour already used for user-driven paths. [10](#0-9) [11](#0-10) 

---

### Proof of Concept

1. User creates a tier position and redelegates to a new validator (creating a redelegation entry tracked in `RedelegationMappings`).
2. The bonus pool is drained to zero (e.g., by other users claiming rewards or simply not being replenished).
3. The source validator is slashed (double-sign evidence submitted). The Cosmos SDK calls `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
4. `processEventsAndClaimBonus` computes a non-zero `totalBonus`, advances `LastBonusAccrual` to `blockTime` and `LastEventSeq` past all events, then hits `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
5. `slashRedelegationPosition` swallows the error and calls `setPositionWithState` with the already-mutated `pos`.
6. The user later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the now-advanced `LastBonusAccrual` and `LastEventSeq`, computing zero bonus for the already-consumed window. The accrued bonus is permanently lost. [12](#0-11) [13](#0-12)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
