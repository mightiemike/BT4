### Title
Interest Rate State Not Updated Before Config Change in `addOrUpdateProduct` - (File: `core/contracts/SpotEngine.sol`)

### Summary
`SpotEngine.addOrUpdateProduct` overwrites the interest rate configuration for an existing product without first calling `_updateState` to apply the old rates for the elapsed period. The next `SpotTick`-triggered `updateStates` call will then apply the new rate parameters retroactively across the entire elapsed interval, corrupting `cumulativeBorrowsMultiplierX18` and `cumulativeDepositsMultiplierX18` for all depositors and borrowers of that product.

### Finding Description
`SpotEngine.addOrUpdateProduct` is the owner-callable function used to update a product's `Config`, which includes the four interest rate parameters: `interestFloorX18`, `interestSmallCapX18`, `interestLargeCapX18`, and `interestInflectionUtilX18`. [1](#0-0) 

When called for an **existing** product (`isNewProduct == false`), the function directly overwrites `configs[productId] = config` at line 83 and returns — no call to `_updateState` is made. [2](#0-1) 

Interest accrual in Nado is lazy: it is only applied when the sequencer submits a `SpotTick` transaction, which calls `spotEngine.updateStates(dt)`. [3](#0-2) 

`updateStates` iterates over all products and calls `_updateState(productId, state, dt)`, where `dt` is the full elapsed time since the previous `SpotTick`. [4](#0-3) 

Inside `_updateState`, the interest rate is computed by reading `configs[productId]` — the **current** config at the time of the call. [5](#0-4) 

The computed `borrowRateMultiplierX18` is then applied to the full `dt` window and used to update `cumulativeBorrowsMultiplierX18` and `cumulativeDepositsMultiplierX18`. [6](#0-5) 

Because `addOrUpdateProduct` replaces the config before `_updateState` is called for the elapsed period, the new rate parameters are applied retroactively to the entire interval `[lastSpotTick, nextSpotTick]`, even though the old parameters were in effect for the portion `[lastSpotTick, configChangeTime]`.

### Impact Explanation
`cumulativeBorrowsMultiplierX18` and `cumulativeDepositsMultiplierX18` are the global accumulators from which every user's actual balance is derived: [7](#0-6) 

Corrupting these multipliers directly misstates the real balance of every depositor and borrower of the affected product:
- **Rate increase**: borrowers are overcharged interest for the pre-change window; depositors receive excess yield.
- **Rate decrease**: borrowers are undercharged; depositors receive less yield than earned.

The magnitude of the error scales with `dt` (the `SpotTick` interval) and the size of the rate change. Because the protocol's health checks, liquidation thresholds, and PnL settlement all depend on these balances, the corruption propagates into collateral accounting.

### Likelihood Explanation
The trigger is a legitimate admin operation — updating interest rate parameters for a live product. This is an expected lifecycle event (e.g., adjusting rates in response to market conditions). The `SpotTick` interval is the window of exposure; the longer the interval between ticks, the larger the retroactive misapplication.

### Recommendation
Before overwriting `configs[productId]`, call `_updateState` to apply the old rates up to the current time for existing products:

```solidity
function addOrUpdateProduct(...) public onlyOwner {
    bool isNewProduct = _addOrUpdateProduct(...);

    if (!isNewProduct && states[productId].totalDepositsNormalized != 0) {
        // Flush accrued interest under the old config before switching
        State memory state = states[productId];
        uint128 dt = uint128(getOracleTime()) - lastSpotTime; // or equivalent
        _updateState(productId, state, dt);
        _setState(productId, state);
    }

    configs[productId] = config;
    ...
}
```

Alternatively, require that a `SpotTick` has been submitted in the same block before any config update is accepted, ensuring `dt == 0` at the moment of the change.

### Proof of Concept
1. Product `P` has `interestFloorX18 = 1e16` (1% APR). Last `SpotTick` was 1 hour ago (`dt = 3600`).
2. Admin calls `addOrUpdateProduct(P, ..., config_new, ...)` where `config_new.interestFloorX18 = 1e17` (10% APR). `configs[P]` is immediately overwritten; no `_updateState` is called.
3. Sequencer submits the next `SpotTick`. `updateStates(3600)` is called. `_updateState` reads `configs[P]` — now the 10% config — and applies it to the full 3600-second window.
4. The 1-hour window that elapsed under the 1% config is charged at 10%, a 10× overcharge on borrowers for that period. [1](#0-0) [8](#0-7)

    function _updateState(
        uint32 productId,
        State memory state,
        uint128 dt
    ) internal {
        int128 borrowRateMultiplierX18;
        int128 totalDeposits = state.totalDepositsNormalized.mul(
            state.cumulativeDepositsMultiplierX18
        );
        int128 totalBorrows = state.totalBorrowsNormalized.mul(
            state.cumulativeBorrowsMultiplierX18
        );
        int128 utilizationRatioX18 = totalBorrows.div(totalDeposits);
        int128 minDepositRateX18;
        {
            Config memory config = configs[productId];

            // annualized borrower rate
            int128 borrowerRateX18 = config.interestFloorX18;
            if (utilizationRatioX18 == 0) {
                // setting borrowerRateX18 to 0 here has the property that
                // adding a product at the beginning of time and not using it until time T
                // results in the same state as adding the product at time T
                borrowerRateX18 = 0;
            } else if (utilizationRatioX18 < config.interestInflectionUtilX18) {
                borrowerRateX18 += config
                    .interestSmallCapX18
                    .mul(utilizationRatioX18)
                    .div(config.interestInflectionUtilX18);
            } else {
                borrowerRateX18 +=
                    config.interestSmallCapX18 +
                    config.interestLargeCapX18.mul(
                        (
                            (utilizationRatioX18 -
                                config.interestInflectionUtilX18).div(
                                    ONE - config.interestInflectionUtilX18
                                )
                        )
                    );
            }

            // convert to per second
            borrowerRateX18 = borrowerRateX18.div(
                MathSD21x18.fromInt(31536000)
            );
            borrowRateMultiplierX18 = (ONE + borrowerRateX18).pow(int128(dt));
            minDepositRateX18 = config.minDepositRateX18;
        }
