### Title
Unbounded-Age Fallback to `previous_proposal_init` L1 Gas Prices Bypasses `max_time_gap_seconds` Staleness Guard — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to `PreviousProposalInitInfo` L1 gas prices when the live provider or oracle fails. The fallback uses prices from the last committed block with no check on how much wall-clock time has elapsed since that block. The `PreviousProposalInitInfo.timestamp` field is present but never compared against the current timestamp in the fallback path. This means the `max_time_gap_seconds` staleness guard in `L1GasPriceProvider::get_price_info` — which fires `StaleL1GasPricesError` precisely to signal that prices are too old — is silently bypassed by the fallback, which can then serve prices that are arbitrarily older than the guard's threshold.

---

### Finding Description

`L1GasPriceProvider::get_price_info` enforces a staleness check:

```rust
// Check if the prices are stale.
if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
    return Err(L1GasPriceProviderError::StaleL1GasPricesError { ... });
}
``` [1](#0-0) 

The default `max_time_gap_seconds` is 900 seconds (15 minutes). [2](#0-1) 

When `get_price_info` returns `StaleL1GasPricesError` (or the oracle fails), `get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to `previous_proposal_init`:

```rust
if let Some(prev_info) = previous_proposal_init {
    let prev_l1_gas_price_wei = prev_info.l1_prices_wei.clone();
    let prev_l1_gas_price = prev_info.l1_prices_fri.clone();
    let eth_to_fri_rate = calculate_eth_to_fri_rate(prev_info);
    match eth_to_fri_rate {
        Ok(eth_to_fri_rate) => {
            return (prev_l1_gas_price, prev_l1_gas_price_wei, eth_to_fri_rate);
        }
        ...
    }
}
``` [3](#0-2) 

`PreviousProposalInitInfo` carries a `timestamp` field (the L2 block timestamp of the last committed block): [4](#0-3) 

This `timestamp` is populated from the committed block header: [5](#0-4) 

**The fallback at lines 186–209 never computes `current_timestamp - prev_info.timestamp` and never rejects the fallback if that gap exceeds any threshold.** The `prev_info.timestamp` is only used elsewhere to enforce monotonicity of block timestamps (new block ≥ previous block), not to bound the age of the price data. [6](#0-5) 

The same fallback is invoked identically during proposal validation: [7](#0-6) 

Because both proposer and validator call the same `get_l1_prices_in_fri_and_wei` with the same `previous_proposal_init`, both sides independently compute the same stale fallback value, the `within_margin` check passes, and the block is accepted with arbitrarily old L1 gas prices. [8](#0-7) 

---

### Impact Explanation

After extended network downtime (e.g., consensus stall, L1 scraper outage lasting longer than `max_time_gap_seconds`), the first blocks produced use L1 gas prices from the last committed block before the outage. If L1 gas prices spiked during the downtime, the sequencer charges users the pre-outage (low) price while paying the post-outage (high) price for L1 data availability. This is a direct, unbounded economic loss to the protocol. The magnitude scales with the duration of the outage and the size of the L1 gas price movement.

This matches the allowed impact: **"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

The trigger requires two simultaneous conditions: (1) the L1 gas price provider's samples become stale (scraper down, L1 RPC unavailable, or chain halted for > 900 s), and (2) L1 gas prices move significantly during that window. Both conditions are realistic during network incidents. The Starknet network has experienced downtime historically. The fallback is reached on every block proposal during the outage, not just the first one, so the window of exposure is the entire duration of the outage.

---

### Recommendation

In `get_l1_prices_in_fri_and_wei_and_conversion_rate`, before returning `previous_proposal_init` prices, compare `timestamp - prev_info.timestamp` against a configurable maximum age (e.g., reuse `max_time_gap_seconds` from `L1GasPriceProviderConfig`, or add a dedicated `max_fallback_age_seconds`). If the gap exceeds the threshold, skip the `previous_proposal_init` fallback and proceed to the minimum-config fallback (path 3), which uses operator-verified safe floor values. The `PreviousProposalInitInfo.timestamp` field is already present and requires no schema change.

```rust
if let Some(prev_info) = previous_proposal_init {
    // NEW: reject fallback if previous block prices are too old.
    if timestamp.saturating_sub(prev_info.timestamp) <= max_fallback_age_seconds {
        // existing fallback logic ...
    }
}
```

### Proof of Concept

1. Chain is running; last committed block at wall-clock T=0 carries L1 gas price = 10 gwei. `previous_proposal_init.timestamp = T=0`, `previous_proposal_init.l1_prices_wei.l1_gas_price = 10 gwei`.
2. L1 gas price scraper stops (L1 RPC outage). `L1GasPriceProvider`'s ring buffer stops receiving new samples.
3. At T=1000 s (> `max_time_gap_seconds=900`), `get_price_info(BlockTimestamp(1000))` returns `StaleL1GasPricesError`.
4. `get_l1_prices_in_fri_and_wei_and_conversion_rate` enters the fallback branch at line 186. It reads `prev_info.l1_prices_wei.l1_gas_price = 10 gwei` and returns it. No check on `1000 - 0 = 1000 > 900`.
5. Meanwhile, L1 gas price has spiked to 1000 gwei.
6. Both proposer and validator independently compute the same 10 gwei fallback; `within_margin` passes; block is committed with `l1_gas_price = 10 gwei`.
7. The sequencer pays 1000 gwei per unit of L1 data but charges users 10 gwei — a 100× undercharge persisting for every block until the scraper recovers. [9](#0-8) [1](#0-0) [10](#0-9)

    previous_proposal_init: Option<&PreviousProposalInitInfo>,
    gas_price_params: &GasPriceParams,
) -> (L1PricesInFri, L1PricesInWei, u128) {
    // One of these paths should fill the return values:
    // 1. Both L1 gas price and eth/strk rate are Ok, use those.
    // 2. Otherwise, use previous block info.
    // 3. If that isn't available either, use min gas prices and default eth/strk rate.

    // Get the eth to fri rate from the oracle, and the L1 gas price (in wei) from the provider.
    let (eth_to_fri_rate, price_info) = tokio::join!(

        l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp))
    );
    if price_info.is_err() {
        warn!("Failed to get l1 gas price from provider: {:?}", price_info);
        CONSENSUS_L1_GAS_PRICE_PROVIDER_ERROR.increment(1);
    }
    if eth_to_fri_rate.is_err() {
        warn!("Failed to get eth to fri rate from oracle: {:?}", eth_to_fri_rate);
    }
    if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
        // Both L1 prices and rate are Ok, so we can use them.
        info!(
            "raw eth_to_fri_rate (from oracle): {eth_to_fri_rate}, raw l1 gas price wei (from \
             provider): {price_info:?}"
        );
        apply_fee_transformations(&mut price_info, gas_price_params);
        let prices_in_wei = L1PricesInWei {
            l1_gas_price: price_info.base_fee_per_gas,
            l1_data_gas_price: price_info.blob_fee,
        };
        // Apply the eth/strk rate to get prices in fri.
        let l1_gas_prices_fri_result =
            L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
        // If conversion fails, leave return_value=None to try backup methods.
        if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
            return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
        } else {
            warn!(
                "Failed to convert L1 gas prices to FRI: {:?}",
                l1_gas_prices_fri_result.clone().err()
            );
        }
    }

    // One or both (oracle/provider) have failed to fetch, or failure in conversion, so we need to
    // try to use the previous block info.
    if let Some(prev_info) = previous_proposal_init {
        let prev_l1_gas_price_wei = prev_info.l1_prices_wei.clone();
        let prev_l1_gas_price = prev_info.l1_prices_fri.clone();
        // This calculation can fail if gas price is too high, or zero, or if the prices cause the
        // rate to be zero.
        let eth_to_fri_rate = calculate_eth_to_fri_rate(prev_info);
        match eth_to_fri_rate {
            Ok(eth_to_fri_rate) => {
                info!(
                    "Using previous block info: wei prices: {:?}, fri prices: {:?}, eth to fri \
                     rate: {:?}",
                    prev_l1_gas_price_wei, prev_l1_gas_price, eth_to_fri_rate
                );
                return (prev_l1_gas_price, prev_l1_gas_price_wei, eth_to_fri_rate);
            }
            Err(error) => {
                warn!(
                    "Error calculating eth to fri rate from previous block info: {:?}: {:?}",
                    prev_info, error
                );
                // Do not use previous block info. Prefer the default values instead.
            }
        }
    }

    let default_l1_gas_prices_wei = L1PricesInWei {
        l1_gas_price: gas_price_params.min_l1_gas_price_wei,
        l1_data_gas_price: gas_price_params.min_l1_data_gas_price_wei,
    };
    let default_l1_gas_prices_fri =
        L1PricesInFri::convert_from_wei(&default_l1_gas_prices_wei, DEFAULT_ETH_TO_FRI_RATE)
            .expect("Default values should be convertible between wei and fri.");
    info!(

        default_l1_gas_prices_fri, default_l1_gas_prices_wei, DEFAULT_ETH_TO_FRI_RATE
    );

//! When combining these in `apollo_consensus_orchestrator`, the fallback chain is:
//!
//! 1. **Live data** — Both sources succeed → transform prices (tip, clamp, multiplier) and convert
//!    to FRI using the live rate. Normal path.
//! 2. **Previous block** — Either source fails → reuse WEI and FRI prices from the last accepted
//!    proposal, back-calculating the ETH→STRK rate from them for consistency.
//! 3. **Minimal config** — No previous block available (e.g. startup) → use configured
//!    `min_l1_gas_price_wei` / `min_l1_data_gas_price_wei` and `DEFAULT_ETH_TO_FRI_RATE`. The
//!    default rate is 10²¹ FRI/ETH (≈ 1 ETH = 1000 STRK).
//!
//! **Why not halt on failure?** A transient oracle or provider outage should not stall block
//! production. Stale prices self-correct once the feed recovers; a halted chain requires manual
//! intervention and breaks liveness for all users.
//!
//! **Why keep fallback prices low?** When the true L1 cost is unknown, undercharging slightly is
//! better than overcharging. Inflated prices cause transactions to fail and make the network
//! unusable precisely when it is already degraded. The configured minimums and the default
//! ETH→STRK rate are safe floors the operator has verified are always economically viable.
