Audit Report

## Title
`SwapAllowlistExtension` checks router address as swapper, allowing any user to bypass per-pool swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of the pool's `swap()` call. When any user routes through `MetricOmmSimpleRouter`, `sender` resolves to the router's address. Because the router must be allowlisted for any router-mediated swap to succeed, every user—including non-allowlisted ones—can bypass the curated pool's access control by routing through the router.

## Finding Description

`MetricOmmPool.swap()` passes its own `msg.sender` as the `sender` argument to `_beforeSwap()`: [1](#0-0) 

`_beforeSwap` in `ExtensionCalling.sol` forwards that value as the first positional argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` receives that value as `sender` and checks it against the per-pool allowlist: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly with no mechanism to forward the original caller's identity: [4](#0-3) 

Because the router is `msg.sender` of the pool call, the extension receives `sender = router_address`. The allowlist check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][original_user]`. The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput`, all of which call `pool.swap()` as `msg.sender`: [5](#0-4) 

This creates an irresolvable dilemma: not allowlisting the router makes the router completely unusable for the pool; allowlisting the router lets every user bypass the guard.

## Impact Explanation

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., KYC-verified counterparties). The allowlist provides zero protection against router-mediated swaps: any non-allowlisted user who routes through `MetricOmmSimpleRouter` trades freely on the curated pool. This is a direct admin-boundary break—the pool admin's configured access-control invariant is violated, allowing unauthorized parties to trade against the pool's LPs.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary supported periphery path for swaps and is permissionlessly callable by any address. No special privileges, flash loans, or multi-step setup are required. Any user aware of the router can execute the bypass in a single transaction, and the condition (router allowlisted) is a prerequisite for legitimate use of the router with the pool.

## Recommendation

The extension must gate on the economically relevant actor—the original user—not the intermediary router. Concrete options:

1. **Trusted-router forwarding**: Add a `swapWithSender(address originalSender, ...)` entry point on the pool, callable only by factory-approved routers, that passes `originalSender` to extension hooks instead of `msg.sender`.
2. **Extension-data attestation**: Require the router to embed the original user's address in `extensionData`; the extension decodes it and verifies the caller is a trusted router before accepting the attested identity.
3. **Direct-only pools**: Document and enforce at the factory level that pools using `SwapAllowlistExtension` must not allowlist any router address.

## Proof of Concept

```
1. Admin deploys pool with SwapAllowlistExtension.
2. Admin calls setAllowedToSwap(pool, alice, true)       // alice is KYC'd
3. Admin calls setAllowedToSwap(pool, router, true)      // required for router-mediated swaps
4. bob (non-allowlisted) calls:
       router.exactInputSingle({pool: pool, tokenIn: T0, tokenOut: T1, ...})
5. Router calls pool.swap(recipient=bob, ...)
       → pool: msg.sender = router
       → _beforeSwap(sender=router, ...)
       → extension: allowedSwapper[pool][router] == true  ✓
6. Swap executes. Bob receives T1 despite never being allowlisted.
```

The check `allowedSwapper[pool][router]` passes because the router was allowlisted in step 3. The extension never observes `bob`'s address.
