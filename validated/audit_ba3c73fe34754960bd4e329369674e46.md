### Title
Empty Bonus Pool Permanently Blocks All Exit Paths for Position Owners — (`x/tieredrewards/keeper/msg_server.go`)

### Summary

When the `x/tieredrewards` bonus rewards pool is empty, every user-driven exit operation (`MsgTierUndelegate`, `MsgExitTierWithDelegation`) fails atomically because both unconditionally call `claimRewards` before executing the exit. There is no exit path that bypasses the bonus claim. A user whose exit commitment has elapsed has no way to recover their principal until governance replenishes the pool — an exact structural analog to the AuraLocker shutdown forcing a penalty on all users.

### Finding Description

Every exit operation in `x/tieredrewards` mandates reward settlement before the delegation is moved. In `TierUndelegate`:

```go
// x/tieredrewards/keeper/msg_server.go:166
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err
}
```

And in `ExitTierWithDelegation`:

```go
// x/tieredrewards/keeper/msg_server.go:532
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err
}
```

`claimRewards` calls `processEventsAndClaimBonus`, which calls `sufficientBonusPoolBalance`:

```go
// x/tieredrewards/keeper/bonus_rewards.go:48-61
func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
    ...
    if !poolBalance.IsAllGTE(bonus) {
        return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
    }
    return nil
}
```

If the pool balance is insufficient, `ErrInsufficientBonusPool` propagates up and the entire transaction is rolled back. The third exit message, `MsgWithdrawFromTier`, does not call `claimRewards`, but it requires the position to already be undelegated — it can only be reached after a successful `TierUndelegate`, which is itself blocked.

The result: a user who has completed their exit commitment and submitted `MsgTierUndelegate` or `MsgExitTierWithDelegation` is completely unable to recover their principal. There is no code path that allows exiting without first paying out accrued bonus rewards, and there is no mechanism to waive the bonus claim.

### Impact Explanation

The corrupted invariant is: *once the exit commitment has elapsed, the position owner must be able to exit*. When the bonus pool is empty, this invariant is broken. The user's principal (staked tokens held by the position's per-position delegator address) is locked in the module indefinitely. This is more severe than the AuraLocker analog — there the user suffered a 20% penalty on rewards; here the user's entire principal is inaccessible. The base rewards already withdrawn to the owner via `distributionKeeper.WithdrawDelegationRewards` are unaffected, but the locked principal cannot be recovered.

### Likelihood Explanation

The bonus pool is funded externally (by governance or a designated funder). It can run dry through:
1. Normal depletion as bonus rewards are paid out over time without timely replenishment.
2. Governance inaction or delay in passing a replenishment proposal.
3. A deliberate governance decision to stop funding the pool while sunsetting a tier (e.g., marking it `CloseOnly`).

The `CloseOnly` scenario is particularly acute: governance marks a tier `CloseOnly` to sunset it, stops funding the pool, and existing position owners who have triggered exit find both `TierUndelegate` and `ExitTierWithDelegation` permanently blocked.

### Recommendation

Separate the bonus claim from the exit precondition. When the bonus pool is insufficient, allow the exit to proceed while forfeiting (or deferring) the unpayable bonus — the same pattern already used in `BeforeRedelegationSlashed` to avoid chain halts:

```go
// x/tieredrewards/keeper/slash.go:54-63
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        // Deliberately forgo bonus rewards if pool is insufficient
    } else {
        return err
    }
}
```

Apply the same pattern in `TierUndelegate` and `ExitTierWithDelegation`: if `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, log the forfeiture and allow the exit to proceed. Alternatively, add a separate `claimBaseRewardsOnly` path that skips the bonus check, so users can always exit even when the pool is empty.

### Proof of Concept

1. User locks tokens via `MsgLockTier` into a tier with non-zero `BonusApy`.
2. Time passes; bonus accrues. The bonus pool is not replenished (governance inaction or `CloseOnly` sunset).
3. User calls `MsgTriggerExitFromTier`. Exit commitment timer starts.
4. Exit commitment elapses (`block_time >= ExitUnlockAt`).
5. User submits `MsgTierUndelegate`:
   - `validateUndelegatePosition` passes (exit triggered, elapsed, delegated).
   - `claimRewards` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
   - Transaction reverts. Principal remains locked.
6. User submits `MsgExitTierWithDelegation`:
   - `validateExitTierWithDelegation` passes.
   - `claimRewards` → same failure path → `ErrInsufficientBonusPool`.
   - Transaction reverts. Principal remains locked.
7. `MsgWithdrawFromTier` is unavailable because the position is still delegated (step 5 and 6 both failed).

func (ms msgServer) WithdrawFromTier(ctx context.Context, msg *types.MsgWithdrawFromTier) (*types.MsgWithdrawFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateWithdrawFromTier(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
	if !balances.IsZero() {
		if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
			return nil, err
		}
	}

	if err := ms.deletePosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventPositionWithdrawn{
		Position: pos.Position,
		Amount:   balances,
	}); err != nil {
		return nil, err
	}

	return &types.MsgWithdrawFromTierResponse{
		Amount: balances,
	}, nil
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
