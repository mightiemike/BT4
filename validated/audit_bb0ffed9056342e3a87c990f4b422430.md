### Title
Wrong `class_hash = Felt::ZERO` in DeclareV1 simulation produces incorrect transaction hash and state diff in `simulate_transactions` / `estimate_fee` — (`crates/apollo_rpc/src/v0_8/api/mod.rs`)

### Summary

When a `BroadcastedDeclareV1Transaction` is converted to `ExecutableTransactionInput::DeclareV1` for RPC simulation or fee estimation, the `class_hash` field of the inner `DeclareTransactionV0V1` is unconditionally set to `ClassHash::default()` (`Felt::ZERO`). This zero value propagates into two protocol-visible outputs: (1) the transaction hash computed for the simulation run, and (2) the `deprecated_declared_classes` list in the `induced_state_diff` returned to the caller.

### Finding Description

**Conversion site** — `crates/apollo_rpc/src/v0_8/api/mod.rs`, lines 494–508:

**Hash computation** — `crates/starknet_api/src/transaction_hash.rs`, line 556:

`get_declare_transaction_v1_hash` feeds `transaction.class_hash.0` directly into the Pedersen hash chain. Because `class_hash = Felt::ZERO`, the computed `tx_hash` is `H(…, H(0), …)` instead of `H(…, H(actual_class_hash), …)`. [2](#0-1) 

**State-diff construction** — `crates/apollo_rpc_execution/src/lib.rs`, lines 740–745, and `crates/apollo_rpc_execution/src/execution_utils.rs`, lines 130–144:

`execute_transactions` extracts `class_hash` from the `DeclareTransactionV0V1` struct before execution and passes it as `deprecated_declared_class_hash` to `induced_state_diff`. That function places it verbatim into `ThinStateDiff::deprecated_declared_classes`:

```rust
deprecated_declared_classes: deprecated_declared_class_hash
    .map_or_else(Vec::new, |class_hash| vec![class_hash]),
```

Because `class_hash = Felt::ZERO`, the simulation output's `induced_state_diff.deprecated_declared_classes` always contains `[0x0]` instead of the actual Pedersen class hash. [3](#0-2) [4](#0-3) 

**RPC entry points** — `simulate_transactions` and `estimate_fee` both convert `BroadcastedTransaction` via `tx.try_into()` (which calls the `TryFrom` above) and then call `exec_simulate_transactions` / `exec_estimate_fee` with `tx_hashes = None`, triggering `calc_tx_hashes` → `calc_tx_hash` → `get_transaction_hash` on the zero-class-hash struct. [5](#0-4) [6](#0-5) 

### Impact Explanation

Two concrete, protocol-visible wrong values are returned to any caller of `starknet_simulateTransactions` or `starknet_estimateFee` with a DeclareV1 transaction:

1. **Wrong transaction hash during execution**: The `tx_hash` passed to `BlockifierTransaction::from_api` is computed with `class_hash = Felt::ZERO`. Any `get_tx_info` syscall inside the account's `__validate__` or `__execute__` receives this wrong hash. For accounts that verify the transaction hash (standard ECDSA accounts), simulation with `validate = true` will fail with a signature error rather than returning a useful result. With `validate = false` the wrong hash silently propagates through the execution context.

2. **Wrong `deprecated_declared_classes` in `induced_state_diff`**: The `SimulatedTransaction.transaction_trace` / state diff returned to the caller always lists `[0x0]` as the declared class hash instead of the actual Pedersen hash of the submitted class. A client relying on this field to determine which class was declared (e.g., to pre-populate a local cache or to verify the simulation matches an expected declaration) receives a completely wrong value.

This satisfies: **High — RPC simulation/fee estimation returns an authoritative-looking wrong value.**

### Likelihood Explanation

The path is unconditional and requires no special privileges. Any unprivileged user who submits a `BroadcastedDeclareV1Transaction` to `starknet_simulateTransactions` or `starknet_estimateFee` triggers both wrong outputs on every call. The comment in the code acknowledges the workaround but does not account for the downstream hash and state-diff effects.

### Recommendation

Compute the actual Pedersen class hash from `sn_api_contract_class` before constructing `DeclareTransactionV0V1`, and use it as `class_hash`. The starknet-api crate already exposes `compute_deprecated_class_hash` (or equivalent) for this purpose. This ensures `get_declare_transaction_v1_hash` and `induced_state_diff` both use the correct value.

### Proof of Concept

```rust
// In crates/apollo_rpc/src/v0_8/api/mod.rs (test)
let broadcasted = BroadcastedDeclareTransaction::V1(BroadcastedDeclareV1Transaction {
    contract_class: some_deprecated_class(),
    sender_address: ContractAddress::default(),
    nonce: Nonce::default(),
    max_fee: Fee(1000),
    signature: TransactionSignature::default(),
    r#type: DeclareType::Declare,
});

let ExecutableTransactionInput::DeclareV1(declare_tx, class, _, _) =
    ExecutableTransactionInput::try_from(broadcasted).unwrap()
else { panic!() };

// Demonstrates the bug: class_hash is always zero
assert_eq!(declare_tx.class_hash, ClassHash::default()); // passes — wrong

// The tx hash is computed with Felt::ZERO as class_hash
let chain_id = ChainId::Mainnet;
let (_, tx_hash) = ExecutableTransactionInput::DeclareV1(declare_tx, class, 0, false)
    .calc_tx_hash(&chain_id).unwrap();

// The actual hash of the class is non-zero; recomputing with the real hash
// produces a different tx_hash, proving divergence.
let actual_class_hash = compute_deprecated_class_hash(&class).unwrap();
assert_ne!(actual_class_hash, ClassHash::default()); // actual hash ≠ 0
// tx_hash was computed with 0, not actual_class_hash → wrong hash returned to caller
```

```rust
pub(crate) fn get_declare_transaction_v1_hash(
    transaction: &DeclareTransactionV0V1,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    Ok(TransactionHash(
        HashChain::new()
        .chain(&DECLARE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address.0.key())
        .chain(&Felt::ZERO) // No entry point selector in declare transaction.
        .chain(&HashChain::new().chain(&transaction.class_hash.0).get_pedersen_hash())
        .chain(&transaction.max_fee.0.into())
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce.0)
        .get_pedersen_hash(),
    ))
}
```

```rust
pub fn induced_state_diff(
    transactional_state: &mut CachedState<MutRefState<'_, CachedState<ExecutionStateReader>>>,
    deprecated_declared_class_hash: Option<ClassHash>,
) -> ExecutionResult<ThinStateDiff> {
    let blockifier_state_diff =
        CommitmentStateDiff::from(transactional_state.to_state_diff()?.state_maps);

    Ok(ThinStateDiff {
        deployed_contracts: blockifier_state_diff.address_to_class_hash,
        storage_diffs: blockifier_state_diff.storage_updates,
        class_hash_to_compiled_class_hash: blockifier_state_diff.class_hash_to_compiled_class_hash,
        deprecated_declared_classes: deprecated_declared_class_hash
            .map_or_else(Vec::new, |class_hash| vec![class_hash]),
        nonces: blockifier_state_diff.address_to_nonce,
    })
```
