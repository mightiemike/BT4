### Title
Duplicate Position ID in `ClaimTierRewards` Enables Double Bonus Reward Drain and Persistent State Corruption — (`File: x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgClaimTierRewards` accepts a list of position IDs, loads them all into a slice before any processing, and then iterates over that slice to claim rewards and persist updated state. There is no deduplication of position IDs in the handler. If an attacker submits the same position ID twice in `PositionIds`, the position is processed twice: bonus rewards are paid out twice from the `RewardsPoolName` module account, and the second `setPosition` call overwrites the correctly-advanced checkpoint state with the stale pre-claim snapshot, permanently resetting `LastEventSeq` and `LastBonusAccrual` to their old values and enabling the attack to be repeated indefinitely.

---

### Finding Description

In `ClaimTierRewards` the handler first loads all requested positions into a local slice: [1](#0-0) 

No deduplication is performed. The slice is then passed to `claimRewardsAndUpdatesPositions`: [2](#0-1) 

Inside `claimRewardsAndUpdatesPositions`, each element is processed independently: [3](#0-2) 

For the **first** occurrence of a duplicated position (index `i`):

1. `claimBaseRewards` calls `distributionKeeper.WithdrawDelegationRewards`, draining accrued base rewards.
2. `processEventsAndClaimBonus` walks events since `pos.LastEventSeq`, computes the bonus (including the current open segment from `pos.LastBonusAccrual` to `blockTime`), calls `bankKeeper.SendCoinsFromModuleToAccount` from `RewardsPoolName`, and advances `pos.LastEventSeq` and `pos.LastBonusAccrual` in the in-memory struct.
3. `setPosition` persists the advanced checkpoint to the store. [4](#0-3) 

For the **second** occurrence of the same position (index `i+1`), the slice element still holds the **pre-claim snapshot** (loaded before the first iteration ran). Therefore:

1. `processEventsAndClaimBonus` is called again with the old `LastEventSeq` and old `LastBonusAccrual`. Even if historical events were deleted by `decrementEventRefCount` in the first pass, the current open-segment bonus is recomputed from the stale `segmentStart = pos.LastBonusAccrual` to `blockTime`, producing a non-zero `totalBonus` and triggering a second `SendCoinsFromModuleToAccount`. [5](#0-4) 

2. `setPosition` is called with the stale position, **overwriting** the correctly-advanced checkpoint that was just persisted in the first pass. This resets `LastEventSeq` and `LastBonusAccrual` to their pre-claim values. [6](#0-5) 

The net result is: bonus paid twice per submission, and the position's accrual checkpoint is permanently reset, so the attacker can repeat the same transaction to drain the rewards pool continuously.

---

### Impact Explanation

The `RewardsPoolName` module account balance is drained by double (or repeated) `SendCoinsFromModuleToAccount` calls for the same position's bonus. The corrupted `LastEventSeq` / `LastBonusAccrual` fields on the position allow the attack to be replayed in subsequent blocks. The exact corrupted values are:

- **`types.RewardsPoolName` module account** — loses `N × bonusCoins` per submission with `N` duplicates.
- **`pos.LastBonusAccrual` / `pos.LastEventSeq`** — reset to stale values, breaking the invariant that these fields monotonically advance. [7](#0-6) 

---

### Likelihood Explanation

Any unprivileged account that owns at least one tiered-rewards position can trigger this by submitting a standard `MsgClaimTierRewards` Cosmos SDK transaction with `PositionIds = [X, X]`. No special role, leaked key, or social engineering is required. The entry path is the normal CLI/gRPC transaction path.

---

### Recommendation

Add a deduplication check in `ClaimTierRewards` before appending to the `positions` slice, e.g., maintain a `seen map[uint64]struct{}` and return an error if a position ID appears more than once. Alternatively, deduplicate inside `claimRewardsAndUpdatesPositions` by re-fetching the position from the store at the start of each iteration rather than relying on the pre-loaded snapshot. [8](#0-7) 

---

### Proof of Concept

1. Attacker owns position with ID `42`, which has accrued bonus rewards `B` since `LastBonusAccrual = T0`.
2. Attacker submits `MsgClaimTierRewards{ Owner: attacker, PositionIds: [42, 42] }`.
3. `ClaimTierRewards` loads position `42` twice into `positions[0]` and `positions[1]`, both with `LastBonusAccrual = T0`.
4. First loop iteration (`i=0`): `processEventsAndClaimBonus` computes bonus `B`, sends `B` from `RewardsPoolName` to attacker, advances `positions[0].LastBonusAccrual` to `blockTime`, calls `setPosition` — store now has `LastBonusAccrual = blockTime`.
5. Second loop iteration (`i=1`): `positions[1]` still has `LastBonusAccrual = T0`. `processEventsAndClaimBonus` recomputes the same open-segment bonus `B` (segment `[T0, blockTime]`), sends another `B` from `RewardsPoolName` to attacker. `setPosition` overwrites the store with `LastBonusAccrual = T0`.
6. Attacker received `2B` instead of `B`. Store is back to `LastBonusAccrual = T0`.
7. Attacker repeats step 2 in the next block, draining the pool. [8](#0-7) [3](#0-2)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)
