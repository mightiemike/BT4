### Title
Unsigned `ContractCodeResponse::V1` Accepted Without Responder Identity Verification Before `SignedContractCodeResponse` Feature Activation — (File: `chain/client/src/stateless_validation/validate.rs`)

### Summary
`validate_contract_code_response` skips all responder-identity and signature checks when `ProtocolFeature::SignedContractCodeResponse` is not yet active. Any T2 network peer can send a `ContractCodeResponse::V1` carrying arbitrary contract-code bytes for a publicly-known `ChunkProductionKey`, and the receiving chunk validator will accept and use those bytes to validate the state witness — with no verification that the sender is a legitimate chunk producer or that the bytes match the requested code hashes.

### Finding Description

`ContractCodeResponse` is a versioned enum with two variants:

- `V1` — unsigned, no `responder` field, no signature
- `V2` — signed by the chunk producer, with a `responder: AccountId` field [1](#0-0) 

The sender-side encoder selects the variant based on the protocol version: [2](#0-1) 

The receiver-side validator mirrors this gate exactly — signature verification is **only performed when the feature is already active**: [3](#0-2) 

Before `SignedContractCodeResponse` is active, the only check performed is `validate_chunk_relevant` (a height/epoch window check). There is no check that:
1. The sender is a chunk producer for the relevant shard/epoch, or
2. The compressed contract bytes match the code hashes that were originally requested.

The accepted contracts are then stored directly into the witness tracker: [4](#0-3) 

The `ContractCodeRequest` path, by contrast, **always** verifies the requester's signature unconditionally: [5](#0-4) 

This asymmetry means the response path is unguarded during the pre-activation window.

The `ExcludeContractCodeFromStateWitness` feature removes contract code bytes from the state witness itself (only the hash is recorded in the trie recording path). Consequently, the validator has no trie-Merkle proof to cross-check the received bytes against — it must trust whatever code it receives via `ContractCodeResponse`. [6](#0-5) 

`verify_signature` on `V1` unconditionally returns `false` and is never called on the receiver side before the feature activates, confirming there is no fallback check.

### Impact Explanation

A malicious T2 peer that knows a valid `ChunkProductionKey` (shard_id, epoch_id, height_created — all publicly derivable from the chain head) can:

1. Craft a `ContractCodeResponse::V1` with the correct key but wrong compressed contract bytes.
2. Route it to a chunk validator before the legitimate chunk producer's response arrives.
3. The validator stores the poisoned bytes, compiles the wrong WASM module, and re-executes the chunk with wrong code.
4. Execution diverges from the canonical result; the validator rejects the valid chunk endorsement.

This is a **liveness attack on stateless validation**: targeted validators stop endorsing valid chunks, stalling block production for the affected shard. Because the `ChunkProductionKey` is public and T2 routing is open to any peer, the attack surface is the entire pre-activation epoch window.

### Likelihood Explanation

- `ProtocolFeature::SignedContractCodeResponse` is a nightly/upcoming feature not yet active at stable protocol version 86.
- Any node with a T2 connection can route `ContractCodeResponse` messages.
- The `ChunkProductionKey` fields (`shard_id`, `epoch_id`, `height_created`) are observable from the public chain.
- The attack requires no privileged role — only network reachability to the target validator.

### Recommendation

1. **Hash-verify received bytes immediately**: when `store_accessed_contract_codes` ingests contracts from a `ContractCodeResponse`, verify `CryptoHash::hash(code_bytes) == requested_hash` for every entry. This is independent of the signature feature and closes the injection window regardless of protocol version.
2. **Ensure `SignedContractCodeResponse` activates before or simultaneously with `ExcludeContractCodeFromStateWitness`** in any epoch where both features could coexist, so the trie-bypass path is never open without identity verification.
3. Consider making the signature check unconditional for `V2` messages even before the feature epoch, so a validator that receives a `V2` always verifies it, and a `V1` received after the feature epoch is rejected.

### Proof of Concept

```
1. Observe the chain head to derive a valid ChunkProductionKey:
      { shard_id: S, epoch_id: E, height_created: H }

2. Construct a ContractCodeResponse::V1 with:
      next_chunk = ChunkProductionKey { shard_id: S, epoch_id: E, height_created: H }
      compressed_contracts = <arbitrary wrong WASM bytes, validly compressed>

3. Borsh-serialize as ContractCodeResponse discriminant 0 (V1).

4. Route the message to a chunk validator for shard S at height H via any T2 peer connection.

5. validate_contract_code_response checks:
      - validate_chunk_relevant → passes (height H is in the relevant window)
      - ProtocolFeature::SignedContractCodeResponse.enabled(version) → false (pre-activation)
      → returns Ok(ChunkRelevance::Relevant)

6. handle_contract_code_response calls decompress_contracts() and
   partial_witness_tracker.store_accessed_contract_codes(key, wrong_contracts).

7. When the validator assembles the state witness, it compiles and executes
   the wrong WASM, producing a divergent state root, and rejects the valid chunk.

#[derive(Debug, Clone, PartialEq, Eq, BorshSerialize, BorshDeserialize, ProtocolSchema)]
#[borsh(use_discriminant = true)]
#[repr(u8)]
pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
}

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())
