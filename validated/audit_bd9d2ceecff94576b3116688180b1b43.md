### Title
Protobuf `ValidResourceBounds` Variant Silently Switches from `AllResources` to `L1Gas` on Zero Fields, Diverging Transaction Hash Between Proposer and Syncing Nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in the P2P sync path silently downgrades a transaction's resource-bounds variant from `AllResources` to `L1Gas` whenever `l2_gas` and `l1_data_gas` are zero. Because `get_tip_resource_bounds_hash` produces a structurally different hash for the two variants (3 vs 4 elements in the Poseidon chain), the hash the proposer commits to (`AllResources`, 4 elements) diverges from the hash a syncing node recomputes (`L1Gas`, 3 elements). A transaction that passes gateway admission and is included in a block will therefore fail hash validation on every node that syncs it over P2P, producing a chain split.

---

### Finding Description

**Two distinct `ValidResourceBounds` variants produce different hashes**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage differently per variant:

```
L1Gas      → [tip, L1_GAS_packed, L2_GAS_packed]          (3 elements)
AllResources → [tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]  (4 elements)
``` [1](#0-0) 

**Proposer always commits the `AllResources` hash**

`InternalRpcInvokeTransactionV3::resource_bounds()` unconditionally wraps the stored `AllResourceBounds` in `ValidResourceBounds::AllResources`, so the hash stored in `InternalRpcTransaction.tx_hash` is always the 4-element variant. [2](#0-1) [3](#0-2) 

**P2P sync path silently switches to `L1Gas` on zero fields**

The storage/sync path deserializes `Transaction` (not `RpcTransaction`) from protobuf. The converter for that path is:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← 3-element hash domain
} else {
    ValidResourceBounds::AllResources(...)       // ← 4-element hash domain
})
``` [4](#0-3) 

When a transaction carries `AllResourceBounds { l1_gas: non-zero, l2_gas: 0, l1_data_gas: 0 }`, the proposer stores hash H₁ (AllResources, 4 elements). After protobuf round-trip in the sync path the variant becomes `L1Gas`, and the syncing node recomputes hash H₂ (L1Gas, 3 elements). H₁ ≠ H₂.

**The consensus path is immune; the sync path is not**

The consensus protobuf converter uses a separate `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` that always produces `AllResourceBounds` regardless of zero values, so consensus peers agree on the hash. Only the historical-sync path (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`) is affected. [5](#0-4) 

**The issue is acknowledged in tests but not fixed**

The consensus round-trip test explicitly works around the problem by injecting a non-zero `l2_gas.max_amount` before serialization:

```rust
// If all the fields of `AllResources` are 0 upon serialization,
// then the deserialized value will be interpreted as the `L1Gas` variant.
resource_bounds.l2_gas.max_amount = GasAmount(1);
``` [6](#0-5) 

**Gateway admits the triggering transaction**

The gateway's zero-bounds check wraps the `AllResourceBounds` in `AllResources` before calling `max_possible_fee`. A transaction with non-zero `l1_gas` but zero `l2_gas` and `l1_data_gas` passes because `l1_gas` alone makes the fee non-zero. [7](#0-6) 

---

### Impact Explanation

A syncing node that receives a block containing such a transaction recomputes a hash that does not match the committed hash. Hash validation fails, the block is rejected, and the syncing node diverges from the canonical chain — a chain split. This maps to **Critical: Wrong state / revert result from execution logic for accepted input** and **High: RPC/gateway rejects valid transactions before sequencing**.

---

### Likelihood Explanation

The gateway does not enforce a minimum non-zero `l2_gas` amount or price independently of the overall fee check. Any user who submits a V3 transaction with `l2_gas = 0` and `l1_data_gas = 0` (but non-zero `l1_gas`) triggers the divergence. No privileged access is required.

---

### Recommendation

1. **Fix the variant selector**: Remove the zero-value heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Introduce an explicit discriminant field in the protobuf schema (e.g., a `bounds_type` enum) so the variant is preserved losslessly across serialization.

2. **Short-term mitigation**: Add a gateway check that rejects any V3 transaction where `l2_gas.max_amount == 0 && l2_gas.max_price_per_unit == 0 && l1_data_gas.max_amount == 0 && l1_data_gas.max_price_per_unit == 0`, preventing the triggering transaction from ever reaching the mempool.

3. **Align the sync converter with the consensus converter**: Use `AllResourceBounds::try_from(protobuf::ResourceBounds)` (which never switches variants) in the sync path as well, and wrap it in `ValidResourceBounds::AllResources` unconditionally for post-0.13.3 blocks.

---

### Proof of Concept

```
1. Craft an RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Submit via gateway → passes stateless validation (max_possible_fee > 0 from l1_gas).

3. TransactionConverter::convert_rpc_tx_to_internal calls
   InternalRpcInvokeTransactionV3::resource_bounds() → ValidResourceBounds::AllResources(...)
   → get_tip_resource_bounds_hash hashes [tip, L1_GAS, L2_GAS(0), L1_DATA_GAS(0)] = H₁ (4 elements).
   tx_hash = H₁ is stored in InternalRpcTransaction.

4. Proposer includes the transaction in a block; block is committed with H₁.

5. A syncing node fetches the block over P2P. The transaction body is deserialized via
   TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
     l1_data_gas.is_zero() && l2_gas.is_zero() → ValidResourceBounds::L1Gas(l1_gas)

6. Syncing node calls get_tip_resource_bounds_hash with L1Gas variant:
   hashes [tip, L1_GAS, L2_GAS(0)] = H₂ (3 elements).  H₂ ≠ H₁.

7. validate_transaction_hash(expected = H₁, computed = H₂) → false.
   Block rejected. Chain split.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
