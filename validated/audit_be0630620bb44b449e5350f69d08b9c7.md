Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of User EOA, Allowing Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps on the `sender` argument, which is `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, that value is the router's address. A pool admin who allowlists the router so that approved users can trade via the standard periphery simultaneously opens the pool to every user on the network, completely nullifying the allowlist.

## Finding Description

**Root cause — wrong actor in the allowlist lookup**

`SwapAllowlistExtension.beforeSwap` performs:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension's direct caller) and `sender` is the first argument forwarded by the pool — which is `msg.sender` of `pool.swap()`.

**How the pool populates `sender`**

`MetricOmmPool.swap` calls `_beforeSwap` with `msg.sender` as the first argument:

```solidity
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
``` [2](#0-1) 

`ExtensionCalling._beforeSwap` encodes that value verbatim as the `sender` argument to the extension: [3](#0-2) 

**How the router breaks identity**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making `msg.sender` inside the pool equal to the router contract, not the user's EOA: [4](#0-3) 

The same pattern applies to `exactInput` (L104-112) and `exactOutputSingle` (L136-137). [5](#0-4) 

**The dilemma the admin faces**

| Admin choice | Effect |
|---|---|
| Do **not** allowlist the router | Allowlisted users cannot use the standard periphery; all router-mediated swaps revert |
| **Allowlist the router** | `allowedSwapper[pool][router] = true` → the check passes for **any** user who calls the router |

The router is a public, permissionless contract with no access control of its own. There are no existing guards in `SwapAllowlistExtension` or `MetricOmmSimpleRouter` that distinguish which EOA initiated the router call. [6](#0-5) 

## Impact Explanation

**Severity: High**

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict trading to a known set of counterparties (e.g., KYC'd addresses, whitelisted market makers). The moment the admin allowlists the router — a necessary step for allowlisted users to access the standard swap UI — the allowlist is nullified. Any unprivileged EOA can call `exactInputSingle` or `exactInput` on the router and trade on the supposedly restricted pool. This is a direct, complete bypass of a core access-control mechanism with no additional preconditions beyond the admin's own reasonable configuration step. The broken invariant is: `allowedSwapper[pool][sender]` is satisfied by the router address for all callers, not just the intended EOAs.

## Likelihood Explanation

**Likelihood: High**

`MetricOmmSimpleRouter` is the canonical periphery swap entry point. Pool admins who want their allowlisted users to trade normally will allowlist the router. The bypass requires no special knowledge, no privileged keys, and no unusual token behavior — any user who can call the router (i.e., everyone) can exploit it the moment the router is allowlisted.

## Recommendation

The extension must gate the **economic actor** — the original user — not the direct caller of `pool.swap()`. Two complementary fixes:

1. **Router-side**: Have `MetricOmmSimpleRouter` encode the originating user's address (`msg.sender`) into `extensionData` so the extension can decode and check it. This requires a convention between the router and the extension.

2. **Extension-side**: Change `SwapAllowlistExtension.beforeSwap` to decode the original actor from `extensionData` when `sender` is a known router, or require that pool admins never allowlist the router itself and document this constraint explicitly.

The cleanest long-term fix:

```diff
- if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
+ address actor = _decodeActorFromExtensionData(extensionData, sender);
+ if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][actor]) {
```

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only `allowedUser` is allowlisted.
// Admin also allowlists the router so allowedUser can trade via UI.
extension.setAllowedToSwap(pool, allowedUser, true);
extension.setAllowedToSwap(pool, address(router), true); // admin's necessary step

// Attacker (not allowlisted):
vm.startPrank(attacker);
token0.approve(address(router), type(uint256).max);

// Router calls pool.swap() → pool sees msg.sender = router
// → allowedSwapper[pool][router] = true → check passes for attacker
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        recipient:        attacker,
        zeroForOne:       true,
        amountIn:         1e18,
        amountOutMinimum: 0,
        priceLimitX64:    0,
        deadline:         block.timestamp,
        extensionData:    ""
    })
);
vm.stopPrank();
// attacker successfully swapped on a pool that should have blocked them
```
