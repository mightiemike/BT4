### Title
Division by Zero in `PerpEngine.socializeSubaccount` Permanently Locks Liquidation Finalization When `openInterest == 0` — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when socializing residual negative `vQuoteBalance`. No guard exists for the case where `openInterest == 0`. When this condition is met, `MathSD21x18.div` explicitly reverts with `"DBZ"`, causing the entire `liquidateSubaccountImpl` transaction to revert. Because `_finalizeSubaccount` is the only code path that resolves an insolvent subaccount, the subaccount becomes permanently unliquidatable and its bad debt unresolvable.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance partially covers a negative `vQuoteBalance`, the remaining deficit is socialized across all open-interest holders:

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← no zero-guard
    );
``` [1](#0-0) 

`MathSD21x18.div` enforces `require(y != 0, ERR_DIV_BY_ZERO)`:

```solidity
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`_finalizeSubaccount` requires `balance.amount == 0` for every perp product before calling `socializeSubaccount`:

```solidity
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [3](#0-2) 

This means the liquidatee contributes zero to `state.openInterest`. If all other participants have also closed their positions in that product, `state.openInterest == 0`. A subaccount can simultaneously have `amount == 0` and `vQuoteBalance < 0` — the `vQuoteBalance` is accumulated realized PnL that persists after a position is closed. `_updateBalance` confirms this: `vQuoteBalance` is updated independently of `amount`:

```solidity
balance.amount += balanceDelta;
balance.vQuoteBalance += deltaQuote;
``` [4](#0-3) 

The call chain is:

```
liquidateSubaccountImpl
  └─ _finalizeSubaccount (txn.productId == type(uint32).max)
       └─ perpEngine.socializeSubaccount(liquidatee, insurance)
            └─ loop: product with openInterest==0, vQuoteBalance<0
                 └─ -vQuoteBalance.div(0) → revert "DBZ"
``` [5](#0-4) 

Because Solidity reverts undo all state changes atomically, there are no partial writes — but the entire liquidation transaction is aborted. Every subsequent attempt to finalize the same subaccount hits the identical revert, permanently blocking resolution.

---

### Impact Explanation

- **Permanent DoS of the liquidation finalization path** for any subaccount that reaches `_finalizeSubaccount` with a perp product satisfying `openInterest == 0` and `vQuoteBalance < 0` after insurance.
- The insolvent subaccount can never be closed out; its bad debt is permanently unresolvable.
- Protocol insurance and socialization mechanisms are rendered inoperable for the affected product.

---

### Likelihood Explanation

The precondition is reachable in normal operation:

1. A perp product with low participation (e.g., a single trader or a product being wound down).
2. The liquidatee closes their position (`amount → 0`) but retains negative `vQuoteBalance` (realized loss).
3. All other participants also close, driving `openInterest → 0`.
4. The liquidatee's health deteriorates (e.g., due to spot liabilities), triggering liquidation.
5. `_finalizeSubaccount` is called; `socializeSubaccount` reverts.

No privileged access or external dependency is required. The scenario is reproducible with a single-participant product in a standard Hardhat test.

---

### Recommendation

Add a zero-guard before the division. When `openInterest == 0`, there are no other participants to socialize the loss against; the remaining deficit should be absorbed by the insurance fund or written off as protocol bad debt:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest > 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    // If openInterest == 0, no one to socialize against; loss is written off.
    balance.vQuoteBalance = 0;
}
``` [6](#0-5)

### Proof of Concept

```solidity
// 1. Deploy PerpEngine with one product (productId = 2).
// 2. Alice opens a short: amount = -10e18, vQuoteBalance = +5e18 (entry credit).
// 3. Alice closes: amount = 0, vQuoteBalance = -3e18 (realized loss locked in).
//    → state.openInterest == 0 (no open positions remain).
// 4. Alice's spot balance goes negative; health < 0 → liquidatable.
// 5. Liquidator calls liquidateSubaccountImpl with productId = type(uint32).max.
// 6. _finalizeSubaccount passes the amount==0 check.
// 7. socializeSubaccount is called; insurance = 0 (or < 3e18).
// 8. balance.vQuoteBalance < 0 after insurance → enters socialization branch.
// 9. -(-3e18).div(0) → MathSD21x18.div reverts with "DBZ".
// 10. Assert: transaction reverts; Alice's subaccount is permanently unliquidatable.
```
