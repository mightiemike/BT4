Audit Report

## Title
Swap allowlist checks `msg.sender` (router) instead of the actual trader, allowing any user to bypass the allowlist via the router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of `pool.swap()` — the router contract, not the end user. When the router is allowlisted (the only way to enable router-based swaps), every user regardless of individual allowlist status can swap in a restricted pool. The allowlist cannot gate individual traders when they route through `MetricOmmSimpleRouter`.

## Finding Description

**Call chain:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle()` (or equivalent).
2. Router calls `pool.swap(recipient, zeroForOne, amountSpecified, priceLimitX64, callbackData, extensionData)`.
3. Inside `MetricOmmPool.swap()`, `msg.sender` is the **router address**.
4. `_beforeSwap(msg.sender, ...)` is called — `sender` = router address.

In `ExtensionCalling._beforeSwap` (line 160–176), the pool encodes `sender` (= router) and calls the extension:

```solidity
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, recipient, zeroForOne, ...)
)
```

In `SwapAllowlistExtension.beforeSwap` (line 37):

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` = pool (correct key), `sender` = **router address** (not the end user). The check is therefore against the router's allowlist entry, not the individual trader's.

**Two broken scenarios:**

- **Router is allowlisted:** Pool admin must allowlist the router to allow any router-based swap. Once the router is allowlisted, `allowedSwapper[pool][router] == true`, so every user — including non-allowlisted ones — passes the check by routing through the router. The per-user allowlist is completely bypassed.
- **Router is not allowlisted:** Individually allowlisted users cannot swap through the router at all, breaking the intended UX.

There is no existing guard that recovers the original `tx.origin` or passes the actual end user through the hook call chain.

## Impact Explanation
Any unprivileged, non-allowlisted trader can bypass the swap allowlist of a restricted pool by calling through `MetricOmmSimpleRouter`. This breaks the core access-control invariant of `SwapAllowlistExtension` — "Gates `swap` by swapper address, per pool" — and allows unauthorized swaps in pools intended to be restricted. Severity: **Medium** (broken core pool access-control functionality; no direct principal loss but allowlist-gated pools are rendered ineffective).

## Likelihood Explanation
Exploitable by any user with no special privileges. The only precondition is that the pool uses `SwapAllowlistExtension` and the router is allowlisted (the expected operational setup). The bypass is deterministic and repeatable on every swap.

## Recommendation
Pass the actual end user through the hook. Options:
1. Have the router forward the original caller as part of `extensionData`, and have the extension decode it (requires trust in the router).
2. Pass `tx.origin` as an additional field in the hook call (weaker, but simpler).
3. Redesign the hook interface to include an `originator` field distinct from `sender`, populated by the pool from a transient-storage context set by the router during its callback.

The cleanest fix is to have the router store the original `msg.sender` in transient storage before calling `pool.swap()`, and have the pool read it and pass it as `originator` to extensions alongside `sender`.

## Proof of Concept

```solidity
// Pool has SwapAllowlistExtension configured.
// Pool admin allowlists the router (required for any router swap).
// allowedSwapper[pool][router] = true
// allowedSwapper[pool][attacker] = false  (attacker is NOT allowlisted)

// Attacker calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: restrictedPool,
    zeroForOne: true,
    recipient: attacker,
    amountSpecified: 1e18,
    priceLimitX64: 0,
    callbackData: "",
    extensionData: ""
}));

// Inside pool.swap(): msg.sender = router
// _beforeSwap(router, attacker, ...)
// Extension checks: allowedSwapper[pool][router] == true  → passes
// Attacker successfully swaps despite not being individually allowlisted.
```
