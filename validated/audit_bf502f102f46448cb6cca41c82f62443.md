Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which equals `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, the router itself becomes `msg.sender`, so the extension always evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any non-allowlisted user can bypass the per-user curation policy by routing through `MetricOmmSimpleRouter`.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router itself `msg.sender` of that call: [3](#0-2) 

The same pattern holds for `exactInput` (all hops): [4](#0-3) 

And for `exactOutputSingle` and `exactOutput` (recursive hops via `_exactOutputIterateCallback`): [5](#0-4) [6](#0-5) 

The extension always evaluates `allowedSwapper[pool][router]`, never `allowedSwapper[pool][actualUser]`. The pool admin faces an impossible choice: if the router is not allowlisted, no user can swap through it (even individually allowlisted ones); if the router is allowlisted, every user can swap through it (even those not individually allowlisted). There is no mechanism to recover the actual user's address inside `beforeSwap`.

By contrast, `DepositAllowlistExtension` does not share this flaw because it gates by the `owner` parameter (the position owner explicitly passed to `addLiquidity`), which the liquidity adder correctly sets to the intended beneficiary: [7](#0-6) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to KYC'd, institutional, or otherwise curated addresses is completely open to any user who routes through `MetricOmmSimpleRouter`. The allowlist provides no protection on the standard periphery path. Any non-allowlisted address can execute live swaps against the pool, receiving real token output and paying real token input, defeating the pool admin's curation policy. This is a direct, unprivileged bypass of an access-control guard with fund-flow consequences: unauthorized trades execute at oracle prices against LP capital.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, documented periphery for swapping. Pool admins who deploy a `SwapAllowlistExtension` and want any user to swap through the router must add the router to the allowlist — the natural and expected configuration. Once the router is allowlisted, the bypass is trivially reachable by any address with no special setup. The trigger requires only a standard `exactInputSingle` or `exactInput` call from any EOA or contract.

## Recommendation
The `beforeSwap` extension hook must receive the original end-user's address to enforce per-user policies correctly. The cleanest fix is for the router to encode `msg.sender` (the actual user) into `extensionData` before calling `pool.swap()`. The `SwapAllowlistExtension` would then decode and check that address instead of `sender` (the router address). Specifically:

1. In `MetricOmmSimpleRouter`, prepend `abi.encode(msg.sender)` to `extensionData` before each `pool.swap()` call.
2. In `SwapAllowlistExtension.beforeSwap`, decode the originating user from `extensionData` and check `allowedSwapper[msg.sender][originatingUser]` instead of `allowedSwapper[msg.sender][sender]`.

## Proof of Concept
```solidity
// Pool is deployed with SwapAllowlistExtension as beforeSwap extension.
// Pool admin allowlists alice and the router (so alice can use the router).
extension.setAllowedToSwap(address(pool), alice, true);
extension.setAllowedToSwap(address(pool), address(router), true);

// charlie is NOT on the allowlist.
// charlie calls the router directly:
vm.prank(charlie);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token1),
        recipient: charlie,
        deadline: block.timestamp + 1,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: false,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// Succeeds: pool calls extension.beforeSwap(router, charlie, ...).
// Extension evaluates allowedSwapper[pool][router] → true and passes.
// Charlie's address is never inspected.
// Charlie receives token0 output despite not being on the allowlist.
```
