Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the actual swapper, allowing any user to bypass per-pool swap allowlists via `MetricOmmSimpleRouter` ‚Äî (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender` ‚Äî the direct caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, that caller is the router contract, not the actual user. A pool admin who allowlists the router to enable router-mediated swaps for their allowlisted users inadvertently opens the pool to every user, because the extension cannot distinguish allowlisted from non-allowlisted callers behind the router.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs its identity check as:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool and `sender` is the first argument forwarded by the pool from its own `msg.sender`:

```solidity
_beforeSwap(
    msg.sender,   // ‚Üê whoever called pool.swap()
    recipient,
    ...
);
``` [2](#0-1) 

When `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)`, the pool's `msg.sender` is the router contract: [3](#0-2) 

So the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. The `setAllowedToSwap` setter only records `(pool, address) ‚Üí bool` with no way to express per-user identity behind the router: [4](#0-3) 

The pool admin faces an impossible choice:
- **Option A** ‚Äî Do not allowlist the router ‚Üí allowlisted users cannot use the router at all.
- **Option B** ‚Äî Allowlist the router ‚Üí every user, including non-allowlisted ones, can bypass the guard by routing through `MetricOmmSimpleRouter`.

Under Option B the bypass is fully unprivileged: any address calls `router.exactInputSingle(restrictedPool, ...)` and the extension passes because `allowedSwapper[pool][router] == true`.

## Impact Explanation
A pool protected by `SwapAllowlistExtension` (e.g., a KYC-gated pool, a private pool with favorable oracle pricing, or a pool with subsidized spreads) can be freely accessed by any user once the pool admin allowlists the router. Unauthorized swappers can drain favorable pricing, extract subsidized output tokens, or interact with a pool contractually restricted to specific counterparties, causing direct loss of value to the pool's LPs and pool admin. This constitutes a broken core allowlist invariant with direct fund impact, meeting High severity thresholds.

## Likelihood Explanation
The pool admin's natural configuration step to let their allowlisted users use the router is to add the router to the allowlist. The extension provides no correct alternative ‚Äî there is no mechanism to gate by actual user identity behind the router. Any pool using `SwapAllowlistExtension` that also wants router support is vulnerable. The trigger is a semi-trusted pool admin action taken in good faith, and the exploit is fully unprivileged and repeatable by any address.

## Recommendation
Gate by the actual user's identity, not the intermediary's address:
1. Have the router forward the actual user's address in `extensionData`; have the extension decode and verify it (requires coordination between router and extension).
2. Add a trusted-forwarder pattern: if `sender` is a known router, decode the actual user from `extensionData` and check that address against the allowlist.
3. Document and enforce at the factory level that `SwapAllowlistExtension` is incompatible with router-mediated swaps, and revert pool creation if both are configured together.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension configured.
2. Pool admin: setAllowedToSwap(pool, alice, true)   // only Alice is allowed
3. Pool admin: setAllowedToSwap(pool, router, true)  // allow router so Alice can use it
4. Bob (non-allowlisted): calls router.exactInputSingle({pool: restrictedPool, ...})
5. Pool calls _beforeSwap(msg.sender=router, ...)
6. Extension checks allowedSwapper[pool][router] == true ‚Üí passes
7. Bob successfully swaps in the restricted pool, bypassing the allowlist entirely.
```
