### Title
Protobuf `ValidResourceBounds` round-trip silently downgrades `AllResources` to `L1Gas`, producing hash/data inconsistency for p2p-synced transactions — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` uses a zero-value heuristic to distinguish pre-0.13.3 (`L1Gas`) from post-0.13.3 (`AllResources`) transactions. A valid `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` is silently downgraded to `L1Gas` after a protobuf round-trip. Because `get_tip_resource_bounds_hash` hashes a different number of elements for `L1Gas` vs `AllResources`, the stored transaction hash (computed at submission with `AllResources`) diverges from any hash recomputed from the stored transaction data (computed with `L1Gas`). This breaks the hash/data canonicalization invariant and causes the RPC to return an authoritative-looking wrong resource-bounds representation.

---

### Finding Description

**Root cause — deserialization fallback (`transaction.rs` line 431–432):**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        // ...
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // ← silently defaults to zero
        // ...
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)                      // ← wrong variant chosen
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
``` [1](#0-0) 

**Serialization side — `L1Gas` and `AllResources(0,0)` produce identical bytes:**

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),          // zero
    l1_data_gas: Some(ResourceBounds::default().into()), // zero — same as AllResources(0,0)
},
``` [2](#0-1) 

Both `ValidResourceBounds::L1Gas(l1_gas)` and `ValidResourceBounds::AllResources(l1_gas, l2_gas=0, l1_data_gas=0)` serialize to identical protobuf bytes. On deserialization the `AllResources` variant is irreversibly lost.

**Hash divergence — `get_tip_resource_bounds_hash` hashes a different number of elements:**

```rust
// crates/starknet_api/src/transaction_hash.rs
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements total
    }
});
``` [3](#0-2) 

- `AllResources(l1_gas, l2_gas=0, l1_data_gas=0)` → hash over `[tip, L1_GAS, L2_GAS(0), L1_DATA_GAS(0)]` → **H₁**
- `L1Gas(l1_gas)` → hash over `[tip, L1_GAS, L2_GAS(0)]` → **H₂**
- **H₁ ≠ H₂** even though the numeric values of all bounds are identical.

**Attack path:**

1. An unprivileged user submits a V3 invoke transaction via the RPC with `AllResourceBounds` where `l1_gas > 0`, `l2_gas = 0`, `l1_data_gas = 0`. The gateway accepts it; fee is enforced on `l1_gas` only.
2. The transaction is included in a finalized block. Its hash **H₁** is computed using the `AllResources` path (3-element hash).
3. A peer node syncs the block via p2p. The `FullTransaction` protobuf carries the transaction body and **H₁** as separate fields.
4. The deserializer reconstructs the transaction body as `L1Gas` (because `l1_data_gas = 0` and `l2_gas = 0`). **H₁** is stored verbatim.
5. The synced node now holds: stored transaction body with `L1Gas` variant + stored hash **H₁** (computed with `AllResources`).
6. Any code that recomputes the hash from the stored body produces **H₂ ≠ H₁**. [4](#0-3) 

---

### Impact Explanation

**High — RPC returns an authoritative-looking wrong value.**

- `starknet_getTransactionByHash` returns `resource_bounds` with the `L1Gas` variant (one resource type) for a transaction that was originally submitted and signed with `AllResources` (three resource types). Clients and block explorers receive a structurally wrong representation.
- The `GasVectorComputationMode` derived from the stored bounds is `NoL2Gas` instead of `All`, so fee estimation and simulation (`starknet_estimateFee`, `starknet_simulateTransactions`) run the wrong gas accounting path for these transactions.
- Any component that recomputes the transaction hash from stored data (e.g., a prover re-executing the block, or a verifier checking the transaction commitment) obtains **H₂** instead of the canonical **H₁**, causing verification failures or wrong committed values.

---

### Likelihood Explanation

**Low-Medium.** The trigger requires a V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` to be included in a finalized block and then synced via p2p. This is a realistic configuration: many existing clients set only `l1_gas` and leave the other two bounds at zero. The backward-compatibility shim (`TODO(Shahak)`) is explicitly marked as temporary but is still active in production code.

---

### Recommendation

1. **Fix the serializer** so that `L1Gas` emits `l1_data_gas: None` (absent) rather than `Some(zero)`:

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),
    l1_data_gas: None,   // absent signals pre-0.13.3
},
```

2. **Fix the deserializer** to use presence/absence of `l1_data_gas` as the discriminant, not its numeric value:

```rust
Ok(match value.l1_data_gas {
    None => ValidResourceBounds::L1Gas(l1_gas),
    Some(dg) => ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas, l2_gas, l1_data_gas: dg.try_into()?,
    }),
})
```

This makes the round-trip lossless: `AllResources(l1_gas, 0, 0)` → `Some(zero)` → `AllResources(l1_gas, 0, 0)`, and `L1Gas(l1_gas)` → `None` → `L1Gas(l1_gas)`.

   starknet_getTransactionByHash(H₁)
