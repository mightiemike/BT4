### Title
`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` panic instead of returning `false` for unrecognized block hashes — (`contract/src/lib.rs`)

---

### Summary

Both SPV proof verification entry points (`verify_transaction_inclusion` and `verify_transaction_inclusion_v2`) declare a `-> bool` return type, implying that all caller-supplied inputs produce a graceful `true`/`false` result. In practice, when the caller-controlled `tx_block_blockhash` field is absent from the contract's state maps, the functions call `env::panic_str(...)` and abort the entire NEAR transaction instead of returning `false`. Any downstream NEAR contract that cross-calls these functions and expects to branch on the boolean result will have its transaction unconditionally reverted.

---

### Finding Description

`verify_transaction_inclusion` performs two state lookups on the caller-supplied `tx_block_blockhash`:

```
// contract/src/lib.rs:298-301
let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));
```

```
// contract/src/lib.rs:310-313
let header = self
    .headers_pool
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));
```

`verify_transaction_inclusion_v2` has the same pattern at its first lookup:

```
// contract/src/lib.rs:353-356
let header = self
    .headers_pool
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));
```

Both functions are documented as returning `True if tx_id is at the claimed position … False otherwise`, yet neither path returns `false` for a missing block — they abort. The `require!` macro used elsewhere in the same functions also calls `env::panic_str` internally, compounding the issue.

Three concrete, unprivileged triggers exist:

1. **GC'd block hash.** `run_mainchain_gc` is called automatically inside every `submit_blocks` invocation and removes old mainchain entries from both `mainchain_height_to_header` and `headers_pool`. A block hash that was valid at submission time becomes panic-inducing after GC removes it.
2. **Fork block hash.** A block stored only in `headers_pool` (fork path, `store_fork_header`) is absent from `mainchain_header_to_height`, so the first lookup in `verify_transaction_inclusion` panics.
3. **Fabricated hash.** Any 32-byte value not present in either map triggers the same abort.

---

### Impact Explanation

The function signature `-> bool` is the contract's public API promise. Downstream NEAR contracts that cross-call `verify_transaction_inclusion_v2` to gate a payment, bridge release, or other state change cannot catch the panic: in NEAR, a panic in a callee causes the entire originating transaction to revert. Protective patterns such as checking the cross-contract call result in a callback are bypassed because the panic propagates before any result is returned. Contracts whose logic requires a `false` return (e.g., "if proof invalid, refund the user") are permanently bricked for any block hash that has been GC'd or was never on the main chain.

---

### Likelihood Explanation

The GC mechanism runs automatically on every `submit_blocks` call with `batch_size = num_of_headers`. With a `gc_threshold` of ~52 704 blocks (≈1 year), any proof submitted for a block older than the threshold will panic. This is a routine operational condition, not a contrived edge case. Additionally, any relayer or user who submits a proof for a block that arrived on a fork (a normal occurrence during chain reorganizations) will trigger the panic path immediately.

---

### Recommendation

Replace each `unwrap_or_else(|| env::panic_str(...))` in both verification functions with an `Option`-returning branch that returns `false` instead of aborting:

```rust
// Instead of:
let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

// Use:
let Some(target_block_height) = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
else {
    return false;
};
```

Apply the same pattern to the `headers_pool` lookup in both functions. The `require!` calls for `confirmations` and empty proof checks may legitimately remain as panics (they represent caller contract bugs, not expected runtime conditions), but missing-block lookups must return `false` to honour the documented API contract.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 10`.
2. Submit 20 blocks so that GC removes the first 10.
3. Call `verify_transaction_inclusion_v2` with the block hash of block 1 (now GC'd).
4. Observe the NEAR transaction receipt shows `ExecutionStatus::Failure` with `"cannot find requested transaction block"` instead of a `false` return value.
5. Repeat with a fork block hash (submit a competing header that loses the chain-work race): the first lookup in `verify_transaction_inclusion` panics with `"block does not belong to the current main chain"`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
