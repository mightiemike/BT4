Audit Report

## Title
Router-Mediated Swaps Substitute Router Address for Original Caller in `SwapAllowlistExtension::beforeSwap`, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool::swap` passes its own `msg.sender` (the router) as the `sender` argument to `_beforeSwap`, which forwards it unchanged to `SwapAllowlistExtension::beforeSwap`. The extension gates swaps on `allowedSwapper[pool][sender]`, so it checks the router's address rather than the originating user's address. Any unprivileged user can bypass a pool's swap allowlist by routing through `MetricOmmSimpleRouter` if the router address is allowlisted, and conversely, legitimately allowlisted users cannot swap through the router because the router itself is not allowlisted.

## Finding Description
**Root cause — `MetricOmmPool::swap` (L230–240):**
The pool calls `_beforeSwap(msg.sender, ...)`. When `MetricOmmSimpleRouter::exactInputSingle` (or any other router entry point) calls `pool.swap()`, `msg.sender` inside the pool is the router contract, not the originating user. [1](#0-0) 

**Forwarding — `ExtensionCalling::_beforeSwap` (L149–177):**
The `sender` value (router address) is forwarded unchanged into the extension call payload. [2](#0-1) 

**Check — `SwapAllowlistExtension::beforeSwap` (L37–39):**
The extension evaluates `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the router. The original user's address is never consulted. [3](#0-2) 

**No forwarding mechanism — `MetricOmmSimpleRouter::exactInputSingle` (L71–80):**
The router calls `pool.swap()` with `extensionData` supplied by the caller but does not inject the original `msg.sender` into it. There is no mechanism to communicate the originating user to the extension. [4](#0-3) 

**Exploit path:**
1. Pool admin deploys pool with `SwapAllowlistExtension` to restrict swaps to specific addresses.
2. Admin calls `setAllowedToSwap(pool, address(router), true)` to enable router-mediated swaps for permitted users.
3. Unprivileged Bob calls `router.exactInputSingle(...)`.
4. Router calls `pool.swap(...)` — pool sees `msg.sender = router`.
5. `beforeSwap(sender=router)` checks `allowedSwapper[pool][router] == true` → passes.
6. Bob's swap executes despite never being allowlisted.

The same path applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) [6](#0-5) 

## Impact Explanation
**High — Allowlist bypass:** A pool admin who allowlists the router to enable router-mediated swaps for permitted users inadvertently grants every unprivileged user the ability to bypass the allowlist. The access control invariant — that only explicitly allowlisted addresses may swap — is fully defeated for any pool that uses `SwapAllowlistExtension` alongside the router. This constitutes a broken core pool functionality and an admin-boundary break reachable by an unprivileged path.

**Medium — Allowlisted users cannot use the router:** If the admin allowlists specific user addresses but not the router, those users' router calls revert because the router's address fails the allowlist check. Core swap functionality is broken for the intended beneficiaries. [7](#0-6) 

## Likelihood Explanation
The router is the canonical public entry point for swaps. Any pool that combines `SwapAllowlistExtension` with router support will encounter this. The bypass requires no special privileges, no flash loans, and no multi-transaction setup — a single `exactInputSingle` call from any EOA suffices. The precondition (router being allowlisted) is a natural and expected admin action. [8](#0-7) 

## Recommendation
Pass the originating user's address through the call chain. One approach: the router encodes `msg.sender` into `extensionData` before calling `pool.swap()`, and `SwapAllowlistExtension::beforeSwap` decodes and checks that address when `sender` is a known router. Alternatively, the pool could accept an explicit `swapper` parameter distinct from `msg.sender`. The hook must gate the economic actor the pool designers intended to allowlist, not the intermediate contract. [4](#0-3) 

## Proof of Concept
```solidity
// Setup
extension.setAllowedToSwap(pool, alice, true);
// Admin allowlists router to enable router-mediated swaps:
extension.setAllowedToSwap(pool, address(router), true);

// Bob (unprivileged) bypasses allowlist in a single transaction:
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: bob,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// pool.swap(msg.sender=router) → beforeSwap(sender=router)
// allowedSwapper[pool][router] == true → PASSES — Bob swaps successfully

// Direct swap by Bob correctly reverts:
pool.swap(bob, true, 1e18, 0, "", "");
// beforeSwap(sender=bob) → allowedSwapper[pool][bob] == false → NotAllowedToSwap ✓

    _checkDeadline(params.deadline);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
