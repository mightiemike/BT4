### Title
Missing Slippage Protection in NLP Mint and Burn Operations — (`core/contracts/Clearinghouse.sol`)

---

### Summary

The `MintNlp` and `BurnNlp` transaction structs contain no minimum output amount field. The conversion between quote tokens and NLP tokens is computed at sequencer execution time using `oraclePriceX18`, which is supplied by the sequencer and reflects the oracle price at processing time — not at signing time. Users have no on-chain mechanism to reject execution at an unfavorable price.

---

### Finding Description

When a user mints NLP tokens they sign `MintNlp{sender, quoteAmount, nonce}`. When burning they sign `BurnNlp{sender, nlpAmount, nonce}`. Neither struct includes a minimum acceptable output amount. [1](#0-0) 

The actual output is computed at execution time using `oraclePriceX18` from the outer `SignedMintNlp` / `SignedBurnNlp` struct, which is populated by the sequencer and is structurally outside the inner `tx` struct that the user signs.

In `Clearinghouse.mintNlp()`:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [2](#0-1) 

In `Clearinghouse.burnNlp()`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [3](#0-2) 

There is a time gap between when the user signs the transaction off-chain and when the sequencer includes it in a batch. During this window the NLP oracle price can change. Because the user's signed struct (`MintNlp` / `BurnNlp`) does not include a minimum output amount, there is no on-chain mechanism to revert execution at an unfavorable price.

The sequencer path that processes these transactions is `EndpointTx.processTransactionImpl()`: [4](#0-3) 

`validateSignedTx` verifies the user's signature against the inner `tx` struct fields (`sender`, `nonce`). The `oraclePriceX18` and `nlpPoolRebalanceX18` fields sit outside the signed `MintNlp` / `BurnNlp` struct and are appended by the sequencer at submission time, meaning the user's signature does not bind them to a specific execution price. [5](#0-4) 

---

### Impact Explanation

**BurnNlp:** If the NLP oracle price drops between signing and execution, the user receives fewer quote tokens than expected. The user's quote balance is credited with `nlpAmount × oraclePriceX18 − burnFee`, where `oraclePriceX18` is lower than the price the user observed when signing.

**MintNlp:** If the NLP oracle price rises between signing and execution, the user receives fewer NLP tokens than expected. The user's NLP balance is credited with `quoteAmount ÷ oraclePriceX18`, where `oraclePriceX18` is higher than the price the user observed when signing.

In both cases the user's on-chain balance is reduced by more than the user intended when constructing the signed transaction. No attacker is required to trigger the loss — normal market price movement during the signing-to-execution window is sufficient. [6](#0-5) 

---

### Likelihood Explanation

Medium. NLP price fluctuates continuously based on the performance of the underlying liquidity pools and the oracle feed. The sequencer processes transactions with inherent latency. During periods of volatility the price at execution can differ materially from the price at signing. No privileged actor or sequencer manipulation is required — the issue manifests under ordinary market conditions with an honest sequencer simply using the current oracle price.

---

### Recommendation

Add `minNlpAmount` to the `MintNlp` struct and `minQuoteAmount` to the `BurnNlp` struct:

```solidity
struct MintNlp {
    bytes32 sender;
    uint128 quoteAmount;
    uint128 minNlpAmount;   // slippage guard
    uint64 nonce;
}

struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint128 minQuoteAmount; // slippage guard
    uint64 nonce;
}
```

In `Clearinghouse.mintNlp()`, after computing `nlpAmount`, add:

```solidity
require(nlpAmount >= int128(txn.minNlpAmount), ERR_SLIPPAGE);
```

In `Clearinghouse.burnNlp()`, after computing the net `quoteAmount`, add:

```solidity
require(quoteAmount >= int128(txn.minQuoteAmount), ERR_SLIPPAGE);
``` [1](#0-0) 

---

### Proof of Concept

1. NLP oracle price is $10. Alice signs `BurnNlp{sender: alice, nlpAmount: 100, nonce: 5}` off-chain, expecting approximately $1,000 in quote tokens.
2. Before the sequencer processes the transaction, the NLP pool incurs losses and the oracle price drops to $5.
3. The sequencer constructs `SignedBurnNlp{tx: BurnNlp{...}, signature: aliceSig, oraclePriceX18: 5e18, nlpPoolRebalanceX18: [...]}` and submits it via `submitTransactionsChecked`.
4. `EndpointTx.processTransactionImpl()` validates Alice's signature (bound only to `{sender, nlpAmount, nonce}`) and calls `clearinghouse.burnNlp(signedTx.tx, 5e18, nlpPools, rebalance)`.
5. `Clearinghouse.burnNlp()` computes `quoteAmount = 100 × 5 = 500`, deducts `burnFee ≈ 1`, and credits Alice with `≈ 499` quote tokens.
6. Alice receives `≈ $499` instead of the `≈ $1,000` she expected when signing. No revert occurs because there is no minimum output check. [7](#0-6)

        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
