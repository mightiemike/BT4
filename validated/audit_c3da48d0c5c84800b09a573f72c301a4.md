### Title
`delistProduct` Does Not Remove Product from `productIds` Array, Causing Persistent State Desynchronization — (`core/contracts/Clearinghouse.sol`, `core/contracts/PerpEngineState.sol`)

---

### Summary

`Clearinghouse.delistProduct()` zeroes out listed subaccount positions but never removes the product from the `productIds` array in `PerpEngine`. This mirrors the Holdefi `removeMarket` / `marketsList` desynchronization exactly: one data structure (the mapping `productToEngine`) is implicitly "cleared" in intent, while the enumeration array (`productIds`) retains the stale entry. Every downstream function that iterates `productIds` continues to process the delisted product as if it were live.

---

### Finding Description

`BaseEngine` maintains two parallel data structures for each registered product:

- `productIds` — a `uint32[]` array used for enumeration by all loop-based operations. [1](#0-0) 
- `productToEngine` — a mapping from `productId → IProductEngine` used for per-product dispatch. [2](#0-1) 

When a new product is registered, both are populated atomically: `productIds.push(productId)` and `_clearinghouse.registerProduct(productId)` (which sets `productToEngine[productId]`). [3](#0-2) 

`Clearinghouse.delistProduct()` is the removal counterpart. It validates the product, settles positions for the explicitly enumerated `txn.subaccounts`, and optionally closes isolated subaccounts. [4](#0-3) 

However, `delistProduct` does **neither** of the following:
1. Remove `txn.productId` from `productIds` in `PerpEngine`.
2. Clear `productToEngine[txn.productId]` in `Clearinghouse`.

After `delistProduct` returns, `perpEngine.getProductIds()` still returns the delisted product ID, and `clearinghouse.getEngineByProduct(productId)` still resolves to the perp engine.

---

### Impact Explanation

Every function that iterates `productIds` continues to process the delisted product:

**1. `PerpEngineState.updateStates()` — continued funding rate accrual**

The loop iterates `productIds[i]` in lockstep with the caller-supplied `avgPriceDiffs[i]`. [5](#0-4) 

If any subaccount was omitted from `txn.subaccounts` during delisting (the sequencer must enumerate them manually), `state.openInterest != 0` for the delisted product, and the `openInterest == 0` early-exit does not fire. [6](#0-5) 

Funding continues to accrue on those residual positions, corrupting `cumulativeFundingLongX18` / `cumulativeFundingShortX18` for a product that is supposed to be settled.

**2. `Clearinghouse.claimSequencerFees()` — delisted product processed in fee sweep**

`perpEngine.getProductIds()` is called to enumerate products for fee collection. [7](#0-6) 

The delisted product is included in the loop, and any residual `FEES_ACCOUNT` balance for it is transferred to `X_ACCOUNT`. [8](#0-7) 

**3. `Clearinghouse.clearNlpPoolPosition()` — delisted product included in NLP pool clearing**

`perpEngine.getProductIds()` is again used to enumerate all products. [9](#0-8) 

The delisted product's balance for the NLP pool subaccount is transferred to `N_ACCOUNT`, even though the product no longer exists in the protocol's intended state. [10](#0-9) 

**4. Health calculations include delisted product positions**

`getHealthContribution` in `PerpEngine` iterates `productIds` via the bitmap. Any subaccount with a residual position in the delisted product has that position included in `getHealth()`, which is called on every `withdrawCollateral`, `transferQuote`, and liquidation path. [11](#0-10) 

A stale or zero price for the delisted product distorts the health value, potentially enabling under-collateralized withdrawals or triggering incorrect liquidations.

---

### Likelihood Explanation

`delistProduct` is a sequencer-submitted slow-mode transaction. The desynchronization is triggered every time a perp product is delisted. The sequencer must manually enumerate all subaccounts with open positions; any omission leaves residual positions in the delisted product. Given that the `productIds` array is never cleaned up, the impact is permanent for the lifetime of the deployment. The likelihood of at least one subaccount being missed during a real delisting event is non-trivial.

---

### Recommendation

In `delistProduct`, after settling all positions, remove the product from `productIds` in `PerpEngine` (e.g., swap-and-pop) and clear `productToEngine[txn.productId]` in `Clearinghouse`. Alternatively, introduce an explicit `isDelisted` flag per product and guard all `productIds` iteration loops with a check against that flag, mirroring the suggestion made to Holdefi.

---

### Proof of Concept

1. Admin registers perp product `productId = 5`. `productIds = [5]`, `productToEngine[5] = perpEngine`.
2. User A opens a long position in product 5. `balances[5][userA].amount = 100`.
3. Sequencer submits `DelistProduct` for product 5, listing only `[userB]` (omitting `userA`).
4. `delistProduct` zeroes `userB`'s balance. `userA`'s balance remains: `balances[5][userA].amount = 100`.
5. `productIds` still contains `5`. `productToEngine[5]` still resolves to `perpEngine`.
6. Sequencer calls `updateStates` with `avgPriceDiffs = [drift_for_product_5]`. The loop processes `productIds[0] = 5`, `openInterest != 0` (userA's position), funding accrues on a delisted product. [12](#0-11) 
7. `userA` calls `withdrawCollateral`. `getHealth(userA, INITIAL)` includes the delisted product's position at whatever price is currently stored, producing an incorrect health value. [13](#0-12) 
8. Depending on the stale price, `userA` may withdraw more collateral than their true risk permits, or be incorrectly blocked from withdrawing.

    function delistProduct(bytes calldata transaction) external onlyEndpoint {
        IEndpoint.DelistProduct memory txn = abi.decode(
            transaction[1:],
            (IEndpoint.DelistProduct)
        );
        // only perp can be delisted
        require(
            productToEngine[txn.productId] == _perpEngine(),
            ERR_INVALID_PRODUCT
        );
        require(txn.priceX18 == _getPriceX18(txn.productId), ERR_INVALID_PRICE);
        IPerpEngine perpEngine = _perpEngine();
        for (uint256 i = 0; i < txn.subaccounts.length; i++) {
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                txn.productId,
                txn.subaccounts[i]
            );
            int128 baseDelta = -balance.amount;
            int128 quoteDelta = -baseDelta.mul(txn.priceX18);
            perpEngine.updateBalance(
                txn.productId,
                txn.subaccounts[i],
                baseDelta,
                quoteDelta
            );
            if (RiskHelper.isIsolatedSubaccount(txn.subaccounts[i])) {
                IOffchainExchange(
                    IEndpoint(getEndpoint()).getOffchainExchange()
                ).tryCloseIsolatedSubaccount(txn.subaccounts[i]);
            }
        }
    }

    function updateStates(uint128 dt, int128[] calldata avgPriceDiffs)
        external
        onlyEndpoint
    {
        int128 dtX18 = int128(dt).fromInt();
        for (uint32 i = 0; i < avgPriceDiffs.length; i++) {
            uint32 productId = productIds[i];
            State memory state = states[productId];
            if (state.openInterest == 0) {
                continue;
            }
            require(dt < 7 * SECONDS_PER_DAY, ERR_INVALID_TIME);
            {
                int128 indexPriceX18 = _risk(productId).priceX18;

                // cap this price diff
                int128 priceDiffX18 = avgPriceDiffs[i];

                int128 maxPriceDiff = MAX_DAILY_FUNDING_RATE.mul(indexPriceX18);

                if (priceDiffX18.abs() > maxPriceDiff) {
                    // Proper sign
                    priceDiffX18 = (priceDiffX18 > 0)
                        ? maxPriceDiff
                        : -maxPriceDiff;
                }

                int128 paymentAmount = priceDiffX18.mul(dtX18).div(ONE_DAY_X18);
                state.cumulativeFundingLongX18 += paymentAmount;
                state.cumulativeFundingShortX18 += paymentAmount;

                emit FundingPayment(
                    productId,
                    dt,
                    state.openInterest,
                    paymentAmount
                );
            }
            _setState(productId, state);
            emit PriceQuery(productId);
        }
