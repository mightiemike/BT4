Audit Report

## Title
Router-Mediated Swap Bypasses `SwapAllowlistExtension` Per-User Gate — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps on `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender` at the time of the `swap` call. When `MetricOmmSimpleRouter` intermediates the swap, the pool's `msg.sender` is the router, not the end user. Any caller of an allowlisted router address bypasses the per-user allowlist entirely, breaking the pool's curation invariant.

## Finding Description
The call chain is confirmed against production code at each hop:

**Hop 1 — Router calls pool with no caller forwarding:**
`MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` directly. [1](#0-0) 
The original `msg.sender` (the end user/attacker) is stored only in transient callback context for payment purposes; it is never forwarded to the pool as the swap initiator.

**Hop 2 — Pool passes its own `msg.sender` (the router) as `sender` to `_beforeSwap`:**
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` where `msg.sender` is the router. [2](#0-1) 

**Hop 3 — `ExtensionCalling._beforeSwap` forwards the router address as `sender` to the extension:** [3](#0-2) 

**Hop 4 — Extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][attacker]`:** [4](#0-3) 

If the pool admin has allowlisted the router (e.g., `allowedSwapper[pool][router] = true`), the check passes for every caller of the router regardless of whether the actual end user is in `allowedSwapper[pool][attacker]`. There is no existing guard that recovers the true originating caller.

## Impact Explanation
Any unprivileged user can execute swaps on a pool whose admin intended to restrict trading to an explicit per-user allowlist, simply by routing through `MetricOmmSimpleRouter`. The pool's curation invariant — that only explicitly allowlisted end-users may trade — is broken. Disallowed users gain full swap access to a curated pool, enabling them to extract LP value at oracle-derived prices from a pool the admin believed was protected. This constitutes broken core pool functionality causing potential loss of funds for LPs.

## Likelihood Explanation
The pool admin must have allowlisted the router address. This is a natural and expected configuration: an admin who wants to permit router-mediated swaps for their curated users would add the router to `allowedSwapper`. The admin has no mechanism to simultaneously allow router-mediated swaps AND enforce per-user identity checks, because the extension only ever sees the router as `sender`. The vulnerability is structural and not dependent on any misconfiguration a careful admin would avoid.

## Recommendation
`SwapAllowlistExtension.beforeSwap` should check the `recipient` parameter (the economic beneficiary) rather than or in addition to `sender`, or the router should forward the original caller's address through `extensionData` so the extension can gate on the true end user. Alternatively, the extension documentation must explicitly warn that allowlisting any intermediary contract (router, multicall) opens the pool to all callers of that contract.

## Proof of Concept
```solidity
// Pool admin setup — natural action to allow router-mediated swaps:
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// attacker is NOT in allowedSwapper[pool][attacker]

// Attacker action:
vm.prank(attacker);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token0),
    tokenOut: address(token1),
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: attacker,
    deadline: block.timestamp + 1,
    priceLimitX64: 0,
    extensionData: ""
}));
// Swap succeeds: allowedSwapper[pool][router] == true, attacker bypasses the gate
assertFalse(swapExtension.isAllowedToSwap(address(pool), attacker)); // attacker never allowlisted
// but swap completed — invariant violated
```
