### Title
ML-DSA-65 Gas-Key Fee Domain Mismatch: Host-Function Path Uses Borsh Length (1953 B) Instead of Trie-ID Length (33 B) - (`runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The `promise_batch_action_transfer_to_gas_key` and `promise_batch_action_add_gas_key_with_full_access` host functions charge gas fees based on the raw borsh-encoded public key length (`public_key_len as usize`). For an ML-DSA-65 key this is **1953 bytes**. The transaction path for the same actions uses `public_key.trie_id_len()`, which for ML-DSA-65 is **33 bytes** (the SHA3-256 hash form stored in the trie). The two paths diverge by a factor of ~59× on every per-byte fee component, breaking the fee-parity invariant that the codebase explicitly documents and tests.

### Finding Description

`PublicKey` has two length methods that diverge only for ML-DSA-65:

- `len()` → borsh-encoded wire size: 1 + 1952 = **1953 bytes**
- `trie_id_len()` → on-trie hash size: 1 + 32 = **33 bytes** [1](#0-0) 

The documentation explicitly warns: *"Callers that still use `len()` for trie-storage costing will misprice ML-DSA-65 keys by ~1900 bytes."* [2](#0-1) 

The **transaction path** correctly uses `trie_id_len()`:

```rust
// runtime/runtime/src/config.rs line 114-116
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
``` [3](#0-2) 

The **host-function path** uses the raw `public_key_len` argument (the borsh length the contract passes in):

```rust
// runtime/near-vm-runner/src/logic/logic.rs lines 3091-3095
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,
);
``` [4](#0-3) 

The same divergence exists in the wasmtime runner: [5](#0-4) 

And for `promise_batch_action_add_gas_key_with_full_access` / `promise_batch_action_add_gas_key_with_function_call`, which pass `public_key_len as usize` to `gas_key_add_key_exec_fee`: [6](#0-5) 

Compared to the transaction path which uses `trie_id_len()`: [7](#0-6) 

The fee functions themselves are correct — they accept a `public_key_len: usize` parameter and apply it faithfully: [8](#0-7) 

The bug is that the host-function callers supply the borsh length (1953) instead of the trie-id length (33).

### Impact Explanation

For every `TransferToGasKey` or `AddKey`-with-gas-key action emitted by a contract using an ML-DSA-65 public key:

- **Send fee per-byte component**: charged at `gas_key_byte × 1953` instead of `gas_key_byte × 33` — a **~59× overcharge**.
- **Exec fee per-byte component**: `access_key_key_len(account_id_len, 1953)` instead of `access_key_key_len(account_id_len, 33)` — **1920 extra bytes** per nonce entry. [9](#0-8) 

A contract that budgets gas based on the documented fee-parity invariant (confirmed by `test_gas_key_fee_parity`) will exhaust its gas budget when using ML-DSA-65 keys. The fee-parity test only exercises ED25519 keys and does not catch this divergence: [10](#0-9) 

### Likelihood Explanation

`PostQuantumSignatures` and `GasKeys` both stabilize at protocol version 85. From that version onward, any contract can call `promise_batch_action_transfer_to_gas_key` or `promise_batch_action_add_gas_key_with_full_access` with a valid ML-DSA-65 public key. No privileged role is required. The contract simply passes the 1953-byte borsh-encoded key, and the host function charges fees based on that length rather than the 33-byte trie-id length.

### Recommendation

In `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, and `promise_batch_action_add_gas_key_with_function_call` (in both `logic/logic.rs` and `wasmtime_runner/logic.rs`), decode the public key first and then pass `decoded_key.trie_id_len()` to the fee helpers instead of `public_key_len as usize`. The decoded key is already available after `self.get_public_key(...)` / `get_public_key(...)` returns.

Extend `test_gas_key_fee_parity` to cover `KeyType::MLDSA65` so the invariant is enforced for all key types.

### Proof of Concept

For an ML-DSA-65 key (`public_key.len()` = 1953, `public_key.trie_id_len()` = 33):

**Transaction path** (`total_send_fees`):
```
send_fee = gas_key_byte.send_fee(sir) × trie_id_len
         = gas_key_byte.send_fee(sir) × 33
```

**Host-function path** (`promise_batch_action_transfer_to_gas_key`):
```
send_fee = gas_key_byte.send_fee(sir) × public_key_len
         = gas_key_byte.send_fee(sir) × 1953   // 59× larger
```

The exact divergent values are `33` (trie-id hash length) vs `1953` (borsh-encoded ML-DSA-65 pubkey length), a difference of **1920 bytes** per operation, applied to every per-byte fee component. [11](#0-10)

    /// Length of this public key's borsh encoding, in bytes - that is,
    /// the on-the-wire size of the raw key bytes plus a 1-byte borsh
    /// discriminant tag (the leading `+ 1` in each arm).
    ///
    /// For storage-fee accounting use [`PublicKey::trie_id_len`] instead;
    /// for ML-DSA-65 those two diverge (1953 wire vs 33 on-trie).
    // `is_empty` always returns false, so there is no point in adding it
    #[allow(clippy::len_without_is_empty)]
    pub fn len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,
        }
    }

    pub fn empty(key_type: KeyType) -> Self {
        match key_type {
            KeyType::ED25519 => {
                PublicKey::ED25519(ED25519PublicKey([0u8; ed25519_dalek::PUBLIC_KEY_LENGTH]))
            }
            KeyType::SECP256K1 => PublicKey::SECP256K1(Secp256K1PublicKey([0u8; 64])),
            KeyType::MLDSA65 => {
                PublicKey::MLDSA65(MlDsa65PublicKey(Box::new([0u8; ML_DSA_65_PUBLIC_KEY_LENGTH])))
            }
        }
    }

    pub fn key_type(&self) -> KeyType {
        match self {
            Self::ED25519(_) => KeyType::ED25519,
            Self::SECP256K1(_) => KeyType::SECP256K1,
            Self::MLDSA65(_) => KeyType::MLDSA65,
        }
    }

    fn key_tag(&self) -> KeyTag {

            PublicKey::ED25519(_) => KeyTag::Ed25519,
            PublicKey::SECP256K1(_) => KeyTag::Secp256k1,

    /// entry owned by this public key. For ed25519/secp256k1 this matches

            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }

pub fn gas_key_transfer_send_fee(
    cfg: &RuntimeFeesConfig,
    sender_is_receiver: bool,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).send_fee(sender_is_receiver);
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .send_fee(sender_is_receiver)
        .checked_mul(public_key_len as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }
