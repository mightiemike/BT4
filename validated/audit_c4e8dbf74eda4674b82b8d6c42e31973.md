Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender` — the direct caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, the router is `msg.sender` to the pool, so the allowlist checks the router's address rather than the real end user's address. Any non-allowlisted user can bypass the curated-pool restriction by routing through `MetricOmmSimpleRouter` instead of calling the pool directly.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the hook.**

`MetricOmmPool.swap()` calls `_beforeSwap` with `msg.sender` as the first argument: [1](#0-0) 

**Step 2 — SwapAllowlistExtension checks `sender` against the allowlist.**

`SwapAllowlistExtension.beforeSwap()` uses `msg.sender` (the pool) as the mapping key and `sender` (the direct caller of `pool.swap()`) as the identity being gated: [2](#0-1) 

**Step 3 — MetricOmmSimpleRouter calls `pool.swap()` directly, making itself `msg.sender`.**

In `exactInputSingle`, the router calls `pool.swap()` directly. The pool's `msg.sender` is therefore the router, not the end user: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

**Result:** The allowlist check resolves to `allowedSwapper[pool][router]`. For router-mediated swaps to work at all, the pool admin must allowlist the router. Once the router is allowlisted, every user — regardless of their own allowlist status — can call any router function and pass the hook. The `sender` field that `beforeSwap` receives is structurally incapable of identifying the real end user when a router intermediary is present.

## Impact Explanation

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict swaps to KYC'd or otherwise vetted addresses. If the router is allowlisted (the only way to enable router-mediated swaps), any non-allowlisted user can call `MetricOmmSimpleRouter.exactInputSingle()` and swap freely against the pool's liquidity. The allowlist guard is completely defeated. LP funds in the curated pool are exposed to unauthorized swappers, which can result in direct loss of LP principal through adverse-selection trades the pool was specifically configured to prevent. This meets the contest threshold for a High-severity finding: broken core pool access-control functionality causing direct exposure of LP assets to unauthorized actors.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entrypoint. No privileged access, no special setup, and no malicious initial configuration is required. The attacker only needs to call a public router function with a valid swap path. The condition is trivially reachable by any on-chain actor who can read the contract interface.

## Recommendation

The `beforeSwap` hook signature receives `sender` (the pool's `msg.sender`) but not the original transaction initiator. The correct fix is to have the pool record the original transaction initiator via transient storage — analogous to how the router already uses `_setNextCallbackContext` — and expose it to extensions. Alternatively, the router should write the real user into a trusted transient-storage slot before calling `pool.swap()`, and the pool should read that slot and pass it as a verified `originator` parameter to extension hooks. Until fixed, `SwapAllowlistExtension` must not be used with `MetricOmmSimpleRouter` on pools where the allowlist is a security boundary. [5](#0-4) 

## Proof of Concept

```
1. Pool admin deploys a pool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   — required so that any router-mediated swap can pass the hook.
3. Alice (allowlisted) and Bob (NOT allowlisted) both want to swap.
4. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
5. Router calls pool.swap(...) — msg.sender to pool = router.
6. Pool calls _beforeSwap(router, recipient, ...).
7. SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][router] == true → passes.
8. Bob's swap executes successfully despite not being on the allowlist.
9. The curated pool's access control is completely bypassed.
``` [6](#0-5) [7](#0-6)
