### Title
Bonus Reward Checkpoints Advance Without Payment on Insufficient Pool During Redelegation Slash — (`File: x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent chain halt. However, by the time the error is returned, `processEventsAndClaimBonus` has already advanced the position's bonus checkpoints in memory (`LastEventSeq`, `LastBonusAccrual`, `LastKnownBonded`) and decremented event reference counts in the store. `slashRedelegationPosition` then persists the position with these advanced checkpoints via `setPositionWithState`. The result is that the position's accrued bonus rewards are permanently forfeited — not deferred — because the checkpoints that gate future claims have already moved past the unpaid period.

---

### Finding Description

`processEventsAndClaimBonus` performs two categories of work before checking pool sufficiency:

**1. In-memory checkpoint updates (on `pos`):**
- `pos.UpdateLastEventSeq(entry.Seq)` — called inside the event loop
- `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` — called after the loop
- `pos.UpdateLastKnownBonded(bonded)` — called after the loop

**2. Store mutations:**
- `decrementEventRefCount(ctx, valAddr, entry.Seq)` — called inside the event loop, writes to `ValidatorEvents` collection

Only after all of the above does the function check pool balance:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // returns AFTER checkpoints advanced and ref counts decremented
}
``` [1](#0-0) 

In `slashRedelegationPosition`, the `ErrInsufficientBonusPool` case is caught and swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)
    } else {
        return err
    }
}
// pos now has advanced checkpoints but no bonus was paid
...
return k.setPositionWithState(ctx, pos, nil)  // saves advanced checkpoints
``` [2](#0-1) 

For a partial slash (`sharesToUnbond < pos.Delegation.Shares`), `ClearBonusCheckpoints()` is **not** called. The position is saved with `LastBonusAccrual` advanced to `blockTime`, `LastEventSeq` advanced to the latest event, and `LastKnownBonded` updated — all without the corresponding bonus payment. The next `processEventsAndClaimBonus` call will start from the new checkpoint, skipping the forfeited period entirely. [3](#0-2) 

The `decrementEventRefCount` store mutations that already fired inside the loop are also not rolled back, meaning events may be garbage-collected even though the position never received payment for them. [4](#0-3) 

---

### Impact Explanation

A position owner whose validator's redelegation is slashed while the bonus pool is empty permanently loses all accrued bonus rewards for the period `[LastBonusAccrual, blockTime]`. The rewards are not deferred — the checkpoints advance past the period, and no future claim can recover them. The corrupted value is `pos.LastBonusAccrual` and `pos.LastEventSeq` in the `Positions` store, which gate all future bonus computations for that position. [5](#0-4) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions: (1) a tier position with an active redelegation, and (2) the bonus pool being empty or insufficient at the time of the slash. The bonus pool can be empty if it was never funded, was drained by the `BeginBlocker` top-up, or was exhausted by other users claiming rewards. Validator slashing is a normal protocol event. Both conditions can realistically coincide in production, particularly during periods of high claim activity or when the pool is being replenished. [6](#0-5) 

---

### Recommendation

Move `sufficientBonusPoolBalance` check **before** any checkpoint mutations and event reference count decrements in `processEventsAndClaimBonus`. Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, explicitly reset the in-memory `pos` checkpoints to their pre-call values before calling `setPositionWithState`, so the position is saved with its original checkpoints and the forfeited period remains claimable once the pool is replenished. [7](#0-6) 

---

### Proof of Concept

1. User calls `MsgLockTier` to create a tier position on validator V.
2. User calls `MsgTierRedelegate` to redelegate to validator V2; a `RedelegationMapping[unbondingId → positionId]` entry is created.
3. Time passes; bonus accrues on V2.
4. The bonus pool is drained to zero (e.g., by BeginBlocker top-up or other claims).
5. Governance slashes validator V2's redelegation entry. The staking module calls `BeforeRedelegationSlashed(ctx, unbondingId, sharesToUnbond)`.
6. `slashRedelegationPosition` calls `processEventsAndClaimBonus`:
   - Event loop runs: `decrementEventRefCount` fires for each pending event (store mutations committed).
   - `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` in memory.
   - `pos.UpdateLastKnownBonded` updates in memory.
   - `sufficientBonusPoolBalance` fails → returns `ErrInsufficientBonusPool`.
7. `slashRedelegationPosition` catches the error, logs it, and falls through to `setPositionWithState(ctx, pos, nil)`.
8. The position is persisted with `LastBonusAccrual = blockTime` and `LastEventSeq = latestSeq` — no bonus paid.
9. Pool is later replenished. User calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the new checkpoint; the period `[original_LastBonusAccrual, slashBlockTime]` is permanently skipped. User receives zero bonus for that period. [8](#0-7) [9](#0-8)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
