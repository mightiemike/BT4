### Title
Crafted `ClassId` Prefix Bypasses `IsAwayFromOrigin` Check, Enabling Unauthorized NFT Unescrow — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` determines whether to mint a voucher or unescrow an NFT solely by checking whether `data.ClassId` starts with `packet.SourcePort/packet.SourceChannel/`. Because `ClassId` is attacker-controlled application-layer data (not validated by IBC core), a malicious counterparty chain can craft a `ClassId` that begins with its own `sourcePort/sourceChannel/` prefix, forcing `IsAwayFromOrigin` to return `false` and triggering `TransferOwner(escrowAddress, receiver)` on an NFT that was never legitimately returned.

---

### Finding Description

**`IsAwayFromOrigin` logic:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

This returns `false` (i.e., "moving back to origin") if and only if `fullClassPath` starts with `sourcePort/sourceChannel/`. The function is called in `processReceivedPacket` with the packet's source port/channel and the **attacker-supplied** `data.ClassId`: [2](#0-1) 

When `isAwayFromOrigin` is `false`, the code takes the unescrow path: [3](#0-2) 

The escrow address used is derived from the **destination** port/channel (chain B's channel), not the source: [4](#0-3) 

**Attack trace:**

1. Chain B (victim) has `destPort='nft'`, `destChannel='channel-0'`. A legitimate user previously sent native NFT `classID='victimClass'`, `tokenID='token1'` from chain B to chain A via channel-0 → it is now held by `GetEscrowAddress('nft','channel-0')` on chain B.

2. Attacker controls chain A, whose IBC channel to chain B has `sourcePort='nft'`, `sourceChannel='channel-5'` (chain A's channel ID).

3. Attacker submits a packet with `ClassId='nft/channel-5/victimClass'`, `TokenIds=['token1']`, `Receiver=attacker`.

4. On chain B:
   - `IsAwayFromOrigin('nft','channel-5','nft/channel-5/victimClass')` → `!HasPrefix(...)` → `false`
   - `unprefixedClassID = RemoveClassPrefix('nft','channel-5','nft/channel-5/victimClass')` → `'victimClass'`
   - `voucherClassID = ParseClassTrace('victimClass').IBCClassID()` → `'victimClass'` (no path, returns base class ID) [5](#0-4) 

   - `TransferOwner(escrowAddress_channel0, 'victimClass', 'token1', attacker)` is called and **succeeds** because the NFT is genuinely owned by that escrow address. [6](#0-5) 

**`TransferOwner` guard:** [7](#0-6) 

`TransferOwner` checks that `srcOwner` (the escrow address) actually owns the NFT. Since the NFT was legitimately escrowed there, this check passes — the guard does not detect the spoofed ClassId.

**`ValidateBasic` on the packet data:** [8](#0-7) 

`ValidateBasic` only checks that `ClassId` is non-blank, `TokenIds`/`TokenUris` lengths match, and addresses are valid. It does **not** validate that the ClassId prefix corresponds to the actual channel through which the packet arrived.

---

### Impact Explanation

An attacker controlling a counterparty chain can unescrow any NFT held in the destination channel's escrow address without burning the corresponding voucher on the counterparty. This results in:
- The original NFT being transferred to the attacker on chain B
- The voucher on chain A remaining in circulation (double-spend / NFT duplication)
- Complete loss of the escrowed NFT for the legitimate owner

---

### Likelihood Explanation

Preconditions are realistic: a valid IBC channel must exist (standard deployment), the attacker must control the counterparty chain or be able to submit crafted IBC packets (the standard IBC adversary model for cross-chain security), and at least one NFT must be escrowed in the target channel's escrow address (routine after any IBC NFT transfer). No governance, key compromise, or social engineering is required.

---

### Recommendation

In `processReceivedPacket`, before taking the unescrow branch, verify that the `ClassId` prefix in the packet data actually corresponds to the **destination** channel (i.e., the channel on which the packet arrived), not just the source channel. Specifically, the receiving chain should check that `data.ClassId` starts with `GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())` before treating the packet as a return transfer. The correct invariant is:

> A packet may trigger unescrow only if `data.ClassId` begins with `destPort/destChannel/`, proving the NFT was previously sent out of this chain on this exact channel.

The `IsAwayFromOrigin` call should use `packet.GetDestPort()` / `packet.GetDestChannel()` (the local channel identifiers) rather than the counterparty-supplied source port/channel, or equivalently, the ClassId should be validated against the dest prefix before the direction decision is made.

---

### Proof of Concept

```go
// Setup:
// - Channel: destPort="nft", destChannel="channel-0"
//            sourcePort="nft", sourceChannel="channel-5" (attacker's chain)
// - Escrow address = GetEscrowAddress("nft", "channel-0")
// - NFT classID="victimClass", tokenID="token1" owned by escrow address
//   (placed there by a legitimate prior transfer)

craftedPacket := channeltypes.Packet{
    SourcePort:      "nft",
    SourceChannel:   "channel-5",
    DestinationPort: "nft",
    DestinationChannel: "channel-0",
    Data: types.NewNonFungibleTokenPacketData(
        "nft/channel-5/victimClass",  // crafted ClassId
        "",
        []string{"token1"},
        []string{""},
        attackerSender,
        attackerReceiver,
    ).GetBytes(),
}

// IsAwayFromOrigin("nft","channel-5","nft/channel-5/victimClass") == false
// → unprefixedClassID = "victimClass"
// → voucherClassID = "victimClass"
// → TransferOwner(escrowAddr_channel0, "victimClass", "token1", attackerReceiver) succeeds
// Result: attacker receives the escrowed NFT; voucher on chain A still exists
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
```
