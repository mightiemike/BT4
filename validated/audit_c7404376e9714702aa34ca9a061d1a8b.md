Audit Report

## Title
`SwapAllowlistExtension.beforeSwap()` checks the router address instead of the actual user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to `SwapAllowlistExtension.beforeSwap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the allowlist check becomes `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actual_user]`. Any non-allowlisted user can bypass the per-user swap restriction by routing through the public, permissionless router.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` directly as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` forwards that `sender` value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` checks `sender` against the per-pool allowlist: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly without encoding the original caller into `extensionData`: [4](#0-3) 

The exploit path is:
1. Pool admin configures `SwapAllowlistExtension`, allowlists `userA` and the router (required for any router-mediated swap to work).
2. `userB` (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
3. The router calls `pool.swap(...)` — inside the pool, `msg.sender` = router address.
4. `_beforeSwap(sender=router, ...)` is dispatched to the extension.
5. The extension evaluates `allowedSwapper[pool][router] == true` → passes.
6. `userB` receives token output from the pool; the allowlist is silently bypassed.

There is no mechanism in the router to recover or forward the original caller's identity. The `extensionData` field is passed through from the user's call parameters but the extension never decodes it — it only reads `sender`. [5](#0-4) 

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers) loses that restriction entirely for any user routing through `MetricOmmSimpleRouter`. Non-allowlisted users receive real token output from the pool; LP providers bear the counterparty risk of trades they were explicitly configured to reject. This is a direct, fund-impacting bypass of the pool's curation guarantee — the exact wrong value is `allowedSwapper[pool][sender]` resolving to `true` for the router when it should resolve to `false` for the actual economic actor.

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract requiring no special role or privilege. The only precondition is that the pool admin has allowlisted the router address — which is a mandatory operational step for any pool that supports router-mediated swaps. This makes the bypass reachable in any realistic deployment of an allowlisted pool that also supports the periphery router. Any user who knows the pool address can execute this bypass with a single call.

## Recommendation

The extension must check the identity of the economic actor, not the intermediary. The cleanest fix is for the router to encode `msg.sender` (the actual user) into `extensionData` before calling `pool.swap()`, and for `SwapAllowlistExtension.beforeSwap()` to decode and check that address when `sender` is a recognized router address. Alternatively, document and enforce that allowlisted pools must not be used with the public router, and add a pool-creation-time guard to prevent this combination.

## Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension configured
  admin calls setAllowedToSwap(pool, userA, true)   // only userA is allowed
  admin calls setAllowedToSwap(pool, router, true)  // router must be allowed for router swaps

Attack:
  userB (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})

  Router calls (MetricOmmSimpleRouter.sol L72-80):
    pool.swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)
    // msg.sender inside pool = router address

  Pool calls (MetricOmmPool.sol L230-240):
    _beforeSwap(sender=router, ...)

  Extension checks (SwapAllowlistExtension.sol L37):
    allowedSwapper[pool][router] == true  ✓  (admin set this)
    → hook passes, swap executes

  Result:
    userB receives token output from the pool.
    The allowlist that was supposed to block userB is silently bypassed.
    LP providers bear the trade they were configured to reject.
```
