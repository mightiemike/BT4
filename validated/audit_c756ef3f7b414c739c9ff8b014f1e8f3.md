### Title
`fee_proposal_fri` excluded from block hash allows P2P peers to corrupt fee market computation — (File: `crates/starknet_api/src/block_hash/block_hash_calculator.rs`, `crates/apollo_protobuf/src/protobuf/protoc_output.rs`)

---

### Summary

The `fee_proposal_fri` field — the proposer's oracle-derived L2 gas price signal used to drive the SNIP-35 fee market — is stored in the block header and transmitted in `SignedBlockHeader` P2P messages, but is **explicitly excluded from the block hash**. Because consensus signatures in `SignedBlockHeader` cover the block hash (not the proposal commitment), a malicious P2P peer can modify `fee_proposal_fri` in any `SignedBlockHeader` without invalidating the signatures. A syncing node that accepts such a modified header populates its `fee_proposals_window` with attacker-controlled values, causing it to compute a wrong `fee_actual` and therefore a wrong `next_l2_gas_price` for all future blocks it proposes.

### Finding Description

**Step 1 — `fee_proposal_fri` is not in the block hash.**

`gas_prices_to_hash` hashes only `l1_gas_price`, `l1_data_gas_price`, and `l2_gas_price`. A TODO comment explicitly defers adding `l2_gas_consumed` and `next_l2_gas_price`, and `fee_proposal_fri` is never mentioned as a future addition: [1](#0-0) 

`BlockHeaderWithoutHash` carries the field but marks it as outside the hash: [2](#0-1) 

**Step 2 — The protobuf wire format explicitly warns the field is unprotected.**

The `SignedBlockHeader` protobuf carries `fee_proposal_fri` at tag 22 with the comment:

> `WARNING: this field is currently not part of the block hash, so the value must be trusted.` [3](#0-2) 

**Step 3 — The proposal commitment does bind `fee_proposal_fri`, but it is not the block hash.**

`proposal_commitment_from` computes `Poseidon(partial_block_hash, fee_proposal_fri)` and this is what consensus validators sign during the round. However, after consensus the `SignedBlockHeader` carries the *block hash* (not the proposal commitment) together with the consensus signatures. The block hash does not include `fee_proposal_fri`, so the signatures in `SignedBlockHeader` do not protect it. [4](#0-3) 

**Step 4 — `fee_proposals_window` is populated from received block headers.**

During sync, `fee_proposal_fri` from `BlockHeaderWithoutHash` is stored and later read into `fee_proposals_window`. `compute_fee_actual` computes the median of this window: [5](#0-4)

pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

/// Bind `fee_proposal_fri` to the proposal commitment hash.
///
/// Pre-V0_14_3 blocks have `fee_proposal = None` and the commitment is just `partial.0`,
/// preserving on-chain behavior. From V0_14_3 onward, the commitment is
/// `Poseidon(partial.0, fee_proposal_fri)`, so a proposer cannot equivocate on
/// `fee_proposal_fri` without changing the commitment that consensus signs over.
///
/// This computation lives in the orchestrator (not in `starknet_api`) because the
/// fee_proposal is a consensus-time signal — `starknet_api`'s block-hash primitives
/// remain unaware of it, and `BlockHash` is unaffected.
pub(crate) fn proposal_commitment_from(
    partial: PartialBlockHash,
    fee_proposal: Option<GasPrice>,
) -> ProposalCommitment {
    let Some(fee_proposal) = fee_proposal else {
        return ProposalCommitment(partial.0);
    };
    ProposalCommitment(Poseidon::hash_array(&[partial.0, Felt::from(fee_proposal.0)]))
}
