Audit Report

## Title
Off-by-one in `_validateDrawdown` allows `drawdownE6 = 1e6`, collapsing `floorMultiplier` to zero and permanently disabling stop-loss — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`_validateDrawdown` uses a strict `>` comparison, so `drawdownE6 = 1e6` (equal to `E6`) passes validation without reverting. When stored and applied, `floorMultiplier = E6 - drawdown = 0`, making the breach condition `metric < 0` — structurally impossible for `uint256` — so `OracleStopLossTriggered` is never emitted and the stop-loss is silently disabled for all subsequent swaps.

## Finding Description

**Validation off-by-one (L305–307):**

```solidity
function _validateDrawdown(uint256 drawdownE6) private pure {
    if (drawdownE6 > E6) revert OracleStopLossDrawdownTooLarge(drawdownE6);
}
```

`drawdownE6 == E6 == 1e6` satisfies `drawdownE6 > E6 → false`, so no revert occurs. [1](#0-0) 

**Floor multiplier collapses to zero (L234):**

```solidity
uint256 floorMultiplier = E6 - drawdown;   // 1e6 - 1e6 = 0
``` [2](#0-1) 

**Breach check becomes `metric < 0` — always false for `uint256` (L333–335):**

```solidity
if (metric >= hwm) return (metric, false);
breached = metric < (hwm * floorMultiplier) / E6;  // metric < 0 → always false
return (hwm, breached);
``` [3](#0-2) 

**The early-return guard at L217 does not fire:**

```solidity
if (drawdown == 0) return;
```

`drawdown = 1e6 ≠ 0`, so the loop runs across all touched bins but never triggers a revert. [4](#0-3) 

**Attack path:**

1. Pool admin calls `proposeOracleStopLossDrawdown(pool, 1e6)` — passes `_validateDrawdown` due to the off-by-one. [5](#0-4) 
2. Waits for `timelock` seconds. If the pool was initialized with `timelock = 0` (no minimum is enforced by the initializer), `_afterTimelock` returns `block.timestamp`, and `_requireElapsed` passes immediately — propose and execute are atomic in the same block. [6](#0-5) 
3. Calls `executeOracleStopLossDrawdown(pool)` — writes `drawdownE6 = 1e6` to storage. [7](#0-6) 
4. All subsequent swaps execute `_afterSwapOracleStopLoss` with `floorMultiplier = 0`; no breach is ever detected regardless of how much value is drained per bin. [8](#0-7) 

This is a cap failure: the cap is intended to prevent 100% drawdown (`drawdownE6 < E6`), but the off-by-one allows exactly `E6`, which is outside the intended boundary.

## Impact Explanation

`OracleValueStopLossExtension` is the sole on-chain guard preventing per-bin value drain. With `floorMultiplier = 0`, the guard is a no-op: any swap that moves value out of a bin — including a fully draining swap — completes without reverting. LPs who deposited under the assumption that the stop-loss extension protects their principal lose that protection entirely, with no on-chain recourse after the timelock elapses. This constitutes a direct, fund-impacting cap failure by the pool admin, which is within the allowed scope for semi-trusted pool admin boundary analysis.

## Likelihood Explanation

- A pool admin who wishes to disable LP protection has a direct, one-step path: propose `drawdownE6 = 1e6`, wait the timelock, execute.
- If the pool was initialized with `timelock = 0` (permitted by the initializer — no minimum timelock is enforced), the disable is atomic within a single block.
- Even with a non-zero timelock, LPs who are not actively monitoring `OracleStopLossDrawdownProposed` events have no automated protection.
- The condition is repeatable: any pool admin on any pool using this extension can trigger it.

## Recommendation

Change the strict inequality to `>=` in `_validateDrawdown`:

```solidity
function _validateDrawdown(uint256 drawdownE6) private pure {
    if (drawdownE6 >= E6) revert OracleStopLossDrawdownTooLarge(drawdownE6);
}
```

This ensures `floorMultiplier` is always at least 1, keeping the breach check meaningful. Apply the same fix symmetrically to `_validateDecay` if a 100% decay rate is also unintended (currently `decayPerSecondE8 > E8` has the same off-by-one pattern).

## Proof of Concept

```solidity
// Foundry test sketch
function test_drawdown_1e6_disables_stop_loss() public {
    // pool admin proposes 100% drawdown — passes _validateDrawdown due to off-by-one
    vm.prank(poolAdmin);
    ext.proposeOracleStopLossDrawdown(pool, 1e6);

    // skip timelock (or 0 if initialized with timelock=0)
    skip(oracleStopLossConfig.timelock);

    vm.prank(poolAdmin);
    ext.executeOracleStopLossDrawdown(pool);

    // assert config stored
    assertEq(ext.oracleStopLossConfig(pool).drawdownE6, 1e6);

    // floorMultiplier = E6 - 1e6 = 0 → breach check always false
    // perform a fully draining swap — expect NO revert, OracleStopLossTriggered never emitted
    pool.swap(/* drain params */);
    // assertion: swap succeeded, stop-loss never fired
}
```
