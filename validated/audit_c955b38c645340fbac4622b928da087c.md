Audit Report

## Title
SwapAllowlistExtension Gating Bypassed via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` binds to `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The allowlist therefore checks the router's address instead of the actual trader's address, allowing any non-allowlisted user to bypass the swap allowlist by calling the public router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← router address when routed
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this verbatim to every configured extension via `abi.encodeCall`. `SwapAllowlistExtension.beforeSwap` receives `sender` = router and checks:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct) and `sender` is the router (wrong actor). The allowlist lookup is `allowedSwapper[pool][router]`. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly — the pool receives `msg.sender = router`. If the router is allowlisted (or `allowAllSwappers[pool] == true`), every non-allowlisted user gains unrestricted swap access. The symmetric failure also exists: legitimately allowlisted users who route through the router are incorrectly blocked because the router is not on the allowlist.

No existing guard prevents this. `BaseMetricExtension.onlyPool` only verifies the caller is a registered pool — it does not validate which address is passed as `sender`. The extension has no mechanism to recover the original EOA from the router call.

## Impact Explanation
A pool operator deploying `SwapAllowlistExtension` to restrict trading to a curated set of addresses (KYC'd counterparties, whitelisted market makers, protocol-controlled addresses) loses that protection entirely. Any address can execute swaps against the pool's LP liquidity by routing through the public `MetricOmmSimpleRouter`. LP capital is exposed to unrestricted trading by actors the pool operator explicitly intended to exclude. This is a broken core access-control invariant with direct fund-impacting consequences.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entry point. Any user who discovers the allowlist restriction on direct pool calls will naturally try the router. No special privileges, flash loans, or unusual token behavior are required — a standard `exactInputSingle` call suffices. The bypass is trivially repeatable.

## Recommendation
The `SwapAllowlistExtension` must gate the economically relevant actor, not the intermediary. The cleanest fix is to change the allowlist lookup key from `sender` to `recipient` in `SwapAllowlistExtension.beforeSwap`, since `MetricOmmSimpleRouter` always sets `recipient` to the caller's intended output address (the actual user). Alternatively, document that pools using `SwapAllowlistExtension` must not configure the public router as an allowed caller and enforce this in the extension's `initialize` or via a separate router-exclusion check.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured as a `beforeSwap` extension.
2. Add address `A` to the allowlist via `setAllowedToSwap(pool, A, true)`; leave address `B` (attacker) off the allowlist.
3. Confirm: `pool.swap(...)` called directly by `B` reverts with `NotAllowedToSwap`.
4. Call `MetricOmmSimpleRouter.exactInputSingle(...)` from `B` with `recipient = B`.
   - Pool receives the call with `msg.sender = router`.
   - `_beforeSwap(router, B, ...)` is dispatched.
   - `SwapAllowlistExtension` checks `allowedSwapper[pool][router]` — if router is allowlisted or `allowAllSwappers[pool] == true`, the check passes.
   - Swap executes successfully for the non-allowlisted attacker `B`.
5. LP funds are now accessible to any address via the router, defeating the allowlist entirely.
