### Title
`ValidResourceBounds::AllResources` with zero `l2_gas` and `l1_data_gas` silently collapses to `L1Gas` during protobuf deserialization, producing a divergent transaction hash for syncing nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion uses the runtime values of `l2_gas` and `l1_data_gas` to decide which enum variant to reconstruct. When both are zero, it always produces `ValidResourceBounds::L1Gas`, even if the originating node stored and hashed the transaction as `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` includes an extra `L1_DATA_GAS` field in the Poseidon preimage for `AllResources` but omits it for `L1Gas`, the two variants produce different transaction hashes for the same wire bytes. A syncing node that deserializes a block containing such a transaction computes a different `transaction_commitment`, causing block-hash verification to fail.

---

### Finding Description

**Root cause — type-collapse in protobuf deserialization**

`crates/apollo_protobuf/src/converters/transaction.rs` lines 417–436:

```rust
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        ...
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        ...
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)          // ← type collapsed
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
``` [1](#0-0) 

The serializer (`From<ValidResourceBounds> for protobuf::ResourceBounds`) faithfully encodes `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` with all three fields present and zero:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),          // 0
        l1_data_gas: Some(l1_data_gas.into()), // 0
    },
``` [2](#0-1) 

On the receiving side the deserializer sees `l2_gas = 0` and `l1_data_gas = 0` and silently returns `L1Gas(X)` — a different variant.

**How the transaction enters the system with `AllResources`**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`). The gateway conversion always wraps it as `AllResources`:

```rust
impl From<RpcInvokeTransactionV3> for InvokeTransactionV3 {
    fn from(tx: RpcInvokeTransactionV3) -> Self {
        Self {
            resource_bounds: ValidResourceBounds::AllResources(tx.resource_bounds),
            ...
        }
    }
}
``` [7](#0-6) 

A user can legitimately submit an `InvokeV3` transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The proposer hashes it as `AllResources` (hash H₁) and commits the block with a `transaction_commitment` derived from H₁. A syncing peer deserializes the same transaction as `L1Gas(X)`, computes hash H₂ ≠ H₁, and derives a different `transaction_commitment`, causing `calculate_block_hash` to produce a different block hash. [8](#0-7) 

---

### Impact Explanation

A syncing node that receives a block containing an `InvokeV3` transaction with `AllResources { l2_gas: 0, l1_data_gas: 0 }` will:

1. Deserialize the transaction as `L1Gas` (type collapse).
2. Compute a transaction hash that differs from the one the proposer committed.
3. Derive a `transaction_commitment` that does not match the block header.
4. Fail block-hash verification, making the block appear invalid.

This maps to **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result** if the wrong hash propagates into the state.

---

### Likelihood Explanation

- Any user can submit an `InvokeV3` transaction with zero `l2_gas` and zero `l1_data_gas` through the standard RPC gateway.
- The gateway's `validate_tx_l2_gas_price_within_threshold` checks the *price* of L2 gas, not the *amount*; a transaction with `max_amount = 0` passes.
- If the transaction's execution consumes zero L2 gas (e.g., a no-op or a simple storage write under certain versioned-constants configurations), it will be included in a block.
- No special privilege is required; any unprivileged user can craft such a transaction.

---

### Recommendation

Replace the value-based type discrimination with an explicit discriminant field in the protobuf message, or add a boolean `is_all_resources` flag. Alternatively, always deserialize as `AllResources` when all three resource fields are present in the wire message, regardless of their values:

```rust
// Proposed fix: presence of l1_data_gas field (even if zero) signals AllResources
Ok(if value.l1_data_gas.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

This preserves the original variant across a protobuf round-trip and eliminates the hash divergence.

---

### Proof of Concept

```
1. Craft an InvokeV3 RPC transaction:
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
   }

2. Submit to the gateway. The gateway converts it to:
   InvokeTransactionV3 { resource_bounds: AllResources { l1_gas: 1000/1, l2_gas: 0/0, l1_data_gas: 0/0 } }

3. Hash H₁ = get_invoke_transaction_v3_hash(AllResources):
   tip_resource_bounds_hash = Poseidon(tip, L1_GAS_concat, L2_GAS_concat(0), L1_DATA_GAS_concat(0))
   → H₁ includes the L1_DATA_GAS(0) element.

4. Block is committed with transaction_commitment derived from H₁.

5. Syncing peer receives the block via P2P sync.
   protobuf::InvokeV3 → TryFrom → ValidResourceBounds::L1Gas(1000/1)
   (because l1_data_gas.is_zero() && l2_gas.is_zero())

6. Hash H₂ = get_invoke_transaction_v3_hash(L1Gas):
   tip_resource_bounds_hash = Poseidon(tip, L1_GAS_concat, L2_GAS_concat(0))
   → H₂ omits the L1_DATA_GAS element.

7. H₁ ≠ H₂ → transaction_commitment mismatch → block hash mismatch → sync failure.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
