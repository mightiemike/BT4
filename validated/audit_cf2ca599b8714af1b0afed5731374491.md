### Title
`starknet_version` Defaulted Instead of Propagated in `update_state_sync_with_new_block`, Producing Wrong Block Hash Version in Stored `SyncBlock` — (`crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs`)

### Summary

`update_state_sync_with_new_block` constructs a `BlockHeaderWithoutHash` using `..Default::default()` for several fields, including `starknet_version`. The `ProposalInit` value (`init.starknet_version = StarknetVersion::LATEST`) is available in scope but is never assigned. The stored `SyncBlock` therefore carries `StarknetVersion::default()` — a version far below the current protocol — instead of the actual block version. Any downstream consumer that reads `starknet_version` from the stored block (e.g., for `BlockHashVersion` selection or `fee_proposal_required` gating) will operate on the wrong version.

### Finding Description

In `decision_reached`, after a block is finalized, `update_state_sync_with_new_block` is called to persist the new block to the state-sync layer. The function builds a `BlockHeaderWithoutHash` struct:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    block_number: height,
    l1_gas_price,
    ...
    fee_proposal_fri: init.fee_proposal_fri,
    // TODO(guy.f): Figure out where/if to get the values below from and fill them.
    ..Default::default()
};
``` [1](#0-0) 

The `..Default::default()` fills in `starknet_version` (among other fields such as `parent_hash` and `state_root`) with their zero/default values. However, `init.starknet_version` is explicitly set to `StarknetVersion::LATEST` in `initiate_build`:

```rust
starknet_version: starknet_api::block::StarknetVersion::LATEST,
``` [2](#0-1) 

The `init` reference is passed into `update_state_sync_with_new_block` and is used for `init.timestamp`, `init.l1_da_mode`, and `init.fee_proposal_fri`, but `init.starknet_version` is silently dropped.

The `BlockHashVersion` selection is gated on `starknet_version`:

```rust
impl TryFrom<StarknetVersion> for BlockHashVersion {
    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }
}
``` [3](#0-2) 

If `StarknetVersion::default()` resolves to any version below `V0_13_2`, this conversion returns an error. If it resolves to a version in `[V0_13_2, V0_13_4)`, it selects the old `STARKNET_BLOCK_HASH0` constant instead of `STARKNET_BLOCK_HASH1`, producing a structurally different hash preimage.

The `fee_proposal_required` gate in `validate_proposal` is also version-gated:

```rust
let fee_proposal_required = init_proposed.starknet_version >= StarknetVersion::V0_14_3;
``` [4](#0-3) 

If a downstream validator reads `starknet_version` from the stored `SyncBlock` (rather than from the live `ProposalInit`), it would incorrectly conclude `fee_proposal` is not required for V0_14_3+ blocks.

### Impact Explanation

Every block decided by the sequencer is stored in state sync with `starknet_version = StarknetVersion::default()`. Any component that retrieves the block via `get_block()` and uses `block_header_without_hash.starknet_version` for:

- **Block hash calculation** (`calculate_block_hash` → `BlockHashVersion::try_from`) — selects the wrong hash domain constant (`STARKNET_BLOCK_HASH0` vs `STARKNET_BLOCK_HASH1`) or returns an error, producing a wrong or missing block hash. This matches **Critical: wrong state/receipt/block hash**.
- **Fee proposal version gating** — incorrectly treats V0_14_3+ blocks as pre-V0_14_3, bypassing `fee_proposal` bounds enforcement. This matches **High: RPC execution / pending view returns authoritative-looking wrong value**.

The `gas_prices_to_hash` function is also version-gated on `BlockHashVersion`:

```rust
if block_hash_version >= &BlockHashVersion::V0_13_4 {
    // Poseidon hash of all 6 gas price fields
} else {
    // Only 4 flat fields (no L2 gas price)
}
``` [5](#0-4) 

Using the wrong version omits `l2_gas_price_wei` and `l2_gas_price_fri` from the block hash preimage, producing a canonically incorrect hash.

### Likelihood Explanation

This code path executes on every block decision (`decision_reached`). The trigger is unprivileged — any finalized block causes `update_state_sync_with_new_block` to be called. The `init` parameter carrying the correct `starknet_version` is already in scope; the omission is a straightforward missing field assignment, not a design choice (the TODO comment confirms it is unintentional).

### Recommendation

Explicitly assign `starknet_version: init.starknet_version` in the `BlockHeaderWithoutHash` construction inside `update_state_sync_with_new_block`, removing it from the `..Default::default()` fallback:

```rust
let block_header_without_hash = BlockHeaderWithoutHash {
    block_number: height,
    l1_gas_price,
    l1_data_gas_price,
    l2_gas_price,
    l2_gas_consumed: l2_gas_used,
    next_l2_gas_price: self.l2_gas_price,
    sequencer,
    timestamp: BlockTimestamp(init.timestamp),
    l1_da_mode: init.l1_da_mode,
    starknet_version: init.starknet_version,  // ADD THIS
    fee_proposal_fri: init.fee_proposal_fri,
    ..Default::default()
};
```

Similarly, `parent_hash` and `state_root` should be populated once those values are available, as noted by the existing TODO.

### Proof of Concept

1. A sequencer node runs at `StarknetVersion::LATEST` (e.g., `V0_14_4`).
2. A block is decided; `decision_reached` calls `update_state_sync_with_new_block` with `init.starknet_version = V0_14_4`.
3. The stored `SyncBlock` has `block_header_without_hash.starknet_version = StarknetVersion::default()` (e.g., `V0_13_0`).
4. A consumer calls `state_sync_client.get_block(N)` and passes the result to `calculate_block_hash` via `PartialBlockHashComponents::new`.
5. `BlockHashVersion::try_from(V0_13_0)` returns `Err(BlockHashVersion)` since `V0_13_0 < V0_13_2`, or if default resolves to `V0_13_2`/`V0_13_3`, selects `STARKNET_BLOCK_HASH0` and omits `l2_gas_price` from `gas_prices_to_hash`, producing a hash that diverges from the on-chain canonical value. pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}

pub fn gas_prices_to_hash(
