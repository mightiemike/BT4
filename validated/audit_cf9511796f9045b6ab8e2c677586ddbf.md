### Title
Governance Can Overwrite `ExitDuration` via `UpdateTier` Without Checking Active Positions, Locking User Funds for an Unexpectedly Long Period — (File: x/tieredrewards/keeper/msg_server_auth.go)

---

### Summary
`UpdateTier` in `x/tieredrewards` allows the governance authority to change a tier's `ExitDuration` with no check for active positions. Because `TriggerExitFromTier` reads the tier's `ExitDuration` at the moment the user calls it, any user who locked funds before the governance update but has not yet triggered exit will be subject to the new — potentially arbitrarily large — exit duration. This is a direct analog to the reported `setToken` class: a privileged setter overwrites a critical parameter without guarding against existing locked positions, causing those positions to be locked for an unexpected period.

---

### Finding Description

**Root cause — `UpdateTier` overwrites `ExitDuration` unconditionally**

`UpdateTier` in `msg_server_auth.go` only calls `claimRewardsAndUpdateTierPositions` when `BonusApy` changes. No equivalent guard or migration is performed for `ExitDuration` changes; the tier is overwritten unconditionally via `SetTier`. [1](#0-0) 

**Trigger path — `TriggerExitFromTier` reads `ExitDuration` from the live tier at trigger time**

`TriggerExitFromTier` fetches the tier at the moment the user calls it and passes `tier.ExitDuration` directly to `pos.TriggerExit`. The `ExitUnlockAt` timestamp stored in the position is therefore computed from whatever `ExitDuration` is current at trigger time, not from the value in effect when the position was created. [2](#0-1) 

**Contrast with `DeleteTier` — the guard that `UpdateTier` is missing**

`deleteTier` explicitly checks `hasPositions` and returns `ErrTierHasActivePositions` if any positions exist. No equivalent guard exists in `UpdateTier` for `ExitDuration` changes. [3](#0-2) 

**Concrete scenario**

1. Tier T is created with `ExitDuration = 30 days`.
2. User calls `LockTier` and locks bond-denom tokens in tier T.
3. Governance passes `MsgUpdateTier` setting `ExitDuration = 10 years`.
4. User calls `TriggerExitFromTier` → `pos.TriggerExit(blockTime, tier.ExitDuration)` sets `ExitUnlockAt = now + 10 years`.
5. User cannot call `WithdrawFromTier` for 10 years. [4](#0-3) 

---

### Impact Explanation

All positions in the affected tier that have not yet triggered exit are locked for the new `ExitDuration`. If governance sets `ExitDuration` to a value near `math.MaxInt64` nanoseconds, the lock is effectively permanent. The corrupted value is `Position.ExitUnlockAt`, which is set to `now + newExitDuration` instead of `now + originalExitDuration`. The locked assets are the bond-denom tokens held in each position's dedicated delegator account. [5](#0-4) 

---

### Likelihood Explanation

Requires a governance proposal to pass. Any token holder can submit `MsgUpdateTier`; passage requires a majority of voting power. This is a supported production entrypoint (governance transaction path). The risk is realistic in adversarial governance scenarios or accidental misconfiguration by a well-intentioned proposal that does not account for existing positions.

---

### Recommendation

In `UpdateTier`, add a check analogous to `DeleteTier`: if `ExitDuration` is being changed and active positions exist for the tier, either reject the update or snapshot `ExitDuration` into each position at creation time (in `createDelegatedPosition`) and read it from the position rather than from the live tier in `TriggerExitFromTier`. The `claimRewardsAndUpdateTierPositions` pattern already used for `BonusApy` changes demonstrates the intended mitigation approach. [1](#0-0) [6](#0-5) 

---

### Proof of Concept

**Entry path:**
`MsgUpdateTier` → `UpdateTier` (msg_server_auth.go:57) → `SetTier` (tier.go:27) overwrites `ExitDuration` with no position count check.

**Trigger path:**
`MsgTriggerExitFromTier` → `TriggerExitFromTier` (msg_server.go:361) → `ms.getTier(ctx, pos.TierId)` reads the updated tier → `pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)` applies the new duration.

**Corrupted state:**
`Position.ExitUnlockAt` is set to `now + newExitDuration` instead of `now + originalExitDuration`, locking the position's delegated bond-denom tokens for the extended period. [7](#0-6) [2](#0-1)

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

func (ms msgServer) TriggerExitFromTier(ctx context.Context, msg *types.MsgTriggerExitFromTier) (*types.MsgTriggerExitFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateTriggerExit(pos.Position, msg.Owner); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitTriggered{
		PositionId:   pos.Id,
		TierId:       pos.TierId,
		Owner:        pos.Owner,
		ExitUnlockAt: pos.ExitUnlockAt,
	}); err != nil {
		return nil, err
	}

	return &types.MsgTriggerExitFromTierResponse{
		ExitUnlockAt: pos.ExitUnlockAt,
		PositionId:   pos.Id,
	}, nil
}

func (k Keeper) deleteTier(ctx context.Context, tierId uint32) error {
	hasPositions, err := k.hasPositionsForTier(ctx, tierId)
	if err != nil {
		return err
	}
	if hasPositions {
		return types.ErrTierHasActivePositions
	}
	if err := k.Tiers.Remove(ctx, tierId); err != nil {
		if stderrors.Is(err, collections.ErrNotFound) {
			return errorsmod.Wrapf(types.ErrTierNotFound, "tier id %d", tierId)
		}
		return errorsmod.Wrapf(err, "%s (tier id %d)", types.ErrTierStore.Error(), tierId)
	}
	return nil
}

func (k Keeper) createDelegatedPosition(
	ctx context.Context,
	owner string,
	tier types.Tier,
	valAddr sdk.ValAddress,
	delAddr sdk.AccAddress,
	triggerExitImmediately bool,
) (types.Position, error) {
	id, err := k.NextPositionId.Next(ctx)
	if err != nil {
		return types.Position{}, err
	}

	lastEventSeq, err := k.getValidatorEventLatestSeq(ctx, valAddr)
	if err != nil {
		return types.Position{}, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()
	blockHeight := uint64(sdkCtx.BlockHeight())

	pos := types.NewPosition(id, owner, tier.Id, delAddr.String(), blockHeight, lastEventSeq, blockTime, true, blockTime)

	ownerAddr, err := sdk.AccAddressFromBech32(owner)
	if err != nil {
		return types.Position{}, err
	}

	if err := k.routeBaseRewardsToOwner(ctx, delAddr, ownerAddr); err != nil {
		return types.Position{}, err
	}

	if triggerExitImmediately {
		pos.TriggerExit(blockTime, tier.ExitDuration)
	}

	return pos, nil
}
