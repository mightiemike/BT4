Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of end user, enabling full allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router to permit router-mediated swaps, every user on the internet can bypass the allowlist by calling the router directly.

## Finding Description
The call chain is:

1. `MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

2. `ExtensionCalling._beforeSwap` forwards that `sender` verbatim to every configured extension: [2](#0-1) 

3. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router — not the end user: [3](#0-2) 

4. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no user-identity forwarding — the router is `msg.sender` to the pool: [4](#0-3) 

The pool admin faces an impossible choice: if the router is **not** allowlisted, all router-mediated swaps revert for every user including approved ones. If the router **is** allowlisted, any non-approved user bypasses the allowlist by routing through the router. The `DepositAllowlistExtension` correctly avoids this by checking `owner` (the position beneficiary) rather than `sender`: [5](#0-4) 

The protocol's own audit pivot explicitly identifies this as a required invariant: "swap allowlist checks must cover the exact actor/action intended and cannot be bypassed through router." [6](#0-5) 

## Impact Explanation
A curated pool (KYC-only, institutional, or RWA pool) deploys `SwapAllowlistExtension` to restrict trading to approved addresses. The pool admin allowlists the router so approved users can use the standard UX. Any non-approved user calls `MetricOmmSimpleRouter.exactInputSingle` targeting the curated pool. The extension sees `sender = router` (allowlisted) and the swap executes. The non-approved user trades on a pool they were explicitly barred from, draining liquidity or extracting value from LP positions priced for a restricted counterparty set. This is direct unauthorized access to restricted pool assets and constitutes a direct loss of LP principal, qualifying as **High** severity under the contest's "direct loss of user principal above Sherlock thresholds" gate.

## Likelihood Explanation
**High.** `MetricOmmSimpleRouter` is a deployed, public, permissionless contract. No special setup is required beyond knowing the pool address. Any user blocked by the allowlist can trivially re-route through the router. The only precondition — that the router is allowlisted — is the natural operational state for any pool that intends to support router-mediated swaps for its approved users.

## Recommendation
**Short term:** Change `SwapAllowlistExtension.beforeSwap` to check `recipient` if the intent is to gate the output beneficiary, or require the actual user identity to be passed in `extensionData` (encoded by the router) and decoded in the extension.

**Long term:** The router should encode `msg.sender` into `extensionData` before forwarding to the pool, and the extension should verify it. Alternatively, gate by `recipient` (always set by the end user and not spoofable by the router) rather than `sender` (the intermediary contract).

## Proof of Concept
```solidity
// Setup: curated pool with SwapAllowlistExtension
// Admin allowlists the router so approved users can swap via router
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Attack: Bob (NOT allowlisted) calls the router directly
// The extension sees sender = router (allowlisted), not Bob
vm.prank(bob); // bob is NOT in allowedSwapper[pool]
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000_000e6,
        amountOutMinimum: 0,
        recipient:        bob,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// swap succeeds — allowlist completely bypassed
// Bob trades on a pool he was explicitly barred from
```
