### Title
`ValidResourceBounds` Variant Silently Collapses to `L1Gas` During Protobuf Round-Trip, Producing a Divergent Transaction Hash Preimage — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in `crates/apollo_protobuf/src/converters/transaction.rs` silently converts `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` into `L1Gas(l1_gas=X)` during protobuf deserialization. Because `get_tip_resource_bounds_hash` includes `L1_DATA_GAS` in the hash preimage only for the `AllResources` variant, the two variants produce structurally different Poseidon inputs. A transaction whose hash was computed on the originating node under `AllResources` will hash to a different value on any node that deserializes it via the P2P state-sync protobuf path, where the variant collapses to `L1Gas`.
