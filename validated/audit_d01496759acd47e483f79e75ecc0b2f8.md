Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating EOA, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is the pool's own `msg.sender` — the router — not the originating EOA. Any pool that allowlists the router address grants unrestricted swap access to every EOA on the network, completely defeating the allowlist's purpose.

## Finding Description

In `MetricOmmPool.swap()`, the pool passes its own `msg.sender` (the router) as the first argument to `_beforeSwap`: [1](#0-0) 

In `SwapAllowlistExtension.beforeSwap`, the guard checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the router — not the originating EOA: [2](#0-1) 

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly, making itself `msg.sender` from the pool's perspective: [3](#0-2) 

The exploit path is:

```
attacker (non-allowlisted EOA)
  → MetricOmmSimpleRouter.exactInputSingle()
      → pool.swap(recipient, ...)          // msg.sender = router
          → _beforeSwap(msg.sender=router, ...)
              → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                  → allowedSwapper[pool][router] == true  ✓  (passes!)
```

The extension has no mechanism to recover the originating EOA. The `extensionData` passed from the router is `""` (empty) for `exactInputSingle`, so no payer identity is forwarded. [3](#0-2) 

## Impact Explanation

The swap allowlist is the sole curation mechanism restricting who may trade in a pool. A pool admin who allowlists only the router (the natural and expected configuration) inadvertently grants swap access to every EOA. Non-allowlisted users can execute real swaps, moving pool reserves and extracting output tokens. This is a direct, fund-impacting bypass of a core access-control feature, meeting the "broken core pool functionality causing loss of funds" threshold.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the canonical swap entry point. Any pool using `SwapAllowlistExtension` with the router allowlisted is immediately vulnerable to any EOA. No special privileges, malicious setup, or non-standard tokens are required — only a public call to `exactInputSingle`. The condition (router allowlisted) is the natural and expected configuration for any pool using this extension.

## Recommendation

The extension must check the originating actor, not the immediate pool caller. The correct fix is for the router to encode `msg.sender` into `extensionData`, and for the extension to decode and verify it. Specifically:

1. In `MetricOmmSimpleRouter`, encode `msg.sender` into `extensionData` before calling `pool.swap()`.
2. In `SwapAllowlistExtension.beforeSwap`, decode the payer from `extensionData` (the last `bytes calldata` parameter) and check `allowedSwapper[msg.sender][decodedPayer]` instead of `allowedSwapper[msg.sender][sender]`.

This requires a convention between router and extension, but is the only clean solution given the current architecture.

## Proof of Concept

```solidity
function test_allowlistBypass() public {
    // Setup: deploy pool with SwapAllowlistExtension
    // Allowlist only the router, NOT the attacker
    swapExtension.setAllowedToSwap(address(pool), address(router), true);
    // attacker is NOT allowlisted

    // Attacker calls router directly
    vm.prank(attacker);
    router.exactInputSingle(ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        tokenOut: address(token1),
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        recipient: attacker,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    }));
    // Swap succeeds — attacker receives token1 despite not being allowlisted
    // Expected: revert NotAllowedToSwap
}
```

The pool passes `msg.sender` (router) as `sender` to the extension: [4](#0-3) 

The extension checks `allowedSwapper[pool][router]`, which is `true`, so the swap proceeds: [2](#0-1)
