Audit Report

## Title
Swap Allowlist Bypassed via Router: `msg.sender` Identity Substitution Lets Any EOA Swap on Restricted Pools - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument forwarded by the pool, which equals `msg.sender` at the pool call site — the router address, not the originating EOA. If the pool admin allowlists the router to enable periphery-routed swaps, the per-user allowlist check degenerates to a single router-level check that passes for every caller, allowing any EOA to swap on a restricted pool by routing through `MetricOmmSimpleRouter`.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly with `""` as `extensionData`, encoding no originating EOA: [4](#0-3) 

At the pool level `msg.sender` is the router contract. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][EOA]`. The same substitution occurs in `exactInput` (multi-hop), `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to specific counterparties is fully open to any EOA once the router is allowlisted. Non-allowlisted users can execute arbitrary swaps against LP positions priced by an external oracle. Because the pool's liquidity is priced by an external oracle, a non-allowlisted user can execute swaps the LP never consented to, resulting in direct loss of LP principal. This constitutes a critical admin-boundary break and direct loss of user principal above Sherlock thresholds.

## Likelihood Explanation

The pool admin faces an unavoidable dilemma: either allowlist the router (breaking per-user allowlist for all users) or do not (forcing every allowlisted user to call the pool directly, bypassing the periphery entirely). Any deployment that allowlists the router — the expected production configuration — is immediately exploitable by any EOA. The router is a public, permissionless contract requiring no special access or privileged setup.

## Recommendation

`SwapAllowlistExtension.beforeSwap` should gate on the originating EOA, not the intermediate caller. The router should encode `msg.sender` (the EOA) into `extensionData`; the extension should decode and verify it, so the allowlist always gates the economically relevant actor. Concretely: in each router entry point, replace the empty `""` `extensionData` with `abi.encode(msg.sender)`, and in `beforeSwap`, decode the originating address from `extensionData` when `sender` is a known router, then check `allowedSwapper[pool][originatingEOA]`.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   — intended to let allowlisted users swap via the router.
3. Attacker (not in allowlist) calls:
     MetricOmmSimpleRouter.exactInputSingle({
       pool: restrictedPool,
       recipient: attacker,
       ...
     })
4. Router calls pool.swap(attacker, ...) — msg.sender at pool = router.
5. Pool calls _beforeSwap(sender=router, ...).
6. Extension checks allowedSwapper[pool][router] == true → passes.
7. Swap executes; attacker receives output tokens.
   LP suffers unintended trade at oracle price.
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
