Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Real User, Allowing Any Caller to Bypass the Swap Allowlist â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which equals `msg.sender` of the `pool.swap` call. When a user swaps through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap`, so the extension checks whether the **router** is allowlisted rather than the actual user. A pool admin who allowlists the router (the necessary step to let allowlisted users use the standard periphery) inadvertently opens the pool to every unprivileged user who routes through the router.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the `msg.sender` of that call. The original user's identity is stored only in transient callback context (for payment), not forwarded to the extension: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

**Exploit flow:**
1. Pool admin deploys pool with `SwapAllowlistExtension` in `beforeSwap` order.
2. Admin calls `setAllowedToSwap(pool, Alice, true)` â€” Alice is allowlisted.
3. Admin calls `setAllowedToSwap(pool, router, true)` â€” necessary so Alice can use the standard router.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, ...)` â€” router is `msg.sender`.
6. Extension checks `allowedSwapper[pool][router] == true` â†’ passes. Bob's swap executes.

Direct swap by Bob (`Bob â†’ pool.swap`) correctly reverts because `allowedSwapper[pool][Bob] == false`. The bypass requires only that the router be allowlisted, which is the expected operational setup.

## Impact Explanation

The `SwapAllowlistExtension` is the sole on-chain mechanism for restricting swap access to curated pools (KYC-gated, institutional, or permissioned pools). A complete bypass means any unprivileged user can execute swaps against a pool configured to be restricted, exposing LP funds to unrestricted trading and defeating the pool admin's access-control intent. This constitutes broken core pool functionality with direct fund-exposure consequences for LP assets in curated pools â€” a High severity impact under the allowed impact gate.

## Likelihood Explanation

High. `MetricOmmSimpleRouter` is the official periphery swap interface. Any pool admin who wants allowlisted users to use the standard router must allowlist the router address â€” this is the expected operational setup. Once the router is allowlisted, the bypass is trivially reachable by any user with no special privileges, no flash loans, and no complex setup: just calling `exactInputSingle` on the public router.

## Recommendation

The extension must resolve the original user's identity rather than the direct caller of `pool.swap`. The cleanest solution, consistent with the protocol's existing use of EIP-1153 transient storage for callback context, is to have the router write the original `msg.sender` into a transient slot before calling `pool.swap`, and have the extension read that slot. If the slot is set (i.e., the call originates from the router), the extension uses the transient value; otherwise it falls back to `sender`. Alternatively, the router can encode `msg.sender` into `extensionData` under a convention shared with the extension, which decodes and checks that value instead of `sender`.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension in beforeSwap order.
  - Pool admin: setAllowedToSwap(pool, Alice, true)
  - Pool admin: setAllowedToSwap(pool, router, true)  // required for Alice to use router
  - Bob is NOT allowlisted.

Direct swap (blocked correctly):
  Bob â†’ pool.swap(...)
    sender = Bob
    allowedSwapper[pool][Bob] = false â†’ revert NotAllowedToSwap âś“

Router bypass (incorrectly passes):
  Bob â†’ MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    Router â†’ pool.swap(recipient, ...)
      sender = Router
      allowedSwapper[pool][Router] = true â†’ passes âś—
    Bob's swap executes against the restricted pool.
```

A Foundry integration test can confirm this by deploying the pool with the extension, configuring the allowlist as above, and asserting that `exactInputSingle` called by an address not in `allowedSwapper` succeeds when the router is allowlisted.
