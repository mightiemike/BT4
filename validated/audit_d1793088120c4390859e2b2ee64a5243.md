### Title
Malicious P2P Peer Can Misclassify Cairo-1 Classes as Deprecated via `compiled_class_hash=None`, Bypassing State Diff Commitment Verification — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

The `TryFrom<protobuf::StateDiffsResponse>` for `DataOrFin<StateDiffChunk>` uses the sole presence/absence of `compiled_class_hash` to discriminate between `StateDiffChunk::DeclaredClass` (Cairo-1) and `StateDiffChunk::DeprecatedDeclaredClass` (Cairo-0). A malicious p2p peer can send a `DeclaredClass` protobuf message with `compiled_class_hash=None` for a class that is canonically Cairo-1. The p2p sync client's `parse_data_for_block` only validates `state_diff_length` (which is length-invariant across the two variants) and never verifies the resulting `ThinStateDiff` against the `state_diff_commitment` stored in the block header. The wrong state diff is therefore committed to storage unconditionally.

---

### Finding Description

**Step 1 — Converter discriminant** (`crates/apollo_protobuf/src/converters/state_diff.rs`, lines 51–59):

```rust
Some(protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(
    declared_class,
)) => match declared_class.compiled_class_hash.as_ref() {
    Some(_compiled_class_hash) => {
        Ok(DataOrFin(Some(StateDiffChunk::DeclaredClass(declared_class.try_into()?))))
    }
    None => Ok(DataOrFin(Some(StateDiffChunk::DeprecatedDeclaredClass(
        declared_class.try_into()?,
    )))),
},
```

`compiled_class_hash == None` unconditionally produces `DeprecatedDeclaredClass`. There is no cross-check against any canonical source of truth. [1](#0-0) 

**Step 2 — Length invariance** (`crates/starknet_api/src/state.rs`, lines 113–115):

```rust
result += self.class_hash_to_compiled_class_hash.len();
result += self.deprecated_declared_classes.len();
```

Both variants contribute exactly 1 to `ThinStateDiff::len()`. Swapping a `DeclaredClass` for a `DeprecatedDeclaredClass` leaves the total length unchanged, so the `state_diff_length` guard in `parse_data_for_block` passes silently. [2](#0-1) 

**Step 3 — No state diff commitment verification** (`crates/apollo_p2p_sync/src/client/state_diff.rs`, lines 99–107):

```rust
if current_state_diff_len != target_state_diff_len {
    return Err(...WrongStateDiffLength...);
}
validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))
```

After the length check and the duplicate-deprecated-class check, the function returns `Ok`. It never reads `block_header.state_diff_commitment` and never calls `calculate_state_diff_hash` to compare against it. The wrong `ThinStateDiff` is returned as valid. [3](#0-2) 

**Step 4 — Wrong state written to storage** (`crates/apollo_storage/src/state/mod.rs`, lines 638–650):

`append_state_diff` writes `class_hash_to_compiled_class_hash` and `deprecated_declared_classes` directly from the `ThinStateDiff` it receives, with no commitment re-check. [4](#0-3) 

**Step 5 — Hash domain divergence** (`crates/starknet_api/src/block_hash/state_diff_hash.rs`, lines 59–80):

`chain_declared_classes` hashes `(class_hash, compiled_class_hash)` pairs; `chain_deprecated_declared_classes` hashes only `class_hash`. Moving class X from the former to the latter produces a completely different `StateDiffCommitment`, but this is never verified during p2p sync. [5](#0-4)

### Impact Explanation

After the attack, the node's storage contains:

- `deprecated_declared_classes` ∋ X (should be absent)
- `class_hash_to_compiled_class_hash` ∌ X (should map X → compiled_class_hash Y)
- `compiled_class_hash` table has no entry for X

Any subsequent execution that looks up the compiled class hash for X (e.g., a `CALL` or `DEPLOY` targeting class X) will either fail to find a compiled class hash or retrieve a stale/wrong one. The `state_diff_commitment` stored in the header is the canonical correct value, but the actual stored state diff diverges from it — the node is permanently out of sync on this class entry without any self-detection mechanism.

This falls under: **Critical — wrong class hash / wrong compiled class hash / wrong state from execution logic for accepted input.**

---

### Likelihood Explanation

Any node participating in p2p sync is reachable by any peer on the network. No special privilege is required beyond establishing a p2p connection. The attack requires sending a single well-formed protobuf message with one field zeroed. The length check is the only guard and it is trivially satisfied. The probability of a targeted attack by a malicious peer is non-negligible in a permissionless p2p network.

---

### Recommendation

In `parse_data_for_block` (`crates/apollo_p2p_sync/src/client/state_diff.rs`), after assembling the final `ThinStateDiff`, compute `calculate_state_diff_hash(&result)` and compare it against `block_header.state_diff_commitment`. If they differ, return `ParseDataError::BadPeer(...)` and report the peer. This is the canonical guard that the header sync already stores for exactly this purpose.

### Proof of Concept

```rust
// 1. Build a canonical Cairo-1 DeclaredClass protobuf message.
let mut msg = protobuf::DeclaredClass {
    class_hash: Some(X.into()),
    compiled_class_hash: Some(Y.into()), // Cairo-1: non-zero compiled_class_hash
};

// 2. Attacker zeroes the compiled_class_hash field.
msg.compiled_class_hash = None;

// 3. Wrap in StateDiffsResponse and decode.
let response = protobuf::StateDiffsResponse {
    state_diff_message: Some(
        protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(msg)
    ),
};
let chunk: DataOrFin<StateDiffChunk> = response.try_into().unwrap();

// 4. Assert variant is DeprecatedDeclaredClass, not DeclaredClass.
assert!(matches!(
    chunk.0.unwrap(),
    StateDiffChunk::DeprecatedDeclaredClass(DeprecatedDeclaredClass { class_hash })
    if class_hash == X
));

// 5. Feed through unite_state_diffs and assert wrong placement.
let mut thin = ThinStateDiff::default();
unite_state_diffs(&mut thin, chunk.0.unwrap()).unwrap();
assert!(thin.deprecated_declared_classes.contains(&X));
assert!(!thin.class_hash_to_compiled_class_hash.contains_key(&X));

// 6. Confirm state diff commitment diverges from canonical.
let canonical = ThinStateDiff {
    class_hash_to_compiled_class_hash: indexmap!{ X => Y },
    ..Default::default()
};
assert_ne!(
    calculate_state_diff_hash(&thin),
    calculate_state_diff_hash(&canonical)
);
```

fn chain_declared_classes(
    declared_classes: &IndexMap<ClassHash, CompiledClassHash>,
    mut hash_chain: HashChain,
) -> HashChain {
    hash_chain = hash_chain.chain(&declared_classes.len().into());
    for (class_hash, compiled_class_hash) in sorted_index_map(declared_classes) {
        hash_chain = hash_chain.chain(&class_hash).chain(&compiled_class_hash.0)
    }
    hash_chain
}

// Chains: [number_of_old_declared_classes, class_hash_0, class_hash_1, ...].
fn chain_deprecated_declared_classes(
    deprecated_declared_classes: &[ClassHash],
    hash_chain: HashChain,
) -> HashChain {
    let mut sorted_deprecated_declared_classes = deprecated_declared_classes.to_vec();
    sorted_deprecated_declared_classes.sort_unstable();
    hash_chain
        .chain(&sorted_deprecated_declared_classes.len().into())
        .chain_iter(sorted_deprecated_declared_classes.iter().map(|class_hash| &class_hash.0))
}
