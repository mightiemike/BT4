Audit Report

## Title
Swap Allowlist Bypassed via Router: `sender` Bound to Router Address, Not Original User - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is the pool's `msg.sender` — the router contract — not the original EOA. When a pool admin allowlists the router to enable router-mediated swaps, every unprivileged user can bypass the per-user allowlist by routing through `MetricOmmSimpleRouter`, because the extension sees the router address instead of the actual trader.

## Finding Description

**Call path:**

1. User (not in allowlist) calls `MetricOmmSimpleRouter.exactInputSingle(params)`.
2. Router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — so `msg.sender` inside `MetricOmmPool.swap` is the **router address**.
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, passing the router address as `sender`.
4. `ExtensionCalling._beforeSwap` encodes and dispatches `IMetricOmmExtensions.beforeSwap(sender=router, ...)` to the configured extension.
5. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`.

If the pool admin has allowlisted the router (required for any router-mediated swap to succeed), this check passes for **every caller of the router**, regardless of whether the original user is in the allowlist.

**Root cause:** `SwapAllowlistExtension.beforeSwap` receives `sender` as the first argument (the pool's `msg.sender`), but the pool's `msg.sender` is the router, not the originating EOA. There is no mechanism in the extension or the pool to recover the original user's address.

**Existing guards are insufficient:** The only guard is `allowedSwapper[msg.sender][sender]` in `SwapAllowlistExtension.beforeSwap` (line 37). The pool correctly passes `msg.sender` as `sender` (line 231 of `MetricOmmPool.sol`), but that `msg.sender` is the router, not the user. No additional check recovers the original caller.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses loses that restriction entirely for the router path. Any unprivileged user can trade on a supposedly gated pool by calling `MetricOmmSimpleRouter.exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput`. This constitutes a broken core pool functionality (allowlist curation failure) and a direct admin-boundary break: the pool admin's intent to restrict swaps to specific addresses is bypassed by an unprivileged actor using a supported public periphery path.

## Likelihood Explanation
The precondition is that the pool admin allowlists the router address (required for any router-based swap to work at all on an allowlisted pool). Once the router is allowlisted, the bypass is trivially reachable by any user calling any of the four router entry points. No special privileges, flash loans, or multi-block setup are required. The attack is repeatable on every block.

## Recommendation
The extension must gate on the **original user**, not the immediate pool caller. Options:
1. Pass the original `msg.sender` (EOA) through the router as an explicit parameter in `extensionData`, and have `SwapAllowlistExtension` decode and verify it — but this requires the router to be trusted to not forge the value.
2. Require direct pool interaction for allowlisted pools (document that the router is incompatible with `SwapAllowlistExtension`).
3. Introduce a trusted-forwarder pattern where the router appends the original caller and the extension verifies the router's identity before trusting the appended address.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, user NOT allowlisted
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// user is NOT in allowedSwapper

// Direct swap reverts (correct):
vm.prank(user);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(user, true, 1000, type(uint128).max, "", "");

// Router swap succeeds (bypass):
vm.prank(user);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    recipient: user,
    tokenIn: address(token0),
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// swap succeeds — allowlist bypassed
```
