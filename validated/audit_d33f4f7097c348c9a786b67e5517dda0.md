### Title
No Slippage Protection in `mintNlp` and `burnNlp` — (`File: core/contracts/Clearinghouse.sol`)

---

### Summary

The `mintNlp` and `burnNlp` functions in `Clearinghouse.sol` execute NLP token mints and burns at a sequencer-supplied `oraclePriceX18` that is **not included in the user's signed digest**. Users cannot specify a minimum NLP amount to receive when minting, or a minimum quote amount to receive when burning. If the NLP oracle price moves unfavorably between the time the user signs and the time the sequencer processes the transaction, the user silently receives fewer tokens than expected with no on-chain protection.

---

### Finding Description

The `MintNlp` and `BurnNlp` structs that users sign contain only `sender`, the relevant amount, and `nonce`: [1](#0-0) 

The `oraclePriceX18` field lives in the outer `SignedMintNlp`/`SignedBurnNlp` wrappers and is **sequencer-supplied at execution time**, not committed to by the user's signature: [2](#0-1) 

The Verifier confirms `oraclePriceX18` is excluded from the signed digest for both transaction types: [3](#0-2) 

In `mintNlp`, the NLP amount the user receives is computed purely from the sequencer-provided price with no minimum-output guard: [4](#0-3) 

In `burnNlp`, the quote amount the user receives is likewise computed from the sequencer-provided price with no minimum-output guard: [5](#0-4) 

The `ERR_SLIPPAGE_TOO_HIGH` error constant exists in the codebase but is never applied to either of these paths: [6](#0-5) 

---

### Impact Explanation

**Impact: High.**

- **`mintNlp`**: A user signs a transaction to spend a fixed `quoteAmount`. If the NLP oracle price rises between signing and execution, `nlpAmount = quoteAmount / oraclePriceX18` yields fewer NLP tokens than the user anticipated. The user has no on-chain mechanism to enforce a minimum NLP output.
- **`burnNlp`**: A user signs a transaction to burn a fixed `nlpAmount`. If the NLP oracle price falls between signing and execution, `quoteAmount = nlpAmount * oraclePriceX18` yields fewer quote tokens than the user anticipated. The user has no on-chain mechanism to enforce a minimum quote output.

In both cases the asset delta is concrete: the user's quote or NLP balance is reduced beyond what they consented to at signing time.

---

### Likelihood Explanation

**Likelihood: Low.**

The sequencer processes transactions in order and is a trusted party. However, NLP oracle prices are updated via `UpdatePrice` transactions that are also sequenced. A legitimate price update processed immediately before a pending `MintNlp` or `BurnNlp` can cause the user to receive materially fewer tokens. No sequencer compromise is required — normal oracle price movement between signing and execution is sufficient to trigger the loss.

---

### Recommendation

Add a user-specified slippage bound to both structs:

- `MintNlp`: add `uint128 minNlpAmount`. In `mintNlp`, require `nlpAmount >= int128(txn.minNlpAmount)`.
- `BurnNlp`: add `uint128 minQuoteAmount`. In `burnNlp`, require `quoteAmount >= int128(txn.minQuoteAmount)`.

Include these new fields in the EIP-712 signed digest inside `Verifier.sol` so the user's slippage bound is cryptographically committed to and cannot be altered by the sequencer.

---

### Proof of Concept

1. User signs `MintNlp { sender, quoteAmount: 1000e18, nonce }` when NLP oracle price is `10e18` — expecting `100` NLP tokens.
2. Before the sequencer processes this transaction, it processes an `UpdatePrice` that raises the NLP oracle price to `20e18`.
3. `mintNlp` executes: `nlpAmount = 1000e18 / 20e18 = 50` NLP tokens.
4. User receives 50 NLP instead of the expected 100, losing 50% of expected output with no revert or warning.

The same scenario applies symmetrically to `burnNlp` when the oracle price drops between signing and execution. [7](#0-6) [8](#0-7)

    function mintNlp(
        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
