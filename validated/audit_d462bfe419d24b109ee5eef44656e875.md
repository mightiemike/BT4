### Title
`LastKnownBonded` Hardcoded to `true` at Position Creation Allows Bonus Reward Overpayment for Unbonded Validator Period — (`File: x/tieredrewards/keeper/position.go`)

---

### Summary

When a tiered-rewards position is created on a validator that is currently unbonding or unbonded, `createDelegatedPosition` unconditionally initialises `LastKnownBonded = true`. Because `LastEventSeq` is simultaneously set to the validator's current (latest) event sequence, the UNBOND event that caused the validator to be unbonded is permanently skipped. When the validator later re-bonds and the user claims rewards, `processEventsAndClaimBonus` starts the replay with `bonded = true` and computes a full bonus segment covering the entire unbonded gap — a period during which no bonus should have accrued. This drains the governance-funded `BonusPool` with unearned rewards.

---

### Finding Description

**Root cause — `createDelegatedPosition` (`position.go:47–56`)**

```go
lastEventSeq, err := k.getValidatorEventLatestSeq(ctx, valAddr)   // line 47
// ...
pos := types.NewPosition(id, owner, tier.Id, delAddr.String(),
    blockHeight, lastEventSeq, blockTime, true, blockTime)          // line 56
//                                                          ^^^^
//                                              LastKnownBonded hardcoded true
```

`lastEventSeq` is set to the validator's current latest sequence, so all historical events — including the UNBOND event that put the validator into its current unbonded state — are skipped for this position. [1](#0-0) 

**Reward replay — `processEventsAndClaimBonus` (`claim_rewards.go:164–212`)**

```go
bonded      := pos.LastKnownBonded   // line 164 — reads the hardcoded true
segmentStart := pos.LastBonusAccrual // line 165 — position creation time
```

When the validator later emits a BOND event (seq `N+1`), the loop body executes with `bonded = true` and computes:

```
bonus = computeSegmentBonus(pos, tier, position_creation_time, BOND_event_time, rate)
```

This segment spans the entire unbonded gap and is paid out in full. [2](#0-1) 

The code's own comment at line 162–163 acknowledges the invariant that must hold:

> *"Use the persisted bonded state from the last replay, not a hardcoded default. This prevents overpaying bonus for unbonded gaps between claims."*

The comment describes the correct intent, but `createDelegatedPosition` violates it by hardcoding `true`. [3](#0-2) 

**Validator event hooks (`hooks.go:27–49`)**

`AfterValidatorBeginUnbonding` appends an UNBOND event with the pre-unbond `tokensPerShare` and a `ReferenceCount` equal to the number of existing positions. A newly created position sets `LastEventSeq` to the seq of this UNBOND event, so the event is never replayed for the new position. [4](#0-3) 

---

### Impact Explanation

The `BonusPool` (a governance-funded module account) is debited for bonus rewards that were never earned. An attacker who creates a position on an unbonded validator and waits for it to re-bond receives bonus CRO for the full unbonded interval. The amount is proportional to `delegationAmount × tier.BonusApy × unbondedDuration / SecondsPerYear`. With a large delegation and a long unbonded period this can be a material drain on the pool. The corrupted value is the `BonusPool` balance and the position's `LastKnownBonded` checkpoint.

---

### Likelihood Explanation

Validator jailing and unbonding are routine on-chain events. Any unprivileged user can observe that a validator is unbonding (via standard staking queries) and immediately submit `MsgLockTier` or `MsgCommitDelegationToTier` targeting that validator. No special role, leaked key, or social engineering is required. The Cosmos SDK staking module permits delegation to validators in any bonded state.

---

### Recommendation

In `createDelegatedPosition`, query the validator's actual bonded status at position-creation time and use it to initialise `LastKnownBonded`:

```go
val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
if err != nil {
    return types.Position{}, err
}
initialBonded := val.IsBonded()

pos := types.NewPosition(id, owner, tier.Id, delAddr.String(),
    blockHeight, lastEventSeq, blockTime, initialBonded, blockTime)
```

This mirrors the intent stated in the comment at `claim_rewards.go:162–163` and closes the gap between the hardcoded initialisation and the actual validator state.

---

### Proof of Concept

1. Validator `V` is in the active set. `AfterValidatorBeginUnbonding` fires → UNBOND event appended at seq `N`.
2. Attacker calls `MsgLockTier` targeting `V` with a large `Amount`.
   - `createDelegatedPosition` sets `LastEventSeq = N`, `LastKnownBonded = true`.
3. Validator `V` re-enters the active set. `AfterValidatorBonded` fires → BOND event appended at seq `N+1`.
4. Attacker calls `MsgClaimTierRewards`.
   - `processEventsAndClaimBonus` fetches events since seq `N` → returns `[BOND @ N+1]`.
   - `bonded = true` (from `LastKnownBonded`).
   - Loop: computes `bonus = shares × tokensPerShare × tier.BonusApy × (BOND_time − creation_time) / SecondsPerYear`.
   - Bonus is paid from `BonusPool` for the entire unbonded gap.
5. Attacker receives unearned bonus CRO; `BonusPool` is drained by the corresponding amount. [5](#0-4) [6](#0-5) [4](#0-3)

func (k Keeper) createDelegatedPosition(
	ctx context.Context,
	owner string,
	tier types.Tier,
	valAddr sdk.ValAddress,
	delAddr sdk.AccAddress,
	triggerExitImmediately bool,
) (types.Position, error) {
	id, err := k.NextPositionId.Next(ctx)
	if err != nil {
		return types.Position{}, err
	}

	lastEventSeq, err := k.getValidatorEventLatestSeq(ctx, valAddr)
	if err != nil {
		return types.Position{}, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()
	blockHeight := uint64(sdkCtx.BlockHeight())

	pos := types.NewPosition(id, owner, tier.Id, delAddr.String(), blockHeight, lastEventSeq, blockTime, true, blockTime)

	ownerAddr, err := sdk.AccAddressFromBech32(owner)
	if err != nil {
		return types.Position{}, err
	}

	if err := k.routeBaseRewardsToOwner(ctx, delAddr, ownerAddr); err != nil {
		return types.Position{}, err
	}

	if triggerExitImmediately {
		pos.TriggerExit(blockTime, tier.ExitDuration)
	}

	return pos, nil

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

func (h Hooks) AfterValidatorBeginUnbonding(ctx context.Context, _ sdk.ConsAddress, valAddr sdk.ValAddress) error {
	count, err := h.k.getPositionCountForValidator(ctx, valAddr)
	if err != nil {
		return err
	}
	if count == 0 {
		return nil
	}

	tokensPerShare, err := h.k.getTokensPerShare(ctx, valAddr)
	if err != nil {
		return err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	_, err = h.k.appendValidatorEvent(ctx, valAddr, types.ValidatorEvent{
		Height:         sdkCtx.BlockHeight(),
		Timestamp:      sdkCtx.BlockTime(),
		EventType:      types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND,
		TokensPerShare: tokensPerShare,
		ReferenceCount: count,
	})
	return err
