### Title
P2P Protobuf `ValidResourceBounds` Round-Trip Silently Downgrades `AllResources` to `L1Gas`, Producing a Divergent Transaction Hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion in the P2P block-sync path uses a zero-value heuristic to decide between `ValidResourceBounds::L1Gas` and `ValidResourceBounds::AllResources`. A valid V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` survives the gateway, is hashed as `AllResources` (4-element fee preimage), but after protobuf round-trip is silently reconstructed as `L1Gas` (3-element fee preimage). Because `get_tip_resource_bounds_hash` produces structurally different Poseidon inputs for the two variants, the transaction hash computed by a syncing node diverges from the hash committed by the sequencer.

---

### Finding Description

**Serialization path — `From<ValidResourceBounds> for protobuf::ResourceBounds`** [1](#0-0) 

`AllResources(l1=X, l2=0, l1_data=0)` is serialized as `{ l1_gas: Some(X), l2_gas: Some(0), l1_data_gas: Some(0) }` — all three fields present.

**Deserialization path — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [2](#0-1) 

`l1_data_gas` is `Some(0)` → `unwrap_or_default()` → zero. The guard `if l1_data_gas.is_zero() && l2_gas.is_zero()` is true, so the result is `ValidResourceBounds::L1Gas(X)`. The `AllResources` variant is silently lost.

**Hash domain divergence — `get_tip_resource_bounds_hash`** [3](#0-2) 

- `AllResources(l1=X, l2=0, l1_data=0)` → Poseidon hashes **4 elements**: `[tip, pack(l1_gas), pack(l2_gas=0), pack(l1_data_gas=0)]`
- `L1Gas(l1=X)` → Poseidon hashes **3 elements**: `[tip, pack(l1_gas), pack(l2_gas=0)]`

These produce different felt values, so the `tip_resource_bounds_hash` component of the transaction hash differs.

**Transaction hash computation** [4](#0-3) 

The `tip_resource_bounds_hash` is chained directly into the Poseidon hash at position 4. A different value here produces a completely different `TransactionHash`.

**P2P block-sync deserialization entry point** [5](#0-4) 

`TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` calls `ValidResourceBounds::try_from(...)`, which is the conversion above. This is the path used when syncing committed blocks over P2P.

**The gateway accepts the triggering transaction** [6](#0-5) 

A transaction with `AllResourceBounds { l1_gas: NON_EMPTY_RESOURCE_BOUNDS, l2_gas: default, l1_data_gas: default }` passes stateless validation. The gateway computes and stores hash H1 (AllResources, 4-element preimage).

---

### Impact Explanation

A syncing node deserializes the same transaction as `L1Gas` and computes hash H2 ≠ H1. Consequences:

1. **Wrong stored hash**: The syncing node indexes the transaction under H2. `starknet_getTransactionByHash(H1)` returns not-found; `starknet_getTransactionByHash(H2)` returns the transaction — an authoritative-looking wrong RPC value.
2. **Execution divergence**: If the syncing node re-executes the block (e.g., for proof generation or state verification), the account contract's `__validate__` receives H2 as the transaction hash. The user's signature was computed over H1. The signature check fails, the transaction reverts, and the syncing node produces a different state root than the sequencer — a critical state divergence.

This matches: **Critical — Wrong state/receipt/revert result from execution logic for accepted input** and **High — Transaction conversion binds the wrong hash**.

---

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a standard V3 invoke transaction with only L1 gas bounds set (l2_gas and l1_data_gas both zero). This is a common pattern for users who do not use L2 gas or data gas. The gateway accepts it, the block is produced, and every syncing peer is affected.

---

### Recommendation

1. **Remove the zero-value heuristic** in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Instead, add an explicit discriminant field to the protobuf `ResourceBounds` message (e.g., `bool is_all_resources`) so the variant is preserved across serialization.

2. **Alternatively**, enforce that `l1_data_gas` is always present for V3 transactions (the existing TODO comment at line 426 already acknowledges this intent): replace `unwrap_or_default()` with `ok_or(missing("ResourceBounds::l1_data_gas"))?` and always reconstruct as `AllResources`. This is consistent with the proto comment: *"Starting from 0.14.0, MempoolTransaction and ConsensusTransaction shouldn't have None here."* [7](#0-6) 

---

### Proof of Concept

```
1. User constructs InvokeV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      ResourceBounds::default(),   // zero
         l1_data_gas: ResourceBounds::default(),   // zero
     }

2. User signs hash H1 = Poseidon(INVOKE, version, sender,
       Poseidon(tip, pack(l1_gas), pack(l2_gas=0), pack(l1_data_gas=0)),  ← 4 elements
       paymaster_hash, chain_id, nonce, da_mode, ...)

3. Gateway accepts tx, computes same H1, stores it, includes in block.

4. Block propagated over P2P as protobuf:
     ResourceBounds { l1_gas: Some(1000/1), l2_gas: Some(0/0), l1_data_gas: Some(0/0) }

5. Syncing node deserializes:
     l1_data_gas = Some(0) → unwrap_or_default() → 0
     l2_gas = 0
     → l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas(l1_gas)          ← variant changed

6. Syncing node computes H2 = Poseidon(INVOKE, version, sender,
       Poseidon(tip, pack(l1_gas), pack(l2_gas=0)),  ← only 3 elements
       paymaster_hash, chain_id, nonce, da_mode, ...)

7. H1 ≠ H2.
   - RPC on syncing node: getTransactionByHash(H1) → not found
   - Re-execution: account.__validate__(H2) → signature mismatch → revert
   - State root on syncing node ≠ state root on sequencer
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
