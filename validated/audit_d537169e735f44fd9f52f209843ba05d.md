Audit Report

## Title
Swap Allowlist Bypassed via Router — Any User Can Swap on Allowlisted Pools by Routing Through `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed from the pool, which is always `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks the router's address against the allowlist instead of the actual user's address. If the router is allowlisted (the natural operational setup), every unprivileged user can bypass the per-pool swap allowlist by routing through the public router.

## Finding Description

**Root cause — identity mismatch across the call boundary:**

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
  msg.sender,   // <-- whoever called pool.swap()
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol:162-175
abi.encodeCall(
  IMetricOmmExtensions.beforeSwap,
  (sender, ...)   // sender == pool's msg.sender
)
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist, where `msg.sender` is the pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

**Router path — the bypass:**

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    ...
  );
```

When this executes, `msg.sender` inside `MetricOmmPool.swap()` is the **router contract address**, not the original user. The extension therefore evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][user]`.

**Exploit flow:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured and allowlists only specific addresses (e.g., KYC'd traders). The router is allowlisted as a convenience for those traders, or is allowlisted by the pool admin expecting the extension to still gate individual users.
2. An unprivileged user (not in the allowlist) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
3. The router calls `pool.swap()`; the pool passes `address(router)` as `sender` to the extension.
4. The extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
5. The disallowed user successfully swaps on a pool that was supposed to be restricted.

Even if the router is not explicitly allowlisted, the multi-hop `exactInput` path passes `address(this)` (the router) for all intermediate hops, meaning intermediate allowlisted pools are also bypassed for hops 2+.

**Existing guards are insufficient:**

The `onlyPool` modifier on `BaseMetricExtension` only verifies that the caller of `beforeSwap` is a registered pool — it does not verify the identity of the original swap initiator. There is no mechanism in the extension or the pool to recover the original `tx.origin` or a user-supplied identity in a trustworthy way.

## Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., for regulatory compliance, private market making, or beta access) loses that restriction entirely for any user who routes through the public `MetricOmmSimpleRouter`. The allowlist policy is silently nullified. This constitutes a broken core pool functionality and an admin-boundary break where an unprivileged path bypasses a configured access control, directly enabling unauthorized trading on restricted pools.

## Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary public swap entrypoint documented in the periphery. Any user aware of the router can exploit this without any special privileges, capital, or timing. The condition is: (1) a pool uses `SwapAllowlistExtension`, and (2) the router is allowlisted or the attacker targets intermediate hops. Both conditions are the natural production configuration. The bypass is repeatable on every swap.

## Recommendation

The extension must gate the **original user**, not the intermediary. Options:

1. **Pass the original initiator explicitly**: Add a trusted `initiator` field to `extensionData` that the router populates with `msg.sender` before calling the pool, and have the extension decode and check that field. This requires the pool admin to trust the router to set it honestly, which is acceptable since the router is a known periphery contract.
2. **Check `tx.origin` as a fallback**: Only viable if the protocol explicitly restricts usage to EOAs, which is not the current design.
3. **Preferred — allowlist the router separately and require user identity in `extensionData`**: The pool admin allowlists only specific users directly; the router is never allowlisted. Users who want to use the router must call the pool directly or the router must forward a signed user identity that the extension verifies.
4. **Document and enforce direct-pool-only access for allowlisted pools**: If the router cannot be fixed, document that pools using `SwapAllowlistExtension` must not allowlist the router and must require direct `pool.swap()` calls.

## Proof of Concept

```solidity
// Foundry test sketch
function test_allowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, only `allowedUser` is allowlisted
    address allowedUser   = makeAddr("allowed");
    address disallowedUser = makeAddr("disallowed");

    vm.prank(poolAdmin);
    swapAllowlist.setAllowedToSwap(address(pool), allowedUser, true);
    // Router is also allowlisted (natural setup so allowedUser can use it)
    vm.prank(poolAdmin);
    swapAllowlist.setAllowedToSwap(address(pool), address(router), true);

    // disallowedUser calls pool directly → reverts correctly
    vm.prank(disallowedUser);
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    pool.swap(disallowedUser, true, 1e18, 0, "", "");

    // disallowedUser routes through router → succeeds (bypass)
    vm.prank(disallowedUser);
    // This should revert but does NOT — the extension sees router as sender
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            recipient: disallowedUser,
            tokenIn: token0,
            zeroForOne: true,
            amountIn: 1e18,
            amountOutMinimum: 0,
            priceLimitX64: 0,
            deadline: block.timestamp,
            extensionData: ""
        })
    );
    // disallowedUser received tokens — allowlist bypassed
}
```
