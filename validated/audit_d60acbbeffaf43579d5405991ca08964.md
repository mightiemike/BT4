Audit Report

## Title
Pool Admin Can Reduce Timelock to Zero and Immediately Execute Drawdown Changes — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension` timelocks drawdown/decay parameter changes so LPs can react. However, neither `initialize` nor `proposeOracleStopLossTimelock` enforce a minimum timelock floor, allowing the pool admin to reduce the timelock to `0`. Once set to `0`, `_afterTimelock` returns exactly `block.timestamp`, causing `_requireElapsed` to pass immediately, so propose and execute of a drawdown increase can be batched in a single transaction — giving LPs zero seconds to react.

## Finding Description

`_afterTimelock` computes the execution timestamp: [1](#0-0) 

When `oracleStopLossConfig[pool_].timelock == 0`, this returns `uint32(block.timestamp + 0) = block.timestamp`.

`_requireElapsed` then checks: [2](#0-1) 

`block.timestamp < block.timestamp` is always `false`, so the revert never fires. Propose and execute can occur in the same block.

`initialize` calls `_validateDrawdown` and `_validateDecay` but has no `_validateTimelock`, accepting any `uint32 timelock` including `0`: [3](#0-2) 

`proposeOracleStopLossTimelock` similarly accepts any `uint32 newTimelock` with no lower-bound check: [4](#0-3) 

**Attack sequence:**

1. Admin calls `proposeOracleStopLossTimelock(pool, 0)` — `pendingTimelockExecuteAfter = block.timestamp + T`.
2. Admin waits `T` seconds (one-time cost).
3. Admin calls `executeOracleStopLossTimelock(pool)` — `oracleStopLossConfig[pool].timelock = 0`.
4. Admin calls `proposeOracleStopLossDrawdown(pool, 1_000_000)` — `pendingDrawdownExecuteAfter = block.timestamp + 0 = block.timestamp`.
5. Admin calls `executeOracleStopLossDrawdown(pool)` in the **same block** — `_requireElapsed(block.timestamp)` passes, drawdown is updated immediately. [5](#0-4) 

Steps 4 and 5 can be batched in a single transaction. With `drawdownE6 = E6 = 1_000_000`, `floorMultiplier = E6 - drawdown = 0`, so `_applyWatermark` computes `breached = metric < (hwm * 0) / E6 = 0`, which is always `false` for a `uint256`, disabling the stop-loss entirely. [6](#0-5) 

## Impact Explanation

The NatDoc invariant states: *"Drawdown and decay changes are timelocked so LPs can react."* [7](#0-6) 

Once the timelock is `0`, the admin can raise `drawdownE6` to `E6` (100%) atomically, setting `floorMultiplier = 0` and making `breached` permanently `false`. All subsequent swaps that would have been blocked by the stop-loss guard now execute freely, allowing value to drain from the pool without LP recourse. This is a direct admin-boundary break: the pool admin bypasses the timelock that is the sole LP protection mechanism, constituting a High-severity loss of LP principal protection.

## Likelihood Explanation

The pool admin is a real, non-factory role assigned per pool at creation, confirmed by the `onlyPoolAdmin` modifier: [8](#0-7) 

The attack requires only a malicious or compromised pool admin. The one-time cost (waiting the initial timelock `T`) is observable on-chain, but LPs who do not monitor continuously will miss the subsequent same-block drawdown change. Steps 4–5 are atomic and produce no advance warning.

## Recommendation

Add a minimum timelock floor in a `_validateTimelock` helper and call it from both `initialize` and `proposeOracleStopLossTimelock`:

```solidity
uint32 private constant MIN_TIMELOCK = 1 hours;

function _validateTimelock(uint32 timelock) private pure {
    if (timelock < MIN_TIMELOCK) revert OracleStopLossTimelockTooShort(timelock);
}
```

Call `_validateTimelock(timelock)` alongside `_validateDrawdown` and `_validateDecay` in `initialize`: [9](#0-8) 

And call `_validateTimelock(newTimelock)` at the top of `proposeOracleStopLossTimelock`: [10](#0-9) 

## Proof of Concept

```solidity
function test_timelockZeroBypass() public {
    uint32 initialTimelock = 1 days;
    // Pool initialized with timelock = 1 day, drawdownE6 = 100_000 (10%)

    // Step 1: propose timelock = 0
    vm.prank(admin);
    extension.proposeOracleStopLossTimelock(pool, 0);

    // Step 2: wait the existing timelock (one-time cost)
    vm.warp(block.timestamp + initialTimelock);

    // Step 3: execute timelock = 0
    vm.prank(admin);
    extension.executeOracleStopLossTimelock(pool);
    assertEq(extension.oracleStopLossConfig(pool).timelock, 0);

    // Step 4+5: propose and execute drawdown increase atomically (same block)
    vm.prank(admin);
    extension.proposeOracleStopLossDrawdown(pool, 1_000_000); // 100% drawdown = disabled
    vm.prank(admin);
    extension.executeOracleStopLossDrawdown(pool); // no revert — _requireElapsed(block.timestamp) passes

    assertEq(extension.oracleStopLossConfig(pool).drawdownE6, 1_000_000);
    // LPs had 0 seconds to react; stop-loss is now permanently disabled
}
