Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as Swapper Instead of Original User, Enabling Allowlist Bypass or Breaking Standard Swap Path — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

When a swap is routed through `MetricOmmSimpleRouter`, `MetricOmmPool.swap()` receives the router contract as `msg.sender` and passes it verbatim as the `sender` argument to `_beforeSwap`. `SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][originalUser]`. This produces two mutually exclusive broken states: either the router is not allowlisted (breaking the standard periphery path for all allowlisted users) or the router is allowlisted (allowing any public user to bypass the guard entirely).

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` directly as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value verbatim to every configured extension via `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` gates access by checking `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the forwarded value: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly: [4](#0-3) 

At that point `msg.sender` inside `pool.swap()` is the router contract address. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. The router passes `params.extensionData` opaquely and `SwapAllowlistExtension` does not parse `extensionData` at all, so there is no existing mechanism to recover the original caller. [5](#0-4) 

The pool admin configures the allowlist per-user via `setAllowedToSwap`, expecting individual user addresses to be checked — not intermediary contracts.

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to specific addresses (KYC-verified counterparties, whitelisted market makers, protocol-owned addresses) cannot simultaneously allow those users to use `MetricOmmSimpleRouter` and exclude everyone else. This is broken core pool functionality: either the standard periphery swap path is completely unusable for all allowlisted users (router not allowlisted), or the allowlist financial control is fully bypassed by any public user routing through the router (router allowlisted). Both outcomes directly impact fund-impacting policy controls on pools where the allowlist restricts who can drain liquidity via swaps.

## Likelihood Explanation

Any pool that deploys `SwapAllowlistExtension` and expects users to interact via `MetricOmmSimpleRouter` (the documented standard periphery path) will immediately encounter this. No special privileges are required: any public user can call the router. The pool admin has no on-chain recourse other than requiring all users to call the pool directly, bypassing the supported periphery entirely.

## Recommendation

`SwapAllowlistExtension.beforeSwap` must gate on the original end-user address rather than the intermediate router. Two approaches:

1. **Pass the original caller through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. Requires coordinated changes in both the router and the extension.
2. **Transient storage**: The pool already uses transient storage for reentrancy guards; it could store the original initiator there, and the extension reads it directly.

## Proof of Concept

```
1. Deploy a pool with SwapAllowlistExtension as a before-swap hook.
2. Pool admin calls setAllowedToSwap(pool, alice, true).
   Alice is the only allowlisted swapper.
3. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...).
   - Router calls pool.swap(recipient, ...) — msg.sender inside pool.swap() is the router.
   - pool.swap() passes msg.sender = router to _beforeSwap.
   - SwapAllowlistExtension checks allowedSwapper[pool][router].
   - If router is allowlisted: Bob's swap succeeds — allowlist bypassed.
   - If router is not allowlisted: Bob's swap reverts correctly.
4. Alice (allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...).
   - Same path: extension checks allowedSwapper[pool][router], not allowedSwapper[pool][alice].
   - Alice's swap reverts even though she is explicitly allowlisted.
=> Either the allowlist is bypassable (router allowlisted) or the standard swap path
   is broken for legitimate users (router not allowlisted).
```
