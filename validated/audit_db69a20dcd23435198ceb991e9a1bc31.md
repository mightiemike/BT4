Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the actual end-user, allowing any user to bypass the swap allowlist via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension` is documented to gate swaps by swapper address per pool, but its `beforeSwap` hook receives `msg.sender` of `MetricOmmPool.swap` as the `sender` argument. When swaps are routed through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the end-user. Any pool admin who allowlists the router (required for legitimate users to use the standard UX path) inadvertently grants every address on the network the ability to bypass the per-user allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` verbatim as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` argument is allowlisted for the calling pool (`msg.sender` inside the extension = the pool): [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router contract the `msg.sender` inside `pool.swap`: [3](#0-2) 

The actual end-user (`msg.sender` of `exactInputSingle`) is stored only in transient storage for the payment callback and is never forwarded to any extension hook. The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

When the router is allowlisted, `allowedSwapper[pool][router] == true`, so the check at line 37 of `SwapAllowlistExtension` passes for every caller who routes through the router, regardless of whether that caller is individually allowlisted.

## Impact Explanation
A pool admin deploying `SwapAllowlistExtension` to restrict swapping to a curated set of addresses must also allowlist the router for those addresses to use the standard periphery UX. The moment the router is allowlisted, the per-user restriction is completely nullified: any unprivileged address can call `exactInputSingle` and pass the allowlist check because the extension sees `sender = router`. Unauthorized traders can execute swaps on a pool intended to be private, extracting LP-side value and causing direct loss of LP principal and owed fees. This is a broken core pool functionality / direct loss of user principal impact.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard periphery swap entry point. Any pool admin who enables `SwapAllowlistExtension` and also wants legitimate allowlisted users to use the router will inevitably allowlist the router, triggering the bypass. No special privilege is required: a single call to `exactInputSingle` with valid swap parameters suffices. The condition is self-inflicted by normal, expected admin behavior.

## Recommendation
The extension must verify the economic actor, not the direct caller of `pool.swap`. The preferred fix is to have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it when the direct `sender` is a known router. A more robust solution is to redesign the hook signature to include an `originalSender` field populated by the pool from a trusted periphery context or signed permit embedded in `extensionData`. At minimum, the `SwapAllowlistExtension` NatSpec must document that allowlisting any intermediary contract grants that contract's entire user base access, and that the extension only provides meaningful access control for direct `pool.swap` callers.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension.
  - Admin calls setAllowedToSwap(pool, alice, true).
  - Admin calls setAllowedToSwap(pool, router, true)  // required for router UX.
  - bob is NOT allowlisted.

Attack:
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool, ...}).
  2. Router calls pool.swap(recipient=bob, ...) — msg.sender inside pool.swap = router.
  3. pool._beforeSwap(sender=router, ...) is called.
  4. SwapAllowlistExtension.beforeSwap receives sender=router.
  5. allowedSwapper[pool][router] == true → check passes.
  6. bob's swap executes; allowlist is bypassed.

Result: bob, a non-allowlisted address, successfully swaps on a pool
        intended to be restricted, extracting value from LPs.
```
