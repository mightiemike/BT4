Audit Report

## Title
Router-Mediated Swaps Pass Router Address as Swapper to `SwapAllowlistExtension`, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension::beforeSwap` gates swaps by checking the `sender` argument against `allowedSwapper[pool][sender]`. When a swap is routed through `MetricOmmSimpleRouter`, the pool receives the router contract as `msg.sender` and forwards it as `sender` to the extension. The original user's address is never passed into the pool's `swap` call. A pool admin who allowlists the router — a natural action to enable router-based swaps — inadvertently opens the pool to all users, completely defeating the allowlist.

## Finding Description

In `MetricOmmPool::swap`, the pool passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling::_beforeSwap` forwards this value unchanged as `sender` to the extension: [2](#0-1) 

`SwapAllowlistExtension::beforeSwap` then checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

In `MetricOmmSimpleRouter::exactInputSingle`, the original `msg.sender` is stored only in transient storage for the payment callback and is never passed into the pool's `swap` arguments: [4](#0-3) 

When the router calls the pool, `msg.sender` to the pool is the router contract. So `sender` received by the extension is the router address, not the original user. If the pool admin has called `setAllowedToSwap(pool, router, true)`, then `allowedSwapper[pool][router] = true` and the check passes for **any** user routing through the router, regardless of whether that user is individually allowlisted. [5](#0-4) 

## Impact Explanation

`SwapAllowlistExtension` is the primary access-control mechanism for restricting which addresses may swap in a pool. Allowlisting the router (to enable router-based swaps for permitted users) causes the extension to treat every user routing through the router as the router itself, which is allowlisted. Any unprivileged user can bypass the allowlist entirely by calling `MetricOmmSimpleRouter::exactInputSingle` targeting a restricted pool. This constitutes broken core pool functionality and, for pools designed as private venues (e.g., for specific market makers), enables unauthorized trades with direct fund impact through unwanted price impact or liquidity drain.

## Likelihood Explanation

Mode A (allowlist bypass) requires the pool admin to allowlist the router address. This is a natural and expected action — any admin who wants their allowlisted users to be able to use the standard router will do exactly this. There is no warning in the code or interface that allowlisting the router opens the pool to all users. `setAllowedToSwap` treats the router address identically to any other address. Once the router is allowlisted, the bypass is unconditional and repeatable by any caller.

## Recommendation

The extension should not rely solely on the `sender` argument passed from the pool. Options include:

1. **Pass the original user through `extensionData`:** The router encodes the original `msg.sender` into `extensionData`, and the extension decodes and verifies it (requires trust in the router, which can be checked against the factory registry).
2. **Check both `sender` and `recipient`:** Gate on the recipient address as a proxy for the intended beneficiary, if the pool design guarantees recipient = user.
3. **Document the limitation clearly:** If the design intent is to gate only the immediate pool caller, document that the extension is incompatible with router-mediated swaps and that allowlisting the router bypasses per-user restrictions.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension configured in beforeSwap slot.
2. Pool admin calls setAllowedToSwap(pool, alice, true)  — only Alice is allowed.
3. Pool admin calls setAllowedToSwap(pool, router, true) — to let Alice use the router.
4. Bob (not allowlisted) calls MetricOmmSimpleRouter::exactInputSingle with the pool.
5. Router calls pool.swap(...) with msg.sender = router.
6. beforeSwap receives sender = router; allowedSwapper[pool][router] = true → passes.
7. Bob's swap executes successfully despite not being allowlisted.
```
