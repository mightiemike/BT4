Based on my investigation, I found a concrete hash-domain divergence analog to the DODO ETH/WETH representation mismatch.

### Title
`ResourceBoundsMapping` → `ValidResourceBounds::L1Gas` Conversion Silently Produces Wrong Transaction Hash for V3 Transactions with Zero L2/Data Gas Bounds - (`crates/apollo_rpc/src/v0_8/transaction.rs`)

### Summary

The `From<ResourceBoundsMapping> for ValidResourceBounds` conversion in the RPC v0_8 layer silently downgrades a post-0.13.3 `AllResources` transaction to the pre-0.13.3 `L1Gas` variant whenever `l1_data_gas == 0` and `l2_gas == 0`. Because `get_tip_resource_bounds_hash` produces structurally different Poseidon preimages for the two variants, the hash computed inside the RPC layer diverges from the hash the signer computed — breaking signature verification in `starknet_simulateTransactions` / `starknet_estimateFee` (with `skip_validate = false`) for any valid V3 transaction that happens to set both optional gas fields to zero.

**Hash function treats the two variants differently**

`get_tip_resource_bounds_hash` always emits `[tip, L1_GAS_entry, L2_GAS_entry]` for `L1Gas`, but emits `[tip, L1_GAS_entry, L2_GAS_entry, L1_DATA_GAS_entry]` for `AllResources`: [3](#0-2) 

Concretely, for a transaction with `l1_gas = X`, `l2_gas = 0`, `l1_data_gas = 0`:

| Variant | Poseidon preimage elements |
|---|---|
| `L1Gas(X)` | `tip ‖ concat(L1_GAS, X) ‖ concat(L2_GAS, 0)` |
| `AllResources{X,0,0}` | `tip ‖ concat(L1_GAS, X) ‖ concat(L2_GAS, 0) ‖ concat(L1_DATA_GAS, 0)` |

These produce **different Poseidon digests** — the `AllResources` hash has one extra field in the preimage.

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
```
