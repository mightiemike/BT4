### Title
Governance can set `ExitDuration` to an arbitrarily large value, permanently trapping user funds in the `x/tieredrewards` module - (File: x/tieredrewards/types/tier.go)

### Summary
`Tier.Validate()` enforces only that `ExitDuration > 0`, with no upper-bound check. A governance proposal using `MsgUpdateTier` can set `ExitDuration` to an astronomically large value (e.g., `math.MaxInt64` nanoseconds ≈ 292 years). Any position that subsequently calls `MsgTriggerExitFromTier` will have its `ExitUnlockAt` set to `blockTime + ExitDuration`, making `CompletedExitLockDuration()` return `false` for centuries. All exit paths (`MsgTierUndelegate`, `MsgExitTierWithDelegation`, `MsgWithdrawFromTier`) gate on this check, permanently blocking fund recovery for affected positions.

### Finding Description

`Tier.Validate()` in `x/tieredrewards/types/tier.go` only checks that `ExitDuration` is strictly positive:

```go
if t.ExitDuration <= 0 {
    return fmt.Errorf("exit duration must be positive")
}
```

There is no maximum bound. A governance proposal (`MsgUpdateTier`) can set `ExitDuration` to any positive value, including `time.Duration(math.MaxInt64)` (≈ 292 years).

When a user calls `MsgTriggerExitFromTier`, the keeper calls `pos.TriggerExit(blockTime, tier.ExitDuration)`:

```go
func (p *Position) TriggerExit(blockTime time.Time, duration time.Duration) {
    p.ExitTriggeredAt = blockTime
    p.ExitUnlockAt = blockTime.Add(duration)
}
```

`ExitUnlockAt` is then set to a timestamp centuries in the future. All three exit paths check `CompletedExitLockDuration`:

- `validateUndelegatePosition` → `ErrExitLockDurationNotReached`
- `validateExitTierWithDelegation` → `ErrExitLockDurationNotReached`
- `validateWithdrawFromTier` → `ErrExitLockDurationNotReached`

The only escape valve, `MsgClearPosition`, is **blocked** when the tier is `CloseOnly`:

```go
if tier.IsCloseOnly() {
    return types.ErrTierIsCloseOnly
}
```

A governance proposal that simultaneously sets `ExitDuration` to a huge value **and** `CloseOnly = true` leaves all positions that have already triggered exit permanently unable to exit, and all positions that have not yet triggered exit unable to cancel (since `ClearPosition` is blocked by `CloseOnly`). Even without `CloseOnly`, positions that have already triggered exit with the new `ExitUnlockAt` cannot use `ClearPosition` after exit has elapsed (it requires `IsDelegated()` and not unbonding), and positions mid-exit cannot use any exit path.

Critically, `ExitUnlockAt` is **stamped at trigger time** using the tier's `ExitDuration` at that moment. A governance update to `ExitDuration` does not retroactively fix already-triggered positions — their `ExitUnlockAt` is already written to state.

### Impact Explanation

**Impact: High.** User principal (delegated tokens) is permanently locked inside the `x/tieredrewards` module. The position's delegator address holds the staking delegation; the owner cannot recover it via any supported message path. The corrupted value is `Position.ExitUnlockAt` — set to a timestamp centuries in the future — which blocks all three exit paths. Affected users lose access to their locked tokens indefinitely.

### Likelihood Explanation

**Likelihood: Low.** Requires a governance proposal to pass. However, the analog to the original report is exact: a parameter with no upper bound that, if set to an extreme value, causes irreversible user-fund lockup. The default `ExitDuration` values in genesis are reasonable (5s–60s in tests, 1 year in production intent), but nothing in the on-chain validation prevents a malicious or erroneous governance proposal from setting it to `math.MaxInt64`.

### Recommendation

Add a maximum allowed `ExitDuration` in `Tier.Validate()`, for example:

```go
const MaxExitDuration = 10 * 365 * 24 * time.Hour // 10 years

if t.ExitDuration > MaxExitDuration {
    return fmt.Errorf("exit duration must not exceed %s: got %s", MaxExitDuration, t.ExitDuration)
}
```

This mirrors the `BonusApy` cap already present in the same function and the `TargetBaseRewardsRate` cap in `Params.Validate()`.

### Proof of Concept

1. Governance submits `MsgUpdateTier` with `exit_duration = 9223372036854775807ns` (max int64, ≈ 292 years) for Tier 1.
2. `Tier.Validate()` passes — only checks `> 0`.
3. `UpdateTier` handler calls `ms.SetTier(ctx, msg.Tier)` — no additional duration check.
4. User calls `MsgTriggerExitFromTier` on their position.
5. `pos.TriggerExit(blockTime, tier.ExitDuration)` sets `ExitUnlockAt = blockTime + 292 years`.
6. User calls `MsgTierUndelegate` → `validateUndelegatePosition` → `CompletedExitLockDuration(blockTime)` returns `false` → `ErrExitLockDurationNotReached`.
7. User calls `MsgExitTierWithDelegation` → same check → `ErrExitLockDurationNotReached`.
8. User calls `MsgWithdrawFromTier` → same check → `ErrExitLockDurationNotReached`.
9. If governance also set `CloseOnly = true`, `MsgClearPosition` → `ErrTierIsCloseOnly`. Funds are permanently trapped.

**Exact corrupted value:** `Position.ExitUnlockAt` stamped at trigger time with `blockTime + math.MaxInt64 ns`.

**Entry path:** Standard governance `MsgUpdateTier` transaction (authority = gov module account), followed by any user's `MsgTriggerExitFromTier` transaction — both are normal, supported production entrypoints. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

func (k Keeper) validateClearPosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
		if err != nil {
			return err
		}
		if isUnbonding {
			return types.ErrPositionUnbonding
		}

		if !pos.IsDelegated() {
			return types.ErrPositionNotDelegated
		}

	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	return nil
}

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}
