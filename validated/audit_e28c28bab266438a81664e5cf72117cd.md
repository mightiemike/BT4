### Title
`block_casm_hash_v1_declares` Version Gate Excludes DeclareTransaction V2, Allowing Stale Poseidon Compiled Class Hash to Persist in State — (`crates/blockifier/src/transaction/transactions.rs`)

### Summary

The `block_casm_hash_v1_declares` versioned-constants flag, introduced in Starknet 0.14.1 to enforce that only Blake2 (V2) compiled class hashes are accepted in new declarations, contains a version gate that silently excludes `DeclareTransaction::V2`. Any user who submits a V2 declare transaction with a Poseidon (V1) compiled class hash bypasses the enforcement entirely, causing the wrong hash to be committed to state even when the flag is `true`.

### Finding Description

In `crates/blockifier/src/transaction/transactions.rs`, the `run_execute` implementation for `DeclareTransaction` applies the V2-hash enforcement only when the transaction version is ≥ V3:

```rust
// crates/blockifier/src/transaction/transactions.rs  lines 183-189
if context.tx_context.block_context.versioned_constants.block_casm_hash_v1_declares
    && self.version() >= TransactionVersion::THREE
{
    self.check_compile_class_hash_v2_declaration()?
}
try_declare(self, state, class_hash, Some(*compiled_class_hash))?
```

`check_compile_class_hash_v2_declaration` (defined in `crates/starknet_api/src/executable_transaction.rs` lines 228–244) recomputes the Blake2 hash of the CASM and rejects the transaction if the supplied `compiled_class_hash` does not match. Because the guard is `>= TransactionVersion::THREE`, a `DeclareTransaction::V2` reaches `try_declare` unconditionally, writing whatever `compiled_class_hash` the submitter provided—including a Poseidon (V1) hash—directly into the state map.

The `block_casm_hash_v1_declares` flag was enabled (`true`) starting in version 0.14.1, as confirmed by the diff-regression file:

```
~ /block_casm_hash_v1_declares: true
~ /enable_casm_hash_migration: true
```

The Python-sequencer bridge (`crates/native_blockifier/src/py_declare.rs` lines 124–131) explicitly handles V2 declare transactions and passes them through to the blockifier without any compiled-class-hash validation of its own, making this path reachable in the current production sequencer.

The `TransactionVersion` constants confirm that V2 (`Felt::TWO`) is strictly less than V3 (`Felt::THREE`), so the condition `self.version() >= TransactionVersion::THREE` is `false` for every V2 transaction.

### Impact Explanation

When `block_casm_hash_v1_declares = true` and a V2 declare transaction carries a Poseidon (V1) compiled class hash:

1. `try_declare` writes the V1 hash into `state.class_hash_to_compiled_class_hash` without any check.
2. The state diff and subsequent Merkle commitment record the wrong hash for that class.
3. The Starknet OS (`migrate_classes_to_v2_casm_hash` in `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/os_utils.cairo`) expects that, after the migration flag is active, all newly declared classes carry V2 hashes. A class declared with a V1 hash via V2 transaction violates this invariant.
4. If `enable_casm_hash_migration` is also `true`, the migration logic in `CasmHashMigrationData::from_state` (`crates/blockifier/src/bouncer.rs` lines 377–400) will attempt to migrate the class, but the migration path assumes the stored hash is already the canonical V1 hash for a *previously* declared class—not a newly declared one that should have been blocked. This can produce an incorrect state diff entry for `class_hash_to_compiled_class_hash`.
5. If migration is disabled, the V1 hash remains in state permanently, causing the OS proof to fail or produce a wrong state root for any block that includes execution of that class.

This matches: **Critical. Wrong state … class hash … from blockifier/syscall/execution logic for accepted input.**

### Likelihood Explanation

- `block_casm_hash_v1_declares` has been `true` since Starknet 0.14.1 in production.
- V2 declare transactions are valid protocol-level transactions; any user can construct and submit one through the Python sequencer's `py_declare` entry point.
- No gateway-level filter blocks V2 declares in the Python sequencer path.
- The submitter only needs to supply a Poseidon hash (computable from the CASM) instead of the Blake2 hash.

### Recommendation

Remove the `&& self.version() >= TransactionVersion::THREE` guard so that `check_compile_class_hash_v2_declaration` is called for all Cairo 1 declare transactions (V2 and V3) whenever `block_casm_hash_v1_declares` is `true`:

```rust
if context.tx_context.block_context.versioned_constants.block_casm_hash_v1_declares {
    self.check_compile_class_hash_v2_declaration()?
}
```

`check_compile_class_hash_v2_declaration` already returns `Ok(())` for `ContractClass::V0`, so Cairo 0 (V0/V1 declare) transactions are unaffected.

### Proof of Concept

1. Compile any Sierra contract to CASM.
2. Compute its Poseidon (V1) compiled class hash using `casm.hash(&HashVersion::V1)`.
3. Construct a `DeclareTransaction::V2` with `compiled_class_hash` set to the V1 hash.
4. Submit via `py_declare` on a node running Starknet ≥ 0.14.1 (where `block_casm_hash_v1_declares = true`).
5. Observe that `run_execute` skips `check_compile_class_hash_v2_declaration` (because `TransactionVersion::TWO < TransactionVersion::THREE`) and calls `try_declare` with the V1 hash.
6. Inspect the resulting state diff: `class_hash_to_compiled_class_hash` contains the Poseidon hash instead of the Blake2 hash, contradicting the invariant enforced by `block_casm_hash_v1_declares`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

    pub fn check_compile_class_hash_v2_declaration(&self) -> Result<(), StarknetApiError> {
        let compiled_class = &self.class_info.contract_class;
        let compiled_class_hash_v2 = match &compiled_class {
            ContractClass::V0(_) => return Ok(()),
            ContractClass::V1((casm, _)) => casm.hash(&HashVersion::V2),
        };
        let compiled_class_hash = self.compiled_class_hash();
        if compiled_class_hash_v2 != compiled_class_hash {
            let err_var = CasmHashMismatch {
                hash: self.class_hash(),
                actual: compiled_class_hash,
                expected: compiled_class_hash_v2,
            };
            return Err(StarknetApiError::DeclareTransactionCasmHashMissMatch(Box::new(err_var)));
        }
        Ok(())
    }
