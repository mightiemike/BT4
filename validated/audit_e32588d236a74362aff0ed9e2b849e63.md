### Title
Unvalidated IBC Packet Token IDs Bypass Local Format Rules, Permanently Locking Received NFTs - (File: x/nft-transfer/keeper/packet.go)

---

### Summary

The `processReceivedPacket` function in `x/nft-transfer/keeper/packet.go` forwards `data.TokenIds` and `data.TokenUris` from an inbound IBC-721 packet directly into `MintNFT` without validating them against the local chain's `ValidateTokenID` rules. `NonFungibleTokenPacketData.ValidateBasic()` only checks that token IDs are non-blank and that counts match; it never calls `ValidateTokenID`. As a result, a counterparty chain can cause this chain to mint NFTs whose token IDs violate local format constraints, making those NFTs permanently unusable via any local transaction (transfer, burn, edit).

---

### Finding Description

**Layer 1 — Packet validation gap**

`NonFungibleTokenPacketData.ValidateBasic()` in `x/nft-transfer/types/packet.go` performs only structural checks:

```go
if len(nftpd.TokenIds) == 0 { ... }
if len(nftpd.TokenIds) != len(nftpd.TokenUris) { ... }
``` [1](#0-0) 

It never calls `ValidateTokenID` on individual token IDs. `ValidateTokenID` in `x/nft/types/validation.go` enforces:
- Length in `[3, 64]`
- Lowercase alphanumeric only (`[a-z0-9]+`)
- Must begin with a letter [2](#0-1) 

**Layer 2 — Unvalidated forwarding through the keeper chain**

`OnRecvPacket` in `relay.go` calls `data.ValidateBasic()` (which passes for non-blank IDs) and then immediately calls `processReceivedPacket`: [3](#0-2) 

`processReceivedPacket` passes `data.TokenIds[i]` and `data.TokenUris[i]` directly to `k.nftKeeper.MintNFT` with no additional format check: [4](#0-3) 

**Layer 3 — No validation inside MintNFT / MintNFTUnverified**

`MintNFT` delegates to `MintNFTUnverified`, which only checks denom existence and duplicate NFT ID — never token ID format: [5](#0-4) 

**Broken invariant**

Every local path that writes an NFT enforces `ValidateTokenID` at the message layer (`MsgMintNFT.ValidateBasic` → `ValidateTokenID`): [6](#0-5) 

The IBC receive path skips this entirely, so the x/nft store can contain NFTs whose IDs violate the chain's own format rules.

**Consequence: local operations permanently blocked**

Every local message that references an NFT by ID calls `ValidateTokenID` in its `ValidateBasic`:

- `MsgTransferNFT.ValidateBasic()` → `ValidateTokenID` [7](#0-6) 
- `MsgBurnNFT.ValidateBasic()` → `ValidateTokenID` [8](#0-7) 
- `MsgEditNFT.ValidateBasic()` → `ValidateTokenID` [9](#0-8) 

Any NFT minted with a non-conforming token ID (e.g., `"1abc"`, `"ABC"`, `"abc-def"`, or a 65-char ID) is rejected by all three messages. The receiver holds an NFT they can never transfer, burn, or edit locally.

The only escape is re-sending via IBC, because `MsgTransfer.ValidateBasic()` only checks that token IDs are non-blank: [10](#0-9) 

---

### Impact Explanation

**Impact: Medium**

The corrupted value is the receiver's NFT ownership record in the x/nft store. NFTs minted with non-conforming token IDs are permanently inaccessible to all local operations. The receiver's asset is effectively frozen for any on-chain use (sale, burn, governance, collateral). The only recovery path — re-sending via IBC — depends on the counterparty chain cooperating, which a malicious counterparty would not do. There is no admin escape hatch in the x/nft module to forcibly burn or transfer such tokens.

---

### Likelihood Explanation

**Likelihood: Medium**

ICS-721 imposes no token ID format constraints. Any counterparty chain that permits token IDs starting with digits, containing hyphens/underscores, using uppercase, or exceeding 64 characters can trigger this path legitimately or maliciously. A single user on such a chain sending one IBC NFT transfer is sufficient. No privileged access, leaked keys, or governance control is required — only a valid IBC channel and a token ID that passes the counterparty's local rules but fails this chain's `ValidateTokenID`.

---

### Recommendation

In `processReceivedPacket`, before calling `MintNFT`, validate each incoming token ID and URI against the local chain's rules:

```go
for i, tokenID := range data.TokenIds {
    if err := nfttypes.ValidateTokenID(tokenID); err != nil {
        return err
    }
    if err := nfttypes.ValidateTokenURI(data.TokenUris[i]); err != nil {
        return err
    }
}
```

Similarly, validate `data.ClassUri` before passing it to `IssueDenom` at line 168 of `packet.go`. [11](#0-10) 

---

### Proof of Concept

1. On counterparty chain B, create an NFT with token ID `"1abc"` (starts with digit — valid on B, invalid on A).
2. User on B submits `MsgTransfer` to send `"1abc"` to chain A via an established IBC-721 channel.
3. Relayer picks up the packet and submits it to chain A.
4. `IBCModule.OnRecvPacket` on chain A deserializes the packet and calls `keeper.OnRecvPacket`.
5. `data.ValidateBasic()` passes — `"1abc"` is non-blank and counts match.
6. `processReceivedPacket` calls `k.nftKeeper.MintNFT(ctx, voucherClassID, "1abc", ...)`.
7. `MintNFTUnverified` stores the NFT with token ID `"1abc"` in the KV store. No error.
8. Receiver on chain A now holds NFT `"1abc"` in denom `voucherClassID`.
9. Receiver submits `MsgTransferNFT{Id: "1abc", ...}` → `ValidateTokenID("1abc")` returns `ErrInvalidTokenID` ("nft id must begin with an english letter"). Transaction rejected.
10. Receiver submits `MsgBurnNFT{Id: "1abc", ...}` → same rejection.
11. NFT is permanently locked for all local operations. The receiver's asset is frozen. [12](#0-11) [13](#0-12) [14](#0-13)

func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil

func ValidateTokenID(tokenID string) error {
	if len(tokenID) < MinDenomLen || len(tokenID) > MaxDenomLen {
		return sdkerrors.Wrapf(ErrInvalidTokenID, "the length of nft id(%s) only accepts value [%d, %d]", tokenID, MinDenomLen, MaxDenomLen)
	}
	if !IsBeginWithAlpha(tokenID) || !IsAlphaNumeric(tokenID) {
		return sdkerrors.Wrapf(ErrInvalidTokenID, "nft id(%s) only accepts lowercase alphanumeric characters, and begin with an english letter", tokenID)
	}
	return nil

func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}

// MintNFTUnverified mints an NFT without verifying if the owner is the creator of denom
// Needed during genesis initialization
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil
```
