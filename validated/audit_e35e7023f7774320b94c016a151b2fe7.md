Audit Report

## Title
Per-Bin Additional Fee Has No Cap, Allowing Pool Admin to Bypass the Factory-Enforced Fee Ceiling - (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary

`MetricOmmPoolFactory` enforces explicit caps on the pool admin's global spread fee via `maxAdminSpreadFeeE6`, but `setPoolBinAdditionalFees` applies no analogous cap on `addFeeBuyE6` / `addFeeSellE6`. A pool admin can set per-bin fees up to the `uint16` maximum (65 535 E6 ≈ 6.55 %) on any bin, which `SwapMath` applies additively on top of the global spread fee during every swap traversing that bin, directly reducing swapper output beyond the protocol-intended ceiling.

## Finding Description

`MetricOmmPoolFactory.setPoolAdminFees` enforces the factory-level cap before updating the global admin spread fee:

```solidity
// MetricOmmPoolFactory.sol L414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

The second admin fee lever, `setPoolBinAdditionalFees`, contains no such guard:

```solidity
// MetricOmmPoolFactory.sol L450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

`MetricOmmPool.setBinAdditionalFees` only validates the bin index, then stores the unchecked values: [3](#0-2) 

During every swap, `MetricOmmPool` constructs the per-bin fee as `baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)` and passes it directly to `SwapMath`: [4](#0-3) [5](#0-4) 

`SwapMath` then uses this combined fee to compute the gross input required from the swapper: [6](#0-5) 

The `uint16` type permits values up to 65 535 E6 (≈ 6.55 %). No existing guard in either `setPoolBinAdditionalFees` or `setBinAdditionalFees` prevents this value from being set.

## Impact Explanation

A pool admin can call `factory.setPoolBinAdditionalFees(pool, bin, 65535, 65535)` in a single transaction with no timelock. Every swapper whose trade traverses that bin pays the global spread fee **plus** up to 6.55 % additional per bin, deducted from their input token amount before settlement. This is a direct, quantifiable loss of user principal on every swap through the affected bin(s). This satisfies the "Admin-boundary break: pool admin exceeds caps" and "direct loss of user principal" criteria.

## Likelihood Explanation

The pool admin role is explicitly semi-trusted: the factory owner caps the global admin spread fee precisely because the admin is not fully trusted. The per-bin path is reachable by any current `poolAdmin[pool]` in a single transaction with no timelock or delay. Any pool whose admin is compromised, acts adversarially, or has misaligned incentives can exploit this immediately and repeatedly.

## Recommendation

Add a cap check in `setPoolBinAdditionalFees` analogous to the global fee guard, using `maxAdminSpreadFeeE6` or a dedicated `maxBinAdditionalFeeE6` constant:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

## Proof of Concept

1. Factory owner deploys a pool with `maxAdminSpreadFeeE6 = 200_000` (20%) and `adminSpreadFeeE6 = 10_000` (1%).
2. Pool admin calls `factory.setPoolBinAdditionalFees(pool, 0, 65535, 65535)` — no revert occurs.
3. A swapper calls `pool.swap(...)` with the active bin at index 0.
4. `MetricOmmPool` constructs `currBinBuyFeeX64 = baseFeeX64 + Math.mulDiv(65535, ONE_X64, 1e6)` and passes it to `SwapMath.buyToken0InBinSpecifiedIn`.
5. `SwapMath` computes `feeAmountScaled = Math.ceilDiv(amountInScaled * currBinBuyFeeX64, ONE_X64)`, charging the swapper the global spread fee (1%) **plus** the per-bin additional fee (6.5535%) = **7.5535% total**.
6. The swapper receives fewer output tokens than the protocol's fee cap system was designed to guarantee — a direct loss of principal on every swap through the affected bin.
