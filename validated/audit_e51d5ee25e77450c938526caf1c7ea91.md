### Title
`burnNlp` Uses MAINTENANCE Health Check Instead of INITIAL, Allowing Post-Burn Undercollateralization — (`core/contracts/Clearinghouse.sol`)

---

### Summary

`burnNlp` enforces only a MAINTENANCE health check after burning NLP tokens, while every other collateral-reducing operation in the protocol enforces the stricter INITIAL health check. The protocol's own inline comment acknowledges that burning NLP can decrease health. This asymmetry allows a subaccount to exit a burn with negative INITIAL health, violating the core invariant that all operations must leave the sender above INITIAL health.

---

### Finding Description

`mintNlp` enforces INITIAL health: [1](#0-0) 

`burnNlp` enforces only MAINTENANCE health: [2](#0-1) 

The comment at lines 523–525 explicitly acknowledges the problem: burning NLP **can decrease health** when the burn fee exceeds the health improvement. The developers chose MAINTENANCE as the guard, but this is insufficient.

All other collateral-reducing paths use INITIAL:
- `withdrawCollateral` — INITIAL [3](#0-2) 
- `transferQuote` — INITIAL via `_isAboveInitial` [4](#0-3) 
- `nlpProfitShare` — INITIAL [5](#0-4) 

**Why burning NLP decreases INITIAL health:**

The burn fee is `max(ONE, quoteAmount / 1000)`. The net INITIAL health change from burning `nlpAmount` NLP is:

```
Δ_initial = -nlpAmount * price * nlpWeightInitial
           + (nlpAmount * price - burnFee) * quoteWeightInitial
```

If NLP and quote carry the same INITIAL weight (both 1.0, as quote is the reference asset and NLP is a liquid pool token), this simplifies to:

```
Δ_initial = -burnFee * quoteWeightInitial < 0
```

Every burn unconditionally decreases INITIAL health by the burn fee. The MAINTENANCE check only catches the case where MAINTENANCE health also goes negative — it does not protect INITIAL health.

`RiskHelper` confirms INITIAL uses `longWeightInitialX18` and MAINTENANCE uses `longWeightMaintenanceX18`, which are distinct values: [6](#0-5) 

---

### Impact Explanation

An attacker can leave a subaccount with negative INITIAL health after `burnNlp`:

1. The subaccount is now in a state unreachable through any other normal operation.
2. The attacker has extracted quote (the burn proceeds) while the subaccount is undercollateralized at the INITIAL level.
3. The position is eligible for liquidation, potentially at a loss to the protocol insurance fund.
4. The attacker can repeat this across multiple subaccounts to systematically drain quote while socializing losses.

---

### Likelihood Explanation

The exploit requires no special privileges. Any user with an NLP balance and a subaccount whose INITIAL health is between 0 and `burnFee` can trigger this. A price drop (via a preceding `updatePrice` in the same sequencer batch) is a natural way to push INITIAL health close to zero before calling `burnNlp`. The minimum `burnFee` is `ONE` ($1), so INITIAL health only needs to be below $1 for the smallest possible burn to trigger the violation. [7](#0-6) 

---

### Recommendation

Replace the MAINTENANCE health check in `burnNlp` with an INITIAL health check, consistent with every other collateral-reducing operation:

```solidity
// Before (line 527):
getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0

// After:
getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0
``` [8](#0-7) 

---

### Proof of Concept

```
Setup:
  - Subaccount S holds 1000 NLP (price = 1.0, worth $1000)
  - Subaccount S holds a perp position such that:
      INITIAL health  = $0.50  (just above zero)
      MAINTENANCE health = $5.00

Step 1: (optional) UpdatePrice drops NLP price slightly, pushing INITIAL health to $0.50.

Step 2: Attacker submits BurnNlp for nlpAmount = 1000 (quoteAmount = $1000).
  burnFee = max(ONE, 1000e18 / 1000) = $1.00

Step 3: After burn:
  INITIAL health    = $0.50 - $1.00 = -$0.50  (NEGATIVE — invariant broken)
  MAINTENANCE health = $5.00 - $1.00 = $4.00  (passes the check at line 527)

Step 4: burnNlp succeeds. Attacker receives $999 quote.
  Subaccount S is now below INITIAL health with no NLP collateral remaining.

Assert: getHealth(S, INITIAL) < 0  ✓ (invariant violated)
```
