Audit Report

## Title
`uint32` Truncation in `_afterTimelock` Allows Pool Admin to Bypass Timelock Instantly — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`_afterTimelock` computes `uint32(block.timestamp + oracleStopLossConfig[pool_].timelock)`. When `timelock` is set to a value that causes the `uint256` sum to exceed `type(uint32).max`, Solidity's explicit cast silently truncates the result to a timestamp already in the past. Every subsequent `_requireElapsed` check then passes immediately, allowing the pool admin to execute any timelocked parameter change — drawdown, decay, watermarks, or the timelock itself — without waiting, defeating the LP-protection guarantee the timelock is designed to enforce.

## Finding Description

**Root cause — no upper-bound validation on `timelock`:**

`initialize()` validates `drawdownE6` and `decayPerSecondE8` but skips `timelock`:

```solidity
// OracleValueStopLossExtension.sol lines 56-58
(uint32 drawdownE6, uint32 decayPerSecondE8, uint32 timelock) = abi.decode(data, (uint32, uint32, uint32));
_validateDrawdown(drawdownE6);
_validateDecay(decayPerSecondE8);
// ← no _validateTimelock(timelock)
``` [1](#0-0) 

`proposeOracleStopLossTimelock` similarly accepts any `uint32` without validation:

```solidity
// lines 78-84
function proposeOracleStopLossTimelock(address pool_, uint32 newTimelock) external onlyPoolAdmin(pool_) {
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    uint32 executeAfter = _afterTimelock(pool_);
    sched.pendingTimelock = newTimelock;
    sched.pendingTimelockExecuteAfter = executeAfter;
    ...
}
``` [2](#0-1) 

**Overflow in `_afterTimelock`:**

```solidity
// lines 297-299
function _afterTimelock(address pool_) private view returns (uint32) {
    return uint32(block.timestamp + oracleStopLossConfig[pool_].timelock);
}
``` [3](#0-2) 

`block.timestamp` is `uint256` (~1.753 × 10⁹ today). The addition is performed in `uint256` space, then **truncated** to `uint32`. In Solidity 0.8.x, explicit casts truncate — they do not revert. `type(uint32).max` = 4,294,967,295. Any `timelock > 4,294,967,295 − 1,753,000,000 ≈ 2,541,967,295` (~80.5 years, still a valid `uint32`) causes the sum to exceed `type(uint32).max`, wrapping `executeAfter` to a value already in the past (e.g., `uint32(1,753,000,000 + 3,000,000,000)` = `uint32(4,753,000,000)` = `458,032,704`, a timestamp in 1984).

**Bypass of `_requireElapsed`:**

```solidity
// lines 301-303
function _requireElapsed(uint32 executeAfter) private view {
    if (block.timestamp < executeAfter) revert OracleStopLossTimelockNotElapsed(executeAfter, block.timestamp);
}
``` [4](#0-3) 

With `executeAfter = 458,032,704` and `block.timestamp ≈ 1,753,000,000`, the condition `block.timestamp < executeAfter` is `false` and the check passes immediately.

**All four timelocked execute functions are affected:**
- `executeOracleStopLossTimelock` (line 89) [5](#0-4) 
- `executeOracleStopLossDrawdown` (line 115) [6](#0-5) 
- `executeOracleStopLossDecay` (line 142) [7](#0-6) 
- `executeOracleStopLossHighWatermarks` (line 172) [8](#0-7) 

## Impact Explanation

The timelock is the sole mechanism protecting LPs from sudden, unannounced parameter changes by the pool admin. With the bypass active, the pool admin can:

1. **Instantly disable the stop-loss** by setting `drawdownE6 = 0`, removing all value-leak protection without any LP reaction window.
2. **Instantly raise watermarks to extreme values**, causing the stop-loss to trigger on the very next swap and permanently blocking one or both swap directions.
3. **Instantly lower decay to zero**, freezing watermarks at their current peak and making the stop-loss permanently hair-trigger.

This is a direct admin-boundary break: the pool admin exceeds the cap the timelock is designed to impose, matching the allowed impact gate ("Admin-boundary break: pool admin exceeds caps, bypasses timelocks"). LP principal is at direct risk from the removal of the stop-loss guard or its weaponization to freeze the pool.

## Likelihood Explanation

The pool admin is a semi-trusted role. The attack does not require malicious initial setup: a pool can launch with a short, reasonable timelock (e.g., 1 day = 86,400 seconds). The admin then calls `proposeOracleStopLossTimelock(pool_, 3_000_000_000)` — a valid `uint32` — waits the current 1-day timelock, executes it, and from that point forward every proposal is immediately executable. No external preconditions, no special tokens, no oracle manipulation required. The overflowing value (3,000,000,000) is indistinguishable from a legitimate large-but-valid `uint32` input.

## Recommendation

Add an upper-bound check for `timelock` in both `initialize()` and `proposeOracleStopLossTimelock()`, mirroring the existing guards for `drawdownE6` and `decayPerSecondE8`:

```solidity
uint32 private constant MAX_TIMELOCK = 30 days;

function _validateTimelock(uint256 t) private pure {
    if (t > MAX_TIMELOCK) revert OracleStopLossTimelockTooLarge(t);
}
```

Apply `_validateTimelock(timelock)` in `initialize()` after decoding, and `_validateTimelock(newTimelock)` at the top of `proposeOracleStopLossTimelock()`. A bound of 30 days is well below the overflow threshold (~80.5 years) and sufficient for any legitimate operational timelock.

## Proof of Concept

```solidity
// Foundry test sketch
function test_timelockOverflowBypass() public {
    // 1. Deploy pool with 1-day timelock
    uint32 initialTimelock = 1 days;
    extension.initialize(pool, abi.encode(uint32(500_000), uint32(58), initialTimelock));

    // 2. Admin proposes overflowing timelock (valid uint32, causes wrap)
    uint32 overflowTimelock = 3_000_000_000;
    vm.prank(poolAdmin);
    extension.proposeOracleStopLossTimelock(pool, overflowTimelock);

    // 3. Wait the current 1-day timelock
    vm.warp(block.timestamp + 1 days + 1);

    // 4. Execute — sets timelock to 3_000_000_000
    vm.prank(poolAdmin);
    extension.executeOracleStopLossTimelock(pool);

    // 5. Now propose drawdown change — _afterTimelock wraps to past timestamp
    vm.prank(poolAdmin);
    extension.proposeOracleStopLossDrawdown(pool, 0); // disable stop-loss

    // 6. Execute immediately — no waiting required
    vm.prank(poolAdmin);
    extension.executeOracleStopLossDrawdown(pool); // succeeds instantly

    assertEq(extension.oracleStopLossConfig(pool).drawdownE6, 0);
}
```
