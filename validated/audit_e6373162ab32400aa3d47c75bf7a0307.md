### Title
Gas-key nonce fee computed from wire-encoded key length instead of trie-ID length in host functions, causing ~59× overcharge for ML-DSA-65 keys — (`runtime/near-vm-runner/src/logic/logic.rs`, `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`)

---

### Summary

The two gas-key host functions (`promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`) pass the raw WASM-memory byte-length of the borsh-encoded public key (`public_key_len as usize`, which is **1953** for ML-DSA-65) to `gas_key_add_key_exec_fee()`. That function uses the value as the on-trie key-identifier length to price each nonce trie entry. The actual trie identifier for an ML-DSA-65 key is the 33-byte SHA3-256 hash, not the 1953-byte borsh encoding. The transaction-path code that computes the same fee correctly calls `public_key.trie_id_len()` (33 bytes). The two representations diverge by **1920 bytes per nonce**, producing a ~59× overcharge that causes gas exhaustion for any ML-DSA-65 gas key with ≥ 2 nonces added via a host function.

---

### Finding Description

**Invariant being broken.** `gas_key_add_key_exec_fee(account_id_len, public_key_len, num_nonces)` is documented and used throughout the runtime to price the trie writes for gas-key nonces. Its `public_key_len` argument must be the on-trie identifier length — the value returned by `PublicKey::trie_id_len()` — because that is the actual byte width of the trie key segment that the runtime writes for each nonce entry.

**Transaction path (correct).** In `runtime/runtime/src/config.rs`, `permission_exec_fees` calls:

```rust
let nonce_fee = gas_key_add_key_exec_fee(
    fees,
    account_id.len(),
    public_key.trie_id_len(),   // 33 for ML-DSA-65
    gas_key_info.num_nonces,
);
``` [1](#0-0) 

**Host-function path (broken).** In `runtime/near-vm-runner/src/logic/logic.rs`, both gas-key host functions call:

```rust
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,    // 1953 for ML-DSA-65 (wire/borsh length)
    num_nonces,
);
``` [2](#0-1) [3](#0-2) 

The same pattern is duplicated in the wasmtime runner: [4](#0-3) [5](#0-4) 

**Why `public_key_len` is the wire length.** The host-function ABI passes the public key as raw borsh bytes in WASM memory. For ML-DSA-65 the borsh encoding is `[tag=2] || 1952-byte raw key` = **1953 bytes**. The contract writes those 1953 bytes into memory and passes `public_key_len = 1953`. The host function reads and validates the key, but then forwards the raw `public_key_len` directly to the fee helper instead of calling `trie_id_len()` on the decoded key.

**The divergent values.** `gas_key_add_key_exec_fee` computes:

```rust
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + size_of::<NonceIndex>();
let per_byte = gas_key_byte_exec_fee
    .checked_mul((nonce_key_len + NONCE_VALUE_LEN) as u64)
    .checked_mul(num_nonces);
``` [6](#0-5) 

With `public_key_len = 1953` (host-function path) vs `public_key_len = 33` (transaction path), the per-nonce byte count differs by **1920 bytes**. At `gas_key_byte` exec fee of 101,435,400 gas/byte:

| num_nonces | Correct fee (Tgas) | Charged fee (Tgas) | Overcharge (Tgas) |
|---|---|---|---|
| 1 | ~4.8 | ~199.6 | ~194.8 |
| 2 | ~9.6 | ~399.2 | ~389.6 |
| 3 | ~14.4 | ~598.8 | ~584.4 |

The `max_total_prepaid_gas` limit is 300 Tgas. Any ML-DSA-65 gas key with **≥ 2 nonces** added via a host function will unconditionally exhaust the gas budget and abort with `GasExceeded`.

**The trie-ID vs wire-length distinction** is explicitly documented as a new contract introduced by the ML-DSA-65 integration: [7](#0-6) 

The storage-stake path and the transaction fee path were both updated to call `trie_id_len()`. The host-function fee path was not.

---

### Impact Explanation

Any on-chain contract that calls `promise_batch_action_add_gas_key_with_full_access` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key and `num_nonces ≥ 2` will have its execution aborted by gas exhaustion, regardless of how much gas was attached. With `num_nonces = 1` the call succeeds but burns ~195 Tgas more than the equivalent transaction, making the operation ~40× more expensive than the protocol intends. The feature is effectively non-functional via the host-function path for the common case of multiple nonces.

---

### Likelihood Explanation

The `PostQuantumSignatures` protocol feature gates ML-DSA-65 key acceptance. It is currently active on nightly builds (protocol version 156) and will reach mainnet upon stabilization. Any contract written to manage ML-DSA-65 gas keys programmatically — the primary use case for the gas-key host functions — will trigger this path. The bug requires no special privileges: any unprivileged contract call suffices.

---

### Recommendation

In both `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call` (in both `logic/logic.rs` and `wasmtime_runner/logic.rs`), decode the public key first, then pass `decoded_key.trie_id_len()` to `gas_key_add_key_exec_fee()` instead of `public_key_len as usize`. The decoded key is already available (it is used for `append_action_add_gas_key_*`), so no extra work is needed:

```rust
let decoded_key = public_key.decode()?;
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    decoded_key.trie_id_len(),   // was: public_key_len as usize
    num_nonces,
);
```

---

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 public key (1953 borsh bytes) and `num_nonces = 2`.
2. Attach 300 Tgas (the maximum).
3. Observe the call fails with `GasExceeded` even though the equivalent `AddKey` transaction with the same key and `num_nonces = 2` succeeds within normal gas limits.

The divergence is exact: the host-function path charges `gas_key_byte_exec_fee × (access_key_key_len(account_id_len, 1953) + NONCE_VALUE_LEN) × num_nonces` while the transaction path charges `gas_key_byte_exec_fee × (access_key_key_len(account_id_len, 33) + NONCE_VALUE_LEN) × num_nonces`. The ratio of the per-nonce byte components is `(1953 + overhead) / (33 + overhead) ≈ 59×`. [8](#0-7) [9](#0-8)

    permission: &AccessKeyPermission,

            let num_bytes = perm

            base_fee.checked_add(all_bytes_fee).unwrap()

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }
```
