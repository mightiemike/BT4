### Title
Bech32 Address Validation of Counterparty Sender in `NonFungibleTokenPacketData.ValidateBasic()` Breaks IBC NFT Transfer Compatibility with Non-Cosmos Chains - (File: x/nft-transfer/types/packet.go)

### Summary
`NonFungibleTokenPacketData.ValidateBasic()` validates the `Sender` field using `sdk.AccAddressFromBech32()`, a Cosmos-specific bech32 decoder. Because `ValidateBasic()` is called inside `OnRecvPacket`, the `Sender` field belongs to the **counterparty (source) chain**, which may use a non-bech32 address format. Any IBC NFT packet arriving from a non-Cosmos chain (e.g., an EVM chain using hex addresses) will be permanently rejected, making cross-chain NFT transfers impossible over any such channel.

### Finding Description

The comment at the top of `ValidateBasic()` explicitly acknowledges the ICS-721 design intent:

> "NOTE: The addresses formats are not validated as the sender and recipient can have different formats defined by their corresponding chains that are not known to IBC."

Despite this, the implementation immediately contradicts itself by calling `sdk.AccAddressFromBech32()` on both the sender and receiver:

```go
// decode the sender address
if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
    return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
}
// decode the receiver address
if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
    return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
}
``` [1](#0-0) 

This `ValidateBasic()` is invoked unconditionally in `OnRecvPacket` before any application logic runs:

```go
func (k Keeper) OnRecvPacket(...) error {
    if err := data.ValidateBasic(); err != nil {
        return err
    }
    return k.processReceivedPacket(ctx, packet, data)
}
``` [2](#0-1) 

When this chain is the **receiver**, `data.Sender` is the address of the user on the **source chain**. If the source chain is an EVM chain (hex address `0xabc...`) or any other non-Cosmos chain, `sdk.AccAddressFromBech32(nftpd.Sender)` returns an error, and the packet is rejected with an error acknowledgement.

The `IBCModule.OnRecvPacket` propagates this as an error acknowledgement: [3](#0-2) 

By contrast, `MsgTransfer.ValidateBasic()` — used on the **sending** side — correctly omits receiver format validation, with the same ICS-721 rationale: [4](#0-3) 

The analog to the external report is exact: the external bug used a 20-byte EVM `address` type where `bytes32` was needed for cross-chain compatibility. Here, `sdk.AccAddressFromBech32` (Cosmos-specific) is applied to a field that belongs to the counterparty chain, breaking compatibility with any non-Cosmos chain.

### Impact Explanation

Every IBC NFT transfer packet arriving from a non-Cosmos chain (any chain whose addresses are not bech32-encoded) is permanently rejected at the `ValidateBasic()` gate. The NFT is never minted or unescrowed on chain-main. The source chain receives an error acknowledgement and refunds the sender. No NFT can ever cross from a non-Cosmos chain to chain-main via the `x/nft-transfer` module. Any IBC channel established between chain-main and such a counterparty is rendered permanently non-functional for NFT transfers.

### Likelihood Explanation

IBC adoption by non-Cosmos chains (EVM chains, Solana, etc.) is an active and growing area. Any governance-approved IBC channel to such a chain would immediately expose this bug. The trigger requires only a standard `MsgTransfer` transaction on the counterparty chain — no privilege, no key compromise, no special configuration. The bug is deterministic and 100% reproducible for every packet from a non-bech32 chain.

### Recommendation

Remove the `sdk.AccAddressFromBech32` calls from `NonFungibleTokenPacketData.ValidateBasic()`. The sender address belongs to the counterparty chain and must not be decoded with a chain-local codec. The receiver address validation is also incorrect per ICS-721 spec (the receiving chain's application logic in `processReceivedPacket` already performs the bech32 decode where it is actually needed). The fix is to replace both bech32 checks with simple non-empty string checks, consistent with the comment and with `MsgTransfer.ValidateBasic()`:

```go
if strings.TrimSpace(nftpd.Sender) == "" {
    return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
}
if strings.TrimSpace(nftpd.Receiver) == "" {
    return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
}
```

### Proof of Concept

1. A non-Cosmos chain (e.g., an EVM chain) establishes an IBC channel with chain-main's `nft` port.
2. A user on the EVM chain calls the IBC NFT transfer entrypoint with their EVM address (e.g., `0xdeadbeef...`) as sender and a valid bech32 address on chain-main as receiver.
3. The IBC relayer delivers the packet to chain-main's `IBCModule.OnRecvPacket`.
4. `types.ModuleCdc.UnmarshalJSON` succeeds (the packet is well-formed JSON).
5. `im.keeper.OnRecvPacket` is called, which immediately calls `data.ValidateBasic()`.
6. `sdk.AccAddressFromBech32("0xdeadbeef...")` fails — `0xdeadbeef...` is not a valid bech32 string.
7. `ValidateBasic()` returns `ErrInvalidAddress: invalid sender address`.
8. `OnRecvPacket` returns an error acknowledgement; no NFT is minted on chain-main.
9. The source chain's `OnAcknowledgementPacket` refunds the sender.
10. This repeats for every packet from this channel — the channel is permanently broken for NFT transfers. [5](#0-4) [6](#0-5) [7](#0-6)

// ValidateBasic is used for validating the nft transfer.
// NOTE: The addresses formats are not validated as the sender and recipient can have different
// formats defined by their corresponding chains that are not known to IBC.
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil

// ValidateBasic performs a basic check of the MsgTransfer fields.
// NOTE: timeout height or timestamp values can be 0 to disable the timeout.
// NOTE: The recipient addresses format is not validated as the format defined by
// the chain is not known to IBC.
func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil
