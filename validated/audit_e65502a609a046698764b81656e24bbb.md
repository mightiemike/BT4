Audit Report

## Title
Swap allowlist bypassed via allowlisted router: any EOA can swap through `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, the pool receives the router as `sender`, not the originating EOA. If the router is allowlisted, every user of the router bypasses the per-EOA allowlist check, completely defeating the access control gate.

## Finding Description
`MetricOmmPool.swap` passes its own `msg.sender` as the first argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // <-- router address when called via router
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards this value as `sender` to each extension's `beforeSwap(address sender, ...)`.

`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()` — the router, not the originating EOA.

`MetricOmmSimpleRouter.exactInputSingle` stores the originating `msg.sender` only in transient storage for the payment callback, then calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
IMetricOmmPoolActions(params.pool).swap(
    params.recipient, params.zeroForOne, ..., params.extensionData
);
```

The originating EOA is never forwarded to the pool. The pool sees the router as `sender`. The check resolves to `allowedSwapper[pool][router]`, which is `true` if the router is allowlisted — so the check passes unconditionally for every caller of the router, regardless of whether their EOA is individually allowlisted.

## Impact Explanation
Pool curation via the swap allowlist is completely broken for any pool that allowlists the router. Non-allowlisted EOAs can freely swap through the router, draining LP positions and violating the core invariant that only explicitly permitted addresses may trade. This constitutes broken core pool functionality with direct fund impact on LPs — matching the "Broken core pool functionality causing loss of funds" allowed impact.

## Likelihood Explanation
Allowlisting a public router is the natural and expected configuration for any pool that wants to permit router-based trading while restricting direct pool access. Any pool admin who does so unknowingly opens the pool to all router users. The attacker path requires no special privileges — only calling a public router function with no preconditions beyond token approval.

## Recommendation
The extension must verify the originating user, not the immediate `pool.swap()` caller. Concrete options:
- Have the router encode the originating EOA into `extensionData` and have the extension decode and verify it (requires a registry of trusted routers to prevent spoofing).
- Alternatively, document clearly that routers must never be allowlisted; only individual EOAs should be allowlisted, and the pool admin UI/docs should enforce this.
- A cleaner protocol-level fix: expose the originating initiator through a dedicated field in the pool's swap call or a separate transient-storage slot readable by extensions.

## Proof of Concept
```solidity
function test_allowlistBypass() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted, attacker EOA not allowlisted
    swapAllowlist.setAllowedToSwap(pool, address(router), true);
    // allowedSwapper[pool][attacker] == false

    // Attacker calls router — router calls pool.swap(sender=router)
    // Extension sees allowedSwapper[pool][router] == true → passes
    vm.prank(attacker);
    router.exactInputSingle(ExactInputSingleParams({
        pool: pool,
        recipient: attacker,
        zeroForOne: true,
        amountIn: 1e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        tokenIn: token0,
        extensionData: ""
    }));
    // Assert: swap succeeded despite attacker not being individually allowlisted
}
```
