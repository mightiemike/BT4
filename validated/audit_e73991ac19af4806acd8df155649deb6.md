### Title
Bonus Accrual Checkpoints Advance Without Payment When Pool Is Insufficient During Redelegation Slash — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, by the time that error is returned, `processEventsAndClaimBonus` has already (a) advanced the position's `LastEventSeq` and `LastBonusAccrual` checkpoints in memory, and (b) decremented event reference counts in the store. The caller then saves the position with those advanced checkpoints. The user's accrued bonus is permanently forfeited — the events are consumed and can never be replayed.

---

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` performs two categories of side effects before it checks whether the bonus pool can cover the payout:

**Category 1 — store writes (inside the event loop):** [1](#0-0) 

`pos.UpdateLastEventSeq(entry.Seq)` mutates the in-memory position, and `decrementEventRefCount` writes to the store — both happen for every event, before the pool-sufficiency check.

**Category 2 — in-memory checkpoint updates (after the loop):** [2](#0-1) 

`applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` advance the position's accrual window. Only after these mutations does the function reach: [3](#0-2) 

If the pool is insufficient, `ErrInsufficientBonusPool` is returned — but all the mutations above have already occurred.

In `slashRedelegationPosition`, this error is explicitly swallowed: [4](#0-3) 

Execution then falls through to `setPositionWithState`, which persists the mutated `pos` — with advanced checkpoints — to the store: [5](#0-4) 

The result: the position's `LastEventSeq` and `LastBonusAccrual` are committed past the events that were processed, the events' reference counts are decremented (and may be garbage-collected when they reach zero), and the bonus is never transferred. The user has no path to recover those rewards.

---

### Impact Explanation

**Impact: Medium.**

The broken invariant is: *"a position's bonus checkpoints advance past an event if and only if the bonus for that event has been paid to the owner."*

When this invariant is violated, the owner of a tier position permanently loses all accrued bonus rewards that were computed during the failed `processEventsAndClaimBonus` call. The corrupted value is `pos.LastEventSeq` / `pos.LastBonusAccrual` in the `tieredrewards` module store, and the decremented `ValidatorEvent.ReferenceCount` entries. There is no recovery mechanism — the events are consumed and the checkpoints cannot be rewound.

---

### Likelihood Explanation

**Likelihood: Medium.**

The trigger requires two concurrent conditions:
1. A tier position holder has an active redelegation on a validator that gets slashed (`BeforeRedelegationSlashed` fires → `slashRedelegationPosition`).
2. The `tieredrewards` rewards pool (`RewardsPoolName`) has insufficient balance to cover the position's accrued bonus at that moment.

Condition 1 is a normal staking lifecycle event (validator slashing is a standard Cosmos SDK operation reachable by any delegator/validator). Condition 2 is realistic: the pool is funded by governance/inflation and can be depleted by concurrent claims or simply be underfunded. Neither condition requires privileged access.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to after the successful `SendCoinsFromModuleToAccount` call, so checkpoints only advance when the bonus is actually paid. Alternatively, restructure `processEventsAndClaimBonus` to separate the "compute and consume events" phase from the "pay and checkpoint" phase, so that if payment fails the event consumption can be rolled back or deferred.

```go
// In processEventsAndClaimBonus — move checkpoint updates AFTER successful payment:

if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
    return nil, err
}

// Only advance checkpoints after confirmed payment:
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
pos.UpdateLastKnownBonded(bonded)
```

---

### Proof of Concept

1. Alice creates a tier position delegating to validator V.
2. Alice redelegates from V to V2; a `RedelegationMapping` entry is created.
3. The `tieredrewards` rewards pool is drained (e.g., by many concurrent claims).
4. V is slashed. The Cosmos SDK staking module fires `BeforeRedelegationSlashed`.
5. `slashRedelegationPosition` is called → `processEventsAndClaimBonus` runs:
   - Events since `pos.LastEventSeq` are iterated; `pos.UpdateLastEventSeq` and `decrementEventRefCount` execute for each.
   - `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` execute.
   - `sufficientBonusPoolBalance` fails → `ErrInsufficientBonusPool` returned.
6. `slashRedelegationPosition` swallows the error, then calls `setPositionWithState(ctx, pos, ...)`.
7. Alice's position is now stored with `LastEventSeq` and `LastBonusAccrual` advanced past the slash event. The event's `ReferenceCount` has been decremented. Alice's bonus is permanently lost.
8. Even after the pool is replenished, Alice's next `MsgClaimTierRewards` will compute zero bonus for the already-consumed events. [6](#0-5) [7](#0-6)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
