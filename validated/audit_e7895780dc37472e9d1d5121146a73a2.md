### Title
Unprivileged Caller Can Trigger Aggressive Mainchain GC at Zero Cost, Breaking SPV Proof Verification - (`contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, deposit-free, role-free state-mutating function callable by any NEAR account. An attacker can invoke it with `batch_size = u64::MAX` at any time to prune the maximum possible number of mainchain block headers in a single transaction. This permanently removes block-to-height index entries that `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` depend on, causing SPV proofs for transactions in pruned blocks to panic and chain reorganization to fail — at zero economic cost to the attacker.

### Finding Description

`submit_blocks` is correctly gated behind `#[trusted_relayer]` and requires a storage deposit. However, the GC function it internally calls is also exposed as a standalone public entry point with no equivalent guards:

```rust
// contract/src/lib.rs:376-416
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    ...
    if amount_of_headers_we_store > self.gc_threshold {
        let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
        let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);
        for height in start_removal_height..end_removal_height {
            let blockhash = &self.mainchain_height_to_header.get(&height)...;
            self.remove_block_header(blockhash);
            self.mainchain_height_to_header.remove(&height);
        }
        self.mainchain_initial_blockhash = ...;
    }
}
``` [1](#0-0) 

When the contract is not paused, the `#[pause]` macro imposes no caller restriction. There is no `#[trusted_relayer]`, no `#[payable]` deposit check, and no role assertion. Any NEAR account can call this function.

By contrast, `submit_blocks` — the only intended path to trigger GC — bounds the batch size to the number of headers submitted in that call (`num_of_headers`), keeping GC incremental:

```rust
// contract/src/lib.rs:175-181
let num_of_headers = headers.len().try_into().unwrap();
for header in headers { self.submit_block_header(header, ...); }
self.run_mainchain_gc(num_of_headers);
``` [2](#0-1) 

An attacker bypasses this bound by calling `run_mainchain_gc(u64::MAX)` directly, removing every block above `gc_threshold` in one transaction.

The pruned entries are the `mainchain_height_to_header` and `mainchain_header_to_height` mappings. Both `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height` and panic if it is absent:

```rust
// contract/src/lib.rs:299-301
let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));
``` [3](#0-2) 

The CLAUDE.md documentation also explicitly acknowledges that aggressive GC breaks chain reorganization:

> "If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound`" [4](#0-3) 

### Impact Explanation

Any dApp or bridge contract that calls `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` to gate a cross-chain action (e.g., token unlock, proof settlement) will receive a panic instead of a result for any transaction whose block was pruned by the attacker. The attacker can selectively time the GC call to prune blocks containing specific high-value transactions immediately after they reach the confirmation threshold, permanently denying their SPV verification. Chain reorganization also becomes impossible across the pruned range, corrupting the canonical chain state.

### Likelihood Explanation

The entry point is a public NEAR function requiring only gas fees (fractions of a cent). No role, no deposit, no PoW, no prior state is needed. The attacker only needs to wait until the mainchain grows beyond `gc_threshold` — a normal operational condition — then call `run_mainchain_gc(18446744073709551615)`. This is trivially scriptable and repeatable.

### Recommendation

Add a `#[trusted_relayer]` (or equivalent role check) to `run_mainchain_gc` so that only authorized relayers or DAO accounts can invoke it directly. Alternatively, make the function `pub(crate)` / private and remove it from the public ABI entirely, since `submit_blocks` already calls it internally with a safe bounded `batch_size`.

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` (recommended production value).
2. Wait for the relayer to submit enough blocks so `mainchain_size > gc_threshold`.
3. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc({ "batch_size": 18446744073709551615 })
   ```
   with zero attached deposit.
4. All blocks above `gc_threshold` are pruned in one transaction.
5. Call `verify_transaction_inclusion` for any transaction whose block was just pruned — the call panics with `"block does not belong to the current main chain"`.
6. Any pending chain reorganization whose fork point falls in the pruned range now panics with `PrevBlockNotFound`. [5](#0-4) [6](#0-5)

    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
