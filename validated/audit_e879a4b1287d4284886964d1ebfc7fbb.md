### Title
Silent `Ok` on Missing `final_execution_outcome` Permanently Skips Block Heights — (`relayer/src/near_client.rs`)

### Summary

`parse_submit_blocks_response` returns `Ok(SubmitResult { gas_burnt: 0 })` when `response.final_execution_outcome` is `None`, instead of returning an `Err`. Because `get_tx_status` uses `wait_until: TxExecutionStatus::Executed`, the NEAR RPC can legally return a response with `final_execution_outcome = None` (the field is `Option<…>` in the SDK type). The caller in `prepare_and_submit_batches` matches this as `Ok(Ok(result))` and unconditionally advances `first_block_height_to_submit` past the batch's last height, even though the contract's `submit_blocks` method was never confirmed to have executed.

---

### Finding Description

**Root cause — `parse_submit_blocks_response` fallthrough:** [1](#0-0) 

The function checks `if let Some(ref final_execution_outcome) = response.final_execution_outcome { … }`. If the field is `None`, execution falls through to line 340 and silently returns `Ok(SubmitResult { gas_burnt: 0 })`. There is no `else` branch that returns an `Err`.

**RPC call uses `wait_until: TxExecutionStatus::Executed`:** [2](#0-1) 

`TxExecutionStatus::Executed` is satisfied when the transaction is included and receipts are executed, but the NEAR JSON-RPC spec does not guarantee that `final_execution_outcome` is populated at this wait level — the field is `Option<FinalExecutionOutcomeViewEnum>` in the SDK. A response with `final_execution_outcome = None` is a valid, non-error RPC response under this wait condition.

**Caller unconditionally advances the height pointer on `Ok(Ok(…))`:** [3](#0-2) 

```
Ok(Ok(result)) => {
    first_block_height_to_submit.store(tx.last_block_height + 1, ...);
    ...
}
```

When `parse_submit_blocks_response` returns the spurious `Ok`, this arm fires and permanently advances the atomic counter past the batch's last height. The next sync loop iteration starts from `last_block_height + 1`, skipping the entire batch.

---

### Impact Explanation

The invariant that `first_block_height_to_submit` only advances past heights durably committed to `mainchain_height_to_header` is broken. The skipped heights are never retried because the relayer has no mechanism to detect or backfill them. Downstream effects:

- `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` return `false` for any transaction in a skipped block, because the block header is absent from the contract's storage.
- `get_last_n_blocks_hashes` returns an incomplete sequence, breaking fork-detection logic in `get_last_correct_block_height`.
- The gap is permanent unless the contract is re-initialized or an operator manually intervenes.

---

### Likelihood Explanation

The NEAR RPC is documented to return `final_execution_outcome = None` when `wait_until` is below `ExecutedOptimistic` or `Final`. Under network congestion or a slow receipt pipeline, a response satisfying `Executed` can arrive before the full outcome view is assembled. This is not a theoretical edge case; it is an expected behavior of the NEAR RPC at the `Executed` wait level.

---

### Recommendation

Replace the silent fallthrough with an explicit error:

```rust
// near_client.rs — parse_submit_blocks_response
if response.final_execution_outcome.is_none() {
    return Err(CustomError::TxExecutionError(
        "final_execution_outcome is None; cannot confirm contract execution".into(),
    ));
}
```

Alternatively, change `wait_until` to `TxExecutionStatus::Final` or `TxExecutionStatus::ExecutedOptimistic`, which guarantees `final_execution_outcome` is populated before the RPC returns. [4](#0-3)

    fn parse_submit_blocks_response(
        response: &RpcTransactionResponse,
    ) -> Result<SubmitResult, CustomError> {
        if let Some(ref final_execution_outcome) = response.final_execution_outcome {
            let outcome = final_execution_outcome.clone().into_outcome();

            if let near_primitives::views::FinalExecutionStatus::Failure(ref err) = outcome.status {
                let err_str = format!("{err:?}");
                if err_str.contains("PrevBlockNotFound") {
                    return Err(CustomError::PrevBlockNotFound);
                } else if err_str.contains("Exceeded the maximum amount of gas") {
                    return Err(CustomError::GasExceeded);
                }
                return Err(CustomError::TxExecutionError(err_str));
            }

            let gas_burnt = outcome.transaction_outcome.outcome.gas_burnt
                + outcome
                    .receipts_outcome
                    .iter()
                    .map(|r| r.outcome.gas_burnt)
                    .sum::<u64>();

            return Ok(SubmitResult { gas_burnt });
        }

        Ok(SubmitResult { gas_burnt: 0 })
    }
