### Title
Hardcoded `bonded=true` in `TierRedelegate.UpdateBonusCheckpoints` Causes Spurious Bonus Accrual for Unbonded Destination Validator Periods — (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`TierRedelegate` unconditionally passes `true` as the `lastKnownBonded` argument to `UpdateBonusCheckpoints` regardless of the destination validator's actual current bonded state. When the destination validator is currently unbonded at redelegate time, this causes `processEventsAndClaimBonus` to treat the post-redelegate gap as a bonded segment, resulting in over-payment from `RewardsPoolName`.

---

### Finding Description

In `TierRedelegate` (`msg_server.go:263`):

```go
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)
```

`UpdateBonusCheckpoints` sets three fields on the position:

```go
// types/position.go:65-69
func (p *Position) UpdateBonusCheckpoints(lastEventSeq uint64, t time.Time, lastKnownBonded bool) {
    p.LastEventSeq = lastEventSeq
    p.LastBonusAccrual = t
    p.LastKnownBonded = lastKnownBonded
}
```

The third argument is hardcoded to `true`. `LastKnownBonded` is the starting bonded state used by `processEventsAndClaimBonus` on the next claim:

```go
// claim_rewards.go:164
bonded := pos.LastKnownBonded
```

The events loop then computes bonus for each bonded segment:

```go
// claim_rewards.go:175-179
if bonded {
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
    totalBonus = totalBonus.Add(bonus)
}
```

The only defensive check against the actual validator state is at the **tail segment** (after all events are processed):

```go
// claim_rewards.go:206
if bonded && val.IsBonded() {
```

This guard does **not** protect segments computed inside the events loop.

**Attack path:**

1. Destination validator `V_dst` is currently unbonded (e.g., jailed, or newly created and not yet in the bonded set). It has `latestSeq=0` (no tieredrewards events, because hooks only record events when `getPositionCountForValidator > 0`, and no positions were on it yet).
2. Attacker calls `TierRedelegate` to `V_dst`. `validateRedelegatePosition` has no check on the destination validator's bonded state.
3. `claimRewards` settles the source validator events, correctly setting `LastKnownBonded=false` (source was unbonded).
4. `UpdateBonusCheckpoints(0, blockTime, true)` overrides `LastKnownBonded` to `true` and sets `LastBonusAccrual = blockTime`.
5. The position is now on `V_dst`. `getPositionCountForValidator(V_dst) > 0`.
6. `V_dst` later becomes bonded. `AfterValidatorBonded` fires, sees `count > 0`, and records a BOND event at seq=1 with timestamp `T_bond`.
7. Attacker calls `ClaimTierRewards`. `processEventsAndClaimBonus` runs:
   - `bonded = true` (from `LastKnownBonded`)
   - Processes BOND event (seq=1): `bonded=true` → computes bonus for `[redelegate_time, T_bond]` — **the validator was unbonded during this entire interval**
   - Tail check: `bonded && val.IsBonded()` = `true` → computes bonus for `[T_bond, now]` (legitimately bonded)
8. Attacker receives bonus for the unbonded gap `[redelegate_time, T_bond]`, which is an over-payment from `RewardsPoolName`.

The same path applies when `latestSeq > 0` and the last recorded event was an UNBOND: `UpdateBonusCheckpoints(N, blockTime, true)` still hardcodes `bonded=true`, and the next BOND event (seq=N+1) triggers the same spurious segment computation.

---

### Impact Explanation

The position accrues bonus for a period `[redelegate_time, T_bond]` during which the destination validator was not bonded. The bonus is paid out via `bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)`. This is a direct, concrete fund loss from the `RewardsPoolName` module account. The magnitude scales with position size, bonus APY, and the length of the unbonded gap.

---

### Likelihood Explanation

- `validateRedelegatePosition` imposes no restriction on the destination validator's bonded status. Cosmos SDK's `BeginRedelegate` also permits redelegation to unbonded validators.
- A newly created validator with insufficient power, or a jailed validator in the unbonding period, satisfies the precondition.
- The attacker controls the timing: they can redelegate immediately after a validator is jailed (while it is unbonding/unbonded) and claim after it is unjailed.
- No governance or privileged access is required; only a standard `MsgTierRedelegate` transaction.

---

### Recommendation

Replace the hardcoded `true` with the actual current bonded state of the destination validator:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

This ensures `LastKnownBonded` correctly reflects whether the destination validator was bonded at the moment of redelegate, so `processEventsAndClaimBonus` does not compute bonus for any unbonded gap preceding the first BOND event.

---

### Proof of Concept

**Keeper test outline (unmodified Go/Cosmos test setup):**

```go
func (s *KeeperSuite) TestTierRedelegate_UnbondedDst_NoBonusForGap() {
    // 1. Setup position on src validator (bonded).
    pos := s.setupNewTierPosition(sdkmath.NewInt(sdk.DefaultPowerReduction.Int64()), false)
    _, bondDenom := s.getStakingData()
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

    // 2. Create dst validator, then jail it so it is unbonded.
    dstValAddr, _ := s.createSecondValidator()
    s.jailAndUnbondValidator(dstValAddr)
    dstVal, _ := s.app.StakingKeeper.GetValidator(s.ctx, dstValAddr)
    s.Require().False(dstVal.IsBonded())

    // 3. Redelegate to unbonded dst validator.
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    tRedelegate := s.ctx.BlockTime()
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner: pos.Owner, PositionId: pos.Id, DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // 4. Advance time. Unjail dst validator → BOND event recorded.
    s.ctx = s.ctx.WithBlockTime(tRedelegate.Add(30 * 24 * time.Hour))
    consAddr, _ := dstVal.GetConsAddr()
    s.Require().NoError(s.app.StakingKeeper.Unjail(s.ctx, consAddr))
    _, _ = s.app.StakingKeeper.ApplyAndReturnValidatorSetUpdates(s.ctx)
    tBond := s.ctx.BlockTime()

    // 5. Advance more time, then claim.
    s.ctx = s.ctx.WithBlockTime(tBond.Add(10 * 24 * time.Hour))
    pos, _ = s.keeper.GetPositionState(s.ctx, pos.Id)
    bonus, err := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &pos)
    s.Require().NoError(err)

    // 6. Assert: bonus must be ZERO for [tRedelegate, tBond] (unbonded gap).
    //    With the bug, bonus is positive for the gap → test fails, proving over-payment.
    tier, _ := s.keeper.Tiers.Get(s.ctx, pos.TierId)
    currentRate, _ := s.keeper.GetTokensPerShare(s.ctx, dstValAddr)
    legitimateBonus := s.keeper.ComputeSegmentBonus(pos, tier, tBond, s.ctx.BlockTime(), currentRate)
    s.Require().Equal(legitimateBonus.String(), bonus.AmountOf(bondDenom).String(),
        "bonus must only cover the post-bond segment, not the unbonded gap")
}
```

With the current code, `bonus` includes the spurious `[tRedelegate, tBond]` segment, causing the assertion to fail and proving the over-payment.

```go
		ReferenceCount: count,
```

```go
func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}
```
