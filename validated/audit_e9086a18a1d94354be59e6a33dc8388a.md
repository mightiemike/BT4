Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of actual end-user, enabling full allowlist bypass via `MetricOmmSimpleRouter` — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives the `sender` argument from `MetricOmmPool.swap`, which passes `msg.sender` verbatim. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the originating user. Any pool that allowlists the router (required for router-mediated swaps to succeed) grants unrestricted swap access to every address on-chain.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← router address when called via router
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards this value verbatim as `sender` to every configured before-swap extension. `SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```

Here `msg.sender` is the pool and `sender` is the router — not the originating user. When `MetricOmmSimpleRouter.exactInputSingle` is called, it calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    ...
    params.extensionData
  );
```

The router passes no user identity in `extensionData` (it passes `""`), so the extension has no way to recover the actual caller. The check `allowedSwapper[pool][router]` passes for every user who routes through the router, regardless of whether they are individually allowlisted.

## Impact Explanation
The `SwapAllowlistExtension` is a production access-control guard whose stated purpose is to restrict which addresses may swap against a pool. When the router is allowlisted — the only way to enable router-based swaps — any unprivileged user can bypass the guard entirely by routing through `MetricOmmSimpleRouter`. Pools intended for restricted counterparties (institutional market makers, whitelisted integrators) become open to arbitrary swappers, breaking the core pool access-control invariant and potentially enabling unauthorized extraction of LP value or violation of regulatory/contractual access requirements.

## Likelihood Explanation
Any pool that deploys `SwapAllowlistExtension` as a before-swap hook and needs to support the public router must allowlist the router. The router is the primary user-facing entry point. The bypass requires no special privileges, no flash loans, and no oracle manipulation — only knowledge of the router address, which is public. The attack is repeatable by any address.

## Recommendation
The extension must gate the actual end-user, not the immediate caller of `pool.swap()`. Two viable approaches:

1. **Router attests the real user via `extensionData`**: The router encodes `msg.sender` (the actual user) into `extensionData` before forwarding to the pool. The extension decodes and checks that address, and also verifies `sender` is a known trusted router when `extensionData` carries a delegated identity.

2. **Reject intermediary calls**: Pools requiring strict allowlisting revert when `sender` is a known router contract, preventing router usage entirely for that pool.

At minimum, documentation must warn that allowlisting the router grants unrestricted swap access to all users.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension as before-swap extension
  pool admin calls setAllowedToSwap(pool, userA, true)    // allowlist userA
  pool admin calls setAllowedToSwap(pool, router, true)   // required for router usage

Attack:
  userC (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({
      pool: pool, tokenIn: token0, amountIn: X, recipient: userC, extensionData: ""
    })

  Execution trace:
    router.exactInputSingle()
      → pool.swap(recipient=userC, ...) [msg.sender = router]
        → _beforeSwap(sender=router, ...)
          → SwapAllowlistExtension.beforeSwap(sender=router, ...)
            → allowedSwapper[pool][router] == true  ✓ (passes)
        → swap executes, userC receives output tokens

Result: userC, not on the allowlist, successfully swaps against the restricted pool.
