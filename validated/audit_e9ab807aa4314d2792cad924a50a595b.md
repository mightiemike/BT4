Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, making allowlist bypassable for all router-mediated swaps — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is the pool's `msg.sender`. When `MetricOmmSimpleRouter` mediates a swap, the pool's `msg.sender` is the router contract, not the end user. The allowlist therefore gates the router address, not the actual swapper. A pool admin cannot simultaneously allow legitimate users to use the router and block non-allowlisted users.

## Finding Description

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, passing the direct caller of the pool as `sender`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is the pool's caller: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`. The end user (`msg.sender` of the router call) is stored only in transient callback context for payment settlement and is never forwarded to the pool or extensions: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. The binary outcome for the pool admin:

| Router allowlisted? | Allowlisted user via router | Non-allowlisted user via router |
|---|---|---|
| No | Blocked (broken UX) | Blocked |
| Yes | Allowed | **Allowed — bypass** |

No configuration allows allowlisted users to use the router while blocking non-allowlisted users.

## Impact Explanation

Any user can bypass a `SwapAllowlistExtension` guard by routing through `MetricOmmSimpleRouter`. If the pool admin allowlists the router to restore normal UX for legitimate users, the allowlist is completely ineffective. Non-allowlisted addresses (e.g., non-KYC'd or sanctioned counterparties) can execute swaps against the pool at oracle-quoted prices, constituting a direct loss of LP principal and a broken core pool invariant: the configured access control does not gate the actual swapper.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap interface. Any user who calls `exactInputSingle` or `exactInput` through the router triggers the bypass without any privileged access, special setup, or non-standard tokens. The bypass is automatic for any router-mediated swap against a pool where the router is allowlisted.

## Recommendation

The extension must check the identity the pool admin intends to gate — the end user — not the intermediary router. Two complementary approaches:

1. **Extension-side**: Extend `beforeSwap` to also check an optional user address passed via `extensionData`, so router-mediated swaps can supply the real user identity for the allowlist check.
2. **Router-side**: Have the router encode the end user's address into `extensionData` so the extension can verify it.

Alternatively, document clearly that `SwapAllowlistExtension` only gates direct pool callers and is incompatible with router-mediated flows, and provide a router-aware allowlist extension that verifies the end user via `extensionData`.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured on beforeSwap.
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (necessary so legitimate users can swap via router).
  - Alice (allowlisted) and Bob (not allowlisted) both use the router.

Attack:
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
  2. Router calls pool.swap(...) — pool's msg.sender = router address.
  3. Pool calls _beforeSwap(sender=router, ...).
  4. SwapAllowlistExtension checks allowedSwapper[pool][router] → true.
  5. Swap executes. Bob receives output tokens.

Result:
  Bob, a non-allowlisted address, successfully swaps against the pool.
  The SwapAllowlistExtension guard is completely bypassed.
```
