Audit Report

## Title
SwapAllowlistExtension Per-User Allowlist Bypassed via MetricOmmSimpleRouter — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is always `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, the router becomes `msg.sender`, so the extension checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any pool admin who allowlists the router to permit approved users to trade via it simultaneously grants every unpermissioned address the ability to bypass the per-user allowlist by routing through the same contract.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the extension caller). `sender` is the first argument forwarded from `MetricOmmPool.swap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // direct caller of pool.swap()
    ...
);
```

`ExtensionCalling._beforeSwap` (L149-177) passes `sender` unchanged into `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

When `MetricOmmSimpleRouter.exactInputSingle` (L72-80) calls `pool.swap(...)`, the router is `msg.sender` inside the pool, so `sender = router`. The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. The same identity loss applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput` (all recursive callback hops via `_exactOutputIterateCallback` L220-228).

**Exploit path:**
1. Pool P is configured with `SwapAllowlistExtension`; `allowAllSwappers[P] = false`.
2. Admin sets `allowedSwapper[P][alice] = true` (alice is the only approved trader).
3. Admin sets `allowedSwapper[P][router] = true` so alice can trade via the router.
4. Bob (never allowlisted) calls `router.exactInputSingle({pool: P, ...})`.
5. Router calls `pool.swap(...)` → `msg.sender = router` inside pool.
6. Pool calls `_beforeSwap(router, ...)` → extension checks `allowedSwapper[P][router] == true` → no revert.
7. Bob's swap executes; LP liquidity is consumed by an actor the pool was designed to exclude.

No existing guard prevents this: the extension has no mechanism to distinguish a router-forwarded call from a direct call, and no check on `extensionData` for an encoded user identity.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a known set of counterparties (e.g., KYC-verified addresses, whitelisted market makers) has its access control fully neutralised for any user who routes through `MetricOmmSimpleRouter`. The attacker executes real swaps settled at oracle prices, debiting actual LP liquidity. The pool admin's allowlist configuration is rendered meaningless for the router path, which is the primary user-facing entry point. This constitutes a broken core pool functionality causing loss of funds (LP principal exposed to adversarial trading from explicitly excluded actors), meeting the contest's allowed impact gate.

## Likelihood Explanation
The prerequisite — the router being allowlisted — is the natural and expected admin action for any pool that wants its approved users to trade via the standard periphery interface. The attacker requires no special privilege, no tokens beyond swap input, and no setup beyond calling a public function on a deployed contract. The condition is self-created by correct admin behaviour, making exploitation highly feasible whenever the extension is deployed with router support intended.

## Recommendation
The extension must gate on the actual end-user identity, not the direct caller of `pool.swap()`. The safest fix is:

1. **Router encodes `msg.sender` into `extensionData`** before forwarding to the pool. The extension decodes the user identity from `extensionData` when `sender` is a factory-registered router, and checks `allowedSwapper[pool][decodedUser]` instead of `allowedSwapper[pool][router]`.
2. **Factory-registered router allowlist**: the extension trusts the encoded user identity only when `sender` is a factory-approved router address, preventing arbitrary contracts from spoofing a user identity via `extensionData`.

For direct calls (where `sender` is not a known router), the extension continues to check `allowedSwapper[pool][sender]` as today.
