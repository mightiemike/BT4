### Title
P2P Mempool Propagation Silently Rejects Valid V3 Transactions with Zero L2/L1DataGas Bounds Due to `ValidResourceBounds` Deserialization Mismatch — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` classifies any V3 transaction whose `l2_gas` and `l1_data_gas` resource bounds are both zero as a pre-0.13.3 `L1Gas` transaction. The gateway, however, always treats V3 transactions as `AllResources`. This creates a split: the gateway accepts and hashes the transaction under the `AllResources` branch (3 resource elements), while every peer that receives it over P2P deserializes it under the `L1Gas` branch and immediately rejects it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction is permanently stuck in the originating node's mempool and never sequenced.

### Finding Description

**Divergent deserialization path — the analog to the external bug**

The external bug had `WalkerLib.modify` using `treeTick(highTick) - 1` while `PoolWalker.settle` used `treeTick(highTick)`, so the two functions walked different routes through the same tree. The sequencer analog is: the gateway always wraps V3 resource bounds as `ValidResourceBounds::AllResources`, while the P2P protobuf deserializer can silently downgrade the same bytes to `ValidResourceBounds::L1Gas`, causing every downstream consumer to operate on a structurally different object.

**Gateway path (always `AllResources`)**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds`. Its `InvokeTransactionV3Trait` implementation unconditionally returns `ValidResourceBounds::AllResources(self.resource_bounds)`. The hash is therefore computed by `get_tip_resource_bounds_hash` over **three** packed resource felts: `[tip, L1_GAS_concat, L2_GAS_concat, L1_DATA_GAS_concat]`. [1](#0-0) [2](#0-1) 

**P2P deserialization path (can produce `L1Gas`)**

When a peer receives the transaction over the mempool P2P channel, the conversion chain is:

```
protobuf::InvokeV3WithProof
  → InvokeTransactionV3          (via TryFrom<protobuf::InvokeV3> for InvokeTransactionV3)
  → RpcInvokeTransactionV3       (via TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3)
```

Step 1 calls `ValidResourceBounds::try_from(protobuf::ResourceBounds)`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← 2-element hash path
} else {
    ValidResourceBounds::AllResources(...)       // ← 3-element hash path
})
``` [3](#0-2) 

Step 2 then rejects the `L1Gas` variant:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => return Err(StarknetApiError::OutOfRange { ... }),
}
``` [4](#0-3) 

The `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` propagates this as `DEPRECATED_RESOURCE_BOUNDS_ERROR`: [5](#0-4) 

**The `AllResourceBounds`-direct path is not used here**

A separate `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` exists in `rpc_transaction.rs` that would preserve all three fields correctly, but it is **not** on the deserialization path for `RpcInvokeTransactionV3`. That path goes through `InvokeTransactionV3` first, which uses the lossy `ValidResourceBounds::try_from`. [6](#0-5) 

### Impact Explanation

A user submits a V3 invoke transaction with `AllResourceBounds` where both `l2_gas` and `l1_data_gas` are `ResourceBounds { max_amount: 0, max_price_per_unit: 0 }` (i.e., `ResourceBounds::default()`). The originating node's gateway accepts it, computes the hash under the `AllResources` branch, and stores it in the local mempool. When the node broadcasts the transaction to peers via the mempool P2P channel, every peer's deserialization fails with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction is never forwarded to other nodes and never sequenced, even though it passed gateway admission. This matches the **High** impact category: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

Additionally, if such a transaction were included in a block (by the originating sequencer), peers syncing the block would compute the transaction hash under the `L1Gas` branch (2 resource elements), producing a different hash than the one stored in the block, which would constitute a **Critical** wrong receipt/hash result.

### Likelihood Explanation

The trigger condition — `l2_gas.is_zero() && l1_data_gas.is_zero()` — requires both `max_amount = 0` and `max_price_per_unit = 0` for both resources. This is the `Default` value for `ResourceBounds`. A user who submits a V3 transaction without explicitly setting L2 or L1 data gas bounds (relying on defaults) would hit this path. The condition is narrow but requires no special privilege and is reachable by any unprivileged user of the RPC gateway.

### Recommendation

Replace the two-step deserialization path in `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` with the direct `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` converter that already exists in `rpc_transaction.rs`. This converter preserves all three fields unconditionally and does not apply the `is_zero()` downgrade:

```rust
// In TryFrom<protobuf::InvokeV3> for InvokeTransactionV3 (or a new direct path):
let resource_bounds = AllResourceBounds::try_from(
    value.resource_bounds.ok_or(missing("InvokeV3::resource_bounds"))?,
)?;
```

Alternatively, remove the `is_zero()` downgrade from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` entirely, since the protobuf wire format already encodes the version distinction via the presence/absence of `l1_data_gas` (which is `optional` in the proto schema). [7](#0-6) 

### Proof of Concept

1. Construct a V3 invoke transaction at the gateway with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },  // default
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },  // default
   }
   ```
2. The gateway accepts it; `InternalRpcInvokeTransactionV3::resource_bounds()` returns `ValidResourceBounds::AllResources(...)`. `get_tip_resource_bounds_hash` hashes 3 resource felts → `tx_hash_A`.
3. The gateway serializes the transaction to `protobuf::InvokeV3WithProof` and broadcasts it.
4. A peer receives the protobuf. `TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` calls `ValidResourceBounds::try_from(...)`. Since `l2_gas.is_zero() && l1_data_gas.is_zero()` is true, it returns `ValidResourceBounds::L1Gas(l1_gas)`.
5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` matches `L1Gas` against the `_` arm and returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`.
6. The peer drops the transaction. It never enters the peer's mempool and is never sequenced.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
