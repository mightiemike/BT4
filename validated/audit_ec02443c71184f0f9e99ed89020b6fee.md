### Title
`ValidResourceBounds::AllResources` Silently Collapses to `L1Gas` in Protobuf Round-Trip, Producing a Divergent Transaction Hash - (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a value-based heuristic: if both `l2_gas` and `l1_data_gas` are zero it emits `ValidResourceBounds::L1Gas`, otherwise `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound felts depending on the variant (2 for `L1Gas`, 3 for `AllResources`), a V3 invoke transaction submitted via RPC with `AllResourceBounds{l2_gas:0, l1_data_gas:0}` receives hash **H1** at the gateway, but any node that deserializes the same transaction from a protobuf block-sync message recomputes hash **H2 ≠ H1**. This is the sequencer analog of the TreasuryV2 strategy-update bug: a tracked invariant (`strategyUnderlying` ↔ the resource-bounds variant) is silently reset during a representation boundary crossing, causing downstream consumers to operate on stale/wrong state.

### Finding Description

**Step 1 – Submission path (hash H1 computed with `AllResources`).**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`. The `TransactionConverter` converts it to `InternalRpcInvokeTransactionV3`, whose `InvokeTransactionV3Trait::resource_bounds()` always returns `ValidResourceBounds::AllResources(self.resource_bounds)`. [1](#0-0) 

`get_invoke_transaction_v3_hash` then calls `get_tip_resource_bounds_hash` with that `AllResources` value. For `AllResources`, the function appends a third element (`L1_DATA_GAS` packed felt) to the hash input: [2](#0-1) 

So for `AllResources{l1_gas:X, l2_gas:0, l1_data_gas:0}` the hash preimage is `[tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)]` → **H1**.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
