Audit Report

## Title
Synthetic-ratio integer truncation to zero permanently stalls `getBidAndAskPrice` for low-value base tokens ‚Äî (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary
In `AnchoredPriceProvider._getBidAndAskPrice`, when `quoteFeedId` is set, the synthetic ratio `mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2)` silently truncates to zero via integer division whenever `mid_base * 1e8 < mid_quote`. No zero-check is performed on the resulting `mid`. The zero value propagates into `_computeBidAsk`, which returns the `(0, type(uint128).max)` sentinel, causing `getBidAndAskPrice` to revert with `FeedStalled()` on every call, permanently disabling swaps through the pool.

## Finding Description
In `_getBidAndAskPrice` (lines 263‚Äì268), after both legs pass their individual guards, the ratio is computed:

```solidity
if (!ok2 || mid2 == 0) return (0, type(uint128).max);
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ‚Üê no post-check
spreadBps += spreadBps2;
```

`ORACLE_DECIMALS = 1e8`. When `mid_base * 1e8 < mid_quote`, Solidity integer division truncates to zero. The guard at line 265 only validates `mid2 != 0` (the denominator), not the result of the division.

The zero `mid` is forwarded to `_computeBidAsk` (lines 309‚Äì312):

```solidity
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
    return (0, type(uint128).max);
}
```

`_bandEdge(0, ...) = Math.mulDiv(0, ...) = 0`, so `refBid == 0` triggers the guard and returns the stall sentinel. `getBidAndAskPrice` (line 216) then reverts unconditionally:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
```

The `_readLeg` function (line 287) validates `mid != 0` per-leg, so a base token with `mid = 1000` (valid, non-zero) passes all per-leg guards. The truncation occurs only at the ratio step, which has no post-division guard.

## Impact Explanation
Every swap through a pool whose `AnchoredPriceProvider` is configured in synthetic ratio mode with a low-value base token reverts with `FeedStalled()`. This is broken core pool functionality: the pool's primary trading mechanism is disabled under normal, valid oracle conditions. LPs who have deposited liquidity into the pool cannot execute trades; the pool is insolvent as a trading venue for the duration of the price disparity. This meets the "Broken core pool functionality causing unusable swap flows" threshold.

## Likelihood Explanation
Synthetic ratio mode (`quoteFeedId != bytes32(0)`) is an explicitly supported and documented feature. The truncation condition (`mid_base * 1e8 < mid_quote`) is met whenever the base token's USD price is less than `quote_price_USD / 1e8`. For ETH at $3,000, any base token priced below $0.00003 triggers this ‚Äî a condition satisfied by a large class of real tokens (SHIB, PEPE, and similar micro-cap assets). The condition is persistent for as long as the price ratio remains large, making the DoS non-transient. No privileged action is required; the condition arises from normal oracle price reporting.

## Recommendation
Add a zero-check on the computed ratio immediately after the `Math.mulDiv` call in `_getBidAndAskPrice`:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max); // ratio truncated ‚Äî fail closed
spreadBps += spreadBps2;
```

This mirrors the existing per-leg guard pattern and ensures the contract fails closed rather than forwarding a semantically invalid zero price.

## Proof of Concept
1. Deploy `AnchoredPriceProvider` with `baseFeedId` = SHIB/USD feed, `quoteFeedId` = ETH/USD feed.
2. Oracle reports SHIB `mid = 1000` (8-decimal, ‚âà $0.00001), ETH `mid = 300_000_000_000` (8-decimal, ‚âà $3,000). Both feeds are fresh, non-stale, within price guards.
3. `_readLeg(baseFeedId)` returns `ok = true`, `mid = 1000`. `_readLeg(quoteFeedId)` returns `ok2 = true`, `mid2 = 300_000_000_000`. Guard at line 265 passes (`ok2 = true`, `mid2 != 0`).
4. `Math.mulDiv(1000, 1e8, 300_000_000_000)` = `1e11 / 3e11` = `0` (integer truncation).
5. `_computeBidAsk(0, spreadBps)`: `_bandEdge(0, ...) = 0` ‚Üí `refBid == 0` ‚Üí returns `(0, type(uint128).max)`.
6. `getBidAndAskPrice()` reverts with `FeedStalled()`.
7. Every swap through the pool reverts; the pool is permanently unusable for trading under these price conditions.
