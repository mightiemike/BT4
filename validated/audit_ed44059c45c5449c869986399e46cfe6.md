### Title
Migration Under-Pays Bonus Rewards for V1 Positions with `LastKnownBonded=false` During Forced Exit — (`x/tieredrewards/migrations/v2/migrate.go`)

### Summary

The v2 migration force-exits all vesting-owned positions via `ForceFullExitWithDelegation`. For positions created under v1 (before `LastKnownBonded` existed), the field deserializes as `false`. `processEventsAndClaimBonus` uses this value as the starting bonded state. If the validator has been continuously bonded since position creation (no UNBOND/BOND events), the event loop is a no-op and the final segment guard `bonded && val.IsBonded()` evaluates to `false && true = false`, paying zero bonus. The rewards pool retains the accrued coins, violating the invariant that all accrued bonus must be settled before position deletion.

### Finding Description

`processEventsAndClaimBonus` initializes `bonded` from `pos.LastKnownBonded`:

```go
bonded := pos.LastKnownBonded   // false for all v1 positions
segmentStart := pos.LastBonusAccrual
```

For a v1 position on a validator that has been continuously bonded (no events recorded since `pos.LastEventSeq`):

- The `for _, entry := range events` loop body never executes (empty slice).
- `bonded` remains `false`.
- The post-loop guard `if bonded && val.IsBonded()` is `false && true = false`.
- `totalBonus` stays zero; no coins are sent to the owner.

The migration does not backfill `LastKnownBonded=true` for positions whose validator is currently bonded. New positions set this field explicitly to `true` at creation time, but v1 positions in the store have the proto zero-value `false`.

### Impact Explanation

Vesting account owners with v1 positions on continuously-bonded validators receive **zero bonus rewards** during the forced migration exit, regardless of how long the position was held. The rewards pool retains the accrued coins permanently (no future claim path exists after position deletion). The economic loss is:

```
loss = shares × tokensPerShare × bonusApy × durationSeconds / SecondsPerYear
```

For a 1M token position held 1 year at 4% APY, the loss is ~40,000 tokens.

### Likelihood Explanation

This is a deterministic, one-time migration path triggered at the v8 upgrade. Every vesting account owner whose position was created under v1 on a validator that has never unbonded since position creation is affected. The condition (bonded validator, no UNBOND/BOND events) is the common case for a healthy validator.

### Recommendation

In `backfillDelegatorAddress` (or a separate migration step), backfill `LastKnownBonded` by querying the current validator bond status for each position:

```go
val, err := stakingKeeper.GetValidator(ctx, valAddr)
if err == nil && val.IsBonded() {
    pos.LastKnownBonded = true
}
```

This must run **before** `exitVestedAccountsPositions` so that `processEventsAndClaimBonus` starts with the correct bonded state.

### Proof of Concept

1. Create a v1 position (set `LastKnownBonded=false` directly in store) on a bonded validator.
2. Advance block time by N days without triggering any validator UNBOND/BOND events.
3. Call `ForceFullExitWithDelegation(posID)`.
4. Observe `bonusRewards` returned is zero.
5. Compute expected bonus with `LastKnownBonded=true` using `computeSegmentBonus`; assert the difference is non-zero.
6. Confirm the rewards pool balance is unchanged (coins retained).

The existing test `TestForceFullExitWithDelegation_VestingOwner_CommitOrigin` uses `advanceForRewards` which triggers validator events, masking this bug — a test with no intervening events would expose it. [7](#0-6)

func backfillDelegatorAddress(ctx context.Context, positions collections.Map[uint64, types.Position]) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	sdkCtx.Logger().Info("v8 migration: backfill delegator address")
	iter, err := positions.Iterate(ctx, nil)
	if err != nil {
		return err
	}
	defer iter.Close()

	kvs, err := iter.KeyValues()
	if err != nil {
		return err
	}

	for _, kv := range kvs {
		pos := kv.Value
		pos.DelegatorAddress = LegacyDelegatorAddress(pos.Id)
		if _, err := sdk.AccAddressFromBech32(pos.DelegatorAddress); err != nil {
			return fmt.Errorf("backfill produced invalid delegator address for position %d: %w", pos.Id, err)
		}
		if err := positions.Set(ctx, pos.Id, pos); err != nil {
			return err
		}
	}
	sdkCtx.Logger().Info("v8 migration: delegator address backfilled", "count", len(kvs))
	return nil

// TestForceFullExitWithDelegation_VestingOwner_CommitOrigin verifies the
// scenario where the vesting account's tier position originated from
// CommitDelegationToTier. The owner first delegated normally (so DV was
// populated by the bank-side TrackDelegation hook), then moved that delegation
// into a position. Pre-migration, DV+DF is stale-high. After force-exit, the
// returning delegation closes the gap; no top-up is required and DV+DF must
// remain unchanged.
func (s *KeeperSuite) TestForceFullExitWithDelegation_VestingOwner_CommitOrigin() {
	s.setupTier(1)
	vals, bondDenom := s.getStakingData()
	val := vals[0]
	valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	lockedAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	owner := s.newVestingOwnerWithBalance(bondDenom, lockedAmount, lockedAmount)

	pos := s.createCommitPositionV1(owner, val, valAddr, lockedAmount)
	initialDV := s.delegatedVesting(owner).AmountOf(bondDenom)
	initialDF := s.delegatedFree(owner).AmountOf(bondDenom)
	s.Require().Equal(lockedAmount, initialDV,
		"after the original Delegate, DV must equal the delegated locked amount")
	s.Require().True(initialDF.IsZero(), "DF must be zero for a fully-locked delegation")

	// Pre-migration: delegation is on the position, owner has none. DV+DF
	// is stale-high relative to the owner's actual delegations.
	s.Require().True(s.totalDelegated(owner).IsZero(),
		"owner has no on-chain delegation pre force-exit; tier position holds it")
	s.Require().Equal(lockedAmount, s.trackedTotal(owner, bondDenom),
		"DV+DF should be stale-high (= lockedAmount) before force exit")

	s.advanceForRewards(valAddr, bondDenom)

	s.Require().NoError(s.keeper.ForceFullExitWithDelegation(s.ctx, pos.Id))

	// Position deleted, delegation back at the owner.
	_, err := s.keeper.GetPosition(s.ctx, pos.Id)
	s.Require().Error(err)
	ownerDel := s.totalDelegated(owner)
	s.Require().Equal(lockedAmount, ownerDel,
		"owner must hold a delegation equal to the original locked amount")

	// DV unchanged: the stale-high tracking now correctly reflects the
	// returned delegation. No top-up was needed.
	s.Require().Equal(initialDV, s.delegatedVesting(owner).AmountOf(bondDenom),
		"DV must remain unchanged across the round trip")
	s.Require().Equal(initialDF, s.delegatedFree(owner).AmountOf(bondDenom),
		"DF must remain unchanged across the round trip")

	// Final invariant.
	s.Require().Equal(ownerDel, s.trackedTotal(owner, bondDenom),
		"DV + DF must equal Σ delegations after force exit")
}
