### Title
Consensus Proposal Permanently Rejected Due to `AllResources`→`L1Gas` Type Collapse in Protobuf Round-Trip — (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` collapses any `AllResources { l1_gas: X, l2_gas: ZERO, l1_data_gas: ZERO }` into `L1Gas(X)`. Because the gateway accepts V3 invoke transactions carrying exactly that bound combination, a single such transaction causes every consensus proposal that includes it to be permanently rejected by validators, stalling the sequencer.

---

### Finding Description

**Serialization side (proposer):**

`RpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds`. When the proposer serializes a `ConsensusTransaction` to protobuf it goes through:

```
RpcInvokeTransactionV3 → InvokeTransactionV3 (resource_bounds = AllResources(…)) → protobuf::InvokeV3
```

The serialization always succeeds and emits `l2_gas` and `l1_data_gas` as zero-value `ResourceLimits` messages.

**Deserialization side (validator):**

`TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` calls `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3`:

```rust
// crates/apollo_protobuf/src/converters/rpc_transaction.rs:115-131
let snapi_invoke: InvokeTransactionV3 = value.invoke.ok_or(...)?.try_into()?;
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
```

The inner `try_into()` for `InvokeTransactionV3` uses `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:417-436
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant
} else {
    ValidResourceBounds::AllResources(…)
})
```

When `l2_gas=0` and `l1_data_gas=0`, this returns `L1Gas`. Then `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` rejects it:

```rust
// crates/starknet_api/src/rpc_transaction.rs:586-611
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { … });  // ← fires
    }
},
```

The error propagates as `DEPRECATED_RESOURCE_BOUNDS_ERROR`, the `TransactionBatch` deserialization fails, and the validator marks the proposal as `Failed`.

**The codebase itself acknowledges this invariant break.** The test helper `add_gas_values_to_transaction` exists solely to paper over it:

```rust
// crates/apollo_protobuf/src/converters/consensus_test.rs:26-43
// If all the fields of `AllResources` are 0 upon serialization,
// then the deserialized value will be interpreted as the `L1Gas` variant.
fn add_gas_values_to_transaction(transactions: &mut [ConsensusTransaction]) {
    …
    resource_bounds.l2_gas.max_amount = GasAmount(1);  // workaround
}
```

This workaround is applied in every consensus round-trip test but is absent from the production path.

**Gateway admits the trigger transaction.** The stateless validator explicitly allows `AllResourceBounds { l1_gas: non_zero, l2_gas: ZERO, l1_data_gas: ZERO }`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator_test.rs:70-82
#[case::valid_l1_gas(
    StatelessTransactionValidatorConfig { validate_resource_bounds: true, … },
    RpcTransactionArgs {
        resource_bounds: AllResourceBounds {
            l1_gas: NON_EMPTY_RESOURCE_BOUNDS,
            ..Default::default()   // l2_gas=0, l1_data_gas=0
        },
        …
    }
)]
```

Once such a transaction enters the mempool it is never purged: the proposer executes it successfully (the blockifier uses `AllResources` from the internal representation, not the protobuf path), includes it in every proposal, and every validator rejects the proposal. Consensus stalls.

---

### Impact Explanation

Every consensus proposal that contains a V3 invoke transaction with `l2_gas=0` and `l1_data_gas=0` is permanently rejected by all validators. The transaction is never removed from the mempool, so every subsequent proposer re-includes it and suffers the same fate. This matches **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

---

### Likelihood Explanation

The trigger is a single, valid, fee-paying invoke V3 transaction with only L1 gas bounds set. This is a normal pre-0.13.3 usage pattern. Any user (or attacker) who submits such a transaction can trigger the stall. No privileged access is required.

---

### Recommendation

Replace the heuristic type-collapse in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminator field in the protobuf schema, or always deserialize into `AllResources` when all three gas fields are present (even if zero), matching the serialization invariant. The `rpc_transaction.rs` path already does this correctly via `TryFrom<protobuf::ResourceBounds> for AllResourceBounds`, which requires all three fields and never collapses to `L1Gas`.

---

### Proof of Concept

1. Submit the following invoke V3 transaction to the gateway:
   ```json
   {
     "type": "INVOKE",
     "version": "0x3",
     "resource_bounds": {
       "l1_gas": { "max_amount": "0x1", "max_price_per_unit": "0x1" },
       "l2_gas": { "max_amount": "0x0", "max_price_per_unit": "0x0" }
     },
     …
   }
   ```
   The gateway accepts it (stateless validator `valid_l1_gas` case). [1](#0-0) 

2. The transaction enters the mempool as `InternalRpcTransaction` with `resource_bounds: AllResourceBounds { l1_gas: {1,1}, l2_gas: ZERO, l1_data_gas: ZERO }`. [2](#0-1) 

3. The proposer includes it in a `ConsensusTransaction`, serializes to protobuf. Serialization succeeds because `From<RpcInvokeTransactionV3> for InvokeTransactionV3` wraps the bounds in `AllResources` before encoding. [3](#0-2) 

4. The validator deserializes the protobuf. `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` sees `l2_gas.is_zero() && l1_data_gas.is_zero()` and returns `L1Gas`. [4](#0-3) 

5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` hits the `_ => Err(…)` arm and returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`. [5](#0-4) 

6. `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` propagates the error; the `TransactionBatch` is rejected; the proposal is marked `Failed`. [6](#0-5) 

7. The transaction is never evicted from the mempool. Every subsequent proposer re-includes it. Consensus stalls indefinitely.

The test file explicitly acknowledges this collapse and works around it by forcing `l2_gas.max_amount = GasAmount(1)` in every round-trip test, confirming the production path is unprotected. [7](#0-6)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ConsensusTransaction) -> Result<Self, Self::Error> {
        let txn = value.txn.ok_or(missing("ConsensusTransaction::txn"))?;
        let txn = match txn {
            protobuf::consensus_transaction::Txn::DeclareV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Declare(
                    RpcDeclareTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::DeployAccountV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::DeployAccount(
                    RpcDeployAccountTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
                    RpcInvokeTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::L1Handler(txn) => {
                ConsensusTransaction::L1Handler(txn.try_into()?)
            }
        };
        Ok(txn)
    }
}
