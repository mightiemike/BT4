Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address as `sender` instead of end user, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender`. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. This creates two irreconcilable failure modes: allowlisted users are locked out of the router, or if the admin allowlists the router to fix that, any non-allowlisted user can bypass the curated pool's access control entirely.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` verbatim as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that same `sender` into the extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is allowlisted: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly — so the pool's `msg.sender` is the router, not the end user. The original `msg.sender` is stored only in transient callback context for payment purposes, never forwarded as the swap originator: [4](#0-3) 

The allowlist is administered per-user address by the pool admin: [5](#0-4) 

**Scenario A (Lockout):** Admin allowlists specific user addresses but not the router. Allowlisted users calling `exactInputSingle` are rejected because the extension sees `sender = router`, which is not allowlisted.

**Scenario B (Bypass):** Admin allowlists the router to enable router-mediated swaps. Now `allowedSwapper[pool][router] = true`. Any non-allowlisted user calling `MetricOmmSimpleRouter.exactInputSingle` on the curated pool passes the check — the extension sees `sender = router` (allowlisted) and the swap executes, completely bypassing the curation policy.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly gates on `owner` (the position beneficiary, second parameter), not `sender` (the payer/adder contract, first parameter), so the deposit path does not share this flaw: [6](#0-5) 

## Impact Explanation
Scenario B is the critical path: a non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle` on a curated pool whose admin was forced to allowlist the router. The extension passes, the swap executes at oracle price, and the user receives output tokens they were explicitly barred from receiving. This is a direct policy bypass on a production curated pool with real LP assets at stake — broken core pool functionality / unauthorized swap execution. Scenario A additionally breaks the supported periphery path for all legitimate allowlisted users.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entrypoint for the protocol. Any pool that configures `SwapAllowlistExtension` and expects users to swap through the router will encounter one of the two failure modes immediately upon first use. No special preconditions, flash loans, or privileged access are required — a standard `exactInputSingle` call is sufficient.

## Recommendation
The extension must check the original end user, not the intermediary router. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`**: forward the original `msg.sender` inside `extensionData` for every swap call so the extension can recover it.
2. **In `SwapAllowlistExtension.beforeSwap`**: if `sender` is a recognized router, decode the true originator from `extensionData` and check that address instead.

The simplest correct fix is to have `MetricOmmSimpleRouter` ABI-encode `msg.sender` into `extensionData` for every hop, and have `SwapAllowlistExtension` decode and check that address when `sender` is a recognized router.

## Proof of Concept
```solidity
// Pool configured with SwapAllowlistExtension.
// Admin allowlists the router so that allowlisted users can swap via it.
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// alice is NOT allowlisted
assertFalse(swapExtension.isAllowedToSwap(address(pool), alice));

// alice calls the router — sender seen by extension = router (allowlisted)
vm.startPrank(alice);
token1.approve(address(router), type(uint256).max);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token1),
        recipient: alice,
        deadline: block.timestamp + 1,
        zeroForOne: false,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        extensionData: ""
    })
);
vm.stopPrank();
// alice successfully swapped on a pool she was explicitly barred from — allowlist bypassed.
```
