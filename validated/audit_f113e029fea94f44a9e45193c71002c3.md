Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as `sender` Instead of Actual User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of its own `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router address, not the end user. A pool admin who allowlists the router to support router-mediated swaps for curated users inadvertently grants unrestricted swap access to any address that calls through the router, completely defeating the allowlist invariant.

## Finding Description
In `MetricOmmPool.swap`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to every configured extension via `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool passed — the router address when the call came through the router: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no mechanism to forward the original `msg.sender` (the end user) into the pool's `sender` slot. The router stores the original caller only in transient storage for the payment callback, not for the extension check: [4](#0-3) 

The result: when `alice` (not allowlisted) calls `router.exactInputSingle(...)`, the pool sees `sender = address(router)`. If the router is allowlisted, `allowedSwapper[pool][router]` is `true`, and the check passes for alice despite her not being in the allowlist. The same flaw applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all router entry points call `pool.swap()` with `msg.sender` as the effective `sender`.

## Impact Explanation
A pool admin who deploys a curated pool with `SwapAllowlistExtension` and allowlists `MetricOmmSimpleRouter` to support router-based swaps for their curated users exposes the pool to unrestricted swaps by any address. Non-allowlisted users can drain liquidity from the pool at oracle-derived prices, bypassing the access control the pool was designed to enforce. This constitutes direct loss of LP principal and a complete failure of the allowlist invariant — matching the "Admin-boundary break bypassed by an unprivileged path" and "direct loss of user principal" allowed impact categories.

## Likelihood Explanation
Any production pool that configures `SwapAllowlistExtension` in its `beforeSwap` hook order and allowlists `MetricOmmSimpleRouter` to support router-based swaps for its curated users is immediately and trivially exploitable by any address. The router is a supported, documented periphery contract; allowlisting it is the expected operational step. No special privileges, flash loans, or oracle manipulation are required — a single `exactInputSingle` call suffices.

## Recommendation
The extension must gate on the economically relevant actor — the end user — not the intermediary. The preferred fix is a pool-level change: pass an explicit `swapper` (the original end user) separately from `sender` (the immediate caller) so extensions can gate on the correct actor regardless of routing path. As an interim measure, `SwapAllowlistExtension` should document that it cannot be used safely with any intermediary router, and the allowlist must contain only direct callers. Alternatively, the router could encode the real user in `extensionData` and the extension could decode and verify it, though this requires coordinated changes to both contracts.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, alice NOT allowlisted
swapExtension.setAllowedToSwap(address(pool), address(router), true); // admin enables router
// alice is NOT in the allowlist

// Direct swap by alice — correctly blocked:
vm.prank(alice);
pool.swap(alice, true, 1000, type(uint128).max, "", "");
// → reverts NotAllowedToSwap ✓

// Router swap by alice — allowlist bypassed:
vm.prank(alice);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: token0,
    tokenOut: token1,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: alice,
    deadline: block.timestamp + 1,
    priceLimitX64: type(uint128).max,
    extensionData: ""
}));
// → succeeds; pool sees sender=router (allowlisted), alice swaps without being in the allowlist ✗
```
