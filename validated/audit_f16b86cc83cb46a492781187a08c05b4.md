Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps on the `sender` argument, which the pool sets to `msg.sender` at the time `pool.swap()` is called. When `MetricOmmSimpleRouter` is used, `msg.sender` to the pool is the router contract, not the end user. A pool admin who allowlists the router to enable router-based swaps for their curated user set inadvertently grants every unprivileged user the ability to bypass the allowlist entirely.

## Finding Description

**Root cause — pool passes direct caller as `sender`:**

`MetricOmmPool.swap()` passes `msg.sender` (the direct caller) as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` value unchanged to each configured extension: [2](#0-1) 

**`SwapAllowlistExtension` keys the check on that `sender`:** [3](#0-2) 

`msg.sender` inside `beforeSwap` is the pool; `sender` is whoever called `pool.swap()`.

**`MetricOmmSimpleRouter` calls `pool.swap()` directly, making itself `msg.sender`:** [4](#0-3) 

The same pattern applies to `exactInputSingle` (line 72) and `exactOutputSingle` (line 136).

**Exploit flow:**
1. Pool admin deploys pool with `SwapAllowlistExtension`.
2. Admin calls `setAllowedToSwap(pool, router, true)` to let allowlisted users reach the pool via the router.
3. Non-allowlisted attacker calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
4. Router calls `pool.swap(...)` — pool sees `msg.sender == router`.
5. Pool calls `_beforeSwap(router, ...)` → extension evaluates `allowedSwapper[pool][router]` → `true`.
6. Swap executes. Attacker receives tokens from a pool they were never individually authorized to trade against.

**No valid configuration exists:** if the router is not allowlisted, individually allowlisted users cannot use the router at all. If the router is allowlisted, every user bypasses the allowlist.

## Impact Explanation

This is an admin-boundary break: a pool admin who configures `SwapAllowlistExtension` to restrict swaps to a curated set (e.g., KYC-verified counterparties) cannot enforce that restriction when the standard periphery router is involved. Any unprivileged address can execute swaps against the restricted pool by routing through `MetricOmmSimpleRouter`, completely nullifying the access control guard. This also constitutes broken core swap functionality — allowlisted users cannot use the router without triggering the bypass side-effect.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entry point. Any pool operator who configures `SwapAllowlistExtension` and wants their users to use the router will naturally allowlist the router, triggering the bypass. No special privileges are required — only the ability to call the public router. The condition is trivially reachable and repeatable.

## Recommendation

The extension must not rely solely on the `sender` argument (the direct pool caller) when a trusted router intermediary is involved. Concrete options:

1. **Router forwards real user identity via `extensionData`**: The router encodes `msg.sender` into `extensionData` for each hop; the extension decodes and verifies it, trusting only a known router address (verified via `msg.sender` to the extension, i.e., the pool, cross-referenced against a registry).
2. **Extension checks `tx.origin` as a fallback for router-mediated calls**: Only viable if the threat model excludes contract-based callers.
3. **Pool interface extended with a `payer` or `originator` field**: Carry the original initiator separately from the direct caller through the pool's swap interface.

## Proof of Concept

```
1. Deploy MetricOmmPool with SwapAllowlistExtension configured.
2. Pool admin calls swapExtension.setAllowedToSwap(pool, router, true).
3. Non-allowlisted attacker calls:
       MetricOmmSimpleRouter.exactInputSingle({
           pool: restrictedPool,
           tokenIn: token0,
           tokenOut: token1,
           recipient: attacker,
           amountIn: X,
           ...
       })
4. Router calls pool.swap(...) — pool sees msg.sender == router.
5. Pool calls _beforeSwap(router, ...).
6. SwapAllowlistExtension checks allowedSwapper[pool][router] → true.
7. Swap executes. Attacker receives tokens from a pool they were never
   individually authorized to trade against.
```
