### Title
`SpiceContractCodeResponse` accepted without producer-signature authentication while the non-SPICE `ContractCodeResponse` path enforces `SignedContractCodeResponse` — (`chain/client/src/spice/chunk_validator_actor.rs`)

---

### Summary

The SPICE contract-code-response path (`SpiceContractCodeResponse`) carries no signature and its handler performs no sender authentication, while the structurally identical non-SPICE path (`ContractCodeResponse`) was explicitly upgraded with `ProtocolFeature::SignedContractCodeResponse` to require a chunk-producer signature before the response is processed. The SPICE path bypasses that authentication invariant entirely: any network peer can send a `SpiceContractCodeResponse` to a chunk validator and have its payload consumed.

---

### Finding Description

**Non-SPICE path (authenticated):**

`validate_contract_code_response()` gates on `ProtocolFeature::SignedContractCodeResponse` and calls `validate_witness_contract_code_response_signature()`, which verifies that the responder is a chunk producer for the relevant shard and that the signature over the Borsh-encoded inner payload is valid. [1](#0-0) 

`ContractCodeResponse::V2` carries `signature: Signature` over a `ContractCodeResponseV2Inner` that includes a `signature_differentiator` to prevent cross-message replay. [2](#0-1) 

**SPICE path (unauthenticated):**

`SpiceContractCodeResponse` is defined with only a `V1` variant that has no signature field and no `verify_signature()` method. [3](#0-2) 

`handle_spice_contract_code_response()` decompresses the payload and immediately resolves pending contract slots across all waiting chunks — with zero sender authentication. [4](#0-3) 

The asymmetry is stark: the **request** side (`SpiceContractCodeRequest`) IS signed and the producer verifies the signature before responding. [5](#0-4) 

`SpiceChunkContractAccesses` is also signed and verified before the validator trusts the expected-hash set. [6](#0-5) 

So the full SPICE contract-distribution chain is: signed accesses → signed request → **unsigned response**. The authentication invariant is broken at the final, payload-bearing step.

---

### Impact Explanation

Any network peer that can deliver a `SpiceContractCodeResponse` message to a chunk validator can inject contract bytes into the validator's pending-chunk state without being a chunk producer. The hash check (`CodeHash(hash(&contract.0))` matched against `trusted.missing`) prevents bytes that do not hash to an expected value from being accepted, so an attacker cannot substitute wrong contract code. However:

1. **Unauthenticated source of correct data**: An attacker who possesses the on-chain contract bytes (public data) can race the legitimate producer and satisfy the validator's missing-contract set from an unauthenticated source. The validator then proceeds to validate the chunk using code supplied by an unverified peer, violating the security model that only chunk producers should supply this data.

2. **Liveness / timing attack**: By sending a response that partially satisfies `trusted.missing` with correct hashes but then withholding the rest, an attacker can manipulate the order and timing of chunk validation without being detectable as a non-producer.

3. **Future-proof risk**: If a future change relaxes the hash-equality check (e.g., for performance), the missing authentication becomes a direct code-injection vector into chunk validation.

**Severity: Medium** — the hash check limits immediate exploitability to timing/liveness manipulation, but the broken authentication invariant is a structural gap that mirrors the `SignedContractCodeResponse` gap that was already recognized and fixed for the non-SPICE path.

---

### Likelihood Explanation

The SPICE feature is gated by `protocol_feature_spice` and is not yet on mainnet, but it is actively developed and tested. Any peer on the SPICE network can send `SpiceContractCodeResponse` messages; no privileged role is required. The attacker only needs the contract bytes (available from on-chain state) and knowledge of which chunk a validator is currently processing (observable from the network).

---

### Recommendation

Add a signature field and a `signature_differentiator` to `SpiceContractCodeResponseV1` (or introduce a `V2`), mirroring `ContractCodeResponseV2Inner`. In `handle_spice_contract_code_response()`, verify the signature against the chunk producer's public key (looked up via `epoch_manager`) before processing the payload, exactly as `validate_witness_contract_code_response_signature()` does for the non-SPICE path. [7](#0-6) 

---

### Proof of Concept

1. Chunk validator V is waiting for contract hash `H` for SPICE chunk `C` (established via a verified `SpiceChunkContractAccesses` from the legitimate producer P).
2. Attacker A (any network peer) reads the contract bytes `B` for hash `H` from on-chain state.
3. A constructs `SpiceContractCodeResponse::V1 { chunk_id: C, compressed_contracts: compress([B]) }` — no signature required.
4. A sends this message to V over the network.
5. V's `handle_spice_contract_code_response()` decompresses `B`, computes `CodeHash(hash(B)) == H`, finds `H` in `trusted.missing`, removes it, and pushes `B` into `entry.contracts`.
6. V proceeds to validate chunk `C` using contract bytes supplied by unauthenticated peer A, not by the legitimate producer P.

The non-SPICE path would have rejected step 4 because `validate_contract_code_response()` would call `validate_witness_contract_code_response_signature()` and find no `responder` field on a `V1` response (or reject an invalid signature on a `V2` response from a non-producer). [8](#0-7) [9](#0-8)

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }

    /// Handles a contract code response from a chunk producer.
    /// Resolves received contracts across all pending chunks (not just the chunk_id
    /// in the response), since multiple chunks may be waiting on the same contract.
    fn handle_spice_contract_code_response(
        &mut self,
        response: SpiceContractCodeResponse,
    ) -> Result<(), Error> {
        let contracts = response.decompress_contracts()?;

        // Map code_hash -> bytes for the received contracts.
        let mut received: HashMap<CodeHash, CodeBytes> = HashMap::new();
        for contract in contracts {
            let hash = CodeHash(hash(&contract.0));
            received.insert(hash, contract);
        }

        // Resolve across all pending chunks that are waiting on any of these contracts.
        let mut maybe_ready: Vec<SpiceChunkId> = Vec::new();
        for (chunk_id, entry) in &mut self.partial_chunk_data {
            let Some(trusted) = &mut entry.trusted else {
                continue;
            };

            let mut resolved_any = false;
            for (hash, bytes) in &received {
                if trusted.missing.remove(hash) {
                    entry.contracts.push(bytes.clone());
                    resolved_any = true;
                }
            }
            if resolved_any && trusted.missing.is_empty() {
                maybe_ready.push(chunk_id.clone());
            }
        }

        let signer = self
            .validator_signer
            .get()
            .ok_or_else(|| Error::NotAValidator("no signer".to_owned()))?;
        for chunk_id in maybe_ready {
            self.try_assemble_and_validate_chunk(&chunk_id, signer.clone())?;
        }
        Ok(())
    }
