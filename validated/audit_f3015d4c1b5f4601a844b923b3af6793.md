### Title
`ValidResourceBounds::AllResources` with zero L2/L1_DATA gas silently mutates to `L1Gas` across protobuf boundary, producing a divergent transaction hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserialization of `ValidResourceBounds` uses value-based discrimination (zero-check on `l2_gas` and `l1_data_gas`) to reconstruct the Rust enum variant. This causes `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` to be silently decoded as `L1Gas(X)` after a protobuf round-trip. Because `get_tip_resource_bounds_hash` produces a structurally different hash for these two variants (3 vs. 2 resource felts in the Poseidon chain), the transaction hash computed before serialization diverges from the hash that would be computed from the deserialized object.

---

### Finding Description

**Forward conversion** (`ValidResourceBounds` → `protobuf::ResourceBounds`): [1](#0-0) 

When `ValidResourceBounds::L1Gas(l1_gas)` is serialized, `l1_data_gas` is explicitly set to `Some(ResourceBounds::default())` (all-zero). When `AllResources(X, 0, 0)` is serialized, `l1_data_gas` is also `Some(zero)`. The two variants produce **identical protobuf bytes**.

**Reverse conversion** (`protobuf::ResourceBounds` → `ValidResourceBounds`): [2](#0-1) 

The discriminator at line 431 — `if l1_data_gas.is_zero() && l2_gas.is_zero()` — cannot distinguish between the two original variants. `AllResources(X, 0, 0)` is decoded as `L1Gas(X)`.

**Hash preimage divergence:**

`get_tip_resource_bounds_hash` produces structurally different Poseidon inputs for the two variants: [3](#0-2) 

- `L1Gas(X)`: chain = `[tip, concat(L1_GAS, X), concat(L2_GAS, 0)]` — **2 resource felts**
- `AllResources(X, 0, 0)`: chain = `[tip, concat(L1_GAS, X), concat(L2_GAS, 0), concat(L1_DATA_GAS, 0)]` — **3 resource felts**

These produce different Poseidon digests, so H1 (computed before serialization with `AllResources`) ≠ H2 (computed after deserialization as `L1Gas`).

**Attacker-controlled path:**

A user submits a V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it (resource-bounds validation only checks L2 gas price threshold, not whether l2_gas is non-zero): [4](#0-3) 

The hash H1 is computed and stored in `InternalRpcTransaction.tx_hash`: [5](#0-4) 

When the block is synced via P2P, the transaction is serialized to protobuf and deserialized on the receiving node as `L1Gas(X)`. Any component that recomputes the hash from the deserialized `Transaction` object (e.g., `validate_transaction_hash`) will produce H2 ≠ H1: [6](#0-5) 

Additionally, the `GasVectorComputationMode` changes from `All` to `NoL2Gas`: [7](#0-6) 

This affects fee computation and resource-bound enforcement in the blockifier.

---

### Impact Explanation

**High — Transaction conversion binds the wrong hash and executable type.**

A transaction signed and hashed as `AllResources` (3-resource Poseidon preimage) is decoded as `L1Gas` (2-resource Poseidon preimage) after the protobuf boundary. Any node that recomputes the transaction hash from the deserialized object — including `validate_transaction_hash` called during P2P block sync — will compute a divergent value and reject the block. Additionally, the wrong `GasVectorComputationMode` (`NoL2Gas` instead of `All`) is applied to fee and resource-bound checks for the deserialized transaction.

---

### Likelihood Explanation

**Low-to-medium.** The trigger requires a V3 transaction with `AllResources` where both `l2_gas` and `l1_data_gas` are zero. This is an unusual but valid configuration (a user who only wants L1 gas bounds but submits a post-0.13.3 transaction format). The gateway does not reject such transactions. The mutation is silent — no error is returned.

---

### Recommendation

Replace value-based variant discrimination with an explicit type tag in the protobuf schema, or add a boolean field `is_all_resources` to `protobuf::ResourceBounds`. Alternatively, the reverse conversion should treat any protobuf message that was produced by the current sequencer (which always sets `l1_data_gas = Some(...)`) as `AllResources` unless a legacy-format flag is present. The TODO comment at line 426–427 acknowledges the backward-compat shim but does not address the type-mutation invariant: [8](#0-7)

    type Error = ProtobufConversionError;

            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));

            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
