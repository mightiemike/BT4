### Title
Protobuf Receipt Conversion Always Emits Wrong `price_unit` for V3 Transactions Without Mandatory Post-Hoc Fix — (`File: crates/apollo_protobuf/src/converters/receipt.rs`)

### Summary

Every `From<TxOutput>` → `protobuf::receipt::*` conversion in the P2P state-sync layer emits `price_unit = Wei` (protobuf default = 0) regardless of transaction version. The code itself documents this: each converter carries the comment *"The returned price_unit isn't correct. It can be fixed by calling `set_price_unit_based_on_transaction`"*. That separate function is the only way to stamp the correct `Fri` unit onto V3 receipts, but it is not called inside the conversion and there is no type-system enforcement that callers invoke it. Any code path that serialises a V3 receipt without the post-hoc call will broadcast an authoritative-looking receipt with `price_unit = Wei` instead of `Fri`.

### Finding Description

**Broken invariant (direct analog of the external bug)**

The external bug: `tokenDistribution` (the payment-type selector) is never initialised, so `claimRewards()` always picks ETH even for ERC-20 missions.

The sequencer analog: `price_unit` (the fee-token-type selector) is never set inside the `From` conversions, so every serialised receipt defaults to `Wei` even for STRK (V3) transactions.

**Affected converters** — `crates/apollo_protobuf/src/converters/receipt.rs`:

```
impl From<InvokeTransactionOutput>       for protobuf::receipt::Invoke        // line 208
impl From<DeclareTransactionOutput>      for protobuf::receipt::Declare       // line 180
impl From<DeployAccountTransactionOutput> for protobuf::receipt::DeployAccount // line 102
impl From<L1HandlerTransactionOutput>    for protobuf::receipt::L1Handler     // line 236
```

Each calls `create_proto_receipt_common_from_txn_output_fields`, which sets `price_unit` to the protobuf integer default `0` = `PriceUnit::Wei`.

**The post-hoc fix** — `crates/apollo_protobuf/src/converters/transaction.rs` lines 1085-1116:

```rust
pub fn set_price_unit_based_on_transaction(
    receipt: &mut protobuf::Receipt,
    transaction: &protobuf::TransactionInBlock,
) {
    let price_unit = match &transaction.txn {
        Some(Txn::DeclareV3(_))       => protobuf::PriceUnit::Fri,
        Some(Txn::DeployAccountV3(_)) => protobuf::PriceUnit::Fri,
        Some(Txn::InvokeV3(_))        => protobuf::PriceUnit::Fri,
        // all others                 => Wei
        _ => return,
    };
    ...
    common.price_unit = price_unit.into();
}
```

This function must be called **after** the `From` conversion and requires the matching `TransactionInBlock` to be available. There is no type-level guarantee that callers do so.

**Default value** — `crates/apollo_protobuf/src/protobuf/protoc_output.rs` line 800:

```rust
pub enum PriceUnit {
    Wei = 0,   // protobuf default
    Fri = 1,
}
```

Protobuf decodes a missing or zero-valued field as `Wei`, so any receipt serialised without the post-hoc call will be decoded by peers as `Wei`.

**Fee-type selection downstream** — `crates/apollo_rpc_execution/src/objects.rs` lines 531-537:

```rust
impl From<PriceUnit> for FeeType {
    fn from(value: PriceUnit) -> Self {
        match value {
            PriceUnit::Wei => FeeType::Eth,
            PriceUnit::Fri => FeeType::Strk,
        }
    }
}
```

A receipt with `price_unit = Wei` causes downstream consumers to select `FeeType::Eth` (ETH gas prices) instead of `FeeType::Strk` (STRK gas prices) when reconstructing fee estimations or serving RPC responses.

### Impact Explanation

Any node that receives a V3 transaction receipt over P2P state sync without the post-hoc fix applied will store and serve `price_unit = Wei` for that receipt. When the RPC layer reconstructs `FeeEstimation` via `tx_execution_output_to_fee_estimation` (lines 161-183 of `objects.rs`), it reads `price_unit` to select which gas-price vector to use:

```rust
gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get()
```

With `price_unit = Wei`, it reads ETH gas prices instead of STRK gas prices. The `overall_fee` amount (already computed in STRK/Fri) is then labelled `WEI` and paired with ETH gas prices, producing an authoritative-looking but wrong fee estimation response.

**Impact class:** High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.

### Likelihood Explanation

The likelihood depends on whether every call site that invokes the `From` conversions also calls `set_price_unit_based_on_transaction`. The code explicitly documents the defect in four separate places with identical comments, indicating the design relies on caller discipline rather than type-system enforcement. Any new call site, refactor, or test that uses the `From` conversions directly (which is the natural Rust idiom) will silently produce wrong `price_unit` values for V3 transactions. The protobuf default of `0 = Wei` means the error is silent — no panic, no error, just a wrong enum value.

### Recommendation

Remove the two-step pattern. Either:

1. **Merge the fix into the conversion**: require `TransactionVersion` (or the transaction variant) as a parameter to the `From` conversion so `price_unit` is set correctly at construction time, eliminating the need for `set_price_unit_based_on_transaction`.

2. **Use a builder/newtype**: wrap the protobuf receipt in a type that cannot be constructed without a correct `price_unit`, making the omission a compile error.

3. **At minimum**: add a `debug_assert` or runtime check inside `set_price_unit_based_on_transaction` that panics in test builds if the receipt's `price_unit` is still the default after the call, and add integration tests that verify V3 receipts round-trip with `Fri`.

### Proof of Concept

1. Construct a V3 invoke transaction output and convert it to protobuf using the `From` impl:

```rust
let output = InvokeTransactionOutput { actual_fee: Fee(1000), ... };
let proto_receipt: protobuf::receipt::Invoke = output.into();
// proto_receipt.common.price_unit == 0 (Wei) — WRONG for V3
```

2. Wrap it in a `protobuf::Receipt` and send it over P2P without calling `set_price_unit_based_on_transaction`.

3. The receiving node decodes `price_unit = 0 = Wei`.

4. When the RPC layer calls `tx_execution_output_to_fee_estimation`, it selects ETH gas prices:

```rust
gas_prices.l1_gas_price(&PriceUnit::Wei.into())  // returns eth_gas_prices.l1_gas_price
// instead of
gas_prices.l1_gas_price(&PriceUnit::Fri.into())  // should return strk_gas_prices.l1_gas_price
```

pub fn set_price_unit_based_on_transaction(
    receipt: &mut protobuf::Receipt,
    transaction: &protobuf::TransactionInBlock,
) {
    let price_unit = match &transaction.txn {
        Some(protobuf::transaction_in_block::Txn::DeclareV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeclareV2(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeclareV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::Deploy(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeployAccountV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeployAccountV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::InvokeV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::InvokeV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::L1Handler(_)) => protobuf::PriceUnit::Wei,
        _ => return,
    };
    let Some(ref mut receipt_type) = receipt.r#type else {
        return;
    };

    let common = match receipt_type {
        protobuf::receipt::Type::Invoke(invoke) => invoke.common.as_mut(),
        protobuf::receipt::Type::L1Handler(l1_handler) => l1_handler.common.as_mut(),
        protobuf::receipt::Type::Declare(declare) => declare.common.as_mut(),
        protobuf::receipt::Type::DeprecatedDeploy(deploy) => deploy.common.as_mut(),
        protobuf::receipt::Type::DeployAccount(deploy_account) => deploy_account.common.as_mut(),
    };

    if let Some(common) = common {
        common.price_unit = price_unit.into();
    }
}

pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

    let gas_vector = tx_execution_output.execution_info.receipt.gas;

    Ok(FeeEstimation {
        gas_consumed: gas_vector.l1_gas.0.into(),
        l1_gas_price,
        data_gas_consumed: gas_vector.l1_data_gas.0.into(),
        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}

// Executes a series of transactions and returns the execution results.
// TODO(yair): Return structs instead of tuples.
#[allow(clippy::too_many_arguments)]
// TODO(Dan, Yair): consider box large elements (because of BadDeclareTransaction) or use ID
// instead.
fn execute_transactions(
    txs: Vec<ExecutableTransactionInput>,
    tx_hashes: Option<Vec<TransactionHash>>,
    chain_id: &ChainId,
    storage_reader: StorageReader,
    maybe_pending_data: Option<PendingData>,
    state_number: StateNumber,
    block_context_block_number: BlockNumber,
    execution_config: &ExecutionConfig,
    charge_fee: bool,
    validate: bool,
    override_kzg_da_to_false: bool,
    class_manager_handle: Option<(SharedClassManagerClient, Handle)>,
) -> ExecutionResult<(Vec<TransactionExecutionOutput>, BlockContext)> {
    // The starknet state will be from right before the block in which the transactions should run.
    let mut cached_state = CachedState::new(ExecutionStateReader {
        storage_reader: storage_reader.clone(),
        state_number,
        maybe_pending_data: maybe_pending_data.clone(),
        missing_compiled_class: Cell::new(None),
        class_manager_handle,
    });

    let block_context = create_block_context(
        &mut cached_state,
        block_context_block_number,
        chain_id.clone(),
        &storage_reader,
        maybe_pending_data.as_ref(),
        execution_config,
        override_kzg_da_to_false,
    )?;

    let (txs, tx_hashes) = match tx_hashes {
        Some(tx_hashes) => (txs, tx_hashes),
        None => {
            let tx_hashes = calc_tx_hashes(txs, chain_id)?;
            trace!("Calculated tx hashes: {:?}", tx_hashes);
            tx_hashes
        }
    };

    let mut res = vec![];
    for (transaction_index, (tx, tx_hash)) in txs.into_iter().zip(tx_hashes).enumerate() {
        let transaction_version = tx.transaction_version();
        // TODO(DanB): consider supporting match instead.
        let price_unit = if transaction_version == TransactionVersion::ZERO
            || transaction_version == TransactionVersion::ONE
            || transaction_version == TransactionVersion::TWO
        {
            PriceUnit::Wei
        } else {
            PriceUnit::Fri
        };
        let mut transactional_state = CachedState::create_transactional(&mut cached_state);
        let deprecated_declared_class_hash = match &tx {
            ExecutableTransactionInput::DeclareV0(
                DeclareTransactionV0V1 { class_hash, .. },
                _,
                _,
                _,
            ) => Some(*class_hash),
            ExecutableTransactionInput::DeclareV1(
                DeclareTransactionV0V1 { class_hash, .. },
                _,
                _,
                _,
            ) => Some(*class_hash),
            _ => None,
        };
        let blockifier_tx = to_blockifier_tx(tx, tx_hash, transaction_index, charge_fee, validate)?;
        // TODO(Yoni): use the TransactionExecutor instead.
        let tx_execution_info_result =
            blockifier_tx.execute(&mut transactional_state, &block_context);
        let state_diff =
            induced_state_diff(&mut transactional_state, deprecated_declared_class_hash)?;
        transactional_state.commit();
        let execution_info = tx_execution_info_result.map_err(|error| {
            if let Some(class_hash) = cached_state.state.missing_compiled_class.get() {
                ExecutionError::MissingCompiledClass { class_hash }
            } else {
                ExecutionError::from((transaction_index, error))
            }
        })?;
        res.push(TransactionExecutionOutput {
            execution_info,
            induced_state_diff: state_diff,
            price_unit,
        });
