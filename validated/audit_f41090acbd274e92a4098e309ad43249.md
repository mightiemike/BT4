### Title
`ValidResourceBounds` Protobuf Deserialization Misclassifies `AllResources` as `L1Gas`, Causing Consensus Validators to Reject Valid Proposals — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a zero-value heuristic to distinguish between `L1Gas` and `AllResources` variants. A valid `AllResourceBounds` transaction with `l2_gas = 0` and `l1_data_gas = 0` (accepted by the gateway) is silently downgraded to `ValidResourceBounds::L1Gas` during consensus protobuf deserialization. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion then hard-rejects any non-`AllResources` variant, causing validators to fail to deserialize the consensus proposal and reject a valid block.

---

### Finding Description

**Step 1 — Gateway accepts the transaction.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`. The stateless validator explicitly allows `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }` as a valid submission. [1](#0-0) 

**Step 2 — Proposer serializes to protobuf.**

`From<RpcInvokeTransactionV3> for InvokeTransactionV3` wraps the bounds as `ValidResourceBounds::AllResources(...)`, then `From<InvokeTransactionV3> for protobuf::InvokeV3` serializes all three resource fields (including the zero-valued `l2_gas` and `l1_data_gas`) into the wire message. [2](#0-1) [3](#0-2) 

**Step 3 — Validator deserializes: zero-value heuristic fires.**

`ValidResourceBounds::try_from(protobuf::ResourceBounds)` checks `l1_data_gas.is_zero() && l2_gas.is_zero()` and, when both are zero, unconditionally produces `ValidResourceBounds::L1Gas` — discarding the `AllResources` variant the proposer intended. [4](#0-3) 

**Step 4 — Conversion to `RpcInvokeTransactionV3` hard-fails.**

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` requires `ValidResourceBounds::AllResources`; any other variant returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`. Because step 3 produced `L1Gas`, this conversion fails. [5](#0-4) 

**Step 5 — Consensus deserialization propagates the error.**

`TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` calls `txn.try_into()?` for the `InvokeV3` arm. The error from step 4 propagates, and the validator rejects the entire consensus proposal. [6](#0-5) [7](#0-6) 

The invariant broken is: **the round-trip `AllResources → protobuf → ValidResourceBounds` must preserve the variant**. It does not when `l2_gas = 0` and `l1_data_gas = 0`.

---

### Impact Explanation

A validator that receives a consensus proposal containing any invoke V3 transaction with `AllResourceBounds { l1_gas: nonzero, l2_gas: 0, l1_data_gas: 0 }` will fail to deserialize the `ConsensusTransaction` and treat the proposal as invalid. If a quorum of validators does this, the block cannot be finalized. This matches:

> **High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

---

### Likelihood Explanation

- The gateway explicitly accepts `AllResourceBounds` with zero `l2_gas` and `l1_data_gas` (test-confirmed).
- Any unprivileged user can submit such a transaction.
- The proposer will include it in a block proposal.
- Every validator will hit the deserialization failure deterministically.
- No special network position or privilege is required.

---

### Recommendation

Replace the zero-value heuristic in `ValidResourceBounds::try_from(protobuf::ResourceBounds)` with an explicit discriminator field in the protobuf schema (e.g., a `oneof` or a boolean `is_all_resources` flag), or always deserialize to `AllResources` when all three fields are present in the wire message — mirroring the fix pattern of using `forceApprove` (an explicit, unambiguous operation) instead of relying on an implicit return-value assumption. [8](#0-7) 

---

### Proof of Concept

```
1. Submit to gateway:
   RpcInvokeTransactionV3 {
       resource_bounds: AllResourceBounds {
           l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
           l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
       },
       ...
   }
   → Gateway accepts (stateless validator allows l1-only non-zero bounds).

2. Proposer serializes to protobuf::ConsensusTransaction::InvokeV3(InvokeV3WithProof {
       invoke: InvokeV3 {
           resource_bounds: ResourceBounds {
               l1_gas:      ResourceLimits { max_amount: 1000, max_price_per_unit: 1 },
               l2_gas:      ResourceLimits { max_amount: 0,    max_price_per_unit: 0 },
               l1_data_gas: ResourceLimits { max_amount: 0,    max_price_per_unit: 0 },
           }, ...
       }, ...
   })

3. Validator deserializes:
   ValidResourceBounds::try_from(resource_bounds)
   → l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → produces ValidResourceBounds::L1Gas(...)   ← WRONG

4. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3:
   match value.resource_bounds {
       ValidResourceBounds::AllResources(bounds) => bounds,
       _ => return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR),  ← FIRES
   }

5. TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction returns Err.
   Validator rejects the proposal. Consensus stalls.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ConsensusTransaction) -> Result<Self, Self::Error> {
        let txn = value.txn.ok_or(missing("ConsensusTransaction::txn"))?;
        let txn = match txn {
            protobuf::consensus_transaction::Txn::DeclareV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Declare(
                    RpcDeclareTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::DeployAccountV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::DeployAccount(
                    RpcDeployAccountTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
                    RpcInvokeTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::L1Handler(txn) => {
                ConsensusTransaction::L1Handler(txn.try_into()?)
            }
        };
        Ok(txn)
    }
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
