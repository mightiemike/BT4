Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address as `sender` instead of end user, allowing any user to bypass per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, the pool receives `msg.sender = router`, so the extension checks the router's allowlist entry rather than the end user's. Since the router must be allowlisted for any router-based swap to succeed, every unpermissioned user can bypass the per-user gate by routing through the router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks: [2](#0-1) 

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When `exactInputSingle` is used, the router calls `pool.swap()` directly: [3](#0-2) 

The pool's `msg.sender` is therefore the router address. The extension receives `sender = address(router)` and checks `allowedSwapper[pool][router]`. For multi-hop `exactInput`, intermediate hops pass `address(this)` (the router) as the payer context: [4](#0-3) 

The pool admin must add the router to `allowedSwapper[pool][router]` for any router-based swap to work. Once that entry exists, the check passes for every user routing through the router — the per-user allowlist is completely bypassed. The wrong value is the `sender` argument resolved by the extension: it is `address(router)` when it should be the originating EOA.

## Impact Explanation
A pool deployer using `SwapAllowlistExtension` intends to restrict swaps to a curated set of addresses (KYC-gated pool, private market-making pool, trusted-integrator-only pool). Because the extension resolves `sender` to the router rather than the end user, any unpermissioned address can execute swaps against the restricted pool by calling `exactInputSingle` or `exactInput` on `MetricOmmSimpleRouter`. LPs absorb unauthorized toxic flow that the allowlist was designed to exclude, resulting in direct LP principal loss. This is a broken core access-control invariant with direct fund impact, qualifying as High severity.

## Likelihood Explanation
No special privileges are required. Any EOA can call `exactInputSingle` on the canonical router. The router allowlist entry (`allowedSwapper[pool][router] = true`) is a mandatory precondition for any legitimate router-based swap on an allowlisted pool, so it will always be present in any real deployment. The bypass is automatic, requires zero additional setup, and is repeatable indefinitely.

## Recommendation
The extension must verify the originating user, not the immediate caller. The cleanest fix is to add an `originator` field to the `IMetricOmmExtensions.beforeSwap` interface: the pool passes both `msg.sender` (immediate caller) and a separately supplied `originator` address; the router sets `originator = msg.sender` before calling the pool; the extension checks `allowedSwapper[pool][originator]`. This requires coordinated changes to `IMetricOmmExtensions.beforeSwap`, `MetricOmmPool.swap`, and `MetricOmmSimpleRouter`. Alternatively, the router can encode `msg.sender` into `extensionData` and the extension can decode and verify it, provided a trusted encoding convention is enforced.

## Proof of Concept
```
Setup:
  - Pool P uses SwapAllowlistExtension E.
  - Pool admin calls E.setAllowedToSwap(P, router, true)
    // required for any router swap to work
  - Pool admin does NOT call E.setAllowedToSwap(P, attacker, true)

Attack:
  1. attacker calls router.exactInputSingle({pool: P, recipient: attacker, ...})
  2. router calls P.swap(attacker, ...)  — pool sees msg.sender = router
  3. P calls E.beforeSwap(sender=router, ...)
  4. E checks: allowedSwapper[P][router] == true  → passes
  5. Swap executes; attacker receives output tokens.

Result:
  - attacker, not in the allowlist, successfully swaps against the restricted pool.
  - LPs absorb flow the allowlist was designed to exclude.
```
