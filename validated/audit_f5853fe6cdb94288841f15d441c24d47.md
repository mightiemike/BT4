Looking at the H-24 vulnerability class — **access control bypass where a privileged operation is reachable through an unprivileged path due to using the wrong dispatch function** — I need to find an analog in `clvm_rs` where an operator or capability that should only be reachable through a gated/privileged path (the softfork guard) can be triggered directly through an ungated path.

I examined the following key files:

- `src/chia_dialect.rs` — operator dispatch, `ClvmFlags`, `MEMPOOL_MODE`
- `src/dialect.rs` — `OperatorSet`, `Dialect` trait
- `src/run_program.rs` — softfork guard logic, `apply_op`, `parse_softfork_arguments`
- `wheel/src/api.rs` — Python-facing `run_serialized_chia_program`
