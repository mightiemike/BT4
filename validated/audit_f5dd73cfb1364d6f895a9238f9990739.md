### Title
`Mempool::validate_tx` Skips Expired-Transaction Eviction Before Duplicate-Nonce Check, Causing Valid Transactions to Be Rejected at Gateway Admission — (File: `crates/apollo_mempool/src/mempool.rs`)

---

### Summary

`Mempool::validate_tx` does not call `remove_expired_txs()` before performing duplicate-nonce and fee-escalation checks, unlike `Mempool::add_tx` which does. An expired transaction still present in the pool (lazy eviction) causes `validate_tx` to return `DuplicateNonce` for a valid incoming transaction, permanently rejecting it at the gateway admission layer before `add_tx` is ever called.

---

### Finding Description

`Mempool::add_tx` calls `self.remove_expired_txs()` as its very first action, ensuring that all TTL-expired transactions are evicted before any duplicate-nonce or fee-escalation checks are performed:

```rust
pub fn add_tx(&mut self, args: AddTransactionArgs) -> MempoolResult<()> {
    // First remove old transactions from the pool.
    let mut account_nonce_updates = self.remove_expired_txs();
    ...
    self.add_tx_validations(tx_reference, &args.tx, args.account_state.nonce)?;
``` [1](#0-0) 

`Mempool::validate_tx`, however, performs no such eviction before calling `validate_incoming_tx` and `validate_fee_escalation`:

```rust
pub fn validate_tx(&mut self, args: ValidationArgs) -> MempoolResult<()> {
    let tx_reference = (&args).into();
    self.validate_incoming_tx(tx_reference, args.account_nonce)?;
    self.validate_fee_escalation(tx_reference)?;
    Ok(())
}
``` [2](#0-1) 

The eviction mechanism is explicitly lazy: `remove_txs_older_than` is only triggered by `add_tx` or `get_txs`, so expired entries remain in `tx_pool` until one of those paths fires. [3](#0-2) 

`validate_fee_escalation` then finds the expired transaction at `(address, nonce)` and either returns `DuplicateNonce` outright (when `enable_fee_escalation` is false) or rejects the incoming transaction if its fee is lower than the expired entry's fee (when escalation is enabled):

```rust
if !self.config.static_config.enable_fee_escalation {
    if self.tx_pool.get_by_address_and_nonce(address, nonce).is_some() {
        return Err(MempoolError::DuplicateNonce { address, nonce });
    };
    ...
}
if !self.should_replace_tx(&existing_tx_reference, &incoming_tx_reference) {
    return Err(MempoolError::DuplicateNonce { address, nonce });
}
``` [4](#0-3) 

The gateway's `StatefulTransactionValidator` calls `mempool_client.validate_tx()` as a mandatory pre-validation step. A failure there causes the transaction to be rejected at the gateway level; `add_tx` is never called. [5](#0-4) 

---

### Impact Explanation

A valid transaction T2 at nonce N for address A is permanently rejected by the gateway because an expired transaction T1 at the same nonce is still in the pool (lazy eviction has not yet fired). `validate_tx` sees T1, compares T2's fee against it, and returns `DuplicateNonce`. Since `add_tx` is never reached, T2 is dropped. The user must wait for an unrelated `add_tx` or `get_txs` call to trigger cleanup before resubmitting.

This matches the allowed impact: **"Mempool/gateway/RPC admission rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The default `transaction_ttl` is 60 seconds; the deployment config sets it to 300 seconds. [6](#0-5) 

Lazy eviction fires only on `add_tx` or `get_txs`. During any quiet window (no new submissions, batcher idle), expired entries accumulate. A user who submits a replacement transaction at the same nonce during this window — even with a perfectly valid fee — is rejected. The window is bounded by the TTL but can be several minutes in production.

---

### Recommendation

Add `self.remove_expired_txs()` at the start of `validate_tx`, mirroring `add_tx`:

```diff
 pub fn validate_tx(&mut self, args: ValidationArgs) -> MempoolResult<()> {
+    self.remove_expired_txs();
     let tx_reference = (&args).into();
     self.validate_incoming_tx(tx_reference, args.account_nonce)?;
     self.validate_fee_escalation(tx_reference)?;
     Ok(())
 }
``` [2](#0-1)

    fn remove_expired_txs(&mut self) -> AddressToNonce {
        let removed_txs = self
            .tx_pool
            .remove_txs_older_than(self.config.dynamic_config.transaction_ttl, &self.state.staged);

        for tx_ref in &removed_txs {
            self.decrement_stuck_txs_if_gap_account(tx_ref.address, 1);
        }

        let queued_txs = self.tx_queue.remove_txs(&removed_txs);

        self.log_and_count_expired_txs(&removed_txs);
        self.update_state_metrics();
        queued_txs
            .into_iter()
            .map(|tx| (tx.address, self.state.resolve_nonce(tx.address, tx.nonce)))
            .collect::<AddressToNonce>()
    }
