Audit Report

## Title
`SwapAllowlistExtension` allowlist check resolves to the router address, not the originating user — any caller bypasses per-user allowlist via router - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract address, not the end user. A pool admin who allowlists the router to enable router-based swaps inadvertently opens the gate to every user, because all router calls present the same `sender = router`. The per-user allowlist invariant is fully broken for all router-mediated swap paths.

## Finding Description

**Confirmed call chain:**

1. `MetricOmmSimpleRouter.exactInputSingle()` calls `IMetricOmmPoolActions(params.pool).swap(...)` directly. [1](#0-0) 

2. Inside `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument — which is the router address. [2](#0-1) 

3. `SwapAllowlistExtension.beforeSwap` receives this router address as `sender` and evaluates `allowedSwapper[msg.sender][sender]` — i.e., `allowedSwapper[pool][router]`. [3](#0-2) 

**Root cause:** The extension is documented as "Gates `swap` by swapper address, per pool" and exposes `setAllowedToSwap(pool, swapper, allowed)` for per-user control. [4](#0-3) 

However, the `sender` it receives is always the immediate caller of `pool.swap()`. When the router is allowlisted (a natural admin action to enable router-based swaps), `allowedSwapper[pool][router] == true` passes for every user who routes through the router, regardless of whether that user is individually allowlisted. There is no mechanism to recover the originating EOA from the router call.

## Impact Explanation

Any non-allowlisted user can swap in a pool intended to be restricted to specific addresses by routing through `MetricOmmSimpleRouter`. The core access-control invariant of `SwapAllowlistExtension` — that only individually allowlisted addresses may swap — is broken for all router-mediated paths. This constitutes broken core pool functionality: the allowlist guard is rendered entirely ineffective once the router is allowlisted, allowing unauthorized swaps in restricted pools.

## Likelihood Explanation

- Pool admins who want to support router-based swaps will naturally call `setAllowedToSwap(pool, router, true)`.
- Once the router is allowlisted, the bypass requires zero privilege: any user calls `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on the public router.
- The bypass is repeatable and requires no special tokens, roles, or timing.

## Recommendation

The extension must receive the originating user identity, not the immediate `pool.swap()` caller. Options:

1. **Pass `msg.sender` from the router as `extensionData`** and have `SwapAllowlistExtension` decode it — but this is forgeable by any direct caller unless combined with a signature or trusted-forwarder pattern.
2. **Preferred:** Add an `originSender` field to the swap hook arguments that the pool populates from a trusted periphery context, or require the router to pass the originating user through `extensionData` with a verifiable signature that the extension can authenticate.
3. **Alternatively:** Document clearly that `SwapAllowlistExtension` only gates direct `pool.swap()` callers and cannot enforce per-user restrictions for router-mediated swaps, and provide a separate router-aware allowlisting mechanism.

## Proof of Concept

```solidity
// Pool admin allowlists the router to enable router-based swaps
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Non-allowlisted attacker routes through the router
vm.prank(attacker); // attacker is NOT in allowedSwapper[pool]
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token0),
    tokenOut: address(token1),
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: attacker,
    deadline: block.timestamp + 1,
    priceLimitX64: 0,
    extensionData: ""
}));
// Swap succeeds — pool receives sender = address(router), which passes allowedSwapper[pool][router] == true
// attacker was never individually allowlisted but the swap executes
```

The pool's `beforeSwap` hook receives `sender = address(router)`, which satisfies `allowedSwapper[pool][router] == true`, and the swap executes despite the attacker never being individually allowlisted. [5](#0-4) [6](#0-5)
