Audit Report

## Title
SwapAllowlistExtension gates the router address instead of the originating user, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the router contract, not the originating user. `SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[pool][router]`, so allowlisting the router to enable legitimate user access simultaneously grants unrestricted swap access to every address on-chain.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` directly as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first positional argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the forwarded value: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the `msg.sender` inside the pool: [4](#0-3) 

The allowlist lookup therefore becomes `allowedSwapper[pool][router]`. Two broken outcomes follow:

1. **Bypass**: Admin allowlists the router so legitimate users can trade through it. Because the router is a shared identity, every address on-chain can call `exactInputSingle` and pass the check — the guard is disabled for all router-mediated swaps.
2. **Lockout**: Admin does not allowlist the router. Every allowlisted user who trades through the router is rejected, making the standard periphery unusable.

No existing guard in the pool, extension, or router prevents this. The router stores the originating `msg.sender` only in transient storage for payment purposes (`_setNextCallbackContext`), not in any field passed to the pool's `swap()` call. [5](#0-4) 

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to KYC'd counterparties, protocol-owned addresses, or whitelisted market makers loses that restriction entirely the moment the router is allowlisted. Any unprivileged address can route through `MetricOmmSimpleRouter` and execute swaps against LP-owned liquidity at oracle-quoted prices the pool admin never intended to offer to arbitrary counterparties. This constitutes direct loss of LP principal and a broken core invariant of the allowlist extension — a High-severity impact under Sherlock criteria. [6](#0-5) 

## Likelihood Explanation

The router is the canonical, documented periphery for end-user swaps. A pool admin deploying a curated pool with `SwapAllowlistExtension` will almost certainly allowlist the router to let approved users trade conveniently. The bypass is triggered by a routine, expected administrative action. Any attacker who observes the router is allowlisted (via on-chain event `AllowedToSwapSet`) can immediately exploit it with no special privileges, no capital requirements beyond the swap input, and can repeat it indefinitely. [7](#0-6) 

## Recommendation

Pass the originating user rather than the immediate caller as `sender` to extensions. Two complementary approaches:

1. **Router-side**: Have the router encode the originating `msg.sender` in `extensionData` and have the extension decode it from there. Requires a trust assumption on the router.
2. **Pool-side (preferred)**: Add a separate `originator` parameter to `swap()` that callers supply explicitly, and pass that to extensions instead of `msg.sender`. The pool enforces `originator == msg.sender` for direct calls; routers supply the real user. This mirrors Uniswap v4's separation of `sender` from `msgSender`.

Until fixed, pool admins must not allowlist the router on curated pools; allowlisted users must call the pool directly.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is the only allowed swapper
  - Pool admin calls setAllowedToSwap(pool, router, true)  // needed so alice can use the router

Attack:
  - bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(
        pool, tokenIn, tokenOut, amountIn, recipient=bob, ...
    )
  - Router calls pool.swap(...) — msg.sender inside pool = router address
  - Pool calls _beforeSwap(sender=router, ...)
  - ExtensionCalling forwards sender=router to SwapAllowlistExtension.beforeSwap
  - Check: allowedSwapper[pool][router] == true → passes
  - Bob's swap executes against LP liquidity at oracle price

Result:
  - bob, who was never allowlisted, successfully swaps on a curated pool.
  - The allowlist invariant is broken; LP funds are exposed to arbitrary counterparties.
```
