### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Enabling 64-Byte Merkle Proof Forgery — (File: `contract/src/lib.rs`)

### Summary
The `verify_transaction_inclusion` function was deprecated in favour of `verify_transaction_inclusion_v2`, which adds a coinbase Merkle proof check to close the 64-byte transaction forgery vulnerability. However, the old function was never access-controlled or removed — it remains a fully public, unprivileged entry point. Any NEAR caller can bypass the coinbase-proof protection entirely by calling the deprecated function directly, obtaining a `true` result for a transaction that was never included in the block.

### Finding Description
`verify_transaction_inclusion` is decorated `#[deprecated]` and `#[pause]`, but it is still declared `pub` with no role guard. [1](#0-0) 

The function's own inline warning acknowledges the broken invariant:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [2](#0-1) 

`verify_transaction_inclusion_v2` was introduced to close this gap by requiring a matching coinbase Merkle proof before delegating to the old function: [3](#0-2) 

But because `verify_transaction_inclusion` is still `pub`, the coinbase-proof gate in v2 is trivially bypassed — a caller simply invokes the v1 endpoint directly.

The Merkle verification itself (`compute_root_from_merkle_proof`) is a pure positional hash walk with no constraint that the leaf must be a real transaction: [4](#0-3) 

An attacker who knows the block's Merkle root can craft a `tx_id` equal to an internal Merkle-tree node and supply a proof path that hashes up to the stored root. The function returns `true`.

### Impact Explanation
Any recipient NEAR contract that calls `verify_transaction_inclusion` to gate a privileged action (e.g., releasing bridged funds, minting tokens, recording a cross-chain event) will receive a forged `true` for a transaction that does not exist. The corrupted value is the **proof result** returned to the caller — the canonical chain state itself is not modified, but the verification oracle is compromised. This is a proof-verification forgery reachable from an unprivileged NEAR account.

### Likelihood Explanation
The 64-byte Merkle forgery technique is publicly documented (the contract's own comment links to the BitMEX research post). The entry point requires no special role, no staking, and no prior relationship with the contract. The only prerequisite is knowledge of a real block hash stored in the mainchain and the ability to compute a valid internal-node proof — both are trivially available from on-chain state. Likelihood is **high**.

### Recommendation
Restrict `verify_transaction_inclusion` to internal use only by removing its `pub` visibility or adding a role guard (e.g., `#[private]`). Because `verify_transaction_inclusion_v2` already calls it internally via `#[allow(deprecated)]`, making it `pub(crate)` or `fn` (private) is sufficient to close the external entry point without breaking v2. Alternatively, inline the v1 logic into v2 and delete the v1 function entirely.

### Proof of Concept

1. Identify any block hash `B` present in `mainchain_header_to_height` (readable via `get_block_hash_by_height`).
2. Retrieve the stored `merkle_root` for block `B` from `get_last_block_header` or `headers_pool`.
3. Construct a `tx_id` equal to an internal Merkle-tree node whose subtree hashes to `merkle_root`, and build the corresponding `merkle_proof` path (standard 64-byte CVE technique).
4. Call `verify_transaction_inclusion` directly (not v2) with:
   - `tx_id` = crafted internal node hash
   - `tx_block_blockhash` = `B`
   - `tx_index` = position consistent with the crafted proof
   - `merkle_proof` = crafted path
   - `confirmations` = 1
5. The function returns `true` for a transaction that was never mined.
6. Any downstream contract that trusts this result executes its privileged action on a fraudulent proof. [5](#0-4)

        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }
