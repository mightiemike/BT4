### Title
Unclaimed Airdrop Tokens Are Permanently Locked With No Recovery Mechanism — (`File: core/contracts/Airdrop.sol`)

### Summary
`Airdrop.sol` holds protocol tokens that users claim via Merkle proof. There is no function allowing the owner to withdraw unclaimed or excess tokens. Any tokens deposited into the contract that are never claimed — due to sanctioned addresses, lost keys, or simply unclaimed allocations — are permanently locked and irrecoverable.

### Finding Description
`Airdrop.sol` accepts ERC20 token deposits (transferred directly to the contract address before or after deployment) and distributes them to eligible claimants via `claim()`. The contract inherits `OwnableUpgradeable` but exposes no `withdraw`, `rescueTokens`, or equivalent function.

The `_verifyProof` function explicitly blocks sanctioned addresses from claiming: [1](#0-0) 

Any allocation assigned to a sanctioned address in the Merkle tree is unclaimable by design, yet the corresponding tokens remain locked in the contract forever. The same applies to any user who loses wallet access, any rounding surplus, or any tokens deposited in excess of the Merkle-tree total.

The only state-changing functions in the contract are `registerMerkleRoot` (owner-only, adds a new week's root) and `claim` (public, transfers tokens to the caller). Neither provides a path to recover tokens. [2](#0-1) 

### Impact Explanation
Protocol tokens deposited into `Airdrop.sol` that are not claimed are permanently irrecoverable. Given that `MAX_SUPPLY`-scale airdrops routinely see 10–30 % of allocations go unclaimed, and that the sanctions check hard-blocks entire addresses, the locked amount can be material. There is no upgrade path that recovers these tokens without replacing the contract and migrating balances, which itself requires the tokens to be withdrawable.

### Likelihood Explanation
Certain. Every real-world airdrop has unclaimed allocations. The sanctions check guarantees at least one class of permanently unclaimable tokens whenever a sanctioned address appears in a Merkle tree. No attacker action is required; the loss is structural.

### Recommendation
Add an owner-controlled recovery function, callable only after a reasonable claim window has elapsed:

```solidity
function withdrawUnclaimed(address to, uint256 amount) external onlyOwner {
    // optionally enforce: block.timestamp > claimDeadline
    SafeERC20.safeTransfer(IERC20(token), to, amount);
}
```

### Proof of Concept
1. Owner calls `registerMerkleRoot(1, root)` where `root` encodes allocations including address `A`.
2. Protocol transfers `N` tokens to `Airdrop.sol`.
3. Address `A` is added to the OFAC sanctions list before claiming.
4. `A` calls `claim(...)` → reverts at `require(!ISanctionsList(sanctions).isSanctioned(sender), "address is sanctioned.")`.
5. No other address can claim `A`'s allocation.
6. Owner has no function to recover the tokens.
7. `A`'s share of `N` tokens is permanently locked in `Airdrop.sol`. [3](#0-2)

contract Airdrop is OwnableUpgradeable, IAirdrop {
    address internal token;
    address internal sanctions;
    uint32 internal pastWeeks;

    mapping(uint32 => bytes32) internal merkleRoots;
    mapping(uint32 => mapping(address => uint256)) internal claimed;

    /// @custom:oz-upgrades-unsafe-allow constructor
    constructor() {
        _disableInitializers();
    }

    function initialize(address _token, address _sanctions)
        external
        initializer
    {
        __Ownable_init();
        token = _token;
        sanctions = _sanctions;
    }

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
}
