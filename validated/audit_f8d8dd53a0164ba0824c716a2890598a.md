Audit Report

## Title
SwapAllowlistExtension Allowlist Collapsed to Router-Address Check When Router Is Allowlisted — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of the pool. When `MetricOmmSimpleRouter` is used, `sender` is the router's address, not the end user's. If a pool admin allowlists the router (the only way to let allowlisted users access the router), every public caller of the router passes the check, completely collapsing the per-user allowlist into a single router-address gate.

## Finding Description
`MetricOmmPool.swap` captures `msg.sender` and passes it as `sender` to `_beforeSwap` (L230–240). `ExtensionCalling._beforeSwap` encodes that value verbatim into the extension call (L149–177). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the immediate caller of `pool.swap` (L37).

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap(...)` directly; there is no mechanism to forward the original user's address into the `sender` field or into authenticated `extensionData`. The router's own address becomes `sender` for every swap it routes.

A pool admin who wants allowlisted users to enjoy the router's UX (slippage caps, multicall, deadline checks) must call `setAllowedToSwap(pool, router, true)`. Once that entry is set, `allowedSwapper[pool][router] == true` for every caller of the router — allowlisted or not — and the per-user gate is permanently bypassed for all router-mediated swaps.

No existing guard in the extension, the pool, or the router prevents this: `allowAllSwappers` is a separate escape hatch, and `extensionData` is user-supplied and unauthenticated.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., KYC'd or institutional traders) loses that restriction entirely for any user who routes through `MetricOmmSimpleRouter`. Non-allowlisted users can execute swaps, extract value via adverse selection, and front-run allowlisted LPs in a pool the admin believed was access-controlled. This is a direct admin-boundary break: an unprivileged path (`MetricOmmSimpleRouter`) bypasses a configured security guard, meeting the "admin-boundary break via unprivileged path" criterion.

## Likelihood Explanation
The trigger is a routine, non-malicious admin action: allowlisting the router so that allowlisted users can use it. No privileged attacker role, no malicious setup, no non-standard tokens, and no off-chain manipulation are required. Any public user can call `MetricOmmSimpleRouter` targeting the pool after the admin performs this natural configuration step. The condition is reachable in normal production operation.

## Recommendation
1. **Pass the original user through the router.** `MetricOmmSimpleRouter` should include `msg.sender` as an authenticated field in `extensionData` (signed or verified by the pool treating the router as a trusted forwarder), and `SwapAllowlistExtension` should decode and verify it when `sender` is a known trusted router.
2. **Document and enforce incompatibility.** If fix (1) is deferred, document that `SwapAllowlistExtension` is incompatible with router-mediated flows and enforce this at the factory or pool-configuration level (e.g., reject pool configurations that pair a swap allowlist extension with a public router).

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension; configure beforeSwap order to call it.
2. Admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
3. Admin calls setAllowedToSwap(pool, alice, true)    // allowlist alice directly
4. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
5. Router calls pool.swap(...) — msg.sender to pool = router address
6. _beforeSwap(sender=router, ...) → allowedSwapper[pool][router] == true → passes
7. Bob's swap executes; allowedSwapper[pool][bob] == false was never checked.

Invariant broken: allowedSwapper[pool][bob] == false, yet Bob's swap settles.
```
Foundry test: deploy pool + extension, configure allowlist with only the router and alice, call `exactInputSingle` from an unlisted address, assert the swap succeeds (demonstrating the bypass).
