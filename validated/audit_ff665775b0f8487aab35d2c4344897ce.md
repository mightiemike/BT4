### Title
Migration Chain Halt: `ForceFullExitWithDelegation` Does Not Suppress `ErrInsufficientBonusPool` During v2 Upgrade — (`x/tieredrewards/keeper/force_exit.go`)

### Summary

The v2 migration calls `ForceFullExitWithDelegation` for every vesting-account-owned position. That function calls `claimRewards`, which calls `processEventsAndClaimBonus`, which calls `sufficientBonusPoolBalance`. If the `rewards_pool` module account has insufficient balance to cover accrued bonus rewards, `ErrInsufficientBonusPool` is returned and propagates unhandled all the way up through the migration, causing the upgrade handler to abort and the chain to halt at upgrade height.

**The existing guard in `slashRedelegationPosition` demonstrates the correct pattern — it explicitly suppresses `ErrInsufficientBonusPool` to prevent chain halt — but this guard is absent in `ForceFullExitWithDelegation`:** [4](#0-3) 

The comment in `slash.go` line 55 reads: *"Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt."* The same reasoning applies to the migration path, but the guard was not applied there.

### Impact Explanation

If any vesting-account-owned position has accrued non-zero bonus rewards at upgrade height and the `rewards_pool` account balance is insufficient to cover them, the upgrade handler returns an error. In Cosmos SDK, a failing upgrade handler causes a chain halt. All validators halt at the same block height, and the chain cannot resume without a coordinated emergency patch and re-upgrade.

### Likelihood Explanation

The conditions are realistic and independently reachable:
- Vesting accounts owning tieredrewards positions are an explicitly supported use case (the migration was written specifically to handle them).
- The `rewards_pool` balance is funded externally and can be depleted by normal bonus claims before the upgrade block.
- Any position that has been bonded for a non-zero duration with a tier that has `BonusApy > 0` will have accrued non-zero bonus.
- The upgrade height is a fixed, publicly known block, giving no opportunity to top up the pool in the same block.

### Recommendation

In `ForceFullExitWithDelegation`, wrap the `claimRewards` call to suppress `ErrInsufficientBonusPool` and log it instead of returning it, matching the pattern already used in `slashRedelegationPosition`:

```go
posState, baseRewards, bonusRewards, err := k.claimRewards(ctx, posState)
if err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        logger.Error("force-exit: insufficient bonus pool, skipping bonus",
            "position_id", posID, "error", err)
        // re-fetch posState without bonus so the exit can continue
    } else {
        return fmt.Errorf("claim rewards for position %d: %w", posID, err)
    }
}
```

Alternatively, add a dedicated `forceClaimRewards` helper used only by the migration that skips bonus payment when the pool is empty.

### Proof of Concept

1. Set up a chain with a tieredrewards tier with `BonusApy > 0`.
2. Create a vesting account and lock tokens into a tier position via `LockTier` or `CommitDelegationToTier`.
3. Advance block time by at least 1 second so `computeSegmentBonus` returns a non-zero value.
4. Ensure the `rewards_pool` module account balance is zero (never funded, or fully drained).
5. Call `v2.Migrate(ctx, positions, ak, keeper)`.
6. Observe: `Migrate` returns a non-nil error wrapping `ErrInsufficientBonusPool`; the upgrade handler aborts; the chain halts.

Expected (correct) behavior: `Migrate` completes without error; bonus is skipped (logged) for the affected position; the delegation is returned to the vesting account owner. [5](#0-4) [6](#0-5) [7](#0-6)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}
