### Title
Protobuf `ValidResourceBounds` Round-Trip Collapses `AllResources` to `L1Gas` for Zero-Bound Transactions, Causing Transaction Hash Divergence Between Proposer and Syncing Nodes - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter silently downcasts any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero into `ValidResourceBounds::L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound elements for `L1Gas` (2) versus `AllResources` (3), the transaction hash computed on the proposer node (always `AllResources`) diverges from the hash that would be computed on any syncing node that deserializes the same transaction from protobuf (`L1Gas`). This breaks the canonicalization invariant that a transaction's hash must be identical across all nodes.

---

### Finding Description

**Root cause — protobuf deserialization collapses the variant:** [1](#0-0) 

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Any `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` is silently converted to `L1Gas` on deserialization. The inverse serializer emits all three fields for both variants, so the protobuf wire bytes are identical — the information loss is one-way and silent. [2](#0-1) 

**Root cause — hash function branches on the variant:** [3](#0-2) 

`get_tip_resource_bounds_hash` builds a different-length element list depending on the variant:

- `L1Gas` → `[tip, L1_GAS_packed, L2_GAS_packed]` (3 elements)
- `AllResources` → `[tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]` (4 elements)

Even when `l1_data_gas = 0`, the packed encoding `L1_DATA_GAS_packed_zero` is a non-zero felt (it encodes the resource-name tag `L1_DATA` in the upper bits), so the two Poseidon hashes are distinct.

**Proposer always uses `AllResources`:**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its trait impl unconditionally wraps it: [4](#0-3) 

The hash is computed from this internal type: [5](#0-4) 

So the proposer always hashes with `AllResources`, including `L1_DATA_GAS_packed_zero` in the preimage.

**Syncing node uses the collapsed `L1Gas` variant:**

P2P block sync deserializes committed `InvokeV3` transactions through: [6](#0-5) 

which calls `ValidResourceBounds::try_from(...)`, producing `L1Gas` for zero-bound transactions. The resulting `InvokeTransactionV3` stored on the syncing node has `resource_bounds = L1Gas`, so any call to `calculate_transaction_hash` on it omits `L1_DATA_GAS_packed_zero` from the preimage and produces a different hash.

**`validate_transaction_hash` would reject the block:** [7](#0-6) 

For `InvokeTransaction::V3` there are no deprecated hashes, so the only candidate is the freshly computed hash. If this function is invoked on the syncing node with the deserialized `L1Gas` transaction and the proposer-computed hash, it returns `false`, causing the block to be rejected.

---

### Impact Explanation

**High — Transaction hash logic binds the wrong hash for the executable payload.**

A valid, unprivileged transaction with `l2_gas = 0` and `l1_data_gas = 0` (accepted by the gateway — see `valid_l1_gas` test case) produces a proposer-side hash that includes `L1_DATA_GAS_packed_zero` in the Poseidon preimage. After protobuf round-trip on any syncing node, the same transaction is stored as `L1Gas`, and recomputing its hash omits that element, yielding a different value. Any node that validates the hash by recomputing it from the stored transaction will reject a legitimately committed block.

---

### Likelihood Explanation

The trigger requires only a standard V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` — a configuration explicitly accepted by the gateway's `validate_resource_bounds` check (non-zero `l1_gas` is sufficient). No special privileges or malformed bytes are needed. The condition `l1_data_gas.is_zero() && l2_gas.is_zero()` is hit on every such transaction during P2P sync.

---

### Recommendation

Remove the `L1Gas`/`AllResources` branching in the protobuf deserializer for V3 transactions. Since all V3 transactions submitted after Starknet 0.13.3 use `AllResources`, the converter should unconditionally produce `AllResources`:

```rust
// In TryFrom<protobuf::ResourceBounds> for ValidResourceBounds
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

The `L1Gas` branch exists only for backward compatibility with pre-0.13.3 transactions, which are not V3 and therefore never reach this code path via `InvokeV3`. Alternatively, gate the downcast on an explicit version check rather than inferring it from zero values, so that the hash-domain variant is preserved across the serialization boundary.

---

### Proof of Concept

```
1. User submits InvokeV3 with AllResourceBounds { l1_gas: 1000, l2_gas: 0, l1_data_gas: 0 }.

2. Gateway (proposer):
   InternalRpcInvokeTransactionV3::resource_bounds() → AllResources
   get_tip_resource_bounds_hash → poseidon([tip, L1_GAS_packed, L2_GAS_packed_zero, L1_DATA_GAS_packed_zero])
   → hash H_proposer

3. Transaction committed to block with hash H_proposer.

4. Syncing node receives block via P2P protobuf:
   TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
     l1_data_gas.is_zero() && l2_gas.is_zero() → L1Gas(l1_gas)
   Stored InvokeTransactionV3.resource_bounds = L1Gas

5. Syncing node recomputes hash:
   get_tip_resource_bounds_hash → poseidon([tip, L1_GAS_packed, L2_GAS_packed_zero])
   → hash H_sync  ≠  H_proposer

6. validate_transaction_hash(tx, block_number, chain_id, H_proposer, options)
   → possible_hashes = [H_sync]
   → H_proposer ∉ [H_sync]  → returns false → block rejected.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
