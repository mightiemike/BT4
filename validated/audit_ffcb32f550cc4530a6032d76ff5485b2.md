Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address as Swapper Instead of Original User, Enabling Allowlist Bypass or Breaking Allowlisted Users' Router Access — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` (the immediate caller) as `sender` to `_beforeSwap`, which forwards it verbatim to `SwapAllowlistExtension.beforeSwap`. When a user routes through `MetricOmmSimpleRouter`, the router's address reaches the extension instead of the original user's address. The extension then checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`, producing a binary broken state: either the router is not allowlisted (blocking all allowlisted users from using the standard periphery path) or the router is allowlisted (allowing any public user to bypass the guard by routing through it).

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` gates access by checking `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the forwarded value: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly — `msg.sender` inside `pool.swap()` is the router contract, so `sender` delivered to the extension is the router's address, not the original user's: [4](#0-3) 

The router does not encode the original `msg.sender` into `extensionData`; it passes `params.extensionData` opaquely from the caller. The extension does not parse `extensionData` at all. There is no mechanism to substitute the original caller's address. The allowlist is configured per-user via `setAllowedToSwap`, which maps individual user addresses — not router addresses — to pools: [5](#0-4) 

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., KYC-verified counterparties, whitelisted market makers) cannot simultaneously allow those users to use `MetricOmmSimpleRouter` and exclude everyone else. This is broken core pool functionality: either the standard periphery swap path is completely unusable for all allowlisted users, or the allowlist is fully bypassed by any public user routing through the router — a direct policy-bypass with fund-impacting consequences on pools where the allowlist is a financial control.

## Likelihood Explanation

Any pool deploying `SwapAllowlistExtension` and expecting users to interact via `MetricOmmSimpleRouter` (the documented standard periphery) will immediately encounter this. No special privileges are required: any public user can call the router. The pool admin's only recourse is to require all users to call the pool directly, bypassing the supported periphery entirely.

## Recommendation

`SwapAllowlistExtension.beforeSwap` must gate on the original end-user address rather than the intermediate router address. Two approaches:

1. **Pass the original caller through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. Requires coordinated changes in both the router and the extension.
2. **Transient storage**: The pool exposes the original initiator via transient storage (already used for reentrancy guards), and the extension reads it directly.

## Proof of Concept

```
1. Deploy a pool with SwapAllowlistExtension as a before-swap hook.
2. Pool admin calls setAllowedToSwap(pool, alice, true).
   Alice is the only allowlisted swapper.
3. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...).
   - Router calls pool.swap(recipient, ...) — msg.sender inside pool.swap() = router.
   - pool.swap() passes msg.sender = router to _beforeSwap.
   - SwapAllowlistExtension checks allowedSwapper[pool][router].
   - If router is allowlisted: Bob's swap succeeds — allowlist bypassed.
   - If router is not allowlisted: Bob's swap reverts correctly.
4. Alice (allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...).
   - Same path: extension checks allowedSwapper[pool][router], not allowedSwapper[pool][alice].
   - Alice's swap reverts even though she is explicitly allowlisted.
   => Either the allowlist is bypassable (router allowlisted) or the standard swap path
      is broken for legitimate users (router not allowlisted).
```
